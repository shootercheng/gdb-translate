28.3 注释为 GDB 输入 ¶

当 GDB 提示输入时，它会对这一事实进行注释，这样就可以知道何时发送输出、给定命令的输出何时结束等情况。

不同类型的输入各有不同的 输入类型。 每种输入类型有三种注释：一种 pre- 注释，它表示正在输出的任何提示的开始；一种普通注释，它表示提示的结束；然后是一种 post- 注释，它表示可能（或可能不）与输入相关的任何回显的结束。 例如， prompt 输入类型具有以下注释： 

^Z^Z预提示
^Z^Z提示
^Z^Z后提示

输入类型有

prompt ¶

当 GDB 提示输入命令时（主 GDB 提示）。

commands ¶

当 GDB 提示输入一组命令时，例如在 commands 命令中。 对于输入的每个命令，注释都会重复。

overload-choice ¶

当 GDB 希望用户在各种重载函数之间进行选择时。

query ¶

当 GDB 希望用户确认一个可能有危险的操作时。

prompt-for-continue ¶

当 GDB 要求用户按回车键继续时。 注意：不要期望这个能很好地工作；相反，使用 set height 0 来禁用提示。 这是因为在有注释的情况下，行计数存在问题。
