23.2.1 默认参数 ¶

你可以告诉 GDB 在使用用户定义的别名时，始终在用户明确提供的参数列表前添加一些默认参数。

如果你反复为某个命令使用相同的参数或选项，你可以为该命令定义一个别名，并告诉 GDB 在使用该别名时，自动将这些参数或选项添加到你明确输入的参数列表前。17.

例如，如果你经常使用命令 thread apply all 指定按升序处理线程，并在遇到错误时继续执行，你可以告诉 GDB 通过使用以下方式自动添加 -ascending 和 -c 选项： 

(gdb) alias thread apply asc - all = thread apply all - ascending - c

一旦你为这个别名定义了默认参数，每次你输入 thread apply asc-all 后面跟着 some arguments, GDB 将执行 thread apply all -ascending -c some arguments.

为了减少输入，你还可以定义一个单字别名： 

(gdb) alias t_a_c = thread apply all - ascending - c

像往常一样，明确的缩写可用于 别名 和 默认参数.

一个命令的不同别名不共享它们的默认参数。例如，你定义一个新别名 bt_ALL 显示所有可能的信息，另一个别名 bt_SMALL 使用以下方式显示非常有限的信息： 

(gdb) alias bt_ALL = backtrace - entry - values both - frame - arg all \
   - past - main - past - entry - full
(gdb) alias bt_SMALL = backtrace - entry - values no - frame - arg none \
   - past - main off - past - entry off

(关于使用 alias 命令的更多信息，请参阅 命令别名.)

默认参数不限于 命令的参数和选项，但如果 命令接受嵌套命令 作为参数，则可以指定嵌套命令。例如，下面定义了 faalocalsoftype 它列出了具有某种类型局部变量的帧，以及匹配的局部变量： 

(gdb) alias faalocalsoftype = frame apply all info locals - q - t
(gdb) faalocalsoftype int
#1  0x55554f5e in sleeper_or_burner (v = 0xdf50) at sleepers.c:86
i = 0
ret = 21845

定义一组嵌套 with 命令的别名以获得特定的临时设置组合也非常有用。例如，下面定义了别名 pp10 它以漂亮的格式打印一个表达式参数，如果表达式是字符串或数组，则最多显示 10 个元素： 

(gdb) alias pp10 = with print pretty -- with print elements 10 -- print

这将别名 pp10 定义为 3 个命令的序列。第一部分 with print pretty -- 临时激活设置 set print pretty, 然后启动分隔符后面的命令。 --第一部分后面的命令也是一个 with 命令，它临时将设置 set print elements 更改为 10，然后启动第二个分隔符后面的命令。 --. print 第三部分 pp10 是该 with 别名将启动的命令，使用设置的临时值和用户明确给出的参数。关于 命令使用的更多信息，请参阅.

默认情况下，请求别名的帮助会显示被别名化命令的文档。当别名是一组嵌套命令时， help 别名的帮助会显示第一个命令的文档。对于像 pp10这样的别名，这种帮助不是特别有用。对于这样的别名，使用 document 命令提供特定的文档很有用（请参阅 文档).

脚注

(17)

GDB 可以很容易地接受预定义命令和别名的默认参数，但认为这样会造成混淆，因此不允许这样做。
