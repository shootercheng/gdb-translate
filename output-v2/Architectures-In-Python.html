23.3.2.37 架构的 Python 表示形式 ¶

GDB 在其各种计算中使用了特定于架构的参数和工件。架构由 gdb.Architecture 类的一个实例表示。

一个 gdb.Architecture 类具有以下方法：

函数： 架构.名称 () ¶

返回架构的名称（字符串值）。

函数： 架构.反汇编 (起始程序计数器 [, 结束程序计数器 [, 数量]]) ¶

返回从内存地址 起始程序计数器开始的反汇编指令列表。可选参数 结束程序计数器 和 数量 确定返回列表中指令的数量。如果两个可选参数 结束程序计数器 和 数量 都被指定，那么返回一个最多包含 数量 条反汇编指令的列表，这些指令的起始地址位于从 起始程序计数器 到 结束程序计数器 的闭内存地址区间内。如果 结束程序计数器 未被指定，但 数量 被指定，那么返回从地址 数量 条从地址 起始程序计数器 开始的指令。如果 数量 未被指定，但 结束程序计数器 被指定，那么返回所有起始地址位于从 起始程序计数器 到 结束程序计数器 的闭内存地址区间内的指令。如果 结束程序计数器 和 数量 都未被指定，那么返回位于 起始程序计数器 的一条指令。对于所有这些情况，返回列表中的每个元素都是一个 Python dict ，具有以下字符串键：

addr

对应此键的值是一个 Python 长整数，捕获指令的内存地址。

asm

对应此键的值是一个字符串值，用汇编语言助记符表示指令。使用的汇编语言风格与当前 CLI 变量指定的相同。 disassembly-flavor。请参阅 源代码和机器代码。

length

对应此键的值是指令的长度（整数值），以字节为单位。

函数： 架构.整数类型 (大小 [, 有符号]) ¶

此函数根据整数类型的 大小，以及可选的是否有符号来查找整数类型。

大小 是所需整数类型的位数。目前仅支持某些大小：0、8、16、24、32、64 和 128。

如果 有符号 未被指定，它默认为 True。如果 有符号 是 False, 返回的类型将是无符号的。

如果找不到指定的类型，此函数将抛出一个 ValueError 异常。

函数： 架构.void_type () ¶

此函数返回一个 void 类型。

函数： 架构.寄存器 ([ 寄存器组 ]) ¶

返回一个 gdb.RegisterDescriptorIterator （请参阅 Python 中的寄存器），包含 寄存器组中所有的寄存器，这是一个表示寄存器组名称的字符串。如果 寄存器组 被省略，或者为空字符串，则假定为寄存器组 ‘所有’。

函数： 架构.寄存器组 () ¶

返回一个 gdb.RegisterGroupsIterator （请参阅 Python 中的寄存器），包含可用于该 gdb.Architecture的所有寄存器组。
