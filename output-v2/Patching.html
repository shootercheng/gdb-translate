17.6 对程序打补丁 ¶

默认情况下， GDB 以只读方式打开包含您程序的可执行代码（或核心转储文件）的文件。这可以防止意外修改机器代码；但这也阻止了您有意对程序的二进制文件进行打补丁。

如果您希望能够对二进制文件打补丁，可以使用 set write 命令显式指定。例如，您可能希望启用内部调试标志，甚至进行紧急修复。

set write on ¶
set write off

如果您指定 ‘set write on’， GDB 以读写方式打开可执行文件和核心文件；如果您指定 set write off （默认设置）， GDB 以只读方式打开它们。

如果您已经加载了一个文件，在更改 exec-file 或 core-file 命令后，您必须再次加载它（使用 set write），以使您的新设置生效。

show write ¶

显示可执行文件和核心文件是否以读写方式打开。
