5.5.2 非停止模式 ¶

对于某些多线程目标， GDB 支持一种可选的操作模式，在这种模式下，您可以在调试器中检查已停止的程序线程，而其他线程可以继续自由执行。这在调试实时系统（例如某些线程有实时约束或必须继续响应外部事件的程序）时将干扰降至最低。这被称为 非停止 模式。

在非停止模式下，当一个线程停止以报告调试事件时， 仅 该线程停止； GDB 与全停止模式的行为不同，不会同时停止其他线程。此外，诸如 continue 和 step 之类的执行命令在非停止模式下默认仅应用于当前线程，而不是像全停止模式那样应用于所有线程。这使您能够以全停止模式无法实现的方式显式控制线程 —— 例如，让一个线程单步执行，同时允许其他线程自由运行；让一个线程单步执行，同时让其他所有线程停止；或者让多个线程独立且同时单步执行。

要进入非停止模式，请在运行或附加到您的程序之前使用以下命令序列： 

# 如果使用命令行界面，分页会破坏非停止模式。
关闭分页功能

# 最后，开启它！
开启非停止模式

您可以使用以下命令来操作非停止模式设置：

set non-stop on ¶

启用非停止模式选择。

set non-stop off

禁用非停止模式选择。

show non-stop

显示当前非停止模式的启用设置。

请注意，这些命令仅反映非停止模式是否已启用，而不反映当前正在执行的程序是否以非停止模式运行。特别是， set non-stop 首选项仅在 GDB 启动或连接到目标程序时才会被参考，并且通常在调试开始后无法切换模式。此外，由于并非所有目标都支持非停止模式，即使您已启用非停止模式， GDB 默认情况下仍可能回退到全停止操作。

在非停止模式下，所有执行命令默认仅应用于当前线程。也就是说， continue 仅继续执行一个线程。要继续执行所有线程，请发出 continue -a 或 c -a命令。

您可以使用 GDB的后台执行命令（请参阅 后台执行），在您继续从 GDB检查或单步执行其他线程时，让一些线程在后台运行。 机器接口（MI）执行命令（请参阅 GDB/MI 程序执行）在非停止模式下始终异步执行。

在后台运行时，使用 interrupt 命令暂停执行，或者在前台执行期间使用 Ctrl - c 。在全停止模式下，这会停止整个进程；但在非停止模式下，中断仅应用于当前线程。要停止整个程序，请使用 interrupt -a命令。

其他执行命令目前不支持 -a 选项。

在非停止模式下，当一个线程停止时， GDB 不会像在全停止模式下那样自动将该线程设为当前线程。这是因为线程停止通知相对于 GDB的命令解释器是异步的，如果 GDB 在您输入对先前的当前线程进行操作的命令时意外切换到另一个线程，会造成混淆。
