23.3.2.14 Python 中的 Xmethods ¶

Xmethods 是 C 语言类现有方法的额外方法或替代方法。++ 当 C 语言源代码中定义的方法可能被编译器内联或优化掉，导致其无法使用时，此功能很有用。++ 对于这种情况，可以定义一个 xmethod 来替代 C 语言源代码中定义的方法。 GDB在这种情况下，人们可以定义一个 xmethod 来替代 C 语言源代码中定义的方法。++ GDB 然后将调用 xmethod，而不是 C 语言方法，来计算表达式。在使用核心文件进行调试时，也可以使用 xmethods。此外，在调试实时程序时，调用 xmethod 无需运行被调试程序（这可能会潜在地扰乱其状态）。因此，即使 C 语言方法可用，如果定义了替代的 xmethod，最好还是使用它。++ ++

Python 中的 xmethods 功能可通过以下概念实现： xmethod 匹配器 和一个 xmethod 工作器。要实现一个 xmethod，必须为其实现一个匹配器和相应的工作器（可以实现多个工作器，每个工作器对应方法的不同重载实例）。在内部， GDB 调用 match 匹配器的方法来匹配类类型和方法名。匹配成功后， match 方法返回一个匹配的 工作器 对象列表。每个工作器对象通常对应 xmethod 的一个重载实例。它们实现了一个 get_arg_types 方法，该方法返回一个与 xmethod 所需参数对应的类型序列。 GDB 使用这个类型序列进行重载解析，并选择一个获胜的 xmethod 工作器。还会从 C 语言源代码中找到的方法中选择一个获胜者。接下来，比较获胜的 xmethod 工作器和获胜的 C 语言方法，以选择一个总体获胜者。如果 xmethod 工作器和 C 语言方法打平，则选择 xmethod 工作器作为获胜者。也就是说，如果发现获胜的 xmethod 工作器与获胜的 C 语言方法等效，则将 xmethod 工作器视为 C 语言方法的替代方法。 GDB ++ ++ ++ ++ ++ GDB 使用总体获胜者来调用方法。如果获胜的 xmethod 工作器是总体获胜者，则通过工作器对象的 __call__ 方法调用相应的 xmethod。

如果想实现一个 xmethod 来替代现有的 C 语言方法，则必须实现一个等效的 xmethod，它的名称与 C 语言方法完全相同，并且接受的参数类型也完全相同。如果用户即使有替代的 xmethod 可用，仍想调用 C 语言方法，则可以禁用 xmethod。++ ++ ++
