27.2.2 GDB/MI 输出语法 ¶

来自 GDB/MI 的输出由零个或多个带外记录组成，之后可选择跟一个单个的结果记录。这个结果记录是针对最近的命令的。输出记录序列以‘(gdb)’结束。

如果输入命令以一个 token 开头，那么该命令对应的输出也将以相同的 令牌。

output →

( out-of-band-record )* [ result-record ] "(gdb)" nl

result-record →

[ token ] "^" result-class ( "," result )* nl

out-of-band-record →

async-record | stream-record

async-record →

exec-async-output | status-async-output | notify-async-output

exec-async-output →

[ token ] "*" async-output nl

status-async-output →

[ token ] "+" async-output nl

notify-async-output →

[ token ] "=" async-output nl

async-output →

async-class ( "," result )*

result-class →

"done" | "running" | "connected" | "error" | "exit"

async-class →

"stopped" | others （其中 其他 将根据需要添加 —— 这仍在开发中）。

result →

variable "=" value

variable →

string

value →

const | tuple | list

const →

c-string

tuple →

"{}" | "{" result ( "," result )* "}"

list →

"[]" | "[" value ( "," value )* "]" | "[" result ( "," result )* "]"

stream-record →

console-stream-output | target-stream-output | log-stream-output

console-stream-output →

"~" c-string nl

target-stream-output →

"@" c-string nl

log-stream-output →

"&" c-string nl

nl →

CR | CR-LF

token →

任意数字序列。

注意：

  • 所有输出序列都以包含一个句点的单行结束。
  • 该 token 来自相应的请求。请注意，对于所有异步输出，虽然语法允许使用令牌，并且未来版本的 GDB 对于某些异步输出消息，通常会省略令牌。前端应将所有异步输出视为报告目标状态的一般变化，并且无需将异步输出与任何先前的命令关联。
  • 状态异步输出 包含关于缓慢操作进度的正在进行的状态信息。它可以被丢弃。所有状态输出都以‘+’开头。
  • 执行异步输出 包含目标上的异步状态变化（停止、启动、消失）。所有异步输出都以‘*’开头。
  • 通知异步输出 包含客户端应处理的补充信息（例如，新的断点信息）。所有通知输出都以‘=’开头。
  • 控制台流输出 是应按原样显示在控制台中的输出。它是对CLI命令的文本响应。所有控制台输出都以‘~’开头。
  • 目标流输出 是目标程序产生的输出。所有目标输出都以‘@’开头。
  • 日志流输出 是来自 GDB内部的输出文本，例如应作为错误日志一部分显示的消息。所有日志输出都以‘&’开头。
  • 新的 GDB/MI 命令应该只输出 列表 包含 值。

参见 GDB/MI 流记录, 了解有关各种输出记录的更多详细信息。
