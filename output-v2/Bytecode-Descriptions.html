F.2 字节码描述

add (0x02): b a 加 b

弹出栈顶的两个元素， b

在这个例子中， add 是字节码的名称，并且 (0x02) b ⇒ a 加 b”显示了字节码执行前后的栈状态。在此之前，栈中必须至少包含两个值， a 和 b；由于栈顶在右侧， b 在栈顶，并且 a a 和 b 从栈中弹出，并将它们替换为一个单一的值， a 加 b. 栈中可能在所示元素下方还有其他值，但字节码仅影响所示元素。

这里是另一个例子：

const8 : ⇒ n

将 8 位整数常量 n 压入栈中，不进行符号扩展。

在这个例子中，字节码 const8 从字节码流中直接获取一个操作数 n ；操作数紧跟在 const8 字节码本身之后。我们在字节码名称之后、冒号之前立即写出任何这样的操作数，并在字节码描述的主体中描述操作数在字节码流中的精确编码。

对于 const8 字节码，在 ⇒ 之前没有给出栈元素；这仅仅意味着该字节码不从栈中消耗任何值。如果一个字节码不消耗任何值，或者不产生任何值，⇒ 两侧的列表可能为空。

如果一个值被写成 a, b, 或 n, 那么字节码将其视为一个整数。如果一个值被写成 地址, 那么字节码将其视为一个地址。

我们在这里没有完全描述浮点运算；虽然这个设计可以以一种简洁的方式扩展以处理浮点值，但客户目前对它们不感兴趣，所以我们避免描述它们以节省时间。

float (0x01): ⇒

浮点字节码的前缀。尚未实现。

add (0x02): a b ⇒ a 加 b

从栈中弹出两个整数，并将它们的和作为整数压入栈中。

sub (0x03): a b ⇒ a 减 b

从栈中弹出两个整数，从次顶值中减去顶值，并将差压入栈中。

mul (0x04): a b ⇒ a 乘 b

从栈中弹出两个整数，将它们相乘，并将乘积压入栈中。请注意，当两个 n位的数相乘得到另一个 n位的数时，这些数是否有符号无关紧要；结果是相同的。

div_signed (0x05): a b ⇒ a 除以 b

从栈中弹出两个有符号整数；将次顶值除以顶值，并将商压入栈中。如果除数为零，则以错误终止。

div_unsigned (0x06): a b ⇒ a 除以 b

从栈中弹出两个无符号整数；将次顶值除以顶值，并将商压入栈中。如果除数为零，则以错误终止。

rem_signed (0x07): a b ⇒ a 模 b

从栈中弹出两个有符号整数；将次顶值除以顶值，并将余数压入栈中。如果除数为零，则以错误终止。

rem_unsigned (0x08): a b ⇒ a 模 b

从栈中弹出两个无符号整数；将次顶值除以顶值，并将余数压入栈中。如果除数为零，则以错误终止。

lsh (0x09): a b ⇒ a 左移 b 位

从栈中弹出两个整数；设 a 为次顶值，并且 b 为顶值。将 a 左移 b 位，并将结果压入栈中。

rsh_signed (0x0a): a b ⇒ (signed)a 右移 b 位（算术右移）

从栈中弹出两个整数；设 a 为次顶值，并且 b 为顶值。将 a 右移 b 位，在高端插入最高位的副本，并将结果压入栈中。

rsh_unsigned (0x0b): a b ⇒ a 右移 b 位（逻辑右移）

从栈中弹出两个整数；设 a 为次顶值，并且 b 为顶值。将 a 右移 b 位，在高端插入零位，并将结果压入栈中。

log_not (0x0e): a ⇒ 非 a

从栈中弹出一个整数；如果它为零，则压入值 1；否则，压入值 0。

bit_and (0x0f): a b ⇒ a 按位与 b

从栈中弹出两个整数，并压入它们的按位 and与。

bit_or (0x10): a b ⇒ a 按位或 b

从栈中弹出两个整数，并压入它们的按位 or或。

bit_xor (0x11): a b ⇒ a 按位异或 b

从栈中弹出两个整数，并压入它们的按位or异或。

bit_not (0x12): a ⇒ 非 a（按位取反）

从栈中弹出一个整数，并压入它的按位取反值。

equal (0x13): a b ⇒ a 等于 b

从栈中弹出两个整数；如果它们相等，则压入值 1；否则，压入值 0。

less_signed (0x14): a b ⇒ a 小于 b

从栈中弹出两个有符号整数；如果次顶值小于顶值，则压入值 1；否则，压入值 0。

less_unsigned (0x15): a b ⇒ a 小于 b

从栈中弹出两个无符号整数；如果次顶值小于顶值，则压入值 1；否则，压入值 0。

ext (0x16) n: a ⇒ a, 从 n 位进行符号扩展

从栈中弹出一个无符号值；将其视为一个 n位的二进制补码值，将其扩展到完整长度。这意味着位 n - 1 （其中最低有效位为位 0）左边的所有位都设置为位 n - 1的值。请注意， n 可能大于或等于字节码引擎栈元素的宽度；在这种情况下，该字节码应该没有效果。

要保留的源位数量， n, 被编码为紧跟在 ext 字节码之后的一个单字节无符号整数。

zero_ext (0x2a) n: a ⇒ a, 从 n 位进行零扩展

从栈中弹出一个无符号值；将除了最低 n 位之外的所有位清零。

要保留的源位数量， n, 被编码为紧跟在 zero_ext 字节码之后的一个单字节无符号整数。

ref8 (0x17): 地址 ⇒ a
ref16 (0x18): 地址 ⇒ a
ref32 (0x19): 地址 ⇒ a
ref64 (0x1a): 地址 ⇒ a

从栈中弹出一个地址 地址 。对于字节码 refn, 从 n位值从 地址, 使用自然目标字节序。将获取的值作为无符号整数压入栈中。

请注意， 地址 可能没有以任何特定方式对齐； refn 字节码应该对任何地址都能正确操作。

如果尝试访问 地址 处的内存会导致某种处理器异常，则以错误终止。

ref_float (0x1b): 地址 ⇒ d
ref_double (0x1c): 地址 ⇒ d
ref_long_double (0x1d): 地址 ⇒ d
l_to_d (0x1e): a ⇒ d
d_to_l (0x1f): d ⇒ a

尚未实现。

dup (0x28): a ⇒ a a

将栈顶元素的另一个副本压入栈中。

swap (0x2b): a b ⇒ b a

交换栈顶的两个元素。

pop (0x29): a ⇒

丢弃栈顶的值。

pick (0x32) n: a … b ⇒ a … b a

从栈中复制一个元素并将其压入栈顶。 n, 一个单字节，表示要复制的栈元素。如果 n 为零，这与 dup相同；如果 n 为 1，它复制栈顶元素下面的元素，依此类推。如果 n 超过栈中元素的数量，则以错误终止。

rot (0x33): a b c ⇒ c a b

旋转栈顶的三个元素。栈顶元素 (c) 变为第三个元素，次顶元素 (b) 变为栈顶元素，从栈顶数第三个元素 (a) 变为次顶元素。

if_goto (0x20) 偏移量: a ⇒

从栈中弹出一个整数；如果它不为零，则跳转到字节码字符串中的给定偏移量处。否则，继续执行字节码流中的下一条指令。换句话说，如果 a 不为零，则将 pc 寄存器设置为 start + 偏移量. 因此，偏移量为零表示表达式的开始。

该 偏移量 存储为一个 16 位无符号值，紧跟在 if_goto 字节码之后。无论目标的正常字节序如何，它总是先存储最高有效字节。偏移量不能保证在字节码流中处于任何特定的对齐位置；因此，在从未对齐地址获取 16 位会引发异常的机器上，你应该一次获取一个字节的偏移量。

goto (0x21) 偏移量: ⇒

无条件跳转到 偏移量；换句话说，将 pc 寄存器设置为 start + 偏移量.

偏移量的存储方式与 if_goto 字节码相同。

const8 (0x22) n: ⇒ n
const16 (0x23) n: ⇒ n
const32 (0x24) n: ⇒ n
const64 (0x25) n: ⇒ n

将整数常量 n 压入栈中，不进行符号扩展。要产生一个小的负值，先压入一个小的二进制补码值，然后使用 ext 字节码进行符号扩展。

常量 n 存储在紧跟在 constb 字节码之后的适当数量的字节中。常量 n 总是先存储最高有效字节，无论目标的正常字节序如何。常量不能保证在字节码流中处于任何特定的对齐位置；因此，在从未对齐地址获取 16 位会引发异常的机器上，你应该一次获取一个字节的 n 。

reg (0x26) n: ⇒ a

将寄存器编号为 n, 的值压入栈中，不进行符号扩展。寄存器编号遵循 GDB 的约定。

寄存器编号 n 编码为一个 16 位无符号整数，紧跟在 reg 字节码之后。无论目标的正常字节序如何，它总是先存储最高有效字节。寄存器编号不能保证在字节码流中处于任何特定的对齐位置；因此，在从未对齐地址获取 16 位会引发异常的机器上，你应该一次获取一个字节的寄存器编号。

getv (0x2c) n: ⇒ v

将跟踪状态变量编号为 n, 的值压入栈中，不进行符号扩展。

变量编号 n 编码为一个 16 位无符号整数，紧跟在 getv 字节码之后。无论目标的正常字节序如何，它总是先存储最高有效字节。变量编号不能保证在字节码流中处于任何特定的对齐位置；因此，在从未对齐地址获取 16 位会引发异常的机器上，你应该一次获取一个字节的寄存器编号。

setv (0x2d) n: v ⇒ v

将跟踪状态变量编号为 n 设置为栈顶的值。栈保持不变，以便如果赋值是更大表达式的一部分，该值可以随时使用。对 n 的处理如 getv所述。

trace (0x0c): 地址 大小 ⇒

将 大小 字节的内容记录在 地址 处的跟踪缓冲区中，以便 GDB 稍后检索。

trace_quick (0x0d) 大小: 地址 ⇒ 地址

将 大小 字节的内容记录在 地址 处的跟踪缓冲区中，以便 GDB 稍后检索。 大小 是紧跟在 trace 操作码之后的一个单字节无符号整数。

这个字节码等同于序列 dup const8 size trace, 但我们还是提供它以节省字节码字符串的空间。

trace16 (0x30) 大小: 地址 ⇒ 地址

与 trace_quick 相同，除了 大小 是一个 16 位大端无符号整数，而不是一个单字节。为了保持一致性，它可能应该被命名为 trace_quick16.

tracev (0x2e) n: ⇒ a

将跟踪状态变量编号为 n 的值记录在跟踪缓冲区中。对 n 的处理如 getv所述。

tracenz (0x2f) 地址 大小 ⇒

将 地址 处的字节记录在跟踪缓冲区中，以便 GDB 稍后检索。在遇到第一个零字节或记录了 大小 字节时停止，以先发生的情况为准。

printf (0x34) 参数数量 字符串 ⇒

进行格式化打印，风格类似于 C 函数 printf). 参数数量 是期望在栈上的参数数量，而 字符串 是格式字符串，前缀为一个两字节的长度。字符串的最后一个字节必须为零，并且包含在长度中。格式字符串包含转义序列，就像它在 C 源文件中一样，例如格式字符串 "\t%d\n" 长度为六个字符，输出将由一个制表符、一个十进制数和一个换行符组成。在栈顶，在要打印的值之上，这个字节码将弹出一个“函数”和一个“通道”。如果函数不为零，那么目标可以将其视为一个函数并调用它，将通道作为第一个参数传递，就像 C 函数 fprintf. 如果函数为零，那么目标可以简单地调用它选择的标准格式化打印函数。总之，这个字节码弹出 2 + 参数数量 个栈元素，并且不压入任何元素。

end (0x27): ⇒

停止执行字节码；结果应该是栈顶元素。如果表达式的目的是计算一个左值或一个内存范围，那么栈的次顶元素是左值的地址，栈顶元素是左值的大小（以字节为单位）。
