23.4.3.10 编写一个Guile漂亮打印机 ¶

一个漂亮打印机由两个基本部分组成：一个用于确定类型是否受支持的查找函数，以及打印机本身。

这里有一个示例展示如何编写一个 std::string 打印机。请参阅 Guile漂亮打印API，了解详细信息。 

(define (make-my-string-printer value)
  "打印一个my::string字符串"
  (make-pretty-printer-worker
   "string"
   (lambda (printer)
     (value-field value "_data"))
   #f))

这里有一个示例展示如何编写上述打印机示例的查找函数。 

(define (str-lookup-function pretty-printer value)
  (let ((tag (type-tag (value-type value))))
    (and tag
         (string-prefix? "std::string<" tag)
         (make-my-string-printer value))))

然后将此打印机注册到全局打印机列表中： 

(append-pretty-printer!
 (make-pretty-printer "my-string" str-lookup-function))

示例查找函数提取值的类型，并尝试将其与可以漂亮打印的类型进行匹配。如果它是打印机可以漂亮打印的类型，它将返回一个 <gdb:pretty-printer-worker> 对象。如果不是，它将返回 #f.

我们建议您将核心漂亮打印机放入一个Guile包中。如果您的漂亮打印机是用于某个库的，我们进一步建议在包名中嵌入版本号。这种做法将使 GDB 能够同时加载您的漂亮打印机的多个版本，因为它们将具有不同的名称。

您应该编写自动加载代码（请参阅 Guile自动加载)，使其可以多次求值而不改变其含义。一个理想的自动加载文件应仅由 import您的打印机模块组成，然后调用一个函数将漂亮打印机注册到当前目标文件中。

总体而言，这种方法可以很好地扩展到多个下级程序，每个程序可能使用不同的库版本。在Guile包名中嵌入版本号将确保 GDB 能够同时加载两组打印机。然后，因为漂亮打印机的搜索是按目标文件进行的，并且因为您的自动加载代码小心地将您的库的打印机注册到特定的目标文件中， GDB 将为每个下级程序使用的特定版本的库找到正确的打印机。

为了继续 my::string 示例， 此代码可能出现在 (my-project my-library v1): 

(use-modules (gdb))
(define (register-printers objfile)
  (append-objfile-pretty-printer!
   (make-pretty-printer "my-string" str-lookup-function)))

然后自动加载文件的相应内容将是： 

(use-modules (gdb) (my-project my-library v1))
(register-printers (current-objfile))

前面的示例展示了一个基本的漂亮打印机。 有一些方面可以改进。 该打印机仅处理一种类型，而一个库通常有几种类型。可以为库中每个所需的类型安装一个查找函数，但也可以让一个单一的查找函数识别几种类型。后者是处理此问题的常规方法。 如果一个漂亮打印机可以处理多种数据类型，那么它的 子打印机 是针对各个数据类型的打印机。

该 (gdb printing) 模块提供了一种正式的方法来解决这个问题（请参阅 Guile打印模块)。 这里有另一个处理多种类型的示例。

这些是我们要漂亮打印的类型： 

struct foo { int a, b; };
struct bar { struct foo x, y; };

这里是打印机： 

(define (make-foo-printer value)
  "打印一个foo对象"
  (make-pretty-printer-worker
   "foo"
   (lambda (printer)
     (format #f "a=<~a> b=<~a>"
             (value-field value "a") (value-field value "a")))
   #f))

(define (make-bar-printer value)
  "打印一个bar对象"
  (make-pretty-printer-worker
   "foo"
   (lambda (printer)
     (format #f "x=<~a> y=<~a>"
             (value-field value "x") (value-field value "y")))
   #f))

这个示例不需要查找函数，这由 (gdb printing) 模块处理。相反，提供了一个函数来构建处理查找的对象。 

(use-modules (gdb printing))

(define (build-pretty-printer)
  (let ((pp (make-pretty-printer-collection "my-library")))
    (pp-collection-add-tag-printer "foo" make-foo-printer)
    (pp-collection-add-tag-printer "bar" make-bar-printer)
    pp))

这里是自动加载支持： 

(use-modules (gdb) (my-library))
(append-objfile-pretty-printer! (current-objfile) (build-pretty-printer))

最后，当此打印机加载到 GDB, 这里是相应的 ‘info pretty-printer’ 输出： 

(gdb) info pretty-printer
my_library.so:
  my-library
    foo
    bar
