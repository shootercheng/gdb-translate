2.1.1 选择文件 ¶

当 GDB 启动时，它会将除选项之外的任何参数视为指定可执行文件和核心文件（或进程ID）。这与通过‘-se’和 ‘-c’（或‘-p’）选项分别指定参数的效果相同。（GDB 会将没有关联选项标志的第一个参数视为等同于‘-se’选项后跟该参数；如果有第二个没有关联选项标志的参数，则将其视为等同于‘-c’/‘-p’选项后跟该参数。） 如果第二个参数以十进制数字开头， GDB 将首先尝试将其作为进程进行附加，如果失败，则尝试将其作为核心文件打开。如果您有一个名称以数字开头的核心文件，您可以通过在其前面加上 GDB 来防止 ./, 例如 ./12345。

如果 GDB 未配置为支持核心文件，例如对于大多数嵌入式目标，那么它会对第二个参数提出抱怨并忽略它。

对于‘-s’、‘-e’和‘-se’选项及其长形式等效项，在文件系统中搜索符号和/或可执行文件的方法与 file 命令使用的方法相同。请参阅 文件。

许多选项都有长形式和短形式；以下列表中同时显示了这两种形式。 GDB 如果您截断长形式选项，只要保留的部分足以明确标识该选项，也会识别它们。（如果您愿意，可以使用‘--’而不是‘-’来标记选项参数，不过我们展示的是更常见的约定。）

-symbols file ¶
-s file

从文件读取符号表 文件。

-exec file ¶
-e file

使用文件 文件 作为适当时候要执行的可执行文件，并用于结合核心转储检查纯数据。

-se file ¶

从文件读取符号表 文件 并将其用作可执行文件。

-core file ¶
-c file

使用文件 文件 作为要检查的核心转储。

-pid number ¶
-p number

连接到进程ID 编号, 就像使用 attach 命令一样。

-command file ¶
-x file

从文件执行命令 文件。此文件的内容将完全按照 source 命令的方式进行计算。请参阅 命令文件。

-eval-command command ¶
-ex command

执行单个 GDB 命令。

此选项可以多次使用以调用多个命令。也可以根据需要与‘-command’交错使用。 

gdb -ex 'target sim' -ex 'load' \
   -x setbreakpoints -ex 'run' a.out
-init-command file ¶
-ix file

在加载被调试程序之前（但在加载gdbinit文件之后）从文件执行命令 文件 。请参阅 启动时的操作 GDB 。

-init-eval-command command ¶
-iex command

在加载被调试程序之前（但在加载gdbinit文件之后）执行单个 GDB 命令。请参阅 启动时的操作 GDB 。

-early-init-command file ¶
-eix file

从以下文件执行命令 文件 在初始化过程的极早期，在产生任何输出之前。请参阅 启动时的操作 GDB 。

-early-init-eval-command command ¶
-eiex command

执行单个 GDB 命令，在初始化过程的极早期，在产生任何输出之前。

-directory directory ¶
-d directory

添加 目录 到搜索源文件和脚本文件的路径中。

-r ¶
-readnow

立即读取每个符号文件的整个符号表，而不是默认的按需逐步读取。这会使启动变慢，但会使后续操作更快。

--readnever

不读取每个符号文件的符号调试信息。这会使启动更快，但代价是无法进行符号调试。DWARF展开信息也不会被读取，这意味着回溯可能会不完整或不准确。此选项的一个用途是，当用户只想执行以下操作序列时：附加、转储核心、分离。在这种情况下，加载调试信息会不必要地导致延迟。
