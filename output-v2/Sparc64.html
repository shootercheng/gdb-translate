21.4.7.1 ADI支持 ¶

M7处理器支持应用程序数据完整性（ADI）功能，该功能可检测无效的数据访问。当软件分配内存并对分配的内存启用ADI时，它会选择一个4位的版本号，将该版本设置在指向该数据的64位指针的高4位中，并将4位版本存储在该数据的每个缓存行中。硬件将后者保存在缓存和内存层次结构的备用位中。在每次加载和存储时，处理器会将高4位VA（虚拟地址）位与缓存行的版本进行比较。如果不匹配，处理器会生成一个版本不匹配陷阱，该陷阱可以是精确的，也可以是中断性的。该陷阱是一种错误条件，内核会将其作为SIGSEGV信号传递给进程。

请注意，只有64位应用程序可以使用ADI，并且需要在启用ADI的情况下进行构建。

可以查看或修改以缓存行（64字节）为粒度的ADI版本标签的值。

adi (examine | x) [ / n ] addr ¶

该 adi examine 命令显示每个缓存行的一个ADI版本标签的值。

n 是一个十进制整数，指定字节数；默认值为1。它指定要显示的ADI版本信息的数量，比例为1:ADI块大小。

addr 是用户地址空间中您希望 GDB 开始显示ADI版本标签的地址。

以下是显示变量“shmaddr”的ADI版本的示例。 

(gdb) adi x/100 shmaddr
   0xfff800010002c000:     0 0
adi (assign | a) [ / n ] addr = tag ¶

该 adi assign 命令用于为一个地址分配新的ADI版本标签。

n 是一个十进制整数，指定字节数；默认值为1。它指定要修改的ADI版本信息的数量，比例为1:ADI块大小。

addr 是用户地址空间中您希望 GDB 开始修改ADI版本标签的地址。

tag 是新的ADI版本标签。

例如，执行以下操作来修改然后验证变量“shmaddr”的ADI版本： 

(gdb) adi a/100 shmaddr = 7
(gdb) adi x/100 shmaddr
   0xfff800010002c000:     7 7
