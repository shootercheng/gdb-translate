27.5.6 GDB/MI 线程信息 ¶

无论何时 GDB 需要报告有关线程的信息时，它会使用一个包含以下字段的元组。除非另有说明，这些字段总是存在。

id

由……分配给线程的全局数字 ID GDB.

target-id

标识线程的特定于目标的字符串。

details

目标提供的有关线程的额外信息。它应该是人类可读的，并且前端不会对其进行解释。此字段是可选的。

name

线程的名称。如果用户使用……指定了一个名称 thread name 命令，那么就给出该名称。否则，如果 GDB 可以从目标中提取线程名称，那么就给出该名称。如果 GDB 无法找到线程名称，那么就省略此字段。

state

线程的执行状态，要么是‘已停止’ 要么是‘正在运行’，这取决于线程当前是否正在运行。

frame

线程当前正在执行的栈帧。只有在线程停止时才会出现此字段。其格式在……中有文档说明 GDB/MI 帧信息.

core

此字段的值是线程最后一次被看到所在的处理器核心的整数值。此字段是可选的。
