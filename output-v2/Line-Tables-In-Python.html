23.3.2.32 使用 Python 操作行表 ¶

Python 代码可以从已加载的符号表中请求并检查行表信息。行表是源行到其在内存中可执行位置的映射。要获取特定符号表的行表信息，请使用 GDB函数（请参阅 linetable Python 中的符号表表示 ）。

一个 gdb.LineTable 是可迭代的。迭代器返回 LineTableEntry 对象，这些对象对应于每个行表条目的源行和地址。 LineTableEntry 对象具有以下属性：

变量： LineTableEntry.line ¶

此 行表条目的源行号。此编号对应于实际的源行。此属性不可写。

变量： LineTableEntry.pc ¶

与行表条目关联的地址，该源行的可执行代码驻留在内存中的该地址处。此属性不可写。

由于单个源行可能有多个地址，因此您可能会收到多个 LineTableEntry 对象，它们的 line 属性匹配，但 pc 属性不同。迭代器按升序排序。以下是一个遍历行表的小示例。 pc 

symtab = gdb.selected_frame().find_sal().symtab
linetable = symtab.linetable()
for line in linetable:
   print ("Line: "+str(line.line)+" Address: "+hex(line.pc))

这将产生以下输出： 

Line: 33 Address: 0x4005c8L
Line: 37 Address: 0x4005caL
Line: 39 Address: 0x4005d2L
Line: 40 Address: 0x4005f8L
Line: 42 Address: 0x4005ffL
Line: 44 Address: 0x400608L
Line: 42 Address: 0x40060cL
Line: 45 Address: 0x400615L

除了能够遍历一个 LineTable之外，它还具有以下直接访问方法：

函数： LineTable.line (line) ¶

返回一个 Python Tuple ，其中包含给定 LineTableEntry 行的行表中任何条目的 对象。行指定了源代码行。如果该源代码 行没有条目，则返回 Python。 None

函数： LineTable.has_line (line) ¶

返回一个 Python Boolean ，指示此源代码行在 行表中是否有条目。如果找到条目，则返回 True ，否则返回 False 。

函数： LineTable.source_lines () ¶

返回一个 Python List ，其中包含符号表中的源代码行号。仅返回具有可执行代码位置的行。 List 的内容将只是表示为 Python Long 值的源行条目。
