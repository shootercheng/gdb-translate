¶

新的 TUI（请参阅 GDB 文本用户界面) 窗口可以用 Python 实现。

函数： gdb.register_window_type (name, factory) ¶

由于 TUI 窗口是根据用户选择的布局创建和销毁的，因此新的窗口类型是通过向 GDB注册一个工厂函数来实现的。

name 是新窗口的名称。尝试替换内置窗口之一是错误的，但其他窗口类型可以被替换。 name 应匹配正则表达式 [a-zA-Z][-_.a-zA-Z0-9]*, 尝试创建一个名称无效的窗口是错误的。

function 是一个工厂函数，用于创建 TUI 窗口。调用此函数时会传入一个类型为 gdb.TuiWindow, 如下所述。它应该返回一个实现 TUI 窗口协议的对象，也如下所述。

如上所述，当调用工厂函数时，会传入一个类型为 gdb.TuiWindow的对象。这个对象有以下这些方法和属性：

函数： TuiWindow.is_valid () ¶

当此窗口有效时，此方法返回 True 。当用户更改 TUI 布局时，新布局中不再可见的窗口将被销毁。此时， gdb.TuiWindow 将不再有效，除了 is_valid 之外的方法（和属性）将抛出异常。

当使用 tui disable 禁用 TUI 时（请参阅 tui disable)，窗口会被隐藏而不是销毁，但 is_valid 仍将返回 False ，其他方法（和属性）仍将抛出异常。

变量： TuiWindow.width ¶

此属性保存窗口的宽度。它是不可写的。

变量： TuiWindow.height ¶

此属性保存窗口的高度。它是不可写的。

变量： TuiWindow.title ¶

此属性保存窗口的标题，是一个字符串。通常显示在窗口上方。此属性可以修改。

函数： TuiWindow.erase () ¶

移除窗口的所有内容。

函数： TuiWindow.write (string [, full_window]) ¶

将 string 写入窗口。 string 可以包含 ANSI 终端转义样式序列； GDB 将根据终端进行适当的转换。

如果 full_window 参数为 True, 那么 string 包含窗口的完整内容。这类似于在 erase 之前调用 write, 但避免了闪烁。

你提供的工厂函数应该返回一个符合 TUI 窗口协议的对象。以下是可以在此对象上调用的方法，下面将此对象称为“窗口对象”。下面记录的方法是可选的；如果对象没有实现其中一个方法， GDB 将不会尝试调用它。未来可能会向窗口协议添加新的方法。 GDB 保证它们将以小写字母开头，因此你可以使用大写字母或下划线开头来实现方法，以避免未来的冲突。

函数： Window.close () ¶

当 TUI 窗口关闭时， gdb.TuiWindow 对象将进入无效状态。此时， GDB 将调用窗口对象的 close 方法。

调用此方法后， GDB 将丢弃它对这个窗口对象持有的任何引用，并且不再调用此对象的方法。

函数: Window.render () ¶

在某些情况下，TUI 窗口的大小可能会改变。例如，如果用户调整终端大小或更改布局，就可能发生这种情况。当这种情况发生时， GDB 将调用窗口对象的 render 方法。

如果你的窗口旨在响应下级程序的变化进行更新，你可能还需要注册事件监听器并将输出发送到 gdb.TuiWindow。

函数： Window.hscroll (num) ¶

这是一个水平滚动窗口的请求。 num 是滚动的量，负数表示向右滚动。在 TUI 模型中，移动的是视口，而不是内容。正参数应使视口向右移动，因此内容应看起来向左移动。

函数： Window.vscroll (num) ¶

这是一个垂直滚动窗口的请求。 num 是滚动的量，负数表示向后滚动。在 TUI 模型中，移动的是视口，而不是内容。正参数应使视口向下移动，因此内容应看起来向上移动。

函数： Window.click (x, y, button) ¶

当在这个窗口中发生鼠标点击时会调用此函数。 x 和 y 是窗口内的鼠标坐标（从左上角开始，基于 0），并且 button 指定使用了哪个鼠标按钮，其值可以是 1（左键）、2（中键）或 3（右键）。

当通过关闭 tui mouse-events 设置禁用 TUI 鼠标事件时（请参阅 set tui mouse-events），则 click 将不会被调用。
