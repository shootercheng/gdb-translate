E.21 分支跟踪配置格式 ¶

对于每个下级线程， GDB 可以使用 ‘qXfer:btrace-conf:read’ （请参阅 qXfer btrace-conf read）数据包获取分支跟踪配置。

该配置描述了分支跟踪格式以及该格式的配置设置。以下信息会被描述：

bts

此线程使用 分支跟踪存储 （BTS）格式。

size

BTS 环形缓冲区的大小（以字节为单位）。

pt

此线程使用 英特尔处理器跟踪 （英特尔 PT）格式。

size

英特尔PT 环形缓冲区的大小（以字节为单位）。

GDB 必须与Expat库链接以支持XML分支跟踪配置发现。请参阅 Expat。

下面给出分支跟踪配置格式的正式DTD： 

<！ELEMENT btrace-conf		(bts?, pt?)> <！ATTLIST btrace-conf		version	CDATA	#FIXED "1.0"> <！ELEMENT bts	EMPTY> <！ATTLIST bts	size		CDATA	#IMPLIED> <！ELEMENT pt	EMPTY> <！ATTLIST pt	size		CDATA	#IMPLIED> <！ATTLIST pt	ptwrite		(yes | no)	#IMPLIED> <！ATTLIST pt	event-tracing	(yes | no)	#IMPLIED>
