E.1 概述 ¶

有时您可能需要了解有关该 协议的某些信息 — 例如，如果到目标 机器只有一个串行端口，您可能希望程序在识别出针对 GDB.

在下面的示例中，‘->’ 和 ‘<-’ 分别用于表示 发送和接收的数据。

所有 GDB 命令和响应（除了确认 和通知，见 通知数据包) 作为一个 数据包. 一个 数据包 以字符 ‘$’ 开始，实际的 数据包数据, 以及终止字符 ‘#’ 后跟一个两位的 校验和: 

$数据包数据#校验和

两位的 校验和 是通过对前导 ‘$’ 和尾随 ‘#’ 之间的所有字符进行模 256 求和计算得出的（一个 八位无符号校验和）。

实现者应注意，在 GDB 5.0 之前，协议 规范还包括一个可选的两位 序列号: 

$序列号:数据包数据#校验和

那个 序列号 被附加到确认信息中。 GDB 从未输出过 序列号。自 GDB 5.0 以来添加的处理数据包的存根不得接受 序列号.

当主机或目标机器接收到一个数据包时，预期的第一个 响应是确认信息：要么是 ‘+’（表示数据包已正确接收），要么是 ‘-’（请求 重新传输）： 

-> $数据包数据#校验和
<- +

‘+’/‘-’ 确认信息在连接建立后可以禁用。 请参阅 数据包确认, 了解详细信息。

主机（GDB) 发送 命令，目标（包含在您程序中的调试存根）发送一个 响应. 对于单步和继续 命令，只有在操作完成且目标再次停止所有 附加进程中的所有线程时才会发送响应。这是默认的全停止模式 行为，但远程协议也支持 GDB的非停止 执行模式；请参阅 远程协议对非停止模式的支持, 了解详细信息。

数据包数据 由一系列字符组成，但不包括 ‘#’ 和 ‘$’（有关其他例外情况，请参阅 ‘X’ 数据包）。

数据包内的字段应使用 ‘,’ ‘;’ 或 ‘:’。除非另有说明，所有数字均以 HEX 表示，前导零被省略。

实现者应注意，在 GDB 5.0 之前，字符 ‘:’ 不能作为数据包中的第三个字符出现（因为它 可能会与 序列号）冲突。

大多数数据包中的二进制数据以每字节二进制数据用两个十六进制 数字进行编码。这使得传统的远程 协议能够在仅支持七位干净的连接上工作。 一些最近设计的数据包假设是八位干净的 连接，并使用更高效的编码来发送和接收 二进制数据。

二进制数据表示使用 7d (ASCII ‘}’) 作为转义字符。任何转义字节都作为转义 字符后跟与 0x20进行异或运算后的原始字符进行传输。 例如，字节 0x7d 将作为两个 字节 0x7d 0x5d进行传输。字节 0x23 (ASCII ‘#’), 0x24 (ASCII ‘$’), 和 0x7d (ASCII ‘}’) 必须始终进行转义。存根发送的响应 也必须对 0x2a (ASCII ‘*’) 进行转义，以免 它被解释为游程编码序列（接下来描述）的开始。

响应 数据 可以进行游程编码以节省空间。 游程编码用重复字符的一个实例，后跟一个 ‘*’ 和一个 重复计数来替换连续相同字符的序列。重复计数本身也进行编码，以避免 数据: 值为 n 被发送为 n+29. 对于大于或等于 3 的重复计数，这会产生一个可打印的 字符，例如重复计数为 3 时的空格（ ASCII 代码 32）。（这是因为游程编码在计数为 3 或更多时开始节省空间。）因此，例如， ‘ASCII 0*’ 是 “0000” 的游程编码：‘ *’ 后面的空格字符 表示重复前导更多次。 0 32 - 29 = 3 更多次。

可打印字符 ‘#’ 和 ‘$’ 或数值大于 126 的字符不得使用。六个重复的序列（‘#’) 或 七个重复的序列（‘$’) 可以仅使用重复计数为 五（‘"’）进行展开。例如，‘00000000’ 可以编码为 ‘0*"00’。

请参阅 标准回复, 了解标准错误响应以及如何 响应表示命令不被支持。

在描述数据包（命令和响应）时，每个描述都有一个 模板，显示整体语法，然后是对该 数据包含义的解释。为了清晰起见，我们在一些模板中包含了空格；这些不是数据包语法的一部分。没有 GDB 数据包使用空格来分隔其组件。例如，像 ‘foo bar baz’ 这样的模板描述了一个以三个 ASCII 字节 ‘foo’ 开头的数据包，后跟一个 bar, 紧接着是一个 baz. GDB 在 ‘foo’ 和 bar之间，或者在 bar 和 baz之间不发送空格字符。

我们将数据包的可选部分放在 [方括号] 中； 例如，像 ‘c [addr]’ 这样的模板描述了一个以单个 ASCII 字符 ‘c’ 开头的数据包，可能 后跟一个 addr.

至少，存根需要支持 ‘?’ 命令以告知 GDB 停止的原因，‘g’ 和 ‘G’ 用于寄存器访问的命令，以及 ‘m’ 和 ‘M’ 用于内存访问的命令。仅控制单线程目标的存根 可以使用 ‘c’（继续）命令实现运行控制，如果 目标架构支持硬件辅助单步执行，则使用 ‘s’（单步）命令。支持多线程 目标的存根应支持 ‘vCont’ 命令。所有其他命令 都是可选的。
