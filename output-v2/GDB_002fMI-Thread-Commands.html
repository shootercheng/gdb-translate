27.11 GDB/MI 线程命令 ¶

该 -thread-info 命令 ¶

概要 ¶

 -thread-info [ 线程 ID ]

如果存在 线程 ID 参数，则报告特定线程的信息；否则报告所有线程的信息。 线程 ID 是线程的全局线程 ID。在打印所有线程的信息时，还会报告当前线程的全局 ID。

GDB 命令 ¶

‘info thread’ 命令会打印所有线程的相同信息。

结果 ¶

结果包含以下属性：

‘线程’

一个线程列表。列表元素的格式在 GDB/MI 线程信息中描述。

‘当前线程 ID’

当前选定线程的全局 ID。如果没有选定线程（例如，当选定的 inferior 未运行，因此没有线程），或者如果向命令传递了 线程 ID 参数，则省略此字段。

示例 ¶

-thread-info
^完成,线程=[
{id="2",目标 ID="线程 0xb7e14b90 (LWP 21257)",
   帧={级别="0",地址="0xffffe410",函数="__kernel_vsyscall",
           参数=[]},状态="运行中"},
{id="1",目标 ID="线程 0xb7e156b0 (LWP 21254)",
   帧={级别="0",地址="0x0804891f",函数="foo",
           参数=[{名称="i",值="10"}],
           文件="/tmp/a.c",完整名称="/tmp/a.c",行号="158",架构="i386:x86_64"},
           状态="运行中"}],
当前线程 ID="1"
(gdb)

该 -thread-list-ids 命令 ¶

概要 ¶

 -thread-list-ids

生成当前已知的全局 GDB 线程 ID 列表。在列表末尾还会打印此类线程的总数。

由于历史原因保留此命令，应使用 -thread-info 命令代替。

GDB 命令 ¶

‘info threads’ 的一部分提供相同的信息。

示例 ¶

(gdb)
-thread-list-ids
^完成,线程 ID={线程 ID="3",线程 ID="2",线程 ID="1"},
当前线程 ID="1",线程数量="3"
(gdb)

该 -thread-select 命令 ¶

概要 ¶

 -thread-select 线程 ID

将全局线程编号为 线程 ID 的线程设为当前线程。它会打印新当前线程的编号以及该线程的最顶层帧。

此命令已弃用，建议在每个命令中显式使用--线程’ 选项。

GDB 命令 ¶

对应的 GDB 命令是 ‘线程’。

示例 ¶

(gdb)
-exec-next
^运行中
(gdb)
*停止,原因="步进范围结束",线程 ID="2",行号="187",
文件="../../../devo/gdb/testsuite/gdb.threads/linux-dp.c"
(gdb)
-thread-list-ids
^完成,
线程 ID={线程 ID="3",线程 ID="2",线程 ID="1"},
线程数量="3"
(gdb)
-thread-select 3
^完成,新线程 ID="3",
帧={级别="0",函数="vprintf",
参数=[{名称="格式",值="0x8048e9c \"%*s%c %d %c\\n\""},
{名称="参数",值="0x2"}],文件="vprintf.c",行号="31",架构="i386:x86_64"}
(gdb)
