C.6 系统范围配置和设置 ¶

GDB 可以配置为拥有一个系统范围的初始化文件和一个系统范围的初始化文件目录；该文件以及该目录中的文件（如果它们具有公认的文件扩展名）将在启动时被读取并执行（请参阅 GDB GDB 在启动时的操作)。

以下是相应的配置选项：

--with-system-gdbinit=file

指定系统范围初始化文件的默认位置为 文件.

--with-system-gdbinit-dir=directory

指定系统范围初始化文件目录的默认位置为 目录.

如果 GDB GDB --prefix=$prefix, 它们可能会被重新定位。有两种可能的情况：

  • 如果此初始化文件/目录的默认位置包含 $prefix, 它将被重新定位。假设配置选项为 --prefix=$prefix --with-system-gdbinit=$prefix/etc/gdbinit; 如果 GDB GDB $prefix 移动到 $install, 则系统初始化文件将被查找为 $install/etc/gdbinit 而不是 $prefix/etc/gdbinit.
  • 相比之下，如果默认位置不包含前缀，它将不会被重新定位。例如，如果 GDB GDB --prefix=/usr/local --with-system-gdbinit=/usr/share/gdb/gdbinit, 那么 GDB GDB /usr/share/gdb/gdbinit, 无论 GDB GDB

如果系统范围初始化文件的配置位置（由 --with-system-gdbinit 选项在配置时指定）位于数据目录（由 --with-gdb-datadir 在配置时指定）或其某个子目录中，那么 GDB GDB --data-directory 命令行选项指定的目录中查找系统范围初始化文件。请注意，系统范围初始化文件仅在 GDB 初始化期间读取一次。如果在 GDB GDB set data-directory 命令启动后更改了数据目录，该文件将不会被重新读取。

这同样适用于 --with-system-gdbinit-dir中指定的系统范围目录。

任何受支持的脚本语言都可以用于这些初始化文件，只要文件扩展名与脚本语言匹配。要被解释为常规 GDB GDB .gdb 扩展名。
