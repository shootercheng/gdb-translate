15.4.10.2 Ada 中的缺失内容 ¶

以下是该子集显著的缺失内容：

  • 仅支持部分属性：
    • 'First, 'Last和 'Length 在数组对象上（不在类型和子类型上）。
    • 'Min 和 'Max.
    • 'Pos 和 'Val.
    • 'Tag.
    • 'Range 在数组对象上（不是子类型），但仅作为成员关系（in) 运算符的右操作数。
    • 'Access, 'Unchecked_Access和 'Unrestricted_Access （GNAT 扩展）。
    • 'Address.
    • 'Size 可用于对象（不是类型）。
    • 'Object_Size 可用，但不适用于不定类型。
  • 中的名称 Characters.Latin_1 不可用。
  • 数组的相等性测试（‘=’ 和 ‘/=’）测试表示形式的按位相等性。它们通常对字符串和元素为整数或枚举类型的数组能正确工作。对于元素类型具有用户定义相等性的数组、实值数组（特别是符合 IEEE 标准的浮点数，因为存在负零和 NaN）以及元素包含具有不确定值的未使用位的数组，它们可能无法正确工作。
  • 其他逐个组件的数组操作（and, or, xor, not, 以及除相等性之外的关系测试）未实现。
  • 对数组和记录聚合的支持有限。它们仅允许在赋值语句的右侧使用，例如以下示例： 
    (gdb) set An_Array := (1, 2, 3, 4, 5, 6)
(gdb) set An_Array := (1, others => 0)
(gdb) set An_Array := (0|4 => 1, 1..3 => 2, 5 => 6)
(gdb) set A_2D_Array := ((1, 2, 3), (4, 5, 6), (7, 8, 9))
(gdb) set A_Record := (1, "Peter", True);
(gdb) set A_Record := (Name => "Peter", Id => 1, Alive => True)

    如果记录中使用了判别式，通过赋值聚合来更改判别式的值会产生未定义的效果。但是，你可以先通过直接赋值来修改判别式（这在 Ada 中通常是不允许的），然后再进行聚合赋值。例如，给定一个变量 A_Rec 声明为具有如下类型： 

    type Rec (Len : Small_Integer := 0) is record
    Id : Integer;
    Vals : IntArray (1 .. Len);
end record;

    你可以通过两次赋值来为不同大小的 Vals 赋值： 

    (gdb) set A_Rec.Len := 4
(gdb) set A_Rec := (Id => 42, Vals => (1, 2, 3, 4))

    如这个示例所示， GDB 对于聚合的通常规则非常宽松。你可以省略数组或记录聚合的一些组件（例如在对 Len 的赋值中的组件）；赋值时它们将保留其原始值。你可以自由地使用动态值作为组件关联中的索引。你甚至可以使用重叠或冗余的组件关联，尽管在这种情况下哪些组件值会被赋值是未定义的。 A_Rec

  • 对分派子程序的调用未实现。
  • 重载算法比真正的 Ada 要受限得多（即选择性更低）。它仅有限地利用子表达式出现的上下文来解析其含义，并且在允许类型匹配的规则上要宽松得多。因此，一些函数调用会有歧义，用户将被要求选择合适的解决方案。
  • 该 new 运算符未实现。
  • 入口调用未实现。
  • 除了打印之外，不支持对原生 VAX 浮点格式进行算术运算。
  • 无法对打包数组进行切片。
  • 名称 True 和 False, 当不是限定名称的一部分时，无论上下文如何，都被解释为好像隐式地以 Standard为前缀。如果你的程序在包或过程中重新定义了这些名称（这充其量是一种可疑的做法），你将不得不使用完全限定名称来访问它们的新定义。
  • 基于实数的字面量未实现。
