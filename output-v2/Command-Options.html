3.5命令选项 ¶

一些命令接受以破折号开头的选项。例如， print -pretty 。与命令名称类似，如果缩写明确，您可以将选项缩写为首几个字母，并且还可以使用 GDB 选项名称的前几个字母，如果该缩写是明确的，并且您还可以使用 制表符 键来 GDB 填充选项中单词的其余部分（或者如果有多种可能性，向您显示可用的替代选项）。

一些命令将原始输入作为参数。例如，print命令处理 GDB支持的任何语言中的任意表达式。对于此类命令，由于原始输入可能以破折号开头，这可能会与选项或其任何缩写混淆，例如 print -p （ print -pretty 的缩写 p或打印负数--？），如果您指定任何命令选项，则必须使用双破折号（

一些选项被描述为接受一个可以是 on 或 off的参数。这些被称为 布尔选项 。与布尔设置命令类似 ——on 和 off 是典型值，但 1、 yes 和 enable 中的任何一个也可以用作“true”值，并且 0、 no 和 disable 中的任何一个也可以用作“false”值。您也可以省略“true”值，因为它默认隐含。

例如，这些是等效的： 

（gdb）打印 - 对象开启 - 美观关闭 - 元素无限制 -- * myptr
（gdb）p - o - p 0 - e u -- * myptr

您可以通过在命令名称后按 - 来发现某个命令接受的选项集。例如： 

（gdb）打印 -制表符制表符
- 地址         - 最大深度               - 对象          - 静态成员
- 数组           - 内存标签违规         - 美观          - 符号
- 数组索引       - 半字节                 - 原始值        - 联合
- 元素           - 空终止               - 重复次数        - vtbl

在某些情况下，自动完成将用关于选项期望何种类型参数的建议来引导您。例如： 

（gdb）打印 - 元素 制表符制表符
数字     无限制

这里，选项期望一个数字（例如 100）, 而不是文字 NUMBER 。这种元语法参数总是以大写形式呈现。
