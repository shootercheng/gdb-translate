函数： Command.dont_repeat () ¶

默认情况下，当用户在命令提示符处输入空行时， GDB 命令会重复执行。命令可以通过在其 dont_repeat 方法的某个点调用 invoke 方法来抑制此行为（通常在出现异常的情况下尽早执行）。这类似于用户命令 dont-repeat, 请参阅 dont-repeat。

函数：
Command.complete (文本, 单词) ¶ ¶

当用户尝试对该命令进行补全时，会调用此方法。所有形式的补全都由该方法处理，即 GDB TAB 和 M-? 键绑定（请参阅 命令补全 ），以及命令（请参阅 complete 补全 ）。)。

参数 文本 和 单词 都是字符串； 文本 包含直到光标位置的完整命令行，而 单词 包含命令行的最后一个单词；这是使用分词启发式算法计算得出的。

该 complete 方法可以返回多个值：

  • 如果返回值是一个序列，则该序列的内容将用作补全项。由 complete 确保这些内容实际上能补全该单词。允许使用长度为零的序列，这意味着没有可用的补全项。序列中只有字符串元素会被使用；序列中的其他元素将被忽略。
  • 如果返回值是下面定义的 ‘COMPLETE_’ 常量之一，则会调用相应的 GDB内部补全函数，并使用其结果。
  • 所有其他结果都被视为没有可用的补全项。

注册新命令时，必须将其声明为某个通用命令类的成员。这用于在在线帮助系统中对顶级命令进行分类；请注意，前缀命令不会列在其自身的类别下，而是列在其顶级命令的类别下。可用的分类由 gdb 模块中定义的常量表示：

gdb.COMMAND_NONE ¶

该命令不属于任何特定类别。此类别中的命令不会显示在任何帮助类别中。

gdb.COMMAND_RUNNING ¶

该命令与运行被调试程序有关。例如， start, step, 和 continue 属于此类别。输入 help running 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_DATA ¶

该命令与数据或变量有关。例如， call, find, 和 print 属于此类别。输入 help data 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_STACK ¶

该命令与栈的操作有关。例如， backtrace, frame, 和 return 属于此类别。输入 help stack 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_FILES ¶

此类用于与文件相关的命令。例如， file, list 和 section 属于此类别。输入 help files 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_SUPPORT ¶

这应该用于 “支持工具”，通常指的是用户与 GDB, 交互时有用的东西，但与被调试程序的状态无关。例如， help, make, 和 shell 属于此类别。输入 help support 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_STATUS ¶

该命令是一个与 ‘info’ 相关的命令，即与 GDB 自身的状态有关。例如， info, macro, 和 show 属于此类别。输入 help status 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_BREAKPOINTS ¶

该命令与断点有关。例如， break, clear, 和 delete 属于此类别。输入 help breakpoints 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_TRACEPOINTS ¶

该命令与跟踪点有关。例如， trace, actions, 和 tfind 属于此类别。输入 help tracepoints 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_TUI ¶

该命令与文本用户界面有关（请参阅 GDB 文本用户界面）。输入 help tui 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_USER ¶

该命令是供用户使用的通用命令，通常不属于其他类别。输入 help user-defined 在 GDB 提示符处查看此类别中的命令列表，以及 GDB 宏列表（请参阅 预定义命令序列）。

gdb.COMMAND_OBSCURE ¶

该命令仅在特殊情况下使用，或者对用户没有普遍吸引力。例如， checkpoint, fork, 和 stop 属于此类别。输入 help obscure 在 GDB 提示符处查看此类别中的命令列表。

gdb.COMMAND_MAINTENANCE ¶

该命令仅对 GDB 维护人员有用。 maintenance 和 flushregs 命令属于此类别。输入 help internals 在 GDB 提示符处查看此类别中的命令列表。

新命令可以使用预定义的补全函数，既可以在初始化时通过参数指定，也可以从 complete 方法返回。这些预定义的补全常量都在 gdb 模块中定义：

gdb.COMPLETE_NONE ¶

此常量表示不进行补全。

gdb.COMPLETE_FILENAME ¶

此常量表示应执行文件名补全。

gdb.COMPLETE_LOCATION ¶

此常量表示应进行位置补全。请参阅 位置规范。

gdb.COMPLETE_COMMAND ¶

此常量表示补全应检查 GDB 命令名称。

gdb.COMPLETE_SYMBOL ¶

此常量表示应使用符号名称作为源进行补全。

gdb.COMPLETE_EXPRESSION ¶

此常量表示应在表达式上进行补全。通常这意味着在符号名称上进行补全，但一些语言解析器也支持在字段名称上进行补全。

以下代码片段展示了如何在 Python 中实现一个简单的 CLI 命令： 

class HelloWorld (gdb.Command):
  """向全世界问好。"""

  def __init__ (self):
    super (HelloWorld, self).__init__ ("hello-world", gdb.COMMAND_USER)

  def invoke (self, arg, from_tty):
    print ("Hello, World!")

HelloWorld ()

最后一行实例化了该类，并且对于触发命令在 GDB中的注册是必要的。根据 Python 代码如何被读入 GDB, 你可能需要显式导入 gdb 模块。
