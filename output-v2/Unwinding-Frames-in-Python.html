23.3.2.13 Python 中的栈帧展开 ¶

在 GDB 术语 “展开” 是指从当前栈帧找到上一个栈帧（即调用者的栈帧）的过程。展开器有三个方法。第一个方法检查它是否可以处理给定的栈帧（“嗅探” 它）。对于它可以嗅探的栈帧，展开器提供两个额外的方法：它可以返回栈帧的 ID，并且可以从上一个栈帧获取寄存器。正在运行的 GDB 维护一个展开器列表，并依次调用每个展开器的嗅探器，直到找到能识别当前栈帧的展开器。有一个 API 用于注册展开器。

随附的展开器 GDB 处理标准栈帧。然而，混合语言应用程序（例如，运行 Java 虚拟机的应用程序）有时使用的栈帧布局无法由 GDB 展开器处理。你可以编写能处理此类自定义栈帧的 Python 代码。

你可以在 Python 中实现一个栈帧展开器，将其作为一个具有两个属性的类 name 和 enabled，具有明显的含义，并且有一个方法 __call__ gdb.UnwindInfo class) 描述它。如果展开器无法识别栈帧，它应该返回 None. 代码中 GDB 允许在 Python 中编写展开器的代码使用此对象在 GDB 核心请求时返回栈帧的 ID 和上一个栈帧的寄存器。

展开器应尽可能少做工作。一些看似无害的操作可能会导致问题（甚至崩溃，因为此代码尚未经过充分强化）。例如，从展开器进行 inferior 调用是不可取的，因为 inferior 调用会重置 GDBGDB 的栈展开过程，可能导致重入展开。

展开器输入 ¶

传递给展开器的对象（一个 gdb.PendingFrame PendingFrame 实例）提供了一个读取栈帧寄存器的方法：

函数： PendingFrame.read_register (register) ¶

此方法返回 register 在栈帧中的内容，作为一个 gdb.Value 对象。有关 register 可接受值的描述，请参阅 Frame.read_register. 如果 register 不是当前架构的寄存器名称，此方法将抛出异常。

请注意，对于有效的寄存器名称，此方法将始终返回一个 gdb.Value 对象。这并不意味着该值是有效的。例如，你可能请求一个早期展开器无法展开的寄存器 —— 该值将不可用。相反，此方法返回的 gdb.Value 对象将是惰性的；也就是说，其底层位直到首次使用时才会被获取。因此，尝试使用这样的值将在使用时引发异常。

返回的 gdb.Value 对象的类型取决于寄存器和架构。寄存器通常具有标量类型，如 long longint；但也可能有许多其他类型，如指针、函数指针、浮点或向量类型。

它还提供了一个工厂方法来创建一个 gdb.UnwindInfo UnwindInfo 实例，以返回给 GDBGDB：

函数： PendingFrame.create_unwind_info (frame_id) ¶

返回一个由给定 gdb.UnwindInfo UnwindInfo 实例。该 frame_id标识的新 frame_id 由 GDB GDB 内部用于标识当前线程栈内的栈帧。 frame_id 的属性决定了在 GDBGDB 中创建的栈帧类型：

sp, pc

栈帧由给定的栈地址和 PC 标识。栈地址必须选择为在栈帧的整个生命周期内保持不变，因此典型的选择是函数开始时栈指针的值 —— 在 DWARF 标准中，这将是 “调用帧地址”。

这是迄今为止最常见的情况。为了完整性，记录了其他情况，但它们仅在特殊情况下有用。

sp, pc, special

栈帧由栈地址、PC 和一个 “特殊” 地址标识。特殊地址用于那些可以有不改变栈但仍然不同的栈帧的架构，例如 IA - 64，它有一个用于寄存器的第二栈。 sp 和 special 在栈帧的整个生命周期内都必须保持不变。

sp

栈帧仅由栈地址标识。任何其他具有匹配 sp 的栈帧将被视为与此栈帧匹配。在 GDB 内部，这被称为 “通配栈帧”。你永远不需要这个。

每个属性值应该是 gdb.Value gdb.Value 实例或一个整数。

在 gdb.unwinder gdb.unwinder 模块中提供了一个辅助类，可用于表示栈帧 ID（请参阅 gdb.unwinder.FrameId)。

返回此 gdb.Architecture 架构（请参阅 Python 中的架构表示)。这表示正在展开的特定栈帧的架构。 gdb.PendingFrame栈帧的

函数： PendingFrame.level () ¶

返回一个整数，即此栈帧的栈帧级别。 请参阅 栈帧.

函数： PendingFrame.name () ¶

返回此待处理栈帧的函数名，或者 None 如果无法获取则为 None。

函数： PendingFrame.is_valid () ¶

如果该对象有效则返回 true，否则返回 false。当创建待处理栈帧的展开器调用返回时，待处理栈帧对象将变为无效。 gdb.PendingFrame 除了此方法之外的所有方法，如果在调用方法时待处理栈帧对象无效，将引发异常。

所有 gdb.PendingFrame 方法，除了这个方法，如果在调用方法时待处理栈帧对象无效，将引发异常。

返回待处理栈帧的恢复地址。

返回待处理栈帧的代码块（请参阅 从 Python 访问代码块)。 如果栈帧没有代码块（例如，若所涉及的代码没有调试信息），则会引发 RuntimeError 异常。

展开器输出：UnwindInfo ¶

使用 PendingFrame.create_unwind_info 上述方法创建一个 gdb.UnwindInfo 实例。使用以下方法指定保存在此栈帧中的调用者寄存器：

函数： gdb.UnwindInfo.add_saved_register (register, value) ¶

register 标识寄存器，有关可接受值的描述，请参阅 Frame.read_register. value 是一个寄存器值（一个 gdb.Value 对象）。

该 gdb.unwinder 模块 ¶

GDB 附带一个 gdb.unwinder 模块，其中包含以下类：

类： gdb.unwinder.Unwinder ¶

该 Unwinder 类是一个基类，用户创建的展开器可以从该类派生，但不要求展开器必须从该类派生，只要任何用户创建的展开器具有所需的 name 和 enabled 属性即可。

函数： gdb.unwinder.Unwinder.__init__ (name) ¶

该 name 是一个字符串，用于在某些 GDB 命令中引用此展开器（请参阅 管理已注册的展开器)。

变量： gdb.unwinder.name ¶

一个只读属性，是一个字符串，表示此展开器的名称。

变量： gdb.unwinder.enabled ¶

一个可修改的属性，包含一个布尔值；当 True为 True 时，展开器启用，并将由 GDB使用。当 False为 False 时，展开器已禁用，将不会被使用。

类： gdb.unwinder.FrameId ¶

这个类适合在调用 gdb.PendingFrame.create_unwind_info时用作栈帧 ID。不要求使用这个类，任何具有所需属性（请参阅 gdb.PendingFrame.create_unwind_info)的类都将被接受，但在大多数情况下，这个类就足够了。

gdb.unwinder.FrameId 有以下方法：

函数： gdb.unwinder.FrameId.__init__ (sp, pc, special = None) ¶

该 sp 和 pc 参数是必需的，应该是一个 gdb.Value 对象，或者是一个整数。

该 special 参数是可选的；如果指定，它应该是一个 gdb.Value 对象，或者是一个整数。

gdb.unwinder.FrameId 有以下只读属性：

变量： gdb.unwinder.sp ¶

传递给构造函数的 sp 值。

变量： gdb.unwinder.pc ¶

传递给构造函数的 pc 值。

变量： gdb.unwinder.special ¶

传递给构造函数的 special 值，或者 None 如果没有传递这样的值。

注册展开器 ¶

目标文件和程序空间可以注册展开器。此外，你可以全局注册展开器。

该 gdb.unwinders 模块提供了注册展开器的函数：

函数： gdb.unwinder.register_unwinder (locus, unwinder, replace=False) ¶

locus 指定将 展开器添加到哪个展开器列表的开头。它可以是一个目标文件（请参阅 Python 中的目标文件）、一个程序空间（请参阅 Python 中的程序空间），或者 NoneNone，在这种情况下，展开器将全局注册。新添加的 展开器 之前调用同一位置的任何其他展开器。同一位置的两个展开器不能有相同的名称。尝试添加具有已存在名称的展开器将引发异常，除非 replace 为 TrueTrue，在这种情况下，旧的展开器将被删除，新的展开器将取而代之进行注册。

GDB 首先按任意顺序调用所有目标文件中的展开器，然后调用当前程序空间中的展开器，接着调用全局注册的展开器，最后调用内置到 GDBGDB 中的展开器。

展开器骨架代码 ¶

以下是一个如何构建用户创建的展开器的示例： 

from gdb.unwinder import Unwinder, FrameId

class MyUnwinder(Unwinder):
    def __init__(self):
        super().__init___("MyUnwinder_Name")

    def __call__(self, pending_frame):
        if not <we recognize frame>:
            return None

        # Create a FrameID.  Usually the frame is identified by a
        # stack pointer and the function address.
        sp = ... compute a stack address ...
        pc = ... compute function address ...
        unwind_info = pending_frame.create_unwind_info(FrameId(sp, pc))

        # Find the values of the registers in the caller's frame and
        # save them in the result:
        unwind_info.add_saved_register(<register-number>, <register-value>)
        ....

        # Return the result:
        return unwind_info

gdb.unwinder.register_unwinder(<locus>, MyUnwinder(), <replace>)

管理已注册的展开器 ¶

GDB 定义了 3 个命令来管理已注册的展开器。这些命令是：

info unwinder [ locus [ name-regexp ] ]

列出所有已注册的展开器。参数 locus 和 name-regexp 都是可选的，可用于过滤要列出的展开器。

该 locus 参数应该是 global, progspace, 或者目标文件的名称。仅列出为指定位置注册的展开器。

该 name-regexp 是一个正则表达式，用于与展开器名称进行匹配。当尝试匹配包含字符串的展开器名称时，将 name-regexp 用引号括起来。

disable unwinder [ locus [ name-regexp ] ]

该 locus 和 name-regexp 的解释与 info unwinder 上述相同，但不是列出匹配的展开器，而是禁用所有匹配的展开器。每个匹配的展开器的 enabled enabled False字段将设置为 False。

enable unwinder [ locus [ name-regexp ] ]

该 locus 和 name-regexp 的解释与 info unwinder 上述相同，但不是列出匹配的展开器，而是启用所有匹配的展开器。每个匹配的展开器的 enabled enabled True字段将设置为 True。
