10.7 内存标记 ¶

内存标记是一种内存保护技术，它使用一对标签来验证通过指针进行的内存访问。标签是整数值，通常由几位组成，具体取决于架构。

在此设置中使用两种类型的标签：逻辑标签和分配标签。逻辑标签存储在指针本身中，通常位于指针的高位。分配标签是与物理地址空间中特定内存范围相关联的标签，指针的逻辑标签会与该标签进行比较。

指针标签（逻辑标签）必须与内存标签（分配标签）匹配，内存访问才有效。如果逻辑标签与分配标签不匹配，将引发内存违规。

分配标签覆盖物理内存的多个连续字节。这个字节范围称为内存标签粒度，并且是特定于架构的。例如，AArch64 的标签粒度为 16 字节，这意味着每个分配标签跨越 16 字节的内存。

如果底层架构支持内存标记，如 AArch64 MTE 或 SPARC ADI 那样， GDB 可以利用它来根据内存分配标签验证指针。

print （请参阅 检查数据) 和 x （请参阅 检查内存) 命令将在适当的时候显示标签信息，并且命令前缀 memory-tag 可以访问各种内存标记命令。

memory-tag 命令如下：

memory-tag print-logical-tag pointer_expression ¶

打印存储在 指针表达式中的逻辑标签。

memory-tag with-logical-tag pointer_expression tag_bytes

打印由 指针表达式给出的指针，并增加一个逻辑标签 标签字节。

memory-tag print-allocation-tag address_expression

打印与由 地址表达式给出的内存地址相关联的分配标签。

memory-tag setatag starting_address length tag_bytes

设置内存范围的分配标签 [起始地址, 起始地址 + 长度) 为 标签字节。

memory-tag check pointer_expression

检查由 指针表达式 给出的指针中的逻辑标签是否与该指针引用的内存的分配标签匹配。

这本质上模拟了通过指针访问带标签内存时进行的硬件验证，但不会像硬件验证期间那样导致内存故障。

它可用于在触发任何故障之前，检查正在运行的进程中潜在的内存标记违规。
