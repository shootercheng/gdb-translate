18.5 索引文件加速 GDB ¶

当 GDB 找到符号文件时，它会按顺序扫描文件中的符号以构建内部符号表。这使得大多数 GDB 操作能够快速进行 —— 代价是早期会有延迟。对于大型程序，这种延迟可能会相当长，因此 GDB 提供了一种构建索引的方法，这可以加快启动速度。

为了方便起见， GDB 附带了一个程序 gdb-add-index, 它可用于向符号文件添加索引。它只接受符号文件作为参数： 

$ gdb-add-index symfile

参见 gdb-add-index.

也可以手动完成这项工作。以下是 gdb-add-index 在幕后所做的事情。

索引作为一个节存储在符号文件中。 GDB 可以将索引写入文件，然后您可以使用 objcopy. 将其放入符号文件中

要创建索引文件，请使用 save gdb-index 命令：

save gdb-index [-dwarf-5] directory ¶

为当前 GDB已知的所有符号文件创建索引文件。对于每个已知的 符号文件, 此命令默认生成一个文件 符号文件.gdb-index. 如果您使用 -dwarf-5 选项调用此命令，它会生成 2 个文件： 符号文件.debug_names 和 符号文件.debug_str. 文件将在给定的 目录.

该 目录 参数支持转义和引用，参见 作为命令参数的文件名.

一旦您创建了索引文件，就可以将其合并到您的符号文件中，这里命名为 symfile, 使用 objcopy: 

$ objcopy --add-section .gdb_index=symfile.gdb-index \
    --set-section-flags .gdb_index=readonly symfile symfile

或者对于 -dwarf-5: 

$ objcopy --dump-section .debug_str=symfile.debug_str.new symfile
$ cat symfile.debug_str >>symfile.debug_str.new
$ objcopy --add-section .debug_names=symfile.gdb-index \
    --set-section-flags .debug_names=readonly \
    --update-section .debug_str=symfile.debug_str.new symfile symfile

GDB 通常会忽略较旧版本的 .gdb_index 已被弃用的节。通常它们被弃用是因为缺少新功能或存在性能问题。 要告诉 GDB 无论如何都使用已弃用的索引节，请指定 set use-deprecated-index-sections on. 默认值是 off. 这可以加快启动速度，但可能会导致一些功能丢失。 参见 .gdb_index 节格式.

警告： 将 use-deprecated-index-sections 设置为 on 必须在 gdb 读取文件之前完成。以下操作将不起作用： 

$ gdb -ex "set use-deprecated-index-sections on" <program>

相反，您必须这样做，例如 

$ gdb -iex "set use-deprecated-index-sections on" <program>

索引仅在使用 DWARF 调试信息时有效，而不适用于 stabs。

18.5.1 自动符号索引缓存 ¶

有可能让 GDB 自动将此索引的副本保存在磁盘缓存中，并在将来加载相同的二进制文件时从那里检索它。可以使用 set index-cache enabled on. 启用此功能。 以下命令可用于调整索引缓存的行为。

set index-cache enabled on ¶
set index-cache enabled off

启用或禁用符号索引缓存的使用。

set index-cache directory directory
show index-cache directory

设置/显示将保存索引文件的目录。

此目录的默认值取决于主机平台。在大多数系统上，如果定义了 gdb 环境变量所指向的目录，则索引会缓存在该目录的 XDG_CACHE_HOME 子目录中，否则会缓存在您主目录的 .cache/gdb 子目录中。但是，在某些系统上，默认值可能会根据本地约定有所不同。

索引缓存使用的磁盘空间没有限制。删除该目录的内容以释放磁盘空间是完全安全的。

show index-cache stats

打印自启动以来缓存命中和未命中的次数 GDB.

18.5.2 使用 GNU 构建索引 gold ¶

GNU gold 链接器可以在链接时将索引写入生成的 ELF 文件，通过传递 --gdb-index 标志给 gold.

如果您打算构建一个程序并立即在 GDB下运行它，这尤其有用。您可能会发现，总体而言，链接器在内存中已有相关信息时写入索引比使用 gdb-add-index从磁盘重新加载数据，或者完全不使用索引调试程序要快，特别是对于大型程序。
