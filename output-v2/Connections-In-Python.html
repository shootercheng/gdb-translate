GDB 允许你在单个会话中运行和调试多个程序。每个正在调试的程序都有一个连接，该连接描述了 GDB 如何控制正在调试的程序。不同连接类型的示例有‘本地’和‘远程’。参见 调试多个下级程序、连接和程序.

在 GDB 中的连接表示为 gdb.TargetConnection的实例，或者是其某个子类的实例。要获取所有连接的列表，请使用 gdb.connections （参见 gdb.connections）。

要获取单个 gdb.Inferior 的连接，请读取其 gdb.Inferior.connection 属性（参见 gdb.Inferior.connection）。

目前 gdb.TargetConnection只有一个子类， gdb.RemoteTargetConnection不过，在未来的 GDB版本中可能会添加更多的子类。因此，你应该避免编写如下代码： 

conn = gdb.selected_inferior().connection
if type(conn) is gdb.RemoteTargetConnection:
  print("This is a remote target connection")

因为当添加更多连接类型时，这段代码可能会失败。相反，你应该编写： 

conn = gdb.selected_inferior().connection
if isinstance(conn, gdb.RemoteTargetConnection):
  print("This is a remote target connection")

一个 gdb.TargetConnection 具有以下方法：

函数： TargetConnection.is_valid () ¶

如果 True ，否则返回 gdb.TargetConnection 对象有效，则返回 False 。如果连接在 gdb.TargetConnection 中不再存在，那么 GDB可能会变得无效，这种情况可能在没有下级程序使用该连接时发生，但可能会延迟到用户替换当前目标时。

如果连接无效，读取任何 gdb.TargetConnection 属性都会抛出异常。

一个 gdb.TargetConnection 具有以下只读属性：

变量： TargetConnection.num ¶

由 GDB 分配的一个整数，用于唯一标识此连接。这与‘编号’列在 info connections 命令输出中显示的值相同（参见 info connections）。

变量： TargetConnection.type ¶

一个字符串，描述此连接的类型。这个字符串将是可以传递给 target 命令的有效名称之一（参见 target command）。

变量： TargetConnection.description ¶

一个字符串，给出此目标类型的简短描述。这与‘描述’列在 info connection 命令输出中显示的字符串相同（参见 info connections）。

变量： TargetConnection.details ¶

一个可选字符串，提供有关此连接的额外信息。如果此连接没有额外细节，此属性可以为 None 。

可能有额外细节的连接类型的一个示例是‘远程’连接，在这种情况下，细节字符串可以包含用于连接到远程目标的‘主机名:端口’。

gdb.RemoteTargetConnection 类是 gdb.TargetConnection的子类，用于表示‘远程’和‘扩展远程连接。除了基类提供的属性和方法外， gdb.TargetConnection 一个 gdb.RemoteTargetConnection 具有以下方法：

函数： RemoteTargetConnection.send_packet (数据包) ¶

此方法发送 数据包 到远程目标并返回响应。 数据包 应该是一个 bytes 对象，或者是一个 Unicode 字符串。

如果 数据包 是一个 Unicode 字符串，那么该字符串将使用 bytes 编解码器编码为一个 ASCII 对象。如果字符串无法编码，则会引发一个 UnicodeError 异常。

如果 数据包 不是一个 bytes 对象，也不是一个 Unicode 字符串，那么会引发一个 TypeError 异常。如果 数据包 为空，则会引发一个 ValueError 异常。

响应以一个 bytes 对象的形式返回。如果已知响应可以表示为字符串，则可以从缓冲区中解码出来。例如，如果已知响应是一个 ASCII 字符串： 

remote_connection.send_packet("some_packet").decode("ascii")

前缀、后缀和校验和（根据远程串行协议的要求）会自动添加到传出的数据包中，并在返回响应内容之前从传入的数据包中移除。

这相当于 maintenance packet 命令（参见 maint packet）。
