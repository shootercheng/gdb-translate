GDB 提供了一个通用的事件机制，以便 Python 代码可以收到各种状态变化的通知，特别是被调试程序中发生的变化。

只是一个描述某些状态变化的对象。对象的类型及其属性将根据变化的细节而有所不同。下面描述了所有现有的事件。

为了收到事件通知，您必须向 事件注册表 gdb.events 模块，该模块会分发特定的事件。注册表提供了注册和注销事件处理程序的方法：

函数： EventRegistry.connect (对象) ¶

添加给定的可调用 对象 到注册表中。当与此注册表对应的事件发生时，将调用此对象。

函数： EventRegistry.disconnect (对象)

移除给定的 对象 从注册表中移除。移除后，该对象将不再收到事件通知。

以下是一个示例： 

def exit_handler (event):
    print ("event type: exit")
    if hasattr (event, 'exit_code'):
        print ("exit code: %d" % (event.exit_code))
    else:
        print ("exit code not available")

gdb.events.exited.connect (exit_handler)

在上面的示例中，我们将处理程序 exit_handler 连接到 events.exited注册表。连接后，当被调试程序退出时， exit_handler 会被调用。在这个示例中，参数 event 的类型为 gdb.ExitedEvent。正如您在示例中看到的， ExitedEvent 对象有一个属性，该属性指示被调试程序的退出代码。

当 GDB 在非停止模式下运行时，某些事件可能是特定于线程的。在 Python 中表示时，这些事件都继承自 gdb.ThreadEvent。此事件是一个基类，不会直接发出；相反，由这个或其他模块发出的事件可能会继承此事件。这些事件的示例包括 gdb.BreakpointEvent 和 gdb.ContinueEvent。 gdb.ThreadEvent 具有以下属性：

变量： ThreadEvent.inferior_thread ¶

在非停止模式下，此属性将设置为参与发出的事件的特定线程。否则，它将设置为 None。

以下是可用的事件注册表列表以及它们发出的事件的详细信息：

events.cont

发出 gdb.ContinueEvent, 它继承自 gdb.ThreadEvent. 此事件表明被调试程序在停止后已继续执行。有关继承的属性，请参阅 gdb.ThreadEvent 上面的内容。

events.exited

发出 events.ExitedEvent, 它表明被调试程序已退出。 events.ExitedEvent 有两个属性：

变量： ExitedEvent.exit_code ¶

一个整数，表示被调试程序返回的退出代码（如果可用）。（例如，如果 GDB 从被调试程序分离，则退出代码可能不可用。）如果退出代码不可用，则该属性不存在。

变量： ExitedEvent.inferior ¶

对触发 exited 事件的被调试程序的引用。

events.stop

发出 gdb.StopEvent, 它继承自 gdb.ThreadEvent.

表明被调试程序已停止。此注册表发出的所有事件都继承自 gdb.StopEvent. 作为 gdb.ThreadEvent, 的子类， gdb.StopEvent 将在 GDB 在非停止模式下运行时指示停止的线程。有关更多详细信息，请参阅 gdb.ThreadEvent 上面的内容。

gdb.StopEvent 具有以下附加属性：

变量： StopEvent.details ¶

一个字典，包含与停止相关的任何详细信息。确切的键和值取决于停止的类型，但与相应的 MI 输出相同（请参阅 GDB/MI 异步记录）。

使用字典（而不是直接向事件对象添加属性）是为了可以不变地使用 MI 键。

当一个 StopEvent 是由一个 finish 命令导致的时，如果可用，它还将包含函数的返回值。这将是return - value字典中名为‘ details ’的条目。该条目的值将是一个 gdb.Value 对象。

发出 gdb.SignalEvent，它继承自 gdb.StopEvent。

此事件表明被调试程序或其某个线程收到了一个信号。 gdb.SignalEvent 具有以下属性：

变量： SignalEvent.stop_signal ¶

一个字符串，表示被调试程序收到的信号。可以通过在 info signals 中的 GDB 命令提示符下运行该命令来获取可能的信号值列表。

还会发出 gdb.BreakpointEvent，它继承自 gdb.StopEvent。

gdb.BreakpointEvent 事件表明一个或多个断点被触发，并且具有以下属性：

变量： BreakpointEvent.breakpoints ¶

一个序列，包含对所有被触发的断点（类型为 gdb.Breakpoint）的引用。有关 使用 Python 操作断点对象的详细信息，请参阅 gdb.Breakpoint 。

变量： BreakpointEvent.breakpoint ¶

对第一个被触发的断点的引用。为了向后兼容而保留此属性，现在已弃用，建议使用 gdb.BreakpointEvent.breakpoints 属性。

events.new_objfile

发出 gdb.NewObjFileEvent ，它表明 GDB已加载了一个新的目标文件。 gdb.NewObjFileEvent 有一个属性：

变量： NewObjFileEvent.new_objfile ¶

对已加载的目标文件（gdb.Objfile）的引用。有关 Python 中的目标文件对象的详细信息，请参阅 gdb.Objfile 。

events.free_objfile

发出 gdb.FreeObjFileEvent ，它表明一个目标文件即将从 GDB中移除。发生这种情况的一个原因是当被调试程序调用 dlclose时。 gdb.FreeObjFileEvent 有一个属性：

变量： FreeObjFileEvent.objfile ¶

对即将被卸载的目标文件（gdb.Objfile）的引用。有关 Python 中的目标文件对象的详细信息，请参阅 gdb.Objfile 。

events.clear_objfiles

发出 gdb.ClearObjFilesEvent ，它表明程序空间的目标文件列表已被重置。 gdb.ClearObjFilesEvent 有一个属性：

变量： ClearObjFilesEvent.progspace ¶

对其目标文件列表已被清除的程序空间（gdb.Progspace）的引用。有关 Python 中的程序空间。

events.inferior_call

在被调试程序中的一个函数被 GDB调用之前和之后发出事件。在被调试程序调用之前，发出类型为 gdb.InferiorCallPreEvent的事件，在被调试程序调用之后，发出类型为 gdb.InferiorCallPostEvent的事件。

gdb.InferiorCallPreEvent ¶

表明被调试程序中的一个函数即将被调用。

变量： InferiorCallPreEvent.ptid ¶

调用将在其中运行的线程。

变量： InferiorCallPreEvent.address ¶

要调用的函数的位置。

gdb.InferiorCallPostEvent ¶

表明被调试程序中的一个函数刚刚被调用。

变量： InferiorCallPostEvent.ptid ¶

调用在其中运行的线程。

变量： InferiorCallPostEvent.address ¶

被调用的函数的位置。

events.memory_changed

发出 gdb.MemoryChangedEvent ，它表明被调试程序的内存已被 GDB 用户修改，例如通过像 set *addr = value这样的命令。该事件具有以下属性：

变量： MemoryChangedEvent.address ¶

更改区域的起始地址。

变量： MemoryChangedEvent.length ¶

更改区域的字节长度。

events.register_changed

发出 gdb.RegisterChangedEvent ，它表明被调试程序中的一个寄存器已被 GDB 用户修改。

变量： RegisterChangedEvent.frame ¶

一个 gdb.Frame 对象，表示寄存器被修改的帧。

变量： RegisterChangedEvent.regnum ¶

表示哪个寄存器被修改。

events.breakpoint_created

当创建一个新的断点时会发出此事件。传递的参数是新的 gdb.Breakpoint 对象。

events.breakpoint_modified

当以某种方式修改了一个断点时会发出此事件。传递的参数是新的 gdb.Breakpoint 对象。

events.breakpoint_deleted

当删除一个断点时会发出此事件。传递的参数是 gdb.Breakpoint 对象。当发出此事件时， gdb.Breakpoint 对象将已经处于无效状态；也就是说， is_valid 方法将返回 False。

events.before_prompt

此事件不携带任何有效负载。每次 GDB 向用户显示提示时都会发出此事件。

events.new_inferior

当创建一个新的被调试程序时会发出此事件。请注意，被调试程序不一定在运行；实际上，它甚至可能没有关联的可执行文件。

该事件的类型为 gdb.NewInferiorEvent。它有一个属性：

变量： NewInferiorEvent.inferior ¶

新的被调试程序，一个 gdb.Inferior 对象。

events.inferior_deleted

当删除一个被调试程序时会发出此事件。请注意，这与进程退出不同；当被调试程序本身被移除时会收到通知，例如通过 remove-inferiors。

该事件的类型为 gdb.InferiorDeletedEvent。它有一个属性：

变量： InferiorDeletedEvent.inferior ¶

正在被移除的被调试程序，一个 gdb.Inferior 对象。

events.new_thread

当 GDB 检测到一个新线程时会发出此事件。该事件的类型为 gdb.NewThreadEvent, 它继承自 gdb.ThreadEvent。它有一个属性：

变量： NewThreadEvent.inferior_thread ¶

新线程。

events.thread_exited

当 GDB 检测到一个线程已退出时会发出此事件。该事件的类型为 gdb.ThreadExitedEvent , 它继承自 gdb.ThreadEvent。它有一个属性：

变量： ThreadExitedEvent.inferior_thread ¶

正在退出的线程。

events.gdb_exiting

当 GDB 退出时会发出此事件。如果 GDB 由于内部错误或意外信号而退出，则不会发出此事件。该事件的类型为 gdb.GdbExitingEvent, 它有一个属性：

变量： GdbExitingEvent.exit_code ¶

一个整数，即 GDB 将返回的退出代码的值。

events.connection_removed

当 GDB 移除一个连接时会发出此事件（请参阅 Python 中的连接）。该事件的类型为 gdb.ConnectionEvent。它有一个只读属性：

变量： ConnectionEvent.connection ¶

正在被移除的 gdb.TargetConnection 。

events.executable_changed

发出 gdb.ExecutableChangedEvent ，它表明 gdb.Progspace.executable_filename 已更改。

当 gdb.Progspace.executable_filename 的值已更改为指定不同的文件，或者 gdb.Progspace.executable_filename 指定的可执行文件在磁盘上已更改，并且 GDB 因此重新加载了它时，会发出此事件。

变量： ExecutableChangedEvent.progspace ¶

当前可执行文件已更改的 gdb.Progspace 。更新后的可执行文件的文件名将在 gdb.Progspace.executable_filename 中可见（请参阅 Python 中的程序空间）。

变量： ExecutableChangedEvent.reload ¶

。 True 的值没有更改，但它指定的文件在磁盘上已更改，并且 gdb.Progspace.executable_filename GDB 重新加载了它，则此属性将为

当此属性为 False时， gdb.Progspace.executable_filename 中的值已更改为指定不同的文件。

请记住， GDB 分别跟踪可执行文件和符号文件，它们分别显示为 gdb.Progspace.executable_filename 和 gdb.Progspace.filename 。使用 文件 命令时， GDB 会更新这两个字段，但会先更新可执行文件，因此当发出此事件时，可执行文件的文件名会已更改，但符号文件名可能仍保留其先前的值。

events.new_progspace

当 GDB 添加一个新的程序空间时会发出此事件（请参阅 Python 中的程序空间）。该事件的类型为 gdb.NewProgspaceEvent, 并且有一个只读属性：

变量： NewProgspaceEvent.progspace ¶

添加到 gdb.Progspace 中的 GDB。

对于分配给第一个被调试程序的第一个程序空间，不会发出 NewProgspaceEvent 。这个第一个程序空间是在 GDB 中创建的，在任何 Python 脚本被加载之前。

events.free_progspace

当 GDB 移除一个程序空间时会发出此事件（请参阅 Python 中的程序空间），例如作为 remove - inferiors 命令（请参阅 remove - inferiors）的结果。该事件的类型为 gdb.FreeProgspaceEvent, 并且有一个只读属性：

变量： FreeProgspaceEvent.progspace ¶

即将从 gdb.Progspace 中移除的 GDB。

events.tui_enabled

当 TUI 启用或禁用时会发出此事件。该事件的类型为 gdb.TuiEnabledEvent, 它有一个只读属性：

变量： TuiEnabledEvent.enabled ¶

如果 TUI 刚刚启用，则此值为 True；否则为 False。
