13.1.9 启动和停止跟踪实验 ¶

tstart ¶

此命令启动跟踪实验并开始收集数据。 它会产生丢弃上一次跟踪实验期间跟踪缓冲区中收集的所有数据的副作用。 如果提供了任何参数，它们将被作为注释并与跟踪实验的状态一起存储。 注释可以是任意文本，在多用户环境中的断开式跟踪中特别有用；注释可以解释跟踪正在做什么，提供用户联系信息等等。

tstop ¶

此命令停止跟踪实验。 如果提供了任何参数，它们将作为注释与实验一起记录。 如果你要停止别人启动的跟踪，这很有用，例如如果跟踪干扰了系统的行为，需要迅速停止。

注意: 如果任何跟踪点的通过计数达到上限（请参阅 跟踪点通过计数），或者跟踪缓冲区已满，跟踪实验和数据收集可能会自动停止。

tstatus ¶

此命令显示当前跟踪数据收集的状态。

以下是我们到目前为止描述的命令的一个示例： 

(gdb) 跟踪 gdb_c_test
(gdb) 动作
为跟踪点 #1 输入动作，每行一个。
> 收集 $regs, $locals, $args
> 单步执行 11 次
  > 收集 $regs
  > 结束
> 结束
(gdb) 启动跟踪
	[一段时间过去 ...]
(gdb) 停止跟踪

你可以选择即使 GDB 与目标断开连接（无论是自愿还是非自愿），仍继续运行跟踪实验。 对于诸如 detach之类的命令，调试器会询问你对跟踪要做什么。 但对于意外终止（GDB 崩溃、网络中断），丢失来之不易的跟踪数据会很可惜，因此变量 disconnected-tracing 可以让你决定跟踪是否应该在没有 GDB的情况下继续运行。

set disconnected-tracing on ¶
set disconnected-tracing off

选择如果 GDB 与目标断开连接，跟踪运行是否应该继续。 请注意， detach 或 quit 无论此变量的设置如何，都会直接询问你对正在运行的跟踪要做什么，因此该变量主要用于处理意外情况，例如网络中断。

show disconnected-tracing ¶

显示当前断开式跟踪的选择。

当你重新连接到目标时，跟踪实验可能仍在运行，也可能已经停止；在此期间跟踪缓冲区可能已经填满，或者由于其他原因停止。 如果它正在运行，重新连接后将继续。

重新连接后，目标将上传有关当前生效的跟踪点的信息。 GDB 然后将该信息与当前定义的跟踪点集进行比较，并尝试匹配它们，考虑到在此期间由于创建和删除操作，跟踪点编号可能已经改变的可能性。 如果目标的某个跟踪点与 GDB中的任何跟踪点都不匹配，调试器将创建一个新的跟踪点，以便你有一个编号来指定该跟踪点。 这种匹配过程必然是启发式的，可能会导致创建无用的跟踪点；如果它们没有用，你可以直接删除它们。

如果你的目标代理支持 循环跟踪缓冲区, 那么你可以无限期地运行跟踪实验而不会填满跟踪缓冲区；当空间不足时，代理会删除已经收集的跟踪帧，从最旧的开始，直到有足够的空间继续收集。 如果你的跟踪点被频繁命中，并且由于缓冲区已满而导致跟踪过早终止，这尤其有用。 要请求一个循环跟踪缓冲区，只需将 ‘循环跟踪缓冲区’ 设置为开启。 你可以在任何时候进行设置，包括在跟踪期间；如果代理可以做到，它将动态更改缓冲区处理方式，否则直到下一次运行时才会生效。

set circular-trace-buffer on ¶
set circular-trace-buffer off

选择跟踪运行应该使用线性还是循环缓冲区来存储跟踪数据。 线性缓冲区不会丢失任何跟踪数据，但可能会过早填满，而循环缓冲区会丢弃旧的跟踪数据，但始终有空间存储最新的跟踪点命中信息。

show circular-trace-buffer ¶

显示当前跟踪缓冲区的选择。 请注意，这可能与代理当前的缓冲区处理方式不匹配，也不能保证与过去运行期间可能生效的设置相匹配，例如如果你正在查看跟踪文件中的帧。

set trace-buffer-size n ¶
set trace-buffer-size unlimited

请求目标使用大小为 n 字节的跟踪缓冲区。 并非所有目标都会满足此请求；它们可能有一个编译时确定的跟踪缓冲区大小，或者有其他限制。 将其设置为 unlimited 或 -1 的值，让目标使用它喜欢的任何大小。 这也是默认设置。

show trace-buffer-size ¶

显示当前请求的跟踪缓冲区大小。 请注意，只有当目标支持设置大小并且能够处理请求的大小时，此值才会与实际大小匹配。 例如，如果目标只能在运行之间更改缓冲区大小，此变量直到下一次运行开始才会反映更改。 使用 tstatus 获取实际缓冲区大小的报告。

set trace-user text ¶
show trace-user ¶
set trace-notes text ¶

设置跟踪运行的注释。

show trace-notes ¶

显示跟踪运行的注释。

set trace-stop-notes text ¶

设置跟踪运行的停止注释。 注释的处理方式与 tstop 参数相同；设置命令是修正错误或不完整的停止注释的便捷方法。

show trace-stop-notes ¶

显示跟踪运行的停止注释。
