10.11 值历史记录 ¶

由该 print 命令打印的值会保存在 GDB 值历史记录中。这允许你在其他表达式中引用它们。 值会一直保留，直到重新读取或丢弃符号表 （例如使用 file 或 symbol-file 命令）。 当符号表发生变化时，值历史记录会被丢弃， 因为这些值可能包含指向符号表中定义类型的指针。

打印出的值会被赋予 历史编号 ，你可以通过这些编号引用它们。这些编号是从 1 开始的连续整数。 print 通过在值前面打印 ‘$编号 = ’ 来显示分配给该值的历史编号；这里 编号 就是历史编号。

要引用任何之前的值，使用 ‘$’ 后跟该值的 历史编号。这种 print 标记其输出的方式是为了 提醒你这一点。仅仅 $ 引用历史记录中最近的值，并且 $$ 引用该值之前的值。 $$n 引用从末尾开始的第 n个值； $$2 是紧接在 $$, $$1 等同于 $$, 并且 $$0 等同于 $。

例如，假设你刚刚打印了一个指向结构体的指针，并且 想查看该结构体的内容。只需输入 

p *$

如果你有一个结构体链，其中成员 next 指向下一个结构体，你可以使用以下命令打印下一个结构体的内容： 

p *$.next

你可以通过重复此命令来打印链中的连续链接 —— 你只需输入 回车键即可。

请注意，历史记录记录的是值，而不是表达式。如果 x 的值为 4，并且你输入以下命令： 

print x
set x=5

那么由 print 命令记录在值历史记录中的值仍然是 4，即使 x 的值已经改变。

show values ¶

打印值历史记录中的最后十个值及其项编号。 这类似于重复执行十次 ‘p $$9’，除了 show values 不会改变历史记录。

show values n

打印以历史项编号 n为中心的十个历史值。

show values +

打印刚刚打印的值之后的十个历史值。如果没有更多 可用的值， show values + 将不显示任何内容。

按下 回车键 重复 show values n 与 ‘show values +’ 具有完全相同的效果。
