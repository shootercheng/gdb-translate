23.3.2.38 Python 中的寄存器 ¶

Python 代码可以从一个 gdb.Architecture 请求有关可用寄存器集的信息 （请参阅 Architecture.registers）。 寄存器信息作为一个 gdb.RegisterDescriptorIterator返回，它是一个迭代器，该迭代器依次返回 gdb.RegisterDescriptor 对象。

一个 gdb.RegisterDescriptor 不提供寄存器的值 （请参阅 Frame.read_register 以读取寄存器的值），相反， RegisterDescriptor 是一种发现特定架构可用寄存器的方式。

一个 gdb.RegisterDescriptor 具有以下只读属性：

变量： RegisterDescriptor.name ¶

此寄存器的名称。

也可以使用以下 gdb.RegisterDescriptorIterator 函数根据寄存器描述符的名称查找它：

函数： RegisterDescriptorIterator.find (名称) ¶

接受 名称 作为参数，该参数必须是字符串，并返回具有该名称的寄存器的 gdb.RegisterDescriptor ，或者如果没有具有该名称的寄存器，则返回 None 。

Python 代码还可以从一个 gdb.Architecture 请求有关给定架构上可用寄存器组集的信息 （请参阅 Architecture.register_groups）。

每个寄存器可以是零个或多个寄存器组的成员。 一些寄存器组在 GDB 内部用于控制诸如在调用被调试程序时必须保存哪些寄存器之类的事情 （请参阅 调用程序函数）。 其他寄存器组的存在是为了让用户在像 info registers 这样的命令中轻松查看相关的寄存器集 （请参阅 info registers reggroup）。

寄存器组信息作为一个 gdb.RegisterGroupsIterator返回，它是一个迭代器，该迭代器依次返回 gdb.RegisterGroup 对象。

一个 gdb.RegisterGroup 对象具有以下只读属性：

变量： RegisterGroup.name ¶

一个字符串，是此寄存器组的名称。
