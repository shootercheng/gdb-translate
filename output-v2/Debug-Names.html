18.6 ‘的扩展.debug_names’ ¶

DWARF规范记录了一个名为 ‘的可选索引节.debug_names’。 GDB 既可以读取也可以创建此 节。 但是，为了使用 GDB, 一些扩展是必要的。

GDB 使用一个增强字符串来指定正在使用哪些扩展，并支持此功能中向后不兼容的更改。 增强字符串的形式为 ‘GDBn’，其中 n 是扩展的整数版本号，当扩展被添加或修改时会递增。 最小的 n 是2；早期版本的 GDB 仅使用‘GDB’且没有版本号，但这些版本的索引不再受支持。

以下是增强字符串版本列表以及每个版本相对于上一个版本引入的更改。

‘GDB2’

指定使用属性 DW_IDX_GNU_internal, DW_IDX_GNU_main, DW_IDX_GNU_language 和 DW_IDX_GNU_linkage_name, 如下所述。

‘GDB3’

更改了 DW_IDX_parent 属性的语义。 使用‘GDB2’时， DW_IDX_parent 提供了到名称表的偏移量。 使用‘GDB3’时，它现在提供相对于条目池区域起始位置的父条目索引条目的偏移量。

GDB 生成的索引使用增强字符串‘GDB3’。

GDB 可以读取使用增强字符串‘GDB2’或 ‘GDB3’的索引。 GDB 不支持读取使用任何其他增强字符串的索引。

GDB 不使用指定的哈希表。 因此，由于此哈希表是可选的， GDB 也不会写入它。

GDB 生成并使用以下非标准索引属性：

DW_IDX_GNU_internal

其值为‘0x2000’。 这是一个标志，当设置时，表示关联的条目具有 static 链接属性。

DW_IDX_GNU_main

其值为‘0x2002’。 这是一个标志，当设置时，表示关联的条目是程序的 main.

DW_IDX_GNU_language

其值为‘0x2003’。 这是一个‘DW_LANG_’常量，表示关联条目的语言。

DW_IDX_GNU_linkage_name

其值为‘0x2004’。 这是一个标志，当设置时，表示关联的条目是链接名称，而不是源名称。
