系统 ¶

概要:
int system(const char *command);
请求:

‘F系统,命令指针/长度’

返回值:

如果 长度 为零，返回值表示是否有可用的 shell。返回值为零表示没有可用的 shell。对于非零的 长度, 出错时返回值为 -1，否则返回命令的返回状态。仅返回命令的退出状态，该状态是从主机的 system 返回值中通过调用 WEXITSTATUS(retval)提取的。在 /bin/sh 无法执行的情况下，返回 127。

错误:
EINTR

调用被用户中断。

GDB 接管了调用必要的主机调用以执行 system 调用的全部任务。 system 在主机上的返回值在返回给目标之前会被简化。子进程的任何终止信号信息都会被丢弃，返回值完全由被调用命令的退出状态组成。

由于安全考虑， system 调用默认情况下会被 GDB拒绝。用户必须使用 set remote system-call-allowed 1 命令显式允许此调用。

set remote system-call-allowed ¶

控制是否允许在远程目标的文件 I/O 协议中进行 system 调用。默认值为零（禁用）。

show remote system-call-allowed ¶

显示是否允许在文件 I/O 协议中进行 system 调用。
