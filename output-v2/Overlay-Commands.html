14.2 覆盖命令 ¶

要使用 GDB的覆盖支持，程序中的每个覆盖必须对应可执行文件的一个独立节。该节的虚拟内存地址和加载内存地址必须是覆盖的映射地址和加载地址。通过节来识别覆盖可以让 GDB 根据覆盖是否已映射来确定函数或变量的合适地址。

GDB的覆盖命令都以单词 overlay开头；你可以将其缩写为 ov 或 ovly。命令如下：

overlay off ¶

禁用 GDB的覆盖支持。当覆盖支持被禁用时， GDB 假定所有函数和变量始终位于其映射地址处。默认情况下， GDB的覆盖支持是禁用的。

overlay manual ¶

启用 手动 覆盖调试。在此模式下， GDB 依赖你使用下面描述的 overlay map-overlay 和 overlay unmap-overlay 命令来告知它哪些覆盖已映射，哪些未映射。

overlay map-overlay overlay ¶
overlay map overlay

告诉 GDB 覆盖 现在已映射； 覆盖 必须是包含该覆盖的目标文件节的名称。当一个覆盖被映射时， GDB 假定可以在其映射地址处找到该覆盖的函数和变量。 GDB 假定任何其他映射范围与 覆盖 重叠的覆盖现在未映射。

overlay unmap-overlay overlay ¶
overlay unmap overlay

告诉 GDB 覆盖 不再映射； 覆盖 必须是包含该覆盖的目标文件节的名称。当一个覆盖未映射时， GDB 假定可以在其加载地址处找到该覆盖的函数和变量。

overlay auto

启用 自动 覆盖调试。在此模式下， GDB 会查询下级程序中覆盖管理器维护的数据结构，以查看哪些覆盖已映射。详细信息，请参阅 自动覆盖调试。

overlay load-target ¶
overlay load

从下级程序重新读取覆盖表。通常情况下， GDB 会在每次下级程序停止时自动重新读取该表，因此只有在你自己使用 GDB 更改了覆盖映射时，此命令才是必要的。此命令仅在使用自动覆盖调试时有用。 GDB

overlay list-overlays ¶
overlay list

显示当前已映射的覆盖列表，以及它们的映射地址、加载地址和大小。

通常，当 GDB 打印代码地址时，它会包含该地址所在函数的名称： 

(gdb) print main
$3 = {int ()} 0x11a0 <main>

当启用覆盖调试时， GDB 可以识别未映射覆盖中的代码，并在未映射函数的名称周围加上星号进行打印。例如，如果 foo 是未映射覆盖中的一个函数， GDB 会这样打印它： 

(gdb) overlay list
No sections are mapped.
(gdb) print foo
$5 = {int (int)} 0x100000 <*foo*>

当 foo的覆盖被映射时， GDB 会正常打印函数的名称： 

(gdb) overlay list
Section .ov.foo.text, loaded at 0x100000 - 0x100034,
        mapped at 0x1016 - 0x104a
(gdb) print foo
$6 = {int (int)} 0x1016 <foo>

当启用覆盖调试时， GDB 可以找到覆盖中函数和变量的正确地址，无论该覆盖是否已映射。这使得大多数 GDB 命令，如 break 和 disassemble, 都能正常工作，即使是针对未映射的代码。然而， GDB的断点支持有一些限制：

  • 只要 GDB 可以在其加载地址处写入覆盖，就可以在未映射覆盖的函数中设置断点。
  • GDB 不能在未映射覆盖中设置基于硬件或模拟器的断点。但是，如果你在覆盖管理器的末尾设置一个断点（并且如果你使用手动覆盖管理，告知 GDB 哪些覆盖现在已映射）， GDB 将正确地重新设置其断点。
