4.9.1 特定下级断点 ¶

在调试多个下级程序时，你可以选择是为所有下级程序设置断点，还是为特定的下级程序设置断点。

break locspec inferior inferior-id ¶
break locspec inferior inferior-id if …

位置规范 指定程序中的一个或多个代码位置。请参阅 位置规范，了解详细信息。

使用限定符 ‘下级程序 下级程序 ID’ 与断点命令一起使用，以指定你只希望 GDB 当特定的下级程序到达此断点时停止。 下级程序 ID 指定符是由 GDB分配的下级程序标识符之一，在 ‘info inferiors’ 输出的第一列中显示。

如果你在设置断点时未指定 ‘下级程序 下级程序 ID’，则该断点适用于 程序的所有 下级程序。

你也可以在条件断点上使用该 inferior 限定符；在这种情况下，将 ‘下级程序 下级程序 ID’ 放在断点条件之前或之后，如下所示： 

(gdb) break frik.c:13 inferior 2 if bartab > lim

当相应的下级程序从 GDB中移除时，特定下级断点会自动删除。例如： 

(gdb) remove-inferiors 2
特定下级断点 3 已删除 - 下级程序 2 已被移除。

一个断点不能既是特定下级断点又是特定线程断点（请参阅 特定线程断点），也不能是特定任务断点（请参阅 Ada 任务扩展）；在创建断点时使用多个 inferior, thread, 或 task 关键字会导致错误。
