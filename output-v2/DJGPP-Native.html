21.1.3 调试功能 DJGPP 程序 ¶

DJGPP 是一个针对 GNU MS - DOS 和 MS - Windows 的开发工具移植版本。 DJGPP 程序是 32 位保护模式程序，它们使用 DPMI （DOS 保护模式接口）API 在实模式 DOS 系统及其模拟环境上运行。

GDB 支持对 DJGPP 程序进行原生调试，并定义了一些特定于该 DJGPP 移植版本的命令。本小节将介绍这些命令。

info dos ¶

这是 DJGPP特定命令的前缀，这些命令用于打印有关目标系统和重要操作系统结构的信息。

info dos sysinfo ¶

此命令显示底层平台的各种信息：CPU 类型和特性、操作系统版本和类型、DPMI 版本，以及可用的常规内存和 DPMI 内存。

info dos gdt ¶
info dos ldt
info dos idt

这 3 个命令分别显示全局描述符表（GDT）、局部描述符表（LDT）和中断描述符表（IDT）中的条目。描述符表是数据结构，用于存储当前正在使用的每个段的描述符。段的选择器是描述符表的索引；该索引对应的表项包含描述符的基地址和界限，以及其属性和访问权限。

典型的 DJGPP 程序使用 3 个段：代码段、数据段（用于数据和栈）和 DOS 段（允许访问 DOS/BIOS 数据结构和常规内存中的绝对地址）。然而，DPMI 主机通常会定义额外的段以支持 DPMI 环境。

这些命令允许显示描述符表中的条目。如果没有参数，则显示指定表中的所有条目。参数应该是一个整数表达式，表示显示索引由该参数指定的单个条目。例如，以下是一种方便的方法来显示有关被调试程序的数据段的信息： 

(gdb) info dos ldt $ds

0x13f: base=0x11970000 limit=0x0009ffff 32-Bit Data (Read/Write, Exp-up)

当你想查看指针是否超出数据段的界限（即 混乱的)时，这很有用。

info dos pde ¶
info dos pte

这两个命令分别显示页目录和页表中的条目。页目录和页表是控制虚拟内存地址如何映射到物理地址的数据结构。页表包含映射到程序地址空间的每个内存页的条目；可能有多个页表，每个页表最多可容纳 4096 个条目。页目录最多有 4096 个条目，每个条目对应一个当前正在使用的页表。

如果没有参数， info dos pde 显示整个页目录，而 info dos pte 显示所有页表中的所有条目。给 info dos pde 命令提供一个整数表达式作为参数，表示只显示页目录表中的该条目。给 info dos pte 命令提供参数表示显示由页目录中指定条目指向的单个页表中的条目。

当你的程序使用 DMA （直接内存访问）时，这些命令很有用，因为 DMA 需要物理地址来对 DMA 控制器进行编程。

这些命令仅在某些 DPMI 服务器中受支持。

info dos address-pte addr ¶

此命令显示指定线性地址的页表条目。参数 addr 是一个线性地址，该地址应该已经加上了相应段的基地址，因为此命令接受可能属于 任何 段的地址。例如，以下是如何显示存储变量 i 的页面的页表条目的方法： 

(gdb) info dos address-pte __djgpp_base_address + (char *)&i

Page Table entry for address 0x11a00d30:

Base=0x02698000 Dirty Acc. Not-Cached Write-Back Usr Read-Write +0xd30

这表明 i 存储在偏移量为 0xd30 的位置，该页面的物理基地址为 0x02698000, 并显示该页面的所有属性。

请注意，你必须将变量的地址强制转换为 char *, 否则 __djgpp_base_address, 即 DJGPP 程序中所有变量和函数的基地址，将根据 C 指针算术规则进行相加：如果 i 被声明为 int, GDB 将把 __djgpp_base_address 的值乘以 4 后加到 i.

这里还有另一个例子，它显示传输缓冲区的页表条目： 

(gdb) info dos address-pte *((unsigned *)&_go32_info_block + 3)

Page Table entry for address 0x29110:

Base=0x00029000 Dirty Acc. Not-Cached Write-Back Usr Read-Write +0x110

（偏移量是因为传输缓冲区的地址是 + 3 结构的第 3 个成员。）输出清楚地表明，这个 DPMI 服务器将常规内存中的地址进行 1:1 映射，即物理（ _go32_info_block 0x00029000 + 0x110)和线性（0x29110)地址是相同的。

此命令仅在某些 DPMI 服务器中受支持。

除了原生调试外，DJGPP 移植版本还支持通过串行数据链路进行远程调试。以下命令是 DJGPP 移植版本中特定于远程串行调试的命令。 GDB.

set com1base addr ¶

此命令设置 COM1 串行端口的基 I/O 端口地址。

set com1irq irq

此命令设置 中断请求 (IRQ)线路，用于 COM1 串行端口。

还有类似的命令‘set com2base’、‘set com3irq’等，用于设置其他 3 个 COM 端口的端口地址和 IRQ 线路。

相关命令‘show com1base’、‘显示 COM1 的中断请求号等。 显示 COM 端口使用的基地址和 IRQ 线路的当前设置。

info serial ¶

此命令打印 4 个 DOS 串行端口的状态。对于每个 端口，它会打印该端口是否处于活动状态、其 I/O 基地址和 中断请求号、是否使用 16550 风格的 FIFO、波特率，以及 到目前为止遇到的各种错误计数。
