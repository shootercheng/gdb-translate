23.3.2.17 Python 中的被调试程序 ¶

下运行的程序称为被调试程序 （参见 GDB 在 ）。Python 脚本可以通过 GDB 类的对象访问和操作由 gdb.Inferior 类。

以下与被调试程序相关的函数在 gdb 模块：

函数： gdb.inferiors () ¶

返回一个包含所有被调试程序对象的元组。

函数： gdb.selected_inferior () ¶

返回一个表示当前被调试程序的对象。

一个 gdb.Inferior 对象具有以下属性：

变量： Inferior.num ¶

被调试程序的 ID，由 GDB分配。例如，你可以使用此 ID 使 Python 断点特定于被调试程序（参见 Breakpoint.inferior 属性）。

变量： Inferior.connection ¶

此被调试程序的 gdb.TargetConnection （参见 Python 中的连接），或者如果此被调试程序没有连接则为 None 。

变量： Inferior.connection_num ¶

被调试程序连接的 ID，由 GDB分配，如果被调试程序未连接到目标则为 None。参见 调试多个被调试程序连接和程序。在 gdb.Inferior.connection.num 的情况下，这等同于 gdb.Inferior.connection 不是 None。

变量： Inferior.pid ¶

被调试程序的进程 ID，由底层操作系统分配。

变量： Inferior.was_attached ¶

布尔值，表示被调试程序是使用 ‘attach’ 创建的，还是由 GDB 自身启动的。

变量： Inferior.main_name ¶

一个字符串，保存此被调试程序的 “main” 函数的名称（如果可以确定）。如果不知道 main 的名称，则为 None。

变量： Inferior.progspace ¶

被调试程序的程序空间。参见 Python 中的程序空间。

变量： Inferior.arguments ¶

被调试程序的命令行参数（如果已知）。这对应于 set args 和 show args 命令。参见 你的程序的参数。

访问时，该值是一个包含所有参数的字符串。内容会像传递给 shell 时一样被引用。如果没有参数，该值为 None。

可以将字符串或字符串序列分配给此属性。当分配字符串时，假设它具有 shell 所需的任何引用；当分配序列时，引用由 GDB应用。

一个 gdb.Inferior 对象具有以下方法：

函数： Inferior.is_valid () ¶

如果 True 对象有效，则返回 gdb.Inferior False 如果不是。一个 gdb.Inferior 对象将变得无效 如果被调试程序在 GDB内不再存在。所有其他 gdb.Inferior 方法在调用时如果对象无效将抛出异常。

函数： Inferior.threads () ¶

此方法返回一个元组，其中包含调用时所有有效的线程。如果没有有效线程，该方法将返回一个空元组。

函数： Inferior.architecture () ¶

返回 gdb.Architecture （参见 架构的 Python 表示形式） 对于此被调试程序。这表示整个被调试程序的架构。某些平台在单个地址空间中可以有多种架构，因此这可能与特定帧的架构不匹配（参见 从 Python 访问被调试程序的栈帧）。

函数： Inferior.read_memory (address, length) ¶

从被调试程序中读取 长度 个可寻址的内存单元，从 地址开始。返回一个 memoryview 对象，其行为类似于数组或字符串。它可以被修改并传递给 Inferior.write_memory 函数。

函数： Inferior.write_memory (address, buffer [, length]) ¶

将 缓冲区 的内容写入被调试程序，从 地址开始。 缓冲区 参数必须是支持缓冲区协议的 Python 对象，即字符串、数组或从 Inferior.read_memory返回的对象。如果给定 长度 ，则确定要写入的 缓冲区 中可寻址内存单元的数量。

函数： Inferior.search_memory (address, length, pattern) ¶

从 地址 开始，使用 长度 和 模式中提供的搜索模式搜索被调试程序的内存区域。 模式 参数必须是支持缓冲区协议的 Python 对象，即字符串、数组或从 gdb.read_memory返回的对象。返回一个 Python Long ，其中包含找到模式的地址，如果 None 未找到模式则返回 None。

函数： Inferior.thread_from_handle (handle) ¶

返回与 句柄对应的线程对象，句柄是特定于线程库的数据结构，例如 pthread_t 对于 pthreads 库实现。

该函数 Inferior.thread_from_thread_handle 提供相同的功能，但不建议使用 Inferior.thread_from_thread_handle 。

可以通过以下方法从 Python 更改传递给被调试程序的环境。这些方法仅在被调试程序启动时生效 - 它们不会影响已经在执行的被调试程序。

函数： Inferior.clear_env () ¶

清除将传递给此被调试程序的当前环境变量。

函数： Inferior.set_env (name, value) ¶

将环境变量 名称 设置为指定的值。两个参数都必须是字符串。

函数： Inferior.unset_env (name) ¶

取消设置环境变量 名称。 名称 必须是字符串。

可以以通常的 Python 方式向 gdb.Inferior 对象添加任意属性。例如，如果需要对与被调试程序相关的一些额外记录进行操作，这将很有用。

在为新属性选择名称时，避免以小写字母开头； GDB 将来添加的属性将以小写字母开头。此外，避免以两个下划线字符开头的属性名称，因为这些可能与 Python 内置属性名称冲突。

在这个人为的示例中，我们记录被调试程序最后一次停止的时间： 

(gdb) python
import datetime

def thread_stopped(event):
    if event.inferior_thread is not None:
        thread = event.inferior_thread
    else:
        thread = gdb.selected_thread()
    inferior = thread.inferior
    inferior._last_stop_time = datetime.datetime.today()

gdb.events.stop.connect(thread_stopped)
