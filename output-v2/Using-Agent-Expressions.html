F.3 使用代理表达式 ¶

代理表达式可以通过几种不同的方式使用， GDB并且调试器可以根据需要生成不同的字节码序列。

一种可能性是在目标端而不是主机端进行表达式求值，例如用于条件跟踪点的条件判断。 在这种情况下， GDB 将源表达式编译成一个字节码序列，该序列只是从寄存器或内存中获取值，进行算术运算，并返回结果。

使用代理表达式的另一种方式是用于跟踪点数据收集。 GDB 为收集生成不同的字节码序列；除了进行计算的字节码之外， GDB 添加 trace 用于保存所使用的内存块的字节码。

  • 用户在程序代码中选择GDB应收集数据的跟踪点。
  • 用户指定在每个跟踪点要计算的表达式。 这些表达式可能表示内存中的对象，在这种情况下，当程序运行时，这些对象的内容会被记录下来；或者表示计算值，在这种情况下，值本身会被记录下来。
  • GDB将跟踪点及其关联的表达式传输到运行在调试目标上的GDB代理。
  • 代理安排在命中跟踪点时得到通知。
  • 当目标端的执行到达一个跟踪点时，代理会计算与该跟踪点关联的表达式，并记录结果值和内存范围。
  • 之后，当用户选择一个给定的跟踪事件并检查记录的对象和表达式值时，GDB会与代理通信，根据需要检索记录的数据以满足用户的请求。 如果用户要求查看其内容未被记录的对象，GDB会报告错误。
