10.14 寄存器 ¶

在表达式中，您可以将机器寄存器的内容作为变量引用，这些变量的名称以 ‘$’ 开头。不同的机器寄存器名称不同；使用 info registers 查看您的机器上使用的名称。

info registers ¶

打印除浮点和向量寄存器之外的所有寄存器的名称和值（在选定的栈帧中）。

info all-registers ¶

打印所有寄存器的名称和值，包括浮点和向量寄存器（在选定的栈帧中）。

info registers reggroup …

打印每个指定的 寄存器组的寄存器名称和值。 寄存器组 可以是 maint print reggroups 返回的任何组（请参阅 维护命令）。

info registers regname …

打印每个指定寄存器的 相对化 值 寄存器名称。如下文详细讨论的，寄存器值通常相对于选定的栈帧。 寄存器名称 可以是您使用的机器上有效的任何寄存器名称，带或不带开头的 ‘$’。

GDB 有四个“标准”寄存器名称，在大多数机器上（只要它们不与架构的寄存器规范助记符冲突）的表达式中都可用。寄存器名称 $pc 和 $sp 用于程序计数器寄存器和栈指针。 $fp 用于包含指向当前栈帧指针的寄存器， $ps 用于包含处理器状态的寄存器。例如，您可以使用 

p/x $pc

以十六进制打印程序计数器，或者使用 

x/i $pc

打印下一条要执行的指令，或者给栈指针加四12 使用 

set $sp += 4

只要可能，即使机器有不同的规范助记符，这四个标准寄存器名称在您的机器上仍然可用，只要不冲突。 info registers 命令显示规范名称。例如，在 SPARC 上， info registers 将处理器状态寄存器显示为 $psr 但您也可以将其称为 $ps；在基于 x86 的机器上 $ps 是 EFLAGS 寄存器的别名。

GDB 当以这种方式检查寄存器时，始终将普通寄存器的内容视为整数。有些机器有只能保存浮点值的特殊寄存器；这些寄存器被认为具有浮点值。没有办法将普通寄存器的内容作为浮点值引用（尽管您可以 打印 它作为浮点值，使用 ‘print/f $寄存器名称’）。

有些寄存器有不同的“原始”和“虚拟”数据格式。这意味着操作系统保存寄存器内容的数据格式与您的程序通常看到的格式不同。例如，68881 浮点协处理器的寄存器总是以“扩展”（原始）格式保存，但所有 C 程序都期望使用“双精度”（虚拟）格式。在这种情况下， GDB 通常只处理虚拟格式（对您的程序有意义的格式），但 info registers 命令以两种格式打印数据。

有些机器有特殊寄存器，其内容可以有几种不同的解释方式。例如，现代基于 x86 的机器有 SSE 和 MMX 寄存器，可以以几种不同的格式一起保存多个值。 GDB 以 struct 表示法引用此类寄存器： 

(gdb) print $xmm1
$1 = {
  v4_float = {0, 3.43859137e-038, 1.54142831e-044, 1.821688e-044},
  v2_double = {9.92129282474342e-303, 2.7585945287983262e-313},
  v16_int8 = "\000\000\000\000\3706;\001\v\000\000\000\r\000\000",
  v8_int16 = {0, 0, 14072, 315, 11, 0, 13, 0},
  v4_int32 = {0, 20657912, 11, 13},
  v2_int64 = {88725056443645952, 55834574859},
  uint128 = 0x0000000d0000000b013b36f800000000
}

要设置此类寄存器的值，您需要告诉 GDB 您希望更改寄存器的哪个视图，就好像您在为一个 struct 成员赋值一样： 

 (gdb) set $xmm1.uint128 = 0x000000000000000000000000FFFFFFFF

通常，寄存器值相对于选定的栈帧（请参阅 选择帧）。这意味着如果退出所有更深层的栈帧并恢复其保存的寄存器，您将获得寄存器将包含的值。为了查看硬件寄存器的真实内容，您必须选择最内层的帧（使用 ‘frame 0’）。

通常，ABI 会保留一些寄存器，被调用者无需保存这些寄存器（也称为“调用者保存”、“调用破坏”或“易失性”寄存器）。因此，如果寄存器的值此后被被调用者更改， GDB 可能无法知道调用之前（换句话说，在外层帧中）寄存器的值。 GDB 会尝试从调试信息、展开信息或编译器生成的机器代码中推断内层帧保存（“被调用者保存”）寄存器的位置。如果某个寄存器未被保存，并且 GDB 知道该寄存器是“调用者保存”的（通过其自身对 ABI 的了解，或者因为调试/展开信息明确表示寄存器的值未定义）， GDB 会显示 ‘<未保存>’ 作为寄存器的值。对于 GDB 不了解寄存器保存约定的目标，如果被调用者未保存某个寄存器，则假设其在外层帧中的值和位置与内层帧相同。这通常没有问题，因为如果寄存器是调用破坏的，调用者要么不关心调用后寄存器中的内容，要么有代码来恢复它关心的值。但是请注意，如果您在外层帧中更改这样的寄存器，您可能也会影响内层帧。此外，您查看的帧越“外层”，调用破坏的寄存器的值就越有可能是错误的，也就是说，它实际上并不代表调用之前寄存器的值。

脚注

(12)

在内存中栈向下增长的机器（现在大多数机器都是这样）上，这是一种从栈中移除一个字的方法。这假设选择了最内层的栈帧；当选择其他栈帧时，不允许设置 $sp 。要从栈中弹出整个帧，无论机器架构如何，使用 return；请参阅 从函数返回。
