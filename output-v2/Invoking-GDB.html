2.1调用 GDB ¶

调用 GDB 通过运行程序 gdb。一旦启动， GDB 它会从终端读取命令，直到你告诉它退出。

你也可以运行 gdb 并带有各种参数和选项，以便在一开始就指定更多的调试环境。

这里描述的命令行选项旨在涵盖各种情况；在某些环境中，其中一些选项可能实际上不可用。

最常见的启动方式 GDB 是使用一个参数， 

指定一个可执行程序： gdb

程序 

你也可以同时指定一个可执行程序和一个核心文件来启动： gdb 程序

核心 -p如果你想调试一个正在运行的进程，你可以改为将进程ID作为第二个参数指定或使用选项 

gdb 程序 1234
 gdb -p 1234

将附加到 GDB 进程 1234。使用选项 -p 你可以省略 程序 文件名。

利用第二个命令行参数需要一个相当完整的操作系统；当你使用 GDB 作为连接到裸板的远程调试器时，可能没有“进程”的概念，而且通常无法获取核心转储。 GDB 如果它无法附加或读取核心转储，它会警告你。

你可以选择让 gdb 使用 --args将可执行文件之后的任何参数传递给被调试程序。此选项停止选项处理。 

gdb --args gcc -O2 -c foo.c

这将导致 gdb 调试 gcc，并将 gcc的命令行参数（见 你的程序的参数)设置为‘-O2 -c foo.c’。

你可以运行 gdb 而不打印描述 GDB的无保修声明的开头材料，通过指定 --silent （或 -q/--quiet)： 

gdb --silent

你可以通过使用命令行选项进一步控制 GDB 如何启动。 GDB 它本身可以提醒你可用的选项。

输入 

gdb -help

以显示所有可用选项并简要描述它们的用法（‘gdb -h’是等效的简短形式）。

你给出的所有选项和命令行参数都按顺序处理。当使用-x’选项时，顺序很重要。
