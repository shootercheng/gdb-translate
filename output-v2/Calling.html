17.5 调用程序函数 ¶

print expr ¶

计算表达式 expr 并显示结果值。 该表达式可能包括对正在调试的程序中的函数的调用。

call expr ¶

计算表达式 expr 不显示 void 返回值。

如果你想执行程序中不返回任何值的函数（也就是 print 命令的这个变体。 一个无返回值函数），但又不想让 void 返回值 GDB 打印出来而使输出混乱。如果结果不是无返回值，它将被打印并保存在值历史记录中。

通过 print 或 call 命令调用的函数可能会产生信号（例如，如果函数中有错误，或者你传递了不正确的参数）。在这种情况下会发生什么由 set unwind-on-signal 命令控制。

同样，对于C++ 程序，通过 print 或 call 命令调用的函数可能会产生一个由于虚拟帧的限制而未被处理的异常。在这种情况下，在帧中抛出但有帧外异常处理程序的任何异常都将无法被找到。GDB为被调试函数调用构建一个虚拟帧，并且展开器无法在这个虚拟帧之外寻找异常处理程序。在这种情况下会发生什么由 set unwind-on-terminating-exception 命令控制。

17.5.1 调用无调试信息的函数 ¶

有时，你想调用的函数缺少调试信息。在这种情况下， GDB 不知道该函数的类型，包括函数参数的类型。为了避免错误地调用被调试函数，这可能导致被调用函数功能异常甚至崩溃， GDB 拒绝调用该函数，除非你告诉它函数的类型。

对于有原型声明的（即ANSI/ISO风格）函数，有两种方法可以做到这一点。最简单的方法是将函数调用强制转换为其声明的返回类型。例如： 

(gdb) p getenv ("PATH")
'getenv' 具有未知的返回类型；将调用强制转换为其声明的返回类型
(gdb) p (char *) getenv ("PATH")
$1 = 0x7fffffffe7ba "/usr/local/bin:/"...

对无调试信息的函数的返回类型进行强制类型转换，等同于将该函数强制转换为一个指向具有与传入参数类型匹配的原型的函数指针，并调用该指针。即，上面的调用等同于： 

(gdb) p ((char * (*) (const char *)) getenv) ("PATH")

假设有一个带有浮点参数的有原型声明的C或C++函数： 

float multiply (float v1, float v2) { return v1 * v2; }

以下调用是等效的： 

(gdb) p (float) multiply (2.0f, 3.0f)
(gdb) p ((float (*) (float, float)) multiply) (2.0f, 3.0f)

如果你想调用的函数被声明为无原型（即旧的K&R风格），你必须使用强制转换为函数指针的语法，这样 GDB 就知道需要应用默认的参数提升（将浮点参数提升为双精度）。见 浮点提升。例如，给定这个带有浮点参数且无调试信息的无原型C函数： 

float
multiply_noproto (v1, v2)
  float v1, v2;
{
  return v1 * v2;
}

你可以这样调用它： 

  (gdb) p ((float (*) ()) multiply_noproto) (2.0f, 3.0f)
