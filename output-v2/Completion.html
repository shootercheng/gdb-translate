3.3 命令补全

GDB 如果只有一种可能性，它可以为你补全命令中的剩余单词；它还可以随时向你显示命令中下一个单词的有效可能性。 这适用于 GDB GDB 子命令、命令选项以及你程序中的符号名称。

每当你想 制表符 键。 如果只有一种可能性 GDB 来补全单词的剩余部分。 如果只有一种可能性， GDB 补全该单词，并等待你完成命令（或按下 回车键 来输入它）。 例如，如果你输入 

(gdb) info bre制表符

GDB 补全单词‘breakpoints’，因为这是唯一 info 以‘bre’开头的子命令： 

(gdb) info breakpoints

此时你可以按下 回车键 来运行 info breakpoints 命令，或者按退格键并输入其他内容，如果‘breakpoints’看起来不是你期望的命令。 （如果你一开始就确定想要 info breakpoints ，你不妨在‘ 回车键 立即在‘info bre’之后按下，以利用命令缩写而不是命令补全）。

如果你按下 制表符时，下一个单词有多种可能性 GDB 会发出提示音。 你可以输入更多字符并再次尝试，或者再次按下 制表符 ； GDB 显示该单词的所有可能补全项。 例如，你可能想在一个以‘make_’开头的子例程上设置断点，但当你输入 b make_制表符 GDB 只会发出提示音。 再次输入 制表符 会显示你程序中所有以这些字符开头的函数名，例如： 

(gdb) b make_制表符

GDB 发出提示音；再次按下 制表符 ，以查看：

make_a_section_from_file     make_environ
make_abs_section             make_function_type
make_blockvector             make_pointer_type
make_cleanup                 make_reference_type
make_command                 make_symbol_completion_list
(gdb) b make_

显示可用可能性后 GDB 复制你的部分输入（在示例中为‘b make_’），以便你完成命令。

如果你要补全的命令期望后面跟一个关键字或一个数字，那么‘数字’将在可用补全项中显示，例如： 

(gdb) print -elements 制表符制表符
数字     无限制
(gdb) print -elements 

这里，该选项期望一个数字（例如 100），而不是字面的 NUMBER。 这种元语法参数总是以大写形式呈现。

如果你一开始就只想查看替代项列表，你可以按下 M-? 而不是按两次 制表符 。 M-? 表示 元键 ?。 你可以通过按住键盘上指定为 元键 的键（如果有的话）同时输入 ?, 或者先按 ESC 再按 ?来输入它。

如果可能的补全项数量很大 GDB 将打印它已收集的尽可能多的列表，并显示一条消息，表明列表可能被截断。 

(gdb) b m制表符制表符
main
<... 其余可能的补全项 ...>
*** 列表可能被截断，已达到最大补全数量。 ***
(gdb) b m

此行为可以通过以下命令进行控制：

set max-completions limit ¶
set max-completions unlimited

设置补全候选的最大数量。 GDB 一旦收集到这么多候选，将停止寻找更多补全项。 当对函数名等内容进行补全时，这很有用，因为收集所有可能的候选可能会很耗时。 默认值为 200。值为零将禁用制表符补全。 请注意，设置无限制或非常大的限制可能会使补全变慢。

show max-completions

显示在补全期间 GDB 将收集并显示的最大候选数量。

有时你需要的字符串，虽然在逻辑上是一个“单词”，但可能包含括号或其他字符， GDB 这些字符通常会从其对单词的定义中排除。 为了允许在这种情况下进行单词补全，你可以将单词用 ' （单引号）括起来，在 GDB 命令中使用。

你可能需要这样做的一种常见情况是在输入涉及 C 的表达式时++ 带有模板参数的符号名。 这是因为在补全表达式时，GDB 将‘<’字符视为单词分隔符，假设它是小于比较运算符（请参阅 C 和 C++ 运算符）。

例如，当你想使用++ 模板函数以交互方式调用 C print 或 call 命令时，你可能需要区分你指的是针对 name 进行特化的版本 int, name<int>(), 还是针对 float, 进行特化的版本 name<float>()。 为了在这种情况下使用单词补全功能，在函数名的开头输入一个单引号 ' 这会提醒 GDB 当你按下 制表符 或 M-? 请求单词补全时，可能需要考虑比平时更多的信息： 

(gdb) p 'func<M-?
func<int>()    func<float>()
(gdb) p 'func<

然而，在设置断点时（请参阅 位置规范），你通常不需要在函数名前输入引号，因为 GDB 理解你想在一个函数上设置断点： 

(gdb) b func<M-?
func<int>()    func<float>()
(gdb) b func<

即使在输入 C 的重载函数（同一函数的多个定义，通过参数类型区分）的名称时也是如此。 例如，当你想设置断点时，你不需要区分你指的是接受++ 的版本 name 一个 int 参数的版本 name(int), 还是接受 float 参数的版本 name(float)。 

(gdb) b bubble(M-?
bubble(int)    bubble(double)
(gdb) b bubble(douM-?
bubble(double)

请参阅 引用名称 以了解其他需要引用的场景的描述。

有关重载函数的更多信息，请参阅 C++ 表达式。 你可以使用命令 set overload-resolution off 来禁用重载解析；请参阅 GDB C 的特性++。

当在查找结构体字段的表达式中进行补全时 GDB 也会尝试3 将补全限制为左侧类型可用的字段名称时，按下 

(gdb) p gdb_stdout.M-?
magic                to_fputs             to_rewind
to_data              to_isatty            to_write
to_delete            to_put               to_write_async_safe
to_flush             to_read

这是因为 gdb_stdout 是一个类型为 struct ui_file 的变量，该类型在 GDB 源文件中定义如下： 

struct ui_file
{
   int *magic;
   ui_file_flush_ftype *to_flush;
   ui_file_write_ftype *to_write;
   ui_file_write_async_safe_ftype *to_write_async_safe;
   ui_file_fputs_ftype *to_fputs;
   ui_file_read_ftype *to_read;
   ui_file_delete_ftype *to_delete;
   ui_file_isatty_ftype *to_isatty;
   ui_file_rewind_ftype *to_rewind;
   ui_file_put_ftype *to_put;
   void *to_data;
}

脚注

(3)

补全器可能会被某些无效表达式弄混淆。 此外，它只检查表达式的静态类型，而不是动态类型。
