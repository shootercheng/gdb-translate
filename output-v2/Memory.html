n, 重复计数

重复计数是一个十进制整数；默认值为 1。它指定要显示多少内存（按单位 u计数）。如果指定一个负数，则从 addr向后检查内存。

f, 显示格式

显示格式是 print 使用的格式之一（‘x’、‘d’、‘u’、‘o’、‘t’、‘a’、‘c’、‘f’、‘s’）、‘i’（用于机器指令）和 ‘m’（用于显示内存标记）。初始默认值是 ‘x’（十六进制）。每次使用 x 或 print, 默认值都会改变。

即使对于格式 ‘ u s’ 和 ‘i’ 忽略了单位大小 n，您可能仍然想使用一个计数；例如，‘3i’ 指定您想查看三条机器指令，包括任何操作数。为了方便起见，特别是在与 display 命令一起使用时，‘i’ 格式还会打印分支延迟槽指令（如果有的话），超出指定的计数，这些指令紧跟在计数范围内的最后一条指令之后。 disassemble 命令提供了一种检查机器指令的替代方法；请参阅 源代码和机器代码。

如果为格式 ‘s’ 或 ‘i’ 指定了负的重复计数，该命令将显示给定地址之前的以空字符结尾的字符串或指令，数量为给定数字的绝对值。对于 ‘i’ 格式，我们在反汇编时使用调试信息中的行号信息来准确确定指令边界。如果没有行号信息，该命令将显示错误消息并停止检查内存。

所有 x 参数的默认值都设计为每次使用 x时，以最少的规格轻松继续扫描内存。例如，在您使用 ‘x/3i addr’ 检查了三条机器指令之后，您可以仅使用 ‘x/7’ 检查接下来的七条。如果您使用 RET 来重复 x 命令，重复计数 n 将再次使用；其他参数的默认值与连续使用 x时相同。

如果架构支持内存标记，可以使用 ‘m’ 来显示标记。请参阅 内存标记。

信息将按颗粒大小（特定内存标记覆盖的字节数）显示一次。例如，AArch64 的颗粒大小为 16 字节，因此它将每 16 字节显示一个标记。

由于 GDB 命令打印信息的方式（未对齐到特定边界），标记信息将引用特定行上显示的初始地址。如果 x x 命令显示的一行中间跨越了内存标记边界，它将在下一行显示。

‘m’ 格式不会影响传递给 x 命令的任何其他指定格式。

命令打印的地址和内容不会保存在值历史记录中，因为通常有太多内容，会造成干扰。相反， x GDB 使这些值可以作为便利变量 $_ 和 $__的值在后续表达式中使用。在执行一次 x 命令后，最后检查的地址可以在便利变量 $_中用于表达式。该地址的检查内容可以在便利变量 $__中找到。

如果 x 命令有重复计数，保存的地址和内容是最后打印的内存单元的；如果最后一行输出打印了多个单元，这与最后打印的地址不同。

大多数目标的可寻址内存单元大小为 8 位。这意味着每个内存地址关联 8 位数据。然而，一些目标有其他可寻址内存单元大小。在 GDB 和本文档中，术语 可寻址内存单元 （或 内存单元 简称）在明确引用该大小的数据块时使用。单词 字节 用于指 8 位的数据块，无论目标的可寻址内存单元大小如何。对于大多数系统，可寻址内存单元是字节的同义词。

当您调试在远程目标机器上运行的程序时（请参阅 调试远程程序），您可能希望将远程机器内存中的程序映像与您下载到目标的可执行文件进行验证。或者，在任何目标上，您可能想检查程序是否损坏了其自己的只读段。为此提供了 compare-sections 命令。

compare-sections [section-name|-r] ¶

比较正在调试的程序的可执行文件中的可加载段 段名 与目标机器内存中相同段的数据，并报告任何不匹配之处。不带参数时，比较所有可加载段。参数为 -r时，比较所有可加载的只读段。

注意：对于远程目标，如果目标支持计算内存块的 CRC 校验和（请参阅 qCRC 数据包），此命令可以加速。
