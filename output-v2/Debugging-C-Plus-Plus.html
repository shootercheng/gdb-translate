15.4.1.7 GDB C 的特性++ ¶

一些 GDB 命令在处理 C 时特别有用，还有一些是专门为 C 设计的。 以下是一个总结：++, 并且有些是专门为与 C 一起使用而设计的。++以下是一个总结：

breakpoint menus ¶

当你想在一个名称重载的函数中设置断点时， GDB 有能力显示一个可能的断点位置菜单，以帮助你指定你想要的函数定义。 请参阅 模糊表达式.

rbreak regex ¶

使用正则表达式设置断点有助于在不属于任何特殊类的重载函数上设置断点。 请参阅 设置断点.

catch throw ¶
catch rethrow
catch catch

调试 C++ 使用这些命令处理异常。 请参阅 设置捕获点.

ptype typename ¶

打印类型的继承关系以及其他信息 类型名. 请参阅 检查符号表.

info vtbl expression.

该 info vtbl 命令可用于显示由 表达式计算出的对象的虚方法表。 这会为每个虚表显示一个条目；当使用多重继承时，可能会有多个虚表。当使用多重继承时，可能会有多个虚表。

demangle name ¶

反修饰 名称. 请参阅 检查符号表, 以获取关于 demangle 命令的更完整描述。

set print demangle ¶
show print demangle
set print asm-demangle
show print asm-demangle

控制 C++ 符号是否以其源代码形式显示，无论是在将代码显示为 C++ 源代码还是在显示反汇编时。 请参阅 打印设置.

set print object
show print object

选择打印对象的派生（实际）类型还是声明类型。 请参阅 打印设置.

set print vtbl
show print vtbl

控制打印虚函数表的格式。 请参阅 打印设置. ( vtbl 命令在使用 HP ANSI C++ 编译器编译的程序上不起作用（aCC).)

set overload-resolution on ¶

启用 C++ 表达式求值的重载解析。 默认是启用的。 对于重载函数， GDB 会计算参数并使用标准 C++ 转换规则（请参阅 C++ 表达式, 了解详细信息）搜索签名与参数类型匹配的函数。 如果找不到匹配项，它会发出一条消息。

set overload-resolution off

禁用 C++ 表达式求值的重载解析。 对于非类成员函数的重载函数， GDB 会选择符号表中找到的第一个指定名称的函数，无论其参数类型是否正确。 对于类成员函数的重载函数， GDB 会搜索签名 完全 匹配参数类型的函数。

show overload-resolution ¶

显示当前的重载解析设置。

Overloaded symbol names

你可以使用在 C 中声明此类符号的相同表示法指定重载符号的特定定义：++类型 symbol(types) 而不仅仅是 符号. 你还可以使用 GDB 命令行单词补全功能列出可用选项，或为你完成类型列表。 请参阅 命令补全, 了解如何执行此操作的详细信息。

Breakpoints in template functions

与处理重载符号的方式类似， GDB 在遇到包含 C++ 模板的符号时，会忽略模板参数列表。 这允许在模板函数族或参数包含模板类型的函数上设置断点。

该 限定 标志可用于覆盖此行为，使 GDB 搜索特定的函数或类型。

该 GDB 命令行单词补全功能也理解模板参数，可用于列出可用选项或为你完成模板参数列表。 请参阅 命令补全, 了解如何执行此操作的详细信息。

Breakpoints in functions with ABI tags

GNU C++ 编译器引入了 ABI “标签” 的概念，这些标签对应于类型、函数或变量的 ABI 变化，而这些变化在修饰名中不会体现。 请参阅 https://developers.redhat.com/blog/2015/02/05/gcc5-and-the-c11-abi/ 以获取更多详细信息。

ABI 标签在 C++ 反修饰名中可见。 例如，一个返回 std::string 的函数： 

std::string function(int);

当为 C++11 ABI 编译时，会标记有 cxx11 ABI 标签，并且 GDB 符号显示如下： 

function[abi:cxx11](int)

你可以像没有标签一样简单地在这类函数上设置断点。 例如： 

(gdb) b function(int)
断点 2 在 0x40060d: 文件 main.cc，第 10 行。
(gdb) info breakpoints
编号    类型          处置 启用 地址     位置
1       断点         保留 y  0x0040060d 在 function[abi:cxx11](int)
                                           于 main.cc:10

在极少数情况下，你需要区分不同的 ABI 标签，你可以通过在函数名中包含 ABI 标签来实现，例如： 

(gdb) b ambiguous[abi:other_tag](int)
