19.2 管理目标的命令 ¶

target type parameters

将 GDB 来指定目标机器的类型或协议。

此外 参数 由目标协议解释，但通常包括诸如要连接的设备名称或主机名、进程号和波特率等内容。

该 target 命令在执行后，如果您按下 回车 不会重复执行。

help target ¶

显示所有可用目标的名称。要显示当前选择的目标，请使用 info target 或 info files （请参阅 指定文件的命令）。

help target name

描述特定目标，包括选择它所需的任何参数。

set gnutarget args ¶

GDB 使用自己的库 BFD 来读取您的文件。 GDB 知道它正在读取的是 可执行文件, 一个 核心转储文件, 还是一个 .o 文件；但是，您可以使用 set gnutarget 命令指定文件格式。与大多数 target 命令不同，使用 gnutarget 时， target 指的是一个程序，而不是一台机器。

警告： 要使用 set gnutarget指定文件格式，您必须知道实际的 BFD 名称。

请参阅 指定文件的命令。

show gnutarget ¶

使用 show gnutarget 命令显示 gnutarget 设置为读取的文件格式。如果您没有设置 gnutarget, GDB 将自动确定每个文件的文件格式，并且 show gnutarget 显示 ‘当前 BFD 目标是 "自动"’。

以下是一些常见目标（根据 GDB 配置情况，可能可用也可能不可用）：

target exec program ¶

一个可执行文件。 ‘目标执行 程序’ 与 ‘执行文件 程序’ 相同。

该 程序 参数支持转义和引号，请参阅 作为命令参数的文件名。

target core filename ¶

一个核心转储文件。 ‘目标核心 文件名’ 与 ‘核心文件 文件名’ 相同。

该 文件名 参数支持转义和引号，请参阅 作为命令参数的文件名。

target remote medium ¶

通过串行线路或网络连接到 GDB 的远程系统。此命令告诉 GDB 使用自己的远程协议通过 介质 进行调试。请参阅 调试远程程序。

例如，如果您有一个连接到 /dev/ttya 运行 GDB的机器上的板卡，您可以这样说： 

目标远程 /dev/ttya

target remote 支持 load 命令。仅当您有其他方法将存根传输到目标系统，并且可以将其放置在内存中不会被下载操作覆盖的位置时，此命令才有用。

target sim [simargs] … ¶

内置 CPU 模拟器。 GDB 包含大多数架构的模拟器。一般来说， 

        目标模拟
        加载
        运行

可行；但是，您不能假定特定的内存映射、设备驱动程序甚至基本 I/O 可用，尽管有些模拟器确实提供了这些。有关任何特定处理器模拟器的详细信息，请参阅 嵌入式 处理器。

target native ¶

本地/原生进程调试的设置。有助于使 run 命令生成原生进程（同样， attach, 等等），即使 set auto-connect-native-target 为 off （请参阅 设置自动连接原生目标）。

不同配置的 GDB上可用的目标不同；您的配置可能有更多或更少的目标。

许多远程目标要求您在成功建立连接后下载可执行文件的代码。您可能希望控制此过程的各个方面。

set hash ¶

此命令控制在将文件下载到远程监视器时是否显示井号 ‘#’。如果开启，每个 S 记录成功下载到监视器后会显示一个井号。

show hash ¶

显示当前井号显示状态。

set debug monitor ¶

启用或禁用 GDB 与远程监视器之间通信消息的显示。

show debug monitor ¶

显示当前 GDB 与远程监视器之间通信的显示状态。

load filename offset ¶

根据配置到 GDB中的远程调试功能， load 命令可能可用。如果存在，它旨在使 文件名 （一个可执行文件）在远程系统上可用于调试 —— 例如通过下载或动态链接。 load 还会记录 文件名 符号表连接到 GDB中，就像 add-symbol-file 命令一样。

如果您的 GDB 没有 load 命令，尝试执行它会得到错误消息 “You can't do that when your target is …”

文件将加载到可执行文件中指定的任何地址。对于某些目标文件格式，您可以在链接程序时指定加载地址；对于其他格式，如 a.out，目标文件格式指定了一个固定地址。

也可以告诉 GDB 将可执行文件加载到由可选参数 偏移量描述的特定偏移处。当 偏移量 提供时， 文件名 也必须提供。

根据远程端的功能， GDB 可能能够将程序加载到闪存中。

load 如果您按下 回车 再次使用它后不会重复执行。

flash-erase ¶

擦除目标上所有已知的闪存区域。
