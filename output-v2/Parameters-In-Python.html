23.3.2.24 Python 中的参数 ¶

你可以使用 Python 实现新的 GDB 参数。新参数作为 gdb.Parameter 类的实例来实现。

参数通过 set 和 show 命令向用户公开。请参阅 获取帮助.

已经存在许多可以设置的参数。 GDB. 两个示例是： set follow fork 和 set charset. 设置这些参数会影响 GDB. 同样，你可以定义可用于影响自定义 Python 脚本和命令行为的参数。

函数： Parameter.__init__ (name, command_class, parameter_class [, enum_sequence]) ¶

的对象初始化器 Parameter 向 GDB注册新参数。此初始化器通常从子类自己的 __init__ 方法中调用。

name 是新参数的名称。如果 name 由多个单词组成，那么会查找初始单词作为前缀参数。可以用一组参数来说明这一点。如果 set print 参数集。如果 name 是 print foo, 那么 print 将作为前缀参数进行搜索。在这种情况下，该参数随后可以在 GDB 中作为 set print foo访问。

如果 name 由多个单词组成，并且找不到前缀参数组，则会抛出异常。

command_class 应该是 ‘COMMAND_’ 常量之一（请参阅 Python 中的 CLI 命令）。此参数告诉 GDB 如何在帮助系统中对新参数进行分类。

parameter_class 应该是下面定义的 ‘PARAM_’ 常量之一。此参数告诉 GDB 新参数的类型；此信息用于输入验证和自动补全。

如果 parameter_class 为 PARAM_ENUM, 那么 enum_sequence 必须是一个字符串序列。这些字符串表示参数的可能值。

如果 parameter_class 不为 PARAM_ENUM, 那么存在第四个参数将导致抛出异常。

新参数的帮助文本包括参数类的 Python 文档字符串（如果有的话）。如果没有文档字符串，则使用默认值。文档字符串包含在参数的 help set 和 help show 命令的输出中，编写时应考虑到这一点。如果参数类的文档字符串为空，则 GDB 将仅使用 Parameter.set_doc 或 Parameter.show_doc （见下文）在 帮助 输出中。

变量： Parameter.set_doc ¶

如果此属性存在且是一个字符串，那么其值将用作此参数的 set 命令帮助文本的第一部分。帮助文本的第二部分取自参数类的文档字符串（如果有的话）。

的值 set_doc 应该给出特定于设置操作的简要摘要，只有当用户运行此参数的 help set 命令时才会显示此文本。类文档应用于更全面地描述参数的作用，此文本会在 help set 和 help show 命令中显示。

当 set_doc 被调用时，会检查 Parameter.__init__ 的值；后续的更改没有效果。

变量： Parameter.show_doc ¶

如果此属性存在且是一个字符串，那么其值将用作此参数的 show 命令。帮助文本的第二部分取自参数类的文档字符串（如果有的话）。

的值 show_doc 应该给出特定于显示操作的简要摘要，只有当用户运行此参数的 help show 命令时才会显示此文本。类文档应用于更全面地描述参数的作用，此文本会在 help set 和 help show 命令中显示。

当 show_doc 被调用时，会检查 Parameter.__init__ 的值；后续的更改没有效果。

变量： Parameter.value ¶

该 value 属性保存参数的底层值。它可以像任何其他属性一样被读取和赋值。 GDB 在进行赋值时会进行验证。

任何 Parameter 类中可能实现两种方法。它们是：

函数： Parameter.get_set_string (self) ¶

如果此方法存在， GDB 在通过 参数的 set API（例如， set foo off）更改参数值时会调用它。 value 属性已经用新值填充，可用于输出。此方法必须返回一个字符串。如果返回的字符串不为空， GDB 会将其呈现给用户。

如果此方法抛出 gdb.GdbError 异常（请参阅 异常处理），那么 GDB 会打印异常字符串，并且 set 命令将失败。但是请注意，在这种情况下， value 属性不会被重置。因此，如果你的参数必须验证值，它应该在内部存储旧值并重置公开的值，如下所示： 

class ExampleParam (gdb.Parameter):
   def __init__ (self, name):
      super (ExampleParam, self).__init__ (name,
                   gdb.COMMAND_DATA,
                   gdb.PARAM_BOOLEAN)
      self.value = True
      self.saved_value = True
   def validate(self):
      return False
   def get_set_string (self):
      if not self.validate():
        self.value = self.saved_value
        raise gdb.GdbError('Failed to validate')
      self.saved_value = self.value
      return ""
函数： Parameter.get_show_string (self, svalue) ¶

GDB 在调用 参数的 show API（例如， show foo）时会调用此方法。参数 svalue 接收当前值的字符串表示形式。此方法必须返回一个字符串。

定义新参数时，必须指定其类型。可用的类型由 gdb 模块中定义的常量表示：

gdb.PARAM_BOOLEAN ¶

值是一个普通布尔值。Python 布尔值 True 和 False 是唯一有效的值。

gdb.PARAM_AUTO_BOOLEAN ¶

值有三种可能的状态：真、假和 ‘自动’。在 Python 中，真和假用布尔常量表示，‘自动’ 用 None表示。

gdb.PARAM_UINTEGER ¶

值是一个无符号整数。 None 的值应解释为 “无限制”（也可以使用字面量 'unlimited' 来设置该值），并且值 0 是保留的，不应使用。

gdb.PARAM_INTEGER ¶

值是一个有符号整数。 None 的值应解释为 “无限制”（也可以使用字面量 'unlimited' 来设置该值），并且值 0 是保留的，不应使用。

gdb.PARAM_STRING ¶

值是一个字符串。当用户修改字符串时，任何转义序列，如 ‘\t’、‘\f’ 和八进制转义，都会被转换为相应的字符并编码为当前主机字符集。

gdb.PARAM_STRING_NOESCAPE ¶

值是一个字符串。当用户修改字符串时，转义字符会原样传递，不进行转换。

gdb.PARAM_OPTIONAL_FILENAME ¶

值要么是一个文件名（字符串），要么是 None。

gdb.PARAM_FILENAME ¶

值是一个文件名。这与 PARAM_STRING_NOESCAPE类似，但使用文件名进行自动补全。

gdb.PARAM_ZINTEGER ¶

值是一个有符号整数。这与 PARAM_INTEGER类似，不同之处在于允许值为 0，并且不支持 None 的值。

gdb.PARAM_ZUINTEGER ¶

值是一个无符号整数。这与 PARAM_UINTEGER类似，不同之处在于允许值为 0，并且不支持 None 的值。

gdb.PARAM_ZUINTEGER_UNLIMITED ¶

值是一个有符号整数。这与 PARAM_INTEGER 类似，包括 None 的值应解释为 “无限制”（也可以使用字面量 'unlimited' 来设置该值），不同之处在于允许值为 0，并且值不能为负，除非设置为 “无限制” 时返回特殊值 -1。

gdb.PARAM_ENUM ¶

值是一个字符串，必须是创建参数时提供的一组字符串常量之一。

gdb.PARAM_COLOR ¶

值是 gdb.Color 实例。

使用 gdb.Parameter创建多个新参数时，通常希望创建一个前缀命令，用于将相关参数分组在一起。例如，如果你想添加参数 插件名称 功能 1 和 插件名称 功能 2，那么 插件名称 需要是一个前缀命令（请参阅 Python 中的 CLI 命令）。

然而，创建参数时，几乎总是需要创建两个前缀命令，一个作为 set 子命令，另一个作为 show 子命令。 GDB 提供了 gdb.ParameterPrefix 辅助类，使创建这两个前缀更容易。

函数： ParameterPrefix.__init__ (name, command_class, doc = None) ¶

的对象初始化器 ParameterPrefix 注册两个新的 gdb.Command 前缀，一个作为 set 子命令，另一个作为 show 子命令。

name, 一个字符串，是新前缀的名称，不包含 set 或 show, 类似于传递给 name 的 gdb.Parameter （请参阅 Python 中的参数）。例如，要创建前缀 set 插件名称 和 show 插件名称, 你将传递字符串 插件名称。

command_class 应该是 ‘COMMAND_’ 常量之一（请参阅 Python 中的 CLI 命令）。此参数告诉 GDB 如何在帮助系统中对新的参数前缀进行分类。

有多种方法可以为两个新的前缀命令提供帮助文本。如果 doc 参数不为 None, 那么它将用作两个前缀命令的文档字符串。

如果 doc 为 None, 但 gdb.ParameterPrefix 已被子类化，那么前缀命令文档将取自子类的文档字符串（即 __doc__ 属性）。

如果 doc 为 None, 并且没有 __doc__ 字符串，那么将使用默认值 ‘此命令未记录。’。

编写帮助文本时，请记住相同的文本用于 set 和 show 前缀命令。

函数： ParameterPrefix.invoke_set (argument, from_tty) ¶

如果子类定义了此方法，那么 GDB 在使用前缀命令时遇到未知子命令时会调用此方法。 argument 和 from_tty 参数与 gdb.Command.invoke （请参阅 Command.invoke）中的参数相同。

如果此方法抛出异常，它将转换为 GDB error 调用。否则，返回值将被忽略。

不是必须让 ParameterPrefix 子类重写此方法。通常，参数前缀仅作为将相关参数分组在一起的一种方式存在。 GDB 会自动处理这种情况，无需实现 invoke_set.

函数： ParameterPrefix.invoke_show (argument, from_tty) ¶

这与 invoke_set 方法类似，但适用于 show 前缀命令。与 invoke_set一样，此方法的实现是可选的，通常不需要。

函数： ParameterPrefix.dont_repeat () ¶

与 Command.dont_repeat （请参阅 Command.dont_repeat）一样，这可以从 ParameterPrefix.invoke_set 或 ParameterPrefix.invoke_show 调用，以防止前缀命令被重复执行。

这里有一个小示例，使用 gdb.ParameterPrefix 和 gdb.Parameter 创建两个新参数 插件名称 功能 1 和 插件名称 功能 2. 由于不需要 invoke_set 或 invoke_show ，此示例不子类化 gdb.ParameterPrefix: 

class ExampleParam(gdb.Parameter):
   def __init__ (self, name):
      super ().__init__ (name, gdb.COMMAND_DATA, gdb.PARAM_BOOLEAN)
      self.value = True

gdb.ParameterPrefix("插件名称", gdb.COMMAND_NONE,
                    """一个示例参数前缀。

                    这将一些参数分组在一起。""")
ExampleParam("插件名称 功能 1")
ExampleParam("插件名称 功能 2")
