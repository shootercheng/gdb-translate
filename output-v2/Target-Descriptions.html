附录 G 目标描述 ¶

使用的挑战之一 GDB 来调试嵌入式系统的挑战之一是，每种正在使用的处理器架构都有许多细微的变体。供应商通常会从一个标准的处理器内核开始，比如 ARM、PowerPC 或 MIPS ，然后进行修改以使其适应特定的市场细分领域。有些架构有数百种变体，来自数十家供应商。这导致了一些问题：

  • 由于有这么多不同的定制处理器，对于 GDB 维护者来说，跟上这些变化是很困难的。
  • 由于个别变体可能寿命较短或受众有限，在 GDB 源代码树中保留关于每个变体的信息可能不值得。
  • 当 GDB 确实支持手头嵌入式系统的架构时，为 set architecture 命令找到正确的架构名称的任务可能容易出错。

为了解决这些问题， GDB 远程协议允许目标系统不仅向 GDB标识自身，而且实际上描述其自身的特性。这使得 GDB 能够支持它以前从未见过的处理器变体——只要描述准确，并且 GDB 能够理解它们。

GDB 必须与 Expat 库链接以支持 XML 目标描述。请参阅 Expat.
