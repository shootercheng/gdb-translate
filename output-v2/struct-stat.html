结构体 stat ¶

目标使用的类型缓冲区 struct stat 并且 GDB 定义如下： 

结构体 stat {
    无符号整型  st_dev;      /* 设备 */
    无符号整型  st_ino;      /* 索引节点 */
    模式类型    st_mode;     /* 保护权限 */
    无符号整型  st_nlink;    /* 硬链接数量 */
    无符号整型  st_uid;      /* 所有者用户 ID */
    无符号整型  st_gid;      /* 所有者组 ID */
    无符号整型  st_rdev;     /* 设备类型（如果是索引节点设备） */
    无符号长整型 st_size;     /* 总大小，以字节为单位 */
    无符号长整型 st_blksize;  /* 文件系统 I/O 的块大小 */
    无符号长整型 st_blocks;   /* 分配的块数量 */
    时间类型      st_atime;    /* 最后访问时间 */
    时间类型      st_mtime;    /* 最后修改时间 */
    时间类型      st_ctime;    /* 最后更改时间 */
};

整数数据类型符合在 相应章节（见 整数数据类型, 详情）中给出的定义，因此这个 结构体大小为 64 字节。

几个字段的值有受限的含义和/或 值的范围。

st_dev

值为 0 表示文件，1 表示控制台。

st_ino

对目标没有有效含义。原样传输。

st_mode

有效的模式位在 常量中描述。任何其他 位目前对目标没有意义。

st_uid
st_gid
st_rdev

对目标没有有效含义。原样传输。

st_atime
st_mtime
st_ctime

这些值的精度依赖于主机和文件系统。特别是在 Windows 主机上，文件系统可能不 支持精确的时间值。

目标获取一个 struct stat 上述表示形式，并负责在继续之前将其转换为目标表示形式。

请注意，由于主机、目标和协议 对 struct stat 成员的表示形式在大小上存在差异，这些成员最终可能 在目标上被截断。
