20.5.3 综合应用 ¶

总之，当你的程序准备好进行调试时，你必须遵循以下步骤。

  1. 确保你已经定义了支持的底层例程（见 存根所需的操作）： 
    getDebugChar, putDebugChar,
flush_i_cache, memset, exceptionHandler.
  2. 在你的程序启动代码中，在调用主程序之前插入以下行： 
    set_debug_traps();
breakpoint();

    在某些机器上，当触发断点陷阱时，硬件会自动将程序计数器（PC）指向断点之后的指令。如果你的机器没有这样做，你可能需要进行调整 handle_exception 以便在第一次调用时让它返回到断点之后的指令，这样你的程序就不会一直停留在初始断点处，而是能够继续执行。

  3. 仅对于680x0存根，你需要提供一个名为 exceptionHook的变量。通常你只需使用： 
    void (*exceptionHook)() = 0;

    但如果在调用 set_debug_traps, 之前，你将其设置为指向你程序中的一个函数，那么当 GDB 在陷阱（例如总线错误）停止后继续执行时，该函数将被调用。由 exceptionHook 指示的函数将接收一个参数：一个 int ，它是异常编号。

  4. 将你的程序、 GDB 目标架构的调试存根以及支持子程序一起编译和链接。
  5. 确保你的目标机器和 GDB 主机之间有串行连接，并确定主机上的串行端口。
  6. 将你的程序下载到目标机器（或者通过制造商提供的任何方式将其传输到目标机器），并启动它。
  7. 在主机上启动 GDB ，并连接到目标（见 连接到远程目标)。
