22.8 自动加载关联文件 ¶

GDB 有时会自动读取包含命令和设置的文件，而无需用户明确指示。我们将此功能称为 自动加载。虽然自动加载对于自动适应项目需求很有用，但有时可能会产生意外结果或引入安全风险（例如，如果文件来自不可信源）。 GDB 。虽然自动加载对于自动适应项目需求很有用，但有时可能会产生意外结果或引入安全风险（例如，如果文件来自不可信源）。

有各种类型的文件 GDB 可以自动加载。除了这些文件之外， GDB 支持自动加载用各种扩展语言编写的代码。参见 自动加载扩展.

请注意，加载这些关联文件（包括本地 .gdbinit 文件）需要相应的配置 auto-load safe-path （参见 自动加载的安全限制）。

出于这些原因， GDB 包含命令和选项，让您可以控制何时自动加载文件以及应自动加载哪些文件。

set auto-load off ¶

全局禁用所有自动加载文件的加载。您可能希望将此命令与 ‘-iex’ 选项一起使用 （参见选项 -init-eval-command 

$ gdb -iex "set auto-load off" 不可信可执行文件 核心文件

请注意，系统初始化文件（参见 系统范围的配置和设置）和主目录中的初始化文件（参见 主目录初始化文件）仍然会被读取（因为它们来自通常可信的目录）。为了防止 GDB 即使这些初始化文件也自动加载，请使用 -nx 选项（参见 选择模式），此外还需要 set auto-load no.

show auto-load ¶

显示每个特定的 ‘自动加载’ 文件是否启用或禁用。 

(gdb) show auto-load
gdb 脚本：命令脚本序列的自动加载已开启。
线程调试库：特定于 inferior 的 libthread_db 的自动加载已开启。
本地 gdbinit 文件：从当前目录自动加载 .gdbinit 脚本已开启。
Python 脚本：Python 脚本的自动加载已开启。
安全路径：可以安全自动加载文件的目录列表是 $debugdir:$datadir/auto-load。
脚本目录：用于加载自动加载脚本的目录列表是 $debugdir:$datadir/auto-load。
info auto-load ¶

打印每个特定的 ‘自动加载’ 文件是否已自动加载。 

(gdb) info auto-load
gdb 脚本：
已加载  脚本
是     /home/user/gdb/gdb-gdb.gdb
线程调试库：未自动加载线程调试库。
本地 gdbinit 文件：本地 .gdbinit 文件 "/home/user/gdb/.gdbinit" 已加载。
Python 脚本：
已加载  脚本
是     /home/user/gdb/gdb-gdb.py
