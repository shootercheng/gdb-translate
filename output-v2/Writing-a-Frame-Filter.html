23.3.2.12 编写帧过滤器 ¶

帧过滤器必须实现三个基本元素：它必须正确实现文档化的接口（参见 过滤帧），必须向 GDB注册自身，最后，必须决定是否对 GDB提供的数据进行处理。在所有情况下，无论是否对迭代器进行处理，每个帧过滤器都必须返回一个迭代器。一个基本的帧过滤器遵循以下示例中的模式。 

import gdb

class FrameFilter():

    def __init__(self):
        # 帧过滤器属性创建。
        #
        # 'name' 是GDB将显示的过滤器名称。
        #
        # 'priority' 是该过滤器相对于其他过滤器的优先级。
        #
        # 'enabled' 是一个布尔值，指示此过滤器是否启用并应执行。

        self.name = "Foo"
        self.priority = 100
        self.enabled = True

        # 将此帧过滤器注册到全局 frame_filters 字典中。
        gdb.frame_filters[self.name] = self

    def filter(self, frame_iter):
        # 直接返回迭代器。
        return frame_iter

上面示例中的帧过滤器实现了所有帧过滤器的三个要求。它实现了API，自行注册，并对迭代器做出决策（在这种情况下，它只是原封不动地返回迭代器）。

第一步是属性创建和赋值，如注释所示，过滤器分配以下属性： name, priority 以及过滤器是否应通过 enabled 属性启用。

第二步是将帧过滤器注册到帧过滤器感兴趣的字典中。如注释所示，此过滤器只是将自身注册到全局字典 gdb.frame_filters中。如前所述， gdb.frame_filters 是在 gdb 模块中初始化的字典，当 GDB 启动时。过滤器注册到哪个字典是一个重要的考虑因素。一般来说，如果一个过滤器特定于一组代码，它应该注册到 objfile 或 progspace 字典中，因为它们特定于当前加载到 GDB中的程序。全局字典始终存在于 GDB 中，并且永远不会卸载。任何注册到全局字典的过滤器将一直存在，直到 GDB 退出。为了避免可能冲突的过滤器，通常最好将帧过滤器注册到与当前过滤器使用更紧密相关的字典中。请参阅 Python自动加载，以获取有关自动加载Python脚本的更多信息。

GDB 对帧过滤器注册采取不干预的方法，因此帧过滤器有责任确保注册已发生，并适当处理任何异常。特别是，您可能希望处理与Python字典键唯一性相关的异常。字典键必须与帧过滤器的 name 属性相同。当用户管理帧过滤器时（参见 帧过滤器的管理。），将显示的名称是包含在 GDB 属性中的名称。 name 属性。

此示例的最后一步是实现 filter 方法。如示例注释所示，我们定义了 filter 方法，并注意到该方法必须接受一个迭代器，并且还必须返回一个迭代器。在这个基本示例中，帧过滤器不是很有用，因为它只是原封不动地返回迭代器。然而，对于设置了 enabled 属性但决定不对任何帧进行操作的帧过滤器来说，这是一个有效的操作。

在下一个示例中，帧过滤器对所有帧进行操作，并使用帧装饰器对帧执行一些工作。请参阅 装饰帧，以获取有关帧装饰器接口的更多信息。

此示例处理内联帧。它通过用 “[inlined]” 标签标记内联帧来突出显示它们。通过使用Python的 itertools imap 方法将帧装饰器应用于所有帧，该示例将操作推迟到帧装饰器。帧装饰器仅在 GDB 打印回溯时处理。

这引入了一个新的决策主题：是在过滤步骤还是在打印步骤执行决策操作。在此示例的方法中，它在过滤步骤除了将帧装饰器映射到每个帧之外，不执行任何过滤决策。这允许在打印每个帧时执行实际的决策。这是一个重要的考虑因素，在设计帧过滤器时非常值得思考。帧过滤器应避免的一个问题是尽可能避免展开栈。有些栈可能非常深，在某些情况下可达数万层。提前搜索每个帧以确定它是否内联在过滤步骤可能代价太高。帧过滤器无法知道它必须迭代多少帧，并且必须遍历所有帧。这最终会重复工作，因为 GDB 在打印帧时会进行此迭代。

在此示例中，决策可以推迟到打印步骤。当 GDB 迭代时，帧装饰器可以依次检查每个帧。从性能角度来看，这是最合适的决策，因为它避免了重复打印步骤无论如何都会进行的工作。此外，如果有许多帧过滤器在过滤期间展开栈，可能会显著延迟回溯的打印，这将导致大量内存使用和糟糕的用户体验。 

class InlineFilter():

    def __init__(self):
        self.name = "InlinedFrameFilter"
        self.priority = 100
        self.enabled = True
        gdb.frame_filters[self.name] = self

    def filter(self, frame_iter):
        frame_iter = itertools.imap(InlinedFrameDecorator,
                                    frame_iter)
        return frame_iter

此帧过滤器与前面的示例有些相似，除了 filter 方法将一个名为 InlinedFrameDecorator 的帧装饰器对象应用于迭代器中的每个元素。 imap Python方法是轻量级的。它不会主动遍历迭代器，而是创建一个新的迭代器来包装现有的迭代器。

以下是此示例的帧装饰器。 

class InlinedFrameDecorator(FrameDecorator):

    def __init__(self, fobj):
        super(InlinedFrameDecorator, self).__init__(fobj)

    def function(self):
        frame = self.inferior_frame()
        name = str(frame.name())

        if frame.type() == gdb.INLINE_FRAME:
            name = name + " [inlined]"

        return name

此帧装饰器仅定义并覆盖了 function 方法。它让提供的 FrameDecorator, 它随 GDB一起提供，执行与打印此帧相关的其他工作。

这两个对象的组合从回溯中产生以下输出： 

#0  0x004004e0 in bar () at inline.c:11
#1  0x00400566 in max [inlined] (b=6, a=12) at inline.c:21
#2  0x00400566 in main () at inline.c:31

因此，在这个示例中，无论帧是否内联，帧装饰器都应用于所有帧。当 GDB 遍历帧过滤器产生的迭代器时， GDB 执行每个帧装饰器，然后在 function 回调中决定打印什么。使用这样的策略是一种将帧内容的决策推迟到打印时间的方法。

省略帧 ¶

可能上述示例不适合表示内联帧，而分层方法可能更受欢迎。如果我们想以分层方式表示帧， elided 帧装饰器接口可能更合适。

此示例使用 elided 方法处理这个问题。这个示例相当长，但非常简单。在本节中编写一个全面涵盖查找和打印内联帧所有方法的完整示例超出了范围。然而，这个示例说明了作者可能使用的方法。

此示例由三个部分组成。 

class InlineFrameFilter():

    def __init__(self):
        self.name = "InlinedFrameFilter"
        self.priority = 100
        self.enabled = True
        gdb.frame_filters[self.name] = self

    def filter(self, frame_iter):
        return ElidingInlineIterator(frame_iter)

此帧过滤器与其他示例非常相似。唯一的区别是此帧过滤器用一个名为frame_iter的自定义迭代器包装提供给它的迭代器（ ElidingInlineIterator）。这再次将操作推迟到 GDB 打印回溯时，因为迭代器直到打印时才会被遍历。

此示例的迭代器如下。在示例的这一部分中，对回溯的内容做出决策。 

class ElidingInlineIterator:
    def __init__(self, ii):
        self.input_iterator = ii

    def __iter__(self):
        return self

    def next(self):
        frame = next(self.input_iterator)

        if frame.inferior_frame().type() != gdb.INLINE_FRAME:
            return frame

        try:
            eliding_frame = next(self.input_iterator)
        except StopIteration:
            return frame
        return ElidingFrameDecorator(eliding_frame, [frame])

此迭代器实现了Python迭代器协议。当 next 函数被调用时（当 GDB 打印每个帧时），迭代器检查这个帧装饰器 frame, 是否包装了一个内联帧。如果不是，它原封不动地返回现有的帧装饰器。如果它包装了一个内联帧，它假设内联帧包含在接下来最旧的帧 eliding_frame, 中，它会获取该帧。然后它创建并返回一个帧装饰器 ElidingFrameDecorator, 它包含省略的帧和省略它的帧。 

class ElidingInlineDecorator(FrameDecorator):

    def __init__(self, frame, elided_frames):
        super(ElidingInlineDecorator, self).__init__(frame)
        self.frame = frame
        self.elided_frames = elided_frames

    def elided(self):
        return iter(self.elided_frames)

此帧装饰器覆盖了一个函数，并在 elided 方法中返回内联帧。和之前一样，它让 FrameDecorator 完成打印此帧涉及的其余工作。这产生了以下输出。 

#0  0x004004e0 in bar () at inline.c:11
#2  0x00400529 in main () at inline.c:25
    #1  0x00400529 in max (b=6, a=12) at inline.c:15

在那个输出中 max 已内联到 main 以分层方式打印。另一种方法是将 function 方法，以及 elided 方法结合起来，既在内联帧中打印一个标记，又显示分层关系。
