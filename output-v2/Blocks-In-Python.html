23.3.2.29 从 Python 访问块 ¶

在 GDB, 符号存储在块中。一个块大致对应于源代码中的一个作用域。块是分层组织的，在 Python 中单独表示为一个 gdb.Block。块依赖于可用的调试信息。

一个帧有一个块。请参阅 从 Python 访问 inferior 栈帧, 以获取关于帧的更深入讨论。

最外层的块称为 全局块。全局块通常包含公共全局变量和函数。

嵌套在全局块内的块是 静态块。静态块通常包含文件作用域的变量和函数。

GDB 提供了一种获取块的父块的方法，但目前没有办法检查块的子块，也无法遍历符号表中的所有块（请参阅 Python 中的符号表表示形式）。

以下是一个简短的示例，有助于解释块： 

/* 这在全局块中。  */
int global;

/* 这在静态块中。  */
static int file_scope;

/* 'function' 在全局块中，'argument' 在 'function' 内部嵌套的块中。  */
int function (int argument)
{
  /* 'local' 在 'function' 内部的一个块中。它可能与 'argument' 在同一个块中，也可能不在。  */
  int local;

  {
     /* 'inner' 在一个块中，其父块是包含 'local' 的块。  */
     int inner;

     /* 如果此调用被编译器展开，您可能会在这里看到一个嵌套块，其函数为 'inline_function'，其父块是包含 'inner' 的块。  */
     inline_function ();
  }
}

一个 gdb.Block 是可迭代的。迭代器返回符号（请参阅 符号的 Python 表示形式），这些符号是块的局部符号。Python 程序不应假设特定的块对象总是包含给定的符号，因为 GDB 功能和基础设施的变化可能会导致符号在符号表中的块之间移动。您还可以使用 Python 的 字典语法 来访问此块中的变量，例如： 

symbol = some_block['variable']  # symbol 的类型为 gdb.Symbol

以下与块相关的函数在 gdb 模块中可用：

函数： gdb.block_for_pc (pc) ¶

返回包含给定 gdb.Block 的最内层 pc 值。如果无法为指定的 pc 值找到块，函数将返回 None。这与 gdb.current_progspace().block_for_pc(pc) 相同，包含此函数是为了保持历史兼容性。

一个 gdb.Block 对象有以下方法：

函数： Block.is_valid () ¶

如果 True 对象有效，则返回 gdb.Block ，否则返回 False 。如果块对象所引用的块在 inferior 中不再存在，则该块对象可能会变得无效。如果在调用方法时块对象无效，所有其他 gdb.Block 方法都会抛出异常。在遍历块的符号时也会检查块的有效性。

一个 gdb.Block 对象有以下属性：

变量： Block.start ¶

块的起始地址。此属性不可写。

变量： Block.end ¶

块中最后一个地址之后的地址。此属性不可写。

变量： Block.function ¶

块的名称，表示为一个 gdb.Symbol。如果块没有名称，则此属性为 None。此属性不可写。

对于普通的函数块，父块是静态块。但是，您应该注意，函数块的父块有可能不是静态块，例如内联函数就是这种情况。

变量： Block.superblock ¶

包含此块的块。如果此父块不存在，此属性为 None。此属性不可写。

变量： Block.subblocks ¶

嵌套在此块中的块的列表。如果没有嵌套块，此属性为一个空列表。此属性不可写。

变量： Block.global_block ¶

与此块关联的全局块。此属性不可写。

变量： Block.static_block ¶

与此块关联的静态块。此属性不可写。

变量： Block.is_global ¶

True 如果 gdb.Block 对象是全局块， False 否则。此属性不可写。

变量： Block.is_static ¶

True 如果 gdb.Block 对象是静态块， False 否则。此属性不可写。
