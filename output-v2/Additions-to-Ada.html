15.4.10.3 Ada 的扩展 ¶

与处理其他语言一样， GDB 对 Ada 进行了某些通用扩展（请参阅 表达式)：

  • 如果表达式 E 是一个驻留在内存中的变量（通常是局部变量或数组元素），并且 N 是一个正整数，那么 E@N 将以数组形式显示 E 的值以及内存中紧随其后的 N - 1 个相邻变量的值。在 Ada 中，这个运算符通常不是必需的，因为它的主要用途是显示数组的部分内容，而在 Ada 中切片通常可以实现这一点。然而，在调试某些调试信息已被优化掉的程序时，偶尔会用到它。
  • B::var 表示“名为 var 的变量，该变量出现在函数或文件 B中”。当 B 是一个文件名时，通常必须用单引号将其括起来。
  • 表达式 {type} addr 表示“类型为 type 且出现在地址 addr的变量”。
  • 以 ‘$’ 开头的名称是一个便利变量（请参阅 便利变量) 或一个机器寄存器（请参阅 寄存器)。

此外， GDB 还提供了一些其他特定于 Ada 的快捷方式和直接扩展：

  • 赋值语句允许作为表达式使用，返回其右操作数作为值。因此，您可以输入 
    (gdb) set x := y + 3
(gdb) print A(tmp := y + 1)
  • 分号允许作为一个“运算符”使用，返回其右操作数的值。这允许，例如，复杂的条件断点： 
    (gdb) break f
(gdb) condition 1 (report(i); k += 1; A(k) > 100)
  • 对基于进制的字面量的扩展可用于指定浮点字面量的确切字节内容。在进制之后，您可以使用零到两个 ‘l’ 字符，后跟一个 ‘f’。‘l’ 字符的数量控制结果实常量的宽度：零表示 Float 使用，一个表示 Long_Float, 两个表示 Long_Long_Float。 
    (gdb) print 16f#41b80000#
$1 = 23.0
  • 与其使用连接和符号字符名称将特殊字符引入字符串，不如使用一种特殊的括号表示法，这种表示法也用于打印字符串。字符串或字符字面量中形式为 ‘["XX"]’ 的字符序列表示其数字编码为 XX （十六进制）的（单个）字符。字符序列 ‘["""]’ 也表示字符串中的单个引号。例如， 
       "One line.["0a"]Next line.["0a"]"

    在每个句点后包含一个 ASCII 换行符（Ada.Characters.Latin_1.LF)。

  • 用作属性前缀的子类型 'Pos, 'Min, 和 'Max 是可选的（并且无论如何都会被忽略）。例如，编写以下内容是有效的 
    (gdb) print 'max(x, y)
  • 在打印数组时， GDB 当数组的下界为 1 时使用位置表示法，否则使用修改后的命名表示法。例如，一个下界为 3 的包含三个整数的一维数组可能会打印为 
    (3 => 10, 17, 1)

    也就是说，与有效的 Ada 不同，只有第一个组件有 => 子句。

  • 您可以在表达式中使用属性名称的任何唯一多字符子序列来缩写属性（精确匹配优先）。例如，您可以使用 a'len, a'gth, 或 a'lh 来代替 a'length。
  • 由于 Ada 不区分大小写，调试器通常会将您输入的标识符映射为小写。GNAT 编译器对其一些内部标识符使用大写字符，这些标识符通常对用户没有意义。在极少数情况下，您实际上需要查看它们时，将它们用尖括号括起来以避免小写映射。例如， 
    (gdb) print <JMPBUF_SAVE>[0]
  • 打印类范围类型的对象或解引用指向类范围的访问值将显示对象特定类型的所有组件（由其运行时标签指示）。同样，对这样的值进行组件选择将对对象的特定类型进行操作。
