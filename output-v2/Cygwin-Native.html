21.1.4 调试MS Windows PE可执行文件的功能 ¶

GDB 支持对MS Windows程序进行原生调试，包括带有和不带有符号调试信息的DLL。

调用……的MS Windows程序 SetConsoleMode 以关闭‘Ctrl - C’按键的特殊含义，不能通过键入 C - c来中断。 出于这个原因， GDB 在MS Windows上支持 C -BREAK 作为替代的中断键序列，即使被调试程序忽略 C - c，也可用于中断被调试程序。

本节描述了各种特定于Cygwin的命令。 关于处理没有调试符号的DLL的内容在 对无调试符号的DLL的支持中描述。

info w32 ¶

这是特定于MS Windows的命令的前缀，用于打印有关目标系统和重要操作系统结构的信息。

info w32 selector

此命令显示由Win32 API返回的信息 GetThreadSelectorEntry 函数。 它接受一个可选参数，该参数被计算为一个长整型值，以提供有关给定选择器的信息。 无参数时，此命令显示有关六个段寄存器的信息。

info w32 thread-information-block

此命令显示存储在线程信息块中的线程特定信息（在X86 CPU系列上使用 $fs 32位程序的选择器和 $gs 64位程序的选择器可读）。

signal-event id ¶

此命令用用户提供的 ID发出一个事件信号。 用于在使用MS Windows JIT调试（AeDebug）附加到崩溃进程时恢复该进程。

要使用它，请在 HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\AeDebug 和/或 HKLM\SOFTWARE\Wow6432Node\Microsoft\Windows NT\CurrentVersion\AeDebug （适用于x86_64版本）中创建或编辑以下键：

  • Debugger （REG_SZ） —— 启动调试器的命令。 建议的命令是： fully-qualified-path-to-gdb.exe -ex "attach %ld" -ex "signal-event %ld" -ex "continue".

    第一个 %ld 将被崩溃进程的进程ID替换，第二个 %ld 将被阻止崩溃进程的事件ID替换，等待 GDB 附加。

  • Auto （REG_SZ） —— 要么 1 要么 0. 1 将使系统自动运行由Debugger键指定的调试器， 0 将弹出一个带有“确定”和“取消”按钮的对话框，允许用户要么终止崩溃进程（确定），要么对其进行调试（取消）。
set cygwin-exceptions mode ¶

如果 模式 是 on, GDB 将在Cygwin DLL内部发生异常时中断。 如果 模式 是 off, GDB 将延迟异常的识别，并且可能会忽略一些似乎由Cygwin DLL内部“簿记”引起的异常。 此选项主要用于调试Cygwin DLL本身；默认值是 off 以避免用虚假的 GDB 信号烦扰 SIGSEGV 用户。

show cygwin-exceptions ¶

显示 GDB 是否将在Cygwin DLL本身内部发生异常时中断。

set new-console mode ¶

如果 模式 是 on 被调试程序将在下一次启动时在新控制台中启动。 如果 模式 是 off, 被调试程序将在与调试器相同的控制台中启动。

show new-console ¶

显示启动被调试程序时是否使用新控制台。

set new-group mode ¶

此布尔值控制被调试程序是应启动一个新组还是与调试器保持在同一组中。 这会影响Windows操作系统处理‘Ctrl - C’的方式。

show new-group ¶

显示新组布尔值的当前值。

set debugevents ¶

此布尔值添加与调试器所看到的与被调试程序相关的内核事件有关的调试输出。 这包括表示线程和进程创建与退出、DLL加载和卸载、控制台中断以及由Windows OutputDebugString API调用产生的调试消息的事件。

set debugexec ¶

此布尔值添加与调试器所看到的执行事件（如恢复线程）有关的调试输出。

set debugexceptions ¶

此布尔值添加与调试器所看到的被调试程序中的异常有关的调试输出。

set debugmemory ¶

此布尔值添加与调试器对被调试程序的内存读写有关的调试输出。

set shell ¶

此布尔值指定被调试程序是通过shell调用还是直接调用（默认值为是）。

show shell ¶

显示被调试程序是否将使用shell启动。
