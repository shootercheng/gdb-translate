23.3.3 Python 自动加载 ¶

当读取一个新的目标文件时（例如，由于 file 命令，或者因为 inferior 加载了一个共享库）， GDB 将通过多种方式查找 Python 支持脚本： 目标文件-gdb.py 和 .debug_gdb_scripts 部分。 请参阅 自动加载扩展.

自动加载功能对于提供特定于应用程序的调试命令和脚本很有用。

可以启用或禁用自动加载，并且可以打印自动加载脚本的列表。

set auto-load python-scripts [on|off] ¶

启用或禁用 Python 脚本的自动加载。

show auto-load python-scripts ¶

显示是否启用或禁用了 Python 脚本的自动加载。

info auto-load python-scripts [regexp] ¶

打印所有 GDB 自动加载的 Python 脚本的列表。

还会打印在 .debug_gdb_scripts 部分中提及但未找到的 Python 脚本列表（请参阅 该 .debug_gdb_scripts 部分) 或由于 auto-load safe-path 拒绝而未自动加载的脚本列表（请参阅 自动加载关联文件)。 这很有用，因为当 GDB 尝试加载它们并失败时，不会打印它们的名称。可能有很多这样的脚本，为每个脚本打印错误消息会有问题。

如果 正则表达式 被提供，则仅打印名称匹配的 Python 脚本。

示例： 

(gdb) info auto-load python-scripts
已加载的脚本
是    py-section-script.py
       完整名称：/tmp/py-section-script.py
否     my-foo-pretty-printers.py

在读取自动加载的文件或脚本时， GDB 设置 当前目标文件. 这可以通过 gdb.current_objfile 函数获得（请参阅 Python 中的目标文件)。这对于注册特定于目标文件的漂亮打印机和帧过滤器很有用。
