22.4 屏幕尺寸 ¶

某些命令 GDB 可能会产生大量的信息输出到屏幕上。为了帮助您阅读所有内容， GDB 会在每一页输出结束时暂停并要求您输入。输入 回车键 当您想查看下一页输出时， q 以丢弃剩余的输出，或者 c 以继续执行当前命令而不进行分页。此外，屏幕宽度设置决定了何时换行输出。根据输出内容的不同， GDB 会尝试在一个易读的位置换行，而不是简单地让其溢出到下一行。

通常情况下 GDB 会从终端驱动软件获取屏幕尺寸信息。例如，在Unix系统上 GDB 会使用termcap数据库以及 TERM 环境变量的值和 stty rows 以及 stty cols 设置。如果这些信息不正确，您可以使用 set height 和 set width 命令来覆盖它：

set height lpp ¶
set height unlimited
show height
set width cpl
set width unlimited
show width

这些 set 命令指定屏幕高度为 每页行数 行，屏幕宽度为 每行字符数 个字符。相关的 show 命令用于显示当前设置。

如果您指定的高度为 unlimited 或零行， GDB 无论输出有多长，在输出过程中都不会暂停。如果输出是到文件或编辑器缓冲区，这很有用。

同样，您可以指定 ‘设置宽度无限制’ 或 ‘设置宽度为0’ 以防止 GDB 对其输出进行换行。

set pagination on ¶
set pagination off

开启或关闭输出分页；默认是开启的。关闭分页是替代 set height unlimited的另一种方法。请注意，运行 GDB 时使用 --batch 选项（见 -batch) 也会自动禁用分页。

show pagination ¶

显示当前的分页模式。
