23.5.1 该 目标文件-gdb.扩展名 文件 ¶

当读取一个新的目标文件时， GDB 会查找一个名为 目标文件-gdb.扩展名 （我们下面称之为 脚本名称 ）的文件，其中 目标文件 是目标文件的名称，并且 扩展名 是扩展语言的文件扩展名：

objfile-gdb.gdb

GDB自身的命令语言

objfile-gdb.py

Python

objfile-gdb.scm

Guile

脚本名称 是通过确保 目标文件 的文件名是绝对路径，遵循所有符号链接，并解析 . 和 .. 组件，然后附加 -gdb.扩展名 后缀来形成的。如果这个文件存在且可读， GDB 将把它作为指定扩展语言的脚本执行。

如果这个文件不存在，那么 GDB 将在下面指定的所有目录中查找 脚本名称 文件。（在MS - Windows/MS - DOS上，可执行文件前导目录的驱动器字母会转换为单字母子目录，例如 d:/usr/bin/ 会转换为 /d/usr/bin/, 因为Windows文件系统不允许文件名中包含冒号。）

请注意，加载这些文件需要相应配置的 auto-load safe-path （参见 自动加载的安全限制）。

对于使用 .exe 后缀的目标文件， GDB 首先会根据其 .exe 文件名正常加载脚本。但如果没有找到脚本 GDB 也会尝试与去掉 .exe 后缀的目标文件匹配的脚本文件名。这种去掉 .exe 后缀的操作不区分大小写，并且会在任何平台上尝试。这使得脚本文件名在Unix和MS - Windows主机之间兼容。

set auto-load scripts-directory [directories] ¶

控制 GDB 自动加载脚本的位置。多个目录条目可以用当前主机平台的路径分隔符分隔（‘:’ 在Unix上，‘;’ 在MS - Windows和MS - DOS上）。

这里的每个条目也需要被安全设置覆盖 set auto-load safe-path （参见 设置自动加载安全路径）。

这个变量的默认值是 $debugdir:$datadir/auto - load. 默认 set auto-load safe-path 值也可以通过 GDB 配置选项 --with - auto - load - dir来覆盖。

任何对 $debugdir 的引用都会被 调试文件目录 的值替换（参见 单独文件中的调试信息），并且任何对 $datadir 的引用都会被 数据目录 替换，该目录是在 GDB 启动时确定的（参见 GDB数据文件）。 $debugdir 和 $datadir 必须作为目录组件放置 —— 可以单独放置，也可以用 / 或 \ 目录分隔符分隔，具体取决于主机平台。

目录列表使用路径分隔符（‘:’ 在GNU和Unix系统上，‘;’ 在MS - Windows和MS - DOS上）来分隔目录，这与 PATH 环境变量类似。

show auto-load scripts-directory ¶

显示 GDB 自动加载脚本的位置。

add-auto-load-scripts-directory [directories…] ¶

将一个条目（或条目列表）添加到自动加载脚本位置列表中。多个条目可以用当前主机平台的路径分隔符分隔。

GDB 不会跟踪它已经以这种方式自动加载了哪些文件。 GDB 每次打开相应的 目标文件 时，都会加载关联的脚本。所以你的 -gdb.扩展名 文件在多次执行时应小心避免出错。
