4.1 为调试而编译 ¶

为了有效地调试程序，你需要在编译时生成调试信息。这些调试信息存储在目标文件中；它描述了每个变量或函数的数据类型，以及源代码行号与可执行代码中地址之间的对应关系。

要请求调试信息，在运行编译器时指定 ‘-g’ 选项。

要交付给客户的程序通常使用 ‘-O’ 编译器选项进行优化编译。然而，一些编译器无法同时处理 ‘-g’ 和 ‘-O’ 选项。使用这些编译器时，你无法生成包含调试信息的优化可执行文件。

GCC, 即 GNU C/C++ 编译器，支持在有或没有 ‘-g’ 的情况下使用 ‘-O’，从而可以调试优化后的代码。我们建议你 始终 在编译程序时使用 ‘-g’。你可能认为自己的程序是正确的，但没必要冒险。更多信息，请参阅 调试优化代码.

较旧版本的 GNU C 编译器允许使用变体选项 ‘-gg’ 用于调试信息。 GDB 不再支持这种格式；如果你的 GNU C 编译器有这个选项，请勿使用。

GDB 了解预处理器宏，并能向你展示它们的展开情况（请参阅 C 预处理器宏)。如果仅指定 -g 标志，大多数编译器不会在调试信息中包含预处理器宏的信息。 GCC, 即 GNU C 编译器的 3.1 版及更高版本，如果使用 DWARF 调试格式并指定选项 -g3.

请参阅 调试你的程序或 GCC 的选项 在 使用 GNU 编译器集合（GCC）, 以获取有关 GCC 影响调试信息的选项的更多信息。

如果你使用编译器支持的最新版本的 DWARF 调试格式，将获得最佳的调试体验。DWARF 目前是 GDB中最具表现力且支持最好的调试格式。
