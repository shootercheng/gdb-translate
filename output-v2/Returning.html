17.4 从函数返回 ¶

return ¶
return expression

你可以使用该命令取消函数调用的执行。 return 命令。如果你给出一个 表达式 参数，其值将作为函数的返回值。

当你使用 return, GDB 会丢弃所选的栈帧（以及其中的所有帧）。你可以将此视为使被丢弃的帧提前返回。如果你希望指定要返回的值，请将该值作为参数传递给 return.

这会弹出所选的栈帧（请参阅 选择一个帧），以及其中的任何其他帧，使其调用者成为最内层剩余的帧。该帧将被选中。指定的值将存储在用于返回函数值的寄存器中。

该 return 命令不会恢复执行；它会使程序停留在函数刚刚返回时的状态。相比之下， finish 命令（请参阅 继续和单步执行) 会继续执行，直到所选的栈帧自然返回。

GDB 需要知道如何为 表达式 参数设置下级进程。具体的寄存器分配取决于操作系统的应用程序二进制接口（ABI）以及所选栈帧返回的类型。例如，操作系统的ABI通常会在浮点运算单元（FPU）寄存器中返回浮点值，而在中央处理器（CPU）寄存器中返回整数值。不过，有些ABI甚至会在CPU寄存器中返回浮点值。较大的整数宽度（例如 long long int) 也有特定的放置规则。 GDB 已经从其当前目标了解了操作系统的ABI，因此它还需要找出要返回的类型，以便将值分配到正确的寄存器中。

通常，所选的栈帧有调试信息。 GDB 在有调试信息可用时，总是会使用调试信息而不是 表达式 的隐式类型。例如，如果你输入 return -1, 并且当前栈帧中的函数声明返回一个 long long int, GDB 会透明地将隐式的 int 值 -1 转换为一个 long long int: 

断点 1, func () 在 gdb.base/return-nodebug.c:29
29        return 31;
(gdb) return -1
现在让 func 返回吗？(y 或 n) y
#0  0x004004f6 在 main () 中，位于 gdb.base/return-nodebug.c:43
43        printf ("result=%lld\n", func ());
(gdb)

但是，如果所选的栈帧没有调试信息，例如，如果函数是在没有调试信息的情况下编译的， GDB 必须从用户那里了解要返回的类型。错误地指定不同的类型可能会将值设置在与调用者代码预期不同的下级进程寄存器中。例如，输入 return -1 及其隐式类型 int 只会为没有调试信息的函数设置部分 long long int 结果（在32位架构上）。因此，用户需要通过适当的强制类型转换显式指定返回类型： 

断点 2, 0x0040050b 在 func () 中
(gdb) return -1
所选栈帧的返回值类型不可用。
请使用显式的强制类型转换来返回值。
(gdb) return (long long int) -1
现在让所选栈帧返回吗？(y 或 n) y
#0  0x00400526 在 main () 中
(gdb)
