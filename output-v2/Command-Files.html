23.1.3 命令文件 ¶

用于 GDB 的命令文件是一个由 GDB 命令行组成的文本文件。也可以包含注释（以 #开头的行）。命令文件中的空行不执行任何操作；它不像在终端中那样意味着重复最后一个命令。

可以使用 source 命令请求执行命令文件。请注意， source 命令也用于计算不是命令文件的脚本。确切的行为可以使用 script-extension 设置进行配置。请参阅 扩展 GDB。

source [-s] [-v] filename ¶

执行命令文件 文件名。

命令文件中的行通常按顺序执行，除非执行顺序被下面描述的 流控制命令 之一改变。命令执行时不会打印出来。任何命令中的错误都会终止命令文件的执行，并将控制权返回给控制台。

GDB 首先在当前目录中搜索 文件名 。如果在那里找不到该文件，并且 文件名 没有指定目录，那么 GDB 也会在源搜索路径（使用‘directory’命令指定）中查找该文件；但 $cdir 不会被搜索，因为编译目录与脚本无关。

如果指定了 -s ，那么 GDB 会在搜索路径中查找 文件名 ，即使 文件名 指定了目录。搜索是通过将 文件名 附加到搜索路径的每个元素上来完成的。例如，如果 文件名 是 mylib/myscript ，并且搜索路径包含 /home/user ，那么 GDB 会查找脚本 /home/user/mylib/myscript。如果 文件名 是绝对路径，也会进行搜索。例如，如果 文件名 是 /tmp/myscript ，并且搜索路径包含 /home/user ，那么 GDB 会查找脚本 /home/user/tmp/myscript。对于类似 DOS 的系统，如果 文件名 包含驱动器说明，在连接之前会将其去除。例如，如果 文件名 是 d:myscript ，并且搜索路径包含 c:/tmp ，那么 GDB 会查找脚本 c:/tmp/myscript。

如果给出 -v（用于详细模式），那么 GDB 会在执行每个命令时显示该命令。该选项必须在 文件名之前给出，否则会被解释为文件名的一部分。

在交互使用时需要确认的命令，在命令文件中使用时会直接执行而不询问。许多 GDB 通常会打印消息说明其操作的命令，从命令文件调用时会省略这些消息。

GDB 也接受来自标准输入的命令输入。在这种模式下，正常输出到标准输出，错误输出到标准错误。标准输入提供的命令文件中的错误不会终止命令文件的执行 —— 执行会继续下一个命令。 

gdb < cmds > log 2>&1

（上面的语法会根据使用的 shell 而有所不同。）这个例子将从文件 cmds中执行命令。所有输出和错误都会定向到 log。

由于存储在命令文件中的命令往往比交互式输入的命令更通用，它们经常需要处理复杂的情况，例如变量和符号的不同或意外值、被调试程序的构建方式的变化等。 GDB 提供了一组流控制命令来处理这些复杂性。使用这些命令，可以编写遍历数据结构、有条件地执行命令等复杂脚本。

if ¶
else

此命令允许在脚本中包含有条件执行的命令。 if 该命令接受一个参数，该参数是一个要计算的表达式。其后跟着一系列命令，只有当表达式为真（其值非零）时才会执行这些命令。然后可以选择有一个 else 行，其后跟着一系列只有当表达式为假时才会执行的命令。列表的结尾由包含 end的行标记。

while ¶

此命令允许编写循环。其语法类似于 if：该命令接受一个参数，该参数是一个要计算的表达式，并且后面必须跟着要执行的命令，每行一个，以 end结束。这些命令被称为 循环体 。 while 循环体中的命令只要表达式计算结果为真就会重复执行。

loop_break ¶

此命令退出包含它的 while 循环。脚本的执行在该 while行之后继续。 end

loop_continue ¶

此命令跳过包含它的 while 循环体中其余命令的执行。执行跳转到 while 循环的开头，在那里计算控制表达式。

end ¶

终止作为 if、 else或 while 流控制命令主体的命令块。
