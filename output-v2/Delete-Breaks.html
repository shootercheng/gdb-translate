5.1.4 删除断点 ¶

一旦断点、观察点或捕获点完成了它的任务，并且你不再希望程序在那里停止，通常就需要将其移除。这被称为 删除 断点。已删除的断点将不再存在；它会被遗忘。

使用 clear 命令，你可以根据断点在程序中的位置来删除它们。使用 delete 命令，你可以通过指定断点编号来删除单个断点、观察点或捕获点。

要越过断点继续执行，并不一定需要删除断点。 GDB 当你在不改变执行地址的情况下继续执行时，会自动忽略要执行的第一条指令上的断点。

clear ¶

删除所选栈帧中下一条要执行的指令处的任何断点（请参阅 选择帧）。当选择最内层帧时，这是删除程序刚刚停止处断点的一个好方法。

clear locspec

删除代码位置对应于 位置规范的任何断点。请参阅 位置规范, 了解 位置规范的各种形式。哪些代码位置对应于 位置规范 取决于位置规范中使用的形式 位置规范:

linenum
filename:linenum
-line linenum
-source filename -line linenum

如果 位置规范 指定了行号，无论是否带有文件名，该命令都会删除代码位置在指定行或指定行内的任何断点 行号 在与指定的 文件名匹配的文件中。如果 文件名 被省略，它默认是当前源文件。

*address

如果 位置规范 指定了一个地址，该命令会删除代码位置在给定 地址。

function
-function function

如果 位置规范 指定了一个函数，该命令会删除代码位置在名称与 函数匹配的任何函数入口处的任何断点。

文件和函数名称的歧义可以按照 位置规范中所述的方法解决。

delete [breakpoints] [list…] ¶

删除作为参数指定的断点列表中的断点、观察点、跟踪点或捕获点。如果未指定参数，则删除所有断点、观察点、跟踪点和捕获点（GDB 会要求确认，除非你已经 set confirm off）。你可以将此命令缩写为 d。
