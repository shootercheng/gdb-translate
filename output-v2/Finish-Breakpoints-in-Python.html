23.3.2.34 完成断点 ¶

完成断点是基于某个命令在帧的返回地址处设置的临时断点。 finish 命令 gdb.FinishBreakpoint 扩展 gdb.Breakpoint。当执行超出断点范围（即 Breakpoint.stop 或 FinishBreakpoint.out_of_scope 触发）时，底层断点将被禁用并删除。完成断点是特定于线程的，必须在选择正确的线程时创建。

函数： FinishBreakpoint.__init__ ([帧] [, 内部]) ¶

在 gdb.Frame 对象 帧的返回地址处创建一个完成断点。如果 帧 未提供，则默认为最新的帧。可选的 内部 参数允许断点对用户不可见。请参阅 使用Python操作断点, 以获取有关此参数的更多详细信息。

函数： FinishBreakpoint.out_of_scope (self) ¶

在某些情况下（例如 longjmp, C++ 异常 GDB return 命令等），函数可能无法正常终止，因此永远不会触发完成断点。当 GDB 注意到这种情况时， out_of_scope 回调将被触发。

你可能想对 gdb.FinishBreakpoint 进行子类化并覆盖此方法： 

class MyFinishBreakpoint (gdb.FinishBreakpoint)
    def stop (self):
        print ("正常完成")
        return True
    
    def out_of_scope ():
        print ("异常完成")
变量： FinishBreakpoint.return_value ¶

当 GDB 在完成断点处停止，并且用于构建 gdb.FinishBreakpoint 对象的帧有调试符号时，此属性将包含一个 gdb.Value 对象，对应于函数的返回值。如果函数返回类型是 None 或者返回值无法计算，则该值将是 void 。此属性不可写。
