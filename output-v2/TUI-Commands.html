TUI有特定的命令来控制文本窗口。这些命令始终可用，即使在 GDB 不在TUI模式下时也是如此。当 GDB 处于标准模式时，大多数这些命令将自动切换到TUI模式。

请注意，如果 GDB的 stdout 未连接到终端，或者 GDB 已使用机器接口解释器启动（参见 GDB/MI 接口），大多数这些命令将因错误而失败，因为启用curses窗口管理是不可能或不可取的。

tui enable ¶

激活TUI模式。如果在当前调试会话中之前使用过TUI模式，则将使用上次活动的TUI窗口布局，否则将使用默认布局。

tui disable ¶

禁用TUI模式，返回控制台解释器。

info win ¶

列出所有当前显示窗口的名称和大小。

tui new-layout name window weight [window weight…] ¶

创建一个新的TUI布局。新布局将命名为 名称，并且可以使用 layout 命令（见下文）访问。

每个 窗口 参数要么是要显示的窗口的名称，要么是窗口描述。窗口将按列出的顺序从上到下显示。

窗口的名称与 focus 命令（见下文）中给出的名称相同；此外，还可以指定 status 窗口。请注意，由于状态窗口的高度是固定的，因此分配给状态窗口的权重并不重要。通常在此处使用‘0’。

窗口描述看起来有点像 tui new-layout的调用，其形式为 {[-horizontal]窗口 权重 [窗口 权重…]}。

这指定了一个子布局。如果 -horizontal 被指定，则此描述中的窗口将并排排列，而不是从上到下排列。

每个 权重 是一个整数。它是此窗口相对于布局中所有其他窗口的权重。这些数字用于计算每个窗口占用屏幕的比例。

例如： 

(gdb) tui new-layout example src 1 regs 1 status 0 cmd 1

这里，新布局名为‘example’。它显示源窗口和寄存器窗口，然后是状态窗口，最后是命令窗口。非状态窗口的权重都相同，因此终端将大致分为三个相等的部分。

这里有一个更复杂的例子，展示了一个水平布局： 

(gdb) tui new-layout example {-horizontal src 1 asm 1} 2 status 0 cmd 1

这将导致源窗口和汇编窗口并排显示；状态窗口和命令窗口位于下方，填满终端的整个宽度。由于源窗口和汇编窗口的权重为2，它们的高度将是命令窗口的两倍。

tui layout name ¶
layout name

更改显示哪些TUI窗口。 名称 参数控制显示哪个布局。它可以是内置布局名称之一，也可以是用户使用 tui new-layout定义的布局名称。

内置布局如下：

next

显示下一个布局。

prev

显示上一个布局。

src

显示源窗口和命令窗口。

asm

显示汇编窗口和命令窗口。

split

显示源窗口、汇编窗口和命令窗口。

regs

当处于 src 布局时，显示寄存器窗口、源窗口和命令窗口。当处于 asm 或 split 布局时，显示寄存器窗口、汇编窗口和命令窗口。

tui focus name ¶
focus name

更改当前用于滚动的TUI窗口。 名称 参数可以是以下任何一个：

next

使下一个窗口成为滚动活动窗口。

prev

使上一个窗口成为滚动活动窗口。

src

使源窗口成为滚动活动窗口。

asm

使汇编窗口成为滚动活动窗口。

regs

使寄存器窗口成为滚动活动窗口。

cmd

使命令窗口成为滚动活动窗口。

tui refresh ¶
refresh

刷新屏幕。这类似于键入 C-L。

tui reg group ¶

将tui寄存器窗口中显示的寄存器组更改为 组 如果当前未显示寄存器窗口，此命令将导致显示寄存器窗口。寄存器组列表及其顺序因目标而异。大多数目标上有以下组可用：

next

重复选择此组将使显示循环遍历所有可用的寄存器组。

prev

重复选择此组将使显示以相反顺序循环遍历所有可用的寄存器组。 下一个。

general

显示通用寄存器。

float

显示浮点寄存器。

system

显示系统寄存器。

vector

显示向量寄存器。

all

显示所有寄存器。

update ¶

更新源窗口和当前执行点。

tui window height name +count ¶
tui window height name -count
winheight name +count
winheight name -count

更改窗口的高度 名称 按 数量 行。正数量增加高度，负数量减小高度。 名称 参数可以是当前任何可见窗口的名称。可以使用以下命令发现当前可见窗口的名称 info win （参见 info win）。

当前可见窗口集必须始终填满终端，因此，只有当有其他可见窗口可以提供或接收额外的终端空间时，才可以调整一个窗口的大小。

tui window width name +count ¶
tui window width name -count
winwidth name +count
winwidth name -count

更改窗口的宽度 名称 按 数量 列。正数量增加宽度，负数量减小宽度。 名称 参数可以是当前任何可见窗口的名称。可以使用以下命令发现当前可见窗口的名称 info win （参见 info win）。

当前可见窗口集必须始终填满终端，因此，只有当有其他可见窗口可以提供或接收额外的终端空间时，才可以调整一个窗口的大小。
