22.9 可选的警告和消息 ¶

默认情况下， GDB 对其内部工作情况保持沉默。如果你在一台运行缓慢的机器上运行，你可能想使用 set verbose 命令。这会使 GDB 在进行冗长的内部操作时告知你，这样你就不会认为它崩溃了。

目前，由 set verbose 控制的消息是那些宣布正在读取源文件的符号表的消息；请参阅 symbol-file 在 指定文件的命令.

set verbose on ¶

启用 GDB 某些信息性消息的输出。

set verbose off

禁用 GDB 某些信息性消息的输出。

show verbose ¶

显示 set verbose 是开启还是关闭。

默认情况下，如果 GDB 在目标文件的符号表中遇到错误，它会保持沉默；但如果你正在调试编译器，你可能会发现这些信息很有用（请参阅 读取符号文件时的错误).

set complaints limit ¶

允许 GDB 输出 限制 每种异常符号的抱怨信息，之后对该问题保持沉默。将 限制 设置为零以抑制所有抱怨；将其设置为一个大数字以防止抱怨被抑制。

show complaints ¶

显示 GDB 被允许产生多少条符号抱怨信息。

默认情况下， GDB 很谨慎，有时会问很多看似愚蠢的问题来确认某些命令。例如，如果你试图运行一个已经在运行的程序： 

(gdb) run
正在调试的程序已经启动。
要从头开始启动它吗？(y 或 n)

如果你愿意毫不退缩地面对自己命令的后果，你可以禁用这个 “功能”：

set confirm off ¶

禁用确认请求。请注意，使用 GDB 带 --batch 选项（请参阅 -batch) 运行时也会自动禁用确认请求。

set confirm on

启用确认请求（默认设置）。

show confirm ¶

显示确认请求的状态。

如果你需要调试用户定义的命令或源文件，你可能会发现启用 命令跟踪. 在此模式下，每个命令在执行时都会被打印出来，前面会加上一个或多个 ‘+’ 符号，符号的数量表示每个命令的调用深度。

set trace-commands on ¶

启用命令跟踪。

set trace-commands off

禁用命令跟踪。

show trace-commands

显示命令跟踪的当前状态。
