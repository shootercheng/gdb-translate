4.5 您程序的工作目录 ¶

每次您使用 run启动程序时，被调试程序将使用由 set cwd 命令指定的当前工作目录进行初始化。如果此命令未指定任何目录，那么在本地调试时，被调试程序将继承 GDB的当前工作目录作为其工作目录；在远程调试时，它将继承远程服务器的当前工作目录。

set cwd [directory]

将被调试程序的工作目录设置为 目录，该目录将进行 glob扩展以解析波浪号（~）。如果未指定任何参数，该命令将清除设置并将其重置为空状态。此设置对 GDB的工作目录没有影响，并且仅在下次启动被调试程序时生效。 ~ 在 目录 中是 主目录的缩写，通常由 HOME 环境变量指向。在MS - Windows系统中，如果 HOME 未定义， GDB 将使用 HOMEDRIVE 和 HOMEPATH 的拼接作为备用。

您还可以使用 GDB的 cd 命令来更改当前工作目录。请参阅 cd命令.

show cwd ¶

显示被调试程序的工作目录。如果 set cwd未指定任何目录，那么默认的被调试程序工作目录与 GDB的工作目录相同。

cd [directory]

将 GDB 工作目录设置为 目录。如果未给出 目录 则使用 '~'.

该 GDB 工作目录作为指定文件供 GDB 操作的命令的默认目录。请参阅 指定文件的命令. 请参阅 set cwd命令.

pwd ¶

打印 GDB 工作目录。

通常不可能找到正在调试的进程的当前工作目录（因为程序在运行过程中可以更改其目录）。如果您使用的系统中 GDB 支持 info proc 命令（请参阅 进程信息），您可以使用 info proc 命令来找出被调试程序的当前工作目录。
