2.1.4 初始化文件 ¶

启动期间（请参阅 什么 GDB 启动期间做什么) GDB 将从几个初始化文件中执行命令。这些初始化文件使用与 命令文件 （请参阅 命令文件) 相同的语法处理，并且以相同的方式由 GDB 处理。

要显示 GDB 在启动时加载的初始化文件列表，按加载顺序显示，可以使用 gdb --help.

早期初始化 文件在 GDB的初始化过程中非常早地加载，在解释器（请参阅 命令解释器) 初始化之前，以及默认目标（请参阅 指定调试目标) 初始化之前。只有 set 或 source 命令应该放在早期初始化文件中，并且唯一可以使用的 set 命令是那些控制 GDB 启动方式的命令。

可以放在早期初始化文件中的命令将在本手册中进行相应的记录。任何未记录为适合早期初始化文件的命令都应该放在通用初始化文件中。传递给 --early-init-command 或 -eix 的命令文件也是早期初始化文件，具有相同的命令限制。只有可以出现在早期初始化文件中的命令才应该传递给 --early-init-eval-command 或 -eiex.

相比之下， 通用初始化 文件在 GDB 完成其自身的内部初始化过程之后稍后处理，这些文件中可以使用任何有效的命令。

在本文档的其余部分，术语 初始化文件 指的是通用初始化文件之一，而不是早期初始化文件。任何关于早期初始化文件的讨论都会特别提及正在讨论的是早期初始化文件。

由于系统范围和主目录初始化文件在大多数命令行选项之前处理，因此设置的更改（例如 ‘set complaints’）可能会影响后续命令行选项和操作数的处理。

以下部分描述了 GDB 在哪里查找早期初始化文件和初始化文件，以及搜索文件的顺序。

  • 主目录早期初始化文件
  • 系统范围的初始化文件
  • 主目录初始化文件
  • 本地目录初始化文件

2.1.4.1 主目录早期初始化文件 ¶

GDB 最初会在用户主目录中查找早期初始化文件1. 有多个位置， GDB 会在主目录中搜索，这些位置会按顺序搜索，并且 GDB 会加载找到的第一个文件，后续位置将不再检查。

在非 macOS 主机上，搜索的位置是：

  • 文件 gdb/gdbearlyinit 在由环境变量指向的目录中，如果该环境变量已定义。 XDG_CONFIG_HOME
  • 文件 .config/gdb/gdbearlyinit 在由环境变量指向的目录中，如果该环境变量已定义。 HOME
  • 文件 .gdbearlyinit 在由环境变量指向的目录中，如果该环境变量已定义。 HOME

相比之下，在 macOS 主机上，搜索的位置是：

  • 文件 Library/Preferences/gdb/gdbearlyinit 在由环境变量指向的目录中，如果该环境变量已定义。 HOME
  • 文件 .gdbearlyinit 在由环境变量指向的目录中，如果该环境变量已定义。 HOME

可以使用 ‘-nx’ 或 ‘-nh’ 命令行选项来阻止加载主目录早期初始化文件，请参阅 选择模式.

2.1.4.2 系统范围的初始化文件 ¶

有两个位置会搜索系统范围的初始化文件。这两个位置总是会被检查：

system.gdbinit

这是一个单一的系统范围的初始化文件。其位置由 --with-system-gdbinit 配置选项 （请参阅 系统范围的配置和设置）。当 GDB 启动时，它会在处理命令行选项之前首先被加载。

system.gdbinit.d

这是系统范围的初始化目录。其位置由 --with-system-gdbinit-dir 配置选项 （请参阅 系统范围的配置和设置）指定。此目录中的文件会在 system.gdbinit （如果启用）之后，按字母顺序立即加载，当 GDB 启动时，在处理命令行选项之前。文件需要有公认的脚本语言扩展名（.py/.scm），或者以 .gdb 扩展名命名，才能被解释为常规 GDB 命令。 GDB 不会递归到该目录的任何子目录中。

可以使用 ‘-nx’ 命令行选项来阻止加载系统范围的初始化文件，请参阅 选择模式。

2.1.4.3 主目录初始化文件 ¶

在加载系统范围的初始化文件之后， GDB 会在用户主目录中查找初始化文件2。有多个位置， GDB 会在主目录中搜索，这些位置会按顺序搜索，并且 GDB 会加载找到的第一个文件，后续位置将不再检查。

在非苹果主机上，搜索的位置是：

$XDG_CONFIG_HOME/gdb/gdbinit
$HOME/.config/gdb/gdbinit
$HOME/.gdbinit

而在苹果主机上，搜索的位置是：

$HOME/Library/Preferences/gdb/gdbinit
$HOME/.gdbinit

可以使用 ‘-nx’ 或 ‘-nh’ 命令行选项来阻止加载主目录初始化文件，请参阅 选择模式。

DJGPP 版本的 GDB 使用名称 gdb.ini 而不是 .gdbinit 或 gdbinit, 这是由于 DOS 文件系统对文件名的限制。Windows 版本的 GDB 使用标准名称，但如果在主目录中找到 gdb.ini 文件，会警告你并建议将文件重命名为标准名称。

2.1.4.4 本地目录初始化文件 ¶

GDB 会检查当前目录中是否有一个名为 .gdbinit. 的文件。它会在处理除 ‘-x’ 和 ‘-ex’ 之外的命令行选项之后最后加载。命令行选项 ‘-x’ 和 ‘-ex’ 会在 .gdbinit 加载之后最后处理，请参阅 选择文件。

如果当前目录中的文件已经作为主目录初始化文件加载过，则不会再次加载。

可以使用 ‘-nx’ 命令行选项来阻止加载本地目录初始化文件，请参阅 选择模式.

脚注

(1)

在 DOS/Windows 系统中，主目录是由 HOME 环境变量指向的目录。

(2)

在 DOS/Windows 系统中，主目录是由 HOME 环境变量指向的目录。
