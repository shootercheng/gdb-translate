以下与录制相关的函数（参见 ）在 gdb 模块：

函数： gdb.start_recording ([方法], [格式]) ¶

使用给定的 方法 和 格式开始录制。如果没有提供 格式 ，则使用录制方法的默认格式。如果没有提供 方法 ，则使用默认方法。成功时返回一个 gdb.Record 对象。失败时抛出异常。

以下字符串可以作为 方法传递：

  • "full"
  • "btrace"： 格式的可能值： "pt", "bts" 或者留空使用默认格式。
函数： gdb.current_recording () ¶

访问当前正在运行的录制。成功时返回一个 gdb.Record 对象。如果当前没有活动的录制，则返回 None 。

函数： gdb.stop_recording () ¶

停止当前的录制。如果当前没有活动的录制，则抛出异常。此调用后，所有记录对象将失效。

一个 gdb.Record 对象具有以下属性：

变量： Record.method ¶

一个包含当前录制方法的字符串，例如 full 或 btrace。

变量： Record.format ¶

一个包含当前录制格式的字符串，例如 bt, pts 或 None。

变量： Record.begin ¶

一个特定于方法的指令对象，表示此录制中的第一条指令。

变量： Record.end ¶

一个特定于方法的指令对象，表示当前指令，但实际上不是录制的一部分。

变量： Record.replay_position ¶

表示当前重放位置的指令。如果没有活动的重放，这将是 None。

变量： Record.instruction_history ¶

一个包含所有录制指令的列表。

变量： Record.function_call_history ¶

一个包含所有录制的函数调用段的列表。

一个 gdb.Record 对象具有以下方法：

函数： Record.goto （指令） ¶

将重放位置移动到给定的 指令。

函数： Record.clear () ¶

清除当前录制的跟踪数据。这会强制对后续命令的跟踪进行重新解码。

录制方法特定的指令对象所继承的通用 gdb.Instruction 类具有以下属性：

变量： Instruction.pc ¶

一个表示此指令地址的整数。

变量： Instruction.data ¶

一个 memoryview 对象，保存原始指令数据。

变量： Instruction.decoded ¶

一个包含反汇编指令的可读字符串。

变量： Instruction.size ¶

指令的字节大小。

此外 gdb.RecordInstruction 具有以下属性：

变量： RecordInstruction.number ¶

一个标识此指令的整数。 number 对应于在 record instruction-history （参见 录制被调试程序的执行并进行重放）中看到的编号。

变量： RecordInstruction.sal ¶

一个 gdb.Symtab_and_line 对象，表示此指令关联的符号表和行。如果没有可用的调试信息，可能为 None 。

变量： RecordInstruction.is_speculative ¶

一个布尔值，指示该指令是否是推测执行的。

如果在录制或解码录制过程中发生错误，此错误由指令列表中的一个 gdb.RecordGap 对象表示。它具有以下属性：

变量： RecordGap.number ¶

一个标识此间隙的整数。 number 对应于在 record instruction-history （参见 录制被调试程序的执行并进行重放）中看到的编号。

变量： RecordGap.error_code ¶

间隙原因的数值表示。该值特定于当前的录制方法。

变量： RecordGap.error_string ¶

一个包含间隙原因的可读字符串。

一些 GDB 功能会将辅助信息写入执行历史记录。此信息由指令列表中的一个 gdb.RecordAuxiliary 对象表示。它具有以下属性：

变量： RecordAuxiliary.number ¶

一个标识此辅助信息的整数。 编号 对应于在 record instruction-history （参见 录制被调试程序的执行并进行重放）中看到的编号。

变量： RecordAuxiliary.data ¶

辅助数据的字符串表示。

一个 gdb.RecordFunctionSegment 对象具有以下属性：

变量： RecordFunctionSegment.number ¶

一个标识此函数段的整数。 number 对应于在 record function-call-history （参见 录制被调试程序的执行并进行重放）中看到的编号。

变量： RecordFunctionSegment.symbol ¶

一个 gdb.Symbol 对象，表示关联的符号。如果没有可用的调试信息，可能为 None 。

变量： RecordFunctionSegment.level ¶

一个表示函数调用栈级别的整数。如果函数调用是一个间隙，可能为 None 。

变量： RecordFunctionSegment.instructions ¶

一个包含 gdb.RecordInstruction 或 gdb.RecordGap 对象的列表，与该函数调用相关联。

变量： RecordFunctionSegment.up ¶

一个 gdb.RecordFunctionSegment 对象，表示调用者的函数段。如果调用未被录制，这将是控制返回的函数段。如果调用和返回都未被录制，这将是 None。

变量： RecordFunctionSegment.prev ¶

一个 gdb.RecordFunctionSegment 对象，表示此函数调用的前一个段。可能为 None。

变量： RecordFunctionSegment.next ¶

一个 gdb.RecordFunctionSegment 对象，表示此函数调用的下一个段。可能为 None。

以下示例演示了如何使用这些对象和函数来创建一个函数，该函数将记录倒回上一次执行不同文件中的函数的时间。这通常用于跟踪库中用户提供的回调函数的执行，这些函数通常在回溯中不可见。 

def bringback ():
    rec = gdb.current_recording ()
    if not rec:
        return

    insn = rec.instruction_history
    if len (insn) == 0:
        return

    try:
        position = insn.index (rec.replay_position)
    except:
        position = -1
    try:
        filename = insn[position].sal.symtab.fullname ()
    except:
        filename = None

    for i in reversed (insn[:position]):
	try:
            current = i.sal.symtab.fullname ()
	except:
            current = None

        if filename == current:
            continue

        rec.goto (i)
        return

另一个可能的应用是编写一个函数，用于计算给定行范围内代码执行的次数。这个行范围可以包含函数的部分内容，也可以跨越多个函数，并且不要求是连续的。 

def countrange (filename, linerange):
    count = 0

    def filter_only (file_name):
        for call in gdb.current_recording ().function_call_history:
            try:
                if file_name in call.symbol.symtab.fullname ():
                    yield call
            except:
                pass

    for c in filter_only (filename):
        for i in c.instructions:
            try:
                if i.sal.line in linerange:
                    count += 1
                    break;
            except:
                    pass

    return count
