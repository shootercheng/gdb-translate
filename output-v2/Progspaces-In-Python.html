23.3.2.26 Python 中的程序空间 ¶

程序空间，表示地址空间的符号视图。 它由程序的所有目标文件组成。 请参阅 Python 中的目标文件 程序空间，以获取有关程序空间的更多详细信息。

以下与程序空间相关的函数在 gdb 模块中可用：

函数： gdb.current_progspace () ¶

此函数返回当前选定 inferior 的程序空间。 请参阅 调试多个 inferior 连接和程序。 这与 gdb.selected_inferior().progspace （请参阅 Python 中的 Inferiors）相同，包含此函数是为了保持历史兼容性。

函数： gdb.progspaces () ¶

返回当前已知的所有程序空间的序列。 GDB。

每个程序空间由 gdb.Progspace 类的一个实例表示。

变量： Progspace.filename ¶

程序空间的主符号文件（已从中加载调试符号）的文件名，以字符串形式表示，例如 symbol-file 或 file 命令的参数。

如果当前没有加载主符号表，那么此属性将为 None。

变量： Progspace.symbol_file ¶

表示 gdb.Objfile 的 gdb.Progspace。 这是由 symbol-file 或 file 命令设置的符号文件。

这将是表示 gdb.Objfile 的 Progspace.filename ，当 Progspace.filename 不是 None时。

如果当前没有加载主符号表，那么此属性将为 None。

如果 Progspace 无效，即当 Progspace.is_valid() 返回 False时，尝试访问此属性将引发 RuntimeError 异常。

变量： Progspace.executable_filename ¶

此程序空间正在使用的可执行文件的文件名，以字符串形式表示。 可执行文件是在使用本地目标时 GDB 为了启动一个 inferior 而调用的文件。 此属性中的文件名由 exec-file 和 file 命令更新。

如果当前在此 Progspace 中没有设置可执行文件，那么此属性包含 None。

如果 Progspace 无效，即当 Progspace.is_valid() 返回 False时，尝试访问此属性将引发 RuntimeError 异常。

变量： Progspace.pretty_printers ¶

该 pretty_printers 属性是一个函数列表。 它用于查找漂亮打印机。 一个 Value 会依次传递给每个函数；如果函数返回 None，则继续搜索。 否则，返回值应该是一个用于格式化值的对象。 请参阅 漂亮打印 API，以获取更多信息。

变量： Progspace.type_printers ¶

该 type_printers 属性是一个类型打印机对象列表。 请参阅 类型打印 API，以获取更多信息。

变量： Progspace.frame_filters ¶

该 frame_filters 属性是一个帧过滤器对象的字典。 请参阅 过滤帧，以获取更多信息。

变量： Progspace.missing_file_handlers ¶

该 missing_file_handlers 属性是一个元组列表。 每个元组为这个程序空间保存一个缺失文件处理程序对象。 有关更多信息，请参阅 Python 中的缺失调试信息和 Python 中的缺失目标文件。

一个程序空间有以下方法：

函数： Progspace.block_for_pc (pc) ¶

返回包含给定 gdb.Block 的最内层 pc 值。 如果无法为指定的 pc 值找到块，该函数将返回 None。

函数： Progspace.find_pc_line (pc) ¶

返回与 gdb.Symtab_and_line 对应的 pc 值。 请参阅 Python 中的符号表表示。 如果传递了无效的 pc 值作为参数，那么返回的 symtab 和 line 属性将分别为 gdb.Symtab_and_line 和 0。 None

函数： Progspace.is_valid () ¶

如果 True 对象有效，则返回 gdb.Progspace ，否则返回 False 。 如果程序空间文件没有被任何 inferior 引用，那么 gdb.Progspace 对象可能会变得无效。 如果在调用方法时对象无效，所有其他 gdb.Progspace 方法将抛出异常。

函数： Progspace.objfiles () ¶

返回此程序空间引用的所有目标文件的序列。 请参阅 Python 中的目标文件。

函数： Progspace.solib_name (address) ¶

以字符串形式返回包含给定 地址 的共享库的名称，或者 None。

函数： Progspace.objfile_for_address (address) ¶

返回包含给定地址的 gdb.Objfile ，如果没有目标文件覆盖该地址，则返回 None 。

可以用通常的 Python 方式为 gdb.Progspace 对象添加任意属性。 例如，如果需要对与程序空间相关的信息进行额外的记录，这将很有用。

请参阅 选择属性名称，以获取为新属性选择合适名称的指导。

在这个人为设计的示例中，我们希望在加载包含某个符号的目标文件时执行一些处理，但由于处理成本较高，我们只想执行一次。 为了实现这一点，我们使用程序空间来记录结果，因为我们无法预测所需的目标文件何时会被加载。 

(gdb) python

def clear_objfiles_handler(event):
    event.progspace.expensive_computation = None
def expensive(symbol):
    """一个模拟例程，用于对符号执行“昂贵”的计算。"""
    print ("正在计算终极问题的答案 ...")
    return 42

def new_objfile_handler(event):
    objfile = event.new_objfile
    progspace = objfile.progspace
    if not hasattr(progspace, 'expensive_computation') or \
            progspace.expensive_computation is None:
        # 为了简化示例，我们使用 'main' 作为符号。
        # 注意：目前没有办法将查找范围限制在一个目标文件内。
        symbol = gdb.lookup_global_symbol('main')
        if symbol is not None:
            progspace.expensive_computation = expensive(symbol)
gdb.events.clear_objfiles.connect(clear_objfiles_handler)
gdb.events.new_objfile.connect(new_objfile_handler)
end

(gdb) file /tmp/hello
正在从 /tmp/hello 读取符号...
正在计算终极问题的答案 ...
(gdb) python print(gdb.current_progspace().expensive_computation)
42
(gdb) run
开始执行程序：/tmp/hello
你好。
[Inferior 1 (进程 4242) 正常退出]
