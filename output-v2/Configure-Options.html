¶

配置 GDB （autoconf）运行 配置脚本, 以获取 配置的完整解释。 

配置 [--帮助]
          [--前缀=目录]
          [--执行前缀=目录]
          [--源目录=目录名]
          [--目标=目标]

您可以使用单个‘-’而不是‘--’来引入选项；但如果使用‘--’，则可以缩写选项名称。

--help

显示如何调用 配置的快速摘要。

--prefix=dir

配置源代码，以便将程序和文件安装到目录 目录下。

--exec-prefix=dir

配置源代码，以便将程序安装到目录 目录下。

--srcdir=dirname

使用此选项可以在与源目录分开的目录中进行配置。除此之外，您可以使用此选项在不同的目录中同时构建（或维护）多个配置。 GDB 配置 在当前目录中写入特定于配置的文件，但安排它们使用目录 目录名中的源文件。 配置 在工作目录下创建与 目录名以下的源目录平行的目录。

--target=target

配置 GDB 用于对在指定的 目标上运行的程序进行交叉调试。如果没有此选项， GDB 将配置为调试在与主机相同的机器上运行的程序，即 GDB 它自身所在的机器。

没有方便的方法来生成所有可用目标的列表。另请参阅下面的 --enable-targets 选项。

有许多特定于 GDB的其他选项。这里只列出了最常见的选项；还有一些非常专业的选项未在此处描述。

--enable-targets=[target]…
--enable-targets=all

配置 GDB 用于对在指定的目标列表上运行的程序进行交叉调试。特殊值‘所有’将配置 GDB 以调试它支持的任何目标上运行的程序。

--enable-binary-file-formats=[format]…
--enable-binary-file-formats=all

配置 GDB 以支持某些二进制文件格式。如果某种格式是给定目标的主要（或唯一）文件格式，配置脚本可能无论如何都会添加对它的支持，并向用户发出警告。如果未指定，将编译 GDB 支持的所有文件格式。

--with-gdb-datadir=path

设置 GDB特定的数据目录。 GDB 将在此处查找某些支持文件或脚本。这默认是 gdb 子目录，位于‘数据目录’（可以使用 --datadir设置）之下。

--with-relocated-sources=dir

设置默认的源路径替换规则，以便调试信息中记录的目录名会自动针对以下任何目录进行调整 目录。 目录 应该是 GDB配置前缀的子目录，即 --prefix 或 --exec-prefix 配置选项中提到的那个。如果 GDB 在构建后要移动到不同的位置，此选项很有用。

--enable-64-bit-bfd

在 32 位主机上启用 BFD 中的 64 位支持。

--disable-gdbmi

构建 GDB 时不使用 GDB/MI 机器接口（请参阅 GDB/MI 接口）。

--enable-tui

构建 GDB 时使用文本模式全屏用户界面（TUI）。需要 curses 库（也支持 ncurses 和 cursesX）。

--with-curses

对于文本模式终端操作，使用 curses 库而不是 termcap 库。

--with-debuginfod

构建 GDB 时使用 libdebuginfod, 即 debuginfod 客户端库。用于根据与任何缺失文件关联的构建 ID 自动从 debuginfod 服务器获取 ELF、DWARF 和源文件。如果在配置时安装并找到了 libdebuginfod ，则默认启用此功能。有关 debuginfod 的更多信息，请参阅 使用 Debuginfod 下载调试资源。

--with-libunwind-ia64

在 ia64 目标平台上使用 libunwind 库展开函数调用栈。请参阅 http://www.nongnu.org/libunwind/index.html 了解详情。

--with-system-readline

使用主机上安装的 readline 库，而不是作为 GDB一部分提供的库。需要 Readline 7 或更高版本；构建系统会强制执行此要求。

--with-system-zlib

使用主机上安装的 zlib 库，而不是作为 GDB一部分提供的库。

--with-expat

构建 GDB 时使用 Expat（一个用于 XML 解析的库）。（如果在配置时安装并找到了 libexpat，则默认会这样做。）此库用于读取随 GDB提供的 XML 文件。如果不可用，一些基于 XML 文件的功能，如远程协议内存映射、目标描述和共享库列表，在 GDB中将不可用。如果您的主机没有安装 libexpat，您可以从 http://expat.sourceforge.net获取最新版本。

--with-libiconv-prefix[=dir]

构建 GDB 时使用 GNU libiconv（一个字符集编码转换库）。默认情况下不会这样做，因为在 GNU 系统上， iconv C 库中内置的功能就足够了。如果您的主机没有可用的 iconv, 您可以从 https://www.gnu.org/software/libiconv/获取最新版本的 GNU iconv。

GDB的构建系统还支持将 GNU libiconv 作为整体构建的一部分进行构建。请参阅 构建要求 GDB。

--with-lzma

构建 GDB 时使用 LZMA（一个压缩库）。（如果在配置时安装并找到了 liblzma，则默认会这样做。）LZMA 用于 GDB的“迷你调试信息”功能，该功能仅在使用 ELF 对象文件格式的平台上有用。如果您的主机没有安装 liblzma，您可以从 https://tukaani.org/xz/获取最新版本。

--with-python[=python]

构建 GDB 时支持 Python 脚本。（如果在配置时存在并找到了 libpython，则默认会这样做。）Python 使 GDB 脚本编写比受限的 CLI 脚本语言强大得多。如果您的主机没有安装 Python，您可以在 http://www.python.org/download/找到它。GDB 支持的最旧 Python 版本是 3.4。可选参数 python 用于查找 Python 头文件和库。它可以是 Python 可执行文件的名称，也可以是 Python 安装目录的名称。

--with-guile[=guile]

构建 GDB 时支持 GNU Guile 脚本。（如果在配置时存在并找到了 libguile，则默认会这样做。）如果您的主机没有安装 Guile，您可以在 https://www.gnu.org/software/guile/找到它。可选参数 guile 可以是一个版本号，这将使 configure 尝试使用该版本的 Guile；也可以是一个 pkg-config 可执行文件的文件名，将查询该文件以获取编译和链接 Guile 所需的信息。

--without-included-regex

不使用随 GDB （作为 libiberty 库的一部分）包含的正则表达式库。在使用 GNU C 库 2 版本的主机上，这是默认设置。

--with-sysroot=dir

使用 目录 作为文件名以 /lib或 /usr/lib'开头的库的默认系统根目录。（ 目录 的值可以在运行时使用 set sysroot 命令进行修改。）如果 目录 在 GDB 配置前缀（使用 --prefix 或 --exec-prefix options设置）之下，当 GDB 移动到不同位置时，默认系统根目录将自动调整。

--with-system-gdbinit=file

配置 GDB 以自动加载系统范围的初始化文件。 文件 应该是一个绝对文件名。如果 文件 位于配置前缀下的目录中，并且 GDB 在构建后移动到另一个位置，系统范围初始化文件的位置将相应调整。

--with-system-gdbinit-dir=directory

配置 GDB 以自动从系统范围的目录加载初始化文件。 目录 应该是一个绝对目录名。如果 目录 位于配置前缀下的目录中，并且 GDB 在构建后移动到另一个位置，系统范围初始化目录的位置将相应调整。

--enable-build-warnings

在构建 GDB 源代码时，要求编译器对任何看起来有点可疑的代码发出警告。根据您使用的编译器的确切版本，它会传递许多不同的警告标志。

--enable-werror

将编译器警告视为错误。它会向编译器添加 -Werror 标志，如果编译器输出任何警告消息，编译将失败。

--enable-ubsan

启用 GCC 未定义行为检查器。默认情况下此功能是禁用的，但将 --enable-ubsan=yes 或 --enable-ubsan=auto 传递给 configure 将启用它。未定义行为检查器检查 C++ 未定义行为。它会有性能开销，因此如果您关注 GDB的性能，应该禁用它。未定义行为检查器在 GCC 4.9 中首次引入。
