E.19 跟踪帧信息格式 ¶

为了能够在检查跟踪点命中时知道可以检查 inferior 中的哪些对象， GDB 需要获取在跟踪帧中收集的内存范围、寄存器和跟踪状态变量的列表。

此列表是使用 ‘qXfer:traceframe-info:read’ （请参阅 qXfer 跟踪帧信息读取) 数据包获取的，并且是一个 XML 文档。

GDB 必须与 Expat 库链接以支持 XML 跟踪帧信息发现。 请参阅 Expat.

文档的顶级结构如下所示： 

<?xml version="1.0"?>
<!DOCTYPE traceframe-info
          PUBLIC "+//IDN gnu.org//DTD GDB Memory Map V1.0//EN"
                 "http://sourceware.org/gdb/gdb-traceframe-info.dtd">
<traceframe-info>
   block...
</traceframe-info>

每个跟踪帧块可以是以下两种类型之一：

  • 从 地址 开始收集的内存区域， 长度 为从该地址起的字节数： 
    <memory start="地址" length="长度"/>
  • 一个块，表示编号为 编号 的跟踪状态变量已被收集： 
    <tvar id="编号"/>

跟踪帧信息格式的正式 DTD 如下所示： 

<!ELEMENT traceframe-info  (memory | tvar)* >
<!ATTLIST traceframe-info  version CDATA   #FIXED  "1.0">

<!ELEMENT memory        EMPTY>
<!ATTLIST memory        start   CDATA   #REQUIRED
                        length  CDATA   #REQUIRED>
<!ELEMENT tvar>
<!ATTLIST tvar          id      CDATA   #REQUIRED>
