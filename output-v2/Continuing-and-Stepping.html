5.2继续和单步执行 ¶

继续 意味着恢复程序执行，直到程序正常完成。相比之下， 单步执行 意味着仅再执行程序的一个“步骤”，其中“步骤”可能指一行源代码或一条机器指令（具体取决于您使用的特定命令）。无论是继续执行还是单步执行，由于断点或信号，程序可能会更快停止。（如果由于信号而停止，您可能想要使用 handle，或者使用 ‘信号0’来恢复执行（请参阅 信号）， 或者您可以单步进入信号的处理程序（请参阅 单步执行和信号处理程序）。）

continue [ignore-count] ¶
c [ignore-count]
fg [ignore-count]

在程序最后停止的地址处恢复程序执行； 在此地址设置的任何断点都将被绕过。可选参数 忽略计数 允许您指定在此位置进一步忽略断点的次数；其效果类似于 ignore （请参阅 断点条件）。

参数 忽略计数 仅在程序由于断点而停止时才有意义。在其他时候， continue 的参数将被忽略。

同义词 c 和 fg （用于 前台，因为被调试的程序被视为前台程序）纯粹是为了方便而提供的，并且具有与 continue完全相同的行为。

要在不同的位置恢复执行，您可以使用 return （请参阅 从函数返回）返回调用函数；或者 jump （请参阅 在不同地址继续）转到程序中的任意位置。

使用单步执行的一种典型技术是在函数或程序中被认为存在问题的部分的开头设置一个断点（请参阅 断点；观察点；和捕获点），运行程序直到它在该断点处停止，然后单步执行可疑区域，检查感兴趣的变量，直到您看到问题发生。

step ¶

继续运行程序，直到控制到达不同的源代码行，然后停止并将控制权返回给 GDB。此命令缩写为 s。

警告： 如果在没有调试信息的情况下编译的函数内使用 step 命令，执行将继续，直到控制到达有调试信息的函数。同样，它不会单步进入没有调试信息的函数。要单步执行没有调试信息的函数，请使用下面描述的 stepi 命令。

命令仅在源代码行的第一条指令处停止。这可以防止在 step command only stops at the first instruction of a source line. This prevents the multiple stops that could otherwise occur in switch 语句、 for 循环等中可能发生的多次停止。 step 如果在该行内调用了有调试信息的函数， step 将继续停止。换句话说， 将单步进入该行内调用的任何函数。

此外， step 命令仅在函数有行号信息时才进入函数。否则，它的行为类似于 next 命令。这避免了在使用 cc -gl 在 MIPS 机器上时出现的问题。以前， step 如果有关于该例程的任何调试信息，就会进入子例程。

step count

像在 step中一样继续运行，但这样做 计数 次。如果在 计数 步之前到达断点或发生与单步执行无关的信号，单步执行将立即停止。

next [count] ¶

继续到当前（最内层）堆栈帧中的下一行源代码。这类似于 step，但代码行内出现的函数调用将在不停止的情况下执行。当控制到达您发出 next 命令时正在执行的原始堆栈级别上的不同代码行时，执行停止。此命令缩写为 n。

参数 计数 是一个重复计数，与 step中的相同。

命令仅在源代码行的第一条指令处停止。这可以防止在 next command only stops at the first instruction of a source line. This prevents multiple stops that could otherwise occur in switch 语句、 for 循环等中可能发生的多次停止。

This is useful in cases where you may be interested in inspecting the machine instructions of a function which has no symbolic info and do not want GDB 自动跳过此函数的情况下很有用。

set step-mode off

导致 step 越过任何不包含调试信息的函数的命令。这是默认设置。

show step-mode

显示是否 GDB 将在没有源行调试信息的函数中停止或越过。

finish ¶

继续运行，直到选定堆栈帧中的函数返回后立即停止。打印返回值（如果有）。此命令可以缩写为 fin.

将此与 return 命令对比（见 从函数返回）。

set print finish [on|off] ¶
show print finish

默认情况下， finish 命令将显示函数返回的值。可以使用 set print finish off禁用此功能。禁用后，该值仍会输入到值历史记录中（见 值历史记录），但不会显示。

until ¶
u

继续运行，直到到达当前堆栈帧中当前行之后的源行。此命令用于避免多次单步执行循环。它类似于 next 命令，不同之处在于当 until 遇到跳转时，它会自动继续执行，直到程序计数器大于跳转地址。

这意味着当你单步执行完循环到达循环末尾时， until 会使你的程序继续执行，直到它退出循环。相比之下，循环末尾的 next 命令只会单步回到循环开头，这会迫使你单步执行下一次迭代。

继续运行你的程序，直到它到达通过解析 locspec得到的代码位置，或者当前堆栈帧返回。 locspec 是 位置规范中描述的任何一种形式。此命令与 until类似，但 advance 不会跳过递归函数调用，并且目标代码位置不必与当前位置在同一帧中。

stepi ¶
stepi arg
si

执行一条机器指令，然后停止并返回调试器。

在通过机器指令单步执行时，执行‘display/i $pc’通常很有用。这会使 GDB 在每次程序停止时自动显示下一条要执行的指令。见 自动显示。

参数是一个重复计数，如在 step中。

nexti ¶
nexti arg
ni

执行一条机器指令，但如果它是函数调用，则继续执行直到函数返回。

参数是一个重复计数，如在 next中。

默认情况下，如果可用， GDB 会利用 目标辅助范围单步执行 step。换句话说，每当你使用单步执行命令（例如 next， GDB ）时， GDB 会告诉目标单步执行相应的指令地址范围，而不是发出多个单步。这加快了行单步执行的速度，特别是对于远程目标。理想情况下，你应该没有理由想要关闭范围单步执行。然而，调试信息中的错误、远程存根（对于远程目标）中的错误，甚至

show range-stepping

控制是否启用范围单步执行。

如果 on，并且目标支持， GDB 会告诉目标自己单步执行一个地址范围，而不是发出多个单步。如果 off， GDB 总是发出单步，即使目标支持范围单步执行。默认值是 on。
