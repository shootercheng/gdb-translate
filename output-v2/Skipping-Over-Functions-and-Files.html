5.3 跳过函数和文件 ¶

您正在调试的程序可能包含一些对调试不感兴趣的函数。 skip 命令允许您指定 GDB 在单步执行时跳过一个函数、文件中的所有函数或特定文件中的特定函数。

例如，考虑以下 C 函数： 

101     int func()
102     {
103         foo(boring());
104         bar(boring());
105     }

假设您希望单步进入函数 foo 和 bar, 但您对单步执行 boring不感兴趣。如果您在第 103 行运行 step , 您将进入 boring(), 但如果您运行 next, 您将跳过 foo 和 boring!

一种解决方案是 step 进入 boring 并使用 finish 命令立即退出它。但如果 boring 从许多地方调用，这可能会变得很繁琐。

更灵活的解决方案是执行 skip boring. 这指示 GDB 永远不要单步进入 boring. 现在，当您在第 103 行执行 step , 您将跳过 boring 并直接进入 foo.

可以通过提供函数名、行规范（请参阅 位置规范）、与函数名匹配的正则表达式、文件名或与文件名匹配的 glob风格的模式来跳过函数。

在 Posix 系统上，正则表达式的形式是 “扩展正则表达式”。例如，请参阅 ‘man 7 regex’ 在 GNU/Linux 系统上。在非 Posix 系统上，正则表达式的形式是底层系统的 regcomp 函数提供的任何形式。 例如，请参阅 ‘man 7 glob’ 在 GNU/Linux 系统上对 glob风格模式的描述。

skip [options] ¶

命令的基本形式接受零个或多个选项，用于指定要跳过的内容。 skip 选项 参数是以下内容的任意有用组合：

-file file
-fi file

位于 文件 中的函数在单步执行时将被跳过。

-gfile file-glob-pattern ¶
-gfi file-glob-pattern

与 文件通配符模式 匹配的文件中的函数在单步执行时将被跳过。 

(gdb) skip -gfi utils/*.c
-function linespec
-fu linespec

由 行规范 指定的函数或包含由 行规范 指定的行的函数在单步执行时将被跳过。 请参阅 位置规范.

-rfunction regexp ¶
-rfu regexp

名称与 正则表达式 匹配的函数在单步执行时将被跳过。

这种形式对于复杂的函数名很有用。 例如，通常不需要单步进入 C++ std::string 构造函数或析构函数。此外，对于 C++ 模板，可能很难写出函数的完整名称，而且通常模板参数是什么并不重要。将要跳过的函数指定为正则表达式会使这变得更容易。 

(gdb) skip -rfu ^std::(allocator|basic_string)<.*>::~?\1 *(

如果您想跳过++ 构造函数和析构函数，您可以这样做： std 命名空间中的每个模板化 C 

(gdb) skip -rfu ^std::([a-zA-z0-9_]+)<.*>::~?\1 *(

如果未指定任何选项，则当前正在调试的函数将被跳过。

skip function [linespec] ¶

运行此命令后，由 行规范 指定的函数或包含由 行规范 指定的行的函数在单步执行时将被跳过。请参阅 位置规范.

如果不指定 行规范, 则当前正在调试的函数将被跳过。

(如果您有一个名为 file 的函数想要跳过，请使用 跳过函数 文件.)

skip file [filename] ¶

运行此命令后，任何源文件位于 文件名 的函数在单步执行时都将被跳过。 

(gdb) skip file boring.c
单步执行时将跳过文件 boring.c。

如果不指定 文件名, 则当前正在调试的文件中的函数将被跳过。

可以像管理断点一样列出、删除、禁用和启用跳过项。 以下是管理跳过项列表的命令：

info skip [range] ¶

打印指定跳过项的详细信息。如果 范围 未指定， 则打印一个表格，其中包含所有标记为跳过的函数和文件的详细信息。 info skip 打印每个跳过项的以下信息：

标识符

一个标识此跳过项的数字。

启用或禁用

启用的跳过项标记为 ‘是’。 禁用的跳过项标记为 ‘否’。

通配符

如果文件名是一个 ‘通配符’ 模式，则为 ‘是’。 否则为 ‘否’。

文件

要跳过的文件的名称或 ‘通配符’ 模式。 如果未指定文件，则为 ‘<无>’。

正则表达式

如果函数名是一个 ‘正则表达式’，则为 ‘是’。 否则为 ‘否’。

函数

要跳过的函数的名称或正则表达式。 如果未指定函数，则为 ‘<无>’。

skip delete [range] ¶

删除指定的跳过项。如果 范围 未指定，则删除所有跳过项。

skip enable [range] ¶

启用指定的跳过项。如果 范围 未指定，则启用所有跳过项。

skip disable [range] ¶

禁用指定的跳过项。如果 范围 未指定，则禁用所有跳过项。

set debug skip [on|off] ¶

设置是否打印有关跳过文件和函数的调试输出。

show debug skip ¶

显示是否打印有关跳过文件和函数的调试输出。
