22.8.4 显示尝试自动加载的文件 ¶

为了更清楚地了解可以放置脚本以便与被调试程序一起自动加载的所有文件位置，或者为了防止意外执行不可信的脚本， GDB 提供了一个功能，用于打印所有尝试加载的文件。既可以打印存在的文件，也可以打印不存在的文件。

例如，可以安全地自动加载文件的目录列表（请参阅 自动加载的安全限制）也适用于规范化的文件名，在设置时可能不太明显。 

(gdb) set debug auto-load on
(gdb) file ~/src/t/true
自动加载：为目标文件 "/tmp/true" 加载命令脚本 "/tmp/true - gdb.gdb" 的预定义序列。
自动加载：更新 "/usr:/opt" 目录。
自动加载：使用目录 "/usr"。
自动加载：使用目录 "/opt"。
警告：文件 "/tmp/true - gdb.gdb" 的自动加载已被您设置的 `auto - load safe - path` "/usr:/opt" 拒绝。
set debug auto-load [on|off] ¶

设置是否打印尝试自动加载的文件名。

show debug auto-load ¶

显示是否开启了打印尝试自动加载的文件名的功能。
