10.4 伪数组 ¶

在内存中打印出几个连续的相同类型的对象通常很有用；可以是数组的一部分，或者是程序中只有一个指针指向的动态确定大小的数组。

你可以通过将一段连续的内存区域称为 伪数组，使用二元运算符 ‘@’ 来实现。‘@’ 的左操作数应该是所需数组的第一个元素，并且是一个单独的对象。右操作数应该是数组的所需长度。结果是一个数组值，其元素的类型都与左参数的类型相同。第一个元素实际上就是左参数；第二个元素来自紧接着存储第一个元素的内存字节，依此类推。这里有一个示例。如果一个程序这样写 

int *array = (int *) malloc (len * sizeof (int));

你可以打印 array 的内容，使用 

p *array@len

‘@’ 的左操作数必须存在于内存中。用 ‘@’ 以这种方式创建的数组值在使用下标操作时的行为与其他数组一样，并且在表达式中使用时会被转换为指针。伪数组最常通过值历史记录（见 值历史记录）在打印出一个之后出现在表达式中。

另一种创建伪数组的方法是使用强制类型转换。这会将一个值重新解释为好像它是一个数组。该值不必存在于内存中： 

(gdb) p/x (short[2])0x12345678
$1 = {0x1234, 0x5678}

为了方便起见，如果你省略数组长度（如在 ‘(类型[])值’ 中） GDB 会计算出填充该值所需的大小（就像 ‘sizeof(值)/sizeof(类型)’ 那样： 

(gdb) p/x (short[])0x12345678
$2 = {0x1234, 0x5678}

有时伪数组机制还不够；在中等复杂的数据结构中，感兴趣的元素实际上可能并不相邻 —— 例如，如果你对数组中指针的值感兴趣。在这种情况下，一个有用的解决方法是使用便利变量（见 便利变量) 作为一个表达式中的计数器，该表达式打印出第一个感兴趣的值，然后通过 回车键重复该表达式。例如，假设你有一个 dtab 指向结构体的指针数组，并且你对每个结构体中的一个字段 fv 的值感兴趣。下面是你可能会输入的示例： 

set $i = 0
p dtab[$i++]->fv
回车键
