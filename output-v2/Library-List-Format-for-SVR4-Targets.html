E.16 SVR4目标的库列表格式 ¶

在SVR4平台上 GDB 可以使用动态加载器的符号表 （例如 ld.so）和正常的内存操作来维护共享库列表。不过，此数据包提供的特殊库列表对于 GDB 远程协议而言更为高效。

‘qXfer:libraries - svr4:read’数据包返回一个XML文档，该文档列出了已加载的库及其SVR4链接器参数。对于SVR4目标上的每个库，将报告以下参数：

  • name, 来自 l_name 字段的绝对文件名 struct link_map.
  • lm 的地址 struct link_map 用于TLS （线程本地存储）访问。
  • l_addr, 从字段读取的位移量 l_addr 的 struct link_map. 对于预链接的库，这不是一个绝对内存地址。它是在库加载期间，相对于文件预链接地址的绝对内存地址的位移量。
  • l_ld, 即 PT_DYNAMIC 段的内存地址
  • lmid, 这是链接器命名空间的标识符，例如 r_debug 包含此命名空间加载映射的对象的内存地址，或者由 dlinfo (3)返回的命名空间标识符。

此外，单个 main-lm 属性指定了 struct link_map 用于主可执行文件的地址。此参数用于TLS访问，并且它是可选的。

GDB 必须与Expat库链接以支持XML SVR4库列表。请参阅 Expat.

一个简单的内存映射，包含两个已加载的库（不使用预链接），如下所示： 

<library - list - svr4 version="1.0" main - lm="0xe4f8f8">
  <library name="/lib/ld - linux.so.2" lm="0xe4f51c" l_addr="0xe2d000"
           l_ld="0xe4eefc" lmid="0xfffe0"/>
  <library name="/lib/libc.so.6" lm="0xe4fbe8" l_addr="0x154000"
           l_ld="0x152350" lmid="0xfffe0"/>
</library - list - svr>

SVR4库列表的格式由以下DTD描述： 

<!-- library - list - svr4: 带有版本信息的根元素 -->
<!ELEMENT library - list - svr4  (library)*>
<!ATTLIST library - list - svr4  version CDATA   #FIXED  "1.0">
<!ATTLIST library - list - svr4  main - lm CDATA   #IMPLIED>
<!ELEMENT library            EMPTY>
<!ATTLIST library            name    CDATA   #REQUIRED>
<!ATTLIST library            lm      CDATA   #REQUIRED>
<!ATTLIST library            l_addr  CDATA   #REQUIRED>
<!ATTLIST library            l_ld    CDATA   #REQUIRED>
<!ATTLIST library            lmid    CDATA   #IMPLIED>
