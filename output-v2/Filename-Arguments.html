3.4 作为命令参数的文件名 ¶

当将文件名（或目录名）作为参数传递给命令时，如果文件名参数不包含任何空格、双引号或单引号，那么对于所有命令，文件名可以写成一个简单的字符串，例如： 

(gdb) file /path/to/some/file

如果文件名确实包含空格、双引号或单引号，那么 GDB 有两种方法来处理这些文件名的格式；使用哪种格式取决于正在使用的命令。

大多数 GDB 命令不需要，也不支持引号和转义。这些命令将命令名之后的任何文本（不是命令选项，见 命令选项）视为文件名，即使文件名包含空格或引号字符。在下面的示例中，用户正在添加 /path/that 包含/two 空格/ 到自动加载安全路径（见 add-auto-load-safe-path): 

(gdb) add-auto-load-safe-path /path/that contains/two spaces/

少数命令要求包含空格或引号字符的文件名要么加引号，要么用反斜杠转义特殊字符。支持这种风格的命令在手册中会有相应标记，任何未标记为接受其文件名参数加引号和转义的命令，都不接受这种文件名参数风格。

例如，要加载文件 /path/with 空格/to/a 文件 使用 file 命令（见 指定文件的命令），你可以用反斜杠转义空格字符： 

(gdb) file /path/with\ spaces/to/a\ file

或者，整个文件名可以用单引号或双引号括起来，在这种情况下不需要反斜杠，例如： 

(gdb) symbol-file "/path/with spaces/to/a file"
(gdb) exec-file '/path/with spaces/to/a file'

可以通过用反斜杠转义来在加引号的文件名中包含引号字符。例如，在双引号括起来的文件名中，双引号字符应该用反斜杠转义，但单引号字符不需要转义。在单引号字符串中，单引号字符需要转义，但双引号字符不需要。

也可以通过用反斜杠转义来包含一个字面反斜杠字符。
