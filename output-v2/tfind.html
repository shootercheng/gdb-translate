13.2.1 tfind n ¶

从缓冲区中选择跟踪快照的基本命令是 tfind n，它查找跟踪快照编号 n，从0开始计数。如果没有参数 n ，则选择下一个快照。

以下是使用该 tfind 命令的各种形式。

tfind start

查找缓冲区中的第一个快照。这是 tfind 0 （因为0是第一个快照的编号）的同义词。

tfind none

停止调试跟踪快照，恢复 实时 调试。

tfind end

与‘tfind none’相同。

tfind

无参数表示查找下一个跟踪快照，如果没有选择跟踪快照，则查找第一个。

tfind -

查找当前快照之前的上一个跟踪快照。这允许回溯更早的步骤。

tfind tracepoint num

查找与跟踪点 编号相关联的下一个快照。搜索从最后检查的跟踪快照向前进行。如果没有参数 编号 ，则表示查找与当前快照相同跟踪点收集的下一个快照。

tfind pc addr

查找与程序计数器的值 地址 相关联的下一个快照。搜索从最后检查的跟踪快照向前进行。如果没有参数 地址 ，则表示查找与当前快照具有相同PC值的下一个快照。

tfind outside addr1, addr2

查找其PC在给定地址范围之外（不包括）的下一个快照。

tfind range addr1, addr2

查找其PC在 地址1 和 地址2 之间（包括）的下一个快照。

tfind line [file:]n

查找与源行 编号相关联的下一个快照。如果给出可选参数 文件 ，则引用该源文件中的第 编号 行。搜索从最后检查的跟踪快照向前进行。如果没有参数 编号 ，则表示查找除当前正在检查的行之外的下一行；因此，反复说 tfind line 可能与在 实时 调试会话中逐行步进具有相同的效果。

这些 tfind 命令的默认参数经过专门设计，以便于浏览跟踪缓冲区。例如， tfind 无参数选择下一个跟踪快照，而 tfind - 无参数选择上一个跟踪快照。因此，通过给出一个 tfind 命令，然后简单地按 回车 键多次，您可以按顺序检查所有跟踪快照。或者，通过说 tfind - 然后按 回车 键多次，您可以按相反顺序检查快照。 tfind line 无参数的 tfind pc 命令选择执行的下一个源行的快照。无参数的 tfind tracepoint 命令选择与当前帧具有相同程序计数器（PC）的下一个快照。无参数的

除了让您手动浏览跟踪缓冲区之外，这些命令还便于构建 GDB 脚本，该脚本浏览跟踪缓冲区并打印出您感兴趣的任何收集数据。因此，如果我们想检查缓冲区中每个跟踪帧的PC、FP和SP寄存器，我们可以这样说： 

(gdb) tfind start
(gdb) while ($trace_frame != -1)
> printf "Frame %d, PC = %08X, SP = %08X, FP = %08X\n", \
          $trace_frame, $pc, $sp, $fp
> tfind
> end

或者，如果我们想检查缓冲区中每个源行的变量 X ： 

(gdb) tfind start
(gdb) while ($trace_frame != -1)
> printf "Frame %d, X == %d\n", $trace_frame, X
> tfind line
> end

Frame 0, X = 1
Frame 7, X = 2
Frame 13, X = 255
