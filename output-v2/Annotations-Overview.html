28.1 什么是注释？ ¶

注释以换行符、两个 ‘控制 - Z’ 字符和注释名称开头。如果没有与该注释相关的额外信息，注释名称后面紧接着一个换行符。如果有额外信息，注释名称后面跟着一个空格、额外信息和一个换行符。额外信息不能包含换行符。

任何不以换行符和两个 ‘控制 - Z’ 字符开头的输出表示来自 GDB. 目前不需要 GDB 输出一个换行符后接两个 ‘控制 - Z’ 字符，但如果有这样的需求，注释可以通过 ‘转义’ 注释进行扩展，该注释表示输出这三个字符。

注释 级别, 它使用 --annotate 命令行选项指定（请参阅 选择模式），控制 GDB 与其提示符、表达式的值、源代码行和其他类型的输出一起打印多少信息。级别 0 表示不使用注释，级别 1 用于 GDB 作为 GNU Emacs 的子进程运行时，级别 3 是适合控制 GDB的程序的最大注释级别，级别 2 的注释已被弃用（请参阅 注释接口的限制 在 GDB 的弃用注释）。

set annotate level ¶

该 GDB 命令 set annotate 将注释级别设置为指定的 级别.

show annotate ¶

显示当前的注释级别。

本章描述级别 3 的注释。

启动 GDB 并使用注释的一个简单示例是： 

$ gdb --annotate=3
GNU gdb 6.0
版权所有 2003 自由软件基金会，Inc.
GDB 是自由软件，受 GNU 通用公共许可证保护，
并且在某些条件下，欢迎您对其进行修改和/或分发副本。
输入 "show copying" 查看条件。
GDB 绝对没有任何保证。输入 "show warranty" 了解详情。
此 GDB 配置为 "i386 - pc - linux - gnu"

^Z^Zpre - prompt
(gdb)
^Z^Zprompt
quit

^Z^Zpost - prompt
$

这里 ‘quit’ 是输入给 GDB的；其余是来自 GDB的输出。以 ‘^Z^Z’ 开头的三行（其中 ‘^Z’ 表示一个 ‘控制 - Z’ 字符）是注释；其余是来自 GDB的输出。
