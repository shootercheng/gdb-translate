27.17 GDB/MI 跟踪点命令 ¶

本节中定义的命令实现了对跟踪点的 MI 支持。有关详细介绍，请参阅 跟踪点.

该 -trace-find 命令 ¶

概要 ¶

 -trace-find 模式 [参数...]

使用由 模式 和 参数定义的条件查找跟踪帧。下表列出了允许的模式及其参数。有关操作细节，请参阅 tfind n.

‘无’

无需参数。停止检查跟踪帧。

‘帧编号’

需要一个整数作为参数。选择具有该索引的跟踪点帧。

‘跟踪点编号’

需要一个整数作为参数。查找与指定编号的跟踪点对应的下一个跟踪帧。

‘程序计数器’

需要一个地址作为参数。查找与指定地址处的任何跟踪点对应的下一个跟踪帧。

‘程序计数器在范围内’

需要两个地址作为参数。查找与指定范围内地址处的跟踪点对应的下一个跟踪帧。两个边界都被视为在范围内。

‘程序计数器在范围外’

需要两个地址作为参数。查找与指定范围外地址处的跟踪点对应的下一个跟踪帧。两个边界都被视为在范围内。

‘行’

需要位置规范作为参数。请参阅 位置规范. 查找与指定位置处的跟踪点对应的下一个跟踪帧。

如果‘无’作为 模式传递，则响应没有字段。否则，响应可能具有以下字段：

‘找到’

此字段的值为‘0’或‘1’，具体取决于是否找到匹配的跟踪点。

‘跟踪帧’

找到的跟踪帧的索引。仅当‘找到’字段的值为‘1’时，此字段才会出现。

‘跟踪点’

找到的跟踪点的索引。仅当‘找到’字段的值为‘1’时，此字段才会出现。

‘帧’

与找到的跟踪帧对应的帧的信息。仅当找到跟踪帧时，此字段才会出现。请参阅 GDB/MI 帧信息, 以获取此字段的描述。

GDB 命令 ¶

对应的 GDB 命令是‘tfind’。

该 -trace-define-variable 命令 ¶

概要 ¶

 -trace-define-variable 名称 [ 值 ]

创建跟踪变量 名称 如果不存在。如果 值 被指定，则将指定跟踪变量的初始值设置为该值。请注意， 名称 应该以‘$’字符开头。

GDB 命令 ¶

对应的 GDB 命令是‘tvariable’。

该 -trace-frame-collected 命令 ¶

概要 ¶

 -trace-frame-collected
    [--var-print-values var_pval]
    [--comp-print-values comp_pval]
    [--registers-format regformat]
    [--memory-contents]

此命令返回在特定跟踪帧处收集的一组对象、寄存器名称、跟踪状态变量名称、内存范围和计算表达式。该命令的可选参数以不同方式影响输出格式。有关更多详细信息，请参阅下面的输出描述表。

报告的名称可以按常规方式用于创建varobjs并检查对象本身。此命令返回的项目按类别分类，以便清楚地知道哪个是变量、哪个是寄存器、哪个是跟踪状态变量、哪个是内存范围以及哪个是计算表达式。

例如，如果操作是 

收集 myVar, myArray[myIndex], myObj.field, myPtr->field, myCount + 2
收集 *(int*)0xaf02bef0@40

则完整收集的对象将是 myVar。该 myArray 对象 myIndex 将被部分收集，因为只收集索引处的元素。其余对象将是计算表达式。

一个示例输出将是： 

(gdb)
-trace-frame-collected
^done,
  explicit-variables=[{name="myVar",value="1"}],
  computed-expressions=[{name="myArray[myIndex]",value="0"},
                        {name="myObj.field",value="0"},
                        {name="myPtr->field",value="1"},
                        {name="myCount + 2",value="3"},
                        {name="$tvar1 + 1",value="43970027"}],
  registers=[{number="0",value="0x7fe2c6e79ec8"},
             {number="1",value="0x0"},
             {number="2",value="0x4"},
             ...
             {number="125",value="0x0"}],
  tvars=[{name="$tvar1",current="43970026"}],
  memory=[{address="0x0000000000602264",length="4"},
          {address="0x0000000000615bc0",length="4"}]
(gdb)

其中：

explicit-variables

已完整收集的对象集（与仅收集数组的几个元素或结构体的几个成员相对）。对于每个对象，则打印其名称和值。 --var-print-values 选项影响值字段的输出方式或是否输出。如果 var_pval 为0，则仅打印名称；如果为1，则也打印其值；如果为2，则打印简单数据类型的名称、类型和值，以及数组、结构体和联合体的名称和类型。

computed-expressions

在当前跟踪帧处收集的计算表达式集。 --comp-print-values 选项以与 --var-print-values 选项影响 explicit-variables 集相同的方式影响此集。请参阅上文。

registers

在当前跟踪帧处收集的寄存器。对于每个收集的寄存器，返回其名称和当前值。值根据 --registers-format 选项进行格式化。有关允许格式的列表，请参阅 -data-list-register-values 命令。默认值为‘x’。

tvars

在当前跟踪帧处收集的跟踪状态变量。对于每个收集的跟踪状态变量，返回其名称和当前值。

memory

在当前跟踪帧处收集的内存范围集。其内容是一个元组列表。每个元组表示一个收集的内存范围，并具有以下字段：

address

内存范围的起始地址，作为十六进制文字。

length

内存范围的长度，作为十进制文字。

contents

内存块的内容，以十六进制表示。仅当指定 --memory-contents 选项时才会出现此字段。

GDB 命令 ¶

没有对应的 GDB 命令。

示例 ¶

该 -trace-list-variables 命令 ¶

概要 ¶

 -trace-list-variables

返回所有已定义跟踪变量的表。表的每个元素具有以下字段：

‘名称’

跟踪变量的名称。此字段始终存在。

‘初始值’

初始值。这是一个64位有符号整数。此字段始终存在。

‘当前值’

跟踪变量当前时刻的值。这是一个64位有符号整数。如果当前值未定义，例如如果跟踪从未运行或当前正在运行，则此字段不存在。

GDB 命令 ¶

The corresponding GDB command is ‘tvariables’.

Example ¶

(gdb)
-trace-list-variables
^done,trace-variables={nr_rows="1",nr_cols="3",
hdr=[{width="15",alignment="-1",col_name="name",colhdr="Name"},
     {width="11",alignment="-1",col_name="initial",colhdr="Initial"},
     {width="11",alignment="-1",col_name="current",colhdr="Current"}],
body=[variable={name="$trace_timestamp",initial="0"}
      variable={name="$foo",initial="10",current="15"}]}
(gdb)

The -trace-save Command ¶

Synopsis ¶

 -trace-save [ -r ] [ -ctf ] filename

Saves the collected trace data to filename. Without the ‘-r’ option, the data is downloaded from the target and saved in a local file. With the ‘-r’ option the target is asked to perform the save.

By default, this command will save the trace in the tfile format. You can supply the optional ‘-ctf’ argument to save it the CTF format. See Using Trace Files for more information about CTF.

GDB Command ¶

The corresponding GDB command is ‘tsave’.

The -trace-start Command ¶

Synopsis ¶

 -trace-start

Starts a tracing experiment. The result of this command does not have any fields.

GDB Command ¶

The corresponding GDB command is ‘tstart’.

The -trace-status Command ¶

Synopsis ¶

 -trace-status

Obtains the status of a tracing experiment. The result may include the following fields:

‘supported’

May have a value of either ‘0’, when no tracing operations are supported, ‘1’, when all tracing operations are supported, or ‘file’ when examining trace file. In the latter case, examining of trace frame is possible but new tracing experiment cannot be started. This field is always present.

‘running’

May have a value of either ‘0’ or ‘1’ depending on whether tracing experiment is in progress on target. This field is present if ‘supported’ field is not ‘0’.

‘stop-reason’

Report the reason why the tracing was stopped last time. This field may be absent iff tracing was never stopped on target yet. The value of ‘request’ means the tracing was stopped as result of the -trace-stop command. The value of ‘overflow’ means the tracing buffer is full. The value of ‘disconnection’ means tracing was automatically stopped when GDB has disconnected. The value of ‘passcount’ means tracing was stopped when a tracepoint was passed a maximal number of times for that tracepoint. This field is present if ‘supported’ field is not ‘0’.

‘stopping-tracepoint’

The number of tracepoint whose passcount as exceeded. This field is present iff the ‘stop-reason’ field has the value of ‘passcount’.

‘帧’
‘已创建的帧’

‘帧’字段是跟踪缓冲区中跟踪帧总数的计数 ，而‘已创建的帧’是运行期间创建的总数 ，包括已丢弃的那些，例如当循环跟踪缓冲区已满时。这两个字段都是可选的。

‘缓冲区大小’
‘可用缓冲区’

这些字段表示跟踪缓冲区的当前大小和 剩余空间。这些字段是可选的。

‘循环的’

循环跟踪缓冲区标志的值。 1 表示 跟踪缓冲区是循环的，如果需要腾出空间，旧的跟踪帧将被丢弃， 0 表示跟踪缓冲区是线性的 并且可能会填满。

‘已断开连接’

已断开连接跟踪标志的值。 1 表示 断开连接后跟踪将继续 GDB ， 0 表示 跟踪运行将停止。

‘跟踪文件’

正在检查的跟踪文件的文件名。此字段是 可选的，并且仅在检查跟踪文件时出现。

GDB 命令 ¶

对应的 GDB 命令是‘tstatus’。

该 -trace-stop 命令 ¶

概要 ¶

 -trace-stop

停止跟踪实验。此命令的结果具有与 -trace-status相同的字段 ，除了‘支持的’和 ‘正在运行’字段不输出。

GDB 命令 ¶

对应的 GDB 命令是‘tstop’。
