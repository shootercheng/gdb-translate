10.24 值的大小 ¶

无论何时 GDB 打印一个值时，都会在内部分配内存 GDB 来保存该值的内容。在一些具有动态类型系统的语言中，一个无效的程序 可能会指示一个大小不正确的值，这反过来可能会导致 GDB 尝试分配过多的内存。

set max-value-size bytes ¶
set max-value-size unlimited

设置 GDB 为一个值的内容分配的最大内存大小为 字节，尝试显示一个需要的内存超过该大小的值将导致错误。

设置此变量不会影响已经在 GDB内分配的值，仅影响未来的分配。

有一个最小大小， max-value-size 可以设置为该大小，以便 GDB 仍然能够正确运行，目前这个最小值是16字节。

该限制适用于一些子表达式的结果以及完整的表达式。例如，一个表示简单 整数组件的表达式，如 x.y.z，如果 x.y 的大小是动态的并且超过 字节。另一方面， GDB 有时很智能；表达式 A[i]，其中 A 是一个大小非恒定的数组变量，通常无论 A的边界如何都会成功，只要组件 大小小于 字节。

的默认值目前是64k。 max-value-size
