8.5 对多个帧应用命令。 ¶

frame apply [all | count | -count | level level…] [option]… command

该 frame apply 命令允许你将指定的 命令 应用到一个或多个帧。

all

指定 all 以应用 命令 到所有帧。

count

使用 计数 以应用 命令 到最内层的 计数 个帧，其中 计数 是一个正数。

-count

使用 -计数 以应用 命令 到最外层的 计数 个帧，其中 计数 是一个正数。

level

使用 level 以应用 命令 到由 级别 列表标识的帧集合。 级别 是一个帧级别或帧级别的范围，格式为 级别1-级别2. 帧级别是 ‘backtrace’ 命令输出中第一个字段显示的数字。 例如，‘2-4 6-8 3’ 表示对级别为2、3、4、6、7、8的帧应用 命令 ，然后再次对级别为3的帧应用。

请注意， frame apply 应用命令的帧也会受到 set backtrace 设置的影响，例如 set backtrace past-main 和 set backtrace limit N. 请参阅 回溯.

该 frame apply 命令还支持一些选项，这些选项允许覆盖相关的 set backtrace 设置：

-past-main [on|off]

回溯是否应继续超过 main. 相关设置： set backtrace past-main.

-past-entry [on|off]

回溯是否应继续超过程序的入口点。 相关设置： set backtrace past-entry.

默认情况下， GDB 在 命令产生的输出之前显示一些帧信息，并且在执行 命令 期间引发的错误将中止 frame apply. 可以使用以下选项来微调这些行为：

-c

标志 -c, 它代表 ‘继续’，会导致 命令 中的任何错误被显示，然后继续执行 frame apply 。

-s

标志 -s, 它代表 ‘静默’，会导致 命令 产生的任何错误或空输出被静默忽略。 也就是说，执行继续，但不打印帧信息和错误。

-q

标志 -q (‘安静’) 禁用帧信息的打印。

以下示例展示了这些标志的使用方法 -c 和 -s 在应用该命令时是有效的 适用于所有帧，其中 变量 p j 只能在最外层成功打印 j 帧。 #1 main 

(gdb) frame apply all p j
#0  some_function (i=5) at fun.c:4
当前上下文中没有符号 "j"。
(gdb) frame apply all -c p j
#0  some_function (i=5) at fun.c:4
当前上下文中没有符号 "j"。
#1  0x565555fb in main (argc=1, argv=0xffffd2c4) at fun.c:11
$1 = 5
(gdb) frame apply all -s p j
#1  0x565555fb in main (argc=1, argv=0xffffd2c4) at fun.c:11
$2 = 5
(gdb)

默认情况下，‘frame apply’ 在命令输出之前打印帧位置 信息： 

(gdb) frame apply all p $sp
#0  some_function (i=5) at fun.c:4
$4 = (void *) 0xffffd1e0
#1  0x565555fb in main (argc=1, argv=0xffffd2c4) at fun.c:11
$5 = (void *) 0xffffd1f0
(gdb)

如果给出标志 -q 则不打印帧信息： 

(gdb) frame apply all -q p $sp
$12 = (void *) 0xffffd1e0
$13 = (void *) 0xffffd1f0
(gdb)
faas command ¶

的快捷方式 frame apply all -s command. 应用 命令 到所有帧，忽略错误和空输出。

例如，它可用于在不知道变量或参数所在帧的情况下打印局部变量或函数 参数，使用： 

(gdb) faas p some_local_var_i_do_not_remember_where_it_is

该 faas 命令接受与 frame apply 命令相同的选项。 请参阅 frame apply.

请注意，命令 tfaas command 应用 命令 到所有线程的所有帧。 请参阅 线程.
