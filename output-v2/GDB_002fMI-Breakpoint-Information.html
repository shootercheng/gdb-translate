27.5.4 GDB/MI 断点信息 ¶

当 GDB 报告有关断点、跟踪点、观察点或捕获点的信息时，它使用一个包含以下字段的元组：

number

断点编号。

type

断点的类型。对于普通断点，此类型将是‘断点’，但可能有许多值。

catch-type

如果断点的类型是‘捕获点’，则这表示捕获点的确切类型。

disp

这是断点的处置方式 —— 要么是‘删除’，表示断点将在下一次停止时被删除；要么是‘保留’，表示断点不会被删除。

enabled

这表示断点是否启用，启用时值为‘是’，禁用时值为‘否’。 请注意，这与字段 enable不同。

addr

断点的地址。这可能是一个十六进制数字，表示地址；或者是字符串‘<待决>’，表示待决断点；或者是字符串‘<多个>’，表示具有多个位置的断点。如果无法确定地址，则此字段将不存在。例如，观察点没有地址。

addr_flags

可选字段，包含与地址相关的任何标志。这些标志依赖于架构；请参阅 架构 以了解特定 CPU 的标志含义。

func

如果已知，断点所在的函数。 如果未知，此字段将不存在。

filename

如果已知，包含此函数的源文件的名称。 如果未知，此字段将不存在。

fullname

如果已知，包含此函数的源文件的完整文件名。 如果未知，此字段将不存在。

line

如果已知，此断点出现的行号。 如果未知，此字段将不存在。

at

如果源文件未知，可能会提供此字段。如果提供，它包含断点的地址，可能后面跟着一个符号名。

pending

如果此断点是待决的，此字段存在并包含用户输入的用于设置断点的文本。

GDB的内部断点（请参阅 维护信息断点) 有时也可能会变成待决的，对于这些断点， 待决 字段将为空，因为 GDB 在加载共享库时会自动创建这些断点。

evaluated-by

此断点的条件在何处评估，要么是‘主机’，要么是‘目标’。

thread

如果这是一个特定线程的断点，则此标识断点可以触发的线程。

inferior

如果这是一个特定子进程的断点，则此标识断点可以触发的子进程。

task

如果此断点仅限于特定的 Ada 任务，则此字段将包含任务标识符。

cond

如果断点是有条件的，这是条件表达式。

ignore

断点的忽略计数。

enable

断点的启用计数。

traceframe-usage

待修复。

static-tracepoint-marker-string-id

对于静态跟踪点，静态跟踪点标记的名称。

mask

对于掩码观察点，这是掩码。

pass

跟踪点的通过计数。

original-location

用户最初指定的断点位置。 此字段是可选的。

times

断点被命中的次数。

installed

此字段仅适用于跟踪点。这要么是‘是’，表示跟踪点已安装；要么是‘否’，表示未安装。

what

一些额外的数据，其确切内容取决于类型。

locations

如果断点有多个位置，则此字段存在。如果断点已启用且有一个单独的禁用位置，此字段也会异常存在。

值是一个位置列表。位置的格式如下所述。

多位置断点中的一个位置表示为一个包含以下字段的元组：

number

位置编号表示为一个点对，如‘1.2’。第一个数字是父断点的编号。第二个数字是该断点内位置的编号。

enabled

有三种可能的值，含义如下：

y

位置已启用。

n

位置被用户禁用。

N

位置因断点条件在此位置无效而被禁用。

addr

此位置的地址，以十六进制数字表示。

addr_flags

可选字段，包含与地址相关的任何标志。这些标志依赖于架构；请参阅 架构 以了解特定 CPU 的标志含义。

func

如果已知，位置所在的函数。 如果未知，此字段将不存在。

file

如果已知，包含此位置的源文件的名称。 如果未知，此字段将不存在。

fullname

如果已知，包含此位置的源文件的完整文件名。 如果未知，此字段将不存在。

line

如果已知，此位置出现的行号。 如果未知，此字段将不存在。

thread-groups

此位置所在的线程组。

例如，以下是 -break-insert （请参阅 GDB/MI 断点命令) 可能是： 

-> -break-insert main
<- ^done,bkpt={number="1",type="breakpoint",disp="keep",
    enabled="y",addr="0x08048564",func="main",file="myprog.c",
    fullname="/home/nickrob/myprog.c",line="68",thread-groups=["i1"],
    times="0"}
<- (gdb)
