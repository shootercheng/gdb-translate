3.1 命令语法 ¶

一条 GDB 命令是一行输入内容。其长度没有限制。它以一个命令名开头，后面跟着参数，参数的含义取决于命令名。例如，命令 step 接受一个参数，该参数是单步执行的次数，如 ‘单步执行 5 次’。你也可以使用不带任何参数的 step 命令。有些命令不允许带任何参数。

GDB 如果缩写不会引起歧义，命令名总是可以缩写。其他可能的命令缩写在各个命令的文档中有列出。在某些情况下，即使是有歧义的缩写也是允许的；例如， s 被特别定义为等同于 step 即使还有其他以 s开头的命令。你可以通过将缩写作为 help 命令的参数来测试它们。

向 GDB 输入空行（只输入 回车键）意味着重复上一条命令。某些命令（例如， run）不会以这种方式重复；这些是那些意外重复可能会导致问题且你不太可能想要重复执行的命令。用户定义的命令可以禁用此功能；请参阅 禁止重复。

当你使用 list 和 x 命令并通过按 回车键重复执行时，它们会构造新的参数，而不是完全按照之前输入的内容重复。这便于浏览源代码或内存。

GDB 你还可以用另一种方式使用 回车键 ：以类似于常用实用程序 more 的方式来分割冗长的输出（请参阅 屏幕大小）。由于在这种情况下很容易多按一次 回车键 ，所以 GDB 会在任何生成此类显示的命令之后禁用命令重复。

从 # 到行尾的任何文本都是注释；它不起任何作用。这主要在命令文件中很有用（请参阅 命令文件）。

Ctrl - o 绑定对于重复一系列复杂的命令很有用。此命令像 回车键一样接受当前行，然后从历史记录中获取相对于当前行的下一行进行编辑。
