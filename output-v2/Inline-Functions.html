11.1 内联函数 ¶

内联 是一种优化技术，它会在每个调用点直接插入函数体的副本，而不是跳转到一个共享的例程。 GDB 会像显示非内联函数一样显示内联函数。它们会出现在回溯信息中。你可以查看它们的参数和局部变量，使用 step进入它们，使用 next跳过它们，使用 finish从它们中跳出。 你可以使用 info frame 命令来检查一个函数是否被内联。

对于 GDB 要支持内联函数，编译器必须在调试信息中记录有关内联的信息 —— GCC 使用 DWARF 2 格式可以做到这一点，其他一些编译器也可以。 GDB 仅在使用 DWARF 2时支持内联函数。 GCC 4.1 之前的版本不会发出两个必需的属性（‘DW_AT_call_file’ 和 ‘DW_AT_call_line’）； GDB 在使用早期版本的 GCC时不会显示内联函数调用。相反，它会将内联函数的参数和局部变量显示为调用者的局部变量。

内联函数的函数体直接包含在其调用点；与非内联函数不同，这里没有专门用于调用的指令。 GDB 仍然会假设调用点和内联函数的起始位置是不同的指令。单步执行到调用点会显示调用点，然后再单步执行会显示内联函数的第一行，即使没有执行额外的指令。

这使得源代码级别的调试更加清晰；你既可以看到调用的上下文，也可以看到调用的效果。只有使用 stepi 或 nexti 进行单条指令的单步执行不会这样；单条指令的单步执行总是会显示内联函数体。

在某些方面， GDB 不会将内联函数调用视为与普通调用相同：

  • 在内联函数的调用点设置断点可能不起作用，因为调用点不包含任何代码。 GDB 可能会错误地将断点移动到包含该调用的函数的下一行。这个限制将在未来版本的 GDB中移除；在此之前，你可以在更早的行或内联函数内部设置断点。
  • GDB 在使用 finish 命令后无法定位内联调用的返回值。这是编译器生成的调试信息的一个限制；在 finish, 你可以单步执行到下一行并打印程序存储返回值的变量。
