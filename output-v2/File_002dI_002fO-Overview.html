E.14.1 文件输入/输出概述 ¶

文件输入/输出远程协议扩展 （简称：文件输入/输出）允许目标系统使用主机的文件系统和控制台输入/输出来执行各种系统调用。目标系统上的系统调用会被转换为发送到主机系统的远程协议数据包，然后主机系统执行所需的操作并向目标系统返回响应数据包。即使在缺乏文件系统的目标系统上，这也能模拟文件系统操作。

该协议被定义为独立于主机和目标系统。它使用自己的数据类型和值的内部表示。 GDB 以及目标系统的 GDB 存根负责在数据传输时将与系统相关的值表示转换为内部协议表示。

通信是同步的。只有当 GDB 正在等待来自‘C’、‘c’、‘S’ 或 ‘s’ 数据包的响应时，系统调用才是可能的。当 GDB 处理系统调用请求时，目标系统会停止，以允许对目标系统的内存进行确定性访问。因此，文件输入/输出不会被目标系统的信号中断。另一方面，可以通过用户中断（‘Ctrl - C’）在 GDB内中断文件输入/输出。

目标系统执行主机系统调用的请求不会结束最近的‘C’、‘c’、‘S’ 或 ‘s’ 操作。这意味着，在完成系统调用后，目标系统会继续之前的活动（继续执行、单步执行）。不需要来自 GDB 的额外继续或单步执行请求。 

(gdb) continue
  < - 目标系统请求‘系统调用X’
  目标系统停止， GDB 执行系统调用
  -> GDB 返回结果
  ... 目标系统继续执行， GDB 返回等待目标系统
  < - 目标系统命中断点并发送一个Txx数据包

该协议仅支持在控制台和主机文件系统上的常规文件进行输入/输出。主机系统上的字符或块特殊设备、管道、命名管道、套接字或任何其他通信方法均不被该协议支持。

文件输入/输出在非停止模式下不受支持。
