以下与目标文件相关的函数在 gdb 模块中可用：

函数： gdb.current_objfile () ¶

在自动加载 Python 脚本时（请参阅 Python 自动加载） GDB 将“当前目标文件”设置为相应的目标文件。此函数返回当前目标文件。如果没有当前目标文件，此函数返回 None。

函数： gdb.objfiles () ¶

返回当前程序空间引用的目标文件序列。请参阅 Python 中的目标文件, 以及 Python 中的程序空间。这与 gdb.selected_inferior().progspace.objfiles() 相同，包含此函数是为了保持历史兼容性。

函数： gdb.lookup_objfile (name [, by_build_id]) ¶

在当前程序空间的目标文件列表中查找 name，它可以是文件名或构建 ID（请参阅 Python 中的程序空间）。 如果未找到目标文件，则抛出 Python ValueError 异常。

如果 name 是相对文件名，则它将匹配任何具有相同尾部组件的源文件名。例如，如果 name 是‘gcc/expr.c’，则它将匹配源文件名 /build/trunk/gcc/expr.c, 但不匹配 /build/trunk/libcpp/expr.c 或 /build/trunk/gcc/x-expr.c.

如果提供了 by_build_id 且其值为 True ，则 name 是目标文件的构建 ID。否则， name 是文件名。 此功能仅在某些操作系统上受支持，特别是那些使用 ELF 格式的二进制文件和 GNU Binutils 的操作系统。有关此功能的更多详细信息，请参阅 --build-id 命令行选项在 命令行选项 中 GNU 链接器。

每个目标文件由 gdb.Objfile 类的一个实例表示。

变量： Objfile.filename ¶

目标文件的文件名，以字符串形式表示，符号链接已解析。

如果目标文件不再有效，则值为 None 。请参阅 gdb.Objfile.is_valid 方法，如下所述。

变量： Objfile.username ¶

用户指定的目标文件的文件名，以字符串形式表示。

如果目标文件不再有效，则值为 None 。请参阅 gdb.Objfile.is_valid 方法，如下所述。

变量： Objfile.is_file ¶

目标文件通常来自普通文件，但在某些情况下，它可能由内存内容构建而成。此属性为 True 对于基于文件的目标文件，以及 False 对于其他类型。

变量： Objfile.owner ¶

对于单独的调试信息目标文件，这是提供调试信息所对应的 gdb.Objfile 对象。否则，这是 None。 单独的调试信息目标文件使用 gdb.Objfile.add_separate_debug_file 方法添加，如下所述。

变量： Objfile.build_id ¶

目标文件的构建 ID，以字符串形式表示。 如果目标文件没有构建 ID，则值为 None。

此功能仅在某些操作系统上受支持，特别是那些使用 ELF 格式的二进制文件和 GNU Binutils 的操作系统。有关此功能的更多详细信息，请参阅 --build-id 命令行选项在 命令行选项 中 GNU 链接器。

变量： Objfile.progspace ¶

目标文件所在的程序空间，作为一个 gdb.Progspace 对象。请参阅 Python 中的程序空间。

变量： Objfile.pretty_printers ¶

该 pretty_printers 属性是一个函数列表。它用于查找漂亮打印机。一个 Value 会依次传递给每个函数；如果函数返回 None, 则继续搜索。否则，返回值应该是一个用于格式化值的对象。请参阅 漂亮打印 API，以获取更多信息。

变量： Objfile.type_printers ¶

该 type_printers 属性是一个类型打印机对象列表。请参阅 类型打印 API，以获取更多信息。

变量： Objfile.frame_filters ¶

该 frame_filters 属性是一个帧过滤器对象字典。请参阅 过滤帧，以获取更多信息。

可以以常规 Python 方式向 gdb.Objfile 对象添加任意属性。 例如，如果需要对与目标文件相关的某些额外记录进行管理，这将很有用。

请参阅 选择属性名称，以获取为新属性选择合适名称的指导。

在这个人为构造的示例中，我们记录了 GDB 加载目标文件的时间。 

(gdb) python
import datetime
def new_objfile_handler(event):
    # 设置新目标文件的 time_loaded 属性。
    event.new_objfile.time_loaded = datetime.datetime.today()
gdb.events.new_objfile.connect(new_objfile_handler)
end

(gdb) file ./hello
Reading symbols from ./hello...
(gdb) python print(gdb.objfiles()[0].time_loaded)
2014-10-09 11:41:36.770345

一个 gdb.Objfile 对象具有以下方法：

函数： Objfile.is_valid () ¶

如果 True 对象有效，则返回 gdb.Objfile , 如果无效，则返回 False 。一个 gdb.Objfile 对象如果其所引用的目标文件不再加载到 GDB 中，则会变得无效。如果在调用方法时对象无效，所有其他 gdb.Objfile 方法将抛出异常。

函数： Objfile.add_separate_debug_file (file) ¶

将 文件 添加到 GDB 将搜索目标文件调试信息的文件列表中。 当调试信息已从程序中移除并存储在单独的文件中时，这很有用。 GDB 对查找单独的调试信息文件有内置支持（请参阅 单独文件中的调试信息），但如果文件不在 GDB 搜索的标准位置之一，则可以使用此函数从不同位置添加调试信息文件。

函数： Objfile.lookup_global_symbol (name [, domain]) ¶

在这个目标文件中搜索名为 name 的全局符号。可选地，可以使用 domain 参数限制搜索范围。 该 domain 参数必须是 gdb 模块中定义的域常量，并在 符号的 Python 表示形式中描述。此函数与 gdb.lookup_global_symbol类似，不同之处在于搜索仅限于此目标文件。

结果是一个 gdb.Symbol 对象，或者如果未找到符号则为 None 。

函数： Objfile.lookup_static_symbol (name [, domain]) ¶

与 Objfile.lookup_global_symbol类似，但在这个目标文件中搜索具有静态链接的名为 name 的全局符号。
