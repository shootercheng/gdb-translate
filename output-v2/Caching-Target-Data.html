10.22 缓存目标数据 ¶

GDB 缓存调试器和目标之间交换的数据。 每个缓存都与 inferior 的地址空间相关联。 请参阅 调试多个 inferior 连接和程序，了解有关 inferior 和地址空间的信息。 这种缓存通常可以提高远程调试的性能 （请参阅 调试远程程序），因为它通过将内存读写捆绑成大块来减少远程协议的开销。 不幸的是，简单地缓存所有内容会导致结果不正确，因为 GDB 不一定了解有关易失性值、内存映射 I/O 地址等信息。此外，在非停止模式下 （请参阅 非停止模式) 内存可能会被更改 当 一个 GDB 命令正在执行时。 因此，默认情况下， GDB 仅缓存已知位于栈上的数据13 或在代码段中。 其他内存区域可以显式标记为可缓存；请参阅 内存区域属性.

set remotecache on ¶
set remotecache off

此选项现在已无作用；它的存在是为了与旧脚本兼容。

show remotecache ¶

显示已弃用的 remotecache 标志的当前状态。

set stack-cache on ¶
set stack-cache off

启用或禁用栈访问的缓存。当 on, 使用 缓存。默认情况下，此选项为 on.

show stack-cache ¶

显示内存访问数据缓存的当前状态。

set code-cache on ¶
set code-cache off

启用或禁用代码段访问的缓存。当 on, 使用缓存。默认情况下，此选项为 on. 这可以提高远程调试中反汇编的性能。

show code-cache ¶

显示代码段访问的目标内存缓存的当前状态。

info dcache [line] ¶

打印有关当前 inferior 地址空间数据缓存性能的信息。显示的信息 包括数据缓存的宽度和深度，以及对于每个缓存行，其编号、地址和被引用的次数。此 命令对于调试数据缓存操作很有用。

如果指定了行号，该行的内容将以十六进制形式打印。

set dcache size size ¶

设置数据缓存中的最大条目数（即上面提到的数据缓存深度）。

set dcache line-size line-size ¶

设置每个数据缓存条目缓存的字节数（即上面提到的数据缓存宽度）。 必须是 2 的幂次方。

show dcache size ¶

显示数据缓存条目的最大数量。请参阅 info dcache.

show dcache line-size ¶

显示数据缓存行的默认大小。

maint flush dcache ¶

清空数据缓存的内容（如果有）。此维护者命令在调试数据缓存实现时很有用。

脚注

(13)

在非停止模式下，运行中的线程以会干扰回溯的方式修改停止线程的栈的情况相对较少，并且栈读取的缓存可以显著提高远程回溯的速度。
