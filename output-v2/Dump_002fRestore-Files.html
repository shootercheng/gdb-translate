10.19 内存和文件之间的复制 ¶

您可以使用以下命令 dump, append, 和 restore 在目标内存和文件之间复制数据。 dump 和 append 命令将数据写入文件，而 restore 命令从文件中读取数据并将其恢复到被调试程序的内存中。文件可以是二进制、摩托罗拉S记录、英特尔十六进制、泰克十六进制或Verilog十六进制格式；然而， GDB 只能追加到二进制文件，并且不能从Verilog十六进制文件中读取。

dump [format] memory filename start_addr end_addr ¶
dump [format] value filename expr

转储从 起始地址 到 结束地址的内存内容，或者 表达式的值到 文件名 以指定的格式。

格式 参数可以是以下任意一种：

binary

原始二进制形式。

ihex

英特尔十六进制格式。

srec

摩托罗拉S记录格式。

tekhex

泰克十六进制格式。

verilog

Verilog十六进制格式。

GDB 对这些格式的定义与 GNU 二进制实用工具相同，如‘objdump’和‘objcopy’。如果 格式 被省略， GDB 将以原始二进制形式转储数据。

append [binary] memory filename start_addr end_addr ¶
append [binary] value filename expr

以原始二进制形式将从 起始地址 到 结束地址的内存内容，或者 表达式的值追加到文件 文件名中。（GDB 只能将数据追加到原始二进制形式的文件中。）

restore filename [binary] bias start end ¶

将文件 文件名 的内容恢复到内存中。 restore 命令可以自动识别任何已知的 BFD 文件格式，但原始二进制格式除外。要恢复原始二进制文件，您必须在文件名后指定可选关键字 binary 。

如果 偏移量 不为零，其值将被加到文件中包含的地址上。二进制文件总是从地址零开始，因此它们将被恢复到地址 偏移量。其他bfd文件有内置的位置；它们将从该位置偏移 偏移量 处恢复。

如果 起始 和/或 结束 不为零，那么只有文件偏移量 起始 和文件偏移量 结束 之间的数据将被恢复。这些偏移量是相对于文件中的地址而言的，在应用 偏移量 参数之前。
