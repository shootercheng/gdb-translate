23.3.2.3 异常处理 ¶

在执行 python 命令时，Python 代码中未捕获的 Python 异常会被转换为对 GDB 错误报告机制的调用。如果调用 python 的命令没有处理该错误， GDB 将终止该命令并打印错误消息。具体打印的内容取决于 set python print-stack （请参阅 Python 命令)。 示例： 

(gdb) python print foo
Traceback (最近一次调用栈):
  File "<string>", line 1, in <module>
NameError: name 'foo' 未定义

GDB 在 GDB Python 代码调用的命令中发生的错误会被转换为 Python 异常。Python 异常的类型取决于错误情况。

gdb.error ¶

这是由 GDB生成的大多数异常的基类。 它派生自 RuntimeError, 以与早期版本的 GDB兼容。

如果在 GDB 中发生的错误不属于更具体的类别，那么生成的异常将具有此类型。

gdb.MemoryError ¶

这是 gdb.error 的一个子类，当操作试图访问被调试程序中的无效内存时会抛出该异常。

KeyboardInterrupt ¶

用户中断（通过 C-c 或在分页提示时输入 q ）会被转换为一个 Python KeyboardInterrupt 异常。

在所有情况下，你的异常处理程序将看到 GDB 错误消息作为其值，并且在最接近 GDB 错误发生位置的 Python 语句处的 Python 调用栈回溯作为回溯信息。

在通过 GDB 用 Python 实现 gdb.Command命令，或通过 gdb.Function实现函数时，能够抛出一个不会导致回溯信息被打印的异常是很有用的。例如，用户可能错误地调用了该命令。 GDB 提供了一个特殊的异常类，可用于此目的。

gdb.GdbError ¶

当从命令或函数中抛出此异常时，该命令或函数将失败，但不会显示 Python 栈。 GDB 本身不会抛出此异常，而是在用户 Python 代码抛出该异常时识别它。示例： 

(gdb) python
>class HelloWorld (gdb.Command):
>  """向整个世界问好。"""
>  def __init__ (self):
>    super (HelloWorld, self).__init__ ("hello-world", gdb.COMMAND_USER)
>  def invoke (self, args, from_tty):
>    argv = gdb.string_to_argv (args)
>    if len (argv) != 0:
>      raise gdb.GdbError ("hello-world 不接受参数")
>    print ("Hello, World!")
>HelloWorld ()
>end
(gdb) hello-world 42
hello-world 不接受参数
