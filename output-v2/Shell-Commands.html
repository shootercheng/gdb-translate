2.3 Shell命令 ¶

如果在调试会话期间需要偶尔执行shell命令，无需退出或暂停调试；你可以直接使用 GDB；你可以 直接使用 shell 命令。

shell command-string ¶
!command-string

调用shell来执行 命令字符串. 请注意， ! 和 命令字符串之间不需要空格。 在GNU和Unix系统上，如果环境变量 SHELL存在，它将决定运行哪个shell。否则， GDB 使用默认shell（/bin/sh 在GNU和Unix系统上， cmd.exe 在MS - Windows上， COMMAND.COM 在MS - DOS等系统上）。

你还可以使用 $_shell 便利函数从表达式中调用shell命令。请参阅 $_shell便利函数.

实用工具 make 在开发环境中经常需要。 你不必为此目的在 shell 中使用 GDB命令：

make make-args ¶

使用指定的参数执行 make 程序。这相当于‘shell make make参数’。

pipe [command] | shell_command
| [command] | shell_command
pipe -d delim command delim shell_command
| -d delim command delim shell_command

执行 命令 并将其输出发送到 shell_command. 请注意， |周围不需要空格。 如果没有提供 命令 ，则重复执行上一个命令。

如果 命令 包含一个 |, 可以使用选项 -d delim 来指定一个替代的分隔字符串 分隔符 ，该分隔符用于分隔 命令 和 shell命令.

示例： 

(gdb) p var
$1 = {
  black = 144,
  red = 233,
  green = 377,
  blue = 610,
  white = 987
}

(gdb) pipe p var|wc
      7      19      80
(gdb) |p var|wc -l
7

(gdb) p /x var
$4 = {
  black = 0x90,
  red = 0xe9,
  green = 0x179,
  blue = 0x262,
  white = 0x3db
}
(gdb) ||grep red
  red => 0xe9,

(gdb) | -d ! echo this contains a | char\n ! sed -e 's/|/PIPE/'
this contains a PIPE char
(gdb) | -d xxx echo this contains a | char!\n xxx sed -e 's/|/PIPE/'
this contains a PIPE char!
(gdb)

便利变量 $_shell_exitcode 和 $_shell_exitsignal 可用于检查由 shell启动的最后一个shell命令的退出状态 make, pipe 和 |. 请参阅 便利变量.
