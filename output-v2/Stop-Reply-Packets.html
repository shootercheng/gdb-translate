E.4 停止回复包 ¶

‘C’, ‘c’, ‘S’, ‘s’, ‘vCont’, ‘vAttach’, ‘vRun’, ‘vStopped’, 和 ‘?’包可以接收以下任何一种作为回复。除了‘?’ 和 ‘vStopped’, 该回复仅在目标停止时返回。在下面， 信号编号 由头文件定义 include/gdb/signals.h 在 GDB 源代码。

在非停止模式下，服务器将简单地回复‘OK’到诸如‘vCont’之类的命令；任何停止都将是未来通知的主题。请参阅 非停止模式的远程协议支持。

正如在请求包的描述中一样，为了清晰起见，我们在回复模板中包含空格；这些空格不是回复包语法的一部分。没有 GDB 停止回复包使用空格来分隔其组件。

‘S AA’

程序接收到信号编号 AA （一个两位十六进制数）。这相当于一个没有T’响应。 n:r 对。

‘T AA n1:r1;n2:r2;…’ ¶

程序接收到信号编号 AA （一个两位十六进制数）。这相当于一个‘S’响应，除了‘n:r’对可以直接在停止回复包中携带重要寄存器的值和其他信息，从而减少往返延迟。单步和断点陷阱以这种方式报告。每个‘n:r’对的解释如下：

  • 如果 n 是一个十六进制数，它是一个寄存器编号，并且相应的 r 给出了该寄存器的值。数据 r 是目标字节顺序的一系列字节，每个字节由一个两位十六进制数表示。
  • 如果 n 是‘线程’，那么 r 是停止线程的线程ID，如 线程ID语法中所指定。
  • 如果 n 是‘核心’，那么 r 是检测到停止事件的核心的十六进制编号。
  • 如果 n 是一个已识别的 停止原因, 它描述了一个更具体的导致目标停止的事件。当前定义的停止原因如下所列。 aa 应该是‘05’，即陷阱信号。最多只能有一个停止原因。
  • 否则， GDB 应该忽略这个‘n:r’对，然后继续处理下一个；这允许我们在未来扩展协议。

当前定义的停止原因如下：

‘观察点’
‘读观察点’
‘读写观察点’

该包表示观察点命中，并且 r 是数据地址，以十六进制表示。

‘系统调用进入’
‘系统调用返回’

该包表示系统调用进入或返回，并且 r 是系统调用编号，以十六进制表示。

‘库’ ¶

该包表示已加载的库发生了变化。 GDB 应该使用‘qXfer:libraries:read’来获取新的已加载库列表。 r 部分将被忽略。

‘重放日志’ ¶

该包表示目标无法继续重放记录的执行事件，因为它已到达日志的末尾（或反向执行时的开头）。 r 的值将是‘开始’或‘结束’。请参阅 反向运行程序, 以获取更多信息。

‘软件断点’

该包表示执行了软件断点指令，无论是否是 GDB 设置的断点，还是断点在程序中硬编码。 r 部分必须留空。

在某些架构（如x86）上，在架构层面，当断点指令执行时，程序计数器指向断点地址加上一个偏移量。在这样的目标上，存根负责调整程序计数器以指向断点地址。

默认情况下不应发送此包；较旧的 GDB 版本不支持它。 GDB 通过提供适当的‘qSupported’功能（请参阅 qSupported）来请求它。远程存根也必须提供适当的‘qSupported’功能以表明支持。

此包对于正确的非停止模式操作是必需的。

‘硬件断点’

该包表示目标因硬件断点而停止。 r 部分必须留空。

关于‘qSupported’和上述非停止模式的相同说明适用。

‘fork’ ¶

该包表示调用了 fork ，并且 r 是新子进程的线程ID，如 线程ID语法中所指定。此包仅适用于支持fork事件的目标。

默认情况下不应发送此包；较旧的 GDB 版本不支持它。 GDB 通过提供适当的‘qSupported’功能（请参阅 qSupported）来请求它。远程存根也必须提供适当的‘qSupported’功能以表明支持。

‘vfork’ ¶

该包表示调用了 vfork ，并且 r 是新子进程的线程ID，如 线程ID语法中所指定。此包仅适用于支持vfork事件的目标。

默认情况下不应发送此包；较旧的 GDB 版本不支持它。 GDB 通过提供适当的‘qSupported’功能（请参阅 qSupported）来请求它。远程存根也必须提供适当的‘qSupported’功能以表明支持。

‘vfork完成’ ¶

该包表示由vfork创建的子进程要么调用了 exec ，要么终止了，因此父进程和子进程的地址空间不再共享。 r 部分将被忽略。此包仅适用于支持vfork完成事件的目标。

默认情况下不应发送此包；较旧的 GDB 版本不支持它。 GDB 通过提供适当的‘qSupported’功能（请参阅 qSupported）来请求它。远程存根也必须提供适当的‘qSupported’功能以表明支持。

‘exec’ ¶

该包表示调用了 execve ，并且 r 是执行的文件的绝对路径名，以十六进制表示。此包仅适用于支持exec事件的目标。

默认情况下不应发送此包；较旧的 GDB 版本不支持它。 GDB 通过提供适当的‘qSupported’功能（请参阅 qSupported）来请求它。远程存根也必须提供适当的‘qSupported’功能以表明支持。

‘clone’

该包表示调用了 clone ，并且 r 是新子线程的线程ID，如 线程ID语法中所指定。此包仅适用于支持clone事件的目标。

默认情况下不应发送此包； GDB 使用 QThreadOptions 包来请求它。

‘创建’

该包表示线程刚刚创建。新线程将保持停止状态，直到 GDB 使用恢复包（请参阅 vCont包）使其运行。默认情况下不应发送此包； GDB 使用 QThreadEvents 包来请求它。另请参阅下面的‘w’（请参阅 线程退出事件）远程回复。 r 部分将被忽略。

‘W AA’
‘W AA ; 进程:pid’

进程已退出，并且 AA 是退出状态。这仅适用于某些目标。

响应的第二种形式，包括已退出进程的进程ID，仅在 GDB 报告支持多进程协议扩展时才能使用；请参阅 多进程扩展。 AA 和 pid 都被格式化为大端十六进制字符串。

‘X AA’
‘X AA ; 进程:pid’

进程因信号 AA而终止。

响应的第二种形式，包括已终止进程的进程ID，仅在 GDB 报告支持多进程协议扩展时才能使用；请参阅 多进程扩展。 AA 和 pid 都被格式化为大端十六进制字符串。

‘w AA ; tid’ ¶

线程已退出，并且 AA 是退出状态。默认情况下不应发送此响应； GDB 使用 QThreadEvents 或 QThreadOptions 包来请求它。另请参阅 线程创建事件 上面。 AA 被格式化为大端十六进制字符串。

‘N’

目标中没有剩余的恢复线程。换句话说，即使进程仍然存活，但最后一个恢复的线程已经退出。例如，假设目标进程有两个线程：线程1和线程2。客户端让线程1停止，并恢复线程2，线程2随后退出。此时，即使进程仍然存活，因此没有发送‘W’停止回复，但实际上也没有线程在执行。‘N’停止回复因此通知客户端它可以停止等待停止回复。默认情况下不应发送此包；较旧的 GDB 版本不支持它。 GDB 通过提供适当的‘qSupported’功能（请参阅 qSupported）来请求它。远程存根也必须提供适当的‘qSupported’功能以表明支持。

‘O XX…’

‘XX…’是 ASCII 数据的十六进制编码，将作为程序的控制台输出写入。这可以在程序运行时的任何时候发生，调试器应该继续等待‘W’, ‘T’等。此回复在非停止模式下不允许。

‘F 调用ID,参数…’

调用ID 是一个标识符，用于指定应该调用哪个主机系统调用。这只是函数的名称。转换为正确的系统调用仅在 GDB中定义时适用。请参阅 文件I/O远程协议扩展, 以获取已实现的系统调用列表。

‘参数…’是为此系统调用定义的参数列表。

当目标期望 GDB 代表目标调用主机系统调用时，它会用此包进行回复。 GDB 用适当的‘F’包进行回复，并继续等待目标的下一个回复包。最新的‘C’, ‘c’, ‘S’ or ‘s’操作预计将继续。请参阅 文件I/O远程协议扩展, 以获取更多详细信息。
