附录I 跟踪文件格式 ¶

跟踪文件由三部分组成：头部、文本描述部分和包含二进制数据的跟踪帧部分。

头部的格式为 \x7fTRACE0\n. 第一个字节是 0x7f ，用于表明文件包含二进制数据，而 0 是一个版本号，未来可能会有不同的值。

描述部分由多行 ASCII 文本组成，这些文本由换行符（0xa) 分隔。这些行可能包含各种可选的描述性或上下文设置信息，例如跟踪点定义或寄存器集大小。 GDB 会忽略任何它不识别的行。空行标志着此部分的结束。

R size

指定寄存器块的大小（以字节为单位）。这等于 g 远程协议中数据包有效负载的大小。 大小 是一个ASCII十进制数。单个跟踪文件中应该只有一行这样的内容。

status status

跟踪状态。 状态 的格式与 qTStatus 远程数据包回复的格式相同。单个跟踪文件中应该只有一行这样的内容。

tp payload

跟踪点定义。 有效负载 的格式与 qTfP/qTsP 远程数据包回复有效负载的格式相同。单个跟踪点的定义可能需要多行，对应多个回复数据包。

tsv payload

跟踪状态变量定义。 有效负载 的格式与 qTfV/qTsV 远程数据包回复有效负载的格式相同。单个变量的定义可能需要多行，对应多个回复数据包。

tdesc payload

XML格式的目标描述。 有效负载 是XML文件的一行。所有这些行应该连接在一起以获得原始的XML文件。此文件的格式与 qXfer features 有效负载相同，并且对应于主 target.xml 文件。不允许包含其他文件。

跟踪帧部分由多个连续的帧组成。每个帧以一个两字节的跟踪点编号开始，后面跟着一个四字节的大小，该大小表示帧中的数据量。帧中的数据由多个块组成，每个块由一个字符表示其类型（至少包括寄存器、内存和跟踪状态变量）。此部分的数据是原始二进制数据，不是十六进制或其他编码；其字节序与目标的字节序相匹配。

R bytes

寄存器块。字节的数量和顺序与 g 远程协议中的数据包相匹配。请注意，这些是实际的字节，按目标顺序排列，而不是十六进制编码。

M address length bytes...

内存块。这是一个连续的内存块，位于8字节的 地址, 长度为2字节 长度, 后面跟着 长度 字节的数据。

V number value

跟踪状态变量块。这记录了编号为 值 的跟踪状态变量的8字节有符号值。 编号.

跟踪文件格式的未来增强可能包括其他类型的块。
