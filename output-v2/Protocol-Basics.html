E.14.2 协议基础 ¶

文件 I/O 协议使用 F 数据包作为请求以及响应数据包。由于文件 I/O 系统调用只能在 GDB 等待继续或单步执行的目标的响应时发生， 文件 I/O 请求是一个响应，该响应是 GDB 作为之前的 ‘C’、‘c’、‘S’ 或 ‘s’ 数据包的结果而必须预期的。 此 F 数据包包含允许 GDB 调用适当的主机系统调用所需的所有信息：

  • 所请求系统调用的唯一标识符。
  • 系统调用的所有参数。指针以目标内存地址空间中的地址形式给出。字符串指针以指针/长度对的形式给出。数值按原样给出。 数值控制标志以特定于协议的表示形式给出。

此时， GDB 必须执行以下操作。

  • 如果参数包含作为系统调用输入所需数据的指针值， GDB 使用标准 m 数据包请求从目标获取此数据。目标实现必须预期这种额外的通信，并将其作为任何其他 m 数据包进行处理。
  • GDB 根据需要将所有值从协议表示形式转换为主机表示形式。数据类型被强制转换为主机类型。
  • GDB 调用系统调用。
  • 然后它将数据类型转换回协议表示形式。
  • 如果系统调用预期在调用的指针参数指定的缓冲区空间中返回数据，则使用 M 或 X 数据包将数据传输到目标。目标实现必须预期此数据包，并将其作为任何其他 M 或 X 数据包进行处理。

最终 GDB 用另一个 F 数据包进行响应，该数据包包含目标继续所需的所有必要信息。这至少包含

  • 返回值。
  • errno, 如果它已被系统调用更改。
  • “Ctrl - C” 标志。

在完成所需的类型和值转换后，目标继续最新的继续或单步操作。
