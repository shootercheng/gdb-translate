8.3 选择一个帧 ¶

大多数用于检查程序中栈和其他数据的命令都作用于当前所选的栈帧。以下是选择栈帧的命令；所有这些命令最后都会打印出刚选择的栈帧的简要描述。

frame [ frame-selection-spec ] ¶
f [ frame-selection-spec ]

该 frame 命令允许选择不同的栈帧。 帧选择规范 可以是以下任何一种：

num ¶
level num

选择帧级别 编号。请记住，帧0是最内层（当前正在执行）的帧，帧1是调用最内层帧的帧，依此类推。最高级别的帧通常是用于 main的帧。

由于这是导航帧栈最常用的方法，字符串 level 可以省略。例如，以下两个命令是等效的： 

(gdb) frame 3
(gdb) frame level 3
address stack-address ¶

选择具有栈地址的帧 栈地址。一个帧的 栈地址 可以在 info frame的输出中看到，例如： 

(gdb) info frame
栈级别1，帧位于0x7fffffffda30:
 rip = 0x40066d 在b (amd64 - entry - value.cc:59)中；保存的rip 0x4004c5
 尾调用帧，位于0x7fffffffda30的帧的调用者
 源语言为c++。
 参数列表地址未知。
 局部变量地址未知，上一帧的sp是0x7fffffffda30

这个帧的 栈地址 是 0x7fffffffda30 ，如以下行所示： 

栈级别1，帧位于0x7fffffffda30:
function function-name ¶

选择函数的栈帧 函数名。如果函数有多个栈帧 函数名 则选择最内层的栈帧。

view stack-address [ pc-addr ] ¶

查看不属于 GDB回溯的帧。查看的帧具有栈地址 栈地址, 并且可选地，程序计数器地址为 程序计数器地址。

如果栈帧的链接因错误而损坏，导致 GDB 无法为所有帧正确分配编号，这主要会很有用。此外，当你的程序有多个栈并在它们之间切换时，这也会很有用。

当使用 frame view 查看当前回溯之外的帧时，你总是可以使用之前的栈帧选择指令之一返回原始栈，例如 frame level 0。

up n ¶

向上移动 n 个栈帧； n 默认为1。对于正数 n, 这会向最外层帧前进，到更高的帧编号，到存在时间更长的帧。

down n ¶

向下移动 n 个栈帧； n 默认为1。对于正数 n, 这会向最内层帧前进，到更低的帧编号，到最近创建的帧。你可以将 down 缩写为 do。

所有这些命令最后都会打印两行描述帧的输出。第一行显示帧编号、函数名、参数以及该帧中执行的源文件和行号。第二行显示该行源代码的文本。

例如： 

(gdb) up
#1  0x22f0 在main (argc = 1, argv = 0xf7fffbf4, env = 0xf7fffbfc)中
    在env.c:10
10              read_input_file (argv[i]);

在这样的打印输出之后， list 不带参数的命令会打印以帧中执行点为中心的十行内容。你也可以通过输入 edit使用你喜欢的编辑程序在执行点编辑程序。详见 打印源代码行。

select-frame [ frame-selection-spec ] ¶

该 select-frame 命令是 frame 的一种变体，它在选择新帧后不显示该帧。此命令主要用于 GDB 命令脚本，在这些脚本中，输出可能是不必要且分散注意力的。 帧选择规范 与 frame 命令中描述的相同，详见 选择一个帧。

up-silently n ¶
down-silently n

这两个命令分别是 up 和 down的变体；它们的不同之处在于，它们默默地完成工作，不会导致显示新帧。它们主要用于 GDB 命令脚本，在这些脚本中，输出可能是不必要且分散注意力的。
