23.4.3.7 Guile 中的类型 ¶

GDB 在类型为的对象中表示来自下级的类型 <gdb:type>.

(gdb) 模块。

Scheme 过程： type? object ¶

返回 #t 如果 object 是一个类型为 <gdb:type>.否则返回 #f.

Scheme 过程： lookup-type name [] ¶

name, 它必须是一个字符串。

block 如果提供了 <gdb:block>, 它是一个类型为 name 的对象，并在该作用域中查找。否则，在全局范围内查找。

通常，此函数将返回一个 <gdb:type>的实例。如果找不到指定名称的类型，则会抛出异常。

Scheme 过程： type-code type ¶

返回 type的类型代码。类型代码将是下面定义的常量之一。 TYPE_CODE_

Scheme 过程： type-tag type ¶

返回 type的标签名。标签名是在 struct, union, 或 enum 之后的名称，在 C 和 C++ 中；并非所有语言都有这个概念。如果此类型没有标签名，则返回++ #f 。

Scheme 过程： type-name type ¶

返回 type的名称。如果此类型没有名称，则返回 #f 。

Scheme 过程： type-print-name type ¶

返回 type的打印名称。即使是匿名类型也会返回一些内容。例如，对于一个匿名 C 结构体，返回 "struct {...}" 。

Scheme 过程： type-sizeof type ¶

返回此类型在目标 char 单位中的大小。通常，目标的 char 类型将是一个 8 位字节。但是，在某些不寻常的平台上，此类型可能具有不同的大小。

Scheme 过程： type-strip-typedefs type ¶

返回一个新的 <gdb:type> ，它表示 type去除所有 typedef 层后的实际类型。

Scheme 过程： type-array type n1 [n2] ¶

返回一个新的 <gdb:type> 对象，该对象表示此类型的数组。如果提供一个参数，它是数组的包含上限；在这种情况下，下限为零。如果提供两个参数，第一个参数是数组的下限，第二个参数是数组的上限。数组的长度不能为负，但边界可以为负。

Scheme 过程： type-vector type n1 [n2] ¶

返回一个新的 <gdb:type> 对象，该对象表示此类型的向量。如果提供一个参数，它是向量的包含上限；在这种情况下，下限为零。如果提供两个参数，第一个参数是向量的下限，第二个参数是向量的上限。向量的长度不能为负，但边界可以为负。

数组和 array 向量之间的区别在于 vector 数组的行为类似于 C 语言中的数组：在表达式中使用时，它们会退化为指向第一个元素的指针，而向量被视为一等值。

Scheme 过程： type-pointer type ¶

返回一个新的 <gdb:type> 对象，该对象表示指向 type的指针。

Scheme 过程： type-range type ¶

返回一个包含两个元素的列表： type的下限和上限。如果 type 没有范围，则抛出异常。

Scheme 过程： type-reference type ¶

返回一个新的 <gdb:type> 对象，该对象表示对 type的引用。

Scheme 过程： type-target type ¶

返回一个新的 <gdb:type> 对象，该对象表示 type.

对于指针类型，目标类型是所指向对象的类型。对于数组类型（即类似 C 语言的数组），目标类型是数组元素的类型。对于函数或方法类型，目标类型是返回值的类型。对于复杂类型，目标类型是元素的类型。对于 typedef，目标类型是别名类型。

如果该类型没有目标，则此方法将抛出异常。

Scheme 过程： type-const type ¶

返回一个新的 <gdb:type> 对象，该对象表示一个 const-限定的 type变体。

Scheme 过程： type-volatile type ¶

返回一个新的 <gdb:type> 对象，该对象表示一个 volatile-限定的 type.

Scheme 过程： type-unqualified type ¶

返回一个新的 <gdb:type> 对象，该对象表示一个未限定的 type变体。也就是说，结果既不是 const 也不是 volatile.

Scheme 过程： type-num-fields ¶

返回 <gdb:type> type的字段数量。

Scheme 过程： type-fields type ¶

以列表形式返回 type 的字段。对于结构和联合类型， fields 具有通常的含义。范围类型有两个字段，即最小值和最大值。枚举类型每个枚举常量有一个字段。函数和方法类型每个参数有一个字段。C 语言类的基类型也表示为字段。如果该类型没有字段，或者不属于这些类别之一，则返回一个空列表。请参阅++ Guile 中类型的字段.

Scheme 过程： make-field-iterator type ¶

以 <gdb:iterator> 对象的形式返回 type 的字段。请参阅 Guile 中的迭代器.

Scheme 过程： type-field type field-name ¶

返回 field-name type 中名为的字段。结果是一个类型为 <gdb:field>的对象。请参阅 Guile 中类型的字段.如果该类型没有字段，或者 field-name 不是 type的字段，则抛出异常。

例如，如果 some-type 是一个 <gdb:type> 实例，持有一个结构类型，你可以使用以下方式访问其 foo 字段： 

(define bar (type-field some-type "foo"))

bar 将是一个 <gdb:field> 对象。

Scheme 过程： type-has-field? type name ¶

如果 #t <gdb:type> type 有一个名为 name的字段，则返回 #f.否则返回

每个类型都有一个代码，该代码指示此类型属于哪个类别。可用的类型类别由 (gdb) 模块中定义的常量表示：

TYPE_CODE_PTR ¶

该类型是一个指针。

TYPE_CODE_ARRAY ¶

该类型是一个数组。

TYPE_CODE_STRUCT ¶

该类型是一个结构。

TYPE_CODE_UNION ¶

该类型是一个联合。

TYPE_CODE_ENUM ¶

该类型是一个枚举。

TYPE_CODE_FLAGS ¶

一种位标志类型，用于状态寄存器等。

TYPE_CODE_FUNC ¶

该类型是一个函数。

TYPE_CODE_INT ¶

该类型是一个整数类型。

TYPE_CODE_FLT ¶

一种浮点类型。

TYPE_CODE_VOID ¶

特殊类型 void.

TYPE_CODE_SET ¶

一种 Pascal 集合类型。

TYPE_CODE_RANGE ¶

一种范围类型，即具有边界的整数类型。

TYPE_CODE_STRING ¶

一种字符串类型。请注意，这仅用于某些具有语言定义字符串类型的语言；C 字符串不是以这种方式表示的。

TYPE_CODE_BITSTRING ¶

一串位。已弃用。

TYPE_CODE_ERROR ¶

一种未知或错误的类型。

TYPE_CODE_METHOD ¶

一种方法类型，如在 C 语言中找到的。++.

TYPE_CODE_METHODPTR ¶

一种指向成员函数的指针。

TYPE_CODE_MEMBERPTR ¶

一种指向成员的指针。

TYPE_CODE_REF ¶

一种引用类型。

TYPE_CODE_RVALUE_REF ¶

一种 C++11 右值引用类型。

TYPE_CODE_CHAR ¶

一种字符类型。

TYPE_CODE_BOOL ¶

一种布尔类型。

TYPE_CODE_COMPLEX ¶

一种复数浮点类型。

TYPE_CODE_TYPEDEF ¶

一种对其他类型的 typedef。

TYPE_CODE_NAMESPACE ¶

一种 C++ 命名空间。

TYPE_CODE_DECFLOAT ¶

一种十进制浮点类型。

TYPE_CODE_INTERNAL_FUNCTION ¶

内部函数 GDB. 这是用于表示便利函数的类型（请参阅 便利函数)。

gdb.TYPE_CODE_XMETHOD ¶

内部方法 GDB. 这是用于表示 x 方法的类型（请参阅 编写 X 方法)。

gdb.TYPE_CODE_FIXED_POINT ¶

一种定点数。

gdb.TYPE_CODE_NAMESPACE ¶

一种 Fortran 命名列表。

在 (gdb types) Guile 模块中提供了对类型的进一步支持（请参阅 Guile 类型模块)。

每个字段都表示为一个类型为 <gdb:field>的对象。

以下与字段相关的过程由 (gdb) 模块提供：

Scheme 过程： field? object ¶

如果 #t object 是一个类型为 <gdb:field>的对象，则返回 #f.否则返回

Scheme 过程： field-name field ¶

返回字段的名称，对于匿名字段返回 #f 。

Scheme 过程： field-type field ¶

返回字段的类型。这通常是一个 <gdb:type>的实例，但在某些情况下可能是 #f 。

Scheme 过程： field-enumval field ¶

返回 <gdb:field> field所表示的枚举值。

Scheme 过程： field-bitpos field ¶

返回 <gdb:field> field的位位置。此属性对于 static 字段（如在 C 语言中）不可用。++

Scheme 过程： field-bitsize field ¶

如果字段是打包的，或者是位字段，则返回 <gdb:field> field 的位大小。否则，返回零；在这种情况下，字段的大小由其类型给出。

Scheme 过程： field-artificial? field ¶

如果字段是人为的，通常意味着它是由编译器提供的而不是用户提供的，则返回 #t 。否则返回 #f.

Scheme 过程： field-base-class? field ¶

如果字段表示 C 语言结构的基类，则返回 #t 。否则返回++ 。 #f
