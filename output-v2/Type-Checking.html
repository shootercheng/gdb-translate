15.3.1 类型检查概述 ¶

某些语言，如C和C++++，是强类型语言，这意味着运算符和函数的参数必须是正确的类型，否则会发生错误。 这些检查可防止类型不匹配错误导致任何运行时问题。 例如， 

int klass::my_method(char *b) { return  b ? 1 : 2; }

(gdb) print obj.my_method (0)
$1 = 2

但是

(gdb) print obj.my_method (0x1234)
无法将方法klass::my_method解析为任何重载实例

第二个示例失败是因为在C++中++ 整数常量‘0x1234’与指针参数类型不兼容。

对于您在 GDB 命令中使用的表达式，您可以告诉 GDB 不强制执行严格的类型检查，或者将任何不匹配视为错误并放弃该表达式；当禁用类型检查时， GDB 可以成功计算像上面第二个示例这样的表达式。

即使关闭了类型检查，可能还有其他与类型相关的原因会阻止 GDB 计算表达式。例如， GDB 不知道如何将一个 int 和一个 struct foo相加。 这些特定的类型错误与所使用的语言无关，通常是由那些无论如何计算都没有多大意义的表达式引起的。

GDB 提供了一些额外的命令来控制类型检查：

set check type on
set check type off

开启或关闭严格的类型检查。 如果在开启类型检查的情况下计算表达式时出现任何类型不匹配， GDB 会打印一条消息并中止表达式的计算。

show check type

显示类型检查的当前设置以及 GDB 是否正在强制执行严格的类型检查规则。
