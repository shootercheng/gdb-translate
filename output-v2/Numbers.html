22.6 数字 ¶

在GDB中，您始终可以按照通常的约定以八进制、十进制或十六进制输入数字： GDB 八进制数字以 ‘0’开头，十进制数字以 ‘.’结尾，十六进制数字 以 ‘0x’开头。既不以 ‘0’ 或 ‘0x’开头，也不以 ‘.’结尾的数字，默认以十进制输入；同样，当未指定特定格式时，数字的默认显示也是十进制。您可以使用下面描述的命令更改输入和输出的默认基数。

set input-radix base ¶

设置数字输入的默认基数。支持的 基数 选择为十进制8、10或16。基数本身必须明确指定或使用当前输入基数；例如，以下任何一个命令 

set input-radix 012
set input-radix 10.
set input-radix 0xa

都将输入基数设置为十进制。另一方面，‘set input-radix 10’ 不会改变输入基数，无论它原来是什么，因为 ‘10’ 没有任何表示其基数的前导或尾随符号，将按照当前基数进行解释。因此，如果当前基数是16， ‘10’ 将被解释为十六进制，即十进制的16，这不会改变基数。

set output-radix base ¶

设置数字显示的默认基数。支持的 基数 选择为十进制8、10或16。基数本身必须明确指定或使用当前输入基数。

show input-radix ¶

显示当前数字输入的默认基数。

show output-radix ¶

显示当前数字显示的默认基数。

set radix [base] ¶
show radix

这些命令用于设置和显示数字输入和输出的默认基数。 set radix 将输入和输出的基数设置为 相同的基数；如果不带参数，则将基数重置为默认值10。
