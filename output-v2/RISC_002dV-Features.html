G.5.13 RISC-V 特性 ¶

‘org.gnu.gdb.riscv.cpu’ 特性是 RISC-V 目标所必需的。它应包含寄存器 ‘x0’ 到 ‘x31’，以及 ‘pc’。可以使用架构名称（‘x0’， ‘x1’ 等），也可以使用 ABI 名称（‘零’，‘返回地址’ 等）。

‘org.gnu.gdb.riscv.fpu’ 特性是可选的。如果存在，它 应包含寄存器 ‘f0’ 到 ‘f31’，‘浮点标志’， ‘浮点舍入模式’，以及 ‘浮点控制状态寄存器’。与 CPU 特性一样，可以使用架构寄存器名称或 ABI 名称。

‘org.gnu.gdb.riscv.virtual’ 特性是可选的。如果存在， 它应包含目标上没有由真实寄存器支持的寄存器，而是虚拟的，其中寄存器值 是从其他目标状态派生而来的。在很多方面，这些类似于 GDB伪寄存器，只是由目标实现。 目前，此集合中预期的唯一寄存器是单字节的 ‘特权’ 寄存器，其最低有效两位包含目标的特权级别。

‘org.gnu.gdb.riscv.csr’ 特性是可选的。如果存在，它 应包含目标的所有标准控制状态寄存器（CSR）。标准 CSR 是 RISC-V 规范文档中定义的那些。此特性与 FPU 特性之间存在一些 重叠；‘浮点标志’， ‘浮点舍入模式’，以及 ‘浮点控制状态寄存器’ 寄存器可以在任一特性中。预期是，如果目标有浮点硬件，这些寄存器将在 FPU 特性中，但如果目标有浮点控制寄存器，但没有其他浮点硬件，则可以将它们移到 CSR 特性中。

‘org.gnu.gdb.riscv.vector’ 特性是可选的。如果存在， 它应包含寄存器 ‘v0’ 到 ‘v31’，所有这些寄存器的大小必须相同。
