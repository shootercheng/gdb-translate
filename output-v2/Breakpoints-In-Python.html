23.3.2.33 使用 Python 操作断点 ¶

Python 代码可以通过 gdb.Breakpoint 类。

gdb.Breakpoint 构造函数。第一种形式接受一个字符串，就像你会传递给 break (请参阅 设置断点) 和 watch (请参阅 设置观察点) 命令，可用于创建断点和观察点。第二种形式接受单独的 Python 参数，类似于 显式位置, 只能用于创建断点。

函数： Breakpoint.__init__ (规范 , 类型 ][, 观察点类 , 内部的 ][, 临时的 ][, 限定的 ]) ¶

根据 规范创建一个新的断点，该规范是一个命名断点位置的字符串，或者是一个定义观察点的表达式。该字符串应使用命令（请参阅 break 设置断点 ) 或在观察点的情况下使用命令（请参阅 watch 设置观察点 ) 识别的格式描述一个位置。).

可选的 类型 参数指定要创建的断点的类型，如下定义。

可选的 观察点类 参数定义要创建的观察点的类，如果 类型 是 gdb.BP_WATCHPOINT. 如果 观察点类 被省略，它默认值为 gdb.WP_WRITE.

可选的 内部的 参数允许断点对用户不可见。断点在创建时不会被报告，也不会在 info breakpoints 的输出中列出（但会在 maint info breakpoints 命令中列出）。

可选的 临时的 参数使断点成为临时断点。临时断点在被命中后会被删除。在断点被命中后对 Python 断点的任何进一步访问都将导致运行时错误（因为该断点现在已被自动删除）。

可选的 限定的 参数是一个布尔值，允许将传入的函数解释为完全限定的名称。它等同于 spec 的 break标志（请参阅 -qualified 行规范位置 和 显式位置 ）。).

函数： Breakpoint.__init__ ([ 源文件 ][, 函数 ][, 标签 ][, 行 ], ][ 内部的 ][, 临时的 ][, 限定的 ]) ¶

创建新断点的第二种形式指定了显式位置（请参阅 显式位置)，使用关键字。新断点将在指定的源文件中创建 源文件，在指定的 函数, 标签 和 行处。

内部的, 临时的 和 限定的 的用法与之前解释的相同。

可用的类型由 gdb 模块中定义的常量表示：

gdb.BP_BREAKPOINT ¶

普通代码断点。

gdb.BP_HARDWARE_BREAKPOINT ¶

硬件辅助代码断点。

gdb.BP_WATCHPOINT ¶

观察点断点。

gdb.BP_HARDWARE_WATCHPOINT ¶

硬件辅助观察点。

gdb.BP_READ_WATCHPOINT ¶

硬件辅助读观察点。

gdb.BP_ACCESS_WATCHPOINT ¶

硬件辅助访问观察点。

gdb.BP_CATCHPOINT ¶

捕获点。目前，创建 gdb.Breakpoint 对象时不能使用此类型，但在 gdb.Breakpoint 报告的对象中会存在（请参阅 gdb.BreakpointEvent Python 中的事件)。

可用的观察点类型由 gdb 模块中定义的常量表示：

gdb.WP_READ ¶

只读观察点。

gdb.WP_WRITE ¶

只写观察点。

gdb.WP_ACCESS ¶

读写观察点。

函数： Breakpoint.stop (self) ¶

该 gdb.Breakpoint 类可以被继承，特别是，你可以选择实现 stop 方法。如果在 gdb.Breakpoint的子类中定义了此方法，当被调试程序到达实例化该子类的断点的任何位置时，将调用此方法。如果该方法返回 True, 被调试程序将在断点位置停止，否则被调试程序将继续执行。

如果在同一位置有多个具有 stop 方法的断点，无论前一个方法的返回状态如何，每个方法都会被调用。这确保了所有 stop 方法都有机会在该位置执行。在这种情况下，如果其中一个方法返回 True 但其他方法返回 False, 被调试程序仍将停止。

你不应改变被调试程序的执行状态（即单步执行、下一步等），改变当前帧上下文（即更改当前活动帧），或更改、添加或删除任何断点。一般来说，此时你不应更改 GDB 或被调试程序内的任何数据。

示例 stop 实现： 

class MyBreakpoint (gdb.Breakpoint):
      def stop (self):
        inf_val = gdb.parse_and_eval("foo")
        if inf_val == 3:
          return True
        return False
函数： Breakpoint.is_valid () ¶

如果此 True 对象有效，则返回 Breakpoint , 否则返回 False 。如果用户删除了断点， Breakpoint 对象可能会变得无效。在这种情况下，对象仍然存在，但底层的断点不存在了。在观察点作用域的情况下，即使被调试程序的执行离开了该观察点的作用域，观察点仍然有效。

函数： Breakpoint.delete () ¶

永久删除 GDB 断点。这也会使 Python Breakpoint 对象无效。对该对象的属性或方法的任何进一步访问都将引发错误。

变量： Breakpoint.enabled ¶

如果断点已启用，此属性为 True , 否则为 False 。此属性是可写的。你可以使用它来启用或禁用断点。

变量： Breakpoint.silent ¶

如果断点是静默的，此属性为 True , 否则为 False 。此属性是可写的。

请注意，如果断点有命令且第一个命令是 silent, 断点也可以是静默的。这不会通过 silent 属性报告。

变量： Breakpoint.pending ¶

如果断点是挂起的，此属性为 True , 否则为 False 。请参阅 设置断点. 此属性是只读的。

变量： Breakpoint.thread ¶

如果断点是特定线程的（请参阅 特定线程的断点），此属性保存线程的全局 ID。如果断点不是特定线程的，此属性为 None. 此属性是可写的。

在任何时候， Breakpoint.thread 或 Breakpoint.inferior 中只有一个可以设置为有效的 ID，也就是说，断点可以是特定线程的，也可以是特定被调试程序的，但不能两者都是。

变量： Breakpoint.inferior ¶

如果断点是特定被调试程序的（请参阅 特定被调试程序的断点），此属性保存被调试程序的 ID。如果断点不是特定被调试程序的，此属性为 None.

此属性可以为类型为 gdb.BP_BREAKPOINT 和 gdb.BP_HARDWARE_BREAKPOINT的断点写入。

变量： Breakpoint.task ¶

如果断点是特定 Ada 任务的，此属性保存 Ada 任务的 ID。如果断点不是特定任务的（或者底层语言不是 Ada），此属性为 None. 此属性是可写的。

变量： Breakpoint.ignore_count ¶

此属性保存断点的忽略计数，是一个整数。此属性是可写的。

变量： Breakpoint.number ¶

此属性保存断点的编号 —— 用户用于操作断点的标识符。此属性不可写。

变量： Breakpoint.type ¶

此属性保存断点的类型 —— 用于确定实际断点类型或用例的标识符。此属性不可写。

变量： Breakpoint.visible ¶

此属性指示断点在设置时或运行 ‘info breakpoints’ 命令时是否对用户可见。此属性不可写。

变量： Breakpoint.temporary ¶

此属性指示断点是否作为临时断点创建。临时断点在被命中后会自动删除。在断点被命中后（因为它已被自动删除），访问此属性以及除 is_valid 函数之外的所有其他属性和函数将导致错误。此属性不可写。

变量： Breakpoint.hit_count ¶

此属性保存断点的命中计数，是一个整数。此属性是可写的，但目前只能设置为零。

变量： Breakpoint.location ¶

此属性保存用户指定的断点位置。它是一个字符串。如果断点没有位置（即它是一个观察点），此属性的值为 None. 此属性不可写。

变量： Breakpoint.locations ¶

获取为此断点插入的最新断点位置列表，元素类型为 gdb.BreakpointLocation （如下所述）。此功能与 info breakpoint 命令（请参阅 设置断点）相匹配，因为它只检索最新的位置列表，因此返回的列表本身不会在后台更新。此属性不可写。

变量： Breakpoint.expression ¶

此属性保存用户指定的断点表达式。它是一个字符串。如果断点没有表达式（断点不是观察点），此属性的值为 None. 此属性不可写。

变量： Breakpoint.condition ¶

此属性保存用户指定的断点条件。它是一个字符串。如果没有条件，此属性的值为 None. 此属性是可写的。

变量： Breakpoint.commands ¶

此属性保存附加到断点的命令。如果有命令，此属性的值是一个包含所有命令的字符串，命令之间用换行符分隔。如果没有命令，此属性为 None. 此属性是可写的。

断点位置 ¶

断点位置是设置断点的实际位置之一，在 Python API 中由 gdb.BreakpointLocation 类型表示。此类型不会由用户直接实例化，而是从 Breakpoint.locations 中检索，它返回当前设置断点的位置列表。如果加载了新的符号文件或关闭了动态加载的库，断点位置可能会变得无效。访问无效断点位置的属性将抛出 RuntimeError 异常。再次访问 Breakpoint.locations 属性以检索新的有效断点位置列表。

变量： BreakpointLocation.source ¶

此属性返回设置此位置的源文件路径和行号。此属性的类型是一个由 字符串 和 长整型组成的元组。如果断点位置没有源位置，它将返回 None，观察点和捕获点就是这种情况。如果位置已无效，这将抛出 RuntimeError 异常。此属性不可写。

变量： BreakpointLocation.address ¶

此属性返回设置此位置的地址。此属性的类型是长整型。如果位置已无效，这将抛出 RuntimeError 异常。此属性不可写。

变量： BreakpointLocation.enabled ¶

此属性保存此位置是否启用的值。此属性是可写的（布尔型）。如果位置已无效，这将抛出 RuntimeError 异常。

变量： BreakpointLocation.owner ¶

此属性保存对 gdb.Breakpoint 所有者对象的引用，从此对象中检索到了此 gdb.BreakpointLocation 。如果位置已无效，这将抛出 RuntimeError 异常。此属性不可写。

变量： BreakpointLocation.function ¶

此属性获取设置此位置的函数的名称。如果找不到函数，此属性返回 None. 如果位置已无效，这将抛出 RuntimeError 异常。此属性不可写。

变量： BreakpointLocation.fullname ¶

此属性获取设置此位置的完整名称。如果找不到完整名称，此属性返回 None. 如果位置已无效，这将抛出 RuntimeError 异常。此属性不可写。

变量： BreakpointLocation.thread_groups ¶

此属性获取设置此位置的线程组。它返回一个 List 线程组 ID 的列表。如果位置已无效，这将抛出 RuntimeError 异常。此属性不可写。
