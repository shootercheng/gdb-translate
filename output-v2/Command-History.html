22.3 命令历史记录 ¶

GDB 可以跟踪您在调试会话期间输入的命令，这样您就可以确切知道发生了什么。使用这些命令来管理 GDB 命令历史记录功能。

GDB 使用 GNU 历史库（Readline 包的一部分）来提供历史记录功能。 请参阅 交互式使用历史记录, 有关历史库的详细描述。

要向 GDB 发出命令而不影响用户看到的某些状态方面，请在命令前加上 ‘服务器 ’ （请参阅 服务器前缀)。这 意味着此命令不会影响命令历史记录，也不会影响 GDB在按下 回车键 时重复哪个命令的概念。

服务器前缀不会影响将值记录到值历史记录中；要打印一个值而不将其记录到值历史记录中，请使用 output 命令而不是 print 命令。

以下是与命令历史记录相关的 GDB 命令的描述。

set history filename [fname] ¶

将 GDB 命令历史记录文件的名称设置为 fname. 这是 GDB 读取初始命令历史记录列表的文件，也是它在退出时写入本次会话命令历史记录的文件。您可以通过历史扩展或下面列出的历史命令编辑字符访问此列表。此文件默认值为环境变量 GDBHISTFILE, 或者如果此变量未设置，则为 ./.gdb_history (./_gdb_history 在 MS - DOS 上)。

在处理任何 GDBHISTFILE 初始化文件（请参阅 GDB 启动时的操作 GDB ）之后以及处理通过命令行选项传递的任何命令（例如) 之后读取 -ex环境变量。

如果未提供 fname 参数，或者 GDBHISTFILE 为空字符串，则 GDB 既不会尝试加载现有的历史记录文件，也不会在退出时尝试保存历史记录。

set history save ¶
set history save on

将命令历史记录记录到一个文件中，该文件的名称可以使用 set history filename 命令指定。默认情况下，此选项是禁用的。当 GDB 退出时，命令历史记录将被记录。如果 set history filename 设置为空字符串，则即使 set history save 为 on.

set history save off

在 set history filename 退出时，不要将命令历史记录记录到 GDB 指定的文件中。

set history size size ¶
set history size unlimited

设置 GDB 在其历史记录列表中保留的命令数量。 此值默认为环境变量 GDBHISTSIZE, 或者如果此变量未设置，则为 256。忽略 GDBHISTSIZE 的非数字值。如果 size 为 unlimited 或者 GDBHISTSIZE 为负数或空字符串，则 GDB 在历史记录列表中保留的命令数量是无限的。

在处理任何 GDBHISTSIZE 初始化文件（请参阅 GDB 启动时的操作 GDB ）之后以及处理通过命令行选项传递的任何命令（例如) 之后读取 -ex环境变量。

set history remove-duplicates count ¶
set history remove-duplicates unlimited

控制命令历史记录列表中重复历史记录条目的删除。 如果 count 不为零， GDB 将查看最后 count 条历史记录条目，并删除与当前要添加到命令历史记录列表中的条目重复的第一个条目。如果 count 为 unlimited ，则此回溯是无界的。如果 count 为 0，则禁用重复历史记录条目的删除。

仅考虑当前会话期间添加的历史记录条目进行删除。此选项默认设置为 0。

历史扩展为字符 !. 请参阅 事件指示符， 有关更多详细信息。

自 ! 在 C 语言中也是逻辑非运算符，因此历史扩展 默认是关闭的。如果您决定使用 set history expansion on 命令启用历史扩展，有时可能需要 在 ! （当它在表达式中用作逻辑非时）后面加上一个空格或制表符，以防止它被扩展。Readline 历史记录功能不会对字符串进行替换 != 和 !(, 即使启用了历史扩展也是如此。

控制历史扩展的命令如下：

set history expansion on ¶
set history expansion

启用历史扩展。历史扩展默认是关闭的。

set history expansion off

禁用历史扩展。

show history ¶
show history filename
show history save
show history size
show history expansion

这些命令显示 GDB 历史参数的状态。 show history 单独使用时会显示所有四种状态。

show commands ¶

显示命令历史记录中的最后十条命令。

show commands n

打印以命令编号 n为中心的十条命令。

show commands +

打印上次打印的命令之后的十条命令。
