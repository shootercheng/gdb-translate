23.1.1 用户定义的命令 ¶

一个 用户定义的命令 是一系列 GDB 命令，你为其分配一个新的命令名。这可以通过 define 命令来完成。

用户命令可以接受无限数量的由空格分隔的参数。参数可以在用户命令中通过 $arg0…$argN访问。参数是文本替换，因此 它们可以引用变量、使用复杂的表达式，甚至执行 底层函数调用。然而，请注意，这种文本 替换意味着处理某些参数很困难。 例如，用户无法传递包含空格的参数；虽然可以使用像 "$arg1"这样的表达式将参数字符串化，但如果参数包含引号，这将失败。对于更复杂和健壮的命令， 我们建议用 Python 编写；请参阅 Python 中的 CLI 命令。

一个简单的例子： 

define adder
  print $arg0 + $arg1 + $arg2
end

要执行该命令，请使用： 

adder 1 2 3

这定义了命令 adder，它打印其三个参数的和。

此外， $argc 可用于确定传递了多少个参数。 

define adder
  if $argc == 2
    print $arg0 + $arg1
  end
  if $argc == 3
    print $arg0 + $arg1 + $arg2
  end
end

与 eval 命令结合使用（请参阅 eval）可以更轻松地处理可变数量的参数： 

define adder
  set $i = 0
  set $sum = 0
  while $i < $argc
    eval "set $sum = $sum + $arg%d", $i
    set $i = $i + 1
  end
  print $sum
end
define commandname ¶

定义一个名为 命令名的命令。如果已经有一个同名的命令，系统会要求你确认是否要重新定义它。 参数 命令名 可以是一个由字母、 数字、破折号、点和下划线组成的简单命令名。它也可以以任何预定义或用户定义的前缀命令开头。 例如，‘define target my-target’ 创建了一个用户定义的 ‘target my-target’ 命令。

命令的定义由其他 GDB 命令行组成，这些命令行在 define 命令之后给出。这些命令的结束由包含 end的行标记。

document commandname ¶

为用户定义的命令 命令名编写文档，以便可以通过 help访问。命令 命令名 必须已经定义。此命令读取文档行的方式与 define 读取命令定义行的方式相同，以 end结束。 在 document 命令完成后， help 在命令 命令名 上显示你编写的文档。

你可以再次使用 document 命令来更改命令的文档。使用 define 重新定义命令不会更改文档。

也可以为用户定义的别名编写文档。别名文档 将被 help 和 apropos 命令使用，而不是被别名命令的文档使用。 为用户定义的别名编写文档在将别名定义为一组嵌套的 with 命令时特别有用（请参阅 默认参数）。

define-prefix commandname ¶

定义或标记命令 命令名 为用户定义的前缀 命令。一旦标记， 命令名 可以被 define 命令用作前缀命令。 请注意， define-prefix 可以与尚未定义的 命令名一起使用。在这种情况下， 命令名 被定义为一个空的用户定义命令。 如果你重新定义一个被标记为用户定义前缀命令的命令，重新定义命令的子命令将被保留（并且 GDB 会向用户表明这一点）。

示例： 

(gdb) define-prefix abc
(gdb) define-prefix abc def
(gdb) define abc def
输入 "abc def" 的定义命令。
以仅包含 "end" 的行结束。
>echo command initial def\n
>end
(gdb) define abc def ghi
输入 "abc def ghi" 的定义命令。
以仅包含 "end" 的行结束。
>echo command ghi\n
>end
(gdb) define abc def
保留前缀命令 "def" 的子命令。
重新定义命令 "def" 吗？(y 或 n) y
输入 "abc def" 的定义命令。
以仅包含 "end" 的行结束。
>echo command def\n
>end
(gdb) abc def ghi
command ghi
(gdb) abc def
command def
(gdb)
dont-repeat ¶

在用户定义的命令中使用，这告诉 GDB 当用户按下 回车键 时，此命令不应重复（请参阅 重复最后一个命令）。

help user-defined ¶

列出所有用户定义的命令和在类 COMMAND_USER 中定义的所有 Python 命令。包含文档或文档字符串的第一行（如果有的话）。

show user ¶
show user commandname

显示用于定义 GDB 命令的 命令名 (但 不是其文档)。如果没有 命令名 被指定，则显示所有用户定义命令的定义。 这对用户定义的 Python 命令不起作用。

show max-user-call-depth ¶
set max-user-call-depth

的值 max-user-call-depth 控制在用户定义的命令中允许的递归 级别数，在 GDB 怀疑存在无限递归并中止命令之前。 这不适用于用户定义的 Python 命令。

除了上述命令之外，用户定义的命令经常 使用控制流命令，在 命令文件中描述。

当执行用户定义的命令时，定义中的命令不会被打印。任何命令中的错误都会停止用户定义命令的执行。

如果交互式使用，那些通常会要求确认的命令在用户定义的命令中使用时不会要求确认。许多 GDB 通常会打印消息说明其操作的命令在用户定义的命令中使用时会省略这些消息。
