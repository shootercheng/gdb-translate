22.2 命令编辑 ¶

GDB 通过 Readline 接口读取其输入命令。这个 GNU 库为向用户提供命令行界面的程序提供一致的行为。优点包括 GNU Emacs 风格的 或 vi风格的命令内联编辑， csh类似的历史记录替换，以及跨调试会话存储和调用命令历史记录。

您可以在 GDB 中使用命令 set.

set editing ¶
set editing on

启用命令行编辑（默认启用）。

set editing off

禁用命令行编辑。

show editing ¶

显示命令行编辑是否已启用。

参见 命令行编辑, 有关 Readline 接口的更多详细信息。不熟悉 GNU Emacs 或 vi 的用户 建议阅读该章节。

GDB 将 Readline 应用程序名称设置为 ‘gdb’。这在 .inputrc.

GDB 定义一个可绑定的 Readline 命令， operate-and-get-next. 此命令默认绑定到 Ctrl - O 。此命令接受当前行以执行，并从历史记录中获取相对于当前行的下一行进行编辑。任何参数都会被忽略。

可以使用 GDB 来发现 显示配置 命令（参见 显示配置) 来了解使用的 Readline 版本。
