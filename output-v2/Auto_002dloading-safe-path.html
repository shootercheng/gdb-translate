22.8.3 自动加载的安全限制 ¶

由于被调试程序的文件可能来自不可信的源（例如由 应用程序用户提交） GDB 并不总是自动加载任何文件。 GDB 提供了 ‘设置自动加载安全路径’ 设置，用于列出 用户未明确请求加载文件时可信任的目录。 每个目录也可以是一个shell通配符模式。

如果路径设置不正确，您将看到一条警告，并且文件将不会 被加载： 

$ ./gdb -q ./gdb
正在从 /home/user/gdb/gdb 读取符号...
警告：文件 "/home/user/gdb/gdb-gdb.gdb" 的自动加载已被
         您的 `自动加载安全路径` 设置拒绝
         为 "$debugdir:$datadir/自动加载"。
警告：文件 "/home/user/gdb/gdb-gdb.py" 的自动加载已被
         您的 `自动加载安全路径` 设置拒绝
         为 "$debugdir:$datadir/自动加载"。

要指示 GDB 继续并使用初始化文件， 请像这样调用 GDB ： 

$ gdb -q -iex "设置自动加载安全路径 /home/user/gdb" ./gdb

可信任目录列表由以下命令控制：

set auto-load safe-path [directories] ¶

设置可信任用于自动 加载和执行脚本的目录（及其子目录）列表。您也可以输入一个特定的可信任文件。 每个目录也可以是一个shell通配符模式；通配符不匹配 目录分隔符 - 请参阅 FNM_PATHNAME 系统函数 fnmatch (请参阅 fnmatch 在 GNU C库参考手册)。 如果您省略 目录, ‘自动加载安全路径’ 将被重置为 编译期间指定的默认值。 GDB

目录列表使用路径分隔符 (‘:’ 在GNU和Unix 系统上，‘;’ 在MS-Windows和MS-DOS上) 来分隔目录，类似于 环境变量。 PATH

show auto-load safe-path ¶

显示可信任用于自动加载和执行脚本的目录列表。

add-auto-load-safe-path ¶

向可信任用于自动 加载和执行脚本的目录列表中添加一个条目（或条目列表）。多个条目可以用 当前主机平台的路径分隔符分隔。

此变量默认设置为 --with-auto-load-dir 已配置的值 (请参阅 带自动加载目录)。 $debugdir 和 $datadir 的替换规则与 设置自动加载脚本目录相同。 默认 set auto-load safe-path 值也可以通过 GDB 配置选项 --带自动加载安全路径覆盖。

将此变量设置为 / 将禁用此安全保护， 对应的 GDB 配置选项是 --不带自动加载安全路径。 此变量应该设置为仅系统超级用户可写的系统目录。用户可以在其主目录的初始化文件中添加他们的源目录（请参阅 主目录初始化文件)。另请参阅当前目录中已弃用的 初始化文件 (请参阅 启动时当前目录中的初始化文件)。

要强制 GDB 加载上一个示例中拒绝加载的文件，您可以使用以下方法之一：

~/.gdbinit: ‘添加自动加载安全路径 ~/src/gdb’

将此可信任目录（或文件）指定为列表的附加组件。 您还必须指定 ‘显示自动加载安全路径’ 显示的任何现有目录（例如 ‘/usr:/bin’ 在这个示例中）。

gdb -iex "设置自动加载安全路径 /usr:/bin:~/src/gdb" …

像上一种情况一样指定此目录，但仅用于单个 GDB 会话。

gdb -iex "设置自动加载安全路径 /" …

为单个 GDB 会话禁用自动加载安全保护。 这假设您在此 GDB 会话期间调试的所有文件都来自可信任的源。

./configure --不带自动加载安全路径

在编译 GDB 时，您可以禁用任何自动加载安全保护。 这假设您将来使用此 GDB 调试的所有文件都来自可信任的源。

另一方面，您也可以明确禁止自动加载文件，这 也会抑制任何此类警告消息：

gdb -iex "设置自动加载为否" …

您可以使用 GDB 命令行选项用于单个 GDB 会话。

~/.gdbinit: ‘设置自动加载为否’

为用户全局禁用自动加载 (请参阅 主目录初始化文件)。虽然可能性不大，但您也可以 使用系统初始化文件代替（请参阅 系统范围的配置和设置)。

此设置适用于用户输入的文件名。如果没有匹配项 GDB 作为最后手段，会尝试将所有文件名解析为 其规范形式（通常是解析符号链接），然后再次比较 这些条目。 GDB 在开始比较之前，它自己已经将大多数文件名规范化了 所以建议输入目录的规范形式。
