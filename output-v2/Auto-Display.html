10.8 自动显示 ¶

如果你发现自己经常想打印某个表达式的值（以查看其如何变化），你可能想将其添加到 自动显示列表 中，以便 GDB 每次程序停止时都打印其值。添加到列表中的每个表达式都会被赋予一个编号以识别它；要从列表中移除一个表达式，你需要指定该编号。自动显示如下所示： 

2: foo = 38
3: bar[5] = (struct hack *) 0x3804

此显示显示了项目编号、表达式及其当前值。与你使用 x 或 print, 手动请求的显示一样，你可以指定你喜欢的输出格式；实际上， display 会根据你的格式规范决定使用 print 还是 x —— 如果你指定了‘ x ；否则使用’或‘s’格式，或者指定了单位大小，则使用 print.

display expr ¶

将表达式 expr 添加到每次程序停止时要显示的表达式列表中。请参阅 表达式.

display 如果你在使用它之后再次按下 RET ，它不会重复。

display/fmt expr

对于 fmt 仅指定显示格式而不指定大小或计数的情况，将表达式 expr 添加到自动显示列表中，并安排每次以指定的格式 fmt显示它。请参阅 输出格式.

display/fmt addr

对于 fmt ‘i’或‘s’，或者包含单位大小或单位数量的情况，将表达式 addr 作为内存地址添加，以便每次程序停止时进行检查。检查实际上意味着执行‘x/fmt addr’。请参阅 检查内存.

例如，‘display/i $pc’可能会很有帮助，以便每次执行停止时查看即将执行的机器指令（‘$pc’是程序计数器的常用名称；请参阅 寄存器).

undisplay dnums… ¶
delete display dnums…

从要显示的表达式列表中移除项目。使用命令参数 dnums指定你想要影响的显示编号。它可以是单个显示编号，即‘info display’显示中第一个字段显示的编号之一；也可以是显示编号的范围，如 2-4.

undisplay 如果你在使用它之后按下 RET ，它不会重复。（否则你会得到错误‘No display number …’。）

disable display dnums… ¶

禁用项目编号 dnums的显示。禁用的显示项目不会自动打印，但不会被遗忘。它稍后可能会再次启用。使用命令参数 dnums指定你想要影响的显示编号。它可以是单个显示编号，即‘info display’显示中第一个字段显示的编号之一；也可以是显示编号的范围，如 2-4.

enable display dnums… ¶

启用项目编号 dnums的显示。它在自动显示其表达式时会再次生效，直到你另行指定。使用命令参数 dnums指定你想要影响的显示编号。它可以是单个显示编号，即‘info display’显示中第一个字段显示的编号之一；也可以是显示编号的范围，如 2-4.

display

显示列表中表达式的当前值，就像程序停止时所做的那样。

info display ¶

打印之前设置为自动显示的表达式列表，每个表达式都有其编号，但不显示值。这包括已禁用的表达式，会进行相应标记。它还包括由于引用当前不可用的自动变量而现在不会显示的表达式。

如果一个显示表达式引用了局部变量，那么在其设置的词法上下文之外就没有意义。当执行进入其某个变量未定义的上下文时，这样的表达式会被禁用。例如，如果你给出命令 display last_char 在带有参数的函数内部时 last_char, GDB 当程序继续在该函数内部停止时，会显示这个参数。当程序在其他地方停止时 —— 那里没有变量 last_char —— 显示会自动禁用。下次程序在 last_char 有意义的地方停止时，你可以再次启用该显示表达式。
