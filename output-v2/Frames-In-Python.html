23.3.2.28 从 Python 访问 inferior 栈帧 ¶

当被调试的程序停止时， GDB 能够分析其调用栈（请参阅 栈帧)。 gdb.Frame 类表示栈中的一个帧。一个 gdb.Frame 对象仅在其对应的帧存在于 inferior 的栈中时才有效。如果你尝试使用无效的帧对象， GDB 将抛出一个 gdb.error 异常（请参阅 异常处理)。

两个 gdb.Frame 对象可以使用 == 运算符进行相等性比较，例如： 

(gdb) python print gdb.newest_frame() == gdb.selected_frame ()
True

以下与帧相关的函数在 gdb 模块中可用：

函数： gdb.selected_frame () ¶

返回选定的帧对象。（请参阅 选择一个帧）。

函数： gdb.newest_frame () ¶

返回选定线程的最新帧对象。

函数： gdb.frame_stop_reason_string (reason) ¶

返回一个字符串，解释 GDB 停止展开帧的原因，由给定的 原因 代码（一个整数，请参阅本节后面的方法）表示。 unwind_stop_reason 方法。

函数： gdb.invalidate_cached_frames ¶

GDB 内部会缓存已展开的帧。此函数使该缓存无效。

普通 Python 代码通常不应调用此函数。为了完整性而记录此函数。

一个 gdb.Frame 对象具有以下方法：

函数： Frame.is_valid () ¶

如果 gdb.Frame 对象有效，则返回 true；否则返回 false。如果帧对象所引用的帧在 inferior 中不再存在，则该对象可能会变得无效。如果在调用方法时对象无效，所有 gdb.Frame 方法都会抛出异常。

函数： Frame.name () ¶

返回帧的函数名，如果无法获取则返回 None 。

函数： Frame.architecture () ¶

返回与帧的架构对应的 gdb.Architecture 对象。请参阅 架构的 Python 表示形式。

函数： Frame.type () ¶

返回帧的类型。该值可以是以下之一：

gdb.NORMAL_FRAME

普通栈帧。

gdb.DUMMY_FRAME

在执行 inferior 函数调用时由 GDB 创建的伪栈帧。

gdb.INLINE_FRAME

表示内联函数的帧。该函数被内联到一个 gdb.NORMAL_FRAME 比此帧更旧的帧中。

gdb.TAILCALL_FRAME

表示尾调用的帧。请参阅 尾调用帧。

gdb.SIGTRAMP_FRAME

信号跳板帧。这是操作系统调用信号处理程序时创建的帧。

gdb.ARCH_FRAME

表示跨架构调用的伪栈帧。

gdb.SENTINEL_FRAME

这类似于 gdb.NORMAL_FRAME, 但仅用于最新的帧。

函数： Frame.unwind_stop_reason () ¶

返回一个整数，表示无法向最外层帧查找更多帧的原因。使用 gdb.frame_stop_reason_string 将此函数返回的值转换为字符串。该值可以是以下之一：

gdb.FRAME_UNWIND_NO_REASON

没有特定原因（应该可以获取更旧的帧）。

gdb.FRAME_UNWIND_NULL_ID

前一帧的分析器返回无效结果。此情况不再被 GDB使用，仅为向后兼容而保留。

gdb.FRAME_UNWIND_OUTERMOST

此帧是最外层帧。

gdb.FRAME_UNWIND_UNAVAILABLE

无法进一步展开，因为这需要知道尚未收集的寄存器或内存的值。

gdb.FRAME_UNWIND_INNER_ID

此帧 ID 看起来应该属于 NEXT 帧，但我们是为 PREV 帧获取到的。通常，这是展开器失败的标志。也可能表示栈损坏。

gdb.FRAME_UNWIND_SAME_ID

此帧的 ID 与前一帧相同。这意味着进一步展开几乎肯定会得到另一个 ID 完全相同的帧，因此断开链。通常，这是展开器失败的标志。也可能表示栈损坏。

gdb.FRAME_UNWIND_NO_SAVED_PC

帧展开器未找到任何保存的程序计数器（PC），但我们需要它才能进一步展开。

gdb.FRAME_UNWIND_MEMORY_ERROR

帧展开器在尝试访问内存时引发了错误。

gdb.FRAME_UNWIND_FIRST_ERROR

任何大于或等于此值的停止原因都表示某种错误。这个特殊值便于编写测试展开错误的代码，即使其他值的列表在未来版本中被修改，该代码也能正常工作。 GDB 使用它，你可以编写： 

reason = gdb.selected_frame().unwind_stop_reason ()
reason_str =  gdb.frame_stop_reason_string (reason)
if reason >=  gdb.FRAME_UNWIND_FIRST_ERROR:
    print ("An error occurred: %s" % reason_str)
函数： Frame.pc () ¶

返回帧的恢复地址。

函数： Frame.block () ¶

返回帧的代码块。请参阅 从 Python 访问块。如果帧没有块（例如，如果相关代码没有调试信息），则会抛出异常。

函数： Frame.function () ¶

返回与此帧对应的函数的符号。请参阅 符号的 Python 表示形式。

函数： Frame.older () ¶

返回调用此帧的帧。如果这是最旧的帧，则返回 None。

函数： Frame.newer () ¶

返回此帧调用的帧。如果这是最新的帧，则返回 None。

函数： Frame.find_sal () ¶

返回帧的符号表和行对象。请参阅 Python 中的符号表表示形式。

函数： Frame.read_register (register) ¶

返回 寄存器 在此帧中的值。返回一个 Gdb.Value 对象。如果 寄存器 不存在，则抛出异常。 寄存器 参数必须是以下之一：

  1. 一个表示有效寄存器名称的字符串（例如， 'sp' 或 'rax'）。
  2. 一个 gdb.RegisterDescriptor 对象（请参阅 Python 中的寄存器)。
  3. 一个 GDB 内部的、特定于平台的数字。出于历史原因支持使用这些数字，但不建议这样做，因为未来对 GDB 的更改可能会改变数字与它们所代表的寄存器之间的映射，从而破坏任何使用特定于平台数字的 Python 代码。这些数字通常可以在相应的 平台-tdep.h 文件中找到，该文件位于 GDB 源代码树中。

使用字符串访问寄存器比其他两种方法稍慢，因为 GDB 必须查找名称与内部寄存器编号之间的映射。如果性能至关重要，可以考虑查找并缓存一个 gdb.RegisterDescriptor 对象。

函数： Frame.read_var (variable [, block]) ¶

返回 变量 在此帧中的值。如果提供了可选参数 块 ，则从该块中搜索变量；否则从帧的当前块开始搜索（该块由帧的当前程序计数器确定）。 变量 参数必须是字符串或 gdb.Symbol 对象； 块 必须是 gdb.Block 对象。

函数： Frame.select () ¶

将此帧设置为选定的帧。请参阅 检查栈。

函数： Frame.static_link () ¶

在某些语言（例如 Ada，还有 GNU C 扩展）中，嵌套函数可以访问外部作用域中的变量。这是通过“静态链接”实现的，它是从嵌套帧到适当外部帧的引用。

此方法返回此帧的静态链接帧（如果存在）。如果没有静态链接，此方法返回 None。

函数： Frame.level () ¶

返回一个整数，即此帧的栈帧级别。请参阅 栈帧。

函数： Frame.language () ¶

返回一个字符串，即此帧的源语言。
