¶

漂亮打印机只是一个持有值并实现特定接口的对象，这里进行了定义。提供了一个示例输出（请参阅 漂亮打印)。

因为 GDB 没有记录漂亮打印机的可扩展性，所以默认情况下 GDB 将假设只有基本的漂亮打印机方法可能可用。下面会对基本方法进行标记。

为了实现可扩展性， GDB 提供了 gdb.ValuePrinter 基类。此类不提供任何属性或行为，而是作为一个标签，可被 GDB识别。对于此类打印机， GDB 保留所有以小写字母开头的属性。也就是说，将来 GDB 可能会向漂亮打印机协议添加新的方法或属性，并且 gdb.ValuePrinter基类的打印机应该能够优雅地处理这种情况。一种简单的方法是在实现的本地属性前加上一个下划线（或两个，遵循 Python 名称修饰方案）。

函数： 漂亮打印机的子项 (self) ¶

GDB 将在漂亮打印机上调用此方法来计算漂亮打印机值的子项。

此方法必须返回一个符合 Python 迭代器协议的对象。迭代器返回的每个项必须是一个包含两个元素的元组。第一个元素是子项的“名称”；第二个元素是子项的值。该值可以是任何可以转换为 GDB 值的 Python 对象。

这是一个基本方法，是可选的。如果该方法不存在， GDB 将认为该值没有子项。

为了提高效率， children 方法应该惰性计算其结果。这将使 GDB 在各种打印设置（请参阅 打印设置）或 -var-list-children （请参阅 GDB/MI 变量对象）限制要显示的元素数量时，只读取必要的元素。

子项可能会根据‘设置 打印最大深度’的值（请参阅 打印设置）而隐藏不显示。

函数： 漂亮打印机的显示提示 (self) ¶

CLI 可能会调用此方法并使用其结果来更改值的格式。该结果也将作为正在打印的变量的‘显示提示’属性提供给 MI 消费者。

这是一个基本方法，是可选的。如果该方法存在，它必须返回一个字符串或特殊值 None。

一些显示提示由 GDB预定义：

‘数组’

表示正在打印的对象是“类似数组”的。CLI 使用此提示来遵循诸如 set print elements 和 set print array之类的参数。

‘映射’

表示正在打印的对象是“类似映射”的，并且可以假设此值的子项在键和值之间交替。

‘字符串’

表示正在打印的对象是“类似字符串”的。如果打印机的 to_string 方法返回某种 Python 字符串，那么 GDB 将调用其内部特定语言的字符串打印函数来格式化该字符串。对于 CLI 来说，这意味着添加引号，可能对某些字符进行转义，遵循 set print elements等等。

特殊值 None 会使 GDB 应用默认的显示规则。

函数： 漂亮打印机转换为字符串 (self) ¶

GDB 将调用此方法来显示传递给对象构造函数的值的字符串表示形式。

这是一个基本方法，是可选的。

从 CLI 打印时，如果 to_string 方法存在，那么 GDB 将其结果添加到 children返回的值之前。具体的格式化方式取决于显示提示，并且随着更多提示的添加可能会发生变化。此外，根据打印设置（请参阅 打印设置），CLI 可能在堆栈跟踪中只打印 to_string 的结果，而省略 children的结果。

如果此方法返回一个字符串，则按原样打印。

否则，如果此方法返回一个 gdb.Value的实例，那么 GDB 将打印此值。这可能会调用另一个漂亮打印机。

如果该方法返回一个可以转换为 gdb.Value的 Python 值，那么 GDB 执行转换并打印 结果值。同样，这可能会调用另一个 漂亮打印机。Python 标量（整数、浮点数和布尔值）以及 字符串可以转换为 gdb.Value；其他类型则不行。

最后，如果此方法返回 None ，则在此方法中不再执行进一步的操作，并且不打印任何内容。

如果结果不是这些类型之一，则会引发异常。

函数： 漂亮打印机的子项数量 () ¶

这不是一个基本方法，因此 GDB 只会对从 gdb.ValuePrinter派生的对象调用它。

如果可用，此方法应返回子项的数量。 None 如果无法轻松计算数量，可以返回。

函数： 漂亮打印机的子项 (n) ¶

这不是一个基本方法，因此 GDB 只会对从 gdb.ValuePrinter派生的对象调用它。

如果可用，此方法应返回由 n指定的子项（即包含此子项的名称和值的元组）。 索引从 0 开始。
