下一项： GDB 远程串行协议   [索引]

¶

数据包允许 GDB 在远程链接的另一端执行 I/O 操作。例如，主机 I/O 用于向具有自己文件系统的远程目标上传和下载文件。主机 I/O 使用与目标发起的文件 I/O 协议相同的常量值和数据结构布局。但是，主机 I/O 数据包的结构不同。目标发起的协议依赖目标内存来存储参数和缓冲区。主机 I/O 请求由 GDB, 并且不涉及目标的内存。请参阅 文件 I/O 远程协议扩展

‘vFile:操作: 参数…’

操作 是特定请求的名称；目标在检查支持的操作时，应比较整个数据包名称直到第二个冒号。 参数 的格式取决于操作。数字总是以十六进制传递。负数有明确的负号（即不使用补码）。字符串（例如文件名）编码为一系列十六进制字节。系统调用的最后一个参数可能是一个转义二进制数据的缓冲区（请参阅 二进制数据）。

主机 I/O 数据包的有效响应如下：

‘F 结果 [, 错误号] [; 附件]’

结果 是此操作返回的整数值，通常成功时为非负数，错误时为 -1。如果发生错误， 错误号 将包含在结果中，指定由文件 I/O 协议定义的值（请参阅 错误号值）。对于返回数据的操作， 附件 以二进制缓冲区的形式提供数据。响应数据包中的二进制缓冲区以正常方式转义（请参阅 二进制数据）。请参阅各个数据包文档以了解 结果 和 附件的解释。

‘’

空响应表示此操作未被识别。

以下是支持的主机 I/O 操作：

‘vFile:open: 文件名, 标志, 模式’

打开 文件名 对应的文件并返回其文件描述符，如果发生错误则返回 -1。 文件名 是一个字符串， 标志 是一个整数，表示打开标志的掩码（请参阅 打开标志），并且 模式 是一个整数，表示如果创建文件时要使用的模式位掩码（请参阅 模式类型值）。请参阅 open，了解打开标志和模式值的详细信息。

‘vFile:close: 文件描述符’

关闭与 文件描述符 对应的打开文件并返回 0，如果发生错误则返回 -1。

‘vFile:pread: 文件描述符, 计数, 偏移量’

从与 文件描述符对应的打开文件中读取数据。从文件起始位置的 字节。目标可能读取的字节数较少；常见原因包括数据包大小限制和文件结束条件。返回读取的字节数。只有在文件末尾成功读取或 处开始，最多将从文件中读取 偏移量 计数 为零时，才应返回零。

成功时，读取的数据应作为二进制附件返回。如果读取的字节数为零，响应应包含一个空的二进制附件（即尾随分号）。返回值是读取的目标字节数；如果某些字符被转义，二进制附件可能会更长。

‘vFile:pwrite: 文件描述符, 偏移量, 数据’

将 数据 （一个二进制缓冲区）写入与 文件描述符对应的打开文件。从文件起始位置的 偏移量 处开始写入。与许多 write 系统调用不同，没有单独的 计数 参数；使用数据包中 数据 的长度。‘vFile:pwrite’ 返回写入的字节数，可能比 数据的长度短，或者如果发生错误则返回 -1。

‘vFile:fstat: 文件描述符’

获取与 文件描述符对应的打开文件的信息。成功时，信息作为二进制附件返回，返回值是该附件的字节大小。如果发生错误，返回值为 -1。返回的二进制附件的格式如 结构体 stat中所述。

‘vFile:stat: 文件名’

获取目标上 文件名 对应的文件的信息，就好像是通过‘stat’ 调用一样。成功时，信息作为二进制附件返回，返回值是该附件的字节大小。如果发生错误，返回值为 -1。返回的二进制附件的格式如 结构体 stat中所述。

如果 文件名 是一个符号链接，那么返回的信息是关于该链接所指向的文件的，这与‘stat’ 库调用一致。

‘vFile:lstat: 文件名’

获取目标上 文件名 对应的文件的信息，就好像是通过‘lstat’ 调用一样。成功时，信息作为二进制附件返回，返回值是该附件的字节大小。如果发生错误，返回值为 -1。返回的二进制附件的格式如 结构体 stat.

此数据包与‘vFile:stat’ 相同，除了如果 文件名 是一个符号链接，那么此数据包返回关于该链接本身的信息，而不是该链接所指向的文件的信息，这与‘lstat’ 库调用一致。

‘vFile:unlink: 文件名’

删除目标上 文件名 对应的文件。返回 0，如果发生错误则返回 -1。 文件名 是一个字符串。

‘vFile:readlink: 文件名’

读取目标上 文件名 对应的符号链接的值。返回读取的字节数，如果发生错误则返回 -1。

成功时，读取的数据应作为二进制附件返回。如果读取的字节数为零，响应应包含一个空的二进制附件（即尾随分号）。返回值是读取的目标字节数；如果某些字符被转义，二进制附件可能会更长。

‘vFile:setfs: 进程 ID’

选择 vFile 操作将在其上进行的文件系统。这对于 文件名 参数的操作是必需的，以便能够访问远程目标上的文件，其中远程存根与下级进程不共享公共文件系统。 GDB

如果 进程 ID 不为零，则选择进程 进程 ID所看到的文件系统。如果 进程 ID 为零，则选择远程存根所看到的文件系统。成功时返回 0，如果发生错误则返回 -1。如果 vFile:setfs: 表示成功，所选文件系统将保持选中状态，直到下一次成功的 vFile:setfs: 操作。
