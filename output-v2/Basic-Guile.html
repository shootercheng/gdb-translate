23.4.3.1 基本Guile ¶

GDB 覆盖Guile的 current-output-port 和 current-error-port 以使用……进行打印。 它还提供了一种将 命令的参数作为表达式进行解析的方法。 在计算数值时也很有用。 例如，这是获取一个 便利变量的值（见 GDB的输出分页流。 一个向这些流之一输出的Guile程序可能会被用户中断其输出（见 屏幕尺寸）。在这种 情况下，一个Guile signal 异常会被抛出，值为 SIGINT。

Guile的历史机制使用与……相同的命名方式， 即使用美元变量（例如，$1、$2等）。 Guile和GDB中的评估结果是分别计数的， GDB $1 在Guile中与 $1 在……中不是同一个值。 GDB。

GDB 不是线程安全的。如果你的Guile程序使用多个 线程，你必须小心只在 GDB特定的 线程中调用……函数。 GDB 线程。

在编写要在……中运行的Guile代码时，必须要小心。 GDB。有两件事特别值得注意：

  • GDB 安装了……的处理程序 SIGCHLD 和 SIGINT。 Guile代码不能覆盖这些处理程序，甚至不能使用……改变选项。 如果你的程序改变了这些信号的处理方式， sigaction GDB 很可能会停止正常工作。注意 不幸的是，GUI工具包通常会安装一个 SIGCHLD 处理程序。
  • GDB 会小心地将其内部文件描述符标记为在执行时关闭。然而，在所有平台上，这不能以线程安全的方式完成。你的Guile程序应该意识到这一点，并且 应该在启动子进程之前，既创建带有执行时关闭标志的新文件描述符，又安排关闭不需要的文件描述符。

GDB 引入了一个新的Guile模块，名为 gdb。所有 由……添加的方法和类都放在这个模块中。 GDB 。 GDB 不会自动 import 该 gdb 模块， 脚本必须自己完成。对于如何导入模块有各种选项，所以 GDB 将如何导入 gdb 模块的选择留给用户。 为了简化交互式使用，建议在你的~/.gdbinit中添加以下内容之一。 

guile (use-modules (gdb))

guile (use-modules ((gdb) #:renamer (symbol-prefix-proc 'gdb:)))

选择哪一个取决于你的偏好。 第二个选项会为所有模块函数 和变量添加 gdb: 作为前缀。

本手册的其余部分假设 gdb 模块已被导入 且没有任何前缀。有关更多信息，请参阅Guile文档中的 use-modules （见 使用Guile模块 在 GNU Guile参考手册）。

示例： 

(gdb) guile (value-type (make-value 1))
ERROR: Unbound variable: value-type
Error while executing Scheme code.
(gdb) guile (use-modules (gdb))
(gdb) guile (value-type (make-value 1))
int
(gdb)

该 (gdb) 模块提供了这些基本的Guile函数。

Scheme过程： 执行 命令 [#:from-tty 布尔值] [#:to-string 布尔值] ¶

评估 命令, 一个字符串，作为一个 GDB CLI命令。 如果在 GDB 命令 运行时发生一个 异常，它将按照 Guile异常处理中所述进行转换。

from-tty 指定是否 GDB 应该将此命令视为用户交互式调用的。 它必须是一个布尔值。如果省略，它默认为 #f。

默认情况下，由 命令 产生的任何输出都会发送到 GDB的标准输出（如果开启了日志记录，也会发送到日志输出）。如果 to-string 参数为 #t, 那么输出将由 execute 收集并作为字符串返回。默认值为 #f, 在这种情况下，返回值未指定。如果 to-string 为 #t, 虚拟终端将临时设置为无限宽度 和高度，并且其分页将被禁用；见 GDB 屏幕尺寸。

Scheme过程： 历史引用 编号 ¶

从……返回一个值 GDB的价值历史记录（见 价值历史记录）。 编号 参数表示要返回哪个历史元素。 如果 编号 为负数，那么 GDB 将取其绝对值 并从最后一个元素（即最近的元素）开始反向计数，以 找到要返回的值。如果 编号 为零，那么 GDB 将 返回最近的元素。如果 编号 指定的元素在价值历史记录中不存在，将抛出一个 gdb:error 异常。

如果没有抛出异常，返回值总是一个……的实例 <gdb:value> （见 Guile中来自下级的数值）。

注意： GDB的价值历史记录与Guile的是独立的。 $1 在 GDB的价值历史记录中包含从 GDB的命令行评估表达式的结果，而 $1 从Guile的 历史记录中包含从Guile的命令行评估表达式的结果。
