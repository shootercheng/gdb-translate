27.9.3 C++ 异常 GDB/MI 捕获点 ¶

以下 GDB/MI 命令可用于创建捕获点，当 C++ 异常被抛出、重新抛出或捕获时，会停止执行。

该 -catch-throw 命令 ¶

概要 ¶

 -catch-throw [ -t ] [ -r 正则表达式]

当调试程序抛出 C++ 异常时停止。 如果 正则表达式 被指定，那么只有类型与正则表达式匹配的异常才会被捕获。

如果指定了 ‘-t’，则捕获点仅在一次停止时启用，在该事件停止一次后，捕获点会自动删除。

GDB 命令 ¶

对应的 GDB 命令是 ‘catch throw’ 和 ‘tcatch throw’（请参阅 设置捕获点)。

示例 ¶

-catch-throw -r exception_type
^完成，断点={编号="1",类型="捕获点",处置="保留",启用="是",
  捕获内容="异常抛出",捕获类型="抛出",
  线程组=["i1"],
  正则表达式="exception_type",触发次数="0"}
(gdb)
-exec-run
^正在运行
(gdb)
~"\n"
~"捕获点 1（抛出异常），0x00007ffff7ae00ed
  在 __cxa_throw () 中，来自 /lib64/libstdc++.so.6\n"

  线程 ID="1"，停止的线程="所有",核心="6"
(gdb)
