E.17 内存映射格式 ¶

为了能够写入闪存， GDB 需要从目标设备获取一个内存映射。本节描述了内存映射的格式。

内存映射是使用‘qXfer:memory-map:read’（参见 qXfer内存映射读取)数据包获取的，它是一个列出内存区域的XML文档。

GDB 必须与Expat库链接以支持XML内存映射。参见 Expat.

文档的顶级结构如下所示： 

<?xml version="1.0"?>
<!DOCTYPE memory-map
          PUBLIC "+//IDN gnu.org//DTD GDB Memory Map V1.0//EN"
                 "http://sourceware.org/gdb/gdb-memory-map.dtd">
<memory-map>
    region...
</memory-map>

每个区域可以是以下两种之一：

  • 从 地址 开始的一段RAM， 长度 字节： 
    <memory type="ram" start="地址" length="长度"/>
  • 一段只读内存： 
    <memory type="rom" start="地址" length="长度"/>
  • 一段闪存，带有擦除块， 块大小 字节长： 
    <memory type="flash" start="地址" length="长度">
  <property name="blocksize">块大小</property>
</memory>

区域不能重叠。 GDB 假设内存映射未覆盖的内存区域是RAM，并使用普通的‘M’和‘X’数据包来写入这些范围内的地址。

内存映射格式的正式DTD如下： 

<!-- ................................................... -->
<!-- Memory Map XML DTD ................................ -->
<!-- File: memory-map.dtd .............................. -->
<!-- .................................... .............. -->
<!-- memory-map.dtd -->
<!-- memory-map: Root element with versioning -->
<!ELEMENT memory-map (memory)*>
<!ATTLIST memory-map    version CDATA   #FIXED  "1.0.0">
<!ELEMENT memory (property)*>
<!-- memory: Specifies a memory region,
             and its type, or device. -->
<!ATTLIST memory        type    (ram|rom|flash) #REQUIRED
                        start   CDATA   #REQUIRED
                        length  CDATA   #REQUIRED>
<!-- property: Generic attribute tag -->
<!ELEMENT property (#PCDATA | property)*>
<!ATTLIST property      name    (blocksize) #REQUIRED>
