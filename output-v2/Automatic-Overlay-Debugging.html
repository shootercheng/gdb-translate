14.3自动覆盖调试 ¶

GDB 在被调试程序中的覆盖管理器进行一些简单协作的情况下，它可以自动跟踪哪些覆盖被映射以及哪些没有被映射。如果使用该命令启用自动覆盖调试（请参阅“覆盖命令”），它会在被调试程序的内存中查找某些描述覆盖当前状态的变量。 overlay auto 命令（请参阅 覆盖命令） GDB 在被调试程序的内存中查找某些描述覆盖当前状态的变量。

以下是你的覆盖管理器必须定义以支持 GDB的自动覆盖调试的变量：

_ovly_table:

此变量必须是以下结构的数组： 

结构体
{
  /* 覆盖的映射地址。  */
  unsigned long vma;

  /* 覆盖的大小，以字节为单位。  */
  unsigned long size;

  /* 覆盖的加载地址。  */
  unsigned long lma;

  /* 如果覆盖当前已映射，则为非零；否则为零。  */
  unsigned long mapped;
}
_novlys:

此变量必须是一个四字节有符号整数，保存 _ovly_table中的元素总数。

为了确定某个特定覆盖是否已映射， GDB 在 _ovly_table 中查找一个条目的 vma 并且 lma 成员等于可执行文件中覆盖段的VMA和LMA。当 GDB 找到匹配的条目时，它会查看该条目的 mapped 成员以确定覆盖当前是否已映射。

此外，你的覆盖管理器可以定义一个名为 _ovly_debug_event的函数。如果定义了此函数， GDB 将在那里悄悄设置一个断点。如果覆盖管理器在每次更改覆盖表时都调用此函数，这将使 GDB 能够准确跟踪哪些覆盖在程序内存中，并更新可能在覆盖中设置的任何断点。这将允许断点即使在覆盖未执行时保存在ROM或其他不可写内存中也能正常工作。
