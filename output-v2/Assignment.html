17.1 给变量赋值 ¶

要改变变量的值，计算赋值表达式。请参阅 表达式。例如， 

print x=4

将值4存储到变量 x中，然后打印赋值表达式的值（即4）。请参阅 使用 GDB 不同语言，以获取有关支持语言中运算符的更多信息。

如果您不想看到赋值的值，请使用 set 命令而不是 print 命令。 set 实际上与 print 相同，只是表达式的值不会被打印，也不会放入值历史记录中（请参阅 值历史记录）。计算表达式仅为了其效果。

如果 set 命令的参数字符串开头看起来与 set 子命令相同，请使用 set variable 命令而不是仅使用 set。此命令与 set 相同，只是它没有子命令。例如，如果您的程序有一个变量 width，如果您尝试仅使用‘set width=13’来设置新值，会得到一个错误，因为 GDB 有命令 set width: 

(gdb) whatis width
type = double
(gdb) p width
$4 = 13
(gdb) set width=47
表达式语法无效。

当然，无效的表达式是‘=47’。为了实际设置程序的变量 width，请使用 

(gdb) set var width=47

因为 set 命令有许多可能与程序变量名冲突的子命令，所以最好使用 set variable 命令而不是仅使用 set。例如，如果您的程序有一个变量 g，如果您尝试仅使用‘set g=4’来设置新值，会遇到问题，因为 GDB 有命令 set gnutarget，缩写为 set g: 

(gdb) whatis g
type = double
(gdb) p g
$1 = 1
(gdb) set g=4
(gdb) p g
$2 = 1
(gdb) r
正在调试的程序已经启动。
要从头开始吗？(y 或 n) y
启动程序：/home/smith/cc_progs/a.out
“/home/smith/cc_progs/a.out”：无法打开以读取符号：
                                 无效的bfd目标。
(gdb) show g
当前的BFD目标是“=4”。

程序变量 g 没有改变，并且您默默地将 gnutarget 设置为无效值。为了设置变量 g, 使用 

(gdb) set var g=4

GDB 在赋值时比C允许更多的隐式转换；您可以自由地将整数值存储到指针变量中，反之亦然，并且可以将任何结构转换为任何长度相同或更短的其他结构。

要将值存储到内存中的任意位置，请使用‘{…}’结构在指定地址生成指定类型的值（请参阅 表达式)。例如， {int}0x83040 将内存位置 0x83040 视为整数（这意味着在内存中有一定的大小和表示形式），并且 

set {int}0x83040 = 4

将值4存储到该内存位置。
