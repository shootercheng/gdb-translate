27.2.1 GDB/MI 输入语法 ¶

command →

cli-command | mi-command

cli-command →

[ token ] cli-command nl，其中 cli命令 是任何现有的 GDB CLI命令。

mi-command →

[ token ] "-" operation ( " " option )* [ " --" ] ( " " parameter )* nl

token →

“任意数字序列”

option →

"-" parameter [ " " parameter ]

parameter →

non-blank-sequence | c-string

operation →

本章中描述的任何操作

non-blank-sequence →

任何内容，前提是它不包含特殊字符，如 “-” 换行符、“""” 当然还有 “ ”

c-string →

""" seven-bit-iso-c-string-content """

nl →

CR | CR-LF

注意事项：

  • CLI命令仍由 MI 解释器处理；其 输出如下所述。
  • 该 token（如果存在）在命令 完成时会被传回。
  • 一些 MI 命令接受可选参数作为参数 列表的一部分。每个选项由一个前导 ‘-’（破折号）标识，并且可能 后跟一个可选的参数。选项在参数 列表中首先出现，可以使用 ‘--’ 与普通参数分隔（当某些参数以破折号开头时，这很有用）。

实际应用：

  • 我们希望能够轻松访问现有的CLI语法（用于调试）。
  • 我们希望能够轻松识别一个 MI 操作。
