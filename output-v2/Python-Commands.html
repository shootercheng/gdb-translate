23.3.1 Python 命令 ¶

GDB 提供了两个用于访问 Python 解释器的命令，以及一个相关设置：

python-interactive [command] ¶
pi [command]

不带参数时， python-interactive 命令可用于启动交互式 Python 提示符。要返回 GDB, 请输入 EOF 字符（例如 Ctrl - D 在空提示符下）。

或者，可以将单行 Python 命令作为参数给出并进行计算。如果命令是一个表达式，则会打印结果；否则，不会打印任何内容。例如： 

(gdb) python - interactive 2 + 3
5
python [command] ¶
py [command]

该 python 命令可用于计算 Python 代码。

如果给出参数， python 命令将把该参数作为 Python 命令进行计算。例如： 

(gdb) python print 23
23

如果不给 python提供参数，它将作为多行命令，类似于 define。在这种情况下，Python 脚本由 python 命令之后给出的后续命令行组成。这个命令列表以包含 end的行结束。例如： 

(gdb) python
>print 23
>end
23
set python print-stack ¶

默认情况下， GDB 在 Python 脚本中发生错误时，只会打印 Python 异常的消息部分。这可以通过 set python print-stack来控制：如果 full, 则启用完整的 Python 堆栈打印；如果 none, 则禁用 Python 堆栈和消息打印；如果 message, 默认情况下，只打印错误的消息部分。

set python ignore-environment [on|off] ¶

默认情况下，此选项为 ‘关闭’，并且当 GDB 初始化其内部 Python 解释器时，Python 解释器将检查环境中会影响其行为的变量，例如 PYTHONHOME, 和 PYTHONPATH18。

如果在 Python 初始化之前将此选项设置为 ‘开启’，则 Python 将忽略所有此类环境变量。由于 Python 在 GDB的启动过程中早期初始化，因此此选项必须放在早期初始化文件中（请参阅 初始化文件) 才能达到预期效果。

此选项相当于向真正的 -E 可执行文件传递 python 。

set python dont-write-bytecode [auto|on|off] ¶

当此选项为 ‘关闭’ 时，一旦 GDB 初始化了 Python 解释器，解释器将对其导入的任何 Python 模块进行字节编译，并将字节码写入磁盘上的 .pyc 文件中。

如果在 Python 初始化之前将此选项设置为 ‘开启’，则 Python 将不再将字节码写入磁盘。由于 Python 在 GDB的启动过程中早期初始化，因此此选项必须放在早期初始化文件中（请参阅 初始化文件) 才能达到预期效果。

默认情况下，此选项设置为 ‘自动’。在此模式下，只要 python ignore-environment 设置为 ‘关闭’，就会检查环境变量 PYTHONDONTWRITEBYTECODE 以确定是否应写出字节码。 PYTHONDONTWRITEBYTECODE 当设置为空字符串或环境变量不存在时，被认为是关闭/禁用的。所有其他设置，包括那些似乎不合理的设置，都表示它是开启/启用的。

此选项相当于向真正的 -B 可执行文件传递 python 。

也可以从 GDB 解释器执行 Python 脚本：

source script-name

脚本名称必须以 ‘.py’ 结尾，并且 GDB 必须使用 script-extension 设置根据文件名扩展名来识别脚本语言。请参阅 扩展 GDB。

以下命令旨在帮助调试 GDB 本身：

set debug py-breakpoint on|off ¶
show debug py-breakpoint

当设置为 ‘开启’ 时 GDB 打印与 Python 断点 API 相关的调试消息。默认情况下为 ‘关闭’。

set debug py-unwind on|off ¶
show debug py-unwind

当设置为 ‘开启’ 时， GDB 打印与 Python 展开器 API 相关的调试消息。默认情况下为 ‘关闭’。
