13.1.8 列出静态跟踪点标记 ¶

info static-tracepoint-markers ¶

显示程序中定义的所有静态跟踪点标记的信息。

对于每个标记，会打印以下列：

计数

一个递增的计数器，用于提高可读性。这不是一个稳定的标识符。

ID

目标报告的标记 ID。

启用或禁用

被探测的标记会标记为 ‘是’。 ‘否’ 表示未启用的标记。

地址

标记在程序中的内存地址。

位置

标记在程序源代码中的文件和行号位置。如果程序中包含的调试信息不允许 GDB 定位标记的源代码，则此列将留空。

此外，还可能为每个标记打印以下信息：

数据

标记调用传递给跟踪库的用户数据。在 UST 后端中，这是作为参数传递给标记调用的格式字符串。

探测该标记的静态跟踪点

附加到该标记的静态跟踪点列表。 

(gdb) info static-tracepoint-markers
计数 ID         启用 地址            位置
1   ust/bar2   是   0x0000000000400e1a 在 stexample.c:25 的 main 函数中
     数据: number1 %d number2 %d
     被静态跟踪点探测: #2
2   ust/bar33  否   0x0000000000400c87 在 stexample.c:24 的 main 函数中
     数据: str %s
