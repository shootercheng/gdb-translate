¶

GDB 提供 便利变量 你可以在其中使用 GDB 来保存一个值并在以后引用它。这些变量完全存在于 GDB；它们不是你程序的一部分，设置便利变量对程序的进一步执行没有直接影响。这就是为什么你可以自由使用它们的原因。

$’。任何以$’ 开头的名称都可以用作便利变量，除非它是预定义的特定于机器的寄存器名称之一（请参阅 寄存器）。（相比之下，值历史记录引用是 数字 ，前面带有 ‘$’。请参阅 值历史记录)

你可以使用赋值表达式将值保存到便利变量中，就像在程序中设置变量一样。例如： 

set $foo = *object_ptr

会将 $foo 中由 object_ptr指向的对象包含的值保存起来。

首次使用便利变量时会创建它，但在你分配新值之前，它的值是 void 。你可以随时通过另一个赋值来更改该值。

便利变量没有固定的类型。你可以为便利变量分配任何类型的值，包括结构体和数组，即使该变量已经有不同类型的值。便利变量在用作表达式时，具有其当前值的类型。

show convenience ¶

打印到目前为止使用的便利变量及其值的列表，以及便利函数的列表。缩写 show conv。

init-if-undefined $variable = expression ¶

如果便利变量尚未设置，则设置它。这对于保留某些状态的用户定义命令很有用。从概念上讲，它类似于在 C 中使用带有初始值设定项的局部静态变量（除了便利变量是全局的）。它还可以用于允许用户覆盖命令脚本中使用的默认值。

如果变量已经定义，则不会计算表达式，因此不会发生任何副作用。

使用便利变量的一种方法是将其用作计数器进行递增，或者用作指针进行推进。例如，要打印结构体数组中连续元素的一个字段： 

set $i = 0
print bar[$i++]->contents

通过输入 回车键。

一些便利变量由 GDB 自动创建，并赋予可能有用的值。

$_ ¶

变量 $_ 由 x 命令自动设置为最后检查的地址（请参阅 检查内存）。其他为 x 提供默认检查地址的命令也会将 $_ 设置为该地址；这些命令包括 info line 和 info breakpoint。 $_ 的类型是 void * ，除非由 x 命令设置，在这种情况下，它是指向 $__类型的指针。

$__ ¶

变量 $__ 由 x 命令自动设置为在最后检查的地址中找到的值。其类型选择为与数据打印的格式相匹配。

$_exitcode ¶

当被调试的程序正常终止时， GDB 会自动将此变量设置为程序的退出代码，并将 $_exitsignal 重置为 void。

$_exitsignal ¶

当被调试的程序因未捕获的信号而终止时， GDB 会自动将此变量设置为该信号的编号，并将 $_exitcode 重置为 void。

为了区分被调试的程序是退出了（即 $_exitcode 不是 void）还是收到了信号（即 $_exitsignal 不是 void），可以使用便利函数 $_isvoid （请参阅 便利函数）。例如，考虑以下源代码： 

#include <signal.h>

int
main (int argc, char *argv[])
{
  raise (SIGALRM);
  return 0;
}

判断被调试的程序是退出还是收到信号的有效方法是： 

(gdb) define has_exited_or_signalled
Type commands for definition of ``has_exited_or_signalled''.
End with a line saying just ``end''.
>if $_isvoid ($_exitsignal)
 >echo The program has exited\n
 >else
 >echo The program has signalled\n
 >end
>end
(gdb) run
Starting program:

Program terminated with signal SIGALRM, Alarm clock.
The program no longer exists.
(gdb) has_exited_or_signalled
The program has signalled

可以看出， GDB 正确地告知被调试的程序收到了信号，因为它调用了 raise 并引发了一个 SIGALRM 信号。如果被调试的程序没有调用 raise, 那么 GDB 会报告正常退出： 

(gdb) has_exited_or_signalled
The program has exited
$_exception

变量 $_exception 被设置为在异常相关的捕获点抛出的异常对象。请参阅 设置捕获点。

$_ada_exception

变量 $_ada_exception 被设置为在 Ada 异常相关的捕获点捕获或抛出的异常的地址。请参阅 设置捕获点。

$_probe_argc
$_probe_arg0…$_probe_arg11

静态探针的参数。请参阅 静态探针点。

$_sdata ¶

变量 $_sdata 包含额外收集的静态跟踪点数据。请参阅 跟踪点动作列表。请注意， $_sdata 如果不检查跟踪缓冲区，或者如果尚未收集额外的静态跟踪点数据，则可能为空。

$_siginfo ¶

变量 $_siginfo 包含额外的信号信息（请参阅 额外的信号信息）。请注意， $_siginfo 如果应用程序尚未接收到任何信号，则可能为空。例如，在执行 run 命令之前它将为空。

$_tlb ¶

变量 $_tlb 在以原生模式调试运行在 MS - Windows 上的应用程序时，或者连接到支持 qGetTIBAddr 请求的 gdbserver 时会自动设置。请参阅 通用查询数据包。此变量包含线程信息块的地址。

$_inferior

当前 inferior 的编号。请参阅 调试多个 inferior 连接和程序。

$_thread

当前线程的线程编号。请参阅 线程编号。

$_gthread

当前线程的全局编号。请参阅 全局线程编号。

$_inferior_thread_count

当前 inferior 中活动线程的数量。请参阅 调试多线程程序。

$_gdb_major ¶
$_gdb_minor

正在运行的 GDB的主版本号和次版本号。开发快照和预测试版本的次版本号会加 1；因此， GDB 预测试版 9.11.90 会为 $_gdb_minor产生值 12。这些变量允许你编写适用于不同版本的 GDB 的脚本，而不会因某些版本中不可用的功能而导致错误。

$_shell_exitcode ¶
$_shell_exitsignal

GDB 诸如 shell 和 | 之类的命令正在启动 shell 命令。当启动的命令终止时， GDB 会根据最后启动的命令的退出状态自动维护变量 $_shell_exitcode 和 $_shell_exitsignal 。这些变量的设置和使用方式与变量 $_exitcode 和 $_exitsignal类似。

$_colorsupport ¶

终端支持的 颜色空间 名称的逗号分隔列表。名称可以是 ‘单色’、‘ANSI 8 色’、‘Aixterm 16 色’、‘Xterm 256 色’、‘RGB 24 位’。例如，对于普通的 Linux 终端，该值可能是 ‘单色,ANSI 8 色’，对于支持真彩色的终端，它可能是 ‘单色,ANSI 8 色,Aixterm 16 色,Xterm 256 色,RGB 24 位’。‘RGB 24 位’的可用性由 COLORTERM 环境变量决定，该变量可能是 ‘真彩色’ 或 ‘24 位’。其他颜色空间由 “Co” 终端能力决定，而 “Co” 又取决于 TERM 环境变量。

$_linker_namespace_count ¶

inferior 中活动链接器命名空间的数量（请参阅 指定文件的命令）。在不支持链接器命名空间的系统中，或者如果 inferior 尚未启动，此变量将始终设置为 ‘0’。

$_linker_namespace ¶

包含 inferior 中当前位置的命名空间。这返回 GDB对该命名空间的内部编号。在不支持链接器命名空间的系统中，此变量将始终设置为 ‘0’。
