5.1.8动态打印 ¶

动态打印命令 dprintf 将断点与程序数据的格式化打印相结合，让你获得即时插入 printf 调用到程序中的效果，而无需重新编译它。

set dprintf-style style

将dprintf输出设置为以下几种不同样式之一进行处理。样式的更改会立即影响所有现有的动态打印。（如果你需要对打印命令进行单独控制，只需使用显式提供的命令列表定义普通断点。）

gdb ¶

使用 GDB printf 命令处理输出。使用此样式时，可以使用‘%V’格式说明符（见 %V格式说明符）。

call ¶

通过调用程序中的函数（通常是 printf）来处理输出。使用此样式时，支持的格式说明符完全取决于所调用的函数。

大多数 GDB的格式说明符与 printf 函数支持的格式说明符一致，但是， GDB的‘%V’格式说明符扩展不被 printf支持。使用‘调用’样式的dprintf时，应注意仅使用输出函数支持的格式说明符，否则结果将是未定义的。

agent ¶

让远程调试代理（如 gdbserver）本身处理输出。此样式仅适用于支持在目标上运行命令的代理。此样式不支持‘%V’格式说明符。

set dprintf-function function

设置如果dprintf样式为 call时要调用的函数。默认情况下，其值为 printf。你可以将其设置为任何可以根据 GDB 命令求值为函数的表达式。 call command.

set dprintf-channel channel

为dprintf设置一个“通道”。如果设置为非空值， GDB 将其求值为一个表达式，并将结果作为第一个参数传递给 dprintf-function，方式与 fprintf 及类似函数相同。否则，dprintf格式字符串将作为第一个参数，方式与 printf相同。

例如，如果要将 dprintf 输出发送到分配给变量 mylog的标准I/O流日志文件中，你可以执行以下操作： 

(gdb) set dprintf-style call
(gdb) set dprintf-function fprintf
(gdb) set dprintf-channel mylog
(gdb) dprintf 25,"at line 25, glob=%d\n",glob
Dprintf 1 at 0x123456: file main.c, line 25.
(gdb) info break
1       dprintf        keep y   0x00123456 in main at main.c:25
        call (void) fprintf (mylog,"at line 25, glob=%d\n",glob)
        continue
(gdb)

请注意， info break 将动态打印命令显示为普通断点命令；因此你可以轻松看到变量设置的效果。

set disconnected-dprintf on ¶
set disconnected-dprintf off

选择如果 dprintf 命令在 GDB 与目标断开连接后是否应继续运行。这仅适用于 dprintf-style 是 agent的情况。

show disconnected-dprintf off ¶

显示当前对于断开连接的 dprintf的选择。

GDB 不会检查函数和通道的有效性，依赖于你提供在使用它们的上下文中有意义的值。例如，函数和通道可能是局部变量的值，但如果是这种情况，那么所有启用的动态打印必须位于这些局部变量的作用域内的位置。如果求值失败， GDB 将报告错误。
