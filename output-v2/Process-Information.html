21.1.2 进程信息 ¶

一些操作系统提供接口，用于获取有关正在运行的进程的额外信息，这些信息超出了内存和每个线程的寄存器状态。如果 GDB 针对具有受支持接口的操作系统进行了配置，则命令 info proc 可用于报告有关运行您的程序的进程或系统上正在运行的任何进程的信息。

一种受支持的接口是名为‘/proc’的工具，可使用文件系统子例程来检查正在运行的进程的映像。此工具在 GNU/Linux和Solaris系统上受支持。

在FreeBSD和NetBSD系统上，使用系统控制节点来查询进程信息。

此外，某些系统可能会在核心文件中提供额外的进程信息。请注意，核心文件可能包含从活动进程获取的信息的子集。目前，可从 GNU/Linux和FreeBSD系统创建的核心文件中获取进程信息。

info proc ¶
info proc process-id

总结有关进程的可用信息。如果通过 进程ID,指定了进程ID，则显示有关该进程的信息；否则，显示有关正在调试的程序的信息。总结信息包括被调试进程的ID、用于调用它的命令行、其当前工作目录以及其可执行文件的绝对文件名。

在某些系统上， 进程ID 可以采用‘[进程ID]/线程ID’的形式，它指定了进程内的某个线程ID。如果可选的 进程ID 部分缺失，则表示来自正在调试的进程的线程（前面的‘/’仍然需要存在，否则 GDB 将把该数字解释为进程ID而不是线程ID）。

info proc cmdline ¶

显示进程的原始命令行。此命令在 GNU/Linux、FreeBSD和NetBSD上受支持。

info proc cwd ¶

显示进程的当前工作目录。此命令在 GNU/Linux、FreeBSD和NetBSD上受支持。

info proc exe ¶

显示进程的可执行文件的名称。此命令在 GNU/Linux、FreeBSD和NetBSD上受支持。

info proc files ¶

显示进程打开的文件描述符。对于每个打开的文件描述符， GDB 会显示其编号、类型（文件、目录、字符设备、套接字）、文件指针偏移量以及在该描述符上打开的资源的名称。资源名称可以是文件名（对于文件、目录和设备）或协议后跟套接字地址（对于网络连接）。此命令在FreeBSD上受支持。

此示例显示了一个使用tty进行标准输入和输出以及两个网络套接字的进程的打开文件描述符：

info proc mappings ¶

报告进程中可访问的内存地址空间范围。在Solaris、FreeBSD和NetBSD系统上，每个内存范围都包含有关进程是否对每个范围具有读、写或执行访问权限的信息。在 GNU/Linux、FreeBSD和NetBSD系统上，每个内存范围都包含映射到该范围的目标文件。

info proc stat ¶
info proc status

显示与进程相关的额外信息，包括用户ID和组ID；虚拟内存使用情况；待处理、阻塞和忽略的信号；其终端；其系统和用户时间消耗；其堆栈大小；其‘优先级’值等。这些命令在 GNU/Linux、FreeBSD和NetBSD上受支持。

对于 GNU/Linux系统，请参阅‘proc’手册页以获取更多信息（在您的shell提示符下输入 man 5 proc ）。

对于FreeBSD和NetBSD系统， info proc stat 是 info proc status的别名。

info proc all

显示上述所有 info proc 子命令下描述的有关进程的所有信息。

set procfs-trace ¶

此命令启用和禁用对 procfs API调用的跟踪。

show procfs-trace ¶

显示 procfs API调用跟踪的当前状态。

set procfs-file file ¶

告诉 GDB 将 procfs API跟踪写入指定的 文件. GDB 将跟踪信息追加到文件的先前内容中。默认情况下，将跟踪显示在标准输出上。

show procfs-file ¶

显示 procfs API跟踪写入的文件。

proc-trace-entry ¶
proc-trace-exit
proc-untrace-entry
proc-untrace-exit

这些命令启用和禁用对进入和退出 syscall 接口的跟踪。
