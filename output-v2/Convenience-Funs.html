10.13便利函数 ¶

GDB 还提供了一些 便利函数。这些函数的语法类似于便利变量。便利函数可以像普通函数一样在表达式中使用；然而，便利函数是在内部实现的， GDB.

这些函数不需要 GDB 进行配置 Python 支持，这意味着它们始终可用。

$_isvoid (expr) ¶

如果表达式 expr 为 void，则返回1。否则返回零。

一个 void 表达式是一个结果类型为 void的表达式。例如，你可以检查一个便利变量（见 便利变量），以检查它是否 void: 

(gdb)打印 $_exitcode
$1 = void
(gdb)打印 $_isvoid ($_exitcode)
$2 = 1
(gdb)运行
启动程序：./a.out
[进程1（进程29572）正常退出]
(gdb)打印 $_exitcode
$3 = 0
(gdb)打印 $_isvoid ($_exitcode)
$4 = 0

在上面的示例中，我们使用 $_isvoid 来检查 $_exitcode 是否 void 在调试程序执行之前和之后。在执行之前，没有退出码可供检查，因此 $_exitcode 是 void。在执行之后，调试程序返回零，因此 $_exitcode 为零，这意味着它不是 void 了。

这个 void 表达式也可以是对正在调试的程序中的函数的调用。例如，给定以下函数： 

void
foo (void)
{
}

在 GDB 内部调用它的结果是 void: 

(gdb)打印 foo ()
$1 = void
(gdb)打印 $_isvoid (foo ())
$2 = 1
(gdb)设置 $v = foo ()
(gdb)打印 $v
$3 = void
(gdb)打印 $_isvoid ($v)
$4 = 1
$_gdb_setting_str (setting) ¶

返回 GDB 设置 的值作为字符串。 设置 是任何可以在 set 或 show 命令中使用的设置（见 控制 GDB）。 

(gdb)显示打印帧参数
打印非标量帧参数为 "标量"。
(gdb) p $_gdb_setting_str("打印帧参数")
$1 = "标量"
(gdb) p $_gdb_setting_str("高度")
$2 = "30"
(gdb)
$_gdb_setting (setting) ¶

返回 GDB 设置的值。返回值的类型取决于设置。

布尔和自动布尔设置的值类型是 int。布尔值 off 和 on 被转换为整数值 0 和 1。值 auto 被转换为值 -1.

整数设置的值类型要么是 unsigned int 要么是 int，取决于设置。

一些整数设置接受一个 unlimited 值。根据设置， set 命令也接受值 0 或值 −1 作为 unlimited的同义词。例如， set height unlimited 等同于 set height 0。

一些其他接受 unlimited 值的设置使用值 0 字面意思表示零。例如， set history size 0 表示不记录命令历史中的任何 GDB 命令。对于这样的设置， −1 是 unlimited的同义词。

请参阅相应的 set 命令的文档，以获取与 unlimited等效的数值。

函数根据 $_gdb_setting 命令使用的内容将无限制值转换为 0 或 −1 值。 set command uses. 

(gdb) p $_gdb_setting_str("高度")
$1 = "30"
(gdb) p $_gdb_setting("高度")
$2 = 30
(gdb)设置高度无限制
(gdb) p $_gdb_setting_str("高度")
$3 = "无限制"
(gdb) p $_gdb_setting("高度")
$4 = 0

(gdb) p $_gdb_setting_str("历史大小")
$5 = "无限制"
(gdb) p $_gdb_setting("历史大小")
$6 = -1
(gdb) p $_gdb_setting_str("反汇编下一行")
$7 = "自动"
(gdb) p $_gdb_setting("反汇编下一行")
$8 = -1
(gdb)

其他设置类型（枚举、文件名、可选文件名、字符串、无转义字符串）作为字符串值返回。

$_gdb_maint_setting_str (setting) ¶

与 $_gdb_setting_str 函数类似，但适用于 maintenance set 变量。

$_gdb_maint_setting (setting) ¶

与 $_gdb_setting 函数类似，但适用于 maintenance set 变量。

$_shell (command-string) ¶

调用一个 shell 来执行 命令字符串。 命令字符串 必须是一个字符串。该 shell 在运行调试器的主机机器上运行。返回命令的退出状态。在 Unix 系统上，以零退出状态退出的命令表示成功，非零退出状态表示失败。当一个命令因致命信号终止，信号编号为 GDB N 时，， GDB 使用值 128 +N 作为退出状态，这是 Unix 外壳中的标准做法。请注意， N 是主机信号编号，而不是目标信号编号。如果是本地调试，它们将是相同的，但如果是交叉调试，主机与目标信号编号可能完全无关。请查阅主机操作系统的文档，以获取主机信号编号与信号名称之间的映射。要运行的 shell 以与 shell 命令相同的方式确定。请参阅 Shell 命令。 

(gdb) print $_shell("true")
$1 = 0
(gdb) print $_shell("false")
$2 = 1
(gdb) p $_shell("echo hello")
hello
$3 = 0
(gdb) p $_shell("foobar")
bash: line 1: foobar: command not found
$4 = 127

这在断点条件中也可能有用。例如： 

(gdb) break function if $_shell("some command") == 0

在这种情况下，你需要确保在断点条件中运行的 shell 命令尽可能少地占用时间。例如，避免运行可能无限期阻塞或在退出前睡眠一段时间的命令。最好使用分析某些状态并立即退出的命令或脚本。这很重要，因为每次调试程序遇到断点时都会停止，然后 GDB 评估断点条件。如果条件为假，程序将透明地重新恢复，而不会通知你停止。因此，一个快速的 shell 命令可以避免不必要地显著减慢调试程序的速度。

注意：与 shell 命令不同， $_shell 便利函数不会影响 $_shell_exitcode 和 $_shell_exitsignal 便利变量。

以下函数需要 GDB 配置为具有 Python 支持。

$_memeq(buf1, buf2, length) ¶

如果由 长度 字节组成的位于 buf1 和 buf2 所给定地址处的内容相等，则返回 1。否则返回 0。

$_regex(str, regex) ¶

如果字符串 str 与正则表达式 regex匹配，则返回 1。否则返回 零。正则表达式的语法由 Python’s 的正则表达式支持指定。

$_streq(str1, str2) ¶

如果字符串 str1 和 str2 相等，则返回 1。否则返回 0。

$_strlen(str) ¶

返回字符串 str的长度。

$_caller_is(name[, number_of_frames]) ¶

如果调用函数的名称等于 name，则返回 1。否则返回 0。

如果提供了可选参数 帧数 ，则它是堆栈中向上查找的帧数。默认值为 1。

示例： 

(gdb) backtrace
#0  bottom_func ()
    at testsuite/gdb.python/py-caller-is.c:21
#1  0x00000000004005a0 in middle_func ()
    at testsuite/gdb.python/py-caller-is.c:27
#2  0x00000000004005ab in top_func ()
    at testsuite/gdb.python/py-caller-is.c:33
#3  0x00000000004005b6 in main ()
    at testsuite/gdb.python/py-caller-is.c:39
(gdb) print $_caller_is ("middle_func")
$1 = 1
(gdb) print $_caller_is ("top_func", 2)
$1 = 1
$_caller_matches(regexp[, number_of_frames]) ¶

如果调用函数的名称与正则表达式 regexp匹配，则返回 1。否则返回 0。

如果提供了可选参数 帧数 ，则它是堆栈中向上查找的帧数。默认值为 1。

$_any_caller_is(name[, number_of_frames]) ¶

如果任何调用函数的名称等于 name，则返回 1。否则返回 0。

如果提供了可选参数 帧数 ，则它是堆栈中向上查找的帧数。默认值为 1。

此函数与 $_caller_is 的不同之处在于，此函数检查从直接调用者到由 帧数指定的帧的所有堆栈帧，而 $_caller_is 仅检查由 帧数指定的帧。

$_any_caller_matches(regexp[, number_of_frames]) ¶

如果任何调用函数的名称与正则表达式 regexp匹配，则返回 1。否则返回 0。

如果提供了可选参数 帧数 ，则它是堆栈中向上查找的帧数。默认值为 1。

此函数与……不同 $_caller_matches 在于此函数会检查从直接调用者到由……指定的帧数所对应的所有栈帧 帧数而 $_caller_matches 仅检查由……指定的帧数所对应的帧 帧数。

$_as_string(value) ¶

此便利函数被视为已弃用，可能会在未来版本中移除。请改用‘ GDB%V’格式说明符（请参阅 %V格式说明符）。

返回……的值的字符串表示形式 值。

此函数对于获取枚举值的文本标签（枚举常量）很有用。例如，假设变量 节点 是枚举类型： 

(gdb) printf "正在访问类型为%s的节点\n", $_as_string(节点)
正在访问类型为NODE_INTEGER的节点
$_cimag(value) ¶

返回复数$_cimag（$_creal）或实数（ 值）部分。

虚部或实部的类型取决于复数的类型，例如，对……使用 $_cimag 在一个 float complex 上会返回类型为 float的虚部。
