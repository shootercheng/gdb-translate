27.8 GDB/MI ¶

本节介绍 GDB/MI 用于操作断点的命令。

-break-after 命令

概要 ¶

 -break-after 编号 计数

断点编号 编号 在被命中之前不起作用 计数 ’ 命令，请参阅 ‘-break-list’ 命令如下。

GDB 命令 ¶

对应的 GDB 命令是 ‘忽略’。

示例 ¶

(gdb)
-break-insert main
^done,bkpt={number="1",type="breakpoint",disp="keep",
enabled="y",addr="0x000100d0",func="main",file="hello.c",
fullname="/home/foo/hello.c",line="5",thread-groups=["i1"],
times="0"}
(gdb)
-break-after 1 3
~
^done
(gdb)
-break-list
^done,BreakpointTable={nr_rows="1",nr_cols="6",
hdr=[{width="3",alignment="-1",col_name="number",colhdr="Num"},
{width="14",alignment="-1",col_name="type",colhdr="Type"},
{width="4",alignment="-1",col_name="disp",colhdr="Disp"},
{width="3",alignment="-1",col_name="enabled",colhdr="Enb"},
{width="10",alignment="-1",col_name="addr",colhdr="Address"},
{width="40",alignment="2",col_name="what",colhdr="What"}],
body=[bkpt={number="1",type="breakpoint",disp="keep",enabled="y",
addr="0x000100d0",func="main",file="hello.c",fullname="/home/foo/hello.c",
line="5",thread-groups=["i1"],times="0",ignore="3"}]}
(gdb)

-break-commands 命令 ¶

概要 ¶

 -break-commands  [

指定当断点 编号 被命中时应执行的 CLI 命令。参数 命令 1 到 命令 N 是这些命令。如果未指定命令，则清除之前设置的任何命令。请参阅 断点命令列表. 此功能的典型用途是跟踪程序，即在断点被命中时打印一些变量的值，然后继续执行。

GDB 命令 ¶

对应的 GDB 命令是 ‘commands’。

示例 ¶

(gdb)
-break-insert main
^done,bkpt={number="1",type="breakpoint",disp="keep",
enabled="y",addr="0x000100d0",func="main",file="hello.c",
fullname="/home/foo/hello.c",line="5",thread-groups=["i1"],
times="0"}
(gdb)
-break-commands 1 "print v" "continue"
^done
(gdb)

该 -break-condition 命令 ¶

概要 ¶

 -break-condition [ --force ] 编号 [ 表达式 ]

断点 编号 只有在 表达式 中的条件为真时才会停止程序。该条件成为 ‘-break-list’ 输出的一部分（请参阅下面 ‘-break-list’ 命令的描述）。如果传递了 ‘--force’ 标志，即使条件对于断点 编号的所有位置都无效，也会强制定义该条件。如果省略了 表达式 参数，断点 编号 变为无条件的。

GDB 命令 ¶

对应的 GDB 命令是 ‘condition’。

示例 ¶

(gdb)
-break-condition 1 1
^done
(gdb)
-break-list
^done,BreakpointTable={nr_rows="1",nr_cols="6",
hdr=[{width="3",alignment="-1",col_name="number",colhdr="Num"},
{width="14",alignment="-1",col_name="type",colhdr="Type"},
{width="4",alignment="-1",col_name="disp",colhdr="Disp"},
{width="3",alignment="-1",col_name="enabled",colhdr="Enb"},
{width="10",alignment="-1",col_name="addr",colhdr="Address"},
{width="40",alignment="2",col_name="what",colhdr="What"}],
body=[bkpt={number="1",type="breakpoint",disp="keep",enabled="y",
addr="0x000100d0",func="main",file="hello.c",fullname="/home/foo/hello.c",
line="5",cond="1",thread-groups=["i1"],times="0",ignore="3"}]}
(gdb)

该 -break-insert 命令 ¶

概要 ¶

 -break-insert [ -t ] [ -h ] [ -f ] [ -d ] [ -a ] [ --qualified ]
    [ -c 条件 ] [ --force-condition ] [ -i 忽略计数 ]
    [ -p 线程 ID ] [ -g 线程组 ID ] [ 位置规范 ]

如果指定， 位置规范, 可以是以下之一：

行规范位置

一个行规范位置。请参阅 行规范位置.

显式位置

一个显式位置。 GDB/MI 显式位置类似于 CLI 使用以下选项名称的显式位置。请参阅 显式位置.

‘--source 文件名’

位置的源文件名。此选项需要使用 ‘--function’ 或 ‘--line’。

‘--function 函数’

函数或方法的名称。

‘--label 标签’

标签的名称。

‘--line 行偏移量’

从位置开始的绝对或相对行偏移量。

地址位置

一个地址位置，*地址. 请参阅 地址位置.

此命令可能的可选参数是：

‘-t’

插入一个临时断点。

‘-h’

插入一个硬件断点。

‘-f’

如果 位置规范 无法解析（例如，如果它引用未知文件或函数），则创建一个挂起的断点。如果没有此标志， GDB 如果 位置规范 无法解析，将报告错误，并且不会创建断点。

‘-d’

创建一个禁用的断点。

‘-a’

创建一个跟踪点。请参阅 跟踪点. 当此参数与 ‘-h’ 一起使用时，将创建一个快速跟踪点。

‘-c 条件’

使断点以 条件. 为条件。

‘--force-condition’

即使条件在所有断点位置都无效，也强制定义断点。

‘-i 忽略计数’

初始化 忽略计数.

‘-p 线程 ID’

将断点限制到具有指定全局 线程 ID. 的线程。 线程 ID 必须是请求断点时的有效线程 ID。使用 线程 ID 创建的断点将在相应线程退出时自动删除。

‘-g 线程组 ID’

将断点限制到具有指定 线程组 ID. 的线程组。

‘--qualified’

此选项使 GDB 将指定的函数名解释为完整的全限定名。

结果 ¶

请参阅 GDB/MI 断点信息, 了解所得断点的格式详细信息。

注意：此格式可能会更改。

GDB 命令 ¶

对应的 GDB 命令是 ‘break’, ‘tbreak’, ‘hbreak’, 和 ‘thbreak’。

示例 ¶

(gdb)
-break-insert main
^done,bkpt={number="1",addr="0x0001072c",file="recursive2.c",
fullname="/home/foo/recursive2.c,line="4",thread-groups=["i1"],
times="0"}
(gdb)
-break-insert -t foo
^done,bkpt={number="2",addr="0x00010774",file="recursive2.c",
fullname="/home/foo/recursive2.c,line="11",thread-groups=["i1"],
times="0"}
(gdb)
-break-list
^done,BreakpointTable={nr_rows="2",nr_cols="6",
hdr=[{width="3",alignment="-1",col_name="number",colhdr="Num"},
{width="14",alignment="-1",col_name="type",colhdr="Type"},
{width="4",alignment="-1",col_name="disp",colhdr="Disp"},
{width="3",alignment="-1",col_name="enabled",colhdr="Enb"},
{width="10",alignment="-1",col_name="addr",colhdr="Address"},
{width="40",alignment="2",col_name="what",colhdr="What"}],
body=[bkpt={number="1",type="breakpoint",disp="keep",enabled="y",
addr="0x0001072c", func="main",file="recursive2.c",
fullname="/home/foo/recursive2.c,"line="4",thread-groups=["i1"],
times="0"},
bkpt={number="2",type="breakpoint",disp="del",enabled="y",
addr="0x00010774",func="foo",file="recursive2.c",
fullname="/home/foo/recursive2.c",line="11",thread-groups=["i1"],
times="0"}]}
(gdb)

该 -dprintf-insert 命令 ¶

概要 ¶

 -dprintf-insert [ -t ] [ -f ] [ -d ] [ --qualified ]
    [ -c 条件 ] [--force-condition] [ -i 忽略计数 ]
    [ -p 线程 ID ] [ 位置规范 ] 格式
    [ 参数... ]

在给定位置插入一个新的动态打印断点。请参阅 动态打印. 格式 是要使用的格式，任何剩余的参数作为要替换的表达式传递。

如果提供， 位置规范 和 --qualified 可以与 -break-insert 命令相同的方式指定。请参阅 -break-insert.

此命令可能的可选参数是：

‘-t’

插入一个临时断点。

‘-f’

如果 位置规范 无法解析（例如，如果它引用未知文件或函数），则创建一个挂起的断点。如果没有此标志， GDB 如果 位置规范 无法解析，将报告错误，并且不会创建断点。

‘-d’

创建一个禁用的断点。

‘-c 条件’

使断点以 条件. 为条件。

‘--force-condition’

即使条件在所有断点位置都无效，也强制定义断点。

‘-i 忽略计数’

设置断点的忽略计数（请参阅 忽略计数) to 忽略计数.

‘-p 线程 ID’

将断点限制到具有指定全局 线程 ID.

结果 ¶

请参阅 GDB/MI 断点信息, 了解所得断点的格式详细信息。

GDB 命令 ¶

对应的 GDB 命令是 ‘dprintf’。

该 -break-list 命令 ¶

概要 ¶

 -break-list

显示已插入的断点列表，显示以下字段：

‘编号’

断点的编号

‘类型’

断点的类型： ‘断点’ 或 ‘观察点’

‘处置方式’

断点被命中时应删除还是禁用： ‘保留’ 或 ‘不保留’

‘启用状态’

断点是否启用： ‘是’ 或 ‘否’

‘地址’

设置断点的内存位置

‘位置信息’

断点的逻辑位置，由函数名、文件名、行号表示

‘线程组’

此断点适用的线程组列表

‘命中次数’

断点被命中的次数

如果没有断点、观察点、跟踪点或捕获点， BreakpointTable body 字段为空列表。

GDB 命令 ¶

对应的 GDB 命令是 ‘info break’。

该 -break-watch 命令 ¶

概要 ¶

 -break-watch [ -a | -r ]

创建一个观察点。使用 ‘-a’ 选项将创建一个 访问 观察点，即当对内存位置进行读或写操作时触发的观察点。使用 ‘-r’ 选项，创建的观察点是一个 读 观察点，即仅当对内存位置进行读访问时触发。如果不使用任何选项，创建的观察点是一个常规观察点，即当对内存位置进行写访问时触发。请参阅 设置观察点.

请注意， ‘-break-list’ 将报告插入的观察点和断点的单个列表。

对应的 GDB 命令是 ‘watch’, ‘awatch’, 和 ‘rwatch’。
