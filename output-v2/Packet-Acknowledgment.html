E.12 数据包确认 ¶

默认情况下，当主机或目标机器接收到一个数据包时，预期的第一个响应是确认信息：要么是 ‘+’（表示数据包已正确接收），要么是 ‘-’（请求重新传输）。这种机制允许 GDB 远程协议在不可靠的传输机制（如串行线路）上运行。

在传输机制本身可靠的情况下（如管道或 TCP 连接），‘+’/‘-’ 确认信息是多余的。在这种情况下，可能希望禁用它们以减少通信开销，或者出于其他原因。这可以通过 ‘QStartNoAckMode’ 数据包来实现；请参阅 QStartNoAckMode.

在无确认模式下，存根和 GDB 都不应发送或期望 ‘+’/‘-’ 协议确认信息。数据包和响应格式仍然包含正常的校验和，如 概述中所述，但接收方可以忽略该校验和。

如果存根支持 ‘QStartNoAckMode’ 并且倾向于在无确认模式下运行，它应该向 GDB 报告这一点，方法是在其对 ‘QStartNoAckMode+’ 的响应中包含 ‘qSupported’；请参阅 qSupported. 如果 GDB 也支持 ‘QStartNoAckMode’ 并且没有通过 set remote noack-packet off 命令禁用它（请参阅 远程配置）， GDB 则可以向存根发送一个 ‘QStartNoAckMode’ 数据包。只有这样，存根才能实际关闭数据包确认功能。 GDB 发送一个最终的 ‘+’ 确认信息，以确认存根的 ‘OK’ 响应，存根可以安全地忽略该确认信息。

请注意， set remote noack-packet 命令仅影响后续连接建立时 GDB 和存根之间的协商；它不会影响任何当前连接的协议确认状态。由于在建立新连接时默认启用 ‘+’/‘-’ 确认信息，因此一旦禁用，也没有协议请求可以为当前连接重新启用确认信息。
