 通信方式

gdbserver 是一个程序，允许您在与正在调试的程序运行的机器不同的机器上运行 GDB

用法（服务器（目标）端） ¶

首先，您需要将想要调试的程序的副本放到目标系统上。如果需要，可以去除程序的符号信息以节省空间，因为 gdbserver 不关心符号。所有符号处理都由运行在主机系统上的 GDB 处理。

要使用服务器，您需要登录到目标系统并运行 gdbserver 程序。您必须告诉它（a）如何与 GDB, （b）您的程序名，以及（c）其参数。一般语法如下： 

目标> gdbserver 通信方式 程序 [参数 ...]

例如，使用串行端口，您可以这样说： 

目标> gdbserver /dev/com1 emacs foo.txt

这告诉 gdbserver 调试带有参数foo.txt的emacs，并通过 GDB 与 /dev/com1通信。 gdbserver 现在耐心等待主机 GDB 与它通信。

要使用TCP连接，您可以这样说： 

目标> gdbserver host:2345 emacs foo.txt

这与上一个示例的意思基本相同，只是我们将通过TCP与 host GDB 通信。 host:2345 参数表示我们期望从 host 到本地TCP端口2345的TCP连接。（目前， host 部分被忽略。）只要不与目标系统上的任何现有TCP端口冲突，您可以选择任何端口号。主机 GDB的命令中必须使用相同的端口号，稍后将对此进行描述。请注意，如果您选择的端口号与其他服务冲突， target remote gdbserver 将打印错误消息并退出。

gdbserver 还可以附加到正在运行的程序。这可以通过 --attach 参数实现。语法如下： 

目标> gdbserver --attach 通信方式 进程ID

进程ID 是当前正在运行的进程的进程ID。不必指向正在运行的进程的二进制文件。 gdbserver

要启动 gdbserver 而不提供要运行的初始命令或要附加的进程ID，请使用 --multi 命令行选项。在这种情况下，您应该使用 目标扩展远程 进行连接以启动您要调试的程序。 

目标> gdbserver --multi 通信方式

用法（主机端） ¶

您的主机系统上需要有目标程序的未去除符号信息的副本，因为 GDB 需要检查其符号表等。像往常一样启动 GDB ，将目标程序作为第一个参数。（如果串行线路的波特率不是9600，您可能需要使用 --baud 选项。）也就是说 gdb TARGET-PROG, 或者 gdb --baud BAUD TARGET-PROG. 之后，您需要了解的唯一新命令是 target remote （或 target extended-remote）。其参数可以是设备名（通常是串行设备，如 /dev/ttyb），或者是 HOST:PORT 描述符。例如： 

(gdb) target remote /dev/ttyb

通过串行线路 /dev/ttyb, 并且： 

(gdb) target remote the-target:2345

通过TCP连接到主机‘the-target’上的端口2345进行通信，您之前在该主机上以相同的端口号启动了 gdbserver 。请注意，对于TCP连接，您必须在使用‘target remote’命令之前启动 gdbserver ，否则您可能会收到类似‘Connection refused’的错误。

gdbserver 还可以同时调试多个子进程，这在 调试多个子进程连接和程序中进行了描述。在这种情况下，使用 extended-remote GDB 命令变体： 

(gdb) target extended-remote the-target:2345

该 gdbserver 选项 --multi 在这种情况下可以使用，也可以不使用。

有三种不同的调用模式 gdbserver:

  • 通过程序名指定特定程序进行调试： 
    gdbserver 通信方式 程序 [参数...]

    该 通信方式 参数指定服务器应如何与 GDB通信；它可以是设备名（使用串行线路）、一个TCP端口号（:1234），或者 - 或者 stdio 使用 gdbserver的标准输入/标准输出。在 程序中指定要调试的程序名。任何剩余的参数将原封不动地传递给该程序。当程序退出时， GDB 将关闭连接，并且 gdbserver 将退出。

  • 通过指定正在运行的程序的进程ID来调试特定程序： 
    gdbserver --attach 通信方式 进程ID

    该 通信方式 参数如上所述。在 进程ID中提供正在运行的程序的进程ID； GDB 将完成其余的操作。与前一种模式一样，当进程 进程ID 退出时， GDB 将关闭连接，并且 gdbserver 将退出。

  • 多进程模式 – 调试多个程序/进程： 
    gdbserver --multi 通信方式

    在这种模式下， GDB 可以指示 gdbserver 要运行哪些命令。与其他两种模式不同， GDB 在被调试的进程退出时不会关闭连接，因此您可以在同一会话中调试多个进程。

在每种模式下，您可以指定这些选项：

--help

列出所有选项，并简要说明。

--version

此选项使 gdbserver 打印其版本号并退出。

--attach

gdbserver 将附加到正在运行的程序。语法如下： 

目标> gdbserver --attach 通信方式 进程ID

进程ID 是当前正在运行的进程的进程ID。不必指向正在运行的进程的二进制文件。 gdbserver

--multi

要启动 gdbserver 而不提供要运行的初始命令或要附加的进程ID，请使用此命令行选项。然后您可以使用 目标扩展远程 进行连接并启动您要调试的程序。语法如下： 

目标> gdbserver --multi 通信方式
--debug[=option1,option2,…]

指示 gdbserver 显示有关调试过程的额外状态信息。此选项用于 gdbserver 开发和向开发人员提交错误报告。

每个 选项 是应启用调试的组件的名称。可能的选项列表为 所有, 线程, 事件循环, 远程. 特殊选项 所有 启用所有组件。选项列表从左到右处理，并且选项可以前缀 -

gdbserver

--debug-file=filename

gdbserver gdbserver

--debug-format=option1[,option2,...]

gdbserver

--wrapper

-- 表示包装器参数的结束。

--once

默认情况下， gdbserver 保持监听的TCP端口打开，以便可以进行额外的连接。但是，如果您使用 gdbserver --once 选项启动，它将在连接到第一个 GDB 会话后停止监听任何进一步的连接尝试。
