15.4.1.1 C和C++++ 运算符 ¶

运算符必须定义在特定类型的值上。例如， + 定义于数字，但不定义于结构体。运算符通常定义于一组类型。

出于C和C++的目的++, 以下定义成立：

  • 整数类型 包括 int 带有任何存储类说明符的； char; enum; 并且，对于C++, bool。
  • 浮点类型 包括 float, double, 和 long double （如果目标平台支持）。
  • 指针类型 包括所有定义为 (type *)的类型。
  • 标量类型 包括上述所有类型。

支持以下运算符。它们按优先级递增的顺序列出：

,

逗号或顺序运算符。逗号分隔列表中的表达式从左到右求值，整个表达式的结果是最后求值的表达式。

=

赋值。赋值表达式的值是被赋的值。定义于标量类型。

op=

用于形式为 a op= b, 的表达式，并转换为 a = a op b。 op= 和 = 具有相同优先级。运算符 op 是以下运算符之一 |, ^, &, <<, >>, +, -, *, /, %。

?:

三元运算符。 a ? b : c 可以认为是：如果 a 则 b 否则 c。参数 a 应该是整数类型。

||

逻辑 OR。定义于整数类型。

&&

逻辑 AND。定义于整数类型。

|

按位 OR。定义于整数类型。

^

按位异或OR。定义于整数类型。

&

按位 AND。定义于整数类型。

==, !=

相等和不相等。定义于标量类型。这些表达式的值，假为0，真为非零。

<, >, <=, >=

小于、大于、小于等于、大于等于。定义于标量类型。这些表达式的值，假为0，真为非零。

<<, >>

左移和右移。定义于整数类型。

@

GDB “伪数组”运算符（见 表达式）。

+, -

加法和减法。定义于整数类型、浮点类型和指针类型。

*, /, %

乘法、除法和取模。乘法和除法定义于整数和浮点类型。取模定义于整数类型。

++, --

自增和自减。当出现在变量之前时，操作在变量用于表达式之前执行；当出现在变量之后时，变量的值在操作执行之前使用。

*

指针解引用。定义于指针类型。与 ++具有相同优先级。

&

取地址运算符。定义于变量。与 ++具有相同优先级。

为了调试C++++, GDB 实现了一种超出C++语言本身允许范围的‘&’的用法：你可以使用‘++ &(&引用)’来检查一个C++++ 引用变量（用‘&引用）被存储。

-

负号。定义于整数和浮点类型。与 ++具有相同优先级。

!

逻辑非。定义于整数类型。与 ++具有相同优先级。

~

按位取反运算符。定义于整数类型。与 ++具有相同优先级。

., ->

结构体成员，以及指向结构体成员的指针。为方便起见， GDB 将两者视为等效，根据存储的类型信息决定是否解引用指针。定义于 struct 和 union 数据。

.*, ->*

成员指针的解引用。

[]

数组索引。 a[i] 定义为 *(a+i)。与 ->具有相同优先级。

()

函数参数列表。与 ->具有相同优先级。

::

C++++ 作用域解析运算符。定义于 struct, union, 和 class 类型。

::

双冒号也表示 GDB 作用域运算符 （见 表达式）。与 ::（上文）具有相同优先级。

如果某个运算符在用户代码中被重新定义， GDB 通常会尝试调用重新定义的版本，而不是使用运算符的预定义含义。
