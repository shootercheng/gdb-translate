gdb-add-index ¶

gdb-add-index [-h | –帮助] [-v | –版本] [–dwarf-5] 文件名

当找到一个符号文件时，它会按顺序扫描文件中的符号，以构建一个内部符号表。这使得大多数 GDB 操作能够快速进行——代价是在一开始会有延迟。对于大型程序，这种延迟可能会相当长，因此 GDB 提供了一种构建索引的方法，这可以加快启动速度。 GDB 要确定一个文件是否包含这样的索引，请使用命令

readelf -S 文件名 : 索引存储在一个名为（DWARF 5 之前）或 .gdb_index 以及 .debug_names （DWARF 5）的节中。索引只能在使用 ELF 二进制文件和 DWARF 调试信息（即名为 .debug_str 的节）的系统上生成。 .debug_*)。

默认情况下 gdb-add-index 会向 .gdb_index 文件名 添加一个 DWARF 5 之前的节。使用 --dwarf-5 则会添加 DWARF 5 节 .debug_names 。

文件名 必须是可写的。

gdb-add-index 使用 GDB, objcopy, 以及 readelf 在 PATH 环境变量中找到的程序。如果您想使用这些程序的不同版本，可以通过相应的环境变量来指定（见下文）。

gdb-add-index 如果成功为 文件名 创建索引，则以状态 0 退出；如果发生错误，则以大于 0 的状态退出。

更多信息请参阅 索引文件可加快速度 GDB.

--dwarf-5

添加 DWARF 5 节而不是之前的 .gdb_index 节。

--help
-h

列出所有选项并简要说明。

--version
-v

打印版本信息然后退出。

GDB

用于生成索引的 gdb 程序的完整文件名。如果未设置，则会在 PATH 中搜索一个 gdb 程序。

OBJCOPY

用于将节信息复制到给定文件的 objcopy 程序的完整文件名。如果未设置，则会在 PATH 中搜索一个 objcopy 程序。

READELF

用于检查给定文件属性的 readelf 程序的完整文件名。如果未设置，则会在 PATH 中搜索一个 readelf 程序。
