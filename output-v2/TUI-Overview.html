25.1 TUI 概述 ¶

在 TUI 模式下， GDB 可以显示多个文本窗口：

命令

此窗口是 GDB 带有 GDB 提示符和 GDB 输出的命令窗口。 GDB 输入仍使用 readline 进行管理。

源代码

源代码窗口显示程序的源文件。当前行和活动断点会在该窗口中显示。

汇编代码

汇编代码窗口显示程序的反汇编输出。

寄存器

此窗口显示处理器寄存器。当寄存器的值发生变化时，它们会被高亮显示。

源代码和汇编代码窗口通过高亮显示当前行并用一个 ‘>’ 标记来显示当前程序位置。默认情况下，高亮显示的文本禁用了源代码和汇编代码样式，但您可以使用 set style tui-current-position on 命令启用它。请参阅 输出样式设置.

断点用两个标记表示。第一个标记表示断点类型：

B

至少命中过一次的断点。

b

从未命中过的断点。

H

至少命中过一次的硬件断点。

h

从未命中过的硬件断点。

第二个标记表示断点是否启用：

+

断点已启用。

-

断点已禁用。

当当前线程改变、帧改变或程序计数器改变时，源代码、汇编代码和寄存器窗口会更新。

这些窗口并非同时可见。命令窗口始终可见。其他窗口可以排列成几种布局：

  • 仅显示源代码
  • 仅显示汇编代码
  • 显示源代码和汇编代码
  • 显示源代码和寄存器
  • 显示汇编代码和寄存器

这些是标准布局，但也可以定义其他布局。

命令窗口上方的状态行显示以下信息：

目标

指示当前的 GDB 目标。（请参阅 指定调试目标)

进程

给出当前进程或线程编号。当没有进程正在调试时，此字段设置为 No process.

焦点

显示具有焦点的 TUI 窗口的名称。

函数

给出所选帧的当前函数名称。如果开启了名称修饰还原（请参阅 打印设置)，则名称会被还原。当没有与当前程序计数器对应的符号时，会显示字符串 ?? 。

行号

指示所选帧的当前行号。当不知道当前行号时，会显示字符串 ?? 。

程序计数器

指示当前程序计数器地址。
