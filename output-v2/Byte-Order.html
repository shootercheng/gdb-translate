19.3 选择目标字节顺序 ¶

某些类型的处理器，例如 MIPS, PowerPC和瑞萨SH， 提供了运行大端字节序或小端字节序的能力。通常，可执行文件或符号会包含一个位来 指定字节序，你无需担心使用哪种字节序。但是，你可能仍然会发现手动调整 GDB对处理器字节序的理解很有用。

set endian big ¶

指示 GDB 假定目标是大端字节序。

set endian little

指示 GDB 假定目标是小端字节序。

set endian auto

指示 GDB 使用与可执行文件关联的字节序。

show endian

显示 GDB当前对目标字节序的理解。

如果 set endian auto 模式有效且未选择任何可执行文件，则使用的字节序是通过 set endian big 和 set endian little 命令之一选择的最后一个字节序，或者是从最后使用的可执行文件推断出来的。如果之前未选择任何字节序，则此模式的默认值是从目标 GDB 所构建的环境推断出来的，并且如果目标CPU的名称有 little 后缀则为 el ，否则为 big 。

请注意，这些命令仅调整主机上符号数据的解释，它们对目标系统绝对没有影响。
