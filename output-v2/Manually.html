15.1.2 设置工作语言 ¶

如果允许 GDB 自动设置语言，那么在调试会话和程序中，表达式的解释方式是相同的。

如果愿意，您可以手动设置语言。为此，请发出命令 ‘设置语言 语言’，其中 语言 是一种语言的名称，例如 c 或 modula-2. 要获取支持的语言列表，请输入 ‘设置语言’。

手动设置语言会阻止 GDB 自动更新工作语言。如果在工作语言与源语言不同的情况下尝试调试程序，当一个表达式在两种语言中都可接受但含义不同时，这可能会导致混淆。例如，如果当前源文件是用C语言编写的，而 GDB 正在解析Modula - 2，像这样的命令： 

打印 a = b + c

可能不会产生您预期的效果。在C语言中，这意味着将 b 和 c 相加，并将结果存储在 a中。打印的结果将是 a的值。在Modula - 2中，这意味着将 a 与 b+c的结果进行比较，得到一个 BOOLEAN 值。
