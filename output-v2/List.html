9.1 打印源代码行 ¶

要从源文件中打印行，请使用 list 命令 （缩写为 l）。默认情况下，会打印十行。 有几种方法可以指定你想要打印文件的哪一部分；请参阅 位置规范，以获取完整列表。

以下是最常用的 list 命令形式：

list linenum

打印以行号为中心的行 行号 在 当前源文件中。

list function

打印以函数开头为中心的行 函数。

list

打印更多行。如果最后打印的行是使用 list 命令打印的，这将打印最后打印行之后的行；但是，如果最后打印的行是作为显示栈帧的一部分单独打印的行（请参阅 检查栈），这将打印以该行中心的行。如果没有使用过 list 命令且没有单独打印过行， 它将打印围绕函数的行 main。

list +

与不使用参数使用该命令相同。

list -

打印最后打印行之前的行。

list .

打印当前选定帧内执行点周围的行。如果程序未运行，则改为打印主函数开始处周围的行。

默认情况下， GDB 使用这些形式的 list 命令时会打印十行源代码。你可以使用 set listsize来更改此设置：

set listsize count ¶
set listsize unlimited

使 list 命令显示 数量 行源代码（除非 list 参数明确指定了其他数量）。 将 数量 设置为 unlimited 或0表示没有限制。

show listsize ¶

显示 list 打印的行数。

使用 list 命令并按 回车键 会丢弃参数， 因此它等同于只输入 list。这比再次列出相同的行更有用。对于参数 ‘-’ 有一个例外；该参数在重复操作时会保留，以便每次重复都会在源文件中向上移动。

一般来说， list 命令期望你提供零个、一个或两个位置规范。这些位置规范会被解释为解析为源代码行；有几种编写它们的方法 （请参阅 位置规范），但效果总是解析为要显示的某些源代码行。

以下是 list可能参数的完整描述：

list locspec

打印围绕解析 位置规范所得到的所有代码位置的行或行的中心的行。

list first,last

打印从 第一行 到 最后一行的行。两个参数都是 位置规范。当一个 list 命令有两个位置规范时， 并且第二个位置规范的源文件被省略，这指的是与第一个位置规范相同的源文件。如果 第一行 或 最后一行 解析为程序中的多行源代码，那么列表命令将显示解析后的源代码行列表，并且不会继续进行源代码列表操作。

list ,last

打印以 最后一行结尾的行。

同样，如果 最后一行 解析为程序中的多行源代码，那么列表命令将打印解析后的源代码行列表，并且不会继续进行源代码列表操作。

list first,

打印从 第一行开始的行。

list +

打印最后打印行之后的行。

list -

打印最后打印行之前的行。

list

如前面的表格所述。
