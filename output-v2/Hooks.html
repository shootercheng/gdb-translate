23.1.2 用户定义的命令钩子 ¶

您可以定义 钩子, 它们是一种特殊的用户定义命令。 每当您运行命令 ‘foo’ 时，如果用户定义的命令 ‘hook-foo’ 存在，它会在该命令执行之前（无参数）执行。

还可以定义一个在您执行的命令之后运行的钩子。 每当您运行命令 ‘foo’ 时，如果用户定义的命令 ‘hookpost-foo’ 存在，它会在该命令执行之后（无参数）执行。 对于同一个命令，执行后钩子可以与执行前钩子同时存在。

钩子调用它所挂钩的命令是有效的。 如果发生这种情况，钩子不会重新执行，从而避免无限递归。

此外，存在一个伪命令 ‘stop’。 定义（‘hook-stop’）会使相关命令在您的程序每次停止执行时执行：在断点命令运行、显示信息打印或栈帧打印之前。

例如，要在单步执行时忽略 SIGALRM 信号，但在正常执行期间正常处理它们，您可以定义： 

define hook-stop
handle SIGALRM nopass
end

define hook-run
handle SIGALRM pass
end

define hook-continue
handle SIGALRM pass
end

再举一个例子，要在 echo 命令的开始和结束处设置钩子，并在消息的开始和结束处添加额外的文本，您可以定义： 

define hook-echo
echo <<<---
end

define hookpost-echo
echo --->>>\n
end

(gdb) echo Hello World
<<<---Hello World--->>>
(gdb)

您可以为 GDB 中的任何单字命令定义钩子，但不能为命令别名定义； 您应该为基本命令名称定义钩子，例如 backtrace 而不是 bt. 您可以通过在命令的最后一个单词后面添加 hook- 或 hookpost- 来为多字命令设置钩子，例如 ‘define target hook-remote’ 为 ‘target remote’ 添加一个钩子。

如果在您的钩子执行期间发生错误， GDB 命令的执行将停止，并且 GDB 会发出一个提示（在您实际输入的命令有机会运行之前）。

如果您尝试定义一个与任何已知命令都不匹配的钩子，您会从 define 命令中收到一个警告。
