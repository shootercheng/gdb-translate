10.18 内存区域属性 ¶

内存区域属性 允许你描述目标内存区域所需的特殊处理。 GDB 使用属性来确定是否允许某些类型的内存访问；是否使用特定宽度的访问；以及是否缓存目标内存。默认情况下，内存区域的描述是从目标获取的（如果当前目标支持此功能），但用户可以覆盖获取的区域。

已定义的内存区域可以单独启用和禁用。当一个内存区域被禁用时， GDB 在访问该区域的内存时使用默认属性。同样，如果没有定义任何内存区域， GDB 在访问所有内存时使用默认属性。

当定义一个内存区域时，会给它一个编号来标识它；要启用、禁用或移除一个内存区域，你需要指定该编号。

mem lower upper attributes… ¶

定义一个由 下限 和 上限 界定的内存区域，其属性为 属性…，并将其添加到由 GDB监控的区域列表中。请注意， 上限 == 0是一种特殊情况：它被视为目标的最大内存地址。（16位目标上为0xffff，32位目标上为0xffffffff等）

mem auto

丢弃用户对内存区域所做的任何更改，并使用目标提供的区域（如果可用），如果目标不支持，则不使用任何区域。

delete mem nums… ¶

从由 编号监控的区域列表中移除内存区域 GDB。

disable mem nums… ¶

禁用对内存区域的监控 编号…。被禁用的内存区域不会被遗忘。稍后可以再次启用它。

enable mem nums… ¶

启用对内存区域的监控 编号…。

info mem ¶

打印所有已定义内存区域的表格，每个区域有以下列：

内存区域编号
启用或禁用状态。

启用的内存区域标记为‘y’。禁用的内存区域标记为‘n’。

低地址

定义内存区域包含下限的地址。

高地址

定义内存区域不包含上限的地址。

属性

为此内存区域设置的属性列表。

10.18.1.1 内存访问模式 ¶

访问模式属性设置 GDB 是否可以对内存区域进行读或写访问。

虽然这些属性可以防止 GDB 执行无效的内存访问，但它们无法防止目标系统、I/O DMA等访问内存。

ro

内存为只读。

wo

内存为只写。

rw

内存为读写。这是默认设置。

10.18.1.2 内存访问大小 ¶

访问大小属性告诉 GDB 在内存区域中使用特定大小的访问。通常，内存映射设备寄存器需要特定大小的访问。如果未指定访问大小属性， GDB 可以使用任何大小的访问。

8

使用8位内存访问。

16

使用16位内存访问。

32

使用32位内存访问。

64

使用64位内存访问。

10.18.1.3 数据缓存 ¶

数据缓存属性设置 GDB 是否会缓存目标内存。虽然这通常通过减少调试协议开销来提高性能，但它可能会导致错误的结果，因为 GDB 不知道易变变量或内存映射设备寄存器的情况。

cache

启用 GDB 缓存目标内存。

nocache

禁用 GDB 缓存目标内存。这是默认设置。

10.18.2 内存访问检查 ¶

GDB 可以被指示拒绝访问未明确描述的内存。如果访问此类区域对特定目标有不良影响，或者为了提供更好的错误检查，这可能会很有用。以下命令控制此行为。

set mem inaccessible-by-default [on|off] ¶

如果 on 被指定时，使 GDB 将内存范围未明确描述的内存视为不存在，并拒绝访问此类内存。只有在至少定义了一个内存范围时才会执行这些检查。如果 off 被指定时，使 GDB 将内存范围未明确描述的内存视为RAM。默认值是 on。

show mem inaccessible-by-default

显示当前对未知内存访问的处理方式。
