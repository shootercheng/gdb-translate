23.5.2 该 .debug_gdb_scripts 节 ¶

对于使用 ELF 和 COFF 等文件格式的系统，当 GDB 加载一个新的目标文件时，它会查找一个名为 .debug_gdb_scripts的特殊节。如果这个节存在，其内容是一个以空字符结尾的条目列表，指定要加载的脚本。每个条目以一个非空的前缀字节开始，该字节指定条目的类型，通常是扩展语言以及脚本是在文件中还是内联在 .debug_gdb_scripts中。

支持以下条目：

SECTION_SCRIPT_ID_PYTHON_FILE = 1
SECTION_SCRIPT_ID_SCHEME_FILE = 3
SECTION_SCRIPT_ID_PYTHON_TEXT = 4
SECTION_SCRIPT_ID_SCHEME_TEXT = 6
  • 脚本文件条目
  • 脚本文本条目

23.5.2.1 脚本文件条目 ¶

如果条目指定了一个文件， GDB 将首先在当前目录中查找该文件，然后沿着源搜索路径查找（请参阅 指定源目录），除了 $cdir 不会被搜索，因为编译目录与脚本无关。

文件条目可以放在节 .debug_gdb_scripts 中，例如，用于 Python 脚本的这个 GCC 宏。 

/* 注意："MS" 节标志用于去除重复项。 */
#define DEFINE_GDB_PY_SCRIPT(script_name) \
  asm("\
.pushsection \".debug_gdb_scripts\", \"MS\",@progbits,1\n\
.byte 1 /* Python */\n\
.asciz \"" script_name "\"\n\
.popsection \n\
");

对于 Guile 脚本，将 .byte 1 替换为 .byte 3。然后可以在头文件或源文件中像这样引用该宏： 

DEFINE_GDB_PY_SCRIPT ("my-app-scripts.py")

如果需要，脚本名称可以包含目录。

请注意，加载此脚本文件还需要相应配置 auto-load safe-path （请参阅 自动加载的安全限制）。

如果宏调用放在头文件中，任何使用此头文件的应用程序或库都将获得对指定脚本的引用，并且通过使用 "MS" 节上的属性，链接器将去除重复项。

23.5.2.2 脚本文本条目 ¶

脚本文本条目允许将可执行脚本直接放在条目中，而不是从文件中加载。条目的第一行，即前缀字节之后到第一个换行符（0xa）字符之间的所有内容，是脚本名称，并且不能包含任何类型的空格字符，例如空格或制表符。条目的其余部分，直到尾随的空字节，是要以指定语言执行的脚本。名称在所有脚本名称中必须是唯一的，因为 GDB 根据脚本名称只执行每个脚本一次。

以下是来自文件 py-section-script.c 在 GDB 测试套件中的一个示例。 

#include "symcat.h"
#include "gdb/section-scripts.h"
asm(
".pushsection \".debug_gdb_scripts\", \"MS\",@progbits,1\n"
".byte " XSTRING (SECTION_SCRIPT_ID_PYTHON_TEXT) "\n"
".ascii \"gdb.inlined-script\\n\"\n"
".ascii \"class test_cmd (gdb.Command):\\n\"\n"
".ascii \"  def __init__ (self):\\n\"\n"
".ascii \"    super (test_cmd, self).__init__ ("
    "\\\"test-cmd\\\", gdb.COMMAND_OBSCURE)\\n\"\n"
".ascii \"  def invoke (self, arg, from_tty):\\n\"\n"
".ascii \"    print (\\\"test-cmd output, arg = %s\\\" % arg)\\n\"\n"
".ascii \"test_cmd ()\\n\"\n"
".byte 0\n"
".popsection\n"
);

加载内联脚本需要正确配置 auto-load safe-path （请参阅 自动加载的安全限制）。在 auto-load safe-path 中指定的路径是包含 .debug_gdb_scripts 节的文件的路径。
