23.3.2.22 GDB/MI Python 中的命令 ¶

可以添加 GDB/MI (参见 GDB/MI 接口) 在 Python 中实现的命令。一个 GDB/MI 命令使用 gdb.MICommand 类的实例来实现，最常见的是使用子类。

函数： MICommand.__init__ (名称) ¶

用于 MICommand 的对象初始化器会在 GDB中注册新命令。这个初始化器通常在子类自己的 __init__ 方法中调用。

名称 是命令的名称。它必须是一个有效的 GDB/MI 命令名称，特别是必须以连字符开头 (-)。不允许重用内置 GDB/MI 的名称，否则会引发一个 RuntimeError 异常。允许使用之前在 Python 中定义的 GDB/MI 命令的名称，之前的命令将被新命令替换。

函数： MICommand.invoke (参数) ¶

当调用新的 MI 命令时，此方法由 GDB 调用。

参数 是一个字符串列表。请注意， --thread 和 --frame 参数由 GDB 本身处理，因此它们不会出现在 arguments中。

如果此方法引发异常，则会将其转换为一个 GDB/MI ^error 响应。只有 gdb.GdbError 异常（或其子类）应用于向用户报告错误，任何其他异常类型都被视为 invoke 方法的失败，并且异常将根据 set python print-stack 设置（参见 set python print-stack) 打印到错误流中。

如果此方法返回 None, 则 GDB/MI 命令将返回一个 ^done 没有额外值的响应。

否则，返回值必须是一个字典，它将被转换为一个 GDB/MI 结果记录 (参见 GDB/MI 输出语法)。 这个字典的键必须是字符串，并用作 变量 名称，这些字符串必须符合下面详细说明的命名规则。这个字典的值将按以下方式递归处理：

  • 如果值是 Python 序列或迭代器，它将被转换为 GDB/MI 列表 ，元素将递归转换。
  • 如果值是 Python 字典，它将被转换为 GDB/MI 元组. 该字典的键必须是字符串， 这些字符串必须符合下面详细说明的 变量 命名规则。值将递归转换。
  • 否则，值首先使用 str () 转换为 Python 字符串，然后转换为 GDB/MI 常量.

用于 变量 名称的字符串在 GDB/MI 输出中必须遵循以下规则；字符串必须至少有一个字符长，第一个字符必须在集合 [a-zA-Z]中，而后续的每个字符必须在集合 [-_a-zA-Z0-9]中。

一个 MICommand 的实例具有以下属性：

变量： MICommand.name ¶

一个字符串，此 GDB/MI 命令的名称，与传递给 __init__ 方法的名称相同。此属性是只读的。

变量： MICommand.installed ¶

一个布尔值，表示此命令是否已安装，可供用户从命令行调用。命令在实例化时会自动安装，之后此属性将变为 True.

如果之后创建了一个同名的新命令，那么 原始命令将被卸载，并且此属性将变为 False.

此属性是可读写的，将此属性设置为 False 将卸载该命令，将其从可用 命令集中移除。将此属性设置为 True 将安装该 命令以供使用。如果已经安装了一个同名的 Python 命令，当前安装的命令将被卸载， 并安装此命令取而代之。

以下代码片段展示了如何在 Python 中实现一些简单的 MI 命令： 

class MIEcho(gdb.MICommand):
    """回显传递给命令的参数。"""

    def __init__(self, name, mode):
        self._mode = mode
        super(MIEcho, self).__init__(name)

    def invoke(self, argv):
        if self._mode == 'dict':
            return { 'dict': { 'argv' : argv } }
        elif self._mode == 'list':
            return { 'list': argv }
        else:
            return { 'string': ", ".join(argv) }


MIEcho("-echo-dict", "dict")
MIEcho("-echo-list", "list")
MIEcho("-echo-string", "string")

最后三行代码三次实例化该类，创建了三个 新的 GDB/MI 命令 -echo-dict, -echo-list, 以及 -echo-string. 每次实例化 gdb.MICommand 的子类时，新命令会自动在 GDB中注册。

根据 Python 代码的读取方式，你可能 需要显式导入 GDB模块。 gdb 以下示例展示了一个

会话，其中添加了上述 命令： GDB 

(gdb)
-echo-dict abc def ghi
^done,dict={argv=["abc","def","ghi"]}
(gdb)
-echo-list abc def ghi
^done,list=["abc","def","ghi"]
(gdb)
-echo-string abc def ghi
^done,string="abc, def, ghi"
(gdb)

相反，也可以从 Python 执行 GDB/MI 命令，结果是一个 Python 对象，而不是 特殊格式的字符串。这可以通过 gdb.execute_mi 函数来完成。

函数： gdb.execute_mi (命令 [, 参数 ]…) ¶

调用一个 GDB/MI 命令。 命令 是命令的名称，是一个字符串。参数 参数, 会传递给 命令。每个参数也必须是一个字符串。

此函数返回一个 Python 字典，其内容反映了相应 命令的输出。有关详细信息，请参考这些命令的文档。列表表示为 Python 列表，元组表示为 Python 字典。 GDB/MI

如果命令执行失败，将引发一个 Python 异常。

以下是使用上述示例中的命令的工作方式： 

(gdb) python print(gdb.execute_mi("-echo-dict", "abc", "def", "ghi"))
{'dict': {'argv': ['abc', 'def', 'ghi']}}
(gdb) python print(gdb.execute_mi("-echo-list", "abc", "def", "ghi"))
{'list': ['abc', 'def', 'ghi']}
(gdb) python print(gdb.execute_mi("-echo-string", "abc", "def", "ghi"))
{'string': 'abc, def, ghi'}
