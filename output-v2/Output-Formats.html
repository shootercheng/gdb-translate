10.5 输出格式 ¶

默认情况下， GDB 会根据值的数据类型打印该值。但有时这并非你想要的结果。例如，你可能想以十六进制形式打印一个数字，或以十进制形式打印一个指针。或者你可能想将某个地址处的内存数据视为字符串或指令。要实现这些功能，在打印值时指定一个 输出格式 。

输出格式最简单的用法是说明如何打印已经计算好的值。这可以通过在 print 命令的参数前加上一个斜杠和一个格式字母来完成。支持的格式字母如下：

x

以十六进制形式打印值的二进制表示。

d

以十进制形式打印值的二进制表示。

u

将值的二进制表示作为无符号十进制数打印。

o

以八进制形式打印值的二进制表示。

t

以二进制形式打印值的二进制表示。字母‘t’代表“二”。 11

a ¶

将其作为地址打印，既以十六进制形式打印绝对地址，也打印相对于最近前一个符号的偏移量。你可以使用此格式来确定一个未知地址位于何处（在哪个函数中）： 

(gdb) p/a 0x54320
$3 = 0x54320 <_initialize_vx+396>

该命令 info symbol 0x54320 会产生类似的结果。请参阅 info symbol。

c

将值转换为整数（与其他格式不同，这不仅仅是重新解释底层位），并将其作为字符常量打印。这会同时打印数值和其字符表示。对于7位\nnn’。 ASCII 范围之外的字符，字符表示会被替换为八进制转义字符‘

如果不使用此格式， GDB 显示 char, unsigned char, 和 signed char 数据作为字符常量。向量的单字节成员会显示为整数数据。

f

将值的位视为浮点数，并使用典型的浮点语法进行打印。

s ¶

如果可能的话，将其视为字符串。使用此格式时，单字节数据的指针会显示为以空字符结尾的字符串，单字节数据的数组会显示为固定长度的字符串。其他值会以其自然类型显示。

如果不使用此格式， GDB 显示指针和数组的 char, unsigned char, 和 signed char 作为字符串。向量的单字节成员会显示为整数数组。

z

与‘x’格式类似，值会被视为整数并以十六进制形式打印，但会打印前导零以将值填充到整数类型的大小。

r ¶

使用‘raw’格式进行打印。默认情况下， GDB 会使用基于Python的漂亮打印机（如果有的话）（请参阅 漂亮打印）。这通常会以更高级的方式显示值的内容。‘r’格式会绕过任何可能存在的Python漂亮打印机。

例如，要以十六进制形式打印程序计数器（请参阅 寄存器），输入 

p/x $pc

请注意，斜杠前不需要空格；这是因为 GDB 中的命令名不能包含斜杠。

要以不同的格式重新打印值历史记录中的最后一个值，你可以使用 print 命令，只指定格式而不指定表达式。例如，‘p/x’会以十六进制形式重新打印最后一个值。

脚注

(11)

‘b’不能使用，因为这些格式字母也用于 x 命令，其中‘b’代表“字节”；请参阅 检查内存。
