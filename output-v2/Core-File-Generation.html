10.20 如何从你的程序生成核心文件 ¶

一个 核心文件 或 核心转储 是一个记录正在运行的进程的内存映像及其进程状态（寄存器值等）的文件。 它的主要用途是对在调试器外部运行时崩溃的程序进行事后调试。 崩溃的程序会自动生成一个核心文件，除非用户禁用了此功能。 请参阅 指定文件的命令, 有关调用的信息 GDB 处于事后调试模式。

偶尔，你可能希望生成正在调试的程序的核心文件，以保存其状态的快照。 GDB 为此有一个特殊的命令。

generate-core-file [file] ¶
gcore [file]

生成下级进程的核心转储。 可选参数 文件 指定存放核心转储的文件名。 如果未指定，文件名默认为 core。进程ID, 其中 进程ID 是下级进程的ID。

如果写入核心文件的文件系统支持， GDB 会生成一个稀疏的核心转储文件。

请注意，此命令仅在某些系统上实现（截至撰写本文时， GNU/Linux、FreeBSD、Solaris和S390）。

在 GNU/Linux上，此命令可以考虑文件 /proc/进程ID/coredump_filter 在生成核心转储时的值（请参阅 设置使用核心转储过滤器），并且默认情况下会遵循 VM_DONTDUMP 标志，对于文件中存在该标志的映射 /proc/进程ID/smaps （请参阅 设置转储排除的映射)。

set use-coredump-filter on
set use-coredump-filter off

启用或禁用在生成核心转储文件时使用文件 /proc/进程ID/coredump_filter 。 此文件由Linux内核用于决定在生成核心转储文件时将转储或忽略哪些类型的内存映射。 进程ID 是当前正在运行的进程的ID。

要使用此功能，你必须在 /proc/进程ID/coredump_filter 文件中写入一个十六进制值，该值是一个表示内存映射类型的位掩码。 如果位掩码中的某一位被设置，则相应类型的内存映射将被转储；否则，它们将被忽略。 此配置会被子进程继承。 有关可以在 /proc/进程ID/coredump_filter 文件中设置的位的更多信息，请参阅 core(5)的手册页。

默认情况下，此选项是 on. 如果此选项被打开 off, GDB 不会读取 核心转储过滤器 文件，而是使用与Linux内核相同的默认值来决定哪些页面将被转储到核心转储文件中。 此值当前是 0x33, 这意味着位 0 （匿名私有映射）、 1 （匿名共享映射）、 4 （ELF头）和 5 （私有大页）是激活的。 这将导致这些内存映射自动被转储。

set dump-excluded-mappings on
set dump-excluded-mappings off

如果 on 被指定 GDB 将转储标记有 VM_DONTDUMP 标志的内存映射。 此标志在文件 /proc/进程ID/smaps 中用首字母缩写 dd表示。

默认值是 off.
