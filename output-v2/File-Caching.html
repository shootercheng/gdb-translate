18.2 文件缓存 ¶

为了加快文件加载速度并减少内存使用， GDB 将 重用 bfd 用于跟踪打开文件的对象。请参阅 BFD 在 二进制文件描述符库。以下命令 允许查看和控制缓存行为。

maint info bfds ¶

这会打印关于每个 bfd 已知的对象的信息 GDB。

maint set bfd-sharing ¶
maint show bfd-sharing

控制是否 bfd 对象可以共享。当共享功能启用时 GDB 会重用已经打开的 bfd 对象，而不是重新打开同一个文件。关闭共享功能不会导致已经共享的 bfd 对象取消共享，但所有未来打开的文件 将创建一个新的 bfd 对象。同样， 重新启用共享功能不会导致多个现有的 bfd 对象合并为一个共享的 bfd 对象。

set debug bfd-cache level ¶

开启 bfd 缓存的调试功能，将级别设置为 级别。

show debug bfd-cache ¶

显示 bfd 缓存的当前调试级别。
