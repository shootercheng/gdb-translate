23.3.2.9 类型打印 API ¶

GDB 为 Python 代码提供了一种自定义类型显示的方法。 这主要用于用规范的 typedef 名称替换类型。

一个 类型打印机 只是一个符合特定协议的 Python 对象。 提供了一个实现该协议的简单基类； 请参阅 gdb.types。 一个类型打印机必须至少提供：

的实例变量 type_printer: 已启用 ¶

一个布尔值，如果打印机已启用则为 True，否则为 False。 这由 enable type-printer 和 disable type-printer 命令进行操作。

的实例变量 type_printer: 名称 ¶

类型打印机的名称。 这必须是一个字符串。 这由 enable type-printer 和 disable type-printer 命令使用。

上的方法 type_printer: 实例化 (self) ¶

这由 GDB 在类型打印开始时调用。 只有在类型打印机已启用时才会调用。 此方法必须返回一个 新对象，该对象提供一个 recognize 方法，如下所述。

当显示一个类型时，例如通过 ptype 命令， GDB 将计算一个类型识别器列表。 这是通过首先遍历每个目标文件的类型打印机（请参阅 Python 中的目标文件）， 接着是每个程序空间的类型打印机（请参阅 Python 中的程序空间），最后是全局类型打印机来完成的。

GDB 将调用每个已启用的 类型打印机的 instantiate 方法。 如果此方法返回 None, 则结果将被忽略； 否则，它将被添加到识别器列表中。

然后，当 GDB 要显示一个类型名称时，它会遍历识别器列表。 对于每个识别器，它会调用识别 函数，如果函数返回非None 值则停止。 识别函数定义如下：

上的方法 type_recognizer: 识别 (self, type) ¶

如果 类型 未被识别，则返回 None. 否则， 返回一个字符串，该字符串将作为 类型的名称进行打印。 类型 参数将是 gdb.Type 的一个实例（请参阅 Python 中的类型).

GDB 使用这种两步方法，以便类型打印机可以有效地缓存信息，而不会长时间保留这些信息。 例如，在类型打印机中查找类型信息并在识别器的生命周期内保留它可能会很方便； 如果只进行一次遍历，那么类型打印机将不得不使用事件系统，以避免保留可能随着 inferior 的变化而过时的信息。
