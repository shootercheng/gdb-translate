23.3.2.8 编写一个漂亮打印机 ¶

一个漂亮打印机由两部分组成：一个用于检测类型是否受支持的查找函数，以及打印机本身。

这里有一个示例，展示了如何 std::string 编写一个打印机。请参阅 漂亮打印 API，了解该类必须提供的 API 详细信息。请注意，此示例使用了 gdb.ValuePrinter 基类，并谨慎地为其局部状态使用了前导下划线。 

class StdStringPrinter(gdb.ValuePrinter):
    "打印一个 std::string"

    def __init__(self, val):
        self.__val = val

    def to_string(self):
        return self.__val['_M_dataplus']['_M_p']

    def display_hint(self):
        return 'string'

这里有一个示例，展示了如何为上述打印机示例编写一个查找函数。 

def str_lookup_function(val):
    lookup_tag = val.type.tag
    if lookup_tag is None:
        return None
    regex = re.compile("^std::basic_string<char,.*>$")
    if regex.match(lookup_tag):
        return StdStringPrinter(val)
    return None

示例查找函数提取值的类型，并尝试将其与可以漂亮打印的类型进行匹配。如果它是打印机可以漂亮打印的类型，它将返回一个打印机对象。如果不是，它将返回 None.

我们建议您将核心漂亮打印机放入一个 Python 包中。如果您的漂亮打印机用于某个库，我们进一步建议在包名中嵌入一个版本号。这种做法将使 GDB 能够同时加载您的多个版本的漂亮打印机，因为它们将有不同的名称。

您应该编写自动加载代码（请参阅 Python 自动加载)，使其可以多次评估而不改变其含义。一个理想的自动加载文件将仅由 import您的打印机模块的

总体而言，这种方法可以很好地扩展到多个 inferior，每个 inferior 可能使用不同的库版本。在 Python 包名中嵌入版本号将确保 GDB 能够同时加载两组打印机。然后，因为漂亮打印机的搜索是按目标文件进行的，并且因为您的自动加载代码小心地将您的库的打印机注册到特定的目标文件， GDB 将为每个 inferior 使用的特定版本的库找到正确的打印机。

为了继续 std::string 示例（请参阅 漂亮打印 API）， 此代码可能出现在 gdb.libstdcxx.v6: 

def register_printers(objfile):
    objfile.pretty_printers.append(str_lookup_function)

然后自动加载文件的相应内容将是： 

import gdb.libstdcxx.v6
gdb.libstdcxx.v6.register_printers(gdb.current_objfile())

前面的示例展示了一个基本的漂亮打印机。 有一些方面可以改进。 打印机没有名称，这使得在已安装的打印机列表中难以识别。查找函数有名称，但查找函数可以有任意甚至相同的名称。

其次，打印机只处理一种类型，而一个库通常有几种类型。可以为库中每个所需的类型安装一个查找函数，但也可以让一个查找函数识别几种类型。后者是处理这种情况的常规方法。 如果一个漂亮打印机可以处理多种数据类型，那么它的 子打印机 是针对各个数据类型的打印机。

gdb.printing 模块提供了一种正式的方法来解决这些问题（请参阅 gdb.printing)。 这里有另一个处理多种类型的示例。

这些是我们要漂亮打印的类型： 

struct foo { int a, b; };
struct bar { struct foo x, y; };

这里是打印机： 

class fooPrinter(gdb.ValuePrinter):
    """打印一个 foo 对象。"""

    def __init__(self, val):
        self.__val = val

    def to_string(self):
        return ("a=<" + str(self.__val["a"]) +
                "> b=<" + str(self.__val["b"]) + ">")

class barPrinter(gdb.ValuePrinter):
    """打印一个 bar 对象。"""

    def __init__(self, val):
        self.__val = val

    def to_string(self):
        return ("x=<" + str(self.__val["x"]) +
                "> y=<" + str(self.__val["y"]) + ">")

这个示例不需要查找函数，这由 gdb.printing 模块处理。相反，提供了一个函数来构建处理查找的对象。 

import gdb.printing

def build_pretty_printer():
    pp = gdb.printing.RegexpCollectionPrettyPrinter(
        "my_library")
    pp.add_printer('foo', '^foo$', fooPrinter)
    pp.add_printer('bar', '^bar$', barPrinter)
    return pp

这里是自动加载支持： 

import gdb.printing
import my_library
gdb.printing.register_pretty_printer(
    gdb.current_objfile(),
    my_library.build_pretty_printer())

最后，当这个打印机加载到 GDB, 这里是 ‘info pretty-printer’的相应输出： 

(gdb) info pretty-printer
my_library.so:
  my_library
    foo
    bar
