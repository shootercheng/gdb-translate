18.4 特殊节中的调试信息 ¶

某些系统会提供预构建的可执行文件和库，这些文件和库有一个特殊的 ‘.gnu_debugdata’ 节。此功能称为 迷你调试信息. 这个节包含一个经过LZMA压缩的对象，用于为回溯提供额外的符号。

这个节的目的是为简单的回溯提供额外的最小调试信息。它并非旨在替代完整的单独调试信息（请参阅 单独文件中的调试信息)。下面的示例展示了预期的使用方式；然而， GDB 目前并没有对该节中可能包含的调试信息类型设置限制。

GDB 支持此扩展。如果该节存在，那么在找不到其他调试信息源且 GDB 已配置为支持LZMA的情况下，会使用该节。

可以使用 objcopy 和其他标准实用工具轻松创建此节： 

# 从主二进制文件中提取动态符号，无需在普通符号表中也包含这些符号。
nm -D 二进制文件 --format=posix --defined-only \
  | awk '{ print $1 }' | sort > dynsyms

# 从调试信息中提取所有文本（即函数）符号。
# （注意，实际上我们也接受 “D” 符号，这是为了像PowerPC64这样使用函数描述符的平台的便利。）
nm 二进制文件 --format=posix --defined-only \
  | awk '{ if ($2 == "T" || $2 == "t" || $2 == "D") print $1 }' \
  | sort > funcsyms

# 保留所有不在动态符号表中的函数符号。
comm -13 dynsyms funcsyms > keep_symbols

# 将完整的调试信息分离到调试二进制文件中。
objcopy --only-keep-debug 二进制文件 调试文件

# 复制完整的调试信息，只保留最小的符号集并移除一些不必要的节。
objcopy -S --remove-section .gdb_index --remove-section .comment \
  --keep-symbols=keep_symbols debug mini_debuginfo

# 从原始二进制文件中删除完整的调试信息。
strip --strip-all -R .comment 二进制文件

# 将压缩数据注入到原始二进制文件的 .gnu_debugdata 节中。
xz mini_debuginfo
objcopy --add-section .gnu_debugdata=mini_debuginfo.xz 二进制文件
