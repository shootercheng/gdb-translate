23.4.3.16 从 Guile 访问块。 ¶

在 GDB, 符号存储在块中。块大致对应于源代码中的一个作用域。块是分层组织的，在 Guile 中分别表示为类型为 <gdb:block>. 块依赖于可用的调试信息。

一个帧有一个块。请参阅 从 Guile 访问 inferior 栈帧。, 以获取关于帧的更深入讨论。

最外层的块称为 全局块. 全局块通常包含公共全局变量和函数。

嵌套在全局块内的块是 静态块. 静态块通常包含文件作用域的变量和函数。

GDB 提供了一种获取块的父块的方法，但目前没有办法检查块的子块，也无法遍历符号表中的所有块（请参阅 Guile 中的符号表表示。)。

以下是一个简短的示例，有助于解释块： 

/* 这在全局块中。  */
int global;

/* 这在静态块中。  */
static int file_scope;

/* 'function' 在全局块中，'argument' 在 'function' 内部嵌套的块中。  */
int function (int argument)
{
  /* 'local' 在 'function' 内部的一个块中。它可能与 'argument' 在同一个块中，也可能不在。  */
  int local;

  {
     /* 'inner' 在一个块中，该块的父块是包含 'local' 的块。  */
     int inner;

     /* 如果此调用被编译器展开，你可能会在这里看到一个嵌套块，其函数为 'inline_function'，父块是包含 'inner' 的块。  */
     inline_function ();
  }
}

以下与块相关的过程由 (gdb) 模块提供：

Scheme 过程： 是否为块？ 对象 ¶

返回 #t 如果 对象 是一个 <gdb:block> 对象。否则返回 #f.

Scheme 过程： 块是否有效？ 块 ¶

返回 #t 如果 <gdb:block> 块 有效， #f 如果无效。如果块所引用的块在 inferior 中不再存在，则块对象可能会变得无效。如果在调用过程时块无效，所有其他 <gdb:block> 方法都会抛出异常。在遍历块的符号时也会检查块的有效性。

Scheme 过程： 块的起始地址 块 ¶

返回 <gdb:block> 块.

Scheme 过程： 块的结束地址 块 ¶

返回 <gdb:block> 块.

Scheme 过程： 块所属的函数 块 ¶

返回 <gdb:block> 块 所代表的 <gdb:symbol> 对象的名称。如果块没有名称，则返回 #f 。

对于普通的函数块，父块是静态块。但是，你应该注意，函数块的父块可能不是静态块，例如内联函数就是这种情况。

Scheme 过程： 块的父块 块 ¶

返回包含 <gdb:block> 块.的块。如果父块不存在，则返回 #f 。

Scheme 过程： 块的全局块 块 ¶

返回与 <gdb:block> 块关联的全局块。

Scheme 过程： 块的静态块 块 ¶

返回与 <gdb:block> 块关联的静态块。

Scheme 过程： 块是否为全局块？ 块 ¶

返回 #t 如果 <gdb:block> 块 是全局块。否则返回 #f.

Scheme 过程： 块是否为静态块？ 块 ¶

返回 #t 如果 <gdb:block> 块 是静态块。否则返回 #f.

Scheme 过程： 块中的符号 ¶

返回 <gdb:block> 块中所有符号（作为 <gdb:symbol> 对象）的列表。

Scheme 过程： 创建块符号迭代器 块 ¶

返回一个类型为 <gdb:iterator> 的对象，该对象将遍历块中的所有符号。Guile 程序不应假设特定的块对象总是包含给定的符号，因为 GDB 特性和基础设施的变化可能会导致符号在符号表中的块之间移动。请参阅 Guile 中的迭代器.

Scheme 过程： 块符号进度对象？ ¶

如果对象是 <gdb:block-symbols-progress> 对象，则返回 #t。此对象将从 progress 元素中获取，该元素来自 <gdb:iterator> 返回的对象。 make-block-symbols-iterator.

Scheme 过程： 查找块 程序计数器 ¶

返回包含给定 <gdb:block> 程序计数器 值的最内层 块。如果找不到指定 程序计数器 值对应的块，函数将返回 #f.
