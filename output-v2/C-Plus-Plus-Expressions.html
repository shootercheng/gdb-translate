15.4.1.3 C++ 表达式 ¶

GDB 表达式处理可以解释大多数 C++ 表达式。

警告： GDB 只有在使用合适的编译器和合适的调试格式时才能调试 C++ 代码。目前， GDB 在调试使用最新版本的++ 编译器编译的 C GCC 代码时效果最佳。首选 DWARF 调试格式； GCC 在大多数流行平台上默认使用此格式。使用 GDB 调试 C++ 代码时，其他编译器和/或调试格式可能效果不佳或根本无法工作。请参阅 为调试而编译.

  1. 允许调用成员函数；你可以使用像 
    count = aml->GetOriginal(x, y) 这样的表达式
  2. 当一个成员函数处于活动状态（在所选的栈帧中）时，你的表达式与该成员函数具有相同的命名空间；也就是说， GDB 允许隐式引用类实例指针 this 遵循与 C 相同的规则++. using 当前作用域中的声明也会被 GDB尊重。
  3. 你可以调用重载函数； GDB 会在一些限制条件下将函数调用解析为正确的定义。 GDB 不会执行涉及用户定义类型转换、构造函数调用或程序中不存在的模板实例化的重载解析。它也无法处理省略号参数列表或默认参数。

    它会执行整数转换和提升、浮点提升、算术转换、指针转换、类对象到基类的转换以及诸如函数或数组到指针的标准转换；它要求函数参数数量完全匹配。

    除非你指定了 set overload-resolution off. 请参阅 GDB C 的特性++.

    你必须指定 set overload-resolution off 才能使用显式函数签名调用重载函数，例如 

    p 'foo(char,int)'('x', 13)

    GDB 命令补全功能可以简化这一过程；请参阅 命令补全.

  4. GDB 理解声明为 C++ 左值或右值引用的变量；你可以在表达式中像在 C++ 源代码中一样使用它们 —— 它们会自动解引用。

    在 GDB 显示帧时显示的参数列表中，引用变量的值不会显示（与其他变量不同）；这避免了混乱，因为引用通常用于大型结构。 引用变量的地址总是会显示，除非你指定了 ‘ set print address off’。

  5. GDB 支持 C++ 名称解析运算符 :: —— 你的表达式可以像程序中的表达式一样使用它。由于一个作用域可能定义在另一个作用域中，如有必要，你可以多次使用 :: ，例如在像 ‘scope1::scope2::name’ 这样的表达式中。 GDB 还允许在 C 和 C++ 调试中通过引用源文件来解析名称作用域（请参阅 程序变量)。
  6. GDB 会根据 C++ 规范执行依赖于参数的查找。
