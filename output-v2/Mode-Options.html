2.1.2 选择模式

你可以以各种替代模式运行 GDB 例如，以批处理模式或安静模式。

-nx ¶
-n

不要执行任何初始化文件中找到的命令（请参阅 初始化文件

-nh ¶

不要执行任何主目录初始化文件中找到的命令（请参阅 主目录初始化文件）。 系统范围和当前目录的初始化文件仍会加载。

-quiet ¶
-silent
-q

“安静”。 不打印介绍性消息和版权消息。 这些消息在批处理模式下也会被抑制。

这也可以使用 set startup-quietly on启用。 默认值是 off。 使用 show startup-quietly 查看当前设置。 将 set startup-quietly on 放入你的早期初始化文件（请参阅 初始化文件），以使未来的 GDB 会话安静启动。

-batch ¶

以批处理模式运行。 在处理完用 ‘ 0 ’ 指定的所有命令文件（以及初始化文件中的所有命令，如果没有用 ‘-x’ 抑制）后，以状态-n’ 退出。 如果在执行命令文件中的 GDB 命令时发生错误，则以非零状态退出。 批处理模式还会禁用分页，设置无限的终端宽度和高度（请参阅 屏幕大小, 并且就像 set confirm off 生效一样（请参阅 可选警告和消息）。

批处理模式对于将 GDB 用作过滤器可能很有用，例如在另一台计算机上下载并运行程序；为了使这更有用，消息 

程序正常退出。

（通常在 GDB 控制下运行的程序终止时发出）在批处理模式下运行时不会发出。

-batch-silent ¶

完全像 ‘-batch’ 一样以批处理模式运行，但完全静默。 所有 GDB 输出到 stdout 的内容都被阻止（stderr 不受影响）。 这比 ‘-silent’ 安静得多，对于交互式会话来说没有用。

这在使用会给出 ‘加载段’ 消息的目标时特别有用。

请注意，通过 GDB而不是直接写入 stdout输出的目标也会被静音。

-return-child-result ¶

的返回码将是子进程（正在调试的进程）的返回码，但以下情况除外： GDB

  • GDB 异常退出。 例如，由于参数不正确或内部错误。 在这种情况下，退出码与没有 ‘-return-child-result’ 时相同。
  • 用户以显式值退出。 例如，‘quit 1’。
  • 子进程从未运行，或不允许终止，在这种情况下，退出码将为 -1。

此选项与 ‘-batch’ 或 ‘-batch-silent’ 结合使用时很有用，当 GDB 用作远程程序加载器或模拟器接口时。

-nowindows ¶
-nw

“无窗口”。 如果 GDB 内置了图形用户界面（GUI），则此选项告诉 GDB 仅使用命令行界面。 如果没有可用的GUI，此选项无效。

-windows ¶
-w

如果 GDB 包含GUI，则此选项要求尽可能使用它。

-cd directory ¶

运行 GDB 使用 目录 作为其工作目录，而不是当前目录。

-data-directory directory ¶
-D directory

运行 GDB 使用 目录 作为其数据目录。 数据目录是 查找其辅助文件的地方。 请参阅 GDB GDB数据文件.

-binary-output ¶

指示 GDB 将其 stdout/stderr 的转换模式设置为二进制。 仅适用于MS-Windows。 对于运行 GDB 测试套件很有用。 默认情况下， GDB 以文本模式打开 stdout/stderr ，并将 ‘\n’ （LF，单个换行符）转换为 ‘\r\n’ （CR-LF，回车换行组合）。 如果设置了此选项，则不进行转换。

-fullname ¶
-f

GNU Emacs在将 GDB 作为子进程运行时会设置此选项。 它告诉 GDB 每次显示堆栈帧时（包括程序每次停止时）以标准、可识别的方式输出完整的文件名和行号。 这种可识别的格式看起来像两个 ‘\032’ 字符，后面跟着用冒号分隔的文件名、行号和字符位置，以及一个换行符。 Emacs到GDB 的接口程序使用两个 ‘\032’ 字符作为信号来显示该帧的源代码。

-annotate level ¶

此选项设置 注释级别 在 GDB内部。 其效果与使用 ‘set annotate 级别’ 相同（请参阅 GDB 注释）。 注释 级别 控制 GDB 在其提示符、表达式的值、源代码行和其他类型的输出中一起打印多少信息。 级别0是正常级别，级别1用于 GDB 作为 GNU Emacs的子进程运行时，级别3是适合控制 GDB的程序的最大注释级别，级别2已被弃用。

注释机制在很大程度上已被 GDB/MI 取代（请参阅 GDB/MI 接口）。

--args ¶

更改命令行的解释方式，以便可执行文件后面的参数作为命令行参数传递给被调试程序。 此选项停止选项处理。

-baud bps ¶
-b bps

设置 GDB 用于远程调试的任何串行接口的线路速度（波特率或每秒位数）。

-l timeout ¶

设置 GDB 用于远程调试的任何通信的超时时间（以秒为单位）。

-tty device ¶
-t device

使用 设备 作为你程序的标准输入和输出运行。

-tui ¶

启动时激活 文本用户界面 。 文本用户界面在终端上管理多个文本窗口，显示源代码、汇编代码、寄存器和 GDB 命令输出（请参阅 GDB 文本用户界面）。 如果你从Emacs运行 GDB ，则不要使用此选项（请参阅 在 GDB 下使用 GNU Emacs）。

-interpreter interp ¶

使用解释器 interp 与控制程序或设备进行接口。 此选项旨在由将 GDB 用作后端进行通信的程序设置。 请参阅 命令解释器.

‘--interpreter=mi’ （或 ‘--interpreter=mi3’）会使 GDB 使用 GDB/MI 接口 版本3（请参阅 GDB/MI 接口），该版本自 GDB 版本9.1起包含。 GDB/MI 版本2（mi2），包含在 GDB 6.0中，版本1（mi1），包含在 GDB 5.3中，也可用。 早期的 GDB/MI 接口不再受支持。

-write ¶

以读写模式打开可执行文件和核心文件。 这相当于 ‘set write on’ 命令在 GDB 内部（请参阅 修补程序）。

-statistics ¶

此选项使 GDB 在完成每个命令并返回提示符后打印有关时间和内存使用的统计信息。

-version ¶

此选项使 GDB 打印其版本号和无担保声明，然后退出。

-configuration ¶

此选项使 GDB 打印有关其构建时配置参数的详细信息，然后退出。 这些详细信息在报告 GDB 错误时可能很重要（请参阅 报告 GDB中的错误
