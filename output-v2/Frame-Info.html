8.4 关于帧的信息 ¶

还有其他几个命令可用于打印有关所选栈帧的信息。

frame
f

如果不带任何参数使用此命令，它不会更改所选的帧，但会打印当前所选栈帧的简要描述。它可以缩写为 f. 带有参数时，此命令用于选择一个栈帧。请参阅 选择一个帧.

info frame ¶
info f

此命令会打印所选栈帧的详细描述，包括：

  • 帧的地址
  • 下一个帧（由该帧调用）的地址
  • 上一个帧（调用该帧的帧）的地址
  • 与此帧对应的源代码所使用的语言
  • 帧的参数地址
  • 帧的局部变量地址
  • 保存在其中的程序计数器（调用帧中的执行地址）
  • 在帧中保存了哪些寄存器

当出现问题导致栈格式不符合通常的约定时，详细描述会很有用。

info frame [ frame-selection-spec ]
info f [ frame-selection-spec ]

打印由 帧选择规范选择的帧的详细描述。 帧选择规范 与 frame 命令的相同（请参阅 选择一个帧）。此命令不会更改所选的帧。

info args [-q] ¶

打印所选帧的参数，每个参数占一行。

可选标志 ‘-q’，代表 ‘安静’，禁用打印标题信息和解释为什么没有打印参数的消息。

info args [-q] [-t type_regexp] [regexp]

与 info args类似，但仅打印使用提供的正则表达式选择的参数。

如果提供了 正则表达式 ，则仅打印名称与正则表达式 正则表达式匹配的参数。

如果提供了 类型正则表达式 ，则仅打印其类型（由 whatis 命令打印）与正则表达式 类型正则表达式匹配的参数。 如果 类型正则表达式 包含空格，则应将其用引号括起来。如有需要，使用反斜杠转义特殊字符或引号的含义。

如果同时提供了 正则表达式 和 类型正则表达式 ，则仅当参数的名称与 正则表达式 匹配且其类型与 类型正则表达式匹配时才会打印该参数。

info locals [-q] ¶

打印所选帧的局部变量，每个变量占一行。这些是在所选帧的执行点可访问的所有变量（声明为静态或自动变量）。

可选标志 ‘-q’，代表 ‘安静’，禁用打印标题信息和解释为什么没有打印局部变量的消息。

info locals [-q] [-t type_regexp] [regexp]

与 info locals类似，但仅打印使用提供的正则表达式选择的局部变量。

如果提供了 正则表达式 ，则仅打印名称与正则表达式 正则表达式匹配的局部变量。

如果提供了 类型正则表达式 ，则仅打印其类型（由 whatis 命令打印）与正则表达式 类型正则表达式. 如果 类型正则表达式 包含空格，则应将其用引号括起来。如有需要，使用反斜杠转义特殊字符或引号的含义。

如果同时提供了 正则表达式 和 类型正则表达式 ，则仅当局部变量的名称与 正则表达式 匹配且其类型与 类型正则表达式.

命令 info locals -q -t 类型正则表达式 可以有效地与以下命令结合使用 frame apply 和 thread apply. 例如，您的程序可能会使用资源获取即初始化（RAII）类型，如 lock_something_t：类型为的每个局部变量 lock_something_t 会自动放置一个锁，当变量超出作用域时该锁会被销毁。然后，您可以通过执行以下操作列出程序中所有已获取的锁 

thread apply all -s frame apply all -s info locals -q -t lock_something_t

或者等效的更简短形式 

tfaas i lo -q -t lock_something_t
