Alpha和 MIPS基于MIPS的计算机使用不寻常的栈帧，这有时需要 GDB 在目标代码中向后搜索以找到函数的起始位置。

为了提高响应时间（特别是对于嵌入式应用程序，其中 GDB 可能只能通过慢速串行线路进行此搜索），你可能想使用以下命令之一来限制此搜索的大小：

set heuristic-fence-post limit ¶

限制 GDB 在搜索函数起始位置时最多检查 限制值 字节。值为 0 （默认值）表示没有限制。然而，除了 0, 限制值越大， heuristic-fence-post 必须搜索的字节就越多，因此运行所需的时间就越长。只有在调试剥离了符号信息的可执行文件时才需要使用此命令。

show heuristic-fence-post

显示当前的限制值。

这些命令仅在 仅 当 GDB 被配置为调试Alpha或 MIPS 处理器上的程序时可用。

几个 MIPS特定的命令在调试 MIPS 程序时可用：

set mips abi arg ¶

告诉 GDB 目标程序使用的是哪种 MIPS 应用二进制接口（ABI）。 arg 的可能值为：

‘自动’

与当前二进制文件关联的默认ABI（这是默认设置）。

‘o32’
‘o64’
‘n32’
‘n64’
‘eabi32’
‘eabi64’
show mips abi ¶

显示 MIPS 被用于调试目标程序的ABI。 GDB 用于调试目标程序。

set mips compression arg ¶

告诉 GDB 目标程序使用的是哪种 MIPS 压缩 指令集架构 （指令集架构）编码。 GDB 用于代码反汇编和其他内部解释目的。只有在没有可执行文件与调试会话关联，或者可执行文件没有提供其使用的编码信息时，才会参考此设置。否则，此设置会根据可执行文件提供的信息自动更新。

参数 arg 的可能值为‘mips16’和‘micromips’。默认的压缩 指令集架构 编码是‘mips16’，因为包含 MIPS16 代码的可执行文件通常不会被识别为这样。

此设置是“粘性”的；也就是说，它在调试会话之间保留其值，直到通过此命令显式重置或从可执行文件隐式重置。

编译器和/或汇编器通常会添加符号表注释来标识为 MIPS16 或 microMIPS 指令集架构如果存在这些函数作用域注释， GDB 则优先使用它们，而不是全局的压缩 指令集架构 编码设置。

show mips compression ¶

显示 MIPS 压缩 指令集架构 被用于调试目标程序的编码。 GDB 用于调试目标程序。

set mipsfpu
show mipsfpu

参见 设置mipsfpu。

set mips mask-address arg ¶

此命令确定64位的MIPS地址的最高32位是否被屏蔽。参数 MIPS 参数 arg 可以是‘开’、‘关’或‘自动’。后者是默认设置，它让 GDB 确定正确的值。

show mips mask-address ¶

显示 MIPS 地址的高32位是否被屏蔽。

set remote-mips64-transfers-32bit-regs ¶

此命令控制与以32位数量传输数据的64位 MIPS 目标的兼容性。如果你有一个旧的 MIPS 64位目标，它对某些寄存器传输32位数据，如 SR 和 FSR, 而对其他寄存器传输64位数据，将此选项设置为‘开’。

show remote-mips64-transfers-32bit-regs ¶

显示与旧的 MIPS 64位目标的当前兼容性设置。

set debug mips ¶

此命令打开和关闭针对 MIPS特定目标代码的调试消息。 GDB。

show debug mips ¶

显示 MIPS 调试消息的当前设置。
