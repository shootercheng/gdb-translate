12 个 C 预处理器宏 ¶

某些语言，如 C 和 C++, 提供了一种定义和调用“预处理器宏”的方法，这些宏会扩展为标记字符串。 GDB 可以计算包含宏调用的表达式，显示宏展开的结果，并显示宏的定义，包括其定义位置。

您可能需要专门编译您的程序，以便为 GDB 提供有关预处理器宏的信息。大多数编译器即使在使用 -g 标志编译时，也不会在调试信息中包含宏。请参阅 为调试而编译.

程序可能在某一点定义一个宏，稍后移除该定义，然后再提供一个不同的定义。因此，在程序的不同点，宏可能有不同的定义，或者根本没有定义。如果有当前栈帧， GDB 会使用该帧源代码行作用域内的宏。否则， GDB 会使用当前列表位置作用域内的宏；请参阅 打印源行.

每当 GDB 计算表达式时，它总是会展开表达式中存在的任何宏调用。 GDB 还提供了以下命令来显式处理宏。

macro expand expression ¶
macro exp expression

显示展开 表达式中所有预处理器宏调用的结果。由于 GDB 只是展开宏，但不解析结果， 表达式 不必是有效的表达式；它可以是任何标记字符串。

macro expand-once expression ¶
macro exp1 expression

（此命令尚未实现。） 显示展开 表达式中显式出现的那些预处理器宏调用的结果。该展开中出现的宏调用保持不变。此命令允许您更清楚地看到特定宏的效果，而不会被进一步的展开所混淆。由于 GDB 只是展开宏，但不解析结果， 表达式 不必是有效的表达式；它可以是任何标记字符串。

info macro [-a|-all] [--] macro ¶

显示命名的 宏, 的当前定义或所有定义，并描述该定义的源位置或编译器命令行。可选的双破折号用于表示参数处理的结束和 宏 的开始，对于非类 C 宏，宏可能以连字符开头。

info macros locspec ¶

显示在解析 位置规范, 后得到的代码位置的源行上生效的所有宏定义，并描述这些定义的源位置或编译器命令行。

macro define macro replacement-list ¶
macro define macro(arglist) replacement-list

为名为 宏, 的预处理器宏引入一个定义，其调用会被 替换列表中给出的标记所替换。此命令的第一种形式定义一个“类对象”宏，它不接受参数；第二种形式定义一个“类函数”宏，它接受 参数列表中给出的参数。

此命令引入的定义在 GDB中计算的每个表达式中都有效，直到使用下面描述的 macro undef 命令将其移除。该定义会覆盖正在调试的程序中所有 宏 的定义，以及任何先前用户提供的定义。

macro undef macro ¶

移除名为 宏. 的宏的任何用户提供的定义。此命令仅影响上面描述的 macro define 命令提供的定义；它不能移除正在调试的程序中存在的定义。

macro list ¶

列出使用 macro define 命令定义的所有宏。

以下是一个展示上述命令实际操作的转录。首先，我们展示我们的源文件： 

$ cat sample.c
#include <stdio.h>
#include "sample.h"

#define M 42
#define ADD(x) (M + x)

main ()
{
#define N 28
  printf ("Hello, world!\n");
#undef N
  printf ("We're so creative.\n");
#define N 1729
  printf ("Goodbye, world!\n");
}
$ cat sample.h
#define Q <
$

现在，我们使用 GNU C 编译器 GCC. 编译程序。我们传递 -gdwarf-214 和 -g3 标志，以确保编译器在调试信息中包含有关预处理器宏的信息。 

$ gcc -gdwarf-2 -g3 sample.c -o sample
$

现在，我们在我们的示例程序上启动 GDB : 

$ gdb -nw sample
GNU gdb 2002-05-06-cvs
版权所有 2002 自由软件基金会
GDB 是自由软件，...
(gdb)

即使程序没有运行，我们也可以展开宏并检查它们的定义。 GDB 使用当前列表位置来决定哪些宏定义在作用域内： 

(gdb) list main
3
4       #define M 42
5       #define ADD(x) (M + x)
6
7       main ()
8       {
9       #define N 28
10        printf ("Hello, world!\n");
11      #undef N
12        printf ("We're so creative.\n");
(gdb) info macro ADD
在 /home/jimb/gdb/macros/play/sample.c:5 处定义
#define ADD(x) (M + x)
(gdb) info macro Q
在 /home/jimb/gdb/macros/play/sample.h:1 处定义
  在 /home/jimb/gdb/macros/play/sample.c:2 处包含
#define Q <
(gdb) macro expand ADD(1)
展开为：(42 + 1)
(gdb) macro expand-once ADD(1)
展开为：once (M + 1)
(gdb)

在上面的示例中，请注意 macro expand-once 仅展开原始文本中显式的宏调用 —— ADD 的调用 —— 但不展开由 M引入的宏 ADD的调用。

一旦程序开始运行, GDB 使用当前栈帧源行生效的宏定义： 

(gdb) break main
断点 1 在 0x8048370 处：文件 sample.c，第 10 行。
(gdb) run
开始执行程序：/home/jimb/gdb/macros/play/sample

在断点 1 处，main () 在 sample.c:10
10        printf ("Hello, world!\n");
(gdb)

在第 10 行，宏的定义 N 在第 9 行生效： 

(gdb) info macro N
在 /home/jimb/gdb/macros/play/sample.c:9 处定义
#define N 28
(gdb) macro expand N Q M
展开为：28 < 42
(gdb) print N Q M
$1 = 1
(gdb)

当我们跳过移除 N定义的指令，然后 给它一个新的定义时， GDB 会找到每个点生效的定义（或不存在定义）： 

(gdb) next
Hello, world!
12        printf ("We're so creative.\n");
(gdb) info macro N
符号 `N' 在 /home/jimb/gdb/macros/play/sample.c:12 处没有作为 C/C++ 预处理器宏的定义
(gdb) next
We're so creative.
14        printf ("Goodbye, world!\n");
(gdb) info macro N
在 /home/jimb/gdb/macros/play/sample.c:13 处定义
#define N 1729
(gdb) macro expand N Q M
展开为：1729 < 42
(gdb) print N Q M
$2 = 0
(gdb)

除了源文件，宏可以在编译命令行上使用 -D名称=值 语法来定义。对于以这种方式定义的宏， GDB 会将其定义位置显示为提交给编译器的源文件的第零行。 

(gdb) info macro __STDC__
在 /home/jimb/gdb/macros/play/sample.c:0 处定义
-D__STDC__=1
(gdb)

脚注

(14)

这是 最小值。最近版本的 GCC 支持 -gdwarf-3 和 -gdwarf-4；我们建议始终选择最新版本的 DWARF。
