15.3.2 范围检查概述 ¶

在某些语言（如Modula - 2）中，超出类型的边界是错误的；这通过运行时检查来强制执行。这种范围检查旨在通过确保计算不会溢出，或者数组元素访问的索引不超过数组的边界来保证程序的正确性。

对于你在 GDB 命令中使用的表达式，你可以告诉 GDB 以三种方式之一处理范围错误：忽略它们，始终将它们视为错误并放弃表达式，或者发出警告但仍计算表达式。

范围错误可能由数值溢出、超出数组索引边界或输入不属于任何类型的常量引起。然而，有些语言不将溢出视为错误。在许多C语言的实现中，数学溢出会导致结果“回绕”到较低的值 —— 例如，如果 m 是最大的整数值，并且 s 是最小的，那么 

m  + 1 ⇒ s

这也因个别语言而异，在某些情况下还因个别编译器或机器而异。请参阅 支持的语言，以获取有关特定语言的更多详细信息。

GDB 提供了一些额外的命令来控制范围检查器：

set check range auto

根据当前使用的语言打开或关闭范围检查。请参阅 支持的语言，以了解每种语言的默认设置。

set check range on
set check range off

打开或关闭范围检查，覆盖当前使用语言的默认设置。如果设置与语言默认设置不匹配，则会发出警告。如果发生范围错误且范围检查已打开，则会打印一条消息并中止表达式的计算。

set check range warn

当 GDB 范围检查器检测到范围错误时输出消息，但仍尝试计算表达式。由于其他原因，计算表达式可能仍然无法进行，例如访问进程不拥有的内存（许多Unix系统中的典型示例）。

show check range

显示范围检查器的当前设置，以及它是否由 GDB自动设置。
