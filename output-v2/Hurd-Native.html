21.1.5 特定于 GNU Hurd系统 ¶

本小节描述 GDB 特定于 GNU Hurd原生调试的命令。

set signals ¶
set sigs

此命令切换对被调试程序信号拦截的状态，通过 GDB。诸如断点陷阱之类的Mach异常不受此命令影响。 sigs 是 signals的简写别名。

show signals ¶
show sigs

显示当前拦截被调试程序信号的状态。

set signal-thread ¶
set sigthread

此命令告知 GDB 哪个线程是 libc 信号线程。当信号传递给正在运行的进程时，该线程会运行。 set sigthread 是 set signal-thread的简写别名。

show signal-thread ¶
show sigthread

这两个命令显示当被调试程序接收到信号时哪个线程将运行。

set stopped ¶

此命令告知 GDB 被调试进程已停止，就像使用 SIGSTOP 信号一样。可以通过向其发送信号来继续已停止的进程。

show stopped ¶

此命令显示 GDB 是否认为被调试程序已停止。

set exceptions ¶

使用此命令关闭对被调试程序中异常的捕获。当异常捕获关闭时，断点和单步执行都将不起作用。要恢复默认设置，请打开异常捕获。

show exceptions ¶

显示当前对被调试程序中异常捕获的状态。

set task pause ¶

此命令在 GDB 获得控制权时切换任务挂起状态。将其设置为开启会立即生效，并且每当 GDB 获得控制权时任务就会挂起。将其设置为关闭将在下次继续执行被调试程序时生效。如果此选项设置为关闭，您可以使用 set thread default pause on 或 set thread pause on （见下文）来暂停单个线程。

show task pause ¶

显示当前任务挂起的状态。

set task detach-suspend-count ¶

此命令设置当 GDB 从任务分离时任务将保留的挂起计数。

show task detach-suspend-count

显示分离时任务将保留的挂起计数。

set task exception-port ¶
set task excp

此命令设置 GDB 将异常转发到的任务异常端口。参数应该是 发送权限 的任务值。 set task excp 是简写别名。

set noninvasive ¶

此命令将 GDB 切换到一种对被调试程序干扰最小的模式。这与使用 set task pause, set exceptions, 和 set signals 将值设置为与默认值相反的情况相同。

info send-rights ¶
info receive-rights
info port-rights
info port-sets
info dead-names
info ports
info psets

这些命令分别显示有关任务的发送权限、接收权限、端口权限、端口集和无效名称的信息。也有简写别名： info ports 对应于 info port-rights 和 info psets 对应于 info port-sets。

set thread pause ¶

此命令在 GDB 获得控制权时切换当前线程的挂起状态。将其设置为开启会立即生效，并且每当 GDB 获得控制权时当前线程就会挂起。将其设置为关闭将在下次继续执行被调试程序时生效。通常，此命令没有效果，因为当 GDB 获得控制权时，整个任务都会挂起。但是，如果您使用了 set task pause off （见上文），此命令可用于仅挂起当前线程。

show thread pause ¶

此命令显示当前线程挂起的状态。

set thread run

此命令设置当前线程是否允许运行。

show thread run

显示当前线程是否允许运行。

set thread detach-suspend-count ¶

此命令设置 GDB 分离线程时将在该线程上留下的挂起计数。此数字相对于 GDB 注意到该线程时找到的挂起计数；使用 set thread takeover-suspend-count 将其强制设置为绝对值。

show thread detach-suspend-count

显示 GDB 分离线程时将在该线程上留下的挂起计数。

set thread exception-port
set thread excp

设置将异常转发到的线程异常端口。这将覆盖由 set task exception-port （见上文）设置的端口。 set thread excp 是简写别名。

set thread takeover-suspend-count

通常， GDB的线程挂起计数是相对于 GDB 注意到每个线程时找到的值。此命令将挂起计数改为绝对值。

set thread default ¶
show thread default

上述每个 set thread 命令都有一个 set thread default 对应项（例如， set thread default pause, set thread default exception-port, 等等）。该 thread default 各种命令为所有线程设置默认线程属性；然后您可以使用非默认命令更改单个线程的属性。
