27.5.3 GDB/MI

记录用于通知 GDB/MI 客户端发生的其他更改。这些更改可能是 GDB/MI 命令（例如，断点已修改）的结果，或者是目标活动（例如，目标已停止）的结果。

*running,thread-id="thread"

字段可以是当前正在运行的线程的全局线程 ID，也可以是 ‘所有’ 如果所有线程都在运行。前端应假设 在发出此通知后，无法与正在运行的线程进行交互。前端不应假设此 通知对于任何命令仅输出一次。 GDB 可能会多次发出此通知，要么是针对不同的线程， 因为它无法同时恢复所有线程，要么甚至是针对单个线程，如果该线程必须在自由运行之前逐步执行某些代码。

*stopped,reason="reason",thread-id="id",stopped-threads="stopped",core="core"

目标已停止。 原因 字段可以具有以下值之一：

breakpoint-hit

到达了一个断点。

watchpoint-trigger

触发了一个观察点。

read-watchpoint-trigger

触发了一个读观察点。

access-watchpoint-trigger

触发了一个访问观察点。

function-finished

完成了 -exec-finish 或类似的 CLI 命令。

location-reached

完成了 -exec-until 或类似的 CLI 命令。

watchpoint-scope

一个观察点已超出范围。

end-stepping-range

完成了 -exec-next、-exec-next-instruction、-exec-step、-exec-step-instruction 或类似的 CLI 命令。

exited-signalled

下级程序因信号而退出。

exited

下级程序已退出。

exited-normally

下级程序正常退出。

signal-received

下级程序收到了一个信号。

solib-event

下级程序因加载或卸载库而停止。 当 stop-on-solib-events （请参阅 指定文件的命令) 被设置或当 catch load 或 catch unload 捕获点正在使用时（请参阅 设置捕获点)。

fork

下级程序进行了分叉。当 catch fork （请参阅 设置捕获点) 被使用时会报告此情况。

vfork

下级程序进行了 vfork。当 catch vfork （请参阅 设置捕获点) 被使用时会报告此情况。

syscall-entry

下级程序进入了一个系统调用。当 catch syscall （请参阅 设置捕获点) 被使用时会报告此情况。

syscall-return

下级程序从一个系统调用返回。当 catch syscall （请参阅 设置捕获点) 被使用时会报告此情况。

exec

下级程序调用了 exec. 当 catch exec （请参阅 设置捕获点) 被使用时会报告此情况。

no-history

没有足够的历史记录来继续反向执行。

id 字段标识直接导致停止的线程的全局线程 ID — 例如通过命中一个断点。 根据是否启用了全停止 模式（请参阅 全停止模式）， GDB 可能会停止所有线程，或者仅停止直接触发停止的线程。 如果所有线程都停止了， 已停止 字段的值将为 "all". 否则， 已停止 字段的值将是一个线程标识符列表。目前，此列表将始终只包含一个线程，但前端应准备好看到列表中有多个线程。 核心 字段报告停止事件发生的处理器核心。如果此类信息不可用，该字段可能会缺失。

=thread-group-added,id="id"
=thread-group-removed,id="id"

一个线程组已添加或已移除。 id 字段 包含线程组的 GDB 标识符。当添加一个线程组时，它通常可能未与正在运行的 进程关联。当移除一个线程组时，其 id 变为无效，不能以任何方式使用。

=thread-group-started,id="id",pid="pid"

一个线程组与一个正在运行的程序关联， 要么是因为程序刚刚启动，要么是因为线程组已附加到一个程序。 id 字段包含线程组的 GDB 标识符。 pid 字段 包含特定于操作系统的进程标识符。

=thread-group-exited,id="id"[,exit-code="code"]

一个线程组不再与一个正在运行的程序关联， 要么是因为程序已退出，要么是因为它已分离。 id 字段包含 GDB 线程组的标识符。 代码 字段是下级程序的退出代码；仅当下级程序以某个代码退出时才存在。

=thread-created,id="id",group-id="gid"
=thread-exited,id="id",group-id="gid"

一个线程已创建或已退出。 id 字段 包含线程的全局 GDB 标识符。 gid 字段标识该线程所属的线程组。

=thread-selected,id="id"[,frame="frame"]

通知所选线程或帧已更改。此通知 不是由 -thread-select 或 -stack-select-frame 命令产生的结果，而是在任何未记录为会更改所选线程和帧的 MI 命令 实际上更改了它们时发出。特别是，直接或间接 （通过用户定义的命令）调用 CLI thread 或 frame 命令， 将生成此通知。从其他 用户界面更改线程或帧（请参阅 命令解释器）也将生成此通知。

帧 字段仅在新选择的线程停止时存在。请参阅 GDB/MI 帧信息 以了解其值的格式。

我们建议前端在收到此通知后， 突出显示所选线程，并使后续命令应用于 该线程。

=library-loaded,...

报告程序加载了一个新的库文件。此 通知有 5 个字段 —id, 目标名称, 主机名称, 符号已加载 和 范围. id 字段是库的不透明标识符。对于远程调试情况， 目标名称 和 主机名称 字段分别给出目标和主机上的库文件名称。对于本地 调试，这两个字段具有相同的值。 符号已加载 字段仅为向后兼容而发出，不应依赖它来传达任何有用信息。 线程组 字段（如果存在）指定加载库的线程组的 id。如果该字段 不存在，则表示库是在所有当前线程组的上下文中加载的。 范围 字段指定属于此库的地址范围。

=library-unloaded,...

报告程序卸载了一个库。此通知 有以下字段 —id, 目标名称, 主机名称 和 范围 其含义与 =library-loaded 通知相同。

一个库可能会在下级程序的库列表中多次出现，但该库仅被映射到下级程序的地址空间一次。当发生这种情况，并且一个库副本被卸载，但仍有剩余副本时， 仍在使用 字段将为 ‘true’。在所有其他 情况下， 仍在使用 字段的值将为 ‘false’。

线程组 字段（如果存在）指定卸载库的线程组的 id。如果该字段 不存在，则表示库是在所有当前线程组的上下文中卸载的。

=traceframe-changed,num=tfnum,tracepoint=tpnum
=traceframe-changed,end

报告跟踪帧已更改，其新编号为 tfnum. 与此跟踪帧关联的跟踪点编号为 tpnum.

=tsv-created,name=name,initial=initial

报告新的跟踪状态变量 名称 已创建，初始值为 初始值.

=tsv-deleted,name=name
=tsv-deleted

报告跟踪状态变量 名称 已删除或所有跟踪状态变量已删除。

=tsv-modified,name=name,initial=initial[,current=current]

报告跟踪状态变量 名称 已修改，初始值为 初始值. 跟踪状态变量的当前值 当前值 是可选的，如果跟踪状态变量的当前值已知，则会报告。

=breakpoint-created,bkpt={...}
=breakpoint-modified,bkpt={...}
=breakpoint-deleted,id=number

分别报告断点已创建、修改或删除。仅将用户可见的断点报告给 MI 用户。

断点 参数的形式与各种断点命令返回的形式相同；请参阅 GDB/MI 断点命令. 编号 是断点的序号。

请注意，如果一个断点在命令的结果记录中发出，则它不会在异步记录中再次发出。

=record-started,thread-group="id",method="method"[,format="format"]
=record-stopped,thread-group="id"

下级程序的执行日志记录已启动或停止。 id 是 GDB 受影响的下级程序对应的线程组的标识符。

方法 字段指示用于记录执行的方法。如果正在使用的方法支持多种记录格式， 格式 将存在并包含当前使用的格式。请参阅 记录下级程序的执行并进行回放, 以了解现有方法和格式的值。

=cmd-param-changed,param=param,value=value

报告命令的参数 set param 已更改为 值. 在多字 set 命令中， 参数 是 set 命令的整个参数列表。 例如，在命令 set check type on, 参数 是 check type 和 值 是 on.

=memory-changed,thread-group=id,addr=addr,len=len[,type="code"]

报告从 地址 到 数据 + 长度 的字节已写入下级程序。 id 是受影响的下级程序对应的线程组的标识符。可选的 type="code" 部分在写入的内存包含可执行代码时报告。
