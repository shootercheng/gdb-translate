3.6 获取帮助

你可以随时询问 GDB 它自身关于其命令的信息，使用命令 help.

help ¶
h

你可以使用 help （缩写为 h），不带任何参数来显示一个简短的命令类列表： 

(gdb) help
命令类列表：

别名 -- 其他命令的用户自定义别名
断点 -- 使程序在某些点停止
数据 -- 检查数据
文件 -- 指定和检查文件
内部 -- 维护命令
晦涩 -- 晦涩的功能
运行 -- 运行程序
栈 -- 检查栈
状态 -- 状态查询
支持 -- 支持工具
跟踪点 -- 在不停止程序的情况下跟踪程序执行
用户自定义 -- 用户自定义命令

输入“help”后跟类名可获取该类中的命令列表。
输入“help”后跟命令名可获取完整文档。
如果命令名缩写不会引起歧义，则允许使用缩写。
(gdb)
help class

使用通用帮助类之一作为参数，你可以获取该类中各个命令的列表。如果一个命令有别名，别名会在命令名后面列出，用逗号分隔。如果一个别名有默认参数，别名的完整定义会在第一行后面给出。例如，以下是 status类的帮助显示： 

(gdb) help status
状态查询。

命令列表：

info, inf, i -- 用于显示正在调试的程序相关信息的通用命令
info address, iamain  -- 描述符号SYM的存储位置。
  别名 iamain = info address main
info all-registers -- 所选栈帧的所有寄存器及其内容列表
...
show, info set -- 用于显示调试器相关信息的通用命令

输入“help”后跟命令名可获取完整文档。
如果命令名缩写不会引起歧义，则允许使用缩写。
(gdb)
help command

以命令名作为 help 参数， GDB 会显示一段关于如何使用该命令的简短段落。如果该命令有一个或多个别名， GDB 会显示第一行，包含命令名及其所有别名，用逗号分隔。这第一行后面会跟着所有有默认参数的别名的完整定义。当询问别名的帮助时，会显示被别名化的命令的文档。

用户自定义别名可以使用 document 命令（请参阅 文档)进行可选的文档记录。 GDB 然后将此别名视为与被别名化的命令不同：此别名不会在被别名化的命令的帮助输出中列出，询问此别名的帮助将显示为该别名提供的文档，而不是被别名化的命令的文档。

apropos [-v] regexp ¶

apropos 命令会在所有 GDB 命令和别名及其文档中搜索 参数中指定的正则表达式。它会打印出所有找到的匹配项。可选标志‘-v’，代表‘详细’，表示输出匹配命令的完整文档，并突出显示文档中与 正则表达式匹配的部分。例如： 

apropos alias

的结果是： 

alias -- 定义一个新命令，该命令是现有命令的别名
aliases -- 其他命令的用户自定义别名

而 

apropos -v cut.*thread apply

的结果如下，如果启用了样式，‘cut for 'thread apply’会被突出显示。 

taas -- 将命令应用于所有线程（忽略错误和空输出）。
用法：taas 命令
‘thread apply all -s 命令’的快捷方式

tfaas -- 将命令应用于所有线程的所有帧（忽略错误和空输出）。
用法：tfaas 命令
‘thread apply all -s frame apply all -s 命令’的快捷方式
complete args ¶

complete args 命令列出命令开头的所有可能补全项。使用 参数指定你想要补全的命令开头。例如： 

complete i

的结果是： 

if
ignore
info
inspect

这是供 GNU Emacs使用的。

除了 help, 你还可以使用 GDB 命令 info 和 show 来查询你的程序的状态，或者 GDB 它自身的状态。每个命令都支持许多查询主题；本手册在适当的上下文中介绍了每个主题。 info 和 show 在命令、变量和函数索引中的列表指向所有子命令。请参阅 命令、变量和函数索引.

info ¶

此命令（缩写为 i）用于描述你的程序的状态。例如，你可以使用 info args显示传递给函数的参数，使用 info registers列出当前正在使用的寄存器，或者使用 info breakpoints列出你设置的断点。你可以使用 info 获取 help info子命令的完整列表。

set ¶

你可以使用 set将表达式的结果赋值给环境变量。例如，你可以使用 GDB 将 set prompt $提示符设置为美元符号。

show ¶

与 info, 相反， show 用于描述 GDB 它自身的状态。你可以通过使用相关命令 show来更改你可以 set做的大多数事情；例如，你可以使用 set radix控制显示使用的数字系统，或者简单地使用 show radix查询当前正在使用的数字系统。

要显示所有可设置的参数及其当前值，你可以不带参数使用 show ；你也可以使用 info set。两个命令产生相同的显示结果。

这里有几个杂项 show 子命令，所有这些子命令都没有对应的 set 命令：

show version ¶

显示正在运行的 GDB 的版本。你应该在 GDB 错误报告中包含此信息。如果你的站点使用多个版本的 GDB ，你可能需要确定你正在运行的是哪个 GDB 版本；随着 GDB 的发展，会引入新的命令，旧的命令可能会被淘汰。此外，许多系统供应商会提供 GDB的不同版本，并且 GDB 在 GNU/Linux发行版中也有不同版本。版本号与你启动 GDB时显示的版本号相同。

show copying ¶
info copying

显示有关复制 GDB的许可信息。

show warranty ¶
info warranty

显示 GNU “无保修”声明，或者如果你的 GDB 版本附带保修，则显示保修声明。

show configuration ¶

显示 GDB 构建时的详细配置信息。这会显示传递给 配置脚本的可选参数，以及 自动检测到的配置参数。在报告 configure错误（请参阅 GDB 报告 中的错误 GDB）时，在报告中包含此信息很重要。

此命令还会显示 GDB 正在使用的Readline库的版本号。版本号后面会显示‘（系统）’，表示 GDB 正在使用系统共享的Readline库，或者‘（内部）’，表示 GDB 正在使用静态链接的Readline库版本。
