22.8.2 自动加载线程调试库 ¶

此功能目前仅存在于 GNULinux本地主机上。

GDB 在某些情况下，会从特定于被调试程序的位置读取线程调试库（请参阅 设置libthread-db搜索路径)。

特殊的‘libthread-db搜索路径’条目‘$sdir’在处理时不会检查这个‘设置自动加载libthread-db’开关，因为一般来说系统库是可信的。在‘libthread-db搜索路径’条目的所有其他情况下， GDB 在尝试打开此类线程调试库之前，首先会检查‘设置 自动加载libthread-db’是否已启用。

请注意，加载此调试库还需要相应的配置 auto-load safe-path （请参阅 自动加载的安全限制)。

set auto-load libthread-db [on|off] ¶

启用或禁用特定于被调试程序的线程调试库的自动加载。

show auto-load libthread-db ¶

显示特定于被调试程序的线程调试库的自动加载是启用还是禁用。

info auto-load libthread-db ¶

打印所有已加载的特定于被调试程序的线程调试库的列表，并为每个此类库打印使用它的被调试程序的 进程ID列表。
