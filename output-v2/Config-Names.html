C.4 为主机和目标指定名称 ¶

在 `configure` 脚本中用于主机和目标的规范基于一个三部分的命名方案，但也支持一些简短的预定义别名。完整的命名方案按以下模式编码三条信息： 配置脚本 脚本基于一个三部分的命名方案，但也支持一些简短的预定义别名。完整的命名方案按以下模式编码三条信息： 

架构-厂商-操作系统

例如，你可以使用别名 sun4 作为 主机 参数，或者作为 目标 在一个 --target=target 选项。等效的完整名称是 ‘sparc - sun - sunos4’。

` 配置脚本 随附的 GDB 不提供任何查询功能来列出所有支持的主机和目标名称或别名。 配置脚本 调用Bourne shell脚本 config.sub 将缩写映射为完整名称；如果你愿意，可以阅读该脚本，或者可以用它来测试你对缩写的猜测 —— 例如： 

% sh config.sub i386 - linux
i386 - pc - linux - gnu
% sh config.sub alpha - linux
alpha - unknown - linux - gnu
% sh config.sub hp9k700
hppa1.1 - hp - hpux
% sh config.sub sun4
sparc - sun - sunos4.1.1
% sh config.sub sun3
m68k - sun - sunos4.1.1
% sh config.sub i986v
无效的配置 `i986v'：未识别的机器 `i986v'

config.sub 也分发在 GDB 源代码目录（gdb - 17.2, 版本17.2）。
