10.23 搜索内存 ¶

可以使用以下命令在内存中搜索特定的字节序列 find 命令。

find [/sn] start_addr, +len, val1 [, val2, …] ¶
find [/sn] start_addr, end_addr, val1 [, val2, …]

在内存中搜索由以下指定的字节序列 值1, 值2, 等等。搜索从地址 起始地址 开始，持续搜索 长度 个字节，或者一直搜索到 结束地址 （包含该地址）。

s 和 n 是可选参数。 它们可以按任意顺序指定，可以分开指定，也可以一起指定。

s, 搜索查询大小

每个搜索查询值的大小。

b

字节

h

半字（两个字节）

w

字（四个字节）

g

巨字（八个字节）

所有值都按照当前语言进行解释。 这意味着，例如，如果当前源语言是C/C++ 那么搜索字符串 “hello” 时会包含结尾的 ’\0’。 可以通过使用类型转换来去除搜索中的空终止符，例如： ‘{char[5]}"hello"’。

如果未指定值的大小，则从当前语言中值的类型获取。 当需要将搜索模式指定为不同类型的混合时，这很有用。 请注意，这意味着，例如，在类似C的语言中，搜索无类型的0x42将搜索 ‘(int) 0x42’ 通常是四个字节。

n, 最大查找次数

要打印的最大匹配次数。默认是打印所有查找结果。

可以使用字符串作为搜索值。用双引号将它们引起来（")。 字符串值会逐字节复制到搜索模式中，无论目标的字节序和大小规范如何。

会打印找到的每个匹配项的地址以及找到的匹配项数量。

最后找到的值的地址存储在便利变量 ‘$_’ 中。 匹配项的数量存储在 ‘$numfound’ 中。

例如，如果在以下位置停止 printf 在这个函数中： 

void
hello ()
{
  static char hello[] = "hello-hello";
  static struct { char c; short s; int i; }
    __attribute__ ((packed)) mixed
    = { 'c', 0x1234, 0x87654321 };
  printf ("%s\n", hello);
}

在调试期间会得到以下结果： 

(gdb) find &hello[0], +sizeof(hello), "hello"
0x804956d <hello.1620+6>
找到1个匹配项
(gdb) find &hello[0], +sizeof(hello), 'h', 'e', 'l', 'l', 'o'
0x8049567 <hello.1620>
0x804956d <hello.1620+6>
找到2个匹配项。
(gdb) find &hello[0], +sizeof(hello), {char[5]}"hello"
0x8049567 <hello.1620>
0x804956d <hello.1620+6>
找到2个匹配项。
(gdb) find /b1 &hello[0], +sizeof(hello), 'h', 0x65, 'l'
0x8049567 <hello.1620>
找到1个匹配项
(gdb) find &mixed, +sizeof(mixed), (char) 'c', (short) 0x1234, (int) 0x87654321
0x8049560 <mixed.1625>
找到1个匹配项
(gdb) print $numfound
$1 = 1
(gdb) print $_
$2 = (void *) 0x8049560
