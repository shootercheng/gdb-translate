19 指定调试目标 ¶

一个 目标 是您的程序所占用的执行环境。

通常情况下， GDB 与您的程序在同一主机环境中运行；在这种情况下，当您使用 file 或 core 命令时，调试目标会作为附带效果被指定。当您需要更多的灵活性时 —— 例如，在物理上独立的主机上运行 GDB ，或者通过串口控制独立系统，或者通过TCP/IP连接控制实时系统 —— 您可以使用 target 命令来指定为 GDB 配置的目标类型之一（请参阅 管理目标的命令）。

可以为多个不同的 GDB 目标 架构构建 GDB 当以这种方式构建时，您可以使用 设置架构 命令选择可用的架构之一。

set architecture arch ¶

此命令将当前目标架构设置为 架构。 架构 的值可以是 "auto", 除了受支持的架构之一。

show architecture

显示当前目标架构。
