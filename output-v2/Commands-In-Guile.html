23.4.3.11 Guile 中的命令 ¶

你可以在 Guile 中实现新的 GDB CLI 命令。一个 CLI 命令对象是通过 make-command Guile 函数创建的，并通过 GDB Guile 函数添加到 register-command! 这种两步走的方法是为了将添加命令的副作用与 GDB 分开。 make-command.

不支持多行命令，即由多行组成并以 end

make-command #:invoke 调用] [#:command-class 命令类别] [#:completer-class 补全器类别] [#:prefix? 前缀？] [#:doc 文档字符串] ¶

参数 名称 是命令的名称。如果 名称 由多个单词组成，则会将初始单词作为前缀命令进行查找。在这种情况下，如果某个前缀命令不存在，则会抛出异常。

结果是表示该命令的 <gdb:command> 对象。在使用 GDB 注册该命令之前，该命令不可用。 register-command!.

其余参数是可选的。

参数 调用 是一个接受三个参数的过程： 自身, 参数 和 来自终端. 参数 自身 是表示该命令的 <gdb:command> 对象。参数 参数 是一个字符串，表示传递给命令的参数，去除了首尾的空白字符。参数 来自终端 是一个布尔标志，指定该命令是否应认为是由用户交互式调用的。如果此函数抛出异常，则会将其转换为 GDB error 调用。否则，返回值将被忽略。

对于非前缀命令， 调用 是必需的。对于前缀命令 调用 是可选的。只有需要处理未知子命令的前缀命令才应提供 调用.

对于不提供 调用, 如果前缀命令在没有子命令名称的情况下使用，则 GDB 将显示每个子命令的帮助文本，除非前缀命令是一个 显示 子命令，在这种情况下 GDB 将列出所有子命令的值。

参数 命令类别 是下面定义的‘COMMAND_’常量之一。此参数告诉 GDB 如何在帮助系统中对新命令进行分类。默认值是 COMMAND_NONE.

参数 补全器 要么是 #f, 下面定义的‘COMPLETE_’常量之一，要么是下面也定义的一个过程。此参数告诉 GDB 如何为此命令执行补全。如果未提供或值为 #f, 则不对该命令执行补全。

参数 前缀 是一个布尔标志，指示新命令是否为前缀命令；可以注册此命令的子命令。

参数 文档字符串 是新命令的帮助文本。如果未提供文档字符串，则使用默认值“此命令未提供文档。”。

Scheme 过程： register-command! 命令 ¶

添加 命令, 一个 <gdb:command> 对象，到 GDB的命令列表中。多次注册同一个命令会导致错误。结果未指定。

Scheme 过程： command? 对象 ¶

返回 #t 如果 对象 是一个 <gdb:command> 对象。否则返回 #f.

Scheme 过程： dont-repeat ¶

默认情况下，当用户在命令提示符处输入空行时，一个 GDB 命令会被重复执行。命令可以通过调用 dont-repeat 函数来抑制这种行为。这类似于用户命令 dont-repeat, 请参阅 dont-repeat.

Scheme 过程： string->argv 字符串 ¶

根据 GDB的参数解析规则将一个字符串转换为字符串列表。建议使用此方法以保持一致性。参数由空格分隔，并且可以加引号。示例： 

scheme@(guile-user)> (string->argv "1 2 \"3 '4 \"5' \"6 '7\"")
$1 = ("1" "2 \"3" "4 \"5" "6 '7")
Scheme 过程： throw-user-error 消息 . 参数 ¶

抛出一个 gdb:user-error 异常。参数 消息 是作为格式字符串的错误消息，类似于 fmt 参数传递给 format Scheme 函数。请参阅 格式化输出 在 GNU Guile 参考手册. 中创建的。参数 参数 是 消息的可选参数列表。

当命令检测到某种用户错误时，例如错误的命令参数，会使用此功能。 

(gdb) guile (use-modules (gdb))
(gdb) guile
(register-command! (make-command "test-user-error"
  #:command-class COMMAND_OBSCURE
  #:invoke (lambda (self arg from-tty)
    (throw-user-error "Bad argument ~a" arg))))
end
(gdb) test-user-error ugh
ERROR: Bad argument ugh
补全器： 自身 文本 单词 ¶

如果 补全器 选项传递给 make-command 的是一个过程，它接受三个参数： 自身 这是 <gdb:command> 对象，以及 文本 和 单词 它们都是字符串。参数 文本 包含直到光标位置的完整命令行。参数 单词 包含命令行的最后一个单词；这是使用分词启发式算法计算得出的。

所有形式的补全都由这个函数处理，即 TAB 和 M-? 键绑定（请参阅 命令补全）以及 complete 命令（请参阅 complete) 分开。

这个过程可以返回几种类型的值：

  • 如果返回值是一个列表，则列表的内容将用作补全项。由 补全器 确保内容实际上能补全单词。允许为空列表，这意味着没有可用的补全项。列表中只有字符串元素会被使用；列表中的其他元素将被忽略。
  • 如果返回值是一个 <gdb:iterator> 对象，则会对其进行迭代以获取补全项。由 completer-procedure 确保结果实际上能补全单词。结果中只有字符串元素会被使用；序列中的其他元素将被忽略。
  • 所有其他结果都被视为没有可用的补全项。

当注册一个新命令时，它会被声明为某个通用命令类的成员。这用于在在线帮助系统中对顶级命令进行分类；请注意，前缀命令不会在其自身的类别下列出，而是在其顶级命令的类别下列出。可用的分类由 gdb 模块中定义的常量表示：

COMMAND_NONE ¶

该命令不属于任何特定类别。此类别中的命令不会在任何帮助类别中显示。这是默认设置。

COMMAND_RUNNING ¶

该命令与运行被调试程序有关。例如， start, step, 和 continue 属于此类别。在 help running 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_DATA ¶

该命令与数据或变量有关。例如， call, find, 和 print 属于此类别。在 help data 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_STACK ¶

该命令与栈的操作有关。例如， backtrace, frame, 和 return 属于此类别。在 help stack 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_FILES ¶

此类用于与文件相关的命令。例如， file, list 和 section 属于此类别。在 help files 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_SUPPORT ¶

这应该用于“支持工具”，通常指的是用户与 GDB交互时有用但与被调试程序的状态无关的事物。例如， help, make, 和 shell 属于此类别。在 help support 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_STATUS ¶

该命令是一个与‘info’相关的命令，即与 GDB 自身的状态有关。例如， info, macro, 和 show 属于此类别。在 help status 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_BREAKPOINTS ¶

该命令与断点有关。例如， break, clear, 和 delete 属于此类别。在 help breakpoints 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_TRACEPOINTS ¶

该命令与跟踪点有关。例如， trace, actions, 和 tfind 属于此类别。在 help tracepoints 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_USER ¶

该命令是一个通用的用户命令，通常不属于其他类别。在 help user-defined 命令提示符处输入 GDB 以查看此类别中的命令列表，以及 gdb 宏列表（请参阅 预定义命令序列)。

COMMAND_OBSCURE ¶

该命令仅在不寻常的情况下使用，或者对用户没有普遍的吸引力。例如， checkpoint, fork, 和 stop 属于此类别。在 help obscure 命令提示符处输入 GDB 以查看此类别中的命令列表。

COMMAND_MAINTENANCE ¶

该命令仅对 GDB 维护人员有用。 maintenance 和 flushregs 命令属于此类别。在 help internals 命令提示符处输入 GDB 以查看此类别中的命令列表。

一个新命令可以使用预定义的补全函数，要么在初始化时通过参数指定，要么从 completer 过程中返回。这些预定义的补全常量都在 gdb 模块中定义：

COMPLETE_NONE ¶

此常量表示不进行补全。

COMPLETE_FILENAME ¶

此常量表示应执行文件名补全。

COMPLETE_LOCATION ¶

此常量表示应执行位置补全。请参阅 位置规范.

COMPLETE_COMMAND ¶

此常量表示补全应检查 GDB 命令名称。

COMPLETE_SYMBOL ¶

此常量表示应使用符号名称作为源进行补全。

COMPLETE_EXPRESSION ¶

此常量表示应在表达式上进行补全。通常这意味着在符号名称上进行补全，但某些语言解析器也支持在字段名称上进行补全。

以下代码片段展示了如何在 Guile 中实现一个简单的 CLI 命令： 

(gdb) guile
(register-command! (make-command "hello-world"
  #:command-class COMMAND_USER
  #:doc "向整个世界问好。"
  #:invoke (lambda (self args from-tty) (display "Hello, World!\n"))))
end
(gdb) hello-world
Hello, World!
