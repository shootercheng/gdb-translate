9.2.2 显式位置 ¶

显式位置 允许用户使用选项 - 值对直接指定源位置的参数。

当程序源中有多个函数、标签或文件名（文件的基名）相同时，显式位置很有用。在这些情况下，显式位置能更准确、明确地指向你所指的源行。此外，在大型程序中使用显式位置可能会更快。

例如，行规范 ‘foo:bar’ 可能指的是一个函数 bar 在名为 foo 的文件中定义的，或者是一个标签 bar 在名为 foo. GDB 的函数中。必须搜索文件系统或符号表才能确定。

有效的显式位置选项列表总结如下表：

-source filename

该值指定源文件名。为了区分具有相同基名的文件，需要在前面添加足够多的目录，以唯一标识所需的文件，例如 foo/bar/baz.c. 否则 GDB 将使用它找到的第一个具有给定基名的文件。此选项需要使用 -function 或 -line.

-function function

该值指定函数的名称。对未被其他选项（如 -label 或 -line) 修改的函数位置进行的操作，指的是函数体开始的行。例如，在 C 语言中，这是带有左花括号的行。

默认情况下，在 C 语言中++ 和 Ada 语言中 函数 被解释为指定所有作用域中名为 函数 的所有函数。对于 C 语言++, 这意味着在所有命名空间和类中。对于 Ada 语言，这意味着在所有包中。

例如，假设一个 C 语言程序中有符号++ 和 A::B::func , 两个命令 B::func break -function func 和 break -function B::func 都会在这两个符号上设置断点。

你可以使用 -qualified 标志来覆盖此行为（见下文）。

-qualified

此标志使 GDB 将使用 -function 指定的函数名解释为完整的完全限定名。

例如，假设一个 C 语言程序中有符号++ 和 A::B::func , B::func break -qualified -function B::func 命令仅在 B::func, 上设置断点。

（注意： -qualified 选项也可以放在行规范之前（见 行规范位置），因此上面的特定示例可以简化为 break -qualified B::func.)

-label label

该值指定标签的名称。当未指定函数名时，会在当前选定栈帧的函数中搜索该标签。

-line number

该值指定位置的行偏移量。偏移量可以是绝对的（-line 3) 或相对的（-line +3)，具体取决于命令。当不指定任何其他选项时，行偏移量相对于当前行。

显式位置选项可以通过省略选项名中任何非唯一的尾部字符来缩写，例如 break -s main.c -li 3.
