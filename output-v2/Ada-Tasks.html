15.4.10.7 Ada任务扩展

调试多线程程序) GDB 提供了以下与任务相关的命令：

info tasks ¶

此命令显示当前Ada任务的列表，如下例所示： 

(gdb) info tasks
  ID       TID P-ID Pri State                 Name
   1   8088000   0   15 Child Activation Wait main_task
   2   80a4000   1   15 Accept Statement      b
   3   809a800   1   15 Child Activation Wait a
*  4   80ae800   3   15 Runnable              c

在此列表中，最后一个任务前的星号表示它是当前正在检查的任务。

ID

表示 GDB的内部任务编号。

TID

Ada任务ID。

P-ID

父任务ID（GDB的内部任务编号）。

优先级

任务的基本优先级。

状态

任务的当前状态。

Unactivated

任务已创建但尚未激活。它不能正在执行。

Runnable

任务没有因Ada已知的任何原因而阻塞。（不过，它可能正在等待互斥锁。）它在概念上以正常模式“执行”。

Terminated

任务已终止，符合ARM 9.3 (5)的定义。任何等待终止选择的依赖项都已被唤醒并自行终止。

Child Activation Wait

任务正在等待已创建的任务完成激活。

Accept or Select Term

任务正在等待接受或选择性等待语句。

Waiting on entry call

任务正在等待入口调用。

Async Select Wait

任务正在等待启动异步选择语句的可中止部分。

Delay Sleep

任务正在等待只有延迟选择项开放的选择语句。

Child Termination Wait

任务在完成自身内的主任务后正在休眠，并正在等待依赖于该主任务的任务终止或等待终止阶段。

Wait Child in Term Alt

任务正在休眠，等待终止选择上的任务完成终止。

Asynchronous Hold

任务已被 Ada.Asynchronous_Task_Control.Hold_Task持有。

Activating

任务已创建并正在变为可运行状态。

Selective Wait

任务正在选择性等待语句中等待。

Accepting RV with taskno

任务正在与任务 任务编号.

Waiting on RV with taskno

任务正在等待与任务 任务编号进行会合。

名称

程序中任务的名称。

info task taskno ¶

此命令显示指定任务的详细信息，如下例所示： 

(gdb) info tasks
  ID       TID P-ID Pri State                  Name
   1   8077880    0  15 Child Activation Wait  main_task
*  2   807c468    1  15 Runnable               task_1
(gdb) info task 2
Ada Task: 0x807c468
Name: "task_1"
Thread: 0
LWP: 0x1fac
Parent: 1 ("main_task")
Base Priority: 15
State: Runnable
task ¶

此命令打印当前任务的ID和名称。 

(gdb) info tasks
  ID       TID P-ID Pri State                  Name
   1   8077870    0  15 Child Activation Wait  main_task
*  2   807c458    1  15 Runnable               some_task
(gdb) task
[Current task is 2 "some_task"]
task taskno ¶

此命令类似于 thread thread-id 命令（见 调试多线程程序）。它将调试上下文从当前任务切换到给定任务。 

(gdb) info tasks
  ID       TID P-ID Pri State                  Name
   1   8077870    0  15 Child Activation Wait  main_task
*  2   807c458    1  15 Runnable               some_task
(gdb) task 1
[Switching to task 1 "main_task"]
#0  0x8067726 in pthread_cond_wait ()
(gdb) bt
#0  0x8067726 in pthread_cond_wait ()
#1  0x8056714 in system.os_interface.pthread_cond_wait ()
#2  0x805cb63 in system.task_primitives.operations.sleep ()
#3  0x806153e in system.tasking.stages.activate_tasks ()
#4  0x804aacc in un () at un.adb:5
task apply [task-id-list | all] [flag]… command

task apply 命令是Ada任务处理中与 thread apply （见 调试多线程程序）类似的命令。它允许你将命名的 命令 应用于一个或多个任务。使用任务ID列表指定你希望受影响的任务，或者指定 all 以应用于所有任务。

标志 参数控制在将 命令 应用于任务时产生什么输出以及如何处理引发的错误。 标志 必须以 - 开头，紧接着是 qcs中的一个字母。如果提供了多个标志，它们必须单独给出，例如 -c -q。

默认情况下， GDB 在 命令产生的输出之前显示一些任务信息，并且在执行 命令 期间引发的错误将中止 task apply。可以使用以下标志来微调此行为：

-c

标志 -c，代表‘继续’，会使 命令 中的任何错误被显示，然后 task apply 的执行继续。

-s

标志 -s，代表‘静默’，会使任何由 命令 产生的错误或空输出被静默忽略。也就是说，执行会继续，但任务信息和错误不会被打印。

-q

标志 -q （‘安静’）会禁用任务信息的打印。

标志 -c 和 -s 不能一起使用。

break locspec task taskno ¶
break locspec task taskno if …

这些命令类似于 break … thread … 命令（见 停止和启动多线程程序）。见 位置规范，了解 位置规范的各种形式。

使用限定符‘任务 任务编号’与断点命令一起使用，以指定你只想 GDB 在特定的Ada任务到达此断点时停止程序。 任务编号 是由 GDB分配的数字任务标识符之一，显示在‘info tasks’显示的第一列中。

如果你在设置断点时不指定‘任务 任务编号’，则该断点适用于 所有 程序任务。

你也可以在条件断点上使用 task 限定符；在这种情况下，将‘任务 任务编号’放在 if之前。

例如， 

(gdb) info tasks
  ID       TID P-ID Pri State                 Name
   1 140022020   0   15 Child Activation Wait main_task
   2 140045060   1   15 Accept/Select Wait    t2
   3 140044840   1   15 Runnable              t1
*  4 140056040   1   15 Runnable              t3
(gdb) b 15 task 2
Breakpoint 5 at 0x120044cb0: file test_task_debug.adb, line 15.
(gdb) cont
Continuing.
task # 1 running
task # 2 running

Breakpoint 5, test_task_debug () at test_task_debug.adb:15
15               flush;
(gdb) info tasks
  ID       TID P-ID Pri State                 Name
   1 140022020   0   15 Child Activation Wait main_task
*  2 140045060   1   15 Runnable              t2
   3 140044840   1   15 Runnable              t1
   4 140056040   1   15 Delay Sleep           t3
