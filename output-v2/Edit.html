9.3编辑源文件 ¶

要编辑源文件中的行，请使用 edit 命令。 将调用您选择的编辑程序，当前行设置为 程序中的活动行。 或者，如果您想查看程序的其他部分，有几种方法可以指定要打印文件的哪一部分：

edit locspec

编辑通过解析得到的代码位置的源文件 locspec。编辑从源文件和源行开始 locspec 解析到的位置。 请参阅 位置规范，了解 locspec 参数的所有可能形式。

如果 locspec 在您的程序中解析为多个源行，那么该命令将打印解析后的源行列表，并且 不会继续进行编辑。

以下是最常用的 edit 命令形式：

edit number

使用 编号 作为活动行号编辑当前源文件。

edit function

编辑包含 函数 定义开头的文件。

9.3.1选择您的编辑器 ¶

您可以自定义 GDB 以使用您想要的任何编辑器 10。 默认情况下，它是 /bin/ex，但您可以通过设置环境变量 EDITOR 在使用 GDB之前进行更改。例如，要配置 GDB 以使用 vi 编辑器，您可以在 sh shell中使用这些命令： 

EDITOR=/usr/bin/vi
 export EDITOR
gdb...

或者在 csh shell中 

setenv EDITOR /usr/bin/vi
gdb...

脚注

(10)

唯一的限制是您的编辑器（例如 ex）识别以下命令行语法： 

ex +编号 文件

可选的数值 +编号 指定在文件中开始编辑的行号。
