27.16 GDB/MI 数据操作 ¶

本节描述了 GDB/MI 用于操作数据的命令：检查内存和寄存器、计算表达式等。

有关可寻址内存单元的详细信息，请参阅 可寻址内存单元

该 -data-disassemble ¶

 -data-disassemble
  ( -s  -e 结束地址
  | -a 地址
  | -f 文件名 -l 行号 [ -n 行数   ]
  [ --source ]
  [ -- 模式 ]

‘’

$pc

结束地址

是要反汇编的函数内的任意地址（或函数名称）。如果指定了地址，将反汇编围绕该地址的整个函数。如果指定了名称，将反汇编具有该名称的整个函数。

行号’

是要围绕其进行反汇编的行号

‘行数’

是要生成的反汇编行数。如果为 -1，在没有指定 结束地址 的情况下，将反汇编整个函数。如果 结束地址 指定为非零值，并且 行数 小于 起始地址 和 结束地址之间的反汇编行数，仅显示 行数 行；如果 行数 大于 起始地址 和 结束地址之间的行数，仅显示到 结束地址 的行。

‘操作码模式’

只能与 模式 0 一起使用，并且应该是以下之一：

‘无’

结果中不包含操作码信息。

‘字节’

操作码将包含在结果中，操作码将按照以下格式进行格式化： 反汇编 /b。

‘显示’

操作码将包含在结果中，操作码将按照以下格式进行格式化： 反汇编 /r。

‘模式’

使用 模式 已被弃用，建议使用 --opcodes 和 --source 选项。当没有指定 模式 时， 模式 将假定为 0。但是， 模式 仍然可用于向后兼容。 模式 应该是以下之一：

‘0’

仅反汇编, 如果未指定模式，这是默认模式。

‘1’

混合源代码和反汇编（已弃用）, 无法使用 --opcodes 和 --source 选项重新创建此模式。

‘2’

带有原始操作码的反汇编, 此模式等同于使用 模式 0 并将 --opcodes bytes 传递给命令。

‘3’

带有原始操作码的混合源代码和反汇编（已弃用）, 无法使用 --opcodes 和 --source 选项重新创建此模式。

‘4’

混合源代码和反汇编, 此模式等同于使用 模式 0 并将 --source 传递给命令。

‘5’

带有原始操作码的混合源代码和反汇编, 此模式等同于使用 模式 0 并将 --opcodes bytes 和 --source 传递给命令。

模式 1 和 3 已被弃用。输出是“以源代码为中心”的，在实践中证明没有用处。请参阅 源代码和机器代码, 以讨论 /m 和 /s 命令输出之间的差异。 disassemble 命令。

该 --source 只能与 模式 0 一起使用。传递此选项将在反汇编结果中包含源代码，就好像使用了 模式 4 或 5 一样。

结果 ¶

该 -data-disassemble 命令的结果将是一个名为asm_insns的列表，此列表的内容取决于与 -data-disassemble 命令一起使用的选项。

对于模式 0 和 2，并且当未使用 --source 选项时，asm_insns列表包含具有以下字段的元组：

address

此指令被反汇编的地址。

func-name

此指令所在的函数名称。

offset

从func-name开始的字节十进制偏移量。

inst

此地址的文本反汇编。

opcodes

此字段仅在模式 2、3 和 5 中存在，或者当使用 --opcodes 选项字节或显示时存在。它包含inst字段的原始操作码字节。

当--opcodes选项未传递给 -data-disassemble, 或者字节值传递给--opcodes时，字节将格式化为一系列单字节，以十六进制表示，按地址升序排列，每个字节之间有一个空格。此格式等同于使用/b选项与 反汇编 命令一起使用（请参阅 反汇编）。

当--opcodes传递的值为显示时，字节将按自然指令显示顺序格式化。这意味着多个字节可以组合在一起，并且字节可能会进行字节交换。此格式等同于使用/r选项与 反汇编 命令一起使用。

对于模式 1、3、4 和 5，或者当使用 --source 选项时，asm_insns列表包含名为src_and_asm_line的元组，每个元组具有以下字段：

line

在文件中的行号。

file

编译单元中的文件名。根据使用的编译命令，这可能是绝对文件名或相对文件名。

fullname

文件的绝对文件名。它使用源文件搜索路径转换为规范形式（请参阅 指定源目录），并在解析所有符号链接之后。

如果未找到源文件，此字段将包含调试信息中存在的路径。

line_asm_insn

这是一个包含行在文件中的反汇编的元组列表。每个元组的字段与 -data-disassemble 在 模式 0 和 2 中相同，因此地址、func-name、偏移量、inst，以及可选的操作码。

请注意，inst字段中包含的任何内容都不会由 GDB/MI直接操作，即无法调整其格式。

该 -data-read-memory-bytes 命令 ¶

概要 ¶

 -data-read-memory-bytes [ -o 偏移量 ]
   地址 数量

其中：

‘地址’

一个表达式，指定要读取的第一个可寻址内存单元的地址。包含嵌入式空格的复杂表达式应使用 C 约定进行引用。

‘数量’

要读取的可寻址内存单元的数量。这应该是一个整数字面量。

‘偏移量’

相对于 地址 开始读取的偏移量。这应该是一个整数字面量。提供此选项是为了使前端不必先计算地址，然后再自行进行地址算术运算。

此命令尝试读取指定范围内的所有可访问内存区域。首先，内存映射中标记为不可读的所有区域（如果定义了）将被跳过。请参阅 内存区域属性。其次， GDB 将尝试读取其余区域。对于每个区域，如果读取整个区域导致错误， GDB 将尝试读取该区域的一个子集。

一般来说，区域中的每个单个内存单元可能可读也可能不可读，读取每个可读单元的唯一方法是在每个地址进行读取，这是不实际的。因此， GDB 将尝试使用二进制划分方案读取区域开头或结尾的所有可访问内存单元。这种启发式方法在跨内存映射边界读取时效果很好。请注意，如果一个区域有一个既不在开头也不在结尾的可读范围， GDB 将不会读取它。

该命令输出的结果记录（请参阅 GDB/MI 结果记录）包含一个名为内存的字段，其内容是一个元组列表。每个元组表示一个成功读取的内存块，具有以下字段：

begin

内存块的起始地址，以十六进制字面量表示。

end

内存块的结束地址，以十六进制字面量表示。

offset

内存块相对于传递给 -data-read-memory-bytes的起始地址的偏移量，以十六进制字面量表示。

contents

内存块的内容，以十六进制表示。

GDB 命令 ¶

对应的 GDB 命令是x。

示例 ¶

(gdb)
-data-read-memory-bytes &a 10

该 -data-write-memory-bytes 命令 ¶

概要 ¶

 -data-write-memory-bytes 地址 内容
 -data-write-memory-bytes 地址 内容 [数量]

其中：

‘地址’

一个表达式，指定要写入的第一个可寻址内存单元的地址。包含嵌入式空格的复杂表达式应使用 C 约定进行引用。

‘内容’

要写入的十六进制编码数据。如果 内容 不代表整数个可寻址内存单元，则为错误。

‘数量’

可选参数，指示要写入的可寻址内存单元的数量。如果 数量 大于 内容的长度， GDB 将重复写入 内容 直到填满 数量 个内存单元。

GDB 命令 ¶

没有对应的 GDB 命令。

示例 ¶

(gdb)
-data-write-memory-bytes &a "aabbccdd"
^done
(gdb)

(gdb)
-data-write-memory-bytes &a "aabbccdd" 16e
^done
(gdb)
