8.2 回溯 ¶

回溯是你的程序如何到达当前位置的摘要。它为许多帧显示每帧一行，从当前正在执行的帧（帧零）开始，接着是其调用者（帧一），以此类推到栈顶。

要打印整个栈的回溯，请使用 backtrace 命令，或其别名 bt. 此命令将为栈中的每个帧打印一行。默认情况下，将打印所有栈帧。你可以随时通过键入系统中断字符（通常是 Ctrl-c）来停止回溯。

backtrace [option]… [qualifier]… [count]
bt [option]… [qualifier]… [count]

打印整个栈的回溯。

可选的 计数 可以是以下之一：

n
n

仅打印最内层的 n 个帧，其中 n 是一个正整数。

-n
-n

仅打印最外层的 n 个帧，其中 n 是一个正整数。

选项：

-full

同时打印局部变量的值。这可以与可选的 计数 结合使用，以限制显示的帧数。

-no-filters

不在此回溯上运行Python帧过滤器。请参阅 过滤帧, 以获取更多信息。此外，使用 disable frame-filter all 来关闭所有帧过滤器。这仅在 GDB 已配置为支持 Python 时相关。

-hide

Python帧过滤器可能会决定“省略”某些帧。通常，这些被省略的帧仍然会被打印，但它们相对于导致它们被省略的过滤帧会有缩进。 -hide 选项会使被省略的帧完全不被打印。

backtrace 命令还支持一些选项，这些选项允许覆盖由 set backtrace 和 set print 子命令设置的相关全局打印设置：

-past-main [on|off]

设置回溯是否应继续越过 main. 相关设置： set backtrace past-main.

-past-entry [on|off]

设置回溯是否应继续越过程序的入口点。 相关设置： set backtrace past-entry.

-entry-values no|only|preferred|if-needed|both|compact|default

设置在函数入口处是否打印函数参数。 相关设置： set print entry-values.

-frame-arguments all|scalars|none

设置是否打印非标量帧参数。 相关设置： set print frame-arguments.

-raw-frame-arguments [on|off]

设置是否以原始形式打印帧参数。 相关设置： set print raw-frame-arguments.

-frame-info auto|source-line|location|source-and-location|location-and-address|short-location

设置是否打印帧信息。 相关设置： set print frame-info.

可选的 限定符 为了向后兼容而保留。 它可以是以下之一：

full

等同于 -full 选项。

no-filters

等同于 -no-filters 选项。

hide

等同于 -hide 选项。

名称 where 和 info stack （缩写为 info s) 是 backtrace的额外别名。

在多线程程序中， GDB 默认情况下仅显示当前线程的回溯。要显示多个或所有线程的回溯，请使用命令 thread apply (请参阅 thread apply)。例如，如果你键入 thread apply all backtrace, GDB 将显示所有线程的回溯；当你调试多线程程序的核心转储时，这很方便。

回溯中的每一行都显示帧号和函数名。程序计数器值也会显示——除非你使用 set print address off. 回溯还会显示源文件名和行号，以及函数的参数。如果程序计数器值位于该行代码的开头，则会省略该值。

以下是一个回溯的示例。它是使用命令 ‘bt 3’ 生成的，因此它显示了最内层的三个帧。 

#0  m4_traceon (obs=0x24eb0, argc=1, argv=0x2b8c8)
    在 builtin.c:993 处
#1  0x6e38 在 expand_macro (sym=0x2b600, data=...) 中，位于 macro.c:242
#2  0x6840 在 expand_token (obs=0x0, t=177664, td=0xf7fffb08)
    在 macro.c:71 处
(更多栈帧随后显示...

帧零的显示不以程序计数器值开头，这表明你的程序在第 993 行代码的开头停止了 builtin.c.

参数的值 data 在帧1中已被替换为 …. 默认情况下， GDB 仅当参数是标量（整数、指针、枚举等）时才打印参数的值。请参阅命令 set print frame-arguments 在 打印设置 中，以获取有关如何配置函数参数值打印方式的更多详细信息。 命令 set print frame-info (请参阅 打印设置) 控制 打印哪些帧信息。

如果你的程序是使用优化选项编译的，一些编译器会在函数调用后如果参数未被使用就将传递给函数的参数优化掉。这种优化会生成通过寄存器传递参数的代码，但不会将这些参数存储在栈帧中。 GDB 无法在除最内层栈帧之外的其他栈帧中显示此类参数。以下是这样一个回溯可能的样子： 

#0  m4_traceon (obs=0x24eb0, argc=1, argv=0x2b8c8)
    在 builtin.c:993 处
#1  0x6e38 在 expand_macro (sym=<优化掉>) 中，位于 macro.c:242
#2  0x6840 在 expand_token (obs=0x0, t=<优化掉>, td=0xf7fffb08)
    在 macro.c:71 处
(更多栈帧随后显示...

未保存在其栈帧中的参数值显示为 ‘<优化掉>’。

如果你需要显示这些被优化掉的参数的值，可以从依赖于你感兴趣的变量的其他变量中推断出来，或者不使用优化选项重新编译。

大多数程序都有一个标准的用户入口点——系统库和启动代码过渡到用户代码的地方。对于C语言，这是 main9. 当 GDB 在回溯中找到入口函数时，它将终止回溯，以避免跟踪到高度特定于系统（通常无趣）的代码。

如果你需要检查启动代码，或者限制回溯的层数，可以更改此行为：

set backtrace past-main
set backtrace past-main on

回溯将继续越过用户入口点。

set backtrace past-main off

回溯在遇到用户入口点时将停止。这是默认设置。

show backtrace past-main ¶

显示当前的用户入口点回溯策略。

set backtrace past-entry
set backtrace past-entry on

回溯将继续越过应用程序的内部入口点。 此入口点在应用程序构建时由链接器编码，并且可能在用户入口点之前 main （或等效操作）被调用。

set backtrace past-entry off

回溯在遇到应用程序的内部入口点时将停止。这是默认设置。

show backtrace past-entry

显示当前的内部入口点回溯策略。

set backtrace limit n
set backtrace limit 0
set backtrace limit unlimited

将回溯限制为 n 层。值为 unlimited 或零表示无限制的层数。

show backtrace limit

显示当前回溯层数的限制。

你可以控制文件名的显示方式。

set filename-display ¶
set filename-display relative

相对于编译目录显示文件名。这是默认设置。

set filename-display basename

仅显示文件名的基本名称。

set filename-display absolute

显示绝对文件名。

show filename-display

显示当前的文件名显示方式。

脚注

(9)

请注意，嵌入式程序（所谓的“独立”环境）不需要有一个 main 函数作为入口点。它们甚至可以有多个入口点。
