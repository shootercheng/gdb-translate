9.2.1 行规范位置 ¶

一个 行规范 是一个由冒号分隔的源位置参数列表， 如文件名、函数名等。以下是指定行规范的所有不同方式：

linenum

指定当前源文件的行号 行号 。

-offset
+offset

指定行 偏移量 当前行之前或之后的 行。 对于命令，当前行是最后打印的那一行；对于断点命令，这是在当前选定的 list 栈帧 中执行停止的行 （参见 帧, 了解栈帧的描述。） 当 用作 list 命令中两个行规范中的第二个时， 这指定了从第一个 行规范向上或向下的 偏移量 行。

filename:linenum

指定源文件中的行 行号 。 文件名 如果 文件名 是一个相对文件名，那么它将匹配任何具有相同尾部组件的源文件名。例如，如果 文件名 是 ‘gcc/expr.c’，那么它将匹配源文件名 /build/trunk/gcc/expr.c, 但不匹配 /build/trunk/libcpp/expr.c 或 /build/trunk/gcc/x-expr.c。

function

指定函数体开始的行 函数. 例如，在 C 语言中，这是带有左花括号的行。

默认情况下，在 C 语言++ 和 Ada 语言中， 函数 被解释为指定所有作用域中名为 函数 的所有函数。对于 C 语言++, 这意味着在所有命名空间和类中。对于 Ada 语言，这意味着在所有包中。

例如，假设一个 C 语言程序中有++ 符号名为 A::B::func 和 B::func, 两个命令 break func 和 break B::func 都会在这两个符号上设置断点。

接受行规范的命令允许你使用 -qualified 选项来覆盖此行为。例如， break -qualified func 在名为 func 的自由函数上设置断点， 忽略任何 C 语言++ 类方法和命名空间函数调用 func。

参见 显式位置。

function:label

指定 标签 出现在 函数中的行。

filename:function

指定文件中函数体开始的行 函数 。 文件名. 当不同源文件中有同名函数时，你只需要文件名和函数名来避免歧义。

label

指定名为 标签 的标签出现在与当前选定栈帧对应的函数中的行。 如果没有当前选定的栈帧（例如，如果子进程未运行），那么 GDB 将不会搜索标签。

-pstap|-probe-stap [objfile:[provider:]]name ¶

该 GNU/Linux 工具 SystemTap 为应用程序提供了一种嵌入静态探测点的方法。参见 静态探测点, 了解有关查找和使用静态探测点的更多信息。这种形式的行规范指定了这样一个静态探测点的位置。

如果 目标文件 被指定，只考虑来自与 目标文件匹配的共享库或可执行文件的探测点 作为正则表达式进行考虑。 如果 提供者 被指定，那么只考虑来自该提供者的探测点。 如果有多个探测点与规范匹配， GDB 将在每个这样的探测点处插入一个断点。
