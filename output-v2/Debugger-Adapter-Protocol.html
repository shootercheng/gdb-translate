29 调试器适配器协议 ¶

调试器适配器协议是一种通用的API，一些集成开发环境（IDE）使用它与调试器进行通信。其文档位于 https://microsoft.github.io/debug-adapter-protocol/.

通常情况下， GDB 按照规定实现调试器适配器协议。然而，在某些情况下，可能需要甚至期望进行扩展。

GDB 定义了一些可以传递给 launch 请求的参数：

args

如果提供，这应该是一个字符串数组。这些字符串将作为命令行参数传递给被调试程序，就好像通过 set args. 请参阅 您程序的参数.

cwd

如果提供，这应该是一个字符串。 GDB 将把它的工作目录更改为这个目录，就好像通过 cd 命令（请参阅 您程序的工作目录)。启动的程序将继承这个目录作为其工作目录。请注意，目录更改发生在 program 参数处理之前。如果 program 是相对文件名，这将影响结果。

env

如果提供，这应该是一个对象。对象的每个键将用作环境变量的名称；每个值必须是一个字符串，将作为该变量的值。被调试程序的环境将被设置为完全按照传入的内容。请参阅 您程序的环境.

program

如果提供，这是一个指定要使用的程序的字符串。这对应于 file 命令。请参阅 指定文件的命令.

stopAtBeginningOfMainSubprogram

如果提供，这必须是一个布尔值。当 ‘真’ 时， GDB 将在程序的主过程处设置一个临时断点，使用与 start 命令相同的方法。请参阅 启动您的程序.

stopOnEntry

如果提供，这必须是一个布尔值。当 ‘真’ 时， GDB 将在程序的第一条指令处设置一个临时断点，使用与 starti 命令相同的方法。请参阅 启动您的程序.

GDB 定义了一些可以传递给 attach 请求的参数。必须指定 pid 或 target ，但如果两者都指定，则 target 将被忽略。

pid

要让 GDB 附加到的进程ID。请参阅 调试已经运行的进程.

program

如果提供，这是一个指定要使用的程序的字符串。这对应于 file 命令。请参阅 指定文件的命令. 在某些情况下， GDB 可以自动确定正在运行的程序。然而，对于许多远程目标，情况并非如此，因此应该提供此信息。

target

要让 GDB 连接到的目标。这是一个字符串，并传递给 target remote 命令。请参阅 连接到远程目标.

作为对 disassemble 请求的响应，DAP允许客户端以实现定义的格式返回每条指令的字节。 GDB 通过发送一个以十六进制编码字节的字符串来实现这一点，例如 "55a2b900".

当 repl 上下文用于 evaluate 请求时， GDB 将提供的表达式作为CLI命令进行计算。

一般来说，计算可能会导致被调试程序继续执行。例如，计算 continue 命令可能会这样做，计算涉及被调试程序函数调用的表达式也可能会这样做。

repl 计算也可能导致 GDB 看起来停止响应请求，例如，如果一个CLI脚本进行了长时间的计算。

可以使用DAP cancel 请求来中断这样的计算。（实际上， cancel 应该对任何请求都有效，但对于大多数请求来说可能用处不大。）

GDB 提供了一些可以在DAP模式下使用的日志记录设置。可以在命令行上使用 -iex 选项来设置这些设置（请参阅 选择文件).

set debug dap-log-file [filename]

启用DAP日志记录。日志将写入 文件名. 如果没有提供 文件名 ，则停止日志记录。

set debug dap-log-level level

设置DAP日志记录级别。默认值是 ‘1’，它会记录DAP协议、开发人员认为有用的任何调试消息以及意外异常。级别 ‘2’ 可用于记录所有异常，包括那些被认为是预期的异常。例如，解析表达式失败将被视为正常异常，通常不会被记录。
