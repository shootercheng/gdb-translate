C.2 调用 GDB 配置 脚本 ¶

GDB 附带一个 配置 脚本，它可以自动完成准备过程 以便进行安装；然后您可以使用 GDB 来 构建 make 程序。 gdb 程序。

该 GDB 发行版包含您所需的所有源代码，用于 GDB 在单个目录中，其名称通常是通过在‘gdb’后面追加版本号组成的。

例如， GDB 17.2版本的发行版位于 gdb - 17.2 目录中。该目录包含：

gdb-17.2/configure (and supporting files)

用于配置的脚本 GDB 及其所有支持库

gdb-17.2/gdb

特定于 GDB 本身的源代码

gdb-17.2/bfd

二进制文件描述符库的源代码

gdb-17.2/include

GNU 头文件

gdb-17.2/libiberty

‘-liberty’自由软件库的源代码

gdb-17.2/opcodes

操作码表和反汇编器库的源代码

gdb-17.2/readline

的源代码 GNU 命令行界面

可能还有其他子目录。

配置和构建 GDB 的最简单方法是运行 配置 从 gdb -版本号 源代码目录，在 这个例子中是 gdb - 17.2 目录。

首先切换到 gdb -版本号 源代码目录 如果您还不在其中；然后运行 配置. 传递 要运行的平台的标识符作为 参数。 GDB 将运行的平台的标识符作为参数。

例如： 

cd gdb - 17.2
./configure
make

运行‘配置’然后运行 make 会构建 包含的支持库，然后是 gdb 本身。配置后的 源文件和二进制文件会留在相应的源 目录中。

配置 是一个Bourne shell（/bin/sh)脚本；如果您的 系统在您运行不同的 shell时不能自动识别它，您可能需要显式地运行 sh 在它上面： 

sh configure

您应该从源 树的顶级目录运行 配置 脚本，即 gdb -版本号 目录。如果您运行 配置 从其中一个子目录运行，您将只配置 该子目录。这通常不是您想要的。特别是， 如果您第一次运行 配置 从 gdb 子目录 在 gdb -版本号 目录中，您将省略对 bfd, readline, 以及 gdb 子目录的其他同级 目录的配置。这会导致构建错误，因为缺少头文件，例如 bfd/bfd.h.

您可以将 GDB 安装在任何地方。最好的方法是将 --prefix 选项传递给 configure, 然后 使用它进行安装 make install.
