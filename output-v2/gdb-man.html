gdb [选项] [程序|程序 进程 ID| 核心文件

像 GDB 这样的调试器的目的是让你在另一个程序执行时查看其“内部”发生了什么——或者在另一个程序崩溃时查看它当时正在做什么。

GDB

  • 启动你的程序，并指定任何可能影响其行为的内容。
  • 让你的程序在指定条件下停止。
  • 当你的程序停止时，检查发生了什么。
  • 更改你程序中的内容，这样你就可以尝试纠正一个错误的影响，并继续了解另一个错误。

你可以使用 GDB 来调试用 C、C++, Fortran 和 Modula - 2 编写的程序。

GDB 通过 shell 命令调用 gdb。一旦启动，它会从终端读取命令，直到你使用 GDB 命令告诉它退出。 quit 或 exit. 你可以通过使用命令 GDB 从 help本身获得在线帮助。

你可以不带任何参数或选项运行 gdb ；但启动 GDB 最常见的方式是使用一个或两个参数，指定一个可执行程序作为参数： 

gdb 程序

你也可以同时指定一个可执行程序和一个核心文件来启动： 

gdb 程序 核心文件

相反，如果你想调试一个正在运行的进程，你可以将进程 ID 作为第二个参数指定，或者使用选项 -p, 例如： 

gdb 程序 1234
gdb -p 1234

将附加 GDB 到进程 1234。使用选项 -p 你可以省略 程序 文件名。

以下是一些最常用的 GDB 命令：

break [file:][function|line]

在 函数 或 行 （在 文件中）设置断点。

run [arglist]

启动你的程序（如果指定了 参数列表）。

bt

回溯：显示程序堆栈。

print expr

显示表达式的值。

c

继续运行你的程序（在停止后，例如在断点处）。

next

执行下一行程序（在停止后）；单步 跳过 该行中的任何函数调用。

edit [file:]function

查看程序当前停止的行。

list [file:]function

输入程序当前停止位置附近的文本。

step

执行下一行程序（在停止后）；单步 进入 该行中的任何函数调用。

help [name]

显示有关 GDB 命令 名称的信息，或有关使用 GDB的一般信息。

quit
exit

从 GDB退出。

除选项外的任何参数指定一个可执行文件和核心文件（或进程 ID）；也就是说，遇到的第一个没有关联选项标志的参数等同于一个 --se 选项，第二个（如果有）如果是文件名则等同于一个 -c 选项。许多选项都有长形式和缩写形式；这里都显示了。如果长形式被截断，只要保留的部分足以明确识别，也会被识别。

这里缩写形式用‘-’表示，长形式用‘--’表示，以反映它们在 --help中的显示方式。然而， GDB 对于大多数选项，识别以下所有约定：

--option=value
--option value
-option=value
-option value
--o=value
--o value
-o=value
-o value

你给出的所有选项和命令行参数按顺序处理。当使用 -x 选项时，顺序会有影响。

--args prog [arglist]

更改命令行的解释，以便此选项之后的参数作为参数传递给 inferior。例如，考虑以下命令：

它将启动 GDB 并带有 -q, 不打印介绍信息。另一方面，使用：

--fullname
-f

Emacs 在将 GDB 作为子进程运行时设置此选项。它告诉 GDB 每次显示堆栈帧时（包括每次程序停止时）以标准、可识别的方式输出完整的文件名和行号。这种可识别的格式看起来像两个‘\032’字符，后面跟着用冒号分隔的文件名、行号和字符位置，以及一个换行符。Emacs 到GDB 的接口程序使用两个‘\032’字符作为显示该帧源代码的信号。
