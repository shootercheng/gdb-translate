27.14 GDB/MI 栈操作命令 ¶

该 -enable-frame-filters 命令 ¶

GDB 允许基于 Python 的帧过滤器影响与栈跟踪相关的 MI 命令的输出。由于无法以完全向后兼容的方式实现此功能，前端必须请求启用此功能。

一旦启用，此功能无法禁用。

GDB, 此命令仍然会成功（但不执行任何操作）。

该 -stack-info-frame 命令 ¶

概要 ¶

 -stack-info-frame

获取所选帧的信息。

GDB 命令 ¶

对应的 GDB 命令是 ‘查看帧信息’ 或 ‘帧’（不带参数）。

示例 ¶

(gdb)
-stack-info-frame
^done,frame={level="1",addr="0x0001076c",func="callee3",
file="../../../devo/gdb/testsuite/gdb.mi/basics.c",
fullname="/home/foo/bar/devo/gdb/testsuite/gdb.mi/basics.c",line="17",
arch="i386:x86_64"}
(gdb)

该 -stack-info-depth 命令 ¶

概要 ¶

 -stack-info-depth [ 最大深度 ]

返回栈的深度。如果指定了整数参数 最大深度 ，则不会计算超过 最大深度 的帧数。

GDB 命令 ¶

没有等效的 GDB 命令。

示例 ¶

对于帧级别从 0 到 11 的栈： 

(gdb)
-stack-info-depth
^done,depth="12"
(gdb)
-stack-info-depth 4
^done,depth="4"
(gdb)
-stack-info-depth 12
^done,depth="12"
(gdb)
-stack-info-depth 11
^done,depth="11"
(gdb)
-stack-info-depth 13
^done,depth="12"
(gdb)

该 -stack-list-arguments 命令 ¶

概要 ¶

 -stack-list-arguments [ --no-frame-filters ] [ --skip-unavailable ] 打印值
    [ 低帧号 高帧号 ]

显示 低帧号 和 高帧号 之间（包括这两个帧）的帧的参数列表。如果 低帧号 和 高帧号 未提供，则列出整个调用栈的参数。如果这两个参数相等，则显示对应级别的单个帧的参数。如果 低帧号 大于实际的帧数，则会出错。另一方面， 高帧号 可以大于实际的帧数，在这种情况下，只会返回存在的帧。

如果 打印值 为 0 或 --no-values, 则只打印变量名；如果为 1 或 --all-values, 则还打印它们的值；如果为 2 或 --simple-values, 则打印简单数据类型的名称、类型和值，以及数组、结构体和联合体的名称和类型。如果提供了选项 --no-frame-filters ，则不会执行 Python 帧过滤器。

如果 --skip-unavailable 选项被指定，则不会列出不可用的参数。但是，部分可用的参数仍然会显示。

使用此命令获取单个帧的参数已被弃用，建议使用 ‘-stack-list-variables’ 命令。

GDB 命令 ¶

GDB 没有等效的命令。 gdbtk 有一个 ‘gdb_get_args’ 命令，其功能与 ‘-stack-list-arguments’ 部分重叠。

frame={level="2",addr="0x0001078c",func="callee2",
file="../../../devo/gdb/testsuite/gdb.mi/basics.c",
fullname="/home/foo/bar/devo/gdb/testsuite/gdb.mi/basics.c",line="22",
arch="i386:x86_64"},
frame={level="3",addr="0x000107b4",func="callee1",

该 -stack-list-frames 命令 ¶

概要 ¶

 -stack-list-frames [ --no-frame-filters 低帧号 高帧号 ]

列出当前栈上的帧。对于每个帧，它会显示以下信息：

‘级别’

帧编号，0 表示最顶层的帧，即最内层的函数。

‘地址’

该 $pc 帧的值。

‘函数名’

函数名称。

‘文件名’

函数所在源文件的名称。

‘完整文件名’

函数所在源文件的完整名称。

‘行号’

对应的行号 $pc.

‘来自’

定义此函数的共享库。只有在帧的函数未知时才会给出此信息。

‘架构’

帧的架构。

如果不带参数调用此命令，则会打印整个栈的回溯信息。如果给定两个整数参数，则会显示级别在这两个参数之间（包括这两个参数）的帧。如果这两个参数相等，则会显示对应级别的单个帧。如果 低帧号 大于实际的帧数，则会出错。另一方面， 高帧号 可以大于实际的帧数，在这种情况下，只会返回存在的帧。如果提供了选项 --no-frame-filters ，则不会执行 Python 帧过滤器。

该 -stack-list-locals 命令 ¶

概要 ¶

 -stack-list-locals [ --no-frame-filters ] [ --skip-unavailable ] 打印值

显示所选帧的局部变量名。如果 打印值 为 0 或 --no-values, 则只打印变量名；如果为 1 或 --all-values, 则还打印它们的值；如果为 2 或 --simple-values, 则打印简单数据类型的名称、类型和值，以及数组、结构体和联合体的名称和类型。在最后一种情况下，前端可以立即显示简单数据类型的值，并在用户希望更详细地探索其他数据类型的值时为其创建变量对象。如果提供了选项 --no-frame-filters ，则不会执行 Python 帧过滤器。

如果 --skip-unavailable 选项被指定，则不会列出不可用的局部变量。但是，部分可用的局部变量仍然会显示。

此命令已被弃用，建议使用 ‘-stack-list-variables’ 命令。

GDB 命令 ¶

‘查看局部变量’ 在 GDB, ‘gdb_get_locals’ 在 gdbtk.

示例 ¶

(gdb)
-stack-list-locals 0
^done,locals=[name="A",name="B",name="C"]
(gdb)
-stack-list-locals --all-values
^done,locals=[{name="A",value="1"},{name="B",value="2"},
  {name="C",value="{1, 2, 3}"}]
-stack-list-locals --simple-values
^done,locals=[{name="A",type="int",value="1"},
  {name="B",type="int",value="2"},{name="C",type="int [3]"}]
(gdb)

该 -stack-list-variables 命令 ¶

概要 ¶

 -stack-list-variables [ --no-frame-filters ] [ --skip-unavailable ] 打印值

显示所选帧的局部变量和函数参数的名称。如果 打印值 为 0 或 --no-values, 则只打印变量名；如果为 1 或 --all-values, 则还打印它们的值；如果为 2 或 --simple-values, 则打印简单数据类型的名称、类型和值，以及数组、结构体和联合体的名称和类型。如果提供了选项 --no-frame-filters ，则不会执行 Python 帧过滤器。

如果 --skip-unavailable 选项被指定，则不会列出不可用的局部变量和参数。但是，部分可用的参数和局部变量仍然会显示。

示例 ¶

(gdb)
-stack-list-variables --thread 1 --frame 0 --all-values
^done,variables=[{name="x",value="11"},{name="s",value="{a = 1, b = 2}"}]
(gdb)

该 -stack-select-frame 命令 ¶

概要 ¶

 -stack-select-frame 帧编号

更改所选帧。选择栈上的不同帧 帧编号 。

此命令已被弃用，建议在每个命令中传递 ‘--frame’ 选项。

GDB 命令 ¶

对应的 GDB 命令是 ‘帧’, ‘上移’, ‘下移’, ‘选择帧’, ‘上移（静默）’, 和 ‘下移（静默）’。

示例 ¶

(gdb)
-stack-select-frame 2
^done
(gdb)
