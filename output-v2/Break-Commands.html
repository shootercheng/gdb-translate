5.1.7 断点命令列表 ¶

当程序因某个断点而停止时，你可以为任何断点（或观察点、捕获点）指定一系列要执行的命令。例如，你可能想打印某些表达式的值，或者启用其他断点。

commands [list…] ¶
… command-list …
end

为给定的断点指定一个命令列表。命令本身出现在后续行中。输入仅包含 end 的行来结束命令。

要移除断点的所有命令，输入 commands 并紧接着输入 end；也就是说，不给出任何命令。

如果没有参数， commands 指的是最后设置的断点、观察点或捕获点（而不是最近遇到的断点）。如果最近的断点是用一个命令设置的，那么 commands 将应用于该命令设置的所有断点。这适用于由 rbreak设置的断点，并且当一个 break 命令创建多个断点时也适用（见 模糊表达式）。

按下 回车键 作为重复上一个 GDB 命令的方式在 命令列表中被禁用。

在命令列表中，你可以使用命令 禁用 $_hit_bpnum 来禁用遇到的断点。

如果你的断点有多个代码位置，命令 禁用 $_hit_bpnum.$_hit_locno 将禁用遇到的特定断点代码位置。如果断点只有一个位置，此命令将禁用遇到的断点。

你可以使用断点命令再次启动你的程序。只需使用 continue 命令，或者 step, 或者任何其他恢复执行的命令。

命令列表中，在恢复执行的命令之后的任何其他命令都将被忽略。这是因为任何时候你恢复执行（即使是简单的 next 或 step），你可能会遇到另一个断点 —— 它可能有自己的命令列表，这会导致关于执行哪个列表的歧义。

如果你在命令列表中指定的第一个命令是 silent, 则通常关于在断点处停止的消息不会被打印。对于那些要打印特定消息然后继续执行的断点，这可能是理想的。如果其余命令都不打印任何内容，你将看不到断点被触发的迹象。 silent 仅在断点命令列表的开头有意义。

命令 echo, output, 和 printf 允许你精确控制输出，并且在静默断点中通常很有用。见 控制输出的命令。

例如，下面展示了如何使用断点命令在每次 x 进入 foo 时打印 x 为正值时的值。 

在 foo 处设置断点，如果 x > 0
命令
静默
打印 "x 是 %d\n",x
继续
结束

断点命令的一个应用是弥补一个错误，以便你可以测试另一个错误。在错误代码行之后设置一个断点，给它一个条件来检测出现错误的情况，并给它命令来为需要的变量赋值。以 continue 命令结束，这样你的程序就不会停止，以 silent 命令开始，这样就不会产生输出。下面是一个例子： 

在 403 行设置断点
命令
静默
设置 x = y + 4
继续
结束
