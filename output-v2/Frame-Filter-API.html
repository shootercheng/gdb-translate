23.3.2.10 过滤帧 ¶

帧过滤器是 Python 对象，当打印回溯（参见 回溯）时，它们可以操纵一个或多个帧的可见性，由 GDB执行。

只有打印回溯的命令，或者在 GDB/MI 命令（参见 该 GDB/MI 接口）的情况下，那些返回帧集合的命令才会受到影响。与帧过滤器一起使用的命令有：

backtrace （参见 回溯命令）， -stack-list-frames （参见 -stack-list-frames 命令）， -stack-list-variables （参见 -stack-list-variables 命令）， -stack-list-arguments 参见 -stack-list-arguments 命令）和 -stack-list-locals （参见 -stack-list-locals 命令）。

帧过滤器的工作方式是将迭代器作为参数，对该迭代器的内容应用操作，然后返回另一个迭代器（或者，在过滤器不执行任何操作的情况下，可能返回提供的同一个迭代器）。通常，帧过滤器利用诸如 Python 的 itertools 模块之类的工具来处理源迭代器并从中创建新的迭代器。无论过滤器选择如何应用操作，它都不得更改底层 GDB 帧或多个帧，也不得尝试在 GDB内更改调用栈。这可以保持 GDB内的数据完整性。帧过滤器按优先级执行，应注意有些帧过滤器可能已经在之前执行过，而有些帧过滤器将在之后执行。

设计帧过滤器时一个重要的考虑因素，并且非常值得思考的是，帧过滤器应尽可能避免展开调用栈。有些栈可能非常深，在某些情况下可达数万层。在帧过滤器执行时搜索每个帧在该步骤可能代价过高。帧过滤器无法知道它必须迭代多少个帧，并且可能必须遍历所有帧。这最终会重复工作，因为 GDB 在打印帧时会执行此迭代。如果过滤器可以将展开帧的操作推迟到帧装饰器执行时，即在最后一个过滤器执行之后，那么它应该这样做。有关装饰器的更多信息，请参见 装饰帧。此外，后续章节中有帧装饰器和过滤器的示例。有关更多信息，请参见 编写帧过滤器。

Python 字典 gdb.frame_filters 包含构成帧过滤器的键/对象对。此字典中的帧过滤器称为 global 帧过滤器，并且在调试所有下级程序时都可用。这些帧过滤器必须直接在字典中注册。除了 global 字典之外，还有其他字典通过自动加载（参见 Python 自动加载）与不同的下级程序一起加载。可以找到帧过滤器字典的另外两个位置是： gdb.Progspace 它包含一个 frame_filters 字典属性，以及每个 gdb.Objfile 对象，它也包含一个 frame_filters 字典属性。

当从 GDB 执行与帧过滤器兼容的命令时， GDB 会合并 global, gdb.Progspace 和所有当前加载的 gdb.Objfile 字典。所有的 gdb.Objfile 字典都会合并，因为可能会使用多个帧，因此可能会使用多个目标文件。 GDB 然后会删除其 enabled 属性为 False的任何帧过滤器。然后根据每个过滤器中的 priority 属性对这个经过筛选的列表进行排序。

一旦字典合并、筛选和排序完成， GDB 会创建一个迭代器，该迭代器将调用栈中的每个帧包装在一个 FrameDecorator 对象中，并按顺序调用每个过滤器。前一个过滤器的输出将始终作为下一个过滤器的输入，依此类推。

帧过滤器有一个必需的接口，每个帧过滤器都必须实现该接口，定义如下：

函数： FrameFilter.filter (迭代器) ¶

GDB 当达到该过滤器在优先级列表中的顺序时，将在帧过滤器上调用此方法。

例如，如果有四个帧过滤器： 

名称         优先级

Filter1      5
Filter2      10
Filter3      100
Filter4      1

帧过滤器的调用顺序是： 

Filter3 -> Filter2 -> Filter1 -> Filter4

请注意， Filter3 的输出会传递给 Filter2的输入，依此类推。

此 filter 方法会传递一个 Python 迭代器。此迭代器包含一系列帧装饰器，这些装饰器包装每个 gdb.Frame, 或者一个帧装饰器包装另一个帧装饰器。在帧过滤器序列中执行的第一个过滤器将接收一个完全由默认 FrameDecorator 对象组成的迭代器。但是，在每个帧过滤器执行之后，前一个帧过滤器可能已经用自己的帧装饰器包装了部分或全部帧装饰器。由于帧装饰器也必须符合必需的接口，因此可以假定这些装饰器以统一的方式工作（参见 装饰帧）。

此方法必须返回一个符合 Python 迭代器协议的对象。迭代器中的每个项必须是一个符合帧装饰器接口的对象。如果帧过滤器不想对这个迭代器执行任何操作，它应该原封不动地返回该迭代器。

此方法不是可选的。如果它不存在， GDB 将引发并打印错误。

变量： FrameFilter.name ¶

该 name 属性必须是 Python 字符串，其中包含显示的过滤器名称 GDB （参见 帧过滤器的管理。）。此属性可以包含字母或数字的任意组合。应注意确保其唯一性。此属性是必需的。

变量： FrameFilter.enabled ¶

该 enabled 属性必须是 Python 布尔值。此属性向 GDB 指示帧过滤器是否启用，并且在执行帧过滤器时应予以考虑。如果 enabled 为 True，则在执行本章前面详细介绍的任何回溯命令时，将执行帧过滤器。如果 enabled 为 False，则不会执行帧过滤器。此属性是必需的。

变量： FrameFilter.priority ¶

该 priority 属性必须是 Python 整数。此属性控制与其他帧过滤器相关的执行顺序。对 priority 的范围没有强制限制，除了它必须是有效的整数。 priority 属性的值越高，与其他帧过滤器相比，该帧过滤器将越早执行。尽管 priority 可以为负数，但建议假定零是帧过滤器可以分配的最低优先级。具有相同优先级的帧过滤器在该优先级槽中以未排序的顺序执行。此属性是必需的。100 是一个不错的默认优先级。
