23.4.3.22 Guile 中的反汇编 ¶

可以从 Scheme 代码中调用反汇编器。此外，反汇编器可以将 Guile 端口作为输入，允许从任何源进行反汇编，而不仅仅是目标内存。

Scheme 过程： arch - disassemble arch start - pc [#:port 端口] [#:offset 偏移量] [#:size 大小] [#:count 数量] ¶

返回从内存地址开始的反汇编指令列表 起始程序计数器（start - pc）。

可选参数 端口 指定从中读取字节的输入端口。如果 端口 是 #f 则从目标内存中读取字节。

可选参数 偏移量 指定了 端口中第一个字节的地址偏移量。例如，当 端口 指定了一个‘字节向量’，并且你希望将该字节向量当作从该地址来的进行反汇编时，这很有用。传递给 起始程序计数器（start - pc） 的读取器的 端口 也会偏移相同的量。

示例： 

(gdb) guile (use - modules (rnrs io ports))
(gdb) guile (define pc (value -> integer (parse - and - eval "$pc")))
(gdb) guile (define mem (open - memory #:start pc))
(gdb) guile (define bv (get - bytevector - n mem 10))
(gdb) guile (define bv - port (open - bytevector - input - port bv))
(gdb) guile (define arch (current - arch))
(gdb) guile (arch - disassemble arch pc #:port bv - port #:offset pc)
(((address . 4195516) (asm . "mov    $0x4005c8,%edi") (length . 5)))

可选参数 大小 和 数量 确定返回列表中指令的数量。如果 大小 或 数量 被指定为零，则不进行反汇编，返回一个空列表。如果两个可选参数 大小 和 数量 都被指定，那么返回最多 数量 条反汇编指令的列表，这些指令的起始地址落在从 起始程序计数器（start - pc） 到（起始程序计数器（start - pc） + 大小 - 1）的闭内存地址区间内。如果 大小 未指定，但 数量 被指定，那么返回从地址 数量 起始程序计数器（start - pc） 条指令。如果 数量 未指定，但 大小 被指定，那么返回所有起始地址落在从 起始程序计数器（start - pc） 到（起始程序计数器（start - pc） + 大小 - 1）的闭内存地址区间内的指令。如果 大小 和 数量 都未指定，那么返回在 起始程序计数器（start - pc） 处的单条指令。

返回列表中的每个元素都是一个关联列表（关联表），包含以下键：

address

与此键对应的值是一个 Guile 整数，表示指令的内存地址。

asm

与此键对应的值是一个字符串，它用汇编语言助记符表示指令。所使用的汇编语言风格与当前 CLI 变量指定的风格相同。 disassembly-flavor参见 源代码和机器代码。

length

与此键对应的值是指令的字节长度。
