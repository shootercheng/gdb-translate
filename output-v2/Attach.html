4.7 调试一个已在运行的进程 ¶

attach process-id

此命令用于附加到一个正在运行的进程 —— 即一个在外部启动的进程 GDB. (info files 显示你当前的活动目标。) 该命令接受一个进程ID作为参数。通常查找 进程ID Unix进程的方法是使用 ps 实用工具，或者使用 ‘jobs -l’ shell命令。

attach 如果你在执行命令后第二次按下 回车键 ，该命令不会重复执行。

要使用 attach, 你的程序必须运行在支持进程的环境中；例如， attach 对于缺乏操作系统的裸机目标上的程序不起作用。你还必须有权限向该进程发送信号。

当你使用 attach, 调试器首先会在当前工作目录中查找该进程中运行的程序，然后（如果未找到该程序）会使用源文件搜索路径（见 指定源目录)。你也可以使用 file 命令来加载程序。请参阅 指定文件的命令.

如果调试器能够确定它所附加的进程中运行的可执行文件与 GDB当前加载的可执行文件不匹配，那么 exec-file-mismatch 选项指定了如何处理这种不匹配情况。 GDB 会尝试通过比较它们的构建ID（见 构建ID）来比较这些文件（如果可用）。

set exec-file-mismatch ‘ask|warn|off’ ¶

是否检测 GDB 当前加载的可执行文件与启动该进程所使用的可执行文件之间的不匹配。如果是 ‘询问’（默认值），则显示一条警告并询问用户是否加载该进程的可执行文件；如果是 ‘警告’，则仅显示一条警告；如果是 ‘关闭’，则不尝试检测不匹配情况。如果用户确认加载该进程的可执行文件，那么其符号也将被加载。

show exec-file-mismatch ¶

显示 exec-file-mismatch的当前值。

GDB 在安排调试指定进程后做的第一件事是停止该进程。你可以使用所有在使用 GDB 启动进程时通常可用的 run命令来检查和修改已附加的进程。你可以插入断点；你可以单步执行和继续执行；你可以修改存储。如果你希望该进程继续运行，你可以在附加 continue 到该进程后使用 GDB 命令。

detach ¶

当你完成对已附加进程的调试后，你可以使用 detach 命令将其从 GDB 控制中释放出来。分离该进程会使其继续执行。在执行 detach 命令后，该进程和 GDB 再次完全独立，并且你可以准备 attach 另一个进程或使用 run启动一个进程。 detach 如果你在执行命令后再次按下 回车键 ，该命令不会重复执行。

如果你在附加了一个进程时退出 GDB ，你将分离该进程。如果你使用 run 命令，你将终止该进程。默认情况下， GDB 如果你尝试做这两件事之一，会要求确认；你可以使用 set confirm 命令（见 可选警告和消息) 来控制是否需要确认。
