actions [num] ¶

此命令会提示输入跟踪点被触发时要执行的操作列表。如果未指定跟踪点编号 num ，此命令将为最近定义的跟踪点设置操作（这样您可以定义一个跟踪点，然后直接说 actions ，而不必关心其编号）。您在接下来的行中逐行指定操作本身，并以仅包含 end的行结束操作列表。到目前为止，唯一定义的操作是 collect, teval, 和 while-stepping。

actions 实际上等同于 commands （请参阅 断点命令列表），但仅允许使用定义的操作；任何其他 GDB 命令都会被拒绝。

要移除跟踪点的所有操作，请输入 ‘actions num’，然后立即输入 ‘end’。 

(gdb) collect data // 收集一些数据

(gdb) while-stepping 5 // 单步执行 5 次，收集数据

(gdb) end              // 表示操作结束。

在下面的示例中，操作列表以 collect 命令开始，这些命令指示跟踪点被触发时要收集的内容。然后，为了在跟踪点之后进行单步执行并收集额外数据，使用了 while-stepping 命令，后面跟着单步序列中每次单步后要收集的内容列表。 while-stepping 命令由其自己的单独 end 命令终止。最后，操作列表由 end 命令终止。 

(gdb) trace foo
(gdb) actions
输入跟踪点 1 的操作，每行一个：
> collect bar,baz
> collect $regs
> while-stepping 12
  > collect $pc, arr[i]
  > end
end
collect[/mods] expr1, expr2, … ¶

当跟踪点被触发时，收集给定表达式的值。此命令接受任何有效表达式的逗号分隔列表。除了全局、静态或局部变量外，还支持以下特殊参数：

$regs

收集所有寄存器。

$args

收集所有函数参数。

$locals

收集所有局部变量。

$_ret

收集返回地址。如果您想查看更多回溯信息，这很有帮助。

注意： 返回地址的位置并不总是能预先可靠地确定，可能最终收集到错误的地址或寄存器。在某些架构上，函数入口处的跟踪点可靠性更高，而在其他架构上则相反。当这种情况发生时，回溯会停止，因为找不到返回地址（除非另一个收集规则恰好匹配它）。

$_probe_argc

收集跟踪点所在的静态探针的参数数量。 请参阅 静态探针点。

$_probe_argn

n 是一个介于 0 和 11 之间的整数。收集跟踪点所在的静态探针的第 n个参数。 请参阅 静态探针点。

$_sdata ¶

收集静态跟踪点标记的特定数据。仅适用于静态跟踪点。请参阅 跟踪点操作列表。在 UST 静态跟踪点库后端，一个插桩点类似于一个 printf 函数调用。跟踪库能够使用对程序进行插桩的程序员提供的格式，将用户指定的数据收集为字符串。其他后端也有类似的机制。以下是一个 UST 标记调用的示例： 

 const char master_name[] = "$your_name";
 trace_mark(channel1, marker1, "hello %s", master_name)

在这种情况下，收集 $_sdata 会收集字符串 ‘hello $yourname’。在分析跟踪缓冲区时，您可以像检查其他可用于$_sdata的变量一样检查它。 GDB.

您可以连续给出多个 collect 命令，每个命令只有一个参数，或者一个 collect 命令带有多个用逗号分隔的参数；效果是一样的。

可选的 mods 会改变参数的常规处理方式。 s 要求将指向字符的指针作为字符串处理，特别是收集指针所指向的内存内容，直到遇到第一个零为止。上限默认是 print characters 变量的值；如果 s 后面跟着一个十进制数，则该数为上限。例如 ‘collect/s25 mystr’ 会在 ‘mystr’ 处收集多达 25 个字符。

命令 info scope （请参阅 info scope) 对于确定要收集哪些数据特别有用。

teval expr1, expr2, … ¶

当跟踪点被触发时，计算给定的表达式。此命令接受一个用逗号分隔的表达式列表。结果会被丢弃，因此这主要用于为跟踪状态变量赋值（请参阅 跟踪状态变量），而不会像使用 collect 操作时那样将这些值添加到跟踪缓冲区中。

while-stepping n ¶

执行 n 次单步指令跟踪，在跟踪点之后进行，每次单步后收集新数据。 while-stepping 命令后面跟着单步时要收集的内容列表（后面跟着它自己的 end 命令）： 

> while-stepping 12
  > collect $regs, myglobal
  > end
>

请注意， $pc 不会被 while-stepping自动收集；如果需要该寄存器，您需要明确指定收集它。您可以将 while-stepping 缩写为 ws 或 stepping。

set default-collect expr1, expr2, … ¶

此变量是一个表达式列表，用于在每次跟踪点触发时收集。它实际上是一个额外的 collect 操作，会添加到每个跟踪点操作列表的前面。表达式会针对每个跟踪点单独解析，因此例如一个名为 xyz 的变量，根据跟踪点的位置，可能会被解释为一个跟踪点的全局变量，而对于另一个跟踪点则是局部变量。

show default-collect ¶

显示每次跟踪点触发时默认收集的表达式列表。
