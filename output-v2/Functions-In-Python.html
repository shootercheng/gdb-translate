23.3.2.25 编写新的便利函数 ¶

你可以在 Python 中实现新的便利函数（参见 便利变量)。便利函数是以下类的子类的一个实例 gdb.Function.

函数： Function.__init__ (名称) ¶

此函数的初始化器 Function 会将新函数注册到 GDB。参数 名称 是函数的名称，为一个字符串。该函数对用户而言将作为一个类型为 internal function的便利变量可见，其名称与给定的 名称相同。

新函数的文档取自新类的文档字符串。

函数： Function.invoke (*参数) ¶

当一个便利函数被求值时，其参数会被转换为 gdb.Value的实例，然后调用该函数的 invoke 方法。请注意， GDB 并不预先确定便利函数的参数数量。相反，所有可用的参数都会按照标准的 Python 调用约定传递给 invoke。特别地，便利函数的参数可以有默认值，且不会产生不良影响。

此方法的返回值将作为其在封闭表达式中的值。如果返回的是普通的 Python 值，它将按照通常的规则转换为 gdb.Value 。

以下代码片段展示了如何在 Python 中实现一个简单的便利函数： 

class Greet (gdb.Function):
  """返回用于问候某人的字符串。
以一个名字作为参数。"""

  def __init__ (self):
    super (Greet, self).__init__ ("greet")

  def invoke (self, name):
    return "Hello, %s!" % name.string ()

Greet ()

最后一行实例化了该类，并且有必要触发将函数注册到 GDB。根据 Python 代码被读入 GDB的方式，你可能需要显式导入 gdb 模块。

现在你可以在表达式中使用该函数： 

(gdb) print $greet("Bob")
$1 = "Hello, Bob!"
