27.6 的简单示例 GDB/MI 交互 ¶

本小节展示了几个使用 该接口进行交互的简单示例。在这些示例中，‘ GDB/MI ->’ 表示以下行作为输入传递给->，而 ‘ GDB/MI <-’ 表示从<-接收到的输出。 GDB/MI.

注意示例中显示的换行仅为了可读性，它们不会出现在实际输出中。

设置断点 ¶

设置断点会生成包含断点详细信息的同步输出。 

-> -break-insert main
<- ^done,bkpt={number="1",type="breakpoint",disp="keep",
    enabled="y",addr="0x08048564",func="main",file="myprog.c",
    fullname="/home/nickrob/myprog.c",line="68",thread-groups=["i1"],
    times="0"}
<- (gdb)

程序执行 ¶

程序执行会生成异步记录，并且 MI 会给出执行停止的原因。 

-> -exec-run
<- ^running
<- (gdb)
<- *stopped,reason="breakpoint-hit",disp="keep",bkptno="1",thread-id="0",
   frame={addr="0x08048564",func="main",
   args=[{name="argc",value="1"},{name="argv",value="0xbfc4d4d4"}],
   file="myprog.c",fullname="/home/nickrob/myprog.c",line="68",
   arch="i386:x86_64"}
<- (gdb)
-> -exec-continue
<- ^running
<- (gdb)
<- *stopped,reason="exited-normally"
<- (gdb)

退出 GDB ¶

退出 GDB 只会打印结果类 ‘^exit’。 

-> (gdb)
<- -gdb-exit
<- ^exit

请注意，‘^exit’ 会立即打印，但 GDB 实际退出可能需要一些时间。在此期间， GDB 会执行必要的清理操作，包括终止正在调试的程序或断开与调试硬件的连接，因此前端应该等待 GDB 退出，并且只有在 GDB 在合理时间内未能退出时才应强制终止它。

一个错误的命令 ¶

如果你传递一个不存在的命令会发生以下情况： 

-> -rubbish
<- ^error,msg="Undefined MI command: rubbish"
<- (gdb)
