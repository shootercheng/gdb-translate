33.3.2 条件初始化构造 ¶

Readline实现了一种在精神上类似于C预处理器条件编译特性的功能，该功能允许根据测试结果进行键绑定和变量设置。使用了四个解析器指令。

$if

该 $if 构造允许根据编辑模式、所使用的终端或使用Readline的应用程序进行绑定。测试文本在任何比较运算符之后一直延伸到行尾；除非另有说明，否则不需要任何字符来分隔它。

mode

该 mode= 形式的 $if 指令用于测试Readline是否处于 emacs 或 vi 模式。例如，这可以与‘设置键映射’命令结合使用，以便仅当Readline最初处于 emacs-standard 和 emacs-ctlx 键映射时设置绑定。 emacs 模式。

term

该 term= 形式可用于包含特定于终端的键绑定，也许是为了绑定终端功能键输出的键序列。‘=’右侧的单词会与终端的全名以及终端名称中第一个‘-’之前的部分进行比较。这允许 sun 匹配 sun 和 sun-cmd, 例如。

version

该 version 测试可用于对特定的Readline版本进行比较。 version 扩展为当前的Readline版本。比较运算符集合包括‘=’（和‘==’）、‘!=’、‘≤’、‘≥’、‘<’，以及‘>’。运算符右侧提供的版本号由主版本号、可选的小数点和可选的次版本号组成（例如，‘7.1’）。如果省略次版本号，则假定为‘0’。运算符可以与字符串 version 以及版本号参数之间用空格分隔。以下示例在使用的Readline版本为7.0或更高版本时设置一个变量： 

$if version ≥ 7.0
set show - mode - in - prompt on
$endif
application

该 应用程序 构造用于包含特定于应用程序的设置。每个使用Readline库的程序都会设置 应用程序名称, 并且可以测试特定的值。这可用于将键序列绑定到对特定程序有用的函数。例如，以下命令添加了一个在Bash中引用当前或前一个单词的键序列： 

$if Bash
# 引用当前或前一个单词
"\C - xq": "\eb\"\ef\""
$endif
variable

该 变量 构造为Readline变量和值提供简单的相等性测试。允许的比较运算符是‘=’、‘==’和‘!=’。变量名必须与比较运算符之间用空格分隔；运算符可以与右侧的值之间用空格分隔。字符串和布尔变量都可以进行测试。布尔变量必须与值 开启 和 关闭进行比较。以下示例等同于上面描述的 mode=emacs 测试： 

$if editing - mode == emacs
set show - mode - in - prompt on
$endif
$endif

如前一个示例所示，此命令终止一个 $if 命令。

$else

如果测试失败，则执行此 $if 指令分支中的命令。

$include

此指令接受单个文件名作为参数，并从该文件中读取命令和绑定。例如，以下指令从 /etc/inputrc读取： 

$include /etc/inputrc
