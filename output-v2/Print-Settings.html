set print symbol-filename off

不打印符号的源文件名和行号。这是默认设置。

show print symbol-filename

显示 GDB 是否将打印符号的源文件名和行号。

另一种显示符号文件名和行号有用的情况是反汇编代码时； GDB 会显示与每条指令对应的行号和源文件。

此外，您可能希望仅当打印的地址与最近的前一个符号足够接近时才显示符号形式：

set print max-symbolic-offset max-offset ¶
set print max-symbolic-offset unlimited

告诉 GDB 仅当最近的前一个符号与地址之间的偏移小于 最大偏移量. 默认值是 unlimited, 这告诉 GDB 如果有任何符号在地址之前，则始终打印地址的符号形式。零等同于 unlimited.

show print max-symbolic-offset

询问 GDB 在符号地址中打印的最大偏移量是多少。

如果您有一个指针，但不确定它指向哪里，可以尝试 ‘set print symbol-filename on’。然后，您可以使用 ‘p/a 指针’ 确定它指向的变量的名称和源文件位置。这将以符号形式解释地址。 例如，这里 GDB 显示一个变量 ptt 指向另一个变量 t, 该变量在 hi2.c中定义： 

(gdb) set print symbol-filename on
(gdb) p/a ptt
$4 = 0xe008 <t in hi2.c>

警告： 对于指向局部变量的指针，即使启用了相应的选项，‘p/a set print ’ 也不会显示被引用对象的符号名称和文件名。

您还可以使用 ‘set print symbol on’ 始终启用类似 ‘/a’ 的格式：

set print symbol on

告诉 GDB 如果存在与地址对应的符号，则打印该符号。

set print symbol off

告诉 GDB 不打印与地址对应的符号。在这种模式下， GDB 仍将打印指向函数的指针对应的符号。这是默认设置。

show print symbol

显示 GDB 是否将显示与地址对应的符号。
