5.5.4 特定线程断点 ¶

当你的程序有多个线程时（请参阅 调试多线程程序），你可以选择是在所有线程上设置断点，还是在特定线程上设置断点。

break locspec thread thread-id ¶
break locspec thread thread-id if …

位置规范 指定了你程序中的一个或多个代码位置。请参阅 位置规范，了解详细信息。

在断点命令中使用限定符 ‘线程 线程ID’ 来指定你只希望 GDB 在特定线程到达此断点时停止程序。 线程ID 指定符是由 GDB分配的线程标识符之一，显示在 ‘信息 线程’ 显示的第一列中。

如果你在设置断点时未指定 ‘线程 线程ID’，则该断点适用于 所有 程序线程。

你也可以在条件断点上使用 thread 限定符；在这种情况下，将 ‘线程 线程ID’ 放在断点条件之前或之后，如下所示： 

(gdb) 在 frik.c:13 处设置断点 线程 28 如果 bartab > lim

当 GDB 检测到相应线程不再在线程列表中时，特定线程断点会自动删除。例如： 

(gdb) 继续
特定线程断点 3 已删除 - 线程 28 不再在线程列表中。

线程消失有几种方式，例如正常线程退出，但当你使用 detach 命令从进程中分离时（请参阅 调试已在运行的进程），或者当 GDB 失去远程连接时（请参阅 调试远程程序）等等。请注意，对于某些目标， GDB 只有在用户明确使用 info threads 命令请求线程列表时才能检测到线程已退出。

一个断点不能既是特定线程的又是特定子进程的（请参阅 特定子进程断点），或者是特定任务的（请参阅 Ada任务扩展); 在创建断点时使用多个 thread, inferior, 或 task 关键字会导致错误。
