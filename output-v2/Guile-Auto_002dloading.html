23.4.4 Guile自动加载 ¶

当读取一个新的目标文件时（例如，由于 file 命令，或者因为 inferior 加载了一个共享库）， GDB 将通过两种方式查找Guile支持脚本： 目标文件-gdb.scm 和 .debug_gdb_scripts 部分。 请参阅 自动加载扩展.

自动加载功能对于提供特定于应用程序的调试命令和脚本很有用。

可以启用或禁用自动加载，并且可以打印自动加载脚本的列表。

set auto-load guile-scripts [on|off] ¶

启用或禁用Guile脚本的自动加载。

show auto-load guile-scripts ¶

显示是否启用或禁用了Guile脚本的自动加载。

info auto-load guile-scripts [regexp] ¶

打印所有 GDB 自动加载的Guile脚本的列表。

还会打印在 .debug_gdb_scripts 部分中提到但未找到的Guile脚本的列表。 这很有用，因为当 GDB 尝试加载它们并失败时，不会打印它们的名称。可能有很多这样的脚本，为每个脚本打印一条错误消息会有问题。

如果 正则表达式 被提供，那么只打印名称匹配的Guile脚本。

示例： 

(gdb) info auto-load guile-scripts
已加载的脚本
是    scm-section-script.scm
       完整名称：/tmp/scm-section-script.scm
否     my-foo-pretty-printers.scm

当读取一个自动加载的文件时， GDB 设置 当前目标文件. 这可以通过 current-objfile 过程获得（请参阅 Guile中的目标文件)。这对于注册特定于目标文件的漂亮打印机很有用。
