23.4.3.4 Guile异常处理 ¶

执行 guile 命令时，Guile代码中未捕获的Guile异常会被转换为对 GDB 错误报告机制的调用。如果调用 guile 的命令未处理该错误， GDB 将根据 guile print-stack 参数的设置终止该命令并报告错误。

该 guile print-stack 参数有三种设置：

none

不打印任何内容。

message

打印一条错误消息，其中包含Guile异常名称、关联值以及异常抛出点的Guile调用栈回溯信息。示例： 

(gdb) guile (display foo)
错误：在过程memoize-variable-access!中：
错误：未绑定的变量：foo
执行Scheme代码时出错。
full

除了错误消息外，还会打印完整的回溯信息。 

(gdb) set guile print-stack full
(gdb) guile (display foo)
Guile回溯信息：

GDB 在 GDB 由Guile代码调用的命令中发生的错误会转换为Guile异常。Guile异常的类型取决于错误情况。

由 GDB 提供的Guile过程可以抛出标准的Guile异常，如 wrong-type-arg 和 out-of-range。

用户中断（通过 C-c 或在分页提示时输入 q ）会转换为一个Guile signal 异常，其值为 SIGINT。

GDB Guile过程还可以抛出以下异常：

gdb:error ¶

此异常是对 GDB内部生成的错误的通用捕获。

gdb:invalid-object ¶

当访问包装底层 GDB 对象的Guile对象时，如果这些对象已变得无效，会抛出此异常。例如，如果用户从命令行删除了一个 <gdb:breakpoint> 对象，该对象在Guile中仍然存在，但它所代表的对象已不存在。对这个断点进行进一步操作将抛出此异常。

gdb:memory-error ¶

当操作试图访问被调试程序中的无效内存时，会抛出此异常。

gdb:pp-type-error ¶

当Guile漂亮打印机将一个错误的对象传递给 GDB时，会抛出此异常。
