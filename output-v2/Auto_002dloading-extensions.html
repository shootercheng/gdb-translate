23.5 自动加载扩展 ¶

GDB 提供了两种机制，用于在读取新的目标文件时自动加载扩展（例如，由于 file 命令，或者因为 inferior 加载了共享库）： 目标文件-gdb.ext （请参阅 《 目标文件-gdb.ext 文件》）以及现代文件格式（如 ELF）的 .debug_gdb_scripts 节（请参阅 《 .debug_gdb_scripts 节》）。有关这两种方法之间差异的讨论，请参阅 选择哪种方式？.

自动加载功能对于提供特定于应用程序的调试命令和功能很有用。

可以启用或禁用自动加载，并且可以打印自动加载脚本的列表。请参阅每种扩展语言的‘自动加载’部分以获取更多信息。对于 GDB 命令文件，请参阅 控制原生脚本的自动加载 GDB 脚本. 对于 Python 文件，请参阅 Python 自动加载.

请注意，加载此脚本文件还需要相应配置 auto-load safe-path （请参阅 自动加载的安全限制）。
