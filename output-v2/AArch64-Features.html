  • AArch64 核心寄存器特性

  • AArch64 SME 寄存器特性
  • AArch64 SME2 寄存器特性
  • AArch64 GCS 寄存器特性

¶

‘org.gnu.gdb.aarch64.core’ 特性是 AArch64 目标所必需的。它必须包含以下内容：

  • ‘x0’ 到 ‘x30’，通用寄存器，大小为 64 位。寄存器 ‘x30’ 也被称为 链接寄存器, 或 ‘lr’.
  • ‘sp’，栈指针寄存器或 ‘x31’。它的大小为 64 位，类型为 ‘数据指针’。
  • ‘pc’，程序计数器寄存器。它的大小为 64 位，类型为 ‘代码指针’。
  • ‘cpsr’，当前程序状态寄存器。它的大小为 32 位，并且有一个自定义标志类型。

‘cpsr’ 中各个标志和字段的语义可能会随着新的架构特性的添加而改变。当前的布局可以在 aarch64 - core.xml 文件中找到。

此特性中允许有额外的寄存器，但它们不会影响 GDB.

G.5.1.2 AArch64 浮点寄存器特性 ¶

‘org.gnu.gdb.aarch64.fpu’ 特性是可选的。如果存在，它必须包含以下寄存器：

  • ‘v0’ 到 ‘v31’，向量寄存器，大小为 128 位。类型是一个自定义向量类型。
  • ‘fpsr’，浮点状态寄存器。它的大小为 32 位，并且有一个自定义标志类型。
  • ‘fpcr’，浮点控制寄存器。它的大小为 32 位，并且有一个自定义标志类型。

‘fpsr’ 和 ‘fpcr’ 中各个标志和字段的语义可能会随着新的架构特性的添加而改变。

向量寄存器、‘fpsr’ 和 ‘fpcr’ 寄存器的类型可以在 aarch64 - fpu.xml 文件中找到。

此特性中允许有额外的寄存器，但它们不会影响 GDB.

G.5.1.3 AArch64 SVE 寄存器特性 ¶

‘org.gnu.gdb.aarch64.sve’ 特性是可选的。如果存在，这意味着目标支持可扩展向量扩展，并且必须包含以下寄存器：

  • ‘z0’ 到 ‘z31’，可扩展向量寄存器。它们的大小是可变的，并且是 128 位的倍数，最大为 2048 位。它们的类型是一个自定义联合类型，有助于可视化不同大小的子向量。
  • ‘fpsr’，浮点状态寄存器。它的大小为 32 位，并且有一个自定义标志类型。
  • ‘fpcr’，浮点控制寄存器。它的大小为 32 位，并且有一个自定义标志类型。
  • ‘p0’ 到 ‘p15’，谓词寄存器。它们的大小是可变的，基于当前向量长度，并且是 16 位的倍数。它们的类型是一个自定义联合类型，有助于可视化子元素。
  • ‘ffr’，首次故障寄存器。它的大小基于当前向量长度可变，并且是 16 位的倍数。其类型与谓词寄存器相同。
  • ‘vg’，向量粒度。它表示 ‘z’ 寄存器中 64 位块的数量。它与当前向量长度密切相关。它的类型是 ‘整数’。

当 GDB 检测到 SVE 特性时，它将假定支持可扩展向量扩展，并将根据 ‘z’、‘p’ 和 ‘ffr’ 寄存器的大小进行相应调整，基于 ‘vg’ 的值。

GDB 还将创建与 ‘v’ 向量寄存器等效的伪寄存器，该向量寄存器来自 ‘org.gnu.gdb.aarch64.fpu’ 特性。

‘z’ 寄存器的前 128 位与 ‘v’ 寄存器的 128 位重叠，因此更改其中一个将触发另一个的更改。

关于 ‘z’、‘p’ 和 ‘ffr’ 寄存器的类型，请查看 aarch64 - sve.c 文件。由于此特性是根据当前向量长度动态生成的，因此没有可用的 XML 文件。

‘fpsr’ 和 ‘fpcr’ 中各个标志和字段的语义可能会随着新的架构特性的添加而改变。

‘fpsr’ 和 ‘fpcr’ 寄存器的类型可以在 aarch64 - sve.c 文件中找到，并且应该与 aarch64 - fpu.xml 中描述的内容相匹配。

此特性中允许有额外的寄存器，但它们不会影响 GDB.

G.5.1.4 AArch64 指针认证寄存器特性 ¶

‘org.gnu.gdb.aarch64.pauth’ 可选特性被引入，以便 GDB 可以检测对指针认证扩展的支持。如果存在，它必须包含两组可能的寄存器之一。

用户模式的指针认证掩码：

  • ‘pauth_dmask’，数据指针的用户模式指针认证掩码。它的大小为 64 位。
  • ‘pauth_cmask’，代码指针的用户模式指针认证掩码。它的大小为 64 位。

用户模式和内核模式的指针认证掩码：

  • ‘pauth_dmask’，数据指针的用户模式指针认证掩码。它的大小为 64 位。
  • ‘pauth_cmask’，代码指针的用户模式指针认证掩码。它的大小为 64 位。
  • ‘pauth_dmask_high’，数据指针的内核模式指针认证掩码。它的大小为 64 位。
  • ‘pauth_cmask_high’，代码指针的内核模式指针认证掩码。它的大小为 64 位。

如果 GDB 检测到此特性中的两组寄存器中的任何一组，它将假定目标能够对指针进行签名。如果是这样， GDB 将在回溯信息中用 ‘[PAC]’ 标记装饰具有需要解掩码/解码的签名链接寄存器值的函数。

GDB 还将使用掩码从指针中移除非地址位。

此特性中允许有额外的寄存器，但它们不会影响 GDB.

请注意，‘org.gnu.gdb.aarch64.pauth’ 特性字符串已被弃用，并且只能用于与旧版本的 GDB 和 gdbserver. 支持指针认证的目标必须通过使用 ‘org.gnu.gdb.aarch64.pauth_v2’ 特性字符串来声明这种能力。

‘org.gnu.gdb.aarch64.pauth_v2’ 特性的内容与特性 ‘org.gnu.gdb.aarch64.pauth’ 完全相同。

引入特性 ‘org.gnu.gdb.aarch64.pauth_v2’ 的原因是之前版本的 GDB （版本 9、10、11 和 12）存在一个 bug。这个 bug 导致 GDB 每当目标报告支持指针认证（使用特性字符串 ‘org.gnu.gdb.aarch64.pauth’）并且还报告了 GDB未考虑到的额外系统寄存器时就会崩溃。这在使用模拟器和裸机调试场景中更为常见。

如果使用较新的 gdbserver 与存在此 bug 的旧 GDB 一起使用，也可能会发生这种情况。在这种情况下，较新的 gdbserver 可能会通过 ‘org.gnu.gdb.aarch64.pauth’ 特性字符串报告指针认证支持，并且还会报告旧的 GDB 不知道的额外寄存器，这可能会导致崩溃。

G.5.1.5 AArch64 TLS 寄存器特性 ¶

‘org.gnu.gdb.aarch64.tls’ 可选特性被引入，以便将 TLS 寄存器暴露给 GDB. 如果存在，它必须包含以下寄存器集之一。

仅 ‘tpidr’：

  • ‘tpidr’，软件线程 ID 寄存器。它的大小为 64 位，类型为 ‘数据指针’。

‘tpidr’ 和 ‘tpidr2’。

  • ‘tpidr’，软件线程 ID 寄存器。它的大小为 64 位，类型为 ‘数据指针’。
  • ‘tpidr2’，第二个软件线程 ID 寄存器。它的大小为 64 位，类型为 ‘数据指针’。它可能会在未来与可扩展矩阵扩展一起用于延迟恢复方案。

如果 GDB 检测到这个特性，它将尝试找到寄存器集的一种变体。如果 ‘tpidr2’ 可用， GDB 可能会在未来对其进行操作以显示额外的数据。

此特性没有 XML 文件，因为 ‘tpidr2’ 的存在是在运行时动态确定的。

此特性中允许有额外的寄存器，但它们不会影响 GDB.

G.5.1.6 AArch64 MTE 寄存器特性 ¶

‘org.gnu.gdb.aarch64.mte’ 可选特性被引入，以便 GDB 可以检测对内存标记扩展的支持并控制内存标记设置。如果存在，此特性必须有以下寄存器：

  • ‘tag_ctl’，标记控制寄存器。它的大小为 64 位，类型为 ‘无符号 64 位整数’。

内存标记检测是通过运行时检查完成的，因此此特性和寄存器的存在不足以启用内存标记支持。

此限制可能会在未来解除。

此特性中允许有额外的寄存器，但它们不会影响 GDB.

G.5.1.7 AArch64 SME 寄存器特性 ¶

‘org.gnu.gdb.aarch64.sme’ 特性是可选的。如果存在，它应该包含寄存器 ZA, SVG 和 SVCR. 请参阅 AArch64 SME.

  • ZA 是一个由 svlxsvl 字节的向量表示的寄存器。请参阅 svl.
  • SVG 是一个 64 位寄存器，包含 svg. 的值。请参阅 svg.
  • SVCR 是一个 64 位状态伪寄存器，有两个有效位。位 0 (SM) 显示流 SVE 模式是启用还是禁用。 位 1 (ZA) 显示 ZA 寄存器状态是活动（正在使用）还是非活动。 请参阅 aarch64 sme svcr.

    此伪寄存器的其余未使用位是未定义和保留的。这些位不应使用，可能会由未来的架构扩展定义。 SVCR

此特性中允许有额外的寄存器，但它们不会影响 GDB.

‘org.gnu.gdb.aarch64.sme’ 特性在目标还报告支持 ‘org.gnu.gdb.aarch64.sme2’ 特性时是必需的。

G.5.1.8 AArch64 SME2 寄存器特性 ¶

‘org.gnu.gdb.aarch64.sme2’ 特性是可选的。如果存在，那么 ‘org.gnu.gdb.aarch64.sme’ 特性也必须存在。‘org.gnu.gdb.aarch64.sme2’ 特性应该包含以下内容： 请参阅 AArch64 SME2.

  • ZT0 是一个 512 位（64 字节）的寄存器。它被定义为一个字节向量。

此特性中允许有额外的寄存器，但它们不会影响 GDB.

G.5.1.9 AArch64 GCS 寄存器特性 ¶

‘org.gnu.gdb.aarch64.gcs’ 特性是可选的。如果存在，这意味着目标支持受保护的控制栈，并且必须包含以下寄存器：

  • gcspr, 它指向线程的受保护控制栈。它的大小为 64 位，类型为 ‘数据指针’。

‘org.gnu.gdb.aarch64.gcs.linux’ 特性是可选的。如果存在，那么 ‘org.gnu.gdb.aarch64.gcs’ 特性也必须存在。‘org.gnu.gdb.aarch64.gcs.linux’ 特性表示 Linux 内核为 GCS 支持提供的功能，应该包含以下内容：

  • gcs_features_enabled 显示当前通过 prctl 或 ptrace 系统调用启用的特性。它被表示为一个具有自定义标志类型的 64 位寄存器。
  • gcs_features_locked 显示当前通过 prctl 或 ptrace 系统调用锁定的特性。它被表示为一个具有自定义标志类型的 64 位寄存器。

自定义标志类型允许 GDB 以人类友好的方式打印 gcs_features_enabled 和 gcs_features_locked 并且应该包含：

  • PR_SHADOW_STACK_ENABLE
  • PR_SHADOW_STACK_WRITE
  • PR_SHADOW_STACK_PUSH

有关更多信息，请参阅 文档 在 Linux 内核中。

这些特性中允许有额外的寄存器，但它们不会影响 GDB.
