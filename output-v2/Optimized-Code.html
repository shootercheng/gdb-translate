11 调试优化代码 ¶

几乎所有编译器都支持优化。禁用优化后，编译器会以一种简单的方式生成与源代码直接对应的汇编代码。随着编译器应用更强大的优化，生成的汇编代码会与原始源代码产生差异。借助编译器生成的调试信息， GDB 可以将运行中的程序映射回原始源代码中的结构。

GDB 在禁用优化的情况下，这种映射更准确。如果可以在不进行优化的情况下重新编译，那么在调试过程中更容易跟踪程序的执行进度。但是，在很多情况下，您可能需要调试优化后的版本。

当您调试使用‘-g -O’编译的程序时，请记住优化器已经重新排列了您的代码；调试器会向您展示实际存在的内容。当执行路径与您的源文件不完全匹配时，不要过于惊讶！一个极端的例子：如果您定义了一个变量，但从未使用它， GDB 调试器永远不会看到该变量 —— 因为编译器将其优化掉了。

有些功能在使用‘-g -O’时不如仅使用‘-g’时表现得好，特别是在具有指令调度功能的机器上。如果有疑问，请仅使用‘-g’重新编译，如果这样能解决问题，请将其作为一个 bug 报告给我们（包括一个测试用例！）。请参阅 程序变量, 以获取有关调试优化代码的更多信息。
