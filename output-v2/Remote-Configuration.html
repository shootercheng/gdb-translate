¶

本节记录了调试远程程序时可用的配置选项。有关远程协议的文件 I/O 扩展相关选项，请参阅 .

set remoteaddresssize bits ¶

将内存数据包中地址的最大大小设置为指定的位数。 GDB

show remoteaddresssize

显示当前远程地址大小的位数。

set serial baud n ¶

将远程串行 I/O 的波特率设置为 n 波特。该值用于设置用于调试远程目标的串行端口的速度。

show serial baud

显示当前远程连接的速度。

set serial parity parity

设置远程串行 I/O 的奇偶校验。支持的 奇偶校验 值有： even, none, 和 odd. 默认值是 none.

show serial parity

显示当前串行端口的奇偶校验设置。

set remotebreak ¶

如果设置为开启， GDB 在你输入 BREAK 信号时向远程发送 Ctrl - c 以中断远程上运行的程序。如果设置为关闭， GDB 则发送 ‘Ctrl - C’ 字符。默认值是关闭，因为大多数远程系统期望看到 ‘Ctrl - C’ 作为中断信号。

show remotebreak

显示 GDB 是否发送 BREAK 或 ‘Ctrl - C’ 以中断远程程序。

set remoteflow on ¶
set remoteflow off

启用或禁用用于与远程目标通信的串行端口上的硬件流控制（RTS/CTS)。

show remoteflow ¶

显示当前硬件流控制的设置。

set remotelogbase base

将记录串行协议通信的基数（也称为进制）设置为 基数. 支持的 基数 值有： ascii, octal, 和 hex. 默认值是 ascii.

show remotelogbase

显示当前记录远程串行协议的进制设置。

set remotelogfile file ¶

将远程串行通信记录到指定的 文件. 默认情况下不进行记录。

show remotelogfile

显示当前用于记录串行通信的文件名设置。

set remotetimeout num ¶

将等待远程目标响应的超时限制设置为 num 秒。默认值是 2 秒。

show remotetimeout

显示当前等待远程目标响应的秒数。

set remote hardware-watchpoint-limit limit
set remote hardware-breakpoint-limit limit

限制 GDB 使用 限制 远程硬件观察点或断点。 限制 可以设置为 0 以禁用硬件观察点或断点，设置为 unlimited 则表示无限制的观察点或断点。

show remote hardware-watchpoint-limit
show remote hardware-breakpoint-limit

显示当前 GDB 可以使用的硬件观察点或断点的数量限制。

set remote hardware-watchpoint-length-limit limit

限制 GDB 使用 限制 字节作为远程硬件观察点的最大长度。 限制 为 0 时禁用硬件观察点，设置为 unlimited 则允许任意长度的观察点。

show remote hardware-watchpoint-length-limit

显示当前远程硬件观察点的最大长度限制（以字节为单位）。

set remote exec-file filename
show remote exec-file

选择用于 run 的文件，使用 target extended-remote. 这应该设置为目标系统上有效的文件名。如果未设置，目标将使用默认文件名（例如，最后运行的程序）。

set remote interrupt-sequence ¶

允许用户从 ‘Ctrl - C’，一个 BREAK 或者 ‘BREAK - g’ 作为向远程目标发送的序列，以中断执行。 ‘Ctrl - C’ 是默认值。某些系统更喜欢 BREAK 在特定时间内使用高级别的串行线路。 Linux 内核更喜欢 ‘BREAK - g’，也就是 Magic SysRq g。 它是 BREAK 字符后面跟随的信号 g.

show remote interrupt-sequence

显示 ‘Ctrl - C’、 BREAK 或者 BREAK-g 由 GDB 发送以中断远程程序。 BREAK-g 是 BREAK 信号后面跟随 g 并且也被称为 Magic SysRq g。

set remote interrupt-on-connect ¶

指定当 GDB 连接到远程目标时，是否向其发送中断序列。当你调试 Linux 内核时，这通常是需要的。Linux 内核期望 BREAK 后面跟随 g 也就是被称为 Magic SysRq g 的内容，以便进行连接 GDB.

show remote interrupt-on-connect

显示当 GDB 连接到远程目标时，是否向其发送中断序列。

set tcp auto-retry on ¶

启用远程 TCP 连接的自动重试功能。如果远程 调试代理与 GDB并行启动，这很有用；因为在 GDB 尝试连接之前，代理可能还未准备好接受连接，所以存在竞争条件。当启用自动重试时，如果初始连接尝试失败， GDB 会使用由 set tcp connect-timeout指定的超时时间重新尝试建立连接。

set tcp auto-retry off

不自动重试失败的 TCP 连接。

show tcp auto-retry

显示当前的自动重试设置。

set tcp connect-timeout seconds ¶
set tcp connect-timeout unlimited

将与远程目标建立 TCP 连接的超时时间设置为 秒. 该超时时间会影响重试失败连接的轮询（由 set tcp auto-retry on启用）以及等待完成较慢的连接，并且表示一个近似的累积值。如果 秒 是 unlimited, 则没有超时时间，并且 GDB 将一直尝试建立连接，除非用 Ctrl - c中断。默认值是 15 秒。

show tcp connect-timeout

显示当前的连接超时设置。

远程协议会自动检测你的调试桩支持的数据包。如果你需要覆盖自动检测，可以使用这些命令来启用或禁用单个数据包。每个数据包可以设置为 ‘ GDB 开启’（远程目标支持此数据包）、‘关闭’（远程目标不支持此数据包），或者 ‘自动’（检测远程目标对此数据包的支持情况）。它们都默认设置为 ‘自动’。有关每个数据包的更多信息，请参阅 GDB 远程串行协议.

在正常使用中，你通常不需要使用这些命令。如果你使用了，可能是你的远程调试桩存在 bug，或者是 GDB存在 bug。你可能想向 GDB 开发者报告这个问题。

对于每个数据包 名称, 启用或禁用该数据包的命令是 set remote name-packet. 如果你配置了一个数据包，在未选择目标的情况下，该配置将应用于所有未来的远程目标。如果选择了一个目标，则仅更改当前目标的配置。所有其他现有远程目标的功能不受影响。打印数据包当前配置的命令是 show remote name-packet. 它会显示当前远程目标的配置。如果未选择远程目标，则显示未来连接的默认配置。可用的设置有：

可以使用命令 set remote memory-read-packet-size 和 set remote memory-write-packet-size配置远程目标每个内存读取或内存写入数据包的字节数。如果设置为 ‘0’（零），将使用默认的数据包大小。实际限制会根据目标进一步降低。指定 ‘固定’ 以禁用与目标相关的限制，指定 ‘限制’ 以启用它。与启用和禁用远程数据包类似，该命令适用于当前选择的目标（如果可用）。如果未选择远程目标，它将适用于所有未来的远程连接。可以使用命令 show remote memory-read-packet-size 和 show remote memory-write-packet-size显示所选目标的配置。如果未选择远程目标，则显示未来连接的默认配置。
