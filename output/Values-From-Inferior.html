23.3.2.4 来自下级进程的值 ¶

GDB 从下级程序获取的值存放在一个 gdb.Value 类型的对象中。GDB 使用这个对象对下级进程的值进行内部记录，并在必要时获取值。

简单标量类型的下级进程值可以直接用于对该值数据类型有效的Python表达式中。这里有一个整数或浮点数 some_val 的示例： 

bar = some_val + 2

这样一来，bar 也将是一个 gdb.Value 对象，其值的类型与 some_val 的类型相同。对表示 struct 或 class 对象的 gdb.Value 对象也可以执行有效的Python操作。对于这种情况，如果存在重载运算符，则使用它来执行操作。例如，如果 val1 和 val2 是表示一个重载了 + 运算符的 class 实例的 gdb.Value 对象，那么可以在Python脚本中使用 + 运算符，如下所示： 

val3 = val1 + val2

操作 val3 的结果也是一个 gdb.Value 对象，它对应于重载的 + 运算符返回的值。一般来说，重载运算符用于以下操作：+（二元加法）、-（二元减法）、*（乘法）、/、%、<<、>>、|、&、^。

结构或某个类的实例的下级进程值可以使用Python的 字典语法 来访问。例如，如果 some_val 是一个持有结构的 gdb.Value 实例，可以使用以下方式访问其 foo 元素： 

bar = some_val['foo']

同样，bar 也将是一个 gdb.Value 对象。也可以使用 gdb.Field 对象作为下标来访问结构元素（有关 gdb.Field 对象的更多信息，请参阅 Python中的类型）。例如，如果 foo_field 是一个与上述结构的 foo 元素对应的 gdb.Field 对象，那么也可以如下访问 bar： 

bar = some_val[foo_field]

如果一个 gdb.Value 具有数组或指针类型，可以使用整数索引来访问元素。 

result = some_array[23]
