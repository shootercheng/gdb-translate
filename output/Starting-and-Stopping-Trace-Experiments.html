13.1.9启动和停止跟踪实验 ¶

tstart ¶

此命令启动跟踪实验，并开始收集数据。 它具有丢弃上一个跟踪实验期间在跟踪缓冲区中收集的所有数据的副作用。如果提供了任何参数，它们将被视为注释并与跟踪实验的状态一起存储。这些注释可以是任意文本，在多用户环境中的断开连接跟踪中特别有用；注释可以解释跟踪正在做什么，提供用户联系信息等等。

tstop ¶

此命令停止跟踪实验。如果提供了任何参数，它们将作为注释与实验一起记录。如果你正在停止由其他人启动的跟踪，这将很有用，例如，如果跟踪干扰了系统行为并且需要快速停止。

注意：如果达到任何跟踪点的通过次数（见 跟踪点通过次数），或者跟踪缓冲区已满，跟踪实验和数据收集可能会自动停止。

tstatus ¶

此命令显示当前跟踪数据收集的状态。

以下是我们到目前为止描述的命令的一个示例： 

(gdb) trace gdb_c_test
(gdb) actions
为跟踪点#1输入操作，每行一个。
> collect $regs,$locals,$args
> while-stepping 11
  > collect $regs
  > end
> end
(gdb) tstart
	[时间流逝...]
(gdb) tstop

即使GDB与目标断开连接（无论是自愿还是非自愿），你也可以选择继续运行跟踪实验。对于诸如 detach 之类的命令，调试器会询问你想如何处理跟踪。但是对于意外终止（GDB崩溃、网络中断），丢失来之不易的跟踪数据将是不幸的，因此变量 disconnected-tracing 让你决定跟踪是否应该在没有GDB的情况下继续运行。

set disconnected-tracing on ¶
set disconnected-tracing off

选择如果GDB已与目标断开连接，跟踪运行是否应继续运行。请注意，无论此变量的设置如何，detach 或 quit 将直接询问你如何处理正在运行的跟踪，因此该变量主要用于处理意外情况，例如网络丢失。

show disconnected-tracing ¶

显示当前断开连接跟踪的选择。

当你重新连接到目标时，跟踪实验可能仍在运行，也可能已经停止；在此期间它可能已经填满了跟踪缓冲区，或者由于其他原因之一而停止。如果它正在运行，重新连接后它将继续。

重新连接后，目标将上传有关当前有效跟踪点的信息。然后GDB将该信息与当前定义的跟踪点集进行比较，并尝试将它们匹配起来，同时考虑到在此期间由于创建和删除可能导致的编号变化。如果目标的一个跟踪点与GDB中的任何跟踪点都不匹配，调试器将创建一个新的跟踪点，以便你有一个编号来指定该跟踪点。这个匹配过程必然是启发式的，可能会导致创建无用的跟踪点；如果它们没有用，你可以简单地删除它们。

如果你的目标代理支持 循环跟踪缓冲区，那么你可以无限期地运行跟踪实验而不会填满跟踪缓冲区；当空间用完时，代理会删除已经收集的跟踪帧， oldest first，直到有足够的空间继续收集。如果你的跟踪点被频繁命中，并且由于缓冲区已满而导致跟踪过早终止，这将特别有用。要请求使用循环跟踪缓冲区，只需将 ‘circular-trace-buffer’ 设置为on。你可以在任何时候设置此选项，包括在跟踪期间；如果代理可以做到，它将即时更改缓冲区处理，否则它将直到下次运行才会生效。

set circular-trace-buffer on ¶
set circular-trace-buffer off

选择跟踪运行是应使用线性还是循环缓冲区来存储跟踪数据。线性缓冲区不会丢失任何跟踪数据，但可能会过早填满，而循环缓冲区会丢弃旧的跟踪数据，但它将始终有空间用于最新的跟踪点命中。

show circular-trace-buffer ¶

显示当前跟踪缓冲区的选择。请注意，这可能与代理当前的缓冲区处理不匹配，也不能保证与过去运行期间可能生效的设置匹配，例如如果你正在查看跟踪文件中的帧。

set trace-buffer-size n ¶
set trace-buffer-size unlimited

请求目标使用大小为 n 字节的跟踪缓冲区。并非所有目标都会遵守此请求；它们可能有一个编译时设置的跟踪缓冲区大小，或者有其他限制。设置为 unlimited 或 -1 以让目标使用其喜欢的任何大小。这也是默认设置。

show trace-buffer-size ¶

显示当前请求的跟踪缓冲区大小。请注意，如果目标支持大小设置并且能够处理请求的大小，这才会与实际大小匹配。例如，如果目标只能在运行之间更改缓冲区大小，此变量将直到下次运行开始才会反映更改。使用 tstatus 来获取实际缓冲区大小的报告。

set trace-user text ¶
show trace-user ¶
set trace-notes text ¶

设置跟踪运行的注释。

show trace-notes ¶

显示跟踪运行的注释。

set trace-stop-notes text ¶

设置跟踪运行的停止注释。注释的处理方式与 tstop 参数相同；设置命令是修正错误或不完整的停止注释的便捷方法。

show trace-stop-notes ¶

显示跟踪运行的停止注释。
