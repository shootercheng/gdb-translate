27.9.3 C++ 异常 GDB/MI 捕获点 ¶

以下 GDB/MI 命令可用于创建捕获点，当C++ 异常被抛出、重新抛出或捕获时停止执行。

The -catch-throw 命令 ¶

概要 ¶

 -catch-throw [ -t ] [ -r 正则表达式]

当被调试程序抛出C++ 异常时停止。如果给出了 正则表达式，那么只有类型与正则表达式匹配的异常才会被捕获。

如果给出了 ‘-t’，那么捕获点仅在一次停止时启用，在因该事件停止一次后捕获点会自动删除。

GDB 命令 ¶

相应的 GDB 命令是 ‘catch throw’ 和 ‘tcatch throw’（见 设置捕获点）。

示例 ¶

-catch-throw -r 异常类型
^done,bkpt={number="1",type="catchpoint",disp="keep",enabled="y",
  what="异常抛出",catch-type="throw",
  thread-groups=["i1"],
  regexp="异常类型",times="0"}
(gdb)
-exec-run
^running
(gdb) 
~"\n"
~"捕获点1（异常抛出），0x00007ffff7ae00ed在__cxa_throw () 中，来自 /lib64/libstdc++.so.6\n"
*stopped,bkptno="1",reason="断点命中",disp="keep",
  frame={addr="0x00007ffff7ae00ed",func="__cxa_throw",
  args=[],from="/lib64/libstdc++.so.6",arch="i386:x86-64"},
  thread-id="1",stopped-threads="all",core="6"
(gdb)

The -catch-rethrow 命令 ¶

概要 ¶

 -catch-rethrow [ -t ] [ -r 正则表达式]

当C++ 异常被重新抛出时停止。如果给出了 正则表达式，那么只有类型与正则表达式匹配的异常才会被捕获。

如果给出了 ‘-t’，那么捕获点仅在一次停止时启用，在捕获第一个事件后捕获点会自动删除。

GDB 命令 ¶

相应的 GDB 命令是 ‘catch rethrow’ 和 ‘tcatch rethrow’（见 设置捕获点）。

示例 ¶

-catch-rethrow -r 异常类型
^done,bkpt={number="1",type="catchpoint",disp="keep",enabled="y",
  what="异常重新抛出",catch-type="rethrow",
  thread-groups=["i1"],
  regexp="异常类型",times="0"}
(gdb)
-exec-run
^running
(gdb) 
~"\n"
~"捕获点1（异常重新抛出），0x00007ffff7ae00ed在__cxa_rethrow () 中，来自 /lib64/libstdc++.so.6\n"
*stopped,bkptno="1",reason="断点命中",disp="keep",
  frame={addr="0x00007ffff7ae00ed",func="__cxa_rethrow",
  args=[],from="/lib64/libstdc++.so.6",arch="i386:x86-64"},
  thread-id="1",stopped-threads="all",core="6"
(gdb)

The -catch-catch 命令 ¶

概要 ¶

 -catch-catch [ -t ] [ -r 正则表达式]

当被调试程序捕获C++ 异常时停止。如果给出了 正则表达式，那么只有类型与正则表达式匹配的异常才会被捕获。

GDB 命令 ¶

-catch-catch -r 异常类型

(gdb) 
~"\n"
~"捕获点1（异常捕获）, 0x0
