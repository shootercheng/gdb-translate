15.1.2 设置工作语言 ¶

如果允许GDB自动设置语言，那么在调试会话和程序中，表达式的解释方式是相同的。

如果愿意，也可以手动设置语言。为此，发出命令“set language lang”，其中lang是一种语言的名称，例如 c 或 modula-2。 要获取支持的语言列表，请输入“set language”。

手动设置语言会阻止GDB自动更新工作语言。如果在工作语言与源语言不同时尝试调试程序，而某个表达式在两种语言中都可接受但含义不同，这可能会导致混淆。例如，如果当前源文件是用C编写的，而GDB正在解析Modula-2，那么像这样的命令： 

print a = b + c

可能不会产生预期的效果。在C中，这意味着将b 和 c 相加并将结果放入 a 中。打印出的结果将是 a 的值。在Modula-2中，这意味着将 a 与 b+c 的结果进行比较，产生一个 BOOLEAN 值。
