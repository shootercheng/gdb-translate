10.9打印设置 ¶

GDB提供了以下几种方式来控制数组、结构体和符号的打印方式。

这些设置对于调试任何语言的程序都很有用：

set print address ¶
set print address on

GDB在显示堆栈跟踪、结构体值、指针值、断点等的位置时会打印内存地址，即使它也会显示这些地址的内容。默认值为on。例如，这是使用set print address on时堆栈帧显示的样子： 

(gdb) f
#0  set_quotes (lq=0x34c78 "<<", rq=0x34c88 ">>")
    at input.c:530
530         if (lquote != def_lquote)
set print address off

在显示内存地址的内容时不打印地址。例如，这是以set print address off显示的相同堆栈帧： 

(gdb) set print addr off
(gdb) f
#0  set_quotes (lq="<<", rq=">>") at input.c:530
530         if (lquote != def_lquote)

你可以使用‘set print address off’来消除GDB界面中所有与机器相关的显示。例如，使用print address off，你应该在所有机器上得到相同的回溯文本 —— 无论它们是否涉及指针参数。

show print address ¶

显示是否要打印地址。

当GDB打印一个符号地址时，它通常会打印最近的前一个符号加上一个偏移量。如果该符号不能唯一标识该地址（例如，它是一个作用域为单个源文件的名称），你可能需要进行澄清。一种方法是使用info line，例如‘info line *0x4537’。或者，你可以设置GDB在打印符号地址时打印源文件名和行号：

set print symbol-filename on ¶

告诉GDB在以符号地址的形式打印符号时，打印该符号的源文件名和行号。

set print symbol-filename off

不打印符号的源文件名和行号。这是默认设置。

show print symbol-filename

显示GDB是否会在以符号地址的形式打印符号时，打印该符号的源文件名和行号。

在反汇编代码时，显示符号文件名和行号也很有帮助；GDB会显示与每条指令对应的行号和源文件。

此外，你可能只想在打印的地址与最近的前一个符号相当接近时才看到符号形式：

set print max-symbolic-offset max-offset ¶
set print max-symbolic-offset unlimited

告诉GDB，只有当最近的前一个符号与该地址之间的偏移量小于max-offset时，才显示该地址的符号形式。默认值是unlimited，这告诉GDB，如果有任何符号在该地址之前，总是打印该地址的符号形式。零等同于unlimited。

show print max-symbolic-offset

询问GDB在符号地址中打印的最大偏移量是多少。

如果你有一个指针，但不确定它指向哪里，可以尝试‘set print symbol-filename on’。然后，你可以使用‘p/a pointer’来确定它指向的变量的名称和源文件位置。这将以符号形式解释该地址。例如，这里GDB显示变量ptt指向另一个在hi2.c中定义的变量t： 

(gdb) set print symbol-filename on
(gdb) p/a ptt
$4 = 0xe008 <t in hi2.c>

警告：对于指向局部变量的指针，即使启用了适当的set print选项，‘p/a’也不会显示被引用对象的符号名称和文件名。

你也可以使用‘set print symbol on’始终启用类似‘/a’的格式化：

set print symbol on

告诉GDB，如果存在对应的符号，则打印与地址对应的符号。

set print symbol off

告诉GDB不打印与地址对应的符号。在这种模式下，GDB仍然会打印指向函数的指针对应的符号。这是默认设置。

show print symbol

显示GDB是否会显示与地址对应的符号。

其他设置控制不同类型对象的打印方式：

set print array ¶
set print array on

以漂亮格式打印数组。这种格式更便于阅读，但会占用更多空间。默认值为关闭。

set print array off

恢复数组的压缩格式。

show print array

显示在显示数组时选择的是压缩格式还是漂亮格式。

set print array-indexes
set print array-indexes on

在显示数组时打印每个元素的索引。这可能更便于在数组中定位给定元素或快速找到打印数组中给定元素的索引。默认值为关闭。

set print array-indexes off

在显示数组时停止打印元素索引。

show print array-indexes

显示在显示数组时是否打印每个元素的索引。

set print nibbles ¶
set print nibbles on

当使用GDB的打印命令并带有‘/t’选项时，以四位一组的形式打印二进制值，即所谓的半字节。例如，这是使用set print nibbles on时的样子： 

(gdb)= 1230
(gdb) print/t val_flags
$2 = 0100 1100 1110
set print nibbles off

不以组的形式打印二进制值。这是默认设置。

show print nibbles

显示是否以四位一组的形式打印二进制值。

set print characters number-of-characters ¶
set print characters elements
set print characters unlimited

设置GDB将打印的字符串字符数限制。如果GDB正在打印一个长字符串，它在打印了由set print characters命令设置的字符数后就会停止打印。这同样适用于多字节和宽字符字符串，也就是说，对于字符类型为wchar_t、char16_t或char32_t的字符串，这个设置控制的是实际字符数，而不是编码使用的底层字节数。 将number-of-characters设置为elements意味着打印字符数的限制遵循数组元素的限制；见set print elements。 将number-of-characters设置为unlimited意味着打印的字符数没有限制。 当GDB启动时，这个限制设置为elements。

show print characters

显示GDB将打印的长字符串的字符数。

set print elements number-of-elements ¶
set print elements unlimited

设置GDB将打印的数组元素数限制。如果GDB正在打印一个大数组，它在打印了由set print elements命令设置的元素数后就会停止打印。默认情况下，这个限制也适用于字符串的显示；见set print characters。 当GDB启动时，这个限制设置为200。 将number-of-elements设置为unlimited或零意味着打印的元素数没有限制。

当打印非常大的数组时，如果其大小大于max-value-size（见max-value-size），并且print elements设置为使得正在打印的元素大小小于或等于max-value-size，那么GDB将打印该数组（直到print elements限制），并且只有价值为max-value-size的数据会被添加到值历史中（见值历史）。

show print elements

显示GDB将打印的大数组的元素数。

set print frame-arguments value ¶

此命令允许控制调试器在打印帧时（见堆栈帧）如何打印参数值。可能的值如下：

all

打印所有参数的值。

scalars

仅当参数是标量时才打印其值。更复杂的参数（如数组、结构体、联合体等）的值将被替换为…。这是默认设置。这里有一个只显示标量参数的示例： 

#1  0x08048361 in call_me (i=3, s=..., ss=0xbf8d508c, u=..., e=green)
  at frame-args.c:23
none

不打印任何参数值。相反，每个参数的值都被替换为…。在这种情况下，上面的示例现在变为： 

#1  0x08048361 in call_me (i=..., s=..., ss=..., u=..., e=...)
  at frame-args.c:23
presence

仅用…表示参数的存在。对于没有任何参数的函数，不打印…。不打印任何参数名称和值。 在这种情况下，上面的示例现在变为： 

#1  0x08048361 in call_me (...) at frame-args.c:23

默认情况下，只打印标量参数。此命令可用于配置调试器以打印所有参数的值，而不管其类型如何。然而，通常不打印更复杂参数的值是有利的。例如，这减少了每个帧中打印的信息量，使回溯更具可读性。此外，在显示Ada帧时，这还提高了性能，因为计算大型参数有时可能会占用大量CPU资源，尤其是在大型应用程序中。将print frame-arguments设置为scalars（默认值）、none或presence可以避免这种计算，从而加快每个Ada帧的显示速度。

show print frame-arguments

显示在打印帧时应如何显示参数值。

set print raw-frame-arguments on

以原始的、不漂亮打印的形式打印帧参数。

set print raw-frame-arguments off

如果有针对该值的漂亮打印机（见漂亮打印），则以漂亮打印的形式打印帧参数，否则以原始形式打印该值。 这是默认设置。

show print raw-frame-arguments

显示是否以原始形式打印帧参数。

set print entry-values value ¶

设置在函数入口处打印帧参数值。在某些情况下，GDB可以确定函数调用者传递的函数参数的值，即使该值在被调用函数内部被修改，因此是不同的。对于优化后的代码，当前值可能不可用，但入口值可能仍然已知。

默认值是default（见下文对其描述）。较旧的GDB的行为与设置no时相同。不支持此功能的编译器在default设置下的行为与在no设置下相同。

此功能目前仅由DWARF 2调试格式支持，并且编译器必须生成‘DW_TAG_call_site’标签。使用GCC时，你需要在编译期间指定-O -g，以获取此信息。

value参数可以是以下之一：

no

仅打印实际参数值，从不打印函数入口点的值。 

#0  equal (val=5)
#0  different (val=6)
#0  lost (val=<optimized out>)
#0  born (val=10)
#0  invalid (val=<optimized out>)
only

仅打印函数入口点的参数值。从不打印实际参数值。
