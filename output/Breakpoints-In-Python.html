23.3.2.33 使用Python操作断点 ¶

Python代码可以通过gdb.Breakpoint类来操作断点。

可以使用gdb.Breakpoint构造函数的两种形式之一来创建断点。第一种形式接受一个字符串，就像传递给break（见 设置断点）和watch（见 设置观察点）命令的字符串一样，可用于创建断点和观察点。第二种形式接受类似于 显式位置 的单独Python参数，并且只能用于创建断点。

函数: Breakpoint.__init__ (spec [, type ][, wp_class ][, internal ][, temporary ][, qualified ]) ¶

根据spec创建一个新的断点，spec是一个命名断点位置的字符串，或者是一个定义观察点的表达式。该字符串应以break命令（见 设置断点）或观察点的情况下，以watch命令（见 设置观察点）所识别的格式描述一个位置。

可选参数 type 指定要创建的断点类型，如下所定义。

可选参数 wp_class 在 type 为 gdb.BP_WATCHPOINT 时定义要创建的观察点类型。如果省略 wp_class，则默认为 gdb.WP_WRITE。

可选参数 internal 允许断点对用户不可见。断点在创建时不会被报告，也不会在 info breakpoints 的输出中列出（但会在 maint info breakpoints 命令中列出）。

可选参数 temporary 使断点成为临时断点。临时断点在被命中后会被删除。在断点被命中后对Python断点的任何进一步访问都将导致运行时错误（因为该断点现在已被自动删除）。

可选参数 qualified 是一个布尔值，允许将 spec 中传递的函数解释为完全限定名称。它等同于 break 的 -qualified 标志（见 行规范位置 和 显式位置）。

函数: Breakpoint.__init__ ([ source ][, function ][, label ][, line ], ][ internal ][, temporary ][, qualified ]) ¶

创建新断点的第二种形式使用关键字指定显式位置（见 显式位置）。新断点将在指定的源文件 source 中，在指定的 function 的 label 和 line 处创建。

internal、temporary 和 qualified 的用法与前面解释的相同。

可用类型由 gdb 模块中定义的常量表示：

gdb.BP_BREAKPOINT ¶

普通代码断点。

Normal code breakpoint.

gdb.BP_HARDWARE_BREAKPOINT ¶

硬件辅助代码断点。

gdb.BP_WATCHPOINT ¶

观察点断点。

gdb.BP_HARDWARE_WATCHPOINT ¶

硬件辅助观察点。

gdb.BP_READ_WATCHPOINT ¶

硬件辅助读观察点。

gdb.BP_ACCESS_WATCHPOINT ¶

硬件辅助访问观察点。

gdb.BP_CATCHPOINT ¶

捕获点。目前，在创建 gdb.Breakpoint 对象时不能使用此类型，但在从 gdb.BreakpointEvent 报告的 gdb.Breakpoint 对象中会存在（见 Python中的事件）。

可用的观察点类型由 gdb 模块中定义的常量表示：

gdb.WP_READ ¶

只读观察点。

gdb.WP_WRITE ¶

只写观察点。

gdb.WP_ACCESS ¶

读写观察点。
