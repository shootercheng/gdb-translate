21.1.5 特定于GNU Hurd系统的命令 ¶

本小节描述特定于GNU Hurd原生调试的GDB命令。

set signals ¶
set sigs

此命令切换GDB对下级信号拦截的状态。Mach异常，如断点陷阱，不受此命令影响。sigs是signals的简写别名。

show signals ¶
show sigs

显示拦截下级信号的当前状态。

set signal-thread ¶
set sigthread

此命令告诉GDB哪个线程是libc信号线程。当信号传递给正在运行的进程时该线程会运行。set sigthread是set signal-thread的简写别名。

show signal-thread ¶
show sigthread

这两个命令显示当下级接收到信号时哪个线程会运行。

set stopped ¶

此命令告诉GDB下级进程已停止，就像使用SIGSTOP信号一样。可以通过向其发送信号来继续已停止的进程。

show stopped ¶

此命令显示GDB是否认为被调试程序已停止。

set exceptions ¶

使用此命令关闭下级中的异常捕获。当异常捕获关闭时，断点和单步执行都将不起作用。要恢复默认设置，需开启异常捕获。

show exceptions ¶

显示下级中捕获异常的当前状态。

set task pause ¶

此命令在GDB控制时切换任务暂停状态。将其设置为开启会立即生效，并且每当GDB获得控制权时任务就会暂停。将其设置为关闭将在下一次继续下级时生效。如果此选项设置为关闭，可以使用set thread default pause on或set thread pause on（见下文）来暂停单个线程。

show task pause ¶

显示任务暂停的当前状态。

set task detach-suspend-count ¶

此命令设置当GDB从任务分离时任务将保留的暂停计数。

show task detach-suspend-count

显示分离时任务将保留的暂停计数。
