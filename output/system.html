system ¶

概要：
int system(const char *command);
请求：

‘Fsystem,commandptr/len’

返回值：

如果len为零，返回值表示是否有可用的 shell。返回值为零表示没有可用的 shell。 对于非零的len，出错时返回值为 -1，否则返回命令的返回状态。只返回命令的退出状态，它是通过调用 WEXITSTATUS(retval) 从主机的 system 返回值中提取的。如果无法执行/bin/sh，则返回127。

错误：
EINTR

调用被用户中断。

GDB 承担了调用必要的主机调用以执行 system 调用的全部任务。在返回给目标之前，主机上 system 的返回值被简化。来自子进程的任何终止信号信息都被丢弃，返回值完全由被调用命令的退出状态组成。

出于安全考虑，GDB 默认拒绝 system 调用。用户必须使用 set remote system-call-allowed 1 命令明确允许此调用。

set remote system-call-allowed ¶

控制是否允许在远程目标的文件I/O协议中进行 system 调用。默认值为零（禁用）。

show remote system-call-allowed ¶

显示在文件I/O协议中是否允许 system 调用。
