22.1 提示符 ¶

GDB 通过打印一个称为 提示符 的字符串来表明它已准备好读取命令。这个字符串通常是 ‘(gdb)’。你可以使用 set prompt 命令更改提示符字符串。例如，当使用 GDB 调试 GDB 时，在其中一个 GDB 会话中更改提示符是很有用的，这样你就可以始终知道你正在与哪个会话进行交互。

注意： set prompt 不会在你设置的提示符后面为你添加一个空格。这使你可以设置以空格结尾或不以空格结尾的提示符。

set prompt 新提示符 ¶

指示 GDB 从此以后使用 新提示符 作为其提示符字符串。

例如，这将设置一个蓝色的 “(gdb)” 提示符： 

set prompt \001\033[0;34m\002(gdb)\001\033[0m\002

它使用 ‘\001’ 和 ‘\002’ 来开始和结束一系列非打印字符，以确保它们不会被计入字符串长度。

show prompt ¶

打印如下形式的一行：‘Gdb的提示符是：你的提示符’

启用了Python脚本的 GDB 版本有提示符扩展。与这些扩展进行交互的命令是：

set extended-prompt 提示符 ¶

设置一个允许替换的扩展提示符。有关可用于替换的转义序列列表，请参阅 gdb.prompt。每次显示提示符时，作为提示符字符串一部分指定的任何转义序列都会被替换为相应的字符串。

例如： 

set extended-prompt 当前工作目录：\w (gdb)

请注意，设置扩展提示符时，它会控制 prompt_hook 钩子。有关更多信息，请参阅 prompt_hook。

show extended-prompt
