17.1给变量赋值 ¶

要改变变量的值，需计算一个赋值表达式。见 表达式。例如， 

print x=4

将值4存储到变量 x 中，然后打印赋值表达式的值（即4）。 有关支持语言中的运算符的更多信息，请参见 使用GDB与不同语言。

如果您不想看到赋值的值，可以使用 set 命令代替 print 命令进行赋值。set 实际上与 print 相同，只是不打印表达式的值，也不将其放入值历史记录中（见 值历史记录）。该表达式仅为其效果而求值。

如果 set 命令的参数字符串开头看起来与 set 子命令相同，请使用 set variable 命令而不是仅使用 set。此命令与 set 相同，只是没有子命令。例如，如果您的程序有一个变量 width，仅尝试使用 ‘set width=13’ 设置新值时会出错，因为GDB有命令 set width： 

(gdb) whatis width
类型 = double
(gdb) p width
$4 = 13
(gdb) set width=47
表达式语法无效。

当然，无效表达式是 ‘=47’。要实际设置程序的变量 width，请使用 

(gdb) set var width=47

由于 set 命令有许多可能与程序变量名冲突的子命令，所以最好使用 set variable 命令而不是仅使用 set 的形式。例如，如果您的程序有一个变量 g，仅尝试使用 ‘set g=4’ 设置新值时会遇到问题，因为GDB有命令 set gnutarget，缩写为 set g： 

(gdb) whatis g
类型 = double
(gdb) p g
$1 = 1
(gdb) set g=4
(gdb) p g
$2 = 
(gdb) r
正在调试的程序已经启动。
从头开始启动它？（y或n）y
启动程序：/home/smith/cc_progs/a.out
“/home/smith/cc_progs/a.out”：无法打开以读取符号：
                                 无效的bfd目标。
(gdb) show g
当前的BFD目标是“=4”。

程序变量 g 没有改变，并且您默默地将 gnutarget 设置为了一个无效值。要设置变量 g，请使用 

(gdb) set var g=4

GDB在赋值中允许比C更多的隐式转换；您可以自由地将整数值存储到指针变量中，反之亦然，并且可以将任何结构体转换为长度相同或更短的任何其他结构体。

要将值存储到内存中的任意位置，请使用 ‘{...}’ 结构在指定地址生成指定类型的值（见 表达式）。例如，{int}0x83040 将内存位置 0x83040 视为一个整数（这意味着在内存中有一定的大小和表示形式），并且 

set {int}0x83040 = 4

将值4存储到该内存位置。
