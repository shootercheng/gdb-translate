10.3程序变量 ¶

最常用的表达式类型是程序中的变量名。

表达式中的变量在选定的栈帧中被理解（见 选择一个帧）；它们必须是：

  • 全局的（或文件静态的）

或者

  • 根据编程语言的作用域规则，从该帧中的执行点可见

这意味着在函数 

foo (a)
     int a;
{
  bar (a);
  {
    int b = test ();
    bar (b);
  }
}

中，只要程序在函数 foo 内执行，你就可以检查和使用变量 a，但只有当程序在声明 b 的块内执行时，你才能使用或检查变量 b。

有一个例外：即使当前执行点不在该文件中，你也可以引用作用域为单个源文件的变量或函数。但是，可能有多个这样的变量或函数具有相同的名称（在不同的源文件中）。如果发生这种情况，引用该名称会产生不可预测的效果。如果你愿意，你可以通过使用双冒号（::）表示法来指定特定函数或文件中的静态变量（并非所有语言都支持此操作，见 表达式）： 

文件::变量
函数::变量

这里 文件 或 函数 是静态 变量 的上下文名称。对于文件名，你可以使用引号以确保GDB将文件名解析为单个单词 —— 例如，要打印在 f2.c 文件中定义的全局变量 x 的值： 

(gdb) p 'f2.c'::x

双冒号（::）表示法通常用于引用静态变量，因为你通常通过选择适当的帧并使用变量的简单名称来消除函数中局部变量使用的歧义。但是你也可以使用此表示法来引用包含选定帧的帧中的局部变量： 

void
foo (int a)
{
  if (a < 10)
    bar (a);
  else
    process (a);    /* 在此处停止 */
}

int
bar (int a)
{
  foo (a + 5);
}

例如，如果在注释行处设置了断点，当程序在执行调用 bar(0) 后停止时，你可能会看到以下内容： 

(gdb) p a
$1 = 10
(gdb) p bar::a
$2 = 5
(gdb) up 2
#2  0x080483d0 in foo (a=5) at foobar.c:12
(gdb) p a
$3 = 5
(gdb) p bar::a
$4 = 0

这些对 ‘::’ 的使用很少与C++ 中相同表示法的非常相似的使用发生冲突。当它们发生冲突时，C++ 的含义优先；但是，这可以通过用单引号引起来文件或函数名称来覆盖。

例如，假设程序在一个具有名为 includefile 的字段的类的方法中停止，并且还有一个名为 includefile 的包含文件，它定义了一个变量 some_global。 

(gdb) p includefile
$1 = 23
(gdb) p includefile::some_global
表达式中存在语法错误，靠近 `'.
(gdb) p 'includefile'::some_global
$2 = 27

警告： 偶尔，局部变量在函数中的某些点可能看起来有错误的值 —— 就在进入新作用域之后，以及就在退出之前。

当你逐机器指令单步执行时，可能会看到这个问题。这是因为，在大多数机器上，设置一个栈帧（包括局部变量定义）需要不止一条指令；如果你逐机器指令单步执行，在栈帧完全构建之前，变量可能看起来有错误的值。在退出时通常也需要不止一条机器指令来销毁一个栈帧；在你开始逐行执行那组指令之后，局部变量定义可能就不存在了。

当编译器进行大量优化时，也可能会发生这种情况。为了始终确保看到准确的值，在编译时关闭所有优化。

编译器优化的另一个可能影响是将未使用的变量优化掉，或者将变量分配到寄存器（而不是内存地址）。根据编译器使用的数据格式对这种情况的支持程度，GDB可能无法显示此类局部变量的值。如果发生这种情况，GDB将打印如下消息： 

当前上下文中没有符号 "foo"。

要解决此类问题，要么不进行优化重新编译，要么如果编译器支持多种数据格式，则使用不同的数据格式。有关选择编译器选项，请参阅 为调试而编译 以获取更多信息。有关最适合C++ 程序的数据格式，请参阅 C和C++ 以获取更多信息。

如果你要求打印一个GDB不知道其内容的对象，例如，因为其数据类型未由调试信息完全指定，GDB将显示 ‘<不完整类型>’。有关此内容，请参阅 不完整类型 以获取更多信息。

如果你尝试检查或使用一个GDB没有类型信息的（全局）变量的值，例如，因为程序没有包含调试信息，GDB会显示一条错误消息。有关未知类型，请参阅 未知类型 以获取更多信息。如果你将变量强制转换为其声明的类型，GDB会使用强制转换后的类型作为变量的类型来获取变量的值。例如，在一个C程序中： 

  (gdb) p var
  'var' 具有未知类型；将其强制转换为其声明的类型
  (gdb) p (float) var
  $1 = 3.14

如果你在函数参数名后附加 @entry 字符串，你将得到函数调用时该参数的值。如果该值不可用，则会打印一条错误消息。入口值仅在某些编译器中可用。根据 设置打印入口值，入口值通常也会在函数参数列表中打印。 

Breakpoint 1, d (i=30) at gdb.base/entry-value.c:29
29	  i++;
(gdb) next
30	  e (i);
(gdb) print i
$1 = 31
(gdb) print i@entry
$2 = 30

字符串被识别为未指定符号性的 char 值数组。signed char 或 unsigned char 数组会被打印为1字节大小整数的数组。-fsigned-char 或 -funsigned-char GCC选项没有效果，因为GDB将字面字符串类型 "char" 定义为无符号的 char。对于程序代码 

char var0[] = "A";
signed char var1[] = "A";

在调试期间你会得到 

(gdb) print var0
$1 = "A"
(gdb) print var1
$2 = {65 'A', 0 '\0'}
