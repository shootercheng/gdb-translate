21.1.3 调试DJGPP程序的功能 ¶

DJGPP是GNU开发工具到MS-DOS和MS-Windows的一个移植。DJGPP程序是32位保护模式程序，它们使用DPMI（DOS保护模式接口）API在实模式DOS系统及其仿真之上运行。

GDB支持对DJGPP程序进行原生调试，并定义了一些特定于DJGPP移植的命令。本小节描述这些命令。

info dos ¶

这是特定于DJGPP的命令前缀，用于打印有关目标系统和重要OS结构的信息。

info dos sysinfo ¶

此命令显示有关底层平台的各种信息：CPU类型和特性、OS版本和风味、DPMI版本以及可用的常规和DPMI内存。

info dos gdt ¶
info dos ldt
info dos idt

这3个命令分别显示全局、局部和中断描述符表（GDT、LDT和IDT）的条目。描述符表是数据结构，用于存储当前正在使用的每个段的描述符。段的选择器是描述符表中的一个索引；该索引的表条目包含描述符的基地址和界限，以及其属性和访问权限。

一个典型的DJGPP程序使用3个段：一个代码段、一个数据段（用于数据和堆栈）和一个DOS段（允许访问DOS/BIOS数据结构和常规内存中的绝对地址）。然而，DPMI主机通常会定义其他段以支持DPMI环境。

这些命令允许显示描述符表的条目。不带参数时，将显示指定表的所有条目。一个参数，应该是一个整数表达式，表示显示索引由该参数给出的单个条目。例如，这是显示已调试程序的数据段信息的一种便捷方法： 

(gdb) info dos ldt $ds

0x13f: base=0x11970000 limit=0x0009ffff 32位数据（读/写，扩展向上）

当你想查看指针是否超出数据段的界限（即乱码）时，这很有用。

info dos pde ¶
info dos pte

这两个命令分别显示页目录和页表的条目。页目录和页表是控制虚拟内存地址如何映射到物理地址的数据结构。一个页表包含映射到程序地址空间的每个内存页的一个条目；可能有几个页表，每个页表最多可容纳4096个条目。一个页目录最多有4096个条目，每个当前正在使用的心页表都有一个条目。

不带参数时，info dos pde显示整个页目录，info dos pte显示所有页表中的所有条目。给info dos pde命令一个参数，一个整数表达式，表示只显示页目录表中的那个条目。给info dos pte命令一个参数，表示显示由页目录中指定条目指向的单个页表的条目。

当你的程序使用DMA（直接内存访问）时这些命令很有用，DMA需要物理地址来对DMA控制器进行编程。

这些命令仅在某些DPMI服务器上受支持。

info dos address-pte addr ¶

此命令显示指定线性地址的页表条目。参数addr是一个线性地址，它应该已经添加了适当段的基地址，因为此命令接受可能属于任何段的地址。例如，这是显示存储变量i的页面的页表条目的方法： 

(gdb) info dos address-pte __djgpp_base_address + (char *)&i

地址0x11a00d30的页表条目：

基地址=0x02698000 已修改 可访问 未缓存 回写 用户 读/写 +0xd30

这表示i存储在物理基地址为0x02698000的页面的偏移量0xd30处，并显示该页面上的所有属性。

请注意，你必须将变量的地址强制转换为char *，否则，DJGPP程序中所有变量和函数的基地址__djgpp_base_address的值将使用C指针算术规则添加：如果i被声明为int型，GDB将把__djgpp_base_address值的4倍添加到i 的地址上。

这是另一个例子，它显示传输缓冲区的页表条目： 

(gdb) info dos address-pte *((unsigned *)&_go32_info_block + 3)

地址0x29110的页表条目：

基地址=0x00029000 已修改 可访问 未缓存 回写 用户 读/写 +0x110

（偏移量+ 3是因为传输缓冲区的地址是_go32_info_block结构的第3个成员。）输出清楚地表明，此DPMI服务器将常规内存中的地址1:1映射，即物理地址（0x00029000 + 0x110）和线性地址（0x29110）是相同的。

此命令仅在某些DPMI服务器上受支持。

除了原生调试外，DJGPP移植还支持通过串行数据链路进行远程调试。以下命令特定于GDB的DJGPP移植中的远程串行调试。

set com1base addr ¶

此命令设置COM1串行端口的基I/O端口地址。

set com1irq irq

此命令设置用于COM1串行端口的中断请求 (IRQ) 线。

有类似的命令 “set com2base”、“set com3irq” 等，用于设置其他3个COM端口的端口地址和IRQ线。

相关命令 “show com1base”、“show com1irq” 等显示COM端口使用的基地址和IRQ线的当前设置。

info serial ¶

此命令打印4个DOS串行端口的状态。对于每个端口，它打印该端口是否处于活动状态、其I/O基地址和IRQ编号、是否使用16550风格的FIFO、其波特率以及到目前为止遇到的各种错误的计数。
