23.‌4.‌3.‌10编写Guile漂亮打印机 ¶

一个漂亮打印机由两个基本部分组成：一个查找函数，用于确定类型是否受支持；以及打印机本身。

以下是一个示例代码，展示了如何编写一个用于打印 std::string 的打印机。有关详细信息，请参阅 Guile漂亮打印API。 

(define (make-my-string-printer value)
  "打印一个my::string字符串"
  (make-pretty-printer-worker
   "string"
   (lambda (printer)
     (value-field value "_data"))
   #f))

以下是一个示例代码，展示了如何为上述打印机编写查找函数。 

(define (str-lookup-function pretty-printer value)
  (let ((tag (type-tag (value-type value))))
    (and tag
         (string-prefix? "std::string<" tag)
         (make-my-string-printer value))))

然后，要将此打印机注册到全局打印机列表中： 

(append-pretty-printer!
 (make-pretty-printer "my-string" str-lookup-function))

示例查找函数提取值的类型，并尝试将其与可以漂亮打印的类型进行匹配。如果它是打印机可以漂亮打印的类型，它将返回一个 <gdb:pretty-printer-worker> 对象。如果不是，它返回 #f。

我们建议将核心漂亮打印机放入Guile包中。如果你的漂亮打印机用于某个库，我们进一步建议在包名中嵌入版本号。这种做法将使 GDB 能够同时加载你的漂亮打印机的多个版本，因为它们将有不同的名称。

你应该编写自动加载代码（请参阅 Guile自动加载），使其可以多次求值而不改变其含义。一个理想的自动加载文件将仅由你的打印机模块的 import 组成，然后是一个调用，用于向当前目标文件注册漂亮打印机。

总体而言，这种方法可以很好地扩展到多个进程，每个进程可能使用不同版本的库。在Guile包名中嵌入版本号将确保 GDB 能够同时加载两组打印机。然后，因为漂亮打印机的搜索是按目标文件进行的，并且因为你的自动加载代码小心地为特定目标文件注册了库的打印机，所以 GDB 将为每个进程使用的库的特定版本找到正确的打印机。

以 my::string 为例，这段代码可能出现在 (my-project my-library v1) 中： 

(use-modules (gdb))
(define (register-printers objfile)
  (append-objfile-pretty-printer!
   (make-pretty-printer "my-string" str-lookup-function)))

然后自动加载文件的相应内容将是： 

(use-modules (gdb) (my-project my-library v1))
(register-printers (current-objfile))

前面的示例说明了一个基本的漂亮打印机。有一些可以改进的地方。该打印机只处理一种类型，而一个库通常有几种类型。可以为库中每个所需类型安装一个查找函数，但也可以有一个单一的查找函数识别几种类型。后者是处理这个问题的传统方法。如果一个漂亮打印机可以处理多种数据类型，那么它的 子打印机 就是用于各个数据类型的打印机。
