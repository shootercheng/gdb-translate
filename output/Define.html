23.1.1 用户定义的命令 ¶

一个用户定义的命令是一组GDB命令序列，你可以为其指定一个新名称作为命令。这是通过define命令完成的。

用户命令可以接受无限数量的由空格分隔的参数。在用户命令中，可以通过$arg0…$argN来访问参数。这些参数是文本替换，因此它们可以引用变量、使用复杂表达式，甚至执行低级函数调用。然而，请注意，这种文本替换意味着处理某些参数会很困难。例如，如果用户要传递一个包含空格的参数，则无法实现；虽然可以使用像"$arg1"这样的表达式将参数转换为字符串，但如果参数包含引号，将会失败。对于更复杂和健壮的命令，我们建议用Python编写；请参阅 Python中的CLI命令。

一个简单的例子： 

define adder
  print $arg0 + $arg1 + $arg2
end

要执行该命令，请使用： 

adder 1 2 3

这定义了命令adder，它会打印其三个参数的和。

此外，$argc可用于查明传递了多少个参数。 

define adder
  if $argc == 2
    print $arg0 + $arg1
  end
  if $argc == 3
    print $arg0 + $arg1 + $arg2
  end
end

与eval命令（请参阅 eval）结合使用，可以更轻松地处理数量可变的参数： 

define adder
  set $i = 0
  set $sum = 0
  while $i < $argc
    eval "set $sum = $sum + $arg%d", $i
    set $i = $i + 1
  end
  print $sum
end
define commandname ¶

定义一个名为commandname的命令。如果已经存在同名的命令，系统会要求你确认是否要重新定义它。参数commandname可以是由字母、数字、破折号、点和下划线组成的简单命令名。它也可以以任何预定义或用户定义的前缀命令开头。 例如，‘define target my-target’ 创建一个用户定义的‘target my-target’ 命令。

命令的定义由其他GDB命令行组成，这些命令行在define命令之后给出。这些命令的结尾由包含end的行标记。

document commandname ¶

记录用户定义的命令commandname，以便可以通过help访问。命令commandname必须已经定义。此命令读取文档行的方式与define读取命令定义行的方式相同，以end结尾。在document命令完成后，对命令commandname执行help将显示你编写的文档。

你可以再次使用document命令来更改命令的文档。使用define重新定义命令不会更改文档。

也可以记录用户定义的别名。然后，help和apropos命令将使用别名文档，而不是被别名命令的文档。当将别名定义为一组嵌套with命令时（请参阅 默认参数），记录用户定义的别名特别有用并且。

define-prefix commandname ¶

将命令commandname定义或标记为用户定义的前缀命令。一旦标记，commandname就可以被define命令用作前缀命令。 请注意，define-prefix可以与尚未定义的commandname一起使用。在这种情况下，commandname被定义为空的用户定义命令。 如果你重新定义一个被标记为用户定义前缀命令的命令，重新定义的命令的子命令将被保留（并且GDB会向用户表明这一点）。

示例： 

(gdb) define-prefix abc
(gdb) define-prefix abc def
(gdb) define abc def
输入“abc def”的定义命令。
以仅包含“end”的行结束。
>echo command initial def\n
>end
(gdb) define abc def ghi
输入“abc def ghi”的定义命令。
以仅包含“end”的行结束。
>echo command ghi\n
>end
(gdb) define abc def
保留前缀命令“def”的子命令。
重新定义命令“def”？（y或n）y
输入“abc def”的定义命令。
以仅包含“end”的行结束。
>echo command def\n
>end
(gdb) abc def ghi
command ghi
(gdb) abc def
command def
(gdb)
