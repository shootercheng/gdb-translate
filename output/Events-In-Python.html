23.3.2.18 Python中的事件 ¶

GDB提供了一个通用的事件机制，以便Python代码能够接收到各种状态变化的通知，特别是下级中发生的变化。

一个事件就是一个描述某种状态变化的对象。对象的类型及其属性会根据变化的细节而有所不同。下面将描述所有现有的事件。

为了接收到事件通知，你必须向一个事件注册表注册一个事件处理程序。事件注册表是gdb.events模块中的一个对象，用于分发特定的事件。注册表提供了注册和注销事件处理程序的方法：

函数： EventRegistry.connect (对象) ¶

将给定的可调用对象对象添加到注册表中。当与该注册表对应的事件发生时，这个对象将会被调用。

函数： EventRegistry.disconnect (对象) ¶

从注册表中移除给定的对象对象。一旦移除，该对象将不再接收事件通知。

下面是一个示例： 

def exit_handler (event):
    print ("事件类型：exit")
    if hasattr (event, 'exit_code'):
        print ("退出代码：%d" % (event.exit_code))
    else:
        print ("退出代码不可用")

gdb.events.exited.connect (exit_handler)

在上面的示例中，我们将处理程序exit_handler连接到注册表events.exited。连接后，当下级退出时，exit_handler就会被调用。此示例中的参数事件是gdb.ExitedEvent类型。正如你在示例中看到的，ExitedEvent对象有一个属性，用于指示下级的退出代码。

当GDB以非停止模式运行时，某些事件可能是特定于线程的。在Python中表示时，这些事件都扩展了gdb.ThreadEvent。这个事件是一个基类，从不直接发出；相反，由这个或其他模块发出的事件可能会扩展这个事件。这些事件的示例包括gdb.BreakpointEvent和gdb.ContinueEvent。gdb.ThreadEvent具有以下属性：

变量： ThreadEvent.inferior_thread ¶

在非停止模式下，如果事件涉及特定线程，则此属性将设置为该线程。否则，它将设置为None。

以下是可用的事件注册表及其发出的事件的详细信息列表：

events.cont

发出gdb.ContinueEvent，它扩展了gdb.ThreadEvent。 此事件表示下级在停止后已继续运行。有关继承属性，请参考上面的gdb.ThreadEvent。

events.exited

发出events.ExitedEvent，表示下级已退出。events.ExitedEvent有两个属性：

变量： ExitedEvent.exit_code ¶

一个整数，表示下级返回的退出代码（如果可用）。（例如，如果GDB与下级分离，退出代码可能不可用。）如果退出代码不可用，则该属性不存在。

变量： ExitedEvent.inferior ¶

对触发exited事件的下级的引用。

events.stop

发出gdb.StopEvent，它扩展了gdb.ThreadEvent。

表示下级已停止。此注册表发出的所有事件都扩展了gdb.StopEvent。作为gdb.ThreadEvent的子类，当GDB以非停止模式运行时，gdb.StopEvent将指示被停止的线程。有关更多详细信息，请参考上面的gdb.ThreadEvent。

gdb.StopEvent具有以下附加属性：

变量： StopEvent.details ¶

一个字典，包含与停止相关的任何详细信息。确切的键和值取决于停止的类型，但与相应的MI输出相同（请参阅 GDB/MI 异步记录）。

使用字典是为了（而不是直接向事件对象添加属性），以便可以不变地使用MI键。

当StopEvent是由finish命令产生时，如果可用，它还将包含函数的返回值。这将是details字典中名为‘return-value’的条目。该条目的值将是一个gdb.Value对象。

发出gdb.SignalEvent，它扩展了gdb.StopEvent。

此事件表示下级或其某个线程已收到信号。gdb.SignalEvent具有以下属性：

变量： SignalEvent.stop_signal ¶

一个字符串，表示下级收到的信号。通过在GDB命令提示符中运行info signals命令，可以获取可能的信号值列表。

还发出gdb.BreakpointEvent，它扩展了gdb.StopEvent。

gdb.BreakpointEvent事件表示一个或多个断点已被命中，并具有以下属性：

变量： BreakpointEvent.breakpoints ¶

一个序列，包含对所有被命中的断点（类型为gdb.Breakpoint）的引用。 有关gdb.Breakpoint对象 的详细信息，请参阅 使用Python操作断点。

变量： BreakpointEvent.breakpoint ¶

对第一个被命中的断点的引用。此属性是为了向后兼容而保留的，现在已被弃用，建议使用gdb.BreakpointEvent.breakpoints属性。
