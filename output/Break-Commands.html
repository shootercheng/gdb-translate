5.1.7断点命令列表 ¶

你可以为任何断点（或观察点或捕获点）指定一系列命令，当你的程序因该断点而停止时执行这些命令。例如，你可能想要打印某些表达式的值，或者启用其他断点。

commands [列表…] ¶
… 命令列表 …
end

为给定的断点指定一个命令列表。命令本身出现在接下来的行中。输入一行只包含end的行来终止命令。

要从一个断点中删除所有命令，输入commands并立即跟一个end；也就是说，不输入任何命令。

不带参数时，commands指的是最后设置的断点、观察点或捕获点（不是最近遇到的断点）。如果最近的断点是用一个命令设置的，那么commands将应用于该命令设置的所有断点。这适用于由rbreak设置的断点，也适用于当一个单一的break命令创建多个断点时（见 模糊表达式）。

在一个命令列表中，按下RET作为重复上一个GDB命令的方式是被禁用的。

在命令列表中，你可以使用命令disable $_hit_bpnum来禁用遇到的断点。

如果你的断点有多个代码位置，命令disable $_hit_bpnum.$_hit_locno将禁用遇到的特定断点代码位置。如果断点只有一个位置，这个命令将禁用遇到的断点。

z

你可以使用断点命令再次启动你的程序。只需使用continue命令，或step，或任何其他恢复执行的命令。

在恢复执行的命令之后，命令列表中的任何其他命令都将被忽略。这是因为每当你恢复执行（即使是使用简单的next或step）时，你可能会遇到另一个断点—它可能有自己的命令列表，这会导致关于执行哪个列表的歧义。

如果你在命令列表中指定的第一个命令是silent，则不会打印通常关于在断点处停止的消息。对于要打印特定消息然后继续的断点，这可能是可取的。如果其余命令都不打印任何内容，你将看不到断点被命中的迹象。silent仅在断点命令列表的开头有意义。

echo、output和printf命令允许你打印精确控制的输出，并且在静默断点中通常很有用。见 用于控制输出的命令。

例如，这是你如何使用断点命令在每次x为正时在进入foo时打印x的值。 

break foo if x>0
commands
silent
printf "x is %d\n",x
cont
end

断点命令的一个应用是弥补一个错误，以便你可以测试另一个错误。在错误的代码行之后设置一个断点，给它一个条件来检测错误发生的情况，并给它命令来为任何需要的变量赋正确的值。以continue命令结束，这样你的程序就不会停止，并以silent命令开始，这样就不会产生输出。这是一个例子： 

break 403
commands
silent
set x = y + 4
cont
end
