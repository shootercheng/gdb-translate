29调试器适配器协议 ¶

调试器适配器协议是一种通用API，一些IDE使用它与调试器进行通信。其文档位于 https://microsoft.github.io/debug-adapter-protocol/。

一般来说，GDB 按文档实现调试器适配器协议。然而，在某些情况下，需要甚至期望进行扩展。

GDB 定义了一些可以传递给 launch 请求的参数：

args

如果提供，这应该是一个字符串数组。这些字符串作为命令行参数提供给子进程，就像通过 set args 一样。见 程序的参数。

cwd

如果提供，这应该是一个字符串。GDB 将其工作目录更改为该目录，就像通过 cd 命令一样（见 程序的工作目录）。启动的程序将继承此作为其工作目录。请注意，目录更改在处理 program 参数之前发生。如果 program 是一个相对文件名，这将影响结果。

env

如果提供，这应该是一个对象。对象的每个键将用作环境变量的名称；每个值必须是一个字符串，并且将是该变量的值。子进程的环境将设置为与传入的完全相同。见 程序的环境。

program

如果提供，这是一个指定要使用的程序的字符串。这对应于 file 命令。见 指定文件的命令。

stopAtBeginningOfMainSubprogram

如果提供，这必须是一个布尔值。当为 ‘True’ 时，GDB 将在程序的主过程中设置一个临时断点，使用与 start 命令相同的方法。见 启动你的程序。

stopOnEntry

如果提供，这必须是一个布尔值。当为 ‘True’ 时，GDB 将在程序的第一条指令处设置一个临时断点，使用与 starti 命令相同的方法。见 启动你的程序。

GDB 定义了一些可以传递给 attach 请求的参数。必须指定 pid 或 target，但如果两者都指定，则 target 将被忽略。

pid

GDB 应附加到的进程ID。见 调试一个正在运行的进程。

program

如果提供，这是一个指定要使用的程序的字符串。这对应于 file 命令。见 指定文件的命令。在某些情况下，GDB 可以自动确定正在运行哪个程序。然而，对于许多远程目标，情况并非如此，因此应提供此参数。

target

GDB 应连接到的目标。这是一个字符串，并传递给 target remote 命令。见 连接到远程目标。

响应 disassemble 请求时，DAP允许客户端以实现定义的格式返回每条指令的字节。GDB 通过发送一个用十六进制编码字节的字符串来实现这一点，例如 "55a2b900"。

当 repl 上下文用于 evaluate 请求时，GDB 将提供的表达式作为CLI命令进行求值。

一般来说，求值可能会导致子进程继续执行。例如，求值 continue 命令可能会这样，求值涉及子进程函数调用的表达式也可能会这样。

repl 求值也可能导致 GDB 似乎停止响应请求，例如，如果一个CLI脚本进行长时间计算。

这样的求值可以使用DAP cancel 请求中断。（实际上，cancel 应该对任何请求都有效，但对大多数请求可能没有用。）

GDB 提供了一些可以在DAP模式下使用的日志记录设置。可以使用 -iex 选项在命令行上设置这些设置（见 选择文件）。

set debug dap-log-file [filename]

启用DAP日志记录。日志写入 filename。如果未给出 filename，则停止日志记录。

set debug dap-log-level level

设置DAP日志记录级别。默认值为 ‘1’，它记录DAP协议、开发人员认为有用的任何调试消息以及意外异常。级别 ‘2’ 可用于记录所有异常，包括被认为是预期的异常。例如，解析表达式失败将被视为正常异常，通常不会记录。
