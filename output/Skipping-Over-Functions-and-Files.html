5.3 跳过函数和文件 ¶

你正在调试的程序可能包含一些调试起来没有意义的函数。skip命令可以让你告诉GDB在单步执行时跳过一个函数、一个文件中的所有函数或者一个特定文件中的某个特定函数。

例如，考虑以下C函数： 

101     int func()
102     {
103         foo(boring());
104         bar(boring());
105     }

假设你希望单步进入函数foo和bar，但你对单步执行boring不感兴趣。如果你在第103行运行step，你将进入boring()，但如果你运行next，你将跳过foo和boring！

一种解决方法是单步进入boring并使用finish命令立即退出它。但如果从很多地方调用boring，这可能会变得很繁琐。

一个更灵活的解决方法是执行skip boring。这会指示GDB永远不要单步进入boring。现在当你在第103行执行step时，你将跳过boring并直接进入foo。

可以通过提供函数名、行规范（见 位置规范）、与函数名匹配的正则表达式、文件名或与文件名匹配的glob风格模式来跳过函数。

在Posix系统上，正则表达式的形式是“扩展正则表达式”。例如，在GNU/Linux系统上查看 ‘man 7 regex’。在非Posix系统上，正则表达式的形式是底层系统的regcomp函数提供的任何形式。 例如，在GNU/Linux系统上查看 ‘man 7 glob’ 以了解glob风格模式的描述。

skip [选项] ¶

skip命令的基本形式接受零个或多个指定要跳过内容的选项。 选项参数是以下任何有用的组合：

-file 文件
-fi 文件

在单步执行时将跳过文件中的函数。

-gfile 文件-glob-模式 ¶
-gfi 文件-glob-模式

在单步执行时将跳过与文件-glob-模式匹配的文件中的函数。 

(gdb) skip -gfi utils/*.c
-function 行规范
-fu 行规范

由行规范命名的函数或包含由行规范命名行的函数在单步执行时将被跳过。 见 位置规范。

-rfunction 正则表达式 ¶
-rfu 正则表达式

名称与正则表达式匹配的函数在单步执行时将被跳过。

这种形式对于复杂的函数名很有用。 例如，通常没有必要单步进入C++ std::string的构造函数或析构函数加上对于C++模板，很难写出函数的全名，而且通常模板参数是什么并不重要。将要跳过的函数指定为正则表达式会使这更容易。 

(gdb) skip -rfu ^std::(allocator|basic_string)<.*>::~?\1 *\(

如果你想跳过std命名空间中每个模板化的C++构造函数和析构函数，你可以这样做： 

(gdb) skip -rfu ^std::([a-zA-z0-9_]+)<.*>::~?\(.*\)

如果未指定选项，将跳过你当前正在调试的函数。

skip function [行规范] ¶

运行此命令后，由行规范命名的函数或包含由行规范命名行的函数在单步执行时将被跳过。见 位置规范。

如果你未指定行规范, 将跳过你当前正在调试的函数。

（如果你有一个名为file的函数想要跳过，使用skip function file。）

skip file [文件名] ¶

运行此命令后，任何源文件位于文件名中的函数在单步执行时将被跳过。 

(gdb) skip file boring.c
文件boring.c在单步执行时将被跳过。

如果你未指定文件名, 将跳过源文件位于你当前正在调试的文件中的函数。

跳过项可以像断点一样被列出、删除、禁用和启用用于管理你的跳过列表的命令如下：

info skip [范围] ¶

打印指定跳过项的详细信息。如果未指定范围，打印一个包含所有标记为要跳过的函数和文件详细信息的表格。info skip会打印每个跳过项的以下信息：

标识符

标识此跳过项的数字。

启用或禁用

启用的跳过项标记为‘y’。 禁用的跳过项标记为‘n’。

Glob

如果文件名是一个‘glob’模式，这是‘y’。 否则是‘n’。

文件

要跳过的文件的名称或‘glob’模式。 如果未指定文件，这是‘<无>’。

RE

如果函数名是一个‘正则表达式’，这是‘y’。 否则是‘n’。

函数

要跳过的函数的名称或正则表达式。 如果未指定函数，这是‘<无>’。

skip delete [范围] ¶

删除指定的跳过项。如果未指定范围，删除所有跳过项。

skip enable [范围] ¶

启用指定的跳过项。如果未指定范围，启用所有跳过项。

skip disable [范围] ¶

禁用指定地跳过项。如果未指定范围，禁用所有跳过项。

set debug skip [on|off] ¶

设置是否打印关于跳过文件和函数的调试输出。

show debug skip ¶

显示是否打印关于跳过文件和函数的调试输出？
