22.5 输出样式 ¶

GDB 可以在支持的终端上设置其输出样式。在大多数系统上，这在默认情况下是启用的，但在批处理模式下默认是禁用的（见 选择模式)。有各种样式设置可用；并且样式也可以完全禁用。

set style enabled ‘on|off’

启用或禁用所有样式。默认值取决于主机，大多数主机默认设置为 ‘on’。

如果 NO_COLOR 环境变量设置为非空值，那么 GDB 将在启动时将此设置更改为 ‘off’。

show style enabled

显示当前的样式状态。

set style emoji ‘auto|on|off’

启用或禁用emoji的使用。在大多数主机上，默认值是 ‘auto’，这意味着只有当主机字符集为 ‘UTF-8’ 时才会使用emoji；但是，在使用控制台时，Windows上的默认值是 ‘off’。请注意，整体禁用样式也会阻止emoji显示。

目前，每当 GDB 报告错误或警告时，都会打印emoji。

show style emoji

显示当前的emoji输出状态。

set style sources ‘on|off’

启用或禁用源代码样式。这会影响源代码（如 list 命令的输出）是否有样式。默认值是 ‘on’。请注意，只有在总体上启用样式并且 GDB 有可用的源突出显示库时，源样式才会起作用。

有两种突出显示的方式。首先，如果 GDB 使用GNU源突出显示库进行链接，那么将使用该库。否则，如果 GDB 配置了Python脚本支持，并且如果有Python Pygments包可用，那么将使用该包。

show style sources

显示当前的源代码样式状态。

set style warning-prefix
show style warning-prefix
set style error-prefix
