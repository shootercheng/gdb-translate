9.3编辑源文件 ¶

要编辑源文件中的行，请使用edit命令。 你选择的编辑程序 会以程序中的活动行为当前行来调用。 或者，如果您想查看程序的其他部分，有几种方法可以指定要打印文件的哪一部分：

edit 位置规范

编辑解析位置规范后得到的代码位置的源文件。编辑从源文件开始，从位置规范解析到的源文件和行开始。 有关位置规范参数的所有可能形式，请参阅 位置规范。

如果位置规范在你的程序中解析为多个源行，那么该命令会打印解析后的源行列表，并且不会继续进行编辑。

以下是最常用的edit命令形式：

edit 编号

以编号作为活动行号编辑当前源文件。

edit 函数

在其定义开始处编辑包含函数 的文件。

  • 选择你的编辑器

9.3.1选择你的编辑器 ¶

你可以自定义GDB以使用任何你想要的编辑器 10。 默认情况下，它是/bin/ex，但你可以在使用GDB之前通过设置环境变量EDITOR来更改此设置。例如，要将GDB配置为使用vi编辑器，你可以在sh shell中使用以下命令： 

EDITOR=/usr/bin/vi
export EDITOR
gdb ...

或者在csh shell中， 

setenv EDITOR /usr/bin/vi
gdb ...

脚注

(10)

唯一的限制是你的编辑器（例如ex）必须识别以下命令行语法： 

ex +编号 文件

可选的数值 +编号 指定文件中开始编辑的行号。
