22.9 可选警告和消息 ¶

默认情况下，GDB 对其内部工作情况保持沉默。如果你在一台慢速机器上运行，你可能想使用 set verbose 命令。这会让 GDB 在执行冗长的内部操作时告诉你，这样你就不会认为它崩溃了。

目前，由 set verbose 控制的消息是那些宣布正在读取源文件的符号表的消息；请参阅 指定文件的命令 中的 symbol-file。

set verbose on ¶

启用 GDB 输出某些信息性消息。

set verbose off

禁用 GDB 输出某些信息性消息。

show verbose ¶

显示 set verbose 是开启还是关闭。

默认情况下，如果 GDB 在目标文件的符号表中遇到错误，它会保持沉默；但如果你正在调试一个编译器，你可能会发现这些信息很有用（请参阅 读取符号文件时的错误）。

set complaints limit ¶

允许 GDB 在对每种异常符号类型输出 limit 条抱怨信息后，才对该问题保持沉默。将 limit 设置为零以抑制所有抱怨；将其设置为一个较大的数字以防止抱怨被抑制。

show complaints ¶

显示 GDB 被允许产生的符号抱怨数量。

默认情况下，GDB 很谨慎，会问一些有时看起来很愚蠢的问题来确认某些命令。例如，如果您尝试运行一个已经在运行的程序： 

(gdb) run
正在调试的程序已经启动。
从开头重新启动它？(y或n)

如果您愿意毫不犹豫地面对自己命令的后果，您可以禁用这个“功能”：

set confirm off ¶
