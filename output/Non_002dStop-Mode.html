5.5.2 不间断模式 ¶

对于某些多线程目标，GDB支持一种可选的操作模式，在这种模式下，你可以在调试器中检查已停止的程序线程，而其他线程继续自由执行。这在调试实时系统（如某些线程有实时约束或必须继续响应外部事件的程序）时将干扰降至最低。这被称为不间断模式。

在不间断模式下，当一个线程停止以报告调试事件时，只有那个线程会停止；与全停止模式的行为不同，GDB不会停止其他线程。此外，在不间断模式下，诸如continue和step等执行命令默认仅应用于当前线程，而不是像在全停止模式下那样应用于所有线程。这使你能够以全停止模式无法实现的方式显式控制线程 —— 例如，单步执行一个线程，同时允许其他线程自由运行；单步执行一个线程，同时保持其他所有线程停止；或者独立且同时地单步执行多个线程。

要进入不间断模式，请在运行或附加到程序之前使用以下命令序列： 

# 如果使用CLI，分页会破坏不间断模式。
set pagination off

# 最后，打开它！
set non-stop on

你可以使用以下命令来操作不间断模式设置：

set non-stop on ¶

启用不间断模式选择。

set non-stop off

禁用不间断模式选择。

show non-stop

显示当前不间断启用设置。

请注意，这些命令仅反映是否启用了不间断模式，而不是当前正在执行的程序是否以不间断模式运行。特别是，set non-stop首选项仅在GDB启动或连接到目标程序时才会被参考，并且一旦调试开始，通常不可能切换模式。此外，由于并非所有目标都支持不间断模式，即使你启用了不间断模式，GDB默认情况下仍可能回退到全停止操作。

在不间断模式下，所有执行命令默认仅应用于当前线程。也就是说，continue仅继续一个线程。要继续所有线程，请发出continue -a或c -a。
