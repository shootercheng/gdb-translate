27.16 GDB/MI数据操作 ¶

[]

本节描述用于操作数据的GDB/MI命令： 检查内存和寄存器、评估表达式等。

有关可寻址存储单元的详细信息，请参阅 可寻址存储单元。

The -data-disassemble命令 ¶

概要 ¶

 -data-disassemble
  ( -s start-addr -e end-addr
  | -a addr
  | -f filename -l linenum [ -n lines ] )
  [ --opcodes opcodes-mode ]
  [ --source ]
  [ -- mode ]

其中：

‘start-addr’

是起始地址（或$pc）

‘end-addr’

是结束地址

‘addr’

是要反汇编的函数内任意位置的地址（或函数名）。如果指定了地址，将反汇编该地址周围的整个函数。如果指定了名称，将反汇编具有该名称的整个函数。

‘filename’

是要反汇编的文件名

‘linenum’

是要围绕其进行反汇编的行号

‘lines’

是要生成的反汇编行数。如果是-1，在未指定end-addr的情况下，将反汇编整个函数。如果end - addr被指定为非零值，并且lines低于start - addr和end - addr之间的反汇编行数，则仅显示lines行；如果lines高于start - addr和end - addr之间的行数，则仅显示到end - addr的行。

‘opcodes-mode’

只能与mode 0一起使用，应为以下之一：

‘none’

结果中不包含操作码信息。

‘bytes’

结果中将包含操作码，操作码将按照 disassemble `/b` 的格式进行格式化。

‘display’

结果中将包含操作码，操作码将按照 disassemble `/r` 的格式进行格式化。

‘mode’

现在不建议使用mode，建议使用 --opcodes 和 --source 选项。如果未给出mode，将假定为mode 0。但是为了向后兼容，mode 仍然可用。mode 应为以下之一：

‘0’

仅反汇编，如果未指定模式，这是默认模式。
