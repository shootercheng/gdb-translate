3.6 获取帮助 ¶

你可以随时使用命令help向GDB本身询问有关其命令的信息。

help ¶
h

你可以在不带参数的情况下使用help（缩写为h）来显示命令的命名类别的简短列表： 

(gdb) help
命令类列表：

别名 -- 其他命令的用户定义别名
断点 -- 使程序在特定点停止
数据 -- 检查数据
文件 -- 指定和检查文件
内部 -- 维护命令
晦涩 -- 晦涩的功能
运行 -- 运行程序
栈 -- 检查栈
状态 -- 状态查询
支持 -- 支持工具
跟踪点 -- 在不停止程序的情况下跟踪程序执行
用户定义 -- 用户定义的命令

在“help”后面跟一个类名以获取该类中的命令列表。
在“help”后面跟命令名以获取完整文档。
如果明确无误，允许使用命令名缩写。
(gdb)
help 类

将通用帮助类之一作为参数使用，你可以获取该类中各个命令的列表。如果一个命令有别名，别名会在命令名之后给出，用逗号分隔。如果一个别名有默认参数，别名的完整定义会在第一行之后给出。 例如，这是类status的帮助显示： 

(gdb) help status  
状态查询。

命令列表：

info, inf, i -- 用于显示有关正在调试的程序的信息的通用命令
        info address, iamain  -- 描述符号SYM存储的位置。
  别名iamain = info address main
info all-registers -- 所选栈帧的所有寄存器及其内容的列表，
        ...
show, info set -- 用于显示有关调试器的信息的通用命令

在“help”后面跟命令名以获取完整文档。
如果明确无误，允许使用命令名缩写。
(gdb)
help 命令

将命令名作为help的参数，GDB会显示一段关于如何使用该命令的简短段落。如果该命令有一个或多个别名，GDB会显示第一行，其中命令名和所有别名用逗号分隔。 这第一行之后将是所有具有默认参数的别名的完整定义。 当询问别名的帮助时，会显示被别名化的命令的文档。

用户定义的别名可以使用document命令进行可选的记录（见 document）。然后GDB会认为这个别名与被别名化的命令不同：这个别名不会列在被别名化的命令帮助输出中，并且询问这个别名的帮助会显示为该别名提供的文档，而不是被别名化的命令的文档。

apropos [-v] 正则表达式 ¶

apropos命令在所有GDB命令、别名及其文档中搜索args中指定的正则表达式。它会打印出找到的所有匹配项。可选标志“‘-v’”，代表“‘详细’”，表示输出匹配命令的完整文档，并突出显示文档中与regexp匹配的部分。例如： 

apropos alias

结果如下： 

alias -- 定义一个新命令，它是现有命令的别名
aliases -- 其他命令的用户定义别名

而 

apropos -v cut.*thread apply

结果如下，其中如果启用了样式，“‘cut for 'thread apply’”会被突出显示。 

taas -- 将一个命令应用于所有线程（忽略错误
和空输出）。
用法：taas COMMAND
'thread apply all -s COMMAND'的快捷方式

tfaas -- 将一个命令应用于所有线程的所有帧
（忽略错误和空输出）。
用法： tfaas COMMAND
'thread apply all -s frame apply all -s COMMAND'的快捷方式
complete 参数 ¶

complete args命令列出命令开头的所有可能的补全。使用args指定要补全的命令的开头。例如： 

complete i

结果如下： 

if
ignore 
info 
inspect

这旨在供GNU Emacs使用。

除了help之外，你还可以使用GDB命令info和show来询问你的程序的状态，或者GDB本身的状态。每个命令都支持许多查询主题；本手册在适当的上下文中介绍了它们中的每一个。命令、变量和函数索引中info和show下的列表指向所有子命令。见 命令 - 变量和函数索引。

info ¶

这个命令（缩写为i）用于描述你的程序的状态。例如，你可以使用info args显示传递给函数的参数，使用info registers列出当前正在使用的寄存器，或者使用info breakpoints列出你设置地断点。 你可以使用help  info获取info子命令的完整列表。

set ¶

你可以使用set将表达式的值赋给一个环境变量。例如，你可以使用set prompt $将GDB提示符设置为美元符号。

show ¶

与info相反，show用于描述GDB本身地状态。 你可以通过使用相关命令set来更改你可以show的大多数内容；例如，你可以使用set radix控制用于显示的数字系统，或者使用show radix简单地询问当前正在使用的数字系统。

要显示所有可设置的参数及其当前值，可以不带参数使用show；你也可以使用info set。这两个命令产生相同的显示。

这里有几个其他地show子命令，它们都很特殊，因为缺少相应的set命令：

show version ¶

显示正在运行的GDB的版本。你应该在GDB错误报告中包含此信息。如果你的站点使用多个版本的GDB，你可能需要确定你正在运行的GDB版本；随着GDB的发展，会引入新命令，旧命令可能会消失。此外，许多系统供应商提供GDB的变体版本，GNU/Linux发行版中也有GDB的变体版本。 版本号与你启动GDB时显示的版本号相同。

show copying ¶
info copying

显示有关复制GDB的权限的信息。

show warranty ¶
info warranty

显示GNU的“无保修”声明，或者如果你的GDB版本附带保修，则显示保修声明。

show configuration ¶

显示有关GDB构建时的配置方式的详细信息。这会显示传递给configure脚本的可选参数，以及configure自动检测到的配置参数。在报告GDB错误时（见 报告GDB中的错误），在你的报告中包含此信息很重要。

此命令还会显示GDB正在使用的Readline库的版本号。版本号后面会出现“‘(系统)’”，表示GDB正在使用共享系统Readline库，或者“‘(内部)’”，表示GDB正在使用静态链接的Readline库版本。
