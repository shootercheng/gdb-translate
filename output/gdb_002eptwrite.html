23.3.4.4 gdb.ptwrite ¶

此模块为记录使用PTWRITE指令的程序提供了额外功能。PTWRITE是一条x86指令，用于将值写入英特尔处理器跟踪（见 记录下级进程的执行并回放）。其对应的GCC内建函数为： 

void _ptwrite32 (unsigned int a)
void _ptwrite64 (unsigned __int64 a)

如果下级进程使用了该指令，GDB默认会将原始有效载荷值作为辅助信息插入到执行历史中。默认情况下，辅助信息会在执行 record instruction-history、record function-call-history 以及所有单步执行命令时打印出来，并且在Python中可以作为一个 RecordAuxiliary 对象访问（见 Python中的记录）。

示例程序： 

#include <immintrin.h>

void
ptwrite64 (unsigned long long value)
{
  _ptwrite64 (value);
}



int
main (void)
{
  ptwrite64 (0x42);
  return 0; /* 在此处打断点。 */
}

以pt格式记录示例程序后GDB的输出： 

(gdb) record instruction-history 12,14
12         0x0040074c <ptwrite64+16>:   ptwrite %rbx
13           [0x42]
14         0x00400751 <ptwrite64+21>:   mov -0x8(%rbp),%rbx
(gdb) record function-call-history
1       main
2       ptwrite64
          [0x42]
3       main

gdb.ptwrite 模块允许自定义 PTWRITE 辅助信息的默认输出。可以注册一个自定义的Python函数作为 PTWRITE 过滤函数。在跟踪解码期间，这个函数将以 PTWRITE 的有效载荷和程序计数器作为参数被调用。该函数可以返回一个字符串，这个字符串将在上述命令执行期间由GDB打印出来，或者返回 None（表示不输出任何内容）。要注册这样一个过滤函数，用户需要提供一个过滤工厂函数，该函数返回一个新的过滤函数对象供GDB调用。

函数： register_filter_factory (filter_factory) ¶

用于注册 PTWRITE 过滤工厂。这个过滤工厂可以是任何可调用对象，它接受一个参数，即当前线程作为一个 gdb.InferiorThread。 它可以返回 None 或者一个可调用对象。这个可调用对象就是指定线程的 PTWRITE 过滤函数。如果工厂函数返回 None，将打印默认的辅助信息。

>
函数： get_filter () ¶

返回当前活动的 PTWRITE 过滤函数。

一个示例： 

(gdb) python-interactive
>>> class my_filter():
...    def __init__(self):
...        self.var = 0
...    def __call__(self, payload, ip):
...        self.var += 1
...        return f"counter: {self.var}, ip: {ip:#x}"
...
>>> def my_filter_factory(thread):
...    if thread.global_num == 1:
...        return my_filter()
...    else:
...        return None
...
>>> import gdb.ptwrite
>>> gdb.ptwrite.register_filter_factory(my_filter_factory)
>>>
