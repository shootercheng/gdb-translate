10.12 便利变量 ¶

GDB 提供了 便利变量，你可以在 GDB 中使用它们来保存一个值，并在以后引用它。这些变量完全存在于 GDB 内部；它们不是你程序的一部分，设置一个便利变量对程序的进一步执行没有直接影响。这就是为什么你可以自由使用它们。

便利变量以 ‘$’ 为前缀。任何以 ‘$’ 开头的名称都可以用作便利变量，除非它是预定义的特定机器寄存器名称之一（见 寄存器）。（相比之下，值历史引用是由 ‘$’ 开头的 数字。见 值历史。）

你可以使用赋值表达式将一个值保存在便利变量中，就像你在程序中设置变量一样。 例如： 

set $foo = *object_ptr

会将 object_ptr 指向的对象中包含的值保存在 $foo 中。

首次使用便利变量时会创建它，但在你赋新值之前它的值是 void。你可以随时用另一个赋值来改变它的值。

便利变量没有固定的类型。你可以给一个便利变量赋任何类型的值，包括结构体和数组，即使该变量已经有了不同类型的值。便利变量在用作表达式时，具有其当前值的类型。

show convenience ¶

打印到目前为止使用的便利变量列表及其值，以及便利函数列表。 缩写为 show conv。

init-if-undefined $变量 = 表达式 ¶

如果便利变量尚未设置，则设置它。这对于保持某些状态的用户定义命令很有用。在概念上，它类似于在C中使用带初始化器的局部静态变量（除了便利变量是全局的）。它还可以用于允许用户覆盖命令脚本中使用的默认值。

如果变量已经定义，则不会计算表达式，因此不会发生任何副作用。

使用便利变量的一种方法是作为计数器进行递增或作为指针进行推进。例如，要打印结构体数组中连续元素的一个字段： 

set $i = 0
print bar[$i++]->contents

按 RET 键重复该命令。

一些便利变量是由 GDB 自动创建并赋予可能有用的值。

$_ ¶

变量 $_ 由 x 命令自动设置为最后检查的地址（见 检查内存）。其他为 x 命令提供默认检查地址的命令也会将 $_ 设置为该地址；这些命令包括 info line 和 info breakpoint。$_ 的类型是 void *，除非由 x 命令设置，在这种情况下它是指向 $__ 类型的指针。

$__ ¶

变量 $__ 由 x 命令自动设置为在最后检查的地址中找到的值。它的类型被选择为与打印数据的格式匹配。

$_exitcode ¶

当被调试的程序正常终止时，GDB 会自动将此变量设置为程序의退出代码，并将 $_exitsignal 重置为 void。

$_exitsignal ¶

当被调试的程序因未捕获的信号而终止时，GDB 会自动将此变量设置为该信号의编号，并将 $_exitcode 重置为 void。

为了区分被调试的程序是正常退出（即 $_exitcode 不是 void）还是因信号终止（即 $_exitsignal 不是 void），可以使用便利函数 $_isvoid（见 便利函数）。例如，可以考虑以下源代码： 

#include <signal.h>

int
main (int argc, char *argv[])
{
  raise (SIGALRM);  
  return 0;
}

判断被调试的程序是正常退出还是因信号终止的有效方法如下： 

(gdb) define has_exited_or_signalled
Type commands for definition of ``has_exited_or_signalled''.
End with a line saying just ``end''.
>if $_isvoid ($_exitsignal)
 >echo The program has exited\n
 >else
 >echo The program has signalled\n
 >end
>end
(gdb) run
Starting program:

Program terminated with signal SIGALRM, Alarm clock.
The program no longer exists.
(gdb) has_exited_or_signalled
The program has signalled

可以看出，GDB 正确地告知被调试的程序因信号终止，因为它调用了 raise 并引发了 SIGALRM 信号。如果被调试的程序没有调用 raise，那么 GDB 将报告正常退出： 

(gdb) has_exited_or_signalled
The program has exited
