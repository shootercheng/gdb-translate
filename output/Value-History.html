10.11 值历史记录 ¶

print命令打印的值保存在GDB的值历史记录中。这使你能够在其他表达式中引用它们。值会一直保留，直到重新读取或丢弃符号表（例如使用file或symbol-file命令）。当符号表发生变化时，值历史记录会被丢弃，因为这些值可能包含指向符号表中定义的类型的指针。

打印出的值会被赋予历史编号，你可以通过这些编号来引用它们。这些编号是从1开始的连续整数。print命令通过在值之前打印“$num = ”来向你显示分配给一个值的历史编号；这里的num就是历史编号。

要引用任何先前的值，使用“$”后跟该值的历史编号。print命令标记其输出的方式就是为了提醒你这一点。单独的“$”指的是历史记录中的最新值，“$$”指的是它之前的值。“$$n”指的是从末尾数第n个值；“$$2”是紧挨着“$$”之前的值，“$$1”等同于“$$”，“$$0”等同于“$”。

例如，假设你刚刚打印了一个指向结构体的指针，想要查看该结构体的内容。只需输入 

p *$

如果你有一串结构体，其中的组件next指向下一个结构体，你可以用这个命令打印下一个结构体的内容： 

p *$.next

你可以通过重复这个命令来打印链表中的后续链接——你只需按下RET键就能做到。

请注意，历史记录记录的是值，而不是表达式。如果x的值是4，你输入这些命令： 

print x
set x=5

那么即使x的值已经改变，print命令记录在值历史记录中的值仍然是4。

show values ¶

打印值历史记录中的最后十个值及其项目编号。这就像是重复十次“p $$9”，只是“show values”不会改变历史记录。

show values n

打印以历史项目编号n为中心的十个历史值。

show values +

在最近打印的值之后打印十个历史值。如果没有更多可用值，“show values +”不会产生显示。

按下RET键重复“show values n”与“show values +”具有完全相同的效果。
