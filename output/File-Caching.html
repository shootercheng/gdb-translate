18.2 文件缓存 ¶

为了加快文件加载速度并减少内存使用，GDB将重用用于跟踪打开文件的bfd对象。请参阅《二进制文件描述符库》中的 BFD。以下命令允许查看和控制缓存行为。

maint info bfds ¶

打印有关GDB已知的每个bfd对象的信息。

maint set bfd-sharing ¶
maint show bfd-sharing

控制是否可以共享bfd对象。启用共享时，GDB会重用已打开的bfd对象，而不是重新打开同一文件。关闭共享不会导致已共享的bfd对象取消共享，但所有未来打开的文件将创建一个新的bfd对象。同样，重新启用共享不会导致多个现有的bfd对象合并为一个共享的bfd对象。

set debug bfd-cache level ¶

开启bfd缓存的调试，将级别设置为level。

show debug bfd-cache ¶

显示bfd缓存的当前调试级别。
