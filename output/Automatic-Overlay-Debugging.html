第14.节 自动覆盖调试 ¶

GDB可以在被调试程序中的覆盖管理器提供一些简单协作的情况下，自动跟踪哪些覆盖被映射以及哪些没有被映射。如果你使用overlay auto命令启用自动覆盖调试（见 覆盖命令），GDB会在被调试程序的内存中查找某些描述覆盖当前状态的变量。

以下是你的覆盖管理器必须定义以支持GDB自动覆盖调试的变量：

_ovly_table:

这个变量必须是一个包含以下结构的数组： 

struct
{
   /* 覆盖的映射地址。  */
   unsigned long vma;

   /* 覆盖的大小，以字节为单位。  */
   unsigned long size;

   /* 覆盖的加载地址。  */
   unsigned long lma;

   /* 如果覆盖当前已映射，则为非零；
      否则为零。  */
   unsigned long mapped;
}
_novlys:

这个变量必须是一个四字节的有符号整数，保存_ovly_table中的元素总数目。

为了确定某个特定的覆盖是否被映射，GDB会在_ovly_table中查找一个条目，其vma和lma成员等于可执行文件中覆盖部分的虚拟内存地址（VMA）和加载内存地址（LMA）。当GDB找到一个匹配的条目时，它会查看该条目的mapped成员来确定覆盖当前是否被映射。

此外，你的覆盖管理器可以定义一个名为_ovly_debug_event的函数。如果定义了这个函数，GDB会在那里默默地设置一个断点。如果覆盖管理器在每次更改覆盖表时调用这个函数，这将使GDB能够准确跟踪哪些覆盖在程序内存中，并更新可能在覆盖中设置的任何断点。这将允许断点即使在覆盖未执行时保存在ROM或其他不可写内存中时也能正常工作。
