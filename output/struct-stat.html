struct stat ¶

目标和GDB使用的类型为struct stat的缓冲区定义如下： 

struct stat {
    unsigned int  st_dev;      /* 设备 */
    unsigned int  st_ino;      /* inode */
    mode_t        st_mode;     /* 保护 */ 
    unsigned int st_nlink;    /* 硬链接数 */
    unsigned int st_uid;      /* 所有者的用户ID */
    unsigned int st_gid;      /* 所有者的组ID */
    unsigned int st_rdev;     /* 设备类型（如果是inode设备） */
    unsigned long st_size;     /* 总大小，以字节为单位 */
    unsigned long st_blksize;  /* 文件系统I/O的块大小 */
    unsigned long st_blocks;   /* 已分配的块数 */
    time_t        st_atime;    /* 最后访问时间 */
    time_t        st_mtime;    /* 最后修改时间 */
    time_t        st_ctime;    /* 最后更改时间 */
};

整数数据类型符合相应章节中给出的定义（有关详细信息，请参阅 整数数据类型），因此这个结构的大小为64字节。

几个字段的值具有受限的含义和/或值范围。

st_dev

值0表示文件，1表示控制台。

st_ino

对目标没有有效意义。原样传输。

st_mode

有效的模式位在 常量 中描述。任何其他位目前对目标没有意义。

st_uid
st_gid
st_rdev

对目标没有有效意义。原样传输。

st_atime
st_mtime
st_ctime

这些值的精度取决于主机和文件系统。特别是在Windows主机上，文件系统可能不支持精确的时间值。

目标获取上述表示形式的 struct stat，并负责在继续之前将其强制转换为目标表示形式。

请注意，由于主机、目标和协议对 struct stat 成员的表示形式之间存在大小差异，这些成员最终可能会在目标上被截断。
