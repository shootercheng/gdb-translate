15.3.2范围检查概述 ¶

在某些语言（如Modula-2）中，超出类型界限是一个错误；这通过运行时检查来强制执行。这种范围检查旨在通过确保计算不会溢出，或者数组元素访问的索引不会超出数组的界限来确保程序的正确性。

对于你在GDB命令中使用的表达式，你可以告诉GDB以三种方式之一处理范围错误：忽略它们、始终将它们视为错误并放弃表达式，或者发出警告但无论如何都评估表达式。

范围错误可能由数值溢出、超出数组索引界限或当你输入一个不是任何类型成员的常量时导致。然而，一些语言并不将溢出视为错误。在许多C语言实现中，数学溢出会导致结果“环绕”到较低的值—例如，如果m是最大整数值，s是最小整数值，那么 

m + 1 ⇒ s

这也特定于个别语言，并且在某些情况下特定于个别编译器或机器。有关特定语言的更多详细信息，请参阅 支持的语言。

GDB提供了一些用于控制范围检查器的附加命令：

set check range auto

根据当前工作语言设置范围检查的开启或关闭。有关每种语言的默认设置，请参阅 支持的语言。

set check range on
set check range off

设置范围检查的开启或关闭，覆盖当前工作语言的默认设置。如果设置与语言默认不匹配，则会发出警告。如果发生范围错误且范围检查开启，则会打印一条消息并中止表达式的评估。

set check range warn

当GDB范围检查器检测到范围错误时输出消息，但无论如何都尝试评估表达式。由于其他原因，例如访问进程不拥有的内存（许多Unix系统中的典型示例）之类的原因，评估表达式可能仍然是不可能的。

show check range

显示范围检查器的当前设置，以及它是否由GDB自动设置。
