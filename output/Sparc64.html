21.4.7.1 ADI支持 ¶

M7处理器支持一种应用数据完整性（ADI）功能，用于检测无效的数据访问。当软件分配内存并在分配的内存上启用ADI时，它会选择一个4位版本号，将该版本设置在指向该数据的64位指针的高4位中，并将4位版本存储在该数据的每个缓存行中。硬件将后者保存在缓存和内存层次结构的备用位中。在每次加载和存储时，处理器会将高4位虚拟地址（VA）与缓存行的版本进行比较。如果不匹配，处理器会生成一个版本不匹配陷阱，该陷阱可以是精确的或中断的。该陷阱是一种错误情况，内核会将其作为SIGSEGV信号传递给进程。

请注意，只有64位应用程序可以使用ADI，并且需要在启用ADI的情况下构建。

ADI版本标签的值，其粒度为一个缓存行（64字节），可以查看或修改。

adi (examine | x) [ / n ] addr ¶

adi examine命令显示每个缓存行的一个ADI版本标签的值。

n是一个十进制整数，指定字节数；默认值为1。它指定要显示的ADI版本信息的量，比例为1:ADI块大小。

addr是用户地址空间中的地址，你希望GDB从该地址开始显示ADI版本标签。

以下是显示变量“shmaddr”的ADI版本的示例。 

(gdb) adi x/100 shmaddr
   0xfff800010002c000:     0 0
adi (assign | a) [ / n ] addr = tag ¶

adi assign命令用于为一个地址分配新的ADI版本标签。

n是一个十进制整数，指定字节数；默认值为1。它指定要修改的ADI版本信息的量，比例为1:ADI块大小。

addr是用户地址空间中的地址，你希望GDB从该地址开始修改ADI版本标签。

tag是新的ADI版本标签。

例如，执行以下操作来修改然后验证变量“shmaddr”的ADI版本： 

(gdb) adi a/
