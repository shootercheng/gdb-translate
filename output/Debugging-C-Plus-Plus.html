15.4.1.7 GDB用于C++的特性 ¶

一些GDB命令在C++中特别有用，有些是专门为与C++一起使用而设计的。以下是一个总结：

断点菜单 ¶

当你想在一个名字被重载的函数中设置断点时，GDB有能力显示一个可能的断点位置菜单，以帮助你指定你想要的函数定义。 请参阅 模糊表达式。

rbreak regex ¶

使用正则表达式设置断点有助于在不是任何特殊类的成员的重载函数上设置断点。 请参阅 设置断点。

catch throw ¶
catch rethrow
catch catch

使用这些命令调试C++异常处理。请参阅 设置捕获点。

ptype typename ¶

打印类型typename的继承关系以及其他信息。 请参阅 检查符号表。

info vtbl expression.

info vtbl命令可用于显示由expression计算出的对象的虚方法表。每个虚表显示一个条目；当使用多重继承时可能有多个虚表。

demangle name ¶

对name进行去混淆。 有关demangle命令的更完整描述，请参阅 检查符号表。

set print demangle ¶
show print demangle
set print asm-demangle
show print asm-demangle

控制在将代码显示为C++源以及显示反汇编时，C++符号是否以其源形式显示。 请参阅 打印设置。

set print object
show print object

选择是打印对象的派生（实际）类型还是声明类型。请参阅 打印设置。
