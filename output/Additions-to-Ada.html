15.4.10 .3 Ada的补充 ¶

与其他语言一样，GDB对Ada进行了一些通用扩展（见表达式）：

  • 如果表达式E是内存中的一个变量（通常是局部变量或数组元素），并且N是一个正整数，那么E@N会将E及其在内存中紧随其后的N - 1个相邻变量的值作为一个数组显示出来。在Ada中，这个操作符通常不是必需的，因为它的主要用途是显示数组的部分内容，而切片通常可以在Ada中完成这个操作。然而，在调试某些调试信息被优化掉的程序时，偶尔会用到它。
  • B::var表示“在函数或文件B中出现的名为var的变量 ”。当B是文件名时，你通常必须用单引号将其括起来。
  • 表达式{type} addr表示“在地址addr处出现的类型为type的变量”。
  • 以‘$’开头的名称是一个便利变量（见便利变量）或机器寄存器（见寄存器）。

此外，GDB还提供了一些其他特定于Ada的快捷方式和直接补充：

  • 允许将赋值语句用作表达式，返回其右操作数作为其值。因此，你可以输入 
    (gdb) set x := y + 3
(gdb) print A(tmp := y + 1)
  • 允许将分号用作“操作符”，返回其右操作数的值作为其值。 这允许进行复杂的条件断点设置，例如： 
    (gdb) break f
(gdb) condition 1 (report(i); k += 1; A(k) > 100)
  • 对基于字面量的扩展可用于指定浮点字面量的确切字节内容。在基数之后，你可以使用零到两个‘l’字符，后跟一个‘f’。‘l’字符的数量控制结果实常量的宽度：零表示使用Float，一表示使用Long_Float，二表示使用Long_Long_Float。 
    (gdb)p rint 16f#41b80000#
$1 = 23.0
  • 不必使用连接和符号字符名称将特殊字符引入字符串，而是可以使用一种特殊的括号表示法，这种表示法也用于打印字符串。字符串或字符字面量中形式为‘["XX"]’的字符序列表示其十六进制数字编码为XX的（单个）字符。字符序列‘["""]’在字符串中也表示一个单引号。例如， 
       "One line.["0a"]Next line.["0a"]"

    在每个句号后包含一个ASCII换行符(Ada.Characters.Latin_1.LF)。

  • 用作属性'Pos‘'Min和 'Max前缀的子类型是可选的（无论如何都会被忽略）。例如，编写以下内容是有效的 
    (gdb) print'max(x, y)
  • 打印数组时，如果数组的下界为1，GDB使用位置表示法，否则使用修改后的命名表示法。例如，一个下界为3的包含三个整数的一维数组可能会打印为 
    (3 => 10, 17, 1)

    也就是说，与有效的Ada不同，只有第一个组件有=>子句。

  • 你可以用其名称的任何唯一的多字符子序列来缩写表达式中的属性（精确匹配优先）。例如，你可以使用a'len、a'gth或a'lh来代替a'length。
  • 由于Ada不区分大小写，调试器通常会将你输入的标识符映射为小写。GNAT编译器对其一些内部标识符使用大写字符，而用户通常对此不感兴趣。在极少数情况下，当你实际需要查看它们时，将它们用尖括号括起来以避免小写映射。例如， 
    (gdb) print <JMPBUF_SAVE>[0]
  • 打印类范围类型的对象或取消引用指向类范围值的指针将显示对象特定类型的所有组件（如运行时标签所示）。同样，对此类值的组件选择将对对象的特定类型进行操作。
