12 C预处理器宏 ¶

一些语言，如C和C++，提供了一种定义和调用“预处理器宏”的方法，这些宏会扩展为一串令牌。GDB可以评估包含宏调用的表达式，显示宏扩展的结果，并显示宏的定义，包括其定义位置。

你可能需要专门编译你的程序，以便为GDB提供有关预处理器宏的信息。大多数编译器在调试信息中不包含宏，即使你使用 -g 标志进行编译。请参阅 为调试而编译。

一个程序可能在某个点定义一个宏，稍后删除该定义，然后再提供一个不同的定义。因此，在程序的不同点，一个宏可能有不同的定义，或者根本没有定义。如果有当前堆栈帧，GDB会使用该帧源代码行中有效的宏。否则，GDB会使用当前列表位置中有效的宏；请参阅 打印源代码行。

每当GDB评估一个表达式时，它总是会扩展表达式中存在的任何宏调用。GDB还提供了以下用于显式处理宏的命令。

macro expand expression ¶
macro exp expression

显示在expression中展开所有预处理器宏调用的结果。由于GDB只是简单地展开宏，而不解析结果，所以expression不必是一个有效的表达式；它可以是任何一串令牌。

macro expand-once expression ¶
macro exp1 expression

（此命令尚未实现。） 显示在expression中显式出现 的那些预处理器宏调用的展开结果。在该展开中出现的宏调用保持不变。此命令允许你更清楚地看到特定宏的效果，而不会被进一步的展开所混淆。由于GDB只是简单地展开宏，而不解析结果，所以expression不必是一个有效的表达式；它可以是任何一串令牌。

info macro [-a|-all] [--] macro ¶

显示命名宏macro的当前定义或所有定义，并描述该定义所在的源位置或编译器命令行。可选的双破折号用于表示参数处理的结束和宏的开始，对于非C风格的宏，宏可能以连字符开头。

info macros locspec ¶

显示在解析locspec后得到的代码位置的源代码行中有效的所有宏定义，并描述这些定义所在的源位置或编译器命令行。

macro define macro replacement-list ¶
macro define macro(arglist) replacement-list

为名为macro的预处理器宏引入一个定义，其调用将被replacement-list中给出的令牌替换。此命令的第一种形式定义一个“对象样”宏，它不带参数；第二种形式定义一个“函数样”宏，它带arglist中给出的参数。

此命令引入的定义在GDB评估的每个表达式中都有效，直到使用下面描述的 macro undef 命令将其删除。该定义会覆盖正在调试的程序中存在的macro的所有定义，以及任何以前用户提供的定义。

macro undef macro ¶

删除名为macro的宏的任何用户提供的定义。此命令仅影响上面描述的 macro define 命令提供的定义；它不能删除正在调试的程序中存在的定义。

macro list ¶

列出使用 macro define 命令定义的所有宏。

以下是一个展示上述命令实际使用情况的记录。首先，我们展示我们的源文件： 

$ cat sample.c
#include <stdio.h>
#include "sample.h"

#define M 42
#define ADD(x) (M + x)

main ()
{
#define N 28
  printf ("Hello, world!\n");
#undef N
  printf ("We're so creative.\n");
#define N 1729
  printf ("Goodbye, world!\n");
}
$ cat sample.h
#define Q <
$

现在，我们使用GNU C编译器GCC编译程序。我们传递 -gdwarf - 214 和 -g3 标志，以确保编译器在调试信息中包含有关预处理器宏的信息。 

$ gcc -gdwarf-2 -g3 sample.c -o sample
$

现在，我们在示例程序上启动GDB： 

$ gdb -nw sample
GNU gdb 2002-05-06-cvs
Copyright 2002 Free Software Foundation, Inc.
GDB is free software, ...
(gdb)
