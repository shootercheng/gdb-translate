13.2.1 tfind n ¶

从缓冲区中选择跟踪快照的基本命令是tfind n，它从0开始计数，查找跟踪快照编号n。如果没有给出参数n，则选择下一个快照。

以下是使用tfind命令的各种形式。

tfind start

查找缓冲区中的第一个快照。这是tfind 0的同义词（因为0是第一个快照的编号）。

tfind none

停止调试跟踪快照，恢复实时调试。

tfind end

与‘tfind none’相同。

tfind

无参数表示查找下一个跟踪快照，如果没有选择跟踪快照，则查找第一个。

tfind -

查找当前快照之前的上一个跟踪快照。这允许回溯更早的步骤。

tfind tracepoint num

查找与跟踪点num相关联的下一个快照。搜索从最后检查的跟踪快照开始向前进行。如果没有给出参数num，则表示查找与当前快照相同跟踪点收集的下一个快照。

tfind pc addr

查找与程序计数器值addr相关联的下一个快照。搜索从最后检查的跟踪快照开始向前进行。如果没有给出参数addr，则表示查找与当前快照具有相同PC值的下一个快照。

tfind outside addr1, addr2

查找下一个PC值在给定地址范围之外（不包括）的快照。

tfind range addr_1, addr_2

查找下一个PC值在addr1和addr2之间（包括）的快照。

tfind line [file:]n

查找与源行n相关联的下一个快照。如果给出了可选参数file，则指的是该源文件中的第n行。搜索从最后检查的跟踪快照开始向前进行。如果没有给出参数n，则表示查找除当前正在检查的行之外的下一行；因此，重复说tfind line可能会产生与在实时调试会话中逐行步进相同的效果。

tfind命令的默认参数经过专门设计，以便于浏览跟踪缓冲区。例如，无参数的tfind选择下一个跟踪快照，无参数的tfind -选择上一个跟踪快照。因此，通过给出一个tfind命令，然后简单地重复按下RET，就可以按顺序检查所有跟踪快照。或者，通过说tfind -，然后重复按下RET，可以按相反顺序检查快照。无参数的tfind line命令选择执行的下一个源行的快照。无参数的tfind pc命令选择与当前帧具有相同程序计数器（PC）的下一个快照。无参数的tfind tracepoint命令选择与当前跟踪点收集的下一个跟踪快照。

除了让你手动浏览跟踪缓冲区之外，这些命令还便于构建GDB脚本，该脚本可以浏览跟踪缓冲区并打印出你感兴趣的任何收集数据。因此，如果我们想检查缓冲区中每个跟踪帧的PC, FP和SP寄存器，我们可以这样说： 

(gdb) tfind start
(gdb) while ($trace_frame != -1)
> printf "Frame %d, PC = %08X, SP = %08X, FP = %08X\n", \
          $trace_frame, $pc, $sp, $fp
> tfind
> end

或者，如果我们想检查缓冲区中每个源行的变量X： 

(gdb) tfind start
(gdb) while ($trace_frame != -1)
> printf "Frame %d, X == %d\n", $trace_frame, X
> tfind line
> end

Frame 0, X = 1
Frame 7, X = 2
Frame 13, X = 2?5
