23.5.1 “objfile-gdb.ext”文件 ¶

当读取一个新的目标文件时，GDB会查找名为objfile-gdb.ext的文件（我们在下面将其称为script-name），其中objfile是目标文件的名称，ext是扩展语言的文件扩展名：

objfile-gdb.gdb

GDB自己的命令语言

objfile-gdb.py

Python

objfile-gdb.scm

Guile

script-name是通过确保objfile的文件名是绝对路径，遵循所有符号链接，并解析“.”和“..”组件，然后附加-gdb.ext后缀来形成的。 如果这个文件存在并且可读，GDB将把它作为指定扩展语言的脚本进行求值。

如果这个文件不存在，那么GDB将在以下指定的所有目录中查找script-name文件。 （在MS-Windows/MS-DOS上，可执行文件主目录的驱动器字母会被转换为一个单字母子目录，例如d:/usr/bin/会被转换为/d/usr/bin/，因为Windows文件系统不允许文件名中包含冒号。）

请注意，加载这些文件需要相应配置的“auto-load safe-path”（见自动加载的安全限制）。

对于使用.exe后缀的目标文件，GDB首先会尝试根据其.exe文件名正常加载脚本。 但如果没有找到脚本，GDB也会尝试使用与目标文件匹配但不包含.exe后缀的脚本文件名。 这种.exe后缀的去除不区分大小写，并且在任何平台上都会尝试。 这使得脚本文件名在Unix和MS-Windows主机之间兼容。

set auto-load scripts-directory [directories] ¶

控制GDB自动加载脚本的位置。多个目录项可以用主机平台使用的路径分隔符分隔（Unix上是“:”，MS-Windows和MS-DOS上是“;”）。

这里的每个条目也需要被安全设置“set auto-load safe-path”覆盖（见设置自动加载安全路径）。
