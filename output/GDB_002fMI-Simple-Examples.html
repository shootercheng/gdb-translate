27.6 GDB/MI交互的简单示例 ¶

本小节展示了几个使用GDB/MI接口进行交互的简单示例。在这些示例中，‘->’表示以下行作为输入传递给GDB/MI，而‘<-’表示从GDB/MI接收到的输出。

请注意，示例中显示的换行仅为了便于阅读，在实际输出中不会出现。

设置断点 ¶

设置断点会生成同步输出，其中包含断点的详细信息。 

-> -break-insert main
<- ^done,bkpt={number="1",type="breakpoint",disp="keep",
    enabled="y",addr="0x08048564",func="main",file="myprog.c",
    fullname="/home/nickrob/myprog.c",line="68",thread-groups=["i1"],
    times="0"}
<- (gdb)

程序执行 ¶

程序执行会生成异步记录，并且MI会给出执行停止的原因。 

-> -exec-run
<- ^running
<- (gdb)
<- *stopped,reason="breakpoint-hit",disp="keep",bkptno="1",thread-id="0",
   frame={addr="0x08048564",func="main",
   args=[{name="argc",value="1"},{name="argv",value="0xbfc4d4d4"}],
   file="myprog.c",fullname="/home/nickrob/myprog.c",line="68",
   arch="i386:x86_64"}
<- (gdb)
-> -exec-continue
<- ^running
<- (gdb)
<- *stopped,reason="exited-normally"
<- (gdb)

退出GDB ¶

退出GDB只会打印结果类‘^exit’。 

-> (gdb)
<- -gdb-exit
<- ^exit

请注意，‘^exit’会立即打印，但GDB实际退出可能需要一些时间。在此期间，GDB会进行必要的清理工作，包括终止正在调试的程序或与调试硬件断开连接，因此前端应该等待GDB退出，只有在GDB未能在合理时间内退出时才应强行终止它。

一个错误的命令 ¶

如果你传递一个不存在的命令会发生以下情况： 

-> -rubbish
<- ^error,msg="未定义的MI命令：rubbish"
<- (gdb)
