open ¶

概要：
int open(const char *pathname, int flags);
int open(const char *pathname, int flags, mode_t mode);
请求：

‘Fopen,pathptr/len,flags,mode’

flags是以下值的按位 OR：

O_CREAT

如果文件不存在，将创建该文件。就文件所有权和时间戳而言，适用主机规则。

O_EXCL

与 O_CREAT 一起使用时，如果文件已存在，则为错误，open() 将失败。

O_TRUNC

如果文件已存在且打开模式允许写入（给定 O_RDWR 或 O_WRONLY）它将被截断为零长度。

O_APPEND

文件以追加模式打开。

O_RDONLY

文件仅以只读方式打开。

O_WRONLY

文件仅以写入方式打开。

O_RDWR

文件以读写方式打开

其他位将被静默忽略。

mode是以下值的按位 OR：

S_IRUSR

用户具有读权限。

S_IWUSR

用户具有写权限。

S_IRGRP

组具有读权限。

S_IWGRP

组具有写权限。

S_IROTH

其他人具有读权限。

S_IWOTH

其他人具有写权限。

其他位将被静默忽略。

返回值：

open 返回新的文件描述符；如果发生错误，则返回 -1。

错误：
EEXIST

pathname 已存在，并且使用了 O_CREAT 和 O_EXCL。

EISDIR

pathname 指向一个目录。

EACCES

请求的访问不被允许。

ENAMETOOLONG

pathname 太长。

ENOENT

pathname中的一个目录组件不存在。

ENODEV

pathname 指向一个设备、管道、命名管道或套接字

EROFS

pathname 指向只读文件系统上的一个文件，并且请求了写入访问权。

EFAULT

pathname 是一个无效的指针值。

ENOSPC

设备上没有空间来创建文件。

EMFILE

进程已经打开了最大数量的文件。

ENFILE

系统上打开的文件总数已达到限制。

EINTR

调用被用户中断。
