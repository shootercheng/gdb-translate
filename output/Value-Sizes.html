10.24 值的大小 ¶

每当GDB打印一个值时，GDB内部都会分配内存来保存该值的内容。在一些具有动态类型系统的语言中，一个无效的程序可能会指示一个过大的值，这反过来可能会导致GDB尝试分配过多的内存。

set max-value-size 字节数 ¶
set max-value-size unlimited

将GDB为一个值的内容分配的最大内存大小设置为字节数，尝试显示需要比该大小更多内存的值将导致错误。

设置此变量不会影响已经在GDB内部分配的值，仅影响未来的分配。

max-value-size可以设置的最小值是为了使GDB仍能正确运行，目前这个最小值是16字节。

这个限制适用于一些子表达式的结果以及完整的表达式。例如，如果x.y的大小是动态的并且超过了字节数，那么表示一个简单整数组件的表达式，如x.y.z，可能会失败。另一方面，GDB有时很聪明；表达式A[i]，其中A是一个大小不固定的数组变量，只要组件大小小于字节数，通常无论A的边界如何都会成功。

max-value-size的默认值目前是64k。

show max-value-size ¶

显示GDB为一个值的内容分配的最大内存大小（以字节为单位）。
