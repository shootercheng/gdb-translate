gdb手册 ¶

gdb [选项] [程序|程序 进程ID|程序 核心文件]

像GDB这样的调试器旨在让你在另一个程序执行时查看其“内部”发生了什么，或者在程序崩溃时查看它当时在做什么。

GDB可以做四种主要的事情（以及其他支持这些的事情）来帮助你当场捕捉错误：

  • 启动你的程序，指定任何可能影响其行为的因素。
  • 使你的程序在指定条件下停止。
  • 检查程序停止时发生了什么。
  • 更改程序中的内容，以便你可以试验纠正一个错误的影响并继续了解另一个错误。

你可以使用GDB调试用C、C++、Fortran和Modula-2编写的程序。

通过 shell 命令gdb调用GDB。启动后，它从终端读取命令，直到你使用GDB命令quit或exit告诉它退出。你可以使用命令help从GDB本身获得在线帮助。

你可以不带任何参数或选项运行gdb；但启动GDB最常见的方法是带一个或两个参数，指定一个可执行程序作为参数： 

gdb 程序

你也可以同时指定一个可执行程序和一个核心文件： 

gdb 程序 核心文件

如果你想调试一个正在运行 的进程，你可以改为指定一个进程ID作为第二个参数或使用选项-p： 

gdb 程序 1234
gdb -p 1234

这将把GDB附加到进程1234。使用选项-p时，你可以省略程序文件名。

以下是一些最常用的GDB命令：

break [文件:][函数|行号]

在函数或行号（在文件中）处设置断点。

run [参数列表]

启动你的程序（如果指定了参数列表）。

bt

回溯：显示程序堆栈。

print 表达式

显示一个表达式的值。

c

继续运行你的程序（在停止后，例如在断点处）。

next

执行下一行程序（在停止后）；跳过该行中的任何函数调用。

edit [文件:]函数

查看当前停止的程序行。

list [文件:]函数

输入当前停止位置附近的程序文本。

step

执行下一行程序（在停止后）；进入该行中的任何函数调用。

help [名称]

显示有关GDB命令名称的信息，或有关使用GDB的一般信息。

quit
exit

退出GDB。

除选项外的任何参数都指定一个可执行文件和核心文件（或进程ID）；也就是说，第一个没有关联选项标志的参数等同于一个--se选项，第二个（如果有）如果是文件名，则等同于一个-c选项。许多选项都有长格式和缩写形式；这里都显示了。如果你截断长格式，只要存在足够的选项以使其明确，GDB也会识别它们。

缩写形式在这里用“-”表示，长格式用“--”表示，以反映它们在--help中的显示方式。然而，GDB对于大多数选项都识别以下所有约定：

--选项=值
--选项 值
-选项=值
-选项 值
--o=值
--o 值
-o=值
-o 值

你给出的所有选项和命令行参数都是按顺序处理的。当使用-x选项时，顺序很重要。

--help
-h

列出所有选项，并给出简要解释。

--symbols=文件
-s 文件

从文件中读取符号表。

--write

启用对可执行文件和核心文件的写入。

--exec=文件
-e 文件

在适当的时候使用文件作为要执行的可执行文件，并结合核心转储检查纯数据。

--se=文件

从文件中读取符号表并将其用作可执行文件。

--core=文件
-c 文件

使用文件作为要检查的核心转储。

--command=文件
-x 文件

从文件中执行GDB命令。

--eval-command=命令
-ex 命令

执行给定的GDB命令。

--init-eval-command=命令
-iex

在加载下级之前执行GDB命令。

--directory=目录
-d 目录

将目录添加到搜索源文件的路径中。

--nh

不执行来自~/.config/gdb/gdbinit、~/.gdbinit、~/.config/gdb/gdbearlyinit或~/.gdbearlyinit的命令。

--nx
-n

不执行来自任何.gdbinit或.gdbearlyinit初始化文件的命令。

--quiet
--silent
-q

“安静”。不打印介绍性和版权信息。这些信息在批处理模式下也会被抑制。

--batch

以批处理模式运行。在处理所有用-x指定的命令文件（以及如果未禁止的话，.gdbinit）后以状态码0退出。如果在执行命令文件中的GDB命令时发生错误，则以非零状态退出。

批处理模式对于将GDB用作过滤器可能很有用，例如在另一台计算机上下载并运行程序；为了使其更有用，在批处理模式下运行时不会发出 

程序正常退出。

（通常在GDB控制下运行的程序终止时会发出此消息）。

--batch-silent

以批处理模式运行，与--batch一样，但完全安静。所有GDB输出都被抑制（stderr不受影响）。这比--silent安静得多，对于交互式会话将毫无用处。

例如在使用给出“加载段”消息的目标时，这特别有用。

请注意，通过GDB输出其输出而不是直接写入stdout的目标也将被设为安静。

--args 程序 [参数列表]

更改命令行的解释，以便此选项之后的参数作为参数传递给下级。例如，对于以下命令： 

gdb ./a.out -q

它将以-q启动GDB，不打印介绍性消息。另一方面，使用： 

gdb --args ./a.out -q

启动GDB时会打印介绍性消息，并将选项传递给下级。

--pid=pid

将GDB附加到一个已经在运行的程序，其PID为pid。

--tui

打开终端用户界面。

--readnow

在首次访问时从给定的符号文件中读取所有符号
