8.5 对多个帧应用命令。 ¶

frame apply [all | count | -count | level level…] [option]… command

frame apply命令允许你将指定的command应用于一个或多个帧。

all

指定all将command应用于所有帧。

count

使用count将command应用于最内层的count个帧，其中count是一个正数。

-count

使用-count将command应用于最外层的count个帧，其中count是一个正数。

level

使用level将command应用于由level列表标识的帧集。level是一个帧级别或帧级别范围，如level1-level2。帧级别是在“backtrace”命令输出的第一列中显示的数字。例如，“2-4 6-8 3”表示将command应用于级别2、3、4、6、7、8的帧，然后再应用于级别3的帧。

请注意，“frame apply”应用命令的帧也会受到诸如“set backtrace past-main”和“set backtrace limit N”等“set backtrace”设置的影响。 请参阅 回溯。

“frame apply”命令还支持许多选项，这些选项允许覆盖相关的“set backtrace”设置：

-past-main [on|off]

回溯是否应继续越过“main”函数。 相关设置：设置回溯越过main。

-past-entry [on|off]

回溯是否应继续越过程序的入口点。 相关设置：设置回溯越过入口。

默认情况下，GDB在command产生的输出之前显示一些帧信息，并且在执行command期间引发的错误将中止“frame apply”。 可以使用以下选项来微调这些行为：

-c

“-c”标志，代表“continue”，会显示command中的任何错误，然后继续执行“frame apply”。

-s

“-s”标志，代表“silent”，会静默忽略command产生的任何错误或空输出。 也就是说，执行继续，但帧信息和错误不会被打印。

-q

“-q”标志（“quiet”）禁用打印帧信息。

以下示例展示了在将命令“p j”应用于所有帧时，“-c”和“-s”标志的工作方式，其中变量“j”只能在最外层的“#1 main”帧中成功打印。 

(gdb) frame apply all p j
#0  some_function (i=5) at fun.c:4
当前上下文中没有符号“j”。
(gdb) frame apply all -c p j
#0  some_function (i=5) at fun.c:4
当前上下文中没有符号“j”。
#1  0x565555fb in main (argc=1, argv=0xffffd2c, at fun.c:11
$1 = 5
(gdb) frame apply all -s p j
#1  0x565555fb in main (argc=1, argv=0xffffd2c, at fun.c:11
$2 = = 5
(gdb)

默认情况下，“frame apply”会在命令输出之前打印帧位置信息： 

(gdb) frame apply all p $sp
#0  some_function (i=5) at fun.c:4
$4 = (void *) 0xffffd1e0
#1  0x565555fb in main (argc=1, argv=0xffffd2c, at fun.c:11
$5 = (void *) 0xffffd1f0
(gdb)

如果给出“-q”标志，则不会打印帧信息： 

(gdb) frame apply all -q p $sp
$12 = (void *) 0xffffd1e0
$13 = (void *) 0xffffd1f0
(gdb)
faas command ¶

“frame apply all -s command”的快捷方式。 将command应用于所有帧，忽略错误和空输出。

例如，它可用于在不知道变量或参数所在帧的情况下打印局部变量或函数参数，使用： 

(gdb) faas p some_local_var_i_do_not_remember_where_it_is

“faas”命令接受与“frame apply”命令相同的选项。 请参阅 帧应用。

请注意，“tfaas command”命令将command应用于所有线程的所有帧。 请参阅 线程。
