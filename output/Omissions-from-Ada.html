15 - 4 - 10 - 2 Ada中的遗漏 ¶

以下是该子集中值得注意的遗漏：

  • 仅支持部分属性：
    • 数组对象上的 'First、'Last 和 'Length（不适用于类型和子类型）。
    • 'Min 和 'Max。
    • 'Pos 和 'Val。
    • 'Tag。
    • 数组对象上的 'Range（不适用于子类型），但仅作为成员资格（in）运算符右边的操作数。
    • 'Access、'Unchecked_Access 和 'Unrestricted_Access（GNAT扩展）。
    • 'Address。
    • 'Size 可用于对象，但不适用于类型。
    • 'Object_Size 可用，但不适用于不定类型。
  • Characters.Latin_1 中的名称不可用。
  • 数组上的相等性测试（‘=’ 和 ‘/=’）测试表示的按位相等性。对于元素具有整数或枚举类型的字符串和数组，它们通常能正确工作。对于元素类型具有用户定义相等性的数组、实值数组（特别是符合IEEE标准的浮点型，因为存在负零和NaN）以及元素包含具有不确定值的未使用位的数组，它们可能无法正确工作。
  • 其他逐个组件的数组操作（and、or、xor、not 以及除相等性之外的关系测试）未实现。
  • 对数组和记录聚合的支持有限。它们仅允许在赋值的右侧使用，如下例所示： 
    (gdb) set An_Array := (1, 2, 3, 4, 5, 6)
(gdb) set An_Array := (1, others => 0)
(gdb) set An_Array := (0|4 => 1, 1..3 =>  2, 5 => 6)
(gdb) set A_2D_Array := ((1, 2, 3), (4, 5, 6), (7, 8, 9))
(gdb) set A_Record := (1, "Peter", True);
(gdb) set A_Record := (Name => "Peter", Id => 1, Alive => True)

    如果在记录中使用判别式，通过赋值聚合来更改判别式的值会产生未定义的效果。但是，你可以先通过直接赋值来修改判别式（这在Ada中通常是不允许的），然后进行聚合赋值。例如，给定一个声明为如下类型的变量 A_Rec： 

    type Rec (Len : Small_Integer := 0) is record
    Id : Integer;
    Vals : IntArray (1 .. Len);
end record;

    你可以通过两次赋值来为 Vals 赋不同大小的值： 

    (gdb) set A_Rec.Len :=  4
(gdb) set A_Rec := (Id => 42, Vals => (1, 2, 3, 4))

    正如这个例子所示，GDB 对聚合的通常规则非常宽松。你可以省略数组或记录聚合的一些组件（例如上面给 A_Rec 赋值时的 Len 组件）；它们在赋值时将保留其原始值。你可以自由地使用动态值作为组件关联中的索引。你甚至可以使用重叠或冗余的组件关联，尽管在这种情况下分配哪些组件值是未定义的。

  • 对调度子程序的调用未实现。
  • 重载算法比实际的Ada要有限得多（即选择性较差）。它在解析子表达式的含义时，仅有限地利用子表达式出现的上下文，并且在允许类型匹配的规则上更加宽松。因此，一些函数调用将是模糊的，用户将被要求选择合适的解析。
  • new 运算符未实现。
  • 入口调用未实现。
  • 除了打印之外，不支持对原生VAX浮点格式进行算术运算。
  • 无法对压缩数组进行切片。
  • 当 True 和 False 不是限定名称的一部分时，无论上下文如何，它们都被解释为好像隐式地以 Standard 为前缀。如果你的程序在包或过程中重新定义了这些名称（这充其量是一种可疑的做法），你将不得不使用完全限定名称来访问它们的新定义。
  • 基于基数的字面量未实现。
