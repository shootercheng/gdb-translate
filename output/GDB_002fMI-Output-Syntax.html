27.2.2 GDB/MI输出语法 ¶

GDB/MI的输出由零个或多个带外记录组成，后面可以选择跟一个结果记录。这个结果记录是最近一个命令的结果。输出记录序列以‘(gdb)’结束。

如果一个输入命令前缀有一个token，那么该命令的相应输出也将以相同的token为前缀。

output →

( out-of-band-record )* [ result-record ] "(gdb)" nl

result-record →

[ token ] "^" result-class ( "," result )* nl

out-of-band-record →

async-record | stream-record

async-record →

exec-async-output | status-async-output | notify-async-output

exec-async-output →

[ token ] "*" async-output nl

status-async-output →

[ token ] "+" async-output nl

notify-async-output →

[ token ] "=" async-output nl

async-output →

async-class ( "," result )*

result-class →

"done" | "running" | "connected" | "error" | "exit"

async-class →

"stopped" | others（其中others将根据需要添加——仍在开发中）。

result →

variable "=" value

variable →

string

value →

const | tuple | list

const →

c-string

tuple →

"{}" | "{" result ( "," result )* "}"

list →

"[]" | "[" value ( "," value )* "]" | "[" result ( "," result )* "]"

stream-record →

console-stream-output | target-stream-output | log-stream-output

console-stream-output →

"~" c-string nl

target-stream-output →

"@" c-string nl

log-stream-output →

"&" c-string nl

nl →

CR | CR-LF

token →

任意数字序列。

注意：

  • 所有输出序列都以包含一个句号的单行结束。
  • token来自相应的请求。请注意，对于所有异步输出，虽然该token在语法上是允许的，并且未来版本的GDB可能会为选定的异步输出消息输出它，但通常会省略。前端应该将所有异步输出视为报告目标状态的一般变化，并无需将异步输出与任何先前的命令相关联。
  • status-async-output包含有关缓慢操作进度的正在进行的状态信息。它可以被丢弃。所有状态输出都以‘+’为前缀。
  • exec-async-output包含目标上的异步状态变化（停止、启动、消失）。所有异步输出都以‘*’为前缀。
  • notify-async-output包含客户端应处理的补充信息（例如，新的断点信息）。所有通知输出都以‘=’为前缀。
  • console-stream-output是应在控制台中按原样显示的输出。它是对CLI命令的文本响应。所有控制台输出都以‘~’为前缀。
  • target-stream-output是目标程序产生的输出。所有目标输出都以‘@’为前缀。
  • log-stream-output是来自GDB内部的输出文本，例如应作为错误日志的一部分显示的消息。所有日志输出都以‘&’为前缀。
  • 新的GDB/MI命令应仅输出包含values 的lists。

有关各种输出记录的更多详细信息，请参阅 GDB/MI流记录。
