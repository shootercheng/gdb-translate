23.3.2.23 Python中的GDB/MI通知 ¶

可以从Python发出GDB/MI通知。使用gdb.notify_mi函数来实现。

函数：gdb.notify_mi (name [, data]) ¶

发出一个GDB/MI异步通知。name是通知名称，由字母数字字符和连字符（-）组成。data是与通知一起发出的任何附加数据，作为Python字典传递。此参数是可选的。该字典将以与Python MI命令的结果相同的方式转换为GDB/MI result记录（见 GDB/MI输出语法）（见 GDB/MI Python中的命令）。

如果 data 为 None，则不发出其他值。

虽然允许使用现有的通知名称（见 GDB/MI异步记录）与 gdb.notify_mi 一起使用，但建议用户在自定义通知前加上连字符（-）以避免可能的冲突。GDB 永远不会引入以连字符开头的通知。

以下是每当与远程GDB服务器的连接关闭时如何发出 =-connection-removed 通知的方法（见 Python中的连接）： 

def notify_connection_removed(event):
    data = {"id": event.connection.num, "type": event.connection.type}
    gdb.notify_mi("-connection-removed", data)


gdb.events.connection_removed.connect(notify_connection_removed)

然后，每次连接关闭时，MI通道上都会有一个通知： 

=-connection-removed,id="1",type="remote"
