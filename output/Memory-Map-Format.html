E.17内存映射格式 ¶

为了能够写入闪存，GDB需要从目标获取内存映射。本节描述内存映射的格式。

内存映射通过‘qXfer:memory-map:read’（见 qXfer内存映射读取）数据包获取，并且是一个列出内存区域的XML文档。

GDB必须与Expat库链接以支持XML内存映射。见 Expat。

文档的顶级结构如下所示： 

<?xml version="1.0"?>
<!DOCTYPE memory-map
          PUBLIC "+//IDN gnu.org//DTD GDB Memory Map V1.0//EN"
                 "http://sourceware.org/gdb/gdb-memory-map.dtd">
<memory-map>
    region...
</memory-map>

每个区域可以是：

  • 一个从addr开始的RAM区域，从那里扩展length字节： 
    <memory type="ram" start="addr" length="length"/>
  • 一个只读内存区域： 
    <memory type="rom" start="addr" length="length"/>
  • 一个闪存区域，擦除块长度为blocksize字节： 
    <memory type="flash" start="addr" length="length">
  <property name="blocksize">blocksize</property>
</memory>

区域不能重叠。GDB假定内存映射未覆盖的内存区域是RAM，并使用普通的‘M’和‘X’数据包写入此范围内的地址。

内存映射格式的正式DTD如下： 

<!-- ................................................... -->
<!-- 内存映射XML DTD ................................ -->
<!-- 文件：memory-map.dtd .............................. -->
<!-- .................................... .............. -->
<!-- memory-map.dtd -->
<!-- memory-map：带有版本控制的根元素 -->

<!ELEMENT memory-map (memory)*>
<!ATTLIST memory-map    version CDATA   #FIXED  "1.0.0">
<!ELEMENT memory (property)*>
<!-- memory：指定一个内存区域，
             及其类型或设备。 -->
<!ATTLIST memory        type    (ram|rom|flash) #REQUIRED
                        start   CDATA   #REQUIRED
                        length  CDATA   #REQUIRED>
<!-- property：通用属性标签 -->
<!ELEMENT property (#PCDATA | property)*>
<!ATTLIST property      name    (blocksize) #REQUIRED>
