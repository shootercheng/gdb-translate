8.2 回溯 ¶

回溯是对程序如何到达当前位置的总结。它为许多栈帧中的每一个显示一行，从当前正在执行的栈帧（栈帧零）开始，接着是它的调用者（栈帧一），并一直向上到栈顶。

要打印整个栈的回溯，使用 backtrace 命令，或其别名 bt。这个命令将为栈中的每个栈帧打印一行。默认情况下，会打印所有栈帧。你可以在任何时候通过输入系统中断字符（通常是 Ctrl-c）来停止回溯。

backtrace [选项]… [限定符]… [计数]
bt [选项]… [限定符]… [计数]

打印整个栈的回溯。

可选的 计数 可以是以下之一：

n
n

只打印最内层的 n 个栈帧，其中 n 是一个正数。

-n
-n

只打印最外层的 n 个栈帧，其中 n 是一个正数。

选项：

-full

也打印局部变量的值。这可以与可选的 计数 结合使用，以限制显示的栈帧数。

-no-filters

不对这个回溯运行Python栈帧过滤器。有关更多信息，请参阅 过滤栈帧。此外，使用 禁用所有栈帧过滤器 来关闭所有栈帧过滤器。这仅在 GDB 配置了 Python 支持时才相关。

-hide

Python栈帧过滤器可能会决定 “省略” 一些栈帧。通常，这些被省略的栈帧仍然会被打印，但它们相对于导致它们被省略的已过滤栈帧会缩进显示。-hide 选项会导致被省略的栈帧根本不被打印。

backtrace 命令还支持许多选项，这些选项允许覆盖由 set backtrace 和 set print 子命令设置的相关全局打印设置：

-past-main [on|off]

设置回溯是否应继续越过 main 函数。相关设置：设置回溯越过main函数。

-past-entry [on|off]

设置回溯是否应继续越过程序的入口点。相关设置：设置回溯越过入口点。

-entry-values no|only|preferred|if-needed|both|compact|default

设置在函数入口处打印函数参数。 相关设置：设置打印入口值。

-frame-arguments all|scalars|none

设置打印非标量栈帧参数。 相关设置：设置打印帧参数。

-raw-frame-arguments [on|off]

设置是否以原始形式打印栈帧参数。 相关设置：设置打印原始帧参数。

-frame-info auto|source-line|location|source-and-location|location-and-address|short-location

设置打印栈帧信息。 相关设置：设置打印帧信息。

可选的 限定符 是为了向后兼容而保留的。它可以是以下之一：

full

等同于 -full选项。

no-filters

等同于 -no-filters选项。

hide

等同于 -hide选项。

名称 where 和 info stack（缩写为 info s）是 backtrace 的另外两个别名。

在多线程程序中，GDB 默认只显示当前线程的回溯。要显示几个或所有线程的回溯，请使用命令 thread apply（见 线程应用）。例如，如果输入 thread apply all backtrace，GDB 将显示所有线程的回溯；当调试多线程程序的核心转储时，这很方便。

回溯中的每一行显示栈帧编号和函数名。程序计数器值也会显示 —— 除非使用 set print address off。回溯还会显示源文件名和行号，以及函数的参数。如果程序计数器值在该行代码的开头，则会省略。

这是一个回溯的示例。它是用命令 ‘bt 3’ 生成的，所以它显示了最内层的三个栈帧。 

#0  m4_traceon (obs=0x24eb0, argc=1, argv=0x2b8c8)
    at builtin.c:993
#1  0x6e38 in expand_macro (sym=0x2b600, data=...) at macro.c:242
#2  0x6840 in expand_token (obs=0x0, t=177664, td=0xf7fffb08)
    at macro.c:71
(更多栈帧如下...)

栈帧零的显示没有以程序计数器值开头，这表明你的程序在 builtin.c 文件的第 993 行代码的开头停止。

栈帧1中参数 data 的值已被替换为 …。默认情况下，GDB 只在参数是标量（整数、指针、枚举等）时才打印其值。有关如何配置函数参数值打印方式的更多详细信息，请参阅 打印设置 中的 set print frame-arguments 命令。set print frame-info 命令（见 打印设置）控制打印哪些栈帧信息。

如果你的程序是用优化选项编译的，一些编译器会在函数调用后如果参数不再使用就将其优化掉。这种优化会生成通过寄存器传递参数但不在栈帧中存储这些参数的代码。GDB 除了最内层的栈帧外，无法在其他栈帧中显示这些参数。这样的回溯可能如下所示： 

#0  m4_traceon (obs=0x24eb0, argc=1, argv=0x2b8c8)
    at builtin.c:993
#1  0x6e38 in expand_macro (sym=<优化掉>) at macro.c:242
#2  0x6840 in expand_token (obs=0x0, t=<优化掉>, td=0xf7fffb08)
    at macro.c:71
(更多栈帧如下...)

未保存在其栈帧中的参数值显示为 ‘<优化掉>’。

如果你需要显示这些被优化掉的参数的值，可以从其他值依赖于你感兴趣的值的变量中推断出来，或者不使用优化选项重新编译。

大多数程序都有一个标准的用户入口点 —— 系统库和启动代码转换为用户代码的地方。对于C语言来说，这是 main9。当 GDB 在回溯中找到入口函数时，它将终止回溯，以避免跟踪到高度特定于系统的（通常无趣的）代码。

如果你需要检查启动代码，或限制回溯的层数，可以更改此行为：

set backtrace past-main
set backtrace past-main on

回溯将继续越过用户入口点。

set backtrace past-main off

回溯在遇到用户入口点时将停止。这是默认设置。

show backtrace past-main ¶

显示当前用户入口点的回溯策略。

set backtrace past-entry
set backtrace past-entry on

回溯将继续越过应用程序的内部入口点。这个入口点在应用程序构建时由链接器编码，并且可能在调用用户入口点 main (或等效函数) 之前。

set backtrace past-entry off
