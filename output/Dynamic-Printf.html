5.1.8 动态printf ¶

动态printf命令dprintf将断点与格式化打印程序数据相结合，让你能够在无需重新编译程序的情况下，即时在程序中插入printf调用。

在其最基本的形式中，输出会发送到GDB控制台。 然而，你可以设置变量dprintf-style以进行其他处理。 例如，你可以通过调用程序的printf函数来格式化输出。 这样做的优点是字符会发送到程序的输出设备，因此可以记录到重定向到文件等操作中。

如果你使用存根程序或代理进行远程调试，你还可以要求远程代理处理printf。 除了确保输出与程序可能产生的任何其他输出一起发送到远程程序的设备外，你还可以要求即使在与远程目标断开连接后，dprintf仍然保持活动状态。 使用存根程序/代理也更高效，因为它可以在无需与GDB通信的情况下完成所有操作。

dprintf locspec,template,expression[,expression…] ¶

每当执行到达由解析locspec产生的代码位置时，在字符串template的控制下打印一个或多个expression的值。 要打印多个值，用逗号分隔它们。

set dprintf-style style

将dprintf输出设置为以下几种不同样式之一进行处理。 样式的更改会立即影响所有现有的动态printf。 （如果你需要对打印命令进行单独控制，只需使用明确提供的命令列表定义普通断点。）

gdb ¶

使用GDB的printf命令处理输出。 使用此样式时，可以使用‘%V’格式说明符（见 %V格式说明符）。

call ¶

通过调用程序中的函数（通常是printf）来处理输出。 使用此样式时，支持的格式说明符完全取决于所调用的函数。

大多数GDB的格式说明符与printf函数支持的格式说明符一致，但是，GDB的‘%V’格式说明符扩展不被printf支持。 当使用‘call’样式的dprintf时，应注意确保只使用输出函数支持的格式说明符，否则结果将是未定义的。

agent ¶

让远程调试代理（如gdbserver）自行处理输出。 此样式仅适用于支持在目标上运行命令的代理。 此样式不支持‘%V’格式说明符。

set dprintf-function function

如果dprintf样式为call，则设置要调用的函数。 默认情况下，其值为printf。 你可以将其设置为GDB可以求值为函数的任何表达式，就像call命令一样。

set dprintf-channel channel

为dprintf设置一个“通道”。 如果设置为非空值，GDB将把它作为一个表达式求值，并将结果作为第一个参数传递给dprintf-function，就像fprintf和类似函数一样。 否则，dprintf格式字符串将是第一个参数，就像printf一样。

例如，如果要将dprintf输出发送到一个日志文件，该日志文件是分配给变量mylog的标准I/O流，你可以这样做： 

(gdb) set dprintf-style call
(gdb) set dprintf-function fprintf
(gdb) set dprintf-channel mylog
(gdb) dprintf 25,"at line 25, glob=%d\n",glob
Dprintf 1 at 0x123456: file main.c, line 25.
(gdb) info break
1       dprintf        keep y   0x00123456 in main at main.c:25
        call (void) fprintf (mylog,"at line 25, glob=%d\n",glob)
        continue
(gdb)

请注意，info break将动态printf命令显示为普通断点命令；因此你可以轻松看到变量设置的效果。

set disconnected-dprintf on ¶
set disconnected-dprintf off

选择如果GDB已与目标断开连接，dprintf命令是否应继续运行。 这仅在dprintf-style为agent时适用。

show disconnected-dprintf off ¶

显示当前对断开连接的dprintf的选择。

GDB不会检查函数和通道的有效性，而是依赖你提供在使用它们的上下文中有意义的值。 例如，函数和通道可能是局部变量的值，但如果是这种情况，那么所有启用的动态打印必须在这些局部变量的作用域内的位置。 如果求值失败，GDB将报告错误。
