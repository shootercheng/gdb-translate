E.20分支跟踪格式 ¶

为了显示一个被调试进程线程的分支跟踪，GDB需要获取分支列表。这个列表表示为通过分支连接的连续代码块的列表。每个块中的代码是按顺序执行的。

这个列表是使用‘qXfer:btrace:read’（见 qXfer btrace read）数据包获取的，并且是一个XML文档。

GDB必须与Expat库链接以支持XML跟踪帧信息发现。见 Expat。

文档顶层结构如下所示： 

<?xml version="1.0"?>
<!DOCTYPE btrace
          PUBLIC "+//IDN gnu.org//DTD GDB Branch Trace V1.0//EN"
                 "http://sourceware.org/gdb/gdb-btrace.dtd">
<btrace>
   block...
</btrace>
  • 从begin开始并在end结束的连续执行指令块： 
    <block begin="begin" end="end"/>

分支跟踪格式的正式DTD如下： 

<!ELEMENT btrace  (block* | pt) >
<!ATTLIST btrace  version CDATA   #FIXED "1..">

<!ELEMENT block        EMPTY>
<!ATTLIST block        begin  CDATA   #REQUIRED
                       end    CDATA   #REQUIRED>

<!ELEMENT pt (pt-config?, raw?)>

<!ELEMENT pt-config (cpu?)>

<!ELEMENT cpu EMPTY>
<!ATTLIST cpu vendor   CDATA #REQUIRED
              family   CDATA #REQUIRED
              model    CDATA #REQUIRED
              stepping CDATA #REQUIRED>

<!ELEMENT raw (#PCDATA)>
