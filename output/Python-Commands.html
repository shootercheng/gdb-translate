23.3.1 Python命令 ¶

GDB提供了两个用于访问Python解释器的命令，以及一个相关设置：

python-interactive [command] ¶
pi [command]

不带参数时，python-interactive命令可用于启动交互式Python提示符。要返回GDB，请输入Ctrl-D（例如，在空提示符上）。

或者，可以将单行Python命令作为参数给出并求值。如果命令是一个表达式，结果将被打印；否则，不会打印任何内容。例如： 

(gdb) python-interactive 2 + 3
5
python [command] ¶
py [command]

python命令可用于求值Python代码。

如果给定一个参数，python命令将把该参数作为Python命令求值。例如： 

(gdb) python print 23
23

如果不向python提供参数，它将作为多行命令，类似于define。在这种情况下，Python脚本由后续的命令行组成，在python命令之后给出。这个命令列表使用包含end的行终止。例如： 

(gdb) python
>print 23
>end
23
set python print-stack ¶

默认情况下，当Python脚本中发生错误时，GDB只会打印Python异常的消息部分。可以使用set python print-stack来控制：如果是full，则启用完整的Python堆栈打印；如果是none，则禁用Python堆栈和消息打印；如果是message（默认），则只打印错误的消息部分。

set python ignore-environment [on|off] ¶

默认情况下，此选项为“off”，当GDB初始化其内部Python解释器时，Python解释器将检查环境中是否存在会影响其行为的变量，例如PYTHONHOME和PYTHONPATH18。

如果在初始化Python之前将此选项设置为“on”，则Python将忽略所有此类环境变量。由于Python在GDB启动过程的早期初始化，因此必须将此选项放入早期初始化文件中（请参阅初始化文件）才能产生预期效果。

此选项等效于将-E传递给实际的python可执行文件。

set python dont-write-bytecode [auto|on|off] ¶

当此选项为“off”时，一旦GDB初始化了Python解释器，解释器将对其导入的任何Python模块进行字节编译，并将字节码写入磁盘上的.pyc文件中。

如果在初始化Python之前将此选项设置为“on”，则Python将不再将字节码写入磁盘。由于Python在GDB启动过程的早期初始化，因此必须将此选项放入早期初始化文件中（请参阅初始化文件）才能产生预期效果。

默认情况下，此选项设置为“auto”。在这种模式下，只要python ignore-environment设置为“off”，就会检查环境变量PYTHONDONTWRITEBYTECODE，以确定是否应该写出字节码。 当PYTHONDONTWRITEBYTECODE设置为空字符串或不存在时，认为它是关闭/禁用的。所有其他设置，包括那些似乎没有意义的设置，都表示它是打开/启用的。

此选项等效于将-B传递给实际的python可执行文件。

也可以从GDB解释器执行Python脚本：

source script-name

脚本名称必须以“.py”结尾，并且GDB必须配置为使用script-extension设置根据文件名扩展名识别脚本语言。请参阅扩展GDB。

以下命令旨在帮助调试GDB本身：

set debug py-breakpoint on|off ¶
show debug py-breakpoint

当为“on”时，GDB会打印与Python断点API相关的调试消息。默认情况下为“off”。

set debug py-unwind on|off ¶
show debug py-unwind

当为“on”时，GDB会打印与Python unwind API相关的调试消息。默认情况下为“off”。
