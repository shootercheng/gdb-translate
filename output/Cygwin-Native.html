21.1.4 调试MS Windows PE可执行文件的功能 ¶

GDB支持对MS Windows程序进行原生调试，包括有和没有符号调试信息的DLL。

调用SetConsoleMode以关闭“Ctrl-C”按键特殊含义的MS-Windows程序不能通过键入 C-c 来中断。因此，MS-Windows上的GDB支持 C-BREAK 作为替代的中断键序列，即使调试对象忽略 C-c，也可用于中断调试对象。

本节介绍了各种其他特定于Cygwin的命令。在 对没有调试符号的DLL的支持 中描述了处理没有调试符号的DLL的情况。

info w32 ¶

这是特定于MS Windows命令的前缀，用于打印有关目标系统和重要OS结构的信息。

info w32 selector

此命令显示Win3 API GetThreadSelectorEntry 函数返回的信息。它接受一个可选参数，该参数被计算为一个长值，以给出有关此给定选择器的信息。无参数时，此命令显示有关六个段寄存器的信息。

info w32 thread-information-block

此命令显示存储在线程信息块中的线程特定信息（在X86 CPU系列上，对于32位程序使用 $fs 选择器，对于64位程序使用 $gs 选择器可读）。

signal-event id ¶

此命令用用户提供的 id 发出一个事件信号。用于在使用MS-Windows JIT调试（AeDebug）附加到崩溃进程时恢复该进程。

要使用它，请在 HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\AeDebug 和/或 HKLM\SOFTWARE\Wow6432Node\Microsoft\Windows NT\CurrentVersion\AeDebug（对于x86_64版本）中创建或编辑以下键：

  • Debugger（REG_SZ） —— 启动调试器的命令。 建议的命令是：fully-qualified-path-to-gdb.exe -ex "attach %ld" -ex "signal-event %ld" -ex "continue"。

    第一个 %ld 将被崩溃进程的进程ID替换，第二个 %ld 将被阻止崩溃进程并等待GDB附加的事件ID替换。

  • Auto（REG_SZ） —— 要么是 数值1 要么是 数值0。数值1 将使系统自动运行由Debugger键指定的调试器，数值0 将导致出现一个带有“确定”和“取消”按钮的对话框，允许用户终止崩溃进程（确定）或调试它（取消）。
set cygwin-exceptions mode ¶

如果 mode 是 on，GDB将在Cygwin DLL内部发生异常时中断。如果 mode 是 off，GDB将延迟对异常的识别，并且可能忽略一些似乎由Cygwin DLL内部“簿记”引起的异常。此选项主要用于调试Cygwin DLL本身；默认值是 off，以避免用虚假的 SIGSEGV 信号打扰GDB用户。

show cygwin-exceptions ¶

显示GDB是否会在Cygwin DLL本身内部发生的异常时中断。

set new-console mode ¶

如果 mode 是 on，调试对象将在下一次启动时在新控制台中启动。 如果 mode 是 off，调试对象将在与调试器相同的控制台中启动。

show new-console ¶

显示启动调试对象时是否使用新控制台。

set new-group mode ¶

此布尔值控制调试对象是应启动新组还是与调试器留在同一组中。 这会影响Windows OS处理“Ctrl-C”的方式。

show new-group ¶

显示new-group布尔值的当前值。

set debugevents ¶

此布尔值会添加与调试器看到的与调试对象相关的内核事件有关的调试输出。这包括发出线程和进程创建及退出信号的事件、DLL加载和卸载、控制台中断以及由Windows OutputDebugString API调用产生的调试消息。

set debugexec ¶

此布尔值会添加与调试器看到的执行事件（如恢复线程）有关的调试输出。

set debugexceptions ¶

此布尔值会添加与调试器看到的调试对象中的异常有关的调试输出。

set debugmemory ¶

此布尔值会添加与调试器对调试对象内存的读取和写入有关的调试输出。

set shell ¶

此布尔值指定调试对象是通过 shell 调用还是直接调用（默认值是开启）。

show shell ¶

显示调试对象是否将通过 shell 启动。
