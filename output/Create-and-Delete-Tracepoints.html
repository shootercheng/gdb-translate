13.1.1 创建和删除跟踪点 ¶

trace locspec ¶

trace命令与break命令非常相似。 其参数locspec可以是任何有效的位置规范。 请参阅 位置规范。trace命令定义一个跟踪点， 这是目标程序中的一个点，调试器将在此短暂停止， 收集一些数据，然后允许程序继续。如果远程存根 支持“InstallInTrace”功能（请参阅 在跟踪中安装跟踪点），则设置跟踪点 或更改其操作将立即生效。 如果远程存根不支持“InstallInTrace”功能，所有 这些更改直到下一个tstart 命令才会生效，并且一旦跟踪实验正在运行，进一步的更改将 直到下一个跟踪实验开始才会有任何效果。此外， GDB支持待决跟踪点 —— 其 地址尚未解析的跟踪点。（这类似于待决断点。） 待决跟踪点不会下载到目标，并且在解析之前不会安装。 待决跟踪点的解析需要GDB支持 —— 当使用远程目标进行调试时，并且 GDB与远程存根断开连接（请参阅 断开连接的跟踪），在GDB断开连接时，待决跟踪点无法解析（并下载到 远程存根）。

以下是使用trace命令的一些示例： 

(gdb) trace foo.c:121    // 一个源文件和行号

(gdb) trace +2           // 向前2行

(gdb) trace my_function  // 函数的第一行源代码

(gdb) trace *my_function // 函数的确切起始地址

(gdb) trace *0x2117c4    // 一个地址

你可以将trace缩写为tr。

trace locspec if cond

设置一个带有条件cond的跟踪点；每次到达跟踪点时评估表达式 cond，并且仅当值非零时才收集数据 —— 也就是说，如果cond求值为真。 有关跟踪点条件的更多信息，请参阅 跟踪点条件。
