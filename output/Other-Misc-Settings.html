22.11其他杂项设置 ¶

set interactive-mode ¶

如果设置为on，则强制GDB假定它是在终端中启动的。实际上，这意味着GDB应该等待用户回答在命令提示符处输入的命令所产生的查询。如果设置为off，则强制GDB以相反的模式运行，并且它使用所有查询的默认答案。如果设置为auto（默认值），GDB会尝试确定其标准输入是否为终端，如果是，则以交互模式工作，否则以非交互模式工作。

在绝大多数情况下，调试器应该能够正确猜出应该使用哪种模式。但此设置在某些特定情况下可能会很有用，例如在cygwin窗口中运行MinGW GDB。

show interactive-mode ¶

显示调试器是否正在以交互模式运行。

set suppress-cli-notifications ¶

如果设置为on，则抑制GDB打印的命令行界面（CLI）通知。如果设置为off，则像往常一样打印通知。默认值为off。当你更改选定的上下文或正在调试的程序停止时，会发生CLI通知，如下所述。

用户选定的上下文更改：

当你更改选定的上下文（即当前进程、线程和/或帧）时，GDB会打印有关新上下文的信息。例如，默认行为如下： 

(gdb) inferior 1
[切换到进程1 [进程634] (/tmp/test)]
[切换到线程1（进程634）]
#0  main () 在test.c:3
3         return 0;
(gdb)

当通知被抑制时，不会打印新上下文： 

(gdb) set suppress-cli-notifications on
(gdb) inferior 1
(gdb)
正在调试的程序停止：

当你正在调试的程序停止时（例如，因为命中断点、完成源步进、中断等），GDB会打印有关停止事件的信息。例如，下面是一个断点命中： 

(gdb) break test.c:�
断点2位于0x555555555155：文件test.c，第3行。
(gdb) continue
继续。

断点2，main () 在test.c:3
3         return 0;
(gdb)

当通知被抑制时，输出变为： 

(gdb) break test.c:3
断点2位于0x555555555155：文件test.c，第3行。
(gdb) set suppress-cli-notifications on
(gdb) continue
继续。
(gdb)

抑制CLI通知在脚本中可能很有用，以便从命令列表中获得简化的输出。

show suppress-cli-notifications ¶

显示是否抑制打印CLI通知。
