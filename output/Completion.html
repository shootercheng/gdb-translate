3.3 命令补全 ¶

如果只有一种可能，GDB可以为你补全命令中的单词；它也可以在任何时候向你展示命令中下一个单词的有效可能性。这适用于GDB命令、GDB子命令、命令选项以及程序中的符号名称。

每当你希望GDB补全单词的其余部分时，按下TAB键。如果只有一种可能，GDB会补全单词，并等待你完成命令（或按下RET键输入它）。例如，如果输入 

(gdb) info breTAB

GDB会补全单词“breakpoints”的其余部分，因为这是唯一以“bre”开头的info子命令： 

(gdb) info breakpoints

此时你可以按下RET键来运行info breakpoints命令，或者按退格键并输入其他内容，如果“breakpoints”看起来不是你期望的命令。（如果你一开始就确定想要info breakpoints，你也可以在“info bre”之后立即按下RET键，以利用命令缩写而不是命令补全）。

当你按下TAB键时，如果下一个单词有多种可能，GDB会发出提示音。你可以提供更多字符并再次尝试，或者再按一次TAB键；GDB会显示该单词的所有可能补全。例如，你可能想在一个以“make_”开头的子例程上设置断点，但当你输入b make_TAB时，GDB只会发出提示音。再次按下TAB键会显示程序中所有以这些字符开头的函数名，例如： 

(gdb) b make_TAB

GDB发出提示音；再按一次TAB键，查看：

make_a_section_from_file     make_environ
make_abs_section             make_function_type
make_blockvector             make_pointer_type
make_cleanup                 make_reference_type
make_command                 make_symbol_completion_list
(gdb) b make_

显示可用可能性后，GDB会复制你的部分输入（例如示例中的“b make_”），以便你完成命令。

如果你试图补全的命令期望接下来是一个关键字或数字，那么“NUMBER”会出现在可用补全中，例如： 

(gdb) print -elements TABTAB
NUMBER     unlimited
(gdb) print -elements 

这里，该选项期望一个数字（例如100），而不是字面的NUMBER。这种元语法参数总是以大写形式呈现。

如果你只是想首先查看替代列表，可以按下M-?而不是按两次TAB键。M-?表示META ?。你可以通过在键盘上按住指定为META shift的键（如果有的话）同时输入?来输入，或者输入ESC后跟 ?。

如果可能的补全数量很大，GDB会打印出它收集到的尽可能多的列表，以及一条表明列表可能被截断的消息。 

(gdb) b mTABTAB
main
<... 其余可能的补全 ...>
*** 列表可能被截断，达到最大补全数。 ***
(gdb) b m

可以使用以下命令控制此行为：

set max-completions limit ¶
set max-completions unlimited

设置补全候选的最大数量。GDB收集到这么多候选后将停止寻找更多补全。在补全函数名等内容时这很有用，因为收集所有可能的候选可能很耗时。 默认值是200。值为零会禁用制表符补全。请注意，设置无限制或非常大的限制可能会使补全变慢。

show max-completions

显示GDB在补全过程中收集并显示的候选的最大数量。

有时你需要的字符串，从逻辑上讲是一个“单词”，但可能包含括号或其他GDB通常在其单词概念中排除的字符。为了在这种情况下允许单词补全起作用，你可以在 GDB 命令中用 '（单引号）将单词括起来。

你可能需要这样做的一个常见情况是在输入一个涉及带有模板参数的C++符号名称的表达式时。这是因为在补全表达式时，GDB将“<”字符视为单词分隔符，假设它是小于比较运算符（见 C和C++运算符）。

例如，当你想使用 print 或 call 命令交互式调用一个C++模板函数时，你可能需要区分你指的是专门针对 int 的 name 版本，即 name<int>()，还是专门针对 float 的版本，即 name<float>()。在这种情况下，要使用单词补全功能，在函数名开头输入一个单引号 '。这会提醒 GDB，当你按下 TAB 或 M-? 请求单词补全时，它可能需要考虑比平时更多的信息： 

(gdb) p 'func<M-?
func<int>()    func<float>()
(gdb) p 'func<

然而，在设置断点时（见 位置规范），你通常不需要在函数名前输入引号，因为 GDB 理解你想在一个函数上设置断点： 

(gdb) b func<M-?
func<int>()    func<float>()
(gdb) b func<

即使在输入C++重载函数（同一函数的多个定义，通过参数类型区分）的名称时也是如此。例如，当你想设置断点时，你不需要区分你指的是接受 int 参数的 name 版本，即 name(int)，还是接受 float 参数的版本，即 name(float)。 

(gdb) b bubble(M-?M-?

有关其他需要加引号的场景的描述，请参阅 引用名称。

有关重载函数的更多信息，请参阅 C++表达式。你可以使用命令 set overload-resolution off 禁用重载解析； 见 GDB的C++功能。

当在一个表达式中补全查找结构体中的字段时，GDB也会尝试3将补全限制为左侧类型中可用的字段名： 

(gdb) p gdb_stdout.M-?
magic                to_fputs             to_rewind
to_data              to_isatty            to_write
to_delete            to_put               to_write_async_safe
to_flush             to_read

这是因为 gdb_stdout 是 struct ui_file 类型的变量，在 GDB 源文件中定义如下： 

struct ui_file
{
   int *magic;
   ui_file_flush_ftype *to_flush;
   ui_file_write_ftype *to_write;
   ui_file_write_async_safe_ftype *to_write_async_safe;
   ui_file_fputs_ftype *to_fputs;
   ui_file_read_ftype *to_read;
   ui_file_delete_ftype *to_delete;
   ui_file_isatty_ftype *to_isatty;
   ui_file_rewind_ftype *to_rewind;
   ui_file_put_ftype *to_put;
   void *to_data;
}

脚注

(③)

某些无效表达式可能会使补全器感到困惑。此外，它只检查表达式的静态类型，而不是动态类型。
