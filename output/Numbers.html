22.6 数字 ¶

在GDB中，你始终可以按照通常的约定以八进制、十进制或十六进制输入数字：八进制数字以‘0’开头，十进制数字以‘.’结尾，十六进制数字以‘0x’开头。既不以‘0’也不以‘0x’开头，且不以‘.’结尾的数字，默认以十进制输入；同样，在未指定特定格式时，数字默认的显示格式也是十进制。你可以使用下面描述的命令更改输入和输出的默认进制。

set input-radix 基数 ¶

设置数字输入的默认进制。基数支持的选择是十进制的8、10或16。基数本身必须明确指定，或者使用当前的输入进制；例如，以下任何一个 

set input-radix 012
set input-radix 10.
set input-radix 0xa

都将输入进制设置为十进制。另一方面，‘set input-radix 10’无论当前进制是什么，都不会改变输入进制，因为‘10’没有任何表示其进制的前导或尾随符号，它是按照当前进制来解释的。因此，如果当前进制是16，‘10’会被解释为十六进制，即十进制的16，这并不会改变进制。

set output-radix 基数 ¶

设置数字显示的默认进制。基数支持的选择是十进制的8、10或16。基数本身必须明确指定，或者使用当前的输入进制。

show input-radix ¶

显示数字输入的当前默认进制。

show output-radix ¶

显示数字显示的当前默认进制。
