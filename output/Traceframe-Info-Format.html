E.19跟踪帧信息格式 ¶

为了能够知道在检查跟踪点命中时可以检查下级中的哪些对象，GDB需要获取在跟踪帧中收集的内存范围、寄存器和跟踪状态变量的列表。

这个列表是使用“qXfer:traceframe-info:read”（见 qXfer跟踪帧信息读取）数据包获取的，并是一个XML文档。

GDB必须与Expat库链接以支持XML跟踪帧信息发现。见 Expat。

文档的顶级结构如下所示： 

<?xml version="1.0"?>
<!DOCTYPE traceframe-info
          PUBLIC "+//IDN gnu.org//DTD GDB Memory Map V1.0//EN"
                 "http://sourceware.org/gdb/gdb-traceframe-info.dtd">
<traceframe-info>
   block...
</traceframe-info>

每个跟踪帧块可以是：

  • 一个从addr开始并从那里延伸length字节的收集内存区域： 
    <memory start="addr" length="length"/>
  • 一个块，指示已收集编号为number的跟踪状态变量： 
    <tvar id="number"/>

跟踪帧信息格式的正式DTD如下所示： 

<!ELEMENT traceframe-info  (memory | tvar)* >
<!ATTLIST traceframe-info  version CDATA   #FIXED  "1.0">

<!ELEMENT memory        EMPTY>
<!ATTLIST memory        start   CDATA   #REQUIRED
                        length  CDATA   #REQUIRED>
<!ELEMENT tvar>
<!ATTLIST tvar          id      CDATA   #REQUIRED>
