23.3.2.27 Python中的目标文件 ¶

GDB 从各种包含符号的文件中为下级加载符号（见 指定文件的命令）。这些文件包括主可执行文件、下级使用的任何共享库以及任何单独的调试信息文件（见 单独文件中的调试信息）。GDB 将这些包含符号的文件称为 目标文件。

gdb 模块中提供了以下与目标文件相关的函数：

函数： gdb.current_objfile () ¶

在自动加载Python脚本时（见 Python自动加载），GDB 将 “当前目标文件” 设置为相应的目标文件。此函数返回当前目标文件。如果没有当前目标文件，此函数返回 None。

函数： gdb.objfiles () ¶

返回当前程序空间引用的目标文件序列。见 Python中的目标文件，以及 Python中的程序空间。这与 gdb.selected_inferior().progspace.objfiles() 相同，为保持历史兼容性而包含。

函数： gdb.lookup_objfile (name [, by_build_id]) ¶

在当前程序空间的目标文件列表中查找 name，它可以是文件名或构建ID（见 Python中的程序空间）。如果未找到目标文件，则抛出Python ValueError 异常。

如果 name 是相对文件名，那么它将匹配任何具有相同尾部组件的源文件名。例如，如果 name 是 ‘gcc/expr.c’，那么它将匹配 /build/trunk/gcc/expr.c 的源文件名，但不匹配 /build/trunk/libcpp/expr.c 或 /build/trunk/gcc/x-expr.c。

如果提供了 by_build_id 且其值为 True，那么 name 是目标文件的构建ID。否则，name 是文件名。此功能仅在某些操作系统上受支持，特别是那些使用ELF格式的二进制文件和 GNU Binutils 的操作系统。有关此功能的更多详细信息，请参阅 The GNU Linker 中 命令行选项 中 --build-id 命令行选项的描述。

每个目标文件由 gdb.Objfile 类的一个实例表示。
