23.3.2.11 装饰帧 ¶

帧装饰器是帧过滤器的姊妹对象（见 过滤帧）。帧装饰器由帧过滤器应用，并且只能与帧过滤器结合使用。

帧装饰器的目的是在执行帧过滤器的命令中自定义每个 gdb.Frame 的打印内容。这个概念叫做装饰帧。帧装饰器用每个API调用中包含的Python代码来装饰一个 gdb.Frame。这将 gdb.Frame 中包含的实际数据与帧装饰器产生的装饰后的数据分开。这种抽象对于维护每个 gdb.Frame 中包含的数据的完整性是必要的。

帧装饰器有一个强制接口，如下定义。

GDB 已经包含一个名为 FrameDecorator 的帧装饰器。它包含大量用于装饰 gdb.Frame 内容的样板代码。建议其他帧装饰器继承并扩展这个对象，并且只覆盖所需的方法。

FrameDecorator 在Python模块 gdb.FrameDecorator 中定义，所以你的代码可以像这样导入它： 

from gdb.FrameDecorator import FrameDecorator
函数：FrameDecorator.elided (self) ¶

elided 方法在一个分层系统中将帧组合在一起。一个例子是解释器，其中多个低级帧在解释语言中构成一个单一调用。在这个例子中，帧过滤器将省略低级帧，并向用户呈现一个单一的高级帧，代表解释语言中的调用。

elided 函数必须返回一个可迭代对象，并且这个可迭代对象必须包含被省略的帧，这些帧用合适的帧装饰器包装。如果没有帧被省略，这个函数可以返回一个空的可迭代对象，或者 None。在 CLI 回溯中，被省略的帧相对于正常帧缩进，或者在 GDB/MI 的情况下，被放在省略帧的 children 字段中。

从源迭代器中过滤出被省略的帧也是帧过滤器的任务。这将避免两次打印帧。

函数：FrameDecorator.function (self) ¶

这个方法返回要打印的帧中的函数名称。

这个方法必须返回一个描述函数的Python字符串，或者 None。

如果这个函数返回 None，GDB 将不会为这个字段打印任何数据。

函数：FrameDecorator.address (self) ¶

这个方法返回要打印帧的地址。

这个方法必须返回一个足够大小的Python数值整数类型来描述帧的地址，或者 None。

如果这个函数返回 None，GDB 将不会为这个字段打印任何数据。

函数：FrameDecorator.filename (self) ¶

这个方法返回与这个帧相关联的文件名和路径。

这个方法必须返回一个包含文件名和支持该帧的目标文件路径的Python字符串，或者 None。

如果这个函数返回 None，GDB 将不会为这个字段打印任何数据。

函数：FrameDecorator.line (self): ¶

这个方法返回与这个帧所指向的函数内当前位置相关联的行号。

这个方法必须返回一个Python整数类型，或者 None。

如果这个函数返回 None，GDB 将不会为这个字段打印任何数据。

函数：FrameDecorator.frame_args (self) ¶

这个方法必须返回一个可迭代对象，或者 None。返回一个空的可迭代对象或 None 意味着不会为这个帧打印帧参数。这个可迭代对象必须包含实现这里描述的两个方法的对象。

这个对象必须实现一个 symbol 方法，该方法接受一个单一的 self 参数，并且必须返回一个 gdb.Symbol（见 Python中的符号表示），或者一个Python字符串。该对象还必须实现一个 value 方法，该方法接受一个单一的 self 参数，并且必须返回一个 gdb.Value（见 来自下级的值）、一个Python值或 None。如果 value 方法返回 None，并且 argument 方法返回一个 gdb.Symbol，GDB 将自动查找并打印 gdb.Symbol 的值。

class SymValueWrapper():

    def __init__(self, symbol, value):
        self.sym = symbol
        self.val = value

    def value(self):
        return self.val

    def symbol(self):
        return self.sym
