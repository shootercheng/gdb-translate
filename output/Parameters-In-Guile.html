23.4.3.12 Guile中的参数 ¶

你可以使用Guile实现新的GDB 参数 19。

已经存在许多可以在GDB中设置的参数。 两个例子是：set follow-fork 和 set charset。 设置这些参数会影响GDB中的某些行为。 同样，你可以定义可用于影响自定义Guile脚本和命令中行为的参数。

使用Guile函数 make-parameter 定义新参数，并使用Guile函数 register-parameter! 将其添加到GDB中。 采取这种两步法是为了将向GDB添加参数的副作用与 make-parameter 分开。

参数通过 set 和 show 命令向用户公开。 请参阅 获取帮助。

Scheme过程： make-parameter name [#:command-class command-class] [#:parameter-type parameter-type] [#:enum-list enum-list] [#:set-func set-func] [#:show-func show-func] [#:doc doc] [#:set-doc set-doc] [#:show-doc show-doc] [#:initial-value initial-value] ¶

参数 name 是新参数的名称。 如果 name 由多个单词组成，则最初的单词会被视为前缀参数进行查找。 这方面的一个例子可以用 set print 这组参数来说明。 如果 name 是 print foo，那么 print 将被作为前缀参数进行搜索。 在这种情况下，该参数随后可以在GDB中通过 set print foo 来访问。 如果 name 由多个单词组成，并且找不到前缀参数组，则会引发异常。

结果是表示该参数的 <gdb:parameter> 对象。 在使用 register-parameter! 将其注册到GDB之前，该参数不可用。

其余参数是可选的。

参数 command-class 应该是 ‘COMMAND_’ 常量之一（请参阅 Guile中的命令）。 此参数告诉GDB如何在帮助系统中对新参数进行分类。 默认值是 COMMAND_NONE。

参数 parameter-type 应该是下面定义的 ‘PARAM_’ 常量之一。 此参数告诉GDB新参数的类型；此信息用于输入验证和自动完成。 默认值是 PARAM_BOOLEAN。

如果 parameter-type 是 PARAM_ENUM，那么 enum-list 必须是一个字符串列表。 这些字符串表示参数的可能值。

如果 parameter-type 不是 PARAM_ENUM，那么 enum-list 的存在将导致抛出异常。

参数 set-func 是一个单参数函数：self，它是表示该参数的 <gdb:parameter> 对象。 当通过 set API更改参数的值时（例如，set foo off），GDB将调用此函数。 参数的值已经设置为新值。 此函数必须返回一个要显示给用户的字符串。 如果字符串不为空，GDB将添加一个换行符。 当设置参数时，GDB通常不打印任何内容，因此通常此函数应返回 ‘""’。 通常应使用非空字符串结果来显示警告和错误。

参数 show-func 是一个双参数函数：self，它是表示该参数的 <gdb:parameter> 对象，以及 svalue，它是当前值的字符串表示。 当调用参数的 show API时（例如，show foo），GDB将调用此函数。 此函数必须返回一个字符串，并将显示给用户。 GDB将添加一个换行符。

参数 doc 是新参数的帮助文本。 如果没有文档字符串，则使用默认值。 如果文档字符串为空，则GDB将只打印 set-doc 和 show-doc 字符串（见下文）。

参数 set-doc 是此参数的 set 命令的帮助文本。

参数 show-doc 是此参数的 show 命令的帮助文本。

参数 initial-value 指定参数的初始值。 如果它是一个函数，它接受一个参数，即 <gdb:parameter> 对象，其结果用作参数的初始值。 初始值必须对参数类型有效，否则将抛出异常。

Scheme过程： register-parameter! parameter ¶

将 parameter（一个 <gdb:parameter> 对象）添加到GDB的参数列表中。 多次注册参数是错误的。 结果未指定。

Scheme过程： parameter? object ¶

如果 object 是一个 <gdb:parameter> 对象，则返回 #t。 否则返回 #f。

Scheme过程： parameter-value parameter ¶

返回 parameter 的值，它可以是一个 <gdb:parameter> 对象或一个命名该参数的字符串。

Scheme过程： set-parameter-value! parameter new-value ¶

将 parameter 的值设置为 new-value。 参数 parameter 必须是 <gdb:parameter> 类型的对象。 GDB在进行赋值时会进行验证。

定义新参数时，必须指定其类型。 可用类型由 gdb 模块中定义的常量表示：

PARAM_BOOLEAN ¶

值是一个普通布尔值。 Guile布尔值 #t 和 #f 是唯一有效的值。

PARAM_AUTO_BOOLEAN ¶

该值有三种可能的状态：真、假和 ‘auto’。 在Guile中，真和假用布尔常量表示，‘auto’ 用 #:auto 表示。

PARAM_UINTEGER ¶

值是一个无符号整数。 #:unlimited 的值应解释为“无限制”，‘0’ 的值保留，不应使用。

PARAM_ZINTEGER ¶

值是一个整数。

PARAM_ZUINTEGER ¶

值是一个无符号整数。

PARAM_ZUINTEGER_UNLIMITED ¶

值是一个范围在 ‘[0, INT_MAX]’ 的整数。 #:unlimited 的值表示“无限制”，‘-1’ 的值保留，不应使用，并且不允许其他负数。

PARAM_STRING ¶

值是一个字符串。 当用户修改字符串时，任何转义序列，如 ‘\t’、‘\f’ 和八进制转义，都会被转换为相应的字符并编码为当前主机字符集。

PARAM_STRING_NOESCAPE ¶

值是一个字符串。 当用户修改字符串时，转义字符将不被转换直接传递。

PARAM_OPTIONAL_FILENAME ¶

值是一个文件名（字符串）或 #f。

PARAM_FILENAME ¶

值是一个文件名。 这与 PARAM_STRING_NOESCAPE 类似，但使用文件名进行自动完成。

PARAM_ENUM ¶

值是一个字符串，它必须是创建参数时提供的一组字符串常量之一。

PARAM_COLOR ¶

值可以是一个字符串或一个无符号整数。 从0到255的整数表示终端颜色调色板中的索引。 字符串可以是 ‘#RRGGBB’ 格式的十六进制RGB三元组或以下颜色名称之一：‘none’（表示终端默认颜色）、‘black’、‘red’、‘green’、‘yellow’、‘blue’、‘magenta’、‘cyan’ 或 ‘white’。

脚注

(19)

请注意，GDB参数绝不能与Guile的参数对象混淆（请参阅 GNU Guile参考手册 中的 参数）。
