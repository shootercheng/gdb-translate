2.1.2选择模式 ¶

你可以在各种替代模式下运行GDB，例如批处理模式或安静模式。

-nx ¶
-n

不执行在任何初始化文件中找到的命令（见 初始化文件）。

-nh ¶

不执行在任何主目录初始化文件中找到的命令（见 主目录初始化文件）。系统范围和当前目录的初始化文件仍会被加载。

-quiet ¶
-silent
-q

“安静”。不打印介绍性和版权信息。这些信息在批处理模式下也会被抑制。

这也可以使用 set startup-quietly on 来启用。默认值是 off。使用 show startup-quietly 来查看当前设置。将 set startup-quietly on 放入你的早期初始化文件（见 初始化文件），以便未来的GDB会话安静地启动。

)
-batch ¶

以批处理模式运行。在处理完所有用‘-x’指定的命令文件（以及初始化文件中的所有命令，如果没有用‘-n’禁止）后，以状态码 0 退出。如果在执行命令文件中的GDB命令时发生错误，则以非零状态退出。批处理模式还会禁用分页，设置无限的终端宽度和高度，见 屏幕大小，并且行为就好像 set confirm off 生效一样（见 可选的警告和消息）。

批处理模式对于将GDB用作过滤器可能很有用，例如在另一台计算机上下载并运行程序；为了使其更有用，消息

-batch-silent ¶

以与‘-batch’完全相同的批处理模式运行，但完全安静。防止所有GDB输出到 stdout（stderr 不受影响）。这比‘-silent’安静得多，对于交互式会话将毫无用处。

例如，当使用给出‘Loading section’消息的目标时，这特别有用。

请注意，通过GDB输出其输出的目标（而不是直接写入 stdout）也将被设为安静。

-return-child-result ¶

GDB的返回码将是子进程（被调试的进程）的返回码，但有以下例外情况：

  • GDB异常退出。例如，由于不正确的参数或内部错误。在这种情况下，如果没有‘-return-child-result’，退出码将与原来相同。
  • 用户使用显式值退出。例如，‘quit 1’。
  • 子进程从未运行，或不被允许终止，在这种情况下，退出码将为 -1。

当GDB用作远程程序加载器或模拟器接口时，如果与‘-batch’或‘-batch-silent’一起使用，此选项很有用。

-nowindows ¶
-nw

“无窗口”。如果GDB内置了图形用户界面（GUI），那么此选项会告诉GDB仅使用命令行界面。如果没有可用的GUI，此选项无效。

-windows ¶
-w

如果GDB包括GUI，那么此选项要求尽可能使用它。

-cd directory ¶

使用 directory 作为其工作目录运行GDB，而不是当前目录。
