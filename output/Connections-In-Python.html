23.3.2.39 Python中的连接 ¶

GDB 允许你在单个会话中运行和调试多个程序。每个被调试的程序都有一个连接，该连接描述了 GDB 如何控制被调试的程序。不同连接类型的示例有 ‘native’ 和 ‘remote’。请参阅 调试多个下级连接和程序。

GDB 中的连接由 gdb.TargetConnection 的实例或其某个子类表示。要获取所有连接的列表，请使用 gdb.connections （请参阅 gdb.connections）。

要获取单个 gdb.Inferior 的连接，请读取其 gdb.Inferior.connection 属性 （请参阅 gdb.Inferior.connection）。

目前，gdb.TargetConnection 只有一个子类 gdb.RemoteTargetConnection， 但是，在 GDB 的未来版本中可能会添加其他子类。因此，你应该避免编写如下代码： 

conn = gdb.selected_inferior().connection
if type(conn) is gdb.RemoteTargetConnection:
  print("这是一个远程目标连接")

因为当添加更多连接类型时，这可能会失败。相反，你应该编写： 

conn = gdb.selected_inferior().connection
if isinstance(conn, gdb.RemoteTargetConnection):
  print("这是一个远程目标连接")

gdb.TargetConnection 有以下方法：

函数： TargetConnection.is_valid () ¶

如果 gdb.TargetConnection 对象有效，则返回 True， 否则返回 False。如果连接在 GDB 中不再存在，gdb.TargetConnection 将变为无效， 当没有下级使用该连接时可能会发生这种情况，但可能会延迟到用户替换当前目标。

如果连接无效，读取任何 gdb.TargetConnection 属性将引发异常。

gdb.TargetConnection 有以下只读属性：

变量： TargetConnection.num ¶

GDB 分配的一个整数，用于唯一标识此连接。这与 info connections 命令输出中的 ‘Num’ 列中显示的值相同（请参阅 info connections）。

变量： TargetConnection.type ¶

一个描述此连接类型的字符串。此字符串将是可以传递给 target 命令的有效名称之一（请参阅 target命令）。

变量： TargetConnection.description ¶

一个给出此目标类型简短描述的字符串。这与 info connection 命令输出中的 ‘Description’ 列中显示的字符串相同（请参阅 info connections）。

变量： TargetConnection.details ¶

一个可选字符串，给出有关此连接的附加信息。如果此连接没有其他详细信息，此属性可以为 None。

可能有附加详细信息的连接类型的一个示例是 ‘remote’ 连接，在这种情况下，详细信息字符串可以包含用于连接到远程目标的 ‘hostname:port’。

gdb.RemoteTargetConnection 类是 gdb.TargetConnection 的子类，用于表示 ‘remote’ 和 ‘extended-remote’ 连接。除了 gdb.TargetConnection 基类可用的属性和方法外， gdb.RemoteTargetConnection 还有以下方法：

函数： RemoteTargetConnection.send_packet (packet) ¶

此方法将 packet 发送到远程目标并返回响应。packet 应该是一个 bytes 对象或一个 Unicode 字符串。

如果 packet 是一个 Unicode 字符串，则使用 ASCII 编解码器将该字符串编码为一个 bytes 对象。如果字符串无法编码，则会引发 UnicodeError。

如果 packet 不是一个 bytes 对象或一个 Unicode 字符串，则会引发 TypeError。如果 packet 为空，则会引发 ValueError。

响应作为一个 bytes 对象返回。如果知道响应可以表示为一个字符串，则可以从缓冲区中解码。例如，如果知道响应是一个 ASCII 字符串： 

remote_connection.send_packet("some_packet").decode("ascii")

前缀、后缀和校验和（根据远程串行协议的要求）会自动添加到传出数据包中，并在返回回复内容之前从传入数据包中删除。

这等效于 maintenance packet 命令 （请参阅 maint packet）。
