23.3.2.5 Python中的类型 ¶

GDB 使用 gdb.Type 类来表示下级中的类型。

gdb 模块中提供了以下与类型相关的函数：

函数： gdb.lookup_type (name [, block]) ¶

此函数通过其 name（必须是字符串）查找类型。

如果给出了 block，则在该作用域中查找 name。 否则，将在全局范围内搜索。

通常，此函数将返回一个 gdb.Type 实例。 如果找不到指定的类型，它将引发异常。

无需按名称查找即可找到整数类型。 有关 integer_type 方法，请参阅 Python中架构的表示。

如果类型是结构体或类类型，或者是枚举类型，则可以使用Python的 字典语法 来访问该类型的字段。 例如，如果 some_type 是一个持有结构体类型的 gdb.Type 实例，则可以使用以下方式访问其 foo 字段： 

bar = some_type['foo']

bar 将是一个 gdb.Field 对象；有关 gdb.Field 类的描述，请参阅下面 Type.fields 方法的描述。

Type 的实例具有以下属性：

变量： Type.alignof ¶

此类型的对齐方式，以字节为单位。类型对齐来自调试信息；如果未指定，则 GDB 将使用相关的ABI来尝试确定对齐方式。在某些情况下，即使这样也不可能，将返回零。

变量： Type.code ¶

此类型的类型代码。类型代码将是下面定义的 TYPE_CODE_ 常量之一。

变量： Type.dynamic ¶

一个布尔值，指示此类型是否为动态类型。在某些情况下，例如Rust enum 类型或Ada变体记录，值的具体类型可能因其内容而异。也就是说，变量的声明类型或 gdb.lookup_type 返回的类型可能是动态的；而变量值的类型将是该动态类型的具体实例。

例如，考虑以下代码： 

int n;
int array[n];

在这里，至少在概念上（您的编译器是否实际这样做是另一个问题），检查 gdb.lookup_symbol("array", ...).type 可能会产生一个 gdb.Type，它报告的大小为 None。 这就是动态类型。

但是，检查 gdb.parse_and_eval("array").type 将产生一个具体类型，其长度是已知的。

变量： Type.name ¶

此类型的名称。如果此类型没有名称，则返回 None。

变量： Type.sizeof ¶

此类型的大小，以目标 char 单位表示。通常，目标的 char 类型将是一个八位字节。但是，在一些不寻常的平台上，此类型可能具有不同的大小。动态类型可能没有固定的大小；在这种情况下，此属性的值将为 None。

变量： Type.tag ¶

此类型的标签名称。标签名称是C和C++ 中 struct、union 或 enum 后面的名称；并非所有语言都有此概念。如果此类型没有标签名称，则返回 None。

变量： Type.objfile ¶

定义此类型的 gdb.Objfile，如果没有关联的目标文件，则为 None。

变量： Type.is_scalar ¶

如果类型是标量类型，则此属性为 True，否则为 False。非标量类型的示例包括结构体、联合体和类。

变量： Type.is_signed ¶

对于标量类型（即 Type.is_scalar 为 True 的类型），如果类型是有符号的，则此属性为 True，否则为 False。

尝试为非标量类型（即 Type.is_scalar 为 False 的类型）读取此属性将引发 ValueError。

变量： Type.is_array_like ¶

一个布尔值，指示此类型是否类似数组。

一些语言具有内部表示为结构体的类似数组的对象。例如，Rust切片类型或Ada无约束数组就是如此。GDB 可能知道这些类型。此判断是基于类型起源的语言进行的。

变量： Type.is_string_like ¶

一个布尔值，指示此类型是否类似字符串。与 Type.is_array_like 一样，这是基于类型起源的语言确定的。

提供了以下方法：

函数： Type.fields () ¶

返回此类型的字段。行为取决于类型代码：

  • 对于结构体和联合体类型，此方法返回字段。
  • 枚举类型每个枚举常量有一个字段。
  • 函数和方法类型每个参数有一个字段。C++ 类的基类型也表示为字段。
  • 数组类型有一个表示数组范围的字段。
  • 如果类型不属于这些类别之一，则会引发 TypeError。

每个字段都是一个 gdb.Field 对象，具有一些预定义的属性：

bitpos

此属性不适用于 enum 或 static（如在C++ 中）字段。值是从包含类型的开头开始以位为单位的位置。请注意，在动态类型中，字段的位置可能不是常量。在这种情况下，值将为 None。此外，动态类型可能有一些在相应具体类型中不出现的字段。

enumval

此属性仅适用于 enum 字段，其值是枚举成员的整数表示。

name

字段的名称，如果是匿名字段则为 None。

artificial

如果字段是人为的（通常意味着它是由编译器提供而不是用户提供的），则此属性为 True。此属性总是提供的，如果字段不是人为的，则为 False。

is_base_class

如果字段表示C++ 结构体的基类，则此属性为 True。此属性总是提供的，如果字段不是 fields 方法的参数类型的基类，或者该类型不是C++ 类，则为 False。

bitsize

如果字段是打包的或位域，则此属性将具有非零值，即字段的位大小。否则，此属性将为零；在这种情况下，字段的大小由其类型给出。

type

字段的类型。这通常是一个 Type 实例，但在某些情况下可能为 None。

parent_type

包含此字段的类型。这是一个 gdb.Type 实例。

函数： Type.array (n1 [, n2]) ¶

返回一个新的 gdb.Type 对象，该对象表示此类型的数组。如果给出一个参数，则它是数组的包含上限；在这种情况下，下限为零。如果给出两个参数，则第一个参数是数组的下限，第二个参数是数组的上限。数组的长度不能为负，但边界可以为负。

函数： Type.vector (n1 [, n2]) ¶

返回一个新的 gdb.Type 对象，该对象表示此类型的向量。如果给出一个参数，则它是向量的包含上限；在这种情况下，下限为零。如果给出两个参数，则第一个参数是向量的下限，第二个参数是向量的上限。向量的长度不能为负，但边界可以为负。

数组和向量的区别在于，数组的行为类似于C中的数组：在表达式中使用时，它们会衰减为指向第一个元素的指针，而向量被视为一等值。

函数： Type.const () ¶

返回一个新的 gdb.Type 对象，该对象表示此类型的 const 限定变体。

函数： Type.volatile () ¶

返回一个新的 gdb.Type 对象，该对象表示此类型的 volatile 限定变体。

函数： Type.unqualified () ¶

返回一个新的 gdb.Type 对象，该对象表示此类型的非限定变体。也就是说，表示既不是 const 也不是 volatile 的类型。

函数： Type.range () ¶

返回一个Python Tuple 对象，该对象包含两个元素：参数类型的下限和上限。如果类型没有范围，GDB 将引发 gdb.error 异常（请参阅 异常处理）。

函数： Type.reference () ¶

返回一个新的 gdb.Type 对象，该对象表示对此类型的引用。

函数： Type.pointer () ¶

返回一个新的 gdb.Type 对象，该对象表示指向此类型的指针。

函数： Type.strip_typedefs () ¶

返回一个新的 gdb.Type，该对象表示去除所有typedef层后的实际类型。

函数： Type.target () ¶

返回一个新的 gdb.Type 对象，该对象表示此类型的目标类型。

对于指针类型，目标类型是指向的对象的类型。对于数组类型（指C风格的数组），目标类型是数组元素的类型。对于函数或方法类型，目标类型是返回值的类型。对于复杂类型，目标类型是元素的类型。对于typedef，目标类型是别名类型。

如果类型没有目标，此方法将引发异常。

函数： Type.template_argument (n [, block]) ¶
