5.5.4线程特定断点 ¶

当你的程序有多个线程时（见 调试多线程程序），你可以选择是在所有线程上设置断点，还是在特定线程上设置断点。

break 位置规格 thread 线程ID ¶
break 位置规格 thread 线程ID if …

位置规格 指定程序中的一个或多个代码位置。 有关详细信息，请参阅 位置规格。

在断点命令中使用限定符 ‘thread 线程ID’ 来指定只有当特定线程到达此断点时才希望GDB停止程序。线程ID 说明符是GDB分配的线程标识符之一，显示在 ‘info threads’ 显示的第一列中。

如果你在设置断点时未指定 ‘thread 线程ID’，则该断点适用于程序的 所有 线程。

你也可以在条件断点上使用 thread 限定符；在这种情况下，将 ‘thread 线程ID’ 放在断点条件之前或之后，如下所示： 

(gdb) break frik.c:13 thread 28 if bartab > lim

当GDB检测到相应线程不再在线程列表中时，线程特定断点将自动删除。例如： 

(gdb) c
线程特定断点3已删除 - 线程28不再在线程列表中。

线程消失有几种方式，例如正常线程退出，但当你使用 detach 命令从进程分离时（见 调试正在运行的进程），或者如果GDB失去远程连接（见 调试远程程序）等。请注意，对于某些目标，只有当用户使用 info threads 命令明确请求线程列表时，GDB才能检测到线程已退出。

一个断点不能同时是线程特定的和进程特定的（见 进程特定断点），也不能是任务特定的（见 Ada任务扩展）；在创建断点时使用多个 thread、inferior 或 task 关键字将导致错误。
