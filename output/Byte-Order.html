19.3选择目标字节序 ¶

某些类型的处理器，如MIPS、PowerPC和瑞萨SH，能够以大端或小端字节序运行。通常，可执行文件或符号会包含一个位来指定字节序，你无需担心使用哪种。然而，手动调整GDB对处理器字节序的理解可能仍然有用。

set endian big ¶

指示GDB假设目标是大端字节序。

set endian little

指示GDB假设目标是小端字节序。

set endian auto

指示GDB使用与可执行文件相关联的字节序。

show endian

显示GDB当前对目标字节序的理解。

如果set endian auto模式有效且未选择可执行文件，则使用的字节序是上次由set endian big和set endian little命令之一选择的，或者是从上次使用的可执行文件推断出来的。如果之前没有选择字节序，则此模式的默认值是根据GDB所构建的目标推断出来的，如果目标CPU的名称有el后缀，则为little，否则为big。

请注意，这些命令仅调整主机上符号数据的解释，对目标系统绝对没有影响。
