23.4.3.8 Guile漂亮打印API ¶

提供了一个示例输出（见 漂亮打印）。

一个漂亮打印机由类型为<gdb:pretty-printer>的对象表示。漂亮打印机对象通过make-pretty-printer创建。

(gdb)模块提供了以下与漂亮打印机相关的过程：

Scheme过程： make-pretty-printer name lookup-function ¶

返回一个名为name的<gdb:pretty-printer>对象。

lookup-function是一个单参数函数：要打印的值。如果该值由这个漂亮打印机处理，那么lookup-function返回一个类型为<gdb:pretty-printer-worker>的对象来执行实际的漂亮打印。否则lookup-function返回#f。

Scheme过程： pretty-printer? object ¶

如果object是一个<gdb:pretty-printer>对象，则返回#t。否则返回#f。

Scheme过程： pretty-printer-enabled? pretty-printer ¶

如果pretty-printer已启用，则返回#t。否则返回#f。

Scheme过程： set-pretty-printer-enabled! pretty-printer flag ¶

将pretty-printer的启用标志设置为flag。返回值未指定。

Scheme过程： pretty-printers ¶

返回全局漂亮打印机列表。

Scheme过程： set-pretty-printers! pretty-printers ¶

将全局漂亮打印机列表设置为pretty-printers。返回值未指定。

Scheme过程： make-pretty-printer-worker display-hint to-string children ¶

返回一个类型为<gdb:pretty-printer-worker>的对象。

此函数接受三个参数：

‘display-hint’

display-hint通过MI向GDB或GDB前端提供一个提示，以更改正在打印的值的格式。该值必须是一个字符串或#f（表示没有提示）。 GDB预定义了几个display-hint的值：

‘array’

表示正在打印的对象是“类似数组的”。CLI使用此提示来遵守诸如set print elements和set print array等参数。

‘map’

表示正在打印的对象是“类似映射的”，并且可以假设此值的子项在键和值之间交替。

‘string’

表示正在打印的对象是“类似字符串的”。如果打印机 的to-string函数返回某种Guile字符串，那么GDB将调用其内部特定于语言的字符串打印函数来格式化该字符串。对于CLI，这意味着添加引号，可能转义一些字符，遵守set print elements等。

‘to-string’

to-string要么是一个单参数函数，即<gdb:pretty-printer-worker>对象，要么是#f。

从CLI打印时，如果to-string方法存在，那么GDB将把它的结果附加到children返回的值之前。这种格式化的确切方式取决于显示提示，并且可能会随着添加更多提示而改变。此外，根据打印设置（见 打印设置），CLI可能在堆栈跟踪中只打印to-string的结果，而省略children的结果。

If this method returns a string, it is printed verbatim.

否则，如果此方法返回一个<gdb:value>实例，那么GDB将打印此值。这可能会导致调用另一个漂亮打印机。

如果该方法返回一个可转换为<gdb:value>的Guile值，那么GDB将执行转换并打印结果值。同样，这可能会导致调用另一个漂亮打印机。Guile标量（整数、浮点数和布尔值）和字符串可转换为<gdb:value>；其他类型则不行。

最后，如果此方法返回#f，则在此方法中不执行进一步操作，也不打印任何内容。

如果结果不是这些类型之一，则会引发异常。

to-string也可以是#f，在这种情况下，将由children来打印该值。

‘children’

children要么是一个单参数函数，即<gdb:pretty-printer-worker>对象，要么是#f。

GDB将在漂亮打印机上调用此函数，以计算漂亮打印机值的子项。

此函数必须返回一个<gdb:iterator>对象。迭代器返回的每个项必须是一个包含两个元素的元组。第一个元素是子项的“名称”；第二个元素是子项的值。该值可以是任何可转换为GDB值的Guile对象。

If children is #f, GDB will act as though the value has no children.

根据“set print max-depth”的值（见 打印设置），子项可能会被隐藏不显示。

GDB提供了一个函数，可用于查找<gdb:value>的默认漂亮打印机：

Scheme过程： default-visualizer value ¶

此函数接受一个<gdb:value>对象作为参数。如果存在针对此值的漂亮打印机，则返回该打印机。如果不存在这样的打印机，则返回#f。
