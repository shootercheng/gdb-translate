8.3 选择一个帧 ¶

大多数用于检查程序中的栈和其他数据的命令，作用于当前选定的任何栈帧。以下是用于选择栈帧的命令；它们都会在最后打印出刚选定的栈帧的简要描述。

frame [ 帧选择规范 ] ¶
f [ 帧选择规范 ]

frame命令允许选择不同的栈帧。帧选择规范可以是以下任何一种：

编号 ¶
level 编号

选择帧级别编号。请记住，帧0是最内层（当前正在执行）的帧，帧1是调用最内层帧的帧，依此类推。最高级别的帧通常是main函数的帧。

由于这是在帧栈中导航的最常用方法，所以字符串level可以省略。例如，以下两个命令是等效的：

(gdb) frame 3
(gdb) frame level 3
address 栈地址 ¶

选择具有栈地址栈地址的帧。一个帧的栈地址可以在info frame的输出中看到，例如： 

(gdb) info frame
栈级别1，帧位于0x7fffffffda30：
 rip = 0x40066d在b (amd64-entry-value.cc:59)；保存的rip 0x4004c5
 尾调用帧，调用者为位于0x7fffffffda30的帧
 源语言c++。
 参数列表位于未知地址。
 局部变量位于未知地址，前一帧的sp为0x7fffffffda30

如该行所示，此帧的栈地址为0x7fffffffda30： 

栈级别1，帧位于0x7fffffffda30：
function 函数名 ¶

选择函数函数名对应的栈帧。如果函数函数名有多个栈帧，则选择最内层的栈帧。

view 栈地址 [ 程序计数器地址 ] ¶

查看一个不属于GDB回溯的帧。要查看的帧具有栈地址栈地址，并且可选地，有一个程序计数器地址程序计数器地址。

这主要在栈帧链接因错误而损坏，导致GDB无法正确为所有帧分配编号时有用。此外，当你的程序有多个栈并在它们之间切换时，这也可能有用。

当使用frame view查看当前回溯之外的帧时，你始终可以使用前面的栈帧选择指令之一返回原始栈，例如frame level 0。

up n ¶

在栈中向上移动n个帧；n默认为1。对于正数n，这会朝着最外层的帧前进，到更高的帧编号，到存在时间更长的帧。

down n ¶

在栈中向下移动n个帧；n默认为1。对于正数n，这会朝着最内层的帧前进，到更低的帧编号，到最近创建的帧。你可以将down缩写为do。

所有这些命令都会在最后打印两行描述该帧的输出。第一行显示帧编号、函数名、参数以及该帧中执行的源文件和行号。第二行显示该源行的文本。

例如： 

(gdb) up
#1  0x22f0 in main (argc = 1, argv = 0xf7fffbf4, env = 0xf7fffbfc)
    at env.c:10
10              read_input_file (argv[i]);

在这样的打印输出之后，不带参数的list命令会打印以该帧中的执行点为中心的十行。你也可以通过输入edit，使用你喜欢的编辑程序在执行点编辑程序。 有关详细信息，请参阅 打印源行。

select-frame [ 帧选择规范 ] ¶

select-frame命令是frame的变体，它在选择帧后不会显示新帧。此命令主要用于GDB命令脚本中，在那里输出可能是不必要的且会分散注意力。帧选择规范与在 选择一个帧 中描述的frame命令相同。

up-silently n ¶
down-silently n

这两个命令分别是up和down的变体；它们的不同之处在于它们默默地执行操作，不会导致新帧的显示。它们主要用于GDB命令脚本中，在那里输出可能是不必要的且会分散注意力。
