23.1.4用于控制输出的命令 ¶

在执行命令文件或用户定义的命令期间，GDB的正常输出会被抑制；唯一出现的输出是定义中的命令明确打印的内容。本节介绍三个有助于生成所需输出的命令。

echo 文本 ¶

打印文本。可以使用C转义序列在文本中包含非打印字符，例如使用’\n’打印换行符。除非指定，否则不会打印换行符。除了标准的C转义序列外，反斜杠后跟一个空格表示一个空格。这对于显示开头或结尾带有空格的字符串很有用，因为否则所有参数的前导和尾随空格都会被修剪掉。 要打印’ 和foo = ’，使用命令’echo \ 和foo = \ ’。

文本末尾的反斜杠可以像在C中一样用于将命令延续到后续行。例如， 

echo This is some text\n\
which is continued\n\
onto several lines.\n

产生的输出与 

echo This is some text\n
echo which is continued\n
echo onto several lines.\n
output 表达式 ¶

打印表达式的值，仅此值：不打印换行符，不打印’$nn = ’。该值也不会被记录到值历史中。有关表达式的更多信息，请参见 表达式。

output/格式 表达式

以格式打印表达式的值。可以使用与print相同的格式。有关更多信息，请参见 输出格式。

printf 模板, 表达式… ¶

在字符串模板的控制下打印一个或多个表达式的值。要打印多个值，使表达式为用逗号分隔的单个表达式列表，这些表达式可以是数字或指针。它们的值将按照模板指定的方式打印，就像C程序执行以下代码一样： 

printf (模板, 表达式...);

与C中的printf一样，模板中的普通字符将按原样打印，而由’%’字符引入的转换说明符会导致后续的表达式被求值，其值根据转换说明符中编码的类型和样式信息进行转换和格式化，然后打印。

例如，可以像这样以十六进制打印两个值： 

printf "foo, bar-foo = 0x%x, 0x%x\n", foo, bar-foo

printf支持所有标准的C转换说明符，包括在’%’字符和转换字母之间的标志和修饰符，但有以下例外：

  • 不支持参数排序修饰符，如’2$’。
  • 不支持用于指定精度或宽度的修饰符’*’。
  • 不支持’'’标志（用于根据LC_NUMERIC'将数字分组）。
  • 不支持类型修饰符’hh’、’j’、’t’和’z’。
  • 不支持转换字母’n’（如’%n’）。
  • 不支持转换字母’a’和’A’。

请注意，只有在用于构建GDB的底层C实现支持long long int类型时，才支持’ll’类型修饰符；只有在有long double类型时，才支持’L’类型修饰符。

与C一样，printf支持简单的反斜杠转义序列，如\n、’\t’、’\\’、’\"’、’\a’和’\f’——由反斜杠后跟一个字符组成。不支持八进制和十六进制转义序列。

此外，printf支持使用以下长度修饰符和浮点说明符对DFP（十进制浮点）类型的转换说明符。 字母：

  • ’H’用于打印Decimal32类型。
  • ’D’用于打印Decimal64类型。
  • ’DD’用于打印Decimal128类型。

如果用于构建GDB的底层C实现支持DFP类型的这三个长度修饰符，GDB也可以使用其他修饰符，如宽度和精度。
