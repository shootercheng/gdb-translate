23.2.1 默认参数 ¶

在使用用户定义的别名时，你可以告诉GDB始终在用户明确提供的参数列表前预先添加一些默认参数。

如果你反复为某个命令使用相同的参数或选项，你可以为该命令定义一个别名，并告诉GDB在使用该别名时自动将这些参数或选项预先添加到你明确输入的参数列表中17。

例如，如果您经常使用命令 thread apply all，并指定按升序处理线程，且在遇到错误时继续执行，您可以通过以下方式告诉GDB自动预先添加 -ascending 和 -c 选项： 

(gdb) alias thread apply asc-all = thread apply all -ascending -c

一旦你定义了带有默认参数的这个别名，每当你输入 thread apply asc-all 并后跟 一些参数 时，GDB将执行 thread apply all -ascending -c 一些参数。

为了减少输入量，你还可以定义一个单字别名： 

(gdb) alias t_a_c = thread apply all -ascending -c

与往常一样，对于 alias 和 default-args 可以使用明确的缩写。

一个命令的不同别名不共享它们的默认参数。 例如，你使用以下方式定义一个新的别名 bt_ALL 来显示所有可能的信息，以及另一个别名 bt_SMALL 来显示非常有限的信息： 

(gdb) alias bt_ALL = backtrace -entry-values both -frame-arg all \
   -past-main -past-entry -full
(gdb) alias bt_SMALL = backtrace -entry-values no -frame-arg none \
   -past-main off -past-entry off

（有关使用 alias 命令的更多信息，请参阅 命令别名。）

默认参数不限于 command 的参数和选项，但如果 command 接受这样的嵌套命令作为参数，则可以指定嵌套命令。 例如，下面定义了 faalocalsoftype，它列出具有特定类型局部变量的帧以及匹配的局部变量： 

(gdb) alias faalocalsoftype = frame apply all info locals -q -t
(gdb) faalocalsoftype int
#1  0x55554f5e in sleeper_or_burner (v=0xdf50) at sleepers.c:86
i = 0
ret = 21845

为一组嵌套的 with 命令定义一个别名以获得特定的临时设置组合也非常有用。例如，下面定义了别名 pp10，它以漂亮的格式打印表达式参数，如果表达式是字符串或数组，则最多显示10个元素： 

(gdb) alias pp10 = with print pretty -- with print elements 10 -- print

这将别名 pp10 定义为一个由3个命令组成的序列。第一部分 with print pretty -- 临时激活设置 set print pretty，然后启动分隔符 -- 后面的命令。 第一部分后面的命令也是一个 with 命令，它将设置 set print elements 临时更改为10，然后启动第二个分隔符 -- 后面的命令。 第三部分 print 是 pp10 别名将使用设置的临时值和用户明确给出的参数启动的命令。 有关 with 命令用法的更多信息，请参阅 命令设置。

默认情况下，请求别名的帮助会显示被别名化命令的文档。当别名是一组嵌套命令时，别名的 help 会显示第一个命令的文档。对于像 pp10 这样的别名，此帮助并不是特别有用。 对于这样的别名，使用 document 命令给出特定的文档是很有用的（请参阅 document）。

脚注

(17)

GDB 可以很容易地接受预定义命令和别名的默认参数，但认为这会造成混淆，因此不允许这样做。
