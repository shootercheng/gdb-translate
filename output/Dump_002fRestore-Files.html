10.19内存与文件之间的复制 ¶

你可以使用命令dump、append和restore在目标内存和文件之间复制数据。dump和append命令将数据写入文件，而restore命令将文件中的数据读回到下级进程的内存中。文件可以是二进制、摩托罗拉S记录、英特尔十六进制、泰克tronix十六进制或Verilog十六进制格式；但是，GDB只能追加到二进制文件，并且不能从Verilog十六进制文件读取。

dump [格式] memory 文件名 起始地址 结束地址 ¶
dump [格式] value 文件名 表达式

将从起始地址到结束地址的内存内容，或表达式的值，以给定格式转储到文件名。

格式参数可以是以下任何一种：

binary

原始二进制形式。

ihex

英特尔十六进制格式。

srec

摩托罗拉S记录格式。

tekhex

泰克tronix十六进制格式。

verilog

Verilog十六进制格式 (GDB不能从Verilog Hex文件读取数据，这里只做格式列举)。

GDB使用与GNU二进制实用程序（如‘objdump’和‘objcopy’）相同的这些格式定义。如果省略格式，GDB将以原始二进制形式转储数据。

append [binary] memory 文件名 起始地址 结束地址 ¶
append [binary] value 文件名 表达式

将从起始地址到结束地址的内存内容，或表达式的值，以原始二进制形式追加到文件文件名。(GDB只能将数据追加到原始二进制形式的文件。)

restore 文件名 [binary] 偏移量 起始位置 结束位置 ¶

将文件文件名的内容恢复到内存中。restore命令可以自动识别任何已知的BFD文件格式，但原始二进制格式除外。要恢复原始二进制文件，你必须在文件名后指定可选关键字binary。

如果偏移量不为零，其值将添加到文件中包含的地址。二进制文件总是从地址零开始，因此它们将在地址偏移量处恢复。其他bfd文件有一个内置位置；它们将从该位置偏移偏移量处恢复。

如果起始位置和/或结束位置不为零，那么只恢复文件偏移量起始位置和文件偏移量结束位置之间的数据。这些偏移量是相对于文件中的地址的，在应用偏移量参数之前。
