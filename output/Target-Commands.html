19.2管理目标的命令 ¶

target 类型 参数

将GDB主机环境连接到目标机器或进程。目标通常是用于与调试工具通信的协议。使用参数类型指定目标机器 的类型或协议。

目标协议会解释进一步的参数，但通常包括要连接的设备名称或主机名、进程号和波特率等内容。

执行命令后，如果再次按下RET，target命令不会重复。

help target ¶

显示所有可用目标的名称。要显示当前选择的目标，可使用info target或info files（见指定文件的命令）。

help target 名称

描述特定目标，包括选择它所需的任何参数。

set gnutarget 参数 ¶

GDB使用自己的库BFD来读取文件。GDB知道它正在读取的是可执行文件、核心文件还是.o文件；但是，可以使用set gnutarget命令指定文件格式。与大多数target命令不同，对于gnutarget，target指的是程序，而不是机器。

警告：要使用set gnutarget指定文件格式，必须知道实际的BFD名称。

见指定文件 的命令。

show gnutarget ¶

使用show gnutarget命令显示gnutarget设置为读取的文件格式。如果尚未设置gnutarget，GDB将自动确定每个文件的文件格式，show gnutarget显示“当前BFD目标是“auto””。

以下是一些常见目标（根据GDB配置是否可用）：

target exec 程序 ¶

一个可执行文件。“target exec 程序”与“exec-file 程序”相同。

程序参数支持转义字符和引号，见作为命令参数的文件名。

target core 文件名 ¶

一个核心转储文件。“target core 文件名”与“core-file 文件名”相同。

文件名参数支持转义字符和引号，见作为命令参数的文件名。

target remote 介质 ¶

通过串行线路或网络连接连接到GDB的远程系统。此命令告诉GDB通过介质使用其自己的远程协议进行调试。见调试远程程序。

例如，如果在运行GDB的机器上有一个连接到/dev/ttya的开发板，可以这样说：

target remote /dev/ttya

target remote支持load命令。只有在有其他方法将存根程序传输到目标系统，并且可以将其放置在内存中不会被下载覆盖的位置时，此命令才有用。

target sim [模拟参数] … ¶

内置CPU模拟器。GDB包括大多数架构的模拟器。一般来说，

        target sim
        load
        run

这样可以工作；但是，不能假定有特定的内存映射、设备驱动程序，甚至基本I/O可用，尽管有些模拟器提供了这些。有关任何特定处理器模拟器细节的信息，请参阅嵌入式处理器中的相应部分。

target native ¶

设置用于本地/原生进程调试。即使“set auto-connect-native-target”为“off”（见设置自动连接本地目标），使“run”命令生成原生进程（同样适用于“attach”等）也很有用。

不同配置的GDB有不同的可用目标；你的配置可能有更多或更少的目标。

许多远程目标要求在成功建立连接后下载可执行文件 的代码。你可能希望控制此过程的各个方面。

set hash ¶

此命令控制在将文件下载到远程监视器时是否显示哈希标记“#”。如果启用，在每个S记录成功下载到监视器后会显示一个哈希标记。

show hash ¶

显示显示哈希标记的当前状态。

set debug monitor ¶

启用或禁用显示GDB与远程监视器之间的通信消息

show debug monitor ¶

显示显示GDB与远程监视器之间通信的当前状态。

load 文件名 偏移量 ¶

根据GDB中配置的远程调试工具，“load”命令可能可用。如果存在，它旨在使文件名（一个可执行文件）在远程系统上可用于调试——例如通过下载或动态链接。“load”还会像“add-symbol-file”命令一样在GDB中记录文件名符号表。 如果你的GDB没有“load”命令，尝试执行它会得到错误消息“当你的目标是……时，你不能这样做”。 文件在可执行文件中指定的任何地址加载。对于某些目标文件格式，可以在链接程序时指定加载地址；对于其他格式，如a.out，目标文件格式指定一个固定地址。 也可以告诉GDB在由可选参数偏移量描述的特定偏移量处加载可执行文件。当提供偏移量时，也必须提供文件名。 根据远程端的功能GDB可能能够将程序加载到闪存中。 使用“load”后，如果再次按下RET，“load”命令不会重复。

flash-erase ¶

擦除目标上所有已知的闪存区域。
