8.4 关于一个帧的信息 ¶

还有其他几个命令用于打印有关选定栈帧的信息。

frame
f

当不带任何参数使用时，此命令不会更改选定的帧，但会打印当前选定栈帧的简要描述。它可以缩写为 f。带参数时，此命令用于选择一个栈帧。 请参阅 选择帧。

info frame ¶
info f

此命令打印选定栈帧的详细描述，包括：

  • 帧的地址
  • 向下的下一个帧（此帧调用的帧）的地址
  • 向上的下一个帧（此帧的调用者）的地址
  • 与此帧对应的源代码所使用的语言
  • 帧的参数的地址
  • 帧的局部变量的地址
  • 保存在其中的程序计数器（调用者帧中的执行地址）
  • 哪些寄存器保存在该帧中

当出现问题导致栈格式不符合通常的约定时，详细描述会很有用。

info frame [ frame-selection-spec ]
info f [ frame-selection-spec ]

打印由 frame-selection-spec 选择的帧的详细描述。frame-selection-spec 与 frame 命令的相同（请参阅 选择帧）。此命令不会更改选定的帧。

info args [-q] ¶

打印选定帧的参数，每个参数占一行。

可选标志 ‘-q’，代表 ‘quiet’，禁用打印标题信息和解释为何未打印任何参数的消息。

info args [-q] [-t type_regexp] [regexp]

与 info args 类似，但只打印用提供的正则表达式选择的参数。

如果提供了 regexp，则只打印名称与正则表达式 regexp 匹配的参数。

如果提供了 type_regexp，则只打印其类型（由 whatis 命令打印）与正则表达式 type_regexp 匹配的参数。 如果 type_regexp 包含空格，应将其括在引号中。如有需要，使用反斜杠转义特殊字符或引号的含义。

如果同时提供了 regexp 和 type_regexp，则仅当参数的名称与 regexp 匹配且其类型与 type_regexp 匹配时才打印该参数。

info locals [-q] ¶

打印选定帧的局部变量，每个变量占一行。这些是在选定帧的执行点可访问的所有变量（声明为静态或自动）。

可选标志 ‘-q’，代表 ‘quiet’，禁用打印标题信息和解释为何未打印任何局部变量的消息。

info locals [-q] [-t type_regexp] [regexp]

与 info locals 类似，但只打印用提供的正则表达式选择 的局部变量。

如果提供了 regexp，则只打印名称与正则表达式 regexp 匹配的局部变量。

如果提供了 type_regexp，则只打印其类型（由 whatis 命令打印）与正则表达式 type_regexp 匹配的局部变量。 如果 type_regexp 包含空格，应将其括在引号中。如有需要，使用反斜杠转义特殊字符或引号的含义。

如果同时提供了 regexp 和 type_regexp；则仅当局部变量的名称与 regexp 匹配且其类型与 type_regexp 匹配时才打印该局部变量。

命令 info locals -q -t type_regexp 可以有效地与命令 frame apply 和 thread apply 结合使用。 例如，你的程序可能使用资源获取即初始化类型（RAII），如 lock_something_t：每个类型为 lock_something_t 的局部变量会自动放置一个锁，当变量超出作用域时该锁会被销毁。然后你可以通过执行以下操作列出程序中所有获取的锁： 

thread apply all -s frame apply all -s info locals -q -t lock_something_t

或等效的简短形式 

tfaas i lo -q -t lock_something_t
