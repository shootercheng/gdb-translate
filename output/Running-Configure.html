C .2调用GDB的configure脚本 ¶

GDB附带了一个configure脚本，该脚本可自动执行准备GDB进行安装的过程；然后你可以使用make来构建gdb程序。

GDB发行版在单个目录中包含了GDB所需的所有源代码，该目录的名称通常是通过将版本号附加到“gdb”后面组成的。

例如，GDB 17.2版本的发行版位于gdb - 17.2目录中。该目录包含：

gdb - 17.2/configure (以及支持文件)

用于配置GDB及其所有支持库的脚本

gdb - 17.2/gdb

GDB本身特定的源代码

gdb - 17.2/bfd

二进制文件描述符库的源代码

gdb - 17.2/include

GNU包含文件

gdb - 17/2/libiberty

“-liberty”自由软件库的源代码

gdb - 17/src/opcodes

操作码表和反汇编器库的源代码

gdb - 17/src/readline

GNU命令行界面的源代码

可能还有其他子目录。

配置和构建GDB的最简单方法是从gdb - 版本号源目录运行configure，在这个例子中就是gdb - 17.2目录。

首先，如果您还没有进入gdb - 版本号源目录，请切换到该目录；然后运行configure。将GDB运行的平台标识符作为参数传递。

例如： 

cd gdb - 17.2
./configure
make

运行“configure”然后运行make会构建包含的支持库，然后构建gdb本身。配置好的源文件和二进制文件会留在相应的源目录中。

configure脚本是一个Bourne shell（/bin/sh）脚本；如果您的系统在运行不同的shell时不能自动识别它，您可能需要显式地在其上运行sh： 

sh configure

您应该从源树的顶级目录，即gdb - 版本号目录运行configure脚本。如果您从其中一个子目录运行configure，您将只配置该子目录。这通常不是您想要的。特别是，如果您从gdb - 版本号目录的gdb子目录运行第一个configure，您将忽略bfd、readline以及gdb子目录的其他同级目录的配置。这会导致关于缺少包含文件（如bfd/bfd.h）的构建错误。

您可以将GDB安装到任何位置。最好的方法是将“--prefix”选项传递给configure，然后使用make install进行安装。
