13.4 使用跟踪文件 ¶

在某些情况下，运行跟踪实验的目标可能不再可用；也许它崩溃了，或者硬件被用于其他活动。为了处理这些情况，你可以安排将跟踪数据转储到文件中，然后通过target tfile命令将该文件用作跟踪数据的源。

tsave [ -r ] 文件名 ¶
tsave [-ctf] 目录名

将跟踪数据保存到文件名。默认情况下，此命令假定文件名指的是主机文件系统，因此如果需要，GDB将从目标复制原始跟踪数据，然后保存它。如果目标支持，你还可以提供可选参数-r（“远程”），以指示目标将数据直接保存到其自己文件系统中的文件名中，如果跟踪缓冲区非常大，这可能更有效。（但是请注意，target tfile只能从主机可访问的文件中读取。）默认情况下，此命令将以tfile格式保存跟踪帧。你可以提供可选参数-ctf以CTF格式保存数据。通用跟踪格式（CTF）被提议作为一种可由多个调试和跟踪工具共享的跟踪格式。请访问‘http://www.efficios.com/ctf’获取更多信息。

target tfile 文件名 ¶
target ctf 目录名

使用名为文件名的文件或名为目录名的目录作为跟踪数据的源。检查数据的命令与处理实时目标时的工作方式相同，但无法运行任何新的跟踪实验。tstatus将报告保存数据时跟踪运行的状态，以及你正在检查的当前跟踪帧。文件名和目录名都必须位于主机可访问的文件系统上。

文件名和目录名参数支持转义和引用，见 作为命令参数的文件名。 

(gdb) target ctf ctf.ctf
(gdb) tfind
找到跟踪帧0，跟踪点2
39            ++a;  /* 在此处设置跟踪点1 */
(gdb) tdump
在跟踪点2，跟踪帧0收集的数据：i = 0
a = 0
b = 1 '\001'
c = {"123", "456", "789", "123", "456", "789"}
d = {{{a = 1, b = 2}, {a = 3, b = 4}}, {{a = 5, b = 6}, {a = 7, b = 8}}}
(gdb) p b
$1 = 1
