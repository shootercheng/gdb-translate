23.4.3.16从Guile访问块。 ¶

在GDB中，符号存储在块中。一个块大致对应于源代码中的一个作用域。块是分层组织的，在Guile中分别表示为类型为<gdb:block>的对象。块依赖于可用的调试信息。

一个帧有一个块。有关帧的更深入讨论，请参阅 从Guile访问下级堆栈帧。。

最外层的块称为全局块。全局块通常包含公共全局变量和函数。

嵌套在全局块内部的块是静态块。静态块通常包含文件作用域的变量和函数。

GDB提供了一种获取块的父块的方法，但目前没有办法检查块的子块，也无法遍历符号表中的所有块（请参阅 Guile中的符号表表示。）。

下面是一个简短的示例，有助于解释块： 

/* 这在全局块中。 */
int global;

/* 这在静态块中。 */
static int file_scope;

/* 'function' 在全局块中，'argument' 在 'function' 内部嵌套的块中。 */
int function (int argument)
{
  /* 'local' 在 'function' 内部的块中。它可能与 'argument' 在同一个块中，也可能不在。 */
  int local;

  {
     /* 'inner' 在一个块中，其父块是包含 'local' 的块。 */
     int inner;

     /* 如果此调用由编译器展开，你可能会在这里看到一个嵌套块，其函数是 'inline_function'，其父块是包含 'inner' 的块。 */
     inline_function ();
  }
}

(gdb) 模块提供了以下与块相关的过程：

Scheme过程： block? 对象 ¶

如果 对象 是一个 <gdb:block> 对象，则返回 #t。否则返回 #f。

Scheme过程： block-valid? 块 ¶

如果 <gdb:block> 块 有效，则返回 #t，否则返回 #f。如果它所引用的块在下级中不再存在，则块对象可能会变得无效。如果在调用过程时块无效，所有其他 <gdb:block> 方法将抛出异常。在遍历块的符号时也会检查块的有效性。

Scheme过程： block-start 块 ¶

返回 <gdb:block> 块 的起始地址。

Scheme过程： block-end 块 ¶

返回 <gdb:block> 块 的结束地址。

Scheme过程： block-function 块 ¶

返回表示为 <gdb:symbol> 对象的 <gdb:block> 块 的名称。如果块没有名称，则返回 #f。

对于普通函数块，父块是静态块。但是，你应该注意，函数块的父块可能不是静态块 —— 例如，内联函数就是这种情况。

Scheme过程： block-superblock 块 ¶

返回包含 <gdb:block> 块 的块。如果父块不存在，则返回 #f。

Scheme过程： block-global-block 块 ¶

返回与 <gdb:block> 块 相关联的全局块。

Scheme过程： block-static-block 块 ¶

返回与 <gdb:block> 块 相关联의静态块。

Scheme过程： block-global? 块 ¶

如果 <gdb:block> 块 是全局块，则返回 #t。否则返回 #f。

Scheme过程： block-static? 块 ¶

如果 <gdb:block> 块 是静态块，则返回 #t。否则返回 #f。

dt class="deffn" id="index-block_002dsymbols">Scheme过程： block-symbols ¶

返回 <gdb:block> 块 中的所有符号（作为 <gdb:symbol> 对象）的列表。

Scheme过程： make-block-symbols-iterator 块 ¶

返回一个类型为 <gdb:iterator> 的对象，该对象将遍历块的所有符号。Guile程序不应假设特定的块对象总是包含给定的符号，因为GDB功能和基础结构的更改可能会导致符号在符号表中的块之间移动。请参阅 Guile中的迭代器。

Scheme过程： block-symbols-progress? ¶

如果对象是 <gdb:block-symbols-progress> 对象，则返回 #t。此对象将从 make-block-symbols-iterator 返回的 <gdb:iterator> 对象의 progress 元素中获取。

Scheme过程： lookup-block pc ¶

返回包含给定 pc 值的最内层 <gdb:block>。如果找不到指定 pc 值的块，函数将返回 #f。
