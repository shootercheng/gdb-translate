22.8自动加载关联文件 ¶

GDB有时会自动读取包含命令和设置的文件，而无需用户明确告知。我们将此功能称为自动加载。虽然自动加载有助于GDB自动适应项目需求，但有时可能会产生意外结果或带来安全风险（例如，如果文件来自不受信任的来源）。

GDB可以自动加载各种类型的文件。除了这些文件，GDB还支持自动加载用各种扩展语言编写的代码。请参阅 自动加载扩展。

请注意，加载这些关联文件（包括本地的.gdbinit文件）需要相应配置的auto-load safe-path（请参阅 自动加载的安全限制）。

出于这些原因，GDB提供了命令和选项，让您可以控制何时自动加载文件以及哪些文件应该自动加载。

set auto-load off ¶

全局禁用所有自动加载文件的加载。 您可能希望将此命令与“-iex”选项一起使用（请参阅 选项-init-eval-command），例如： 

$ gdb -iex "set auto-load off" untrusted-executable corefile

请注意，系统初始化文件（请参阅 系统范围的配置和设置）以及您主目录中的初始化文件（请参阅 主目录初始化文件）仍会被读取（因为它们来自通常受信任的目录）。 要防止GDB自动加载这些初始化文件，除了使用set auto-load no之外，还可以使用-nx选项（请参阅 选择模式）。

show auto-load ¶

显示每个特定的“auto-load”文件的自动加载是否启用。 

(gdb) show auto-load
gdb-scripts: 自动加载命令脚本序列已启用。
libthread-db: 自动加载特定于进程的libthread_db已启用。
local-gdbinit: 从当前目录自动加载.gdbinit脚本已启用。
python-scripts: 自动加载Python脚本已启用。
safe-path: 可安全自动加载文件的目录列表为$debugdir:$datadir/auto-load。
scripts-directory: 用于加载自动加载脚本的目录列表为$debugdir:$datadir/auto-load。
info auto-load ¶

打印每个特定的“auto-load”文件是否已自动加载。 

(gdb) info auto-load
gdb-scripts:
已加载 脚本
是     /home/user/gdb/gdb-gdb.gdb  
libthread-db: 没有自动加载的libthread-db。
local-gdbinit: 本地.gdbinit文件“/home/user/gdb/.gdbinit”已加载。
python-scripts:
已加载 脚本
是     /home/user/gdb/gdb-gdb.py

这些是用于自动加载的GDB控制命令：

请参阅 set auto-load off。全局禁用自动加载。
请参阅 show auto-load。显示各种文件的设置。
