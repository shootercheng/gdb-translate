23.3.2.29从Python访问块 ¶

在GDB中，符号存储在块中。一个块大致对应于源代码中的一个作用域。块是分层组织的，在Python中分别表示为一个gdb.Block。块依赖于可用的调试信息。

一个帧有一个块。有关帧的更深入讨论，请参阅从Python访问下级堆栈帧。

最外层的块称为全局块。全局块通常包含公共全局变量和函数。

嵌套在全局块内部的块是静态块。静态块通常包含文件作用域的变量和函数。

GDB提供了一种获取块的父块的方法，但目前没有办法检查块的子块，也无法遍历符号表中的所有块（请参阅Python中的符号表表示）_。

下面是一个简短的示例，应该有助于解释块： 

/* 这在全局块中。 */
int global;

/* 这在静态块中。 */
static int file_scope;

/* 'function' 在全局块中，'argument' 在 'function' 内部嵌套的块中。 */
int function (int argument)
{
  /* 'local' 在 'function' 内部的块中。它可能与 'argument' 在同一个块中，也可能不在。 */ 
  int local;

  {
     /* 'inner' 在一个块中，其父块是包含 'local' 的块。 */
     int inner;

     /* 如果此调用被编译器展开，你可能会在这里看到一个嵌套块，其函数是 'inline_function'，其父块是包含 'inner' 的块。 */
     inline_function ();
  }
}

一个gdb.Block是可迭代的。迭代器返回该块本地的符号（请参阅符号的Python表示）。Python程序不应假设特定的块对象总是包含给定的符号，因为GDB功能和基础结构的变化可能会导致符号在符号表中的块之间移动。你还可以使用Python的字典语法来访问此块中的变量，例如： 

symbol = some_block['variable']  # symbol的类型是gdb.Symbol
