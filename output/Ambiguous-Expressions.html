10.2模糊表达式 ¶

表达式有时可能包含一些模糊的元素。例如，一些编程语言（特别是Ada、C++和Objective-C）允许一个函数名被多次定义，以应用于不同的上下文。这被称为重载。另一个涉及Ada的例子是泛型。一个泛型包类似于C++模板，通常会被多次实例化，导致相同的函数名在不同的上下文中被定义。

在某些情况下，根据语言的不同，可以调整表达式以消除模糊性。例如在C++中，你可以指定要中断的函数的签名，如break function(types)。在Ada中，使用函数的完全限定名通常也会使表达式变得明确。

当检测到需要解决的模糊性时，调试器能够为每个可能性显示一个编号选择的菜单，然后等待使用提示符“>”进行选择。第一个选项总是“[0]取消”，输入0 RET会中止当前命令。如果使用该表达式的命令允许选择多个选项，则菜单中的下一个选项是“[1]全部”，输入1 RET会选择所有可能的选项。

例如下面的会话摘录展示了尝试在重载符号String::after处设置断点的过程。我们选择了该函数名的三个特定定义：

set multiple-symbols mode ¶

此选项允许你在表达式模糊时调整调试器的行为。

默认情况下, mode 设置为 all。如果使用该表达式的命令允许选择多个选项，那么GDB会自动选择所有可能的选项。例如，在使用模糊名称的函数上插入断点会在每个可能的匹配处插入一个断点。但是，如果必须做出唯一的选择，那么GDB会使用菜单来帮助你消除表达式的模糊性。例如，打印重载函数的地址会导致使用菜单。

当 mode 设置为 ask 时，调试器在检测到模糊性时总是使用菜单。

最后，当 mode 设置为 cancel 时，调试器会由于模糊性报告一个错误，并且命令会被中止。

show multiple-symbols ¶

显示“multiple-symbols”设置的当前值。
