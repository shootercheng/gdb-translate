18.5索引文件加快GDB速度 ¶

当GDB找到一个符号文件时，它会扫描该文件中的符号以构建一个内部符号表。这使得大多数GDB操作能够快速运行 —— 但代价是一开始会有延迟。对于大型程序，这种延迟可能会相当长，所以GDB提供了一种构建索引(index)的方法，这可以加快启动速度。

为了方便起见，GDB附带了一个程序gdb-add-index，可用于将索引添加到符号文件中。它将符号文件作为唯一的参数： 

$ gdb-add-index symfile

请参阅 gdb-add-index。

也可以手动完成这项工作。下面是gdb-add-index在幕后所做的事情。

索引作为符号文件中的一个节存储。GDB可以将索引写入一个文件，然后你可以使用objcopy将其放入符号文件中。

要创建索引文件，使用save gdb-index命令：

save gdb-index [-dwarf-5] directory ¶

为GDB当前已知所有符号文件创建索引文件。对于每个已知的符号文件，此命令默认生成一个单一文件符号文件.gdb-index。如果你使用-dwarf-5选项调用此命令，它会生成2个文件：符号文件.debug_names和符号文件.debug_str。这些文件在给定的目录中创建。

目录参数支持转义字符和引号，见 作为命令参数的文件名。

一旦你创建了一个索引文件，你可以使用objcopy将其合并到你的符号文件中，这里假设符号文件名为symfile： 

$ objcopy --add-section .gdb_index=symfile.gdb-index \
    --set-section-flags .gdb_index=readonly symfile symfile

或者对于-dwarf-5： 

$ objcopy --dump-section .debug_str=symfile.debug_str.new symfile
$ cat symfile.debug_str >>symfile.debug_str.new
$ objcopy --add-section .debug_names=symfile.gdb-index \
    --set-section-flags .debug_names=readonly \
    --update-section .debug_str=symfile.debug_str.new symfile symfile

GDB通常会忽略已弃用的旧版本的.gdb_index节。通常它们被弃用是因为缺少新功能或存在性能问题。 要告诉GDB无论如何都要使用已弃用的索引节，请指定set use-deprecated-index-sections on。默认值是off。 这可以加快启动速度，但可能会导致一些功能丢失。 请参阅 '.gdb_index'节格式。

警告：必须在gdb读取文件之前将use-deprecated-index-sections设置为on。以下操作将不起作用： 

$ gdb -ex "set use-deprecated-index-sections on" <program>

相反，你必须这样做，例如： 

$ gdb -iex "set use-deprecated-index-sections on" <program>

索引仅在使用DWARF调试信息时有效，不适用于stabs。

18.5.1自动符号索引缓存 ¶

GDB可以自动将此索引的副本保存在磁盘上的缓存中，并在将来加载相同二进制文件时从那里检索它。可以使用set index-cache enabled on启用此功能。 以下命令可用于调整索引缓存的行为。

set index-cache enabled on ¶
set index-cache enabled off

启用或禁用符号索引缓存的使用。

set index-cache directory directory
show index-cache directory

设置/显示将保存索引文件的目录。

此目录的默认值取决于主机平台。在大多数系统上，如果定义了环境变量XDG_CACHE_HOME，则索引缓存在其指向目录的gdb子目录中，否则在你的主目录的.cache/gdb子目录中缓存。但是，在某些系统上，默认值可能因本地约定而异[1]。

索引缓存所使用的磁盘空间没有限制。删除该目录的内容以释放磁盘空间是完全安全的[2]。

show index-cache stats

打印自GDB启动以来的缓存命中和未命中次数。

18.5.2使用GNU gold构建索引 ¶

GNU gold链接器可以在链接时通过将--gdb-index标志传递给gold，将索引写入生成的ELF文件中。

如果你打算构建一个程序并立即在GDB下运行它，这特别有用。你可能会发现，对于链接器来说，在它仍在内存中拥有相关信息时写入索引，总体上比使用gdb-add-index从磁盘重新加载数据，或者根本不使用索引调试程序要快，特别是对于大型程序来说。
