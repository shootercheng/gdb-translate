22.3 命令历史记录 ¶

GDB 可以跟踪你在调试会话期间输入的命令，这样你就可以确切知道发生了什么。使用这些命令来管理 GDB 命令历史记录功能。

GDB 使用 GNU 历史库（Readline包的一部分）来提供历史记录功能。 有关历史库的详细描述，请参阅 交互式使用历史记录。

要向 GDB 发出命令而不影响用户看到的某些状态方面，在命令前加上 ‘server ’（请参阅 服务器前缀）。 这意味着此命令不会影响命令历史记录，也不会影响在一行上单独按下 RET 时 GDB 重复哪个命令的概念。

服务器前缀不会影响将值记录到值历史记录中；要打印一个值而不将其记录到值历史记录中，请使用 output 命令而不是 print 命令。

以下是与命令历史记录相关的 GDB 命令的描述。

set history filename [fname] ¶

将 GDB 命令历史记录文件的名称设置为 fname。 这是 GDB 读取初始命令历史记录列表的文件，也是它在退出时写入本次会话命令历史记录的文件。你可以通过历史替换或通过下面列出的命令历史编辑字符来访问此列表。此文件默认为环境变量 GDBHISTFILE 的值，如果未设置此变量，则默认为 ./.gdb_history（MS-DOS 上为 ./_gdb_history）。

GDBHISTFILE 环境变量在处理任何 GDB 初始化文件（请参阅 GDB启动时的操作）以及处理使用命令行选项（例如 -ex）传递的任何命令之后读取。

如果未给出 fname 参数，或者 GDBHISTFILE 为空字符串，则 GDB 既不会尝试加载现有的历史记录文件，也不会在退出时尝试保存历史记录。

set history save ¶
set history save on

将命令历史记录记录到一个文件中，该文件的名称可以使用 set history filename 命令指定。默认情况下，此选项处于禁用状态。命令历史记录将在 GDB 退出时记录下来。如果 set history filename 设置为空字符串，则即使 set history save 设置为 on，历史记录保存也会被禁用。

set history save off

当 GDB 退出时不要将命令历史记录记录到由 set history filename 指定的文件中。

set history size size ¶
set history size unlimited

设置 GDB 在其历史记录列表中保留的命令数量。 这默认为环境变量 GDBHISTSIZE 的值，如果未设置此变量，则默认为256。GDBHISTSIZE 的非数值将被忽略。如果 size 为 unlimited，或者 GDBHISTSIZE 为负数或空字符串，则 GDB 在历史记录列表中保留的命令数量是无限的。

GDBHISTSIZE 环境变量在处理任何 GDB 初始化文件（请参阅 GDB启动时的操作）以及处理使用命令行选项（例如 -ex）传递的任何命令之后读取。

set history remove-duplicates count ¶
set history remove-duplicates unlimited

控制在命令历史记录列表中移除重复历史记录项。 如果 count 不为零，GDB 将查看最后 count 条历史记录项，并移除与当前要添加到命令历史记录列表中的项重复的第一个项。如果 count 为 unlimited，则此回溯是无界的。如果 count 为0，则禁用移除重复历史记录项。

仅考虑在当前会话期间添加的历史记录项进行移除。此选项默认设置为0。

历史替换为字符 ! 指定了特殊含义。 有关更多详细信息，请参阅 事件指示符。

由于 ! 也是C语言中的逻辑非运算符，历史替换默认处于关闭状态。如果你决定使用 set history expansion on 命令启用历史替换，有时你可能需要在 !（当它在表达式中用作逻辑非时）后面跟一个空格或制表符，以防止它被替换。即使启用了历史替换，readline历史记录功能也不会对字符串 != 和 !( 进行替换尝试。

控制历史替换开关的命令如下：

set history expansion on ¶
set history expansion

启用历史替换。历史替换默认处于关闭状态。

set history expansion off

禁用历史替换。

show history ¶
show history filename
show history save
show history size
show history expansion

这些命令显示 GDB 历史记录参数的状态。 单独使用 show history 会显示所有四个状态。

show commands ¶

显示命令历史记录中的最后十条命令。

show commands n

打印以命令编号 n 为中心的十条命令。

show commands +

在最后打印的命令之后打印十条命令。
