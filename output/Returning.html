17.4 从函数返回 ¶

return ¶
return 表达式

你可以使用return命令取消函数调用的执行。如果你给出一个 表达式参数，其值将被用作函数的返回值。

当你使用return时，GDB会丢弃选定的栈帧（以及其中的所有帧）。你可以认为这是使被丢弃的帧过早返回。如果你希望指定一个要返回的值，将该值作为return的参数给出。

这会弹出选定的栈帧（见 选择一个帧），以及其中的任何其他帧，使调用者成为剩余的最内层帧。该帧将被选中。指定的值将被存储在用于函数返回值的寄存器中。

return命令不会恢复执行；它会使程序停留在函数刚刚返回时应有的状态。相比之下，finish命令（见 继续和单步执行）会恢复执行直到选定的栈帧自然返回。

GDB需要知道如何为下级设置表达式参数。具体的寄存器分配取决于操作系统应用二进制接口（OS ABI）以及选定栈帧返回的类型。例如，对于OS ABI来说，在FPU寄存器中返回浮点值而在CPU寄存器中返回整数值是很常见的。不过，有些ABI甚至在CPU寄存器中返回浮点值。更大的整数宽度（如 long long int）也有特定的放置规则。GDB已经从其当前目标中知道了OS ABI，所以它还需要找出返回的类型，以便将值分配到正确的寄存器中。

通常，选定的栈帧有调试信息。当有调试信息时，GDB总是会使用调试信息而不是表达式的隐式类型。例如，如果您输入 return -1，并且当前栈帧中的函数被声明返回一个 long long int，GDB会将隐式的 int 值 -1 透明地转换为 long long int： 

断点1，func () 位于gdb.base/return-nodebug.c:29
29        return 31;
(gdb) return -1
现在让func返回？(y或n) y
#0  0x004004f6 in main () 位于gdb.base/return-nodebug.c:43
43        printf ("result=%lld\n", func ());
(gdb)

然而，如果选定的栈帧没有调试信息，例如，如果函数是在没有调试信息的情况下编译的，GDB必须从用户那里找出要返回的类型。错误地指定不同的类型可能会在下级寄存器中设置与调用者代码预期不同的值。例如，在没有调试信息的函数（在32位架构上）中输入 return -1 及其隐式类型 int 只会设置 long long int 结果的一部分。因此，要求用户通过适当的强制类型转换显式指定返回类型： 

断点2，位于func () 的0x0040050b处
(gdb) return -1
选定栈帧的返回值类型不可用。
请使用要返回值的显式强制类型转换。
(gdb) return (long long int) -1
现在让选定的栈帧返回？(y或n) y
#0  0x00400526 in main ()
(gdb)
