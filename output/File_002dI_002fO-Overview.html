E.14.1文件I/O概述 ¶

文件I/O远程协议扩展（简称：文件I/O）允许目标使用主机的文件系统和控制台I/O来执行各种系统调用。目标系统上的系统调用会被转换为发往主机系统的远程协议数据包，然后主机系统执行所需的操作并向目标系统返回响应数据包。这甚至能在缺乏文件系统的目标上模拟文件系统操作。

该协议被定义为独立于主机和目标系统。它使用自己内部的数据类型和值表示。在传输数据时，GDB和目标的GDB存根都负责将与系统相关的值表示转换为内部协议表示。

通信是同步的。只有当GDB正在等待来自‘C’、‘c’、‘S’或‘s’数据包的响应时，才可能进行系统调用。当GDB处理系统调用请求时，目标会停止，以允许对目标内存进行确定性访问。因此，文件I/O不会被目标信号中断。另一方面，可以在GDB内部通过用户中断（‘Ctrl-C’）中断文件I/O。

目标执行主机系统调用的请求不会结束最新的‘C’、‘c’、‘ S’或‘s’操作。这意味着，在完成系统调用后，目标会返回继续之前的活动（继续、单步执行）。不需要GDB发出额外的继续或单步执行请求。 

(gdb) continue
  <- 目标请求‘系统调用X’
  目标停止，GDB执行系统调用
  -> GDB返回结果
  ... 目标继续，GDB返回等待目标
  <- 目标命中断点并发送一个Txx数据包

该协议仅支持控制台I/O以及主机文件系统上的常规文件。不支持主机系统上的字符或块特殊设备、管道、命名管道、套接字或任何其他通信方法。

非停止模式下不支持文件I/O。
