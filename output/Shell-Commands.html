2.3 Shell命令 ¶

如果在调试会话期间需要偶尔执行一些shell命令，无需离开或暂停 GDB；可以直接使用 shell 命令。

shell 命令字符串 ¶
!命令字符串

调用一个shell来执行 命令字符串。 请注意，在 ! 和 命令字符串 之间不需要空格。 在GNU和Unix系统上，如果环境变量 SHELL 存在，则它决定运行哪个shell。否则，GDB 使用默认的shell（在GNU和Unix系统上是 /bin/sh，在MS-Windows上是 cmd.exe，在MS-DOS上是 COMMAND.COM 等）。

也可以使用 $_shell 便利函数从表达式中调用shell命令。请参阅 $_shell便利函数。

在开发环境中通常需要使用 make 实用程序。在 GDB 中为此目的不必使用 shell 命令：

make make参数 ¶

使用指定的参数执行 make 程序。这等同于 ‘shell make make参数’。

pipe [命令] | shell命令
| [命令] | shell命令
pipe -d 分隔符 命令 分隔符 shell命令
| -d 分隔符 命令 分隔符 shell命令

执行 命令 并将其输出发送到 shell命令。 请注意，在 | 周围不需要空格。 如果未提供 命令，则重复执行最后执行的命令。

如果 命令 中包含 |，可以使用选项 -d 分隔符 指定一个替代的分隔符字符串 分隔符，用于分隔 命令 和 shell命令。

示例： 

(gdb) p var
$1 = {
  black = 144,
  red = 233,
  green = 377,
  blue = 610,
  white = 987
}

(gdb) pipe p var|wc
      7      19      80
(gdb) |p var|wc -l
7

(gdb) p /x var
$4 = {
  black = 0x90,
  red = 0xe9,
  green =
