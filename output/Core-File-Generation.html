10.20如何从你的程序生成核心文件 ¶

一个核心文件或核心转储是一个记录正在运行的进程的内存映像及其进程状态（寄存器值等）的文件。它的主要用途是对在调试器外部运行时崩溃的程序进行事后调试。一个崩溃的程序会自动生成一个核心文件，除非该功能被用户禁用。有关在事后调试模式下调用GDB的信息，请参阅 指定文件的命令。

偶尔，你可能希望生成正在调试的程序的核心文件，以便保存其状态的快照。GDB有一个专门的命令用于此目的。

generate-core-file [文件] ¶
gcore [文件]

生成子进程的核心转储。可选参数文件指定放置核心转储的文件名。如果未指定，文件名默认为core.pid，其中pid是子进程ID。

如果写入核心文件的文件系统支持，GDB会生成一个稀疏核心转储文件。

请注意，此命令仅在某些系统上实现（截至本文撰写时，GNU/Linux、FreeBSD、Solaris和S390）。

在GNU/Linux上，此命令在生成核心转储时可以考虑文件/proc/pid/coredump_filter的值（请参阅 设置使用核心转储过滤器），并且默认情况下会尊重文件 /proc/pid/smaps 中存在的映射的 VM_DONTDUMP 标志（请参阅 设置转储排除的映射）。

设置使用核心转储过滤器开启
设置使用核心转储过滤器关闭

在生成核心转储文件时启用或禁用对文件/proc/pid/coredump_filter的使用。Linux内核使用此文件来决定在生成核心转储文件时哪些类型的内存映射将被转储或忽略。pid是当前正在运行的进程的进程ID。

要使用此功能，你必须在/proc/pid/coredump_filter文件中写入一个十六进制值，该值是一个表示内存映射类型的位掩码。如果位掩码中的一位被设置，则相应类型的内存映射将被转储；否则，它们将被忽略。此配置由子进程继承。有关可以在/proc/pid/coredump_filter文件中设置的位的更多信息，请参考 core(5) 的手册页。

默认情况下，此选项为开启。如果此选项被设置为关闭，GDB不会读取coredump_filter文件，而是使用与Linux内核相同的默认值来决定哪些页面将被转储到核心转储文件中。此值当前为0x33，这意味着位0（匿名私有映射）、1（匿名共享映射）、4（ELF头）和5（私有大页面）是活动的。这将导致这些内存映射被自动转储。

设置转储排除的映射开启
设置转储排除的映射关闭

如果指定为开启，GDB将转储标记有VM_DONTDUMP标志的内存映射。此标志在文件/proc/pid/smaps中表示为缩写dd。

默认值为关闭。
