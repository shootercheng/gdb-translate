E.16 SVR4目标的库列表格式 ¶

在SVR4平台上，GDB可以使用动态加载器（例如ld.so）的符号表和常规内存操作来维护共享库列表。不过，此数据包提供的特殊库列表对于GDB远程协议而言效率更高。

‘qXfer:libraries-svr4:read’数据包返回一个XML文档，该文档列出已加载的库及其SVR4链接器参数。对于SVR4目标上的每个库，会报告以下参数：

  • name，来自struct link_map的l_name字段的绝对文件名。
  • lm，用于TLS（线程局部存储）访问的struct link_map的地址。
  • l_addr，从struct link_map的l_addr字段读取的偏移量。对于预链接库，这不是绝对内存地址。它是相对于库加载期间文件被预链接到的地址的绝对内存地址的偏移量。
  • l_ld，即PT_DYNAMIC段的内存地址。
  • lmid，它是链接器命名空间的标识符，例如包含此命名空间加载映射的r_debug对象的内存地址或由dlinfo(3)返回的命名空间标识符。

此外，单个main-lm属性指定用于主可执行文件的struct link_map的地址。此参数用于TLS访问，其存在是可选的。

GDB必须与Expat库链接以支持XML SVR4库列表。请参阅 Expat。

一个简单的内存映射，有两个已加载的库（不使用预链接），如下所示： 

<library-list-svr4 version="1.0" main-lm="0xe4f8f8">
  <library name="/lib/ld-linux.so.2" lm="0xe4f51c" l_addr="0xe2d000"
           l_ld="0xe4eefc" lmid="0xfffe0"/>
  <library name="/lib/libc.so.6" lm="0xe4fbe8" l_addr="0x154000"
           l_ld="0x152350" lmid="0xfffe0"/>
</library-list-svr>

SVR4库列表的格式由以下DTD描述： 

<!-- library-list-svr: 带有版本控制的根元素 -->
<!ELEMENT library-list-svr4  (library)*>
<!ATTLIST library-list-svr4  version CDATA   #FIXED  "1.0">
<!ATTLIST library-list-svr4  main-lm CDATA   #IMPLIED>
<!ELEMENT library            EMPTY>
<!ATTLIST library            name    CDATA   #REQUIRED>
<!ATTLIST library            lm      CDATA   #REQUIRED>
<!ATTLIST library            l_addr  CDATA   #REQUIRED>
<!ATTLIST library            l_ld    CDATA   #REQUIRED>
<!ATTLIST library            lmid    CDATA   #IMPLIED>
