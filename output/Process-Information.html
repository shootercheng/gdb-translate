21.1.2进程信息 ¶

一些操作系统提供接口，用于获取有关正在运行的进程的额外信息，这些信息超出了内存和每个线程的寄存器状态。如果为具有受支持接口的操作系统配置了GDB，则可以使用info proc命令来报告有关运行您的程序的进程或系统上正在运行的任何进程的信息

一个受支持的接口是一个名为“/proc”的工具，可用于使用文件系统子例程检查正在运行的进程的映像。GNU/Linux和Solaris系统支持此工具。

在FreeBSD和NetBSD系统上，使用系统控制节点来查询进程信息。

此外，一些系统可能会在核心文件中提供额外的进程信息。请注意，核心文件可能只包含活动进程可用信息的一个子集。目前，GNU/Linux和FreeBSD系统上创建的核心文件中可以获取进程信息。

info proc ¶
info proc 进程ID

总结有关进程的可用信息。如果进程ID指定了一个进程ID，则显示有关该进程的信息；否则，显示有关正在调试程序的信息。总结内容包括调试的进程ID、用于调用它的命令行、其当前工作目录以及其可执行文件的绝对文件名。

在某些系统上，进程ID的形式可以是“[pid]/tid”，它指定了进程中的某个线程ID。如果缺少可选的pid部分，则表示来自正在调试的进程的一个线程（仍然需要有前导的“/”，否则GDB将把这个数字解释为进程ID而不是线程ID）。

info proc cmdline ¶

显示进程的原始命令行。此命令在GNU/Linux、FreeBSD和NetBSD上受支持。

info proc cwd ¶

显示进程的当前工作目录。此命令在GNU/Linux、FreeBSD和NetBSD上受支持。

info proc exe ¶

显示进程的可执行文件名。此命令在GNU/Linux、FreeBSD和NetBSD上受支持。

info proc files ¶

显示进程打开的文件描述符。对于每个打开的文件描述符，GDB会显示其编号、类型（文件、目录、字符设备、套接字）、文件指针偏移量以及描述符上打开的资源名称。资源名称可以是文件名（对于文件、目录和设备），也可以是套接字地址后面跟的协议（对于网络连接）。此命令在FreeBSD上受支持。

这个例子展示了一个使用tty作为标准输入和输出以及两个网络套接字的进程的打开文件描述符： 

(gdb) info proc files 22136
进程22136
打开的文件：

      FD   类型     偏移   标志   名称
    text   文件          - r-------- /usr/bin/ssh
    ctty    字符设备       - rw------- /dev/pts/20
     cwd    目录          - r-------- /usr/home/john
    root    目录          - r-------- /
       0    字符设备  0x32933a4 rw------- /dev/pts/20
       1    字符设备  0x32933a4 rw------- /dev/pts/20
       2    字符设备   0x32933a4 rw------- /dev/pts/20
       3  套接字        0x0 rw----n-- tcp4 10.0.1.2:53014 -> 10.0.1.10:22
       4  套接字        0x0 rw-------  Unix流:/tmp/ssh-FIt89oAzOn5f/agent. 2456
info proc mappings ¶

报告进程中可访问的内存地址空间范围。在Solaris、FreeBSD和NetBSD系统上，每个内存范围包括进程对每个范围是否具有读、写或执行访问权限的信息。在GNU/Linux、FreeBSD和NetBSD系统上，每个内存范围包括映射到该范围的目标文件。

info proc stat ¶
info proc status

显示与进程相关的其他信息，包括用户ID和组ID；虚拟内存使用情况；挂起、阻塞和忽略的信号；其TTY；其系统和用户时间消耗；其堆栈大小；其“nice”值等。这些命令在GNU/Linux、FreeBSD和NetBSD上受支持。

对于GNU/Linux系统，请参阅“proc”手册页以获取更多信息（从你的 shell 提示符输入 man 5 proc）。

对于FreeBSD和NetBSD系统，info proc stat是info proc status 的别名。

info proc all

显示上述所有info proc子命令所描述的进程的所有信息。

set procfs-trace ¶

此命令启用和禁用对procfs API调用的跟踪。

show procfs-trace ¶

显示procfs API调用跟踪的当前状态。

set procfs-file 文件 ¶

告诉GDB将procfs API跟踪写入指定的文件。GDB将跟踪信息追加到文件的先前内容之后。默认是在标准输出上显示跟踪信息。

show procfs-file ¶

显示写入procfs API跟踪的文件。

proc-trace-entry ¶
proc-trace-exit
proc-untrace-entry
proc-untrace-exit

这些命令启用和禁用对进入和离开syscall接口的跟踪。
