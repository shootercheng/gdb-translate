22.4屏幕大小 ¶

某些GDB命令可能会在屏幕上输出大量信息。为了帮助你阅读所有这些信息，GDB会在每一页输出结束时暂停并要求你输入。当你想再看一页输出时，按RET，按q丢弃剩余输出，或按c继续而不进行分页显示当前命令的剩余部分。此外，屏幕宽度设置决定了何时换行输出。根据输出内容，GDB会尝试在可读的位置换行，而不是简单地让它溢出到下一行。

通常GDB从终端驱动软件中知道屏幕的大小。例如，在Unix上，GDB使用termcap数据库以及TERM环境变量的值和stty rows与stty cols设置。如果这不正确，你可以使用set height和set width命令覆盖它：

set height lpp ¶
set height unlimited
show height
set width cpl
set width unlimited
show width

这些set命令指定屏幕高度为lpp行，屏幕宽度为cpl个字符。相关的show命令显示当前设置。

如果你指定高度为unlimited或零行，无论输出多长，GDB在输出期间都不会暂停。如果将输出发送到文件或编辑器缓冲区，这很有用。

同样，你可以指定‘set width unlimited’或‘set width 0’以防止GDB换行输出。

set pagination on ¶
set pagination off

打开或关闭输出分页；默认是打开。关闭分页是set height unlimited的替代方法。请注意，使用--batch选项运行GDB（见 -batch）也会自动禁用分页。

show pagination ¶

显示当前的分页模式。
