14.2 Overlay Commands ¶

要使用GDB的覆盖支持，程序中的每个覆盖必须对应可执行文件的一个单独部分。该部分的虚拟内存地址和加载内存地址必须是覆盖的映射地址和加载地址。通过用部分来标识覆盖，GDB可以根据覆盖是否被映射来确定函数或变量的适当地址。

GDB中的覆盖命令都以单词overlay开头；你可以将其缩写为ov或ovly。这些命令如下：

overlay off ¶

禁用GDB的覆盖支持。当覆盖支持被禁用时，GDB假定所有函数和变量始终存在于它们的映射地址处。默认情况下，GDB的覆盖支持是禁用的。

overlay manual ¶

启用手动覆盖调试。在这种模式下，GDB依赖于你使用下面描述的overlay map-overlay和overlay unmap-overlay命令来告诉它哪些覆盖被映射，哪些没有被映射。

overlay map-overlay overlay ¶
overlay map overlay

告诉GDB overlay现在已被映射；overlay必须是包含该覆盖的目标文件部分的名称。当一个覆盖被映射时，GDB假定它可以在其映射地址处找到该覆盖的函数和变量。GDB假定任何其他其映射范围与overlay重叠的覆盖现在未被映射。

overlay unmap-overlay overlay ¶
overlay unmap overlay

告诉GDB overlay不再被映射；overlay必须是包含该覆盖的目标文件部分的名称。当一个覆盖未被映射时，GDB假定它可以在其加载地址处找到该覆盖的函数和变量。

overlay auto

启用自动覆盖调试。在这种模式下，GDB会参考覆盖管理器在下位程序中维护的数据结构，以查看哪些覆盖被映射。有关详细信息，请参阅Automatic Overlay Debugging。

overlay load-target ¶
overlay load

从下位程序重新读取覆盖表。通常，每次下位程序停止时，GDB会自动重新读取该表，所以只有当你自己使用GDB更改了覆盖映射时，才需要这个命令。这个命令仅在使用自动覆盖调试时有用。

overlay list-overlays ¶
overlay list

显示当前已映射的覆盖列表，以及它们的映射地址、加载地址和大小。

通常，当GDB打印一个代码地址时，它会包含该地址所在函数的名称： 

(gdb) print main
$3 = {int ()} 0x11a0 <main>

当启用覆盖调试时，GDB会识别未映射覆盖中的代码，并在未映射函数的名称周围加上星号进行打印。例如，如果foo是未映射覆盖中的一个函数，GDB会这样打印： 

(gdb) overlay list
No sections are mapped.
(gdb) print foo
$5 = {int (int)} 0x100000 <*foo*>

当foo的覆盖被映射时，GDB会正常打印函数的名称： 

(gdb) overlay list
Section .ov.foo.text, loaded at 0x100000 - 0x100034,
        mapped at 0x1016 - 0x104a
(gdb) print foo
$6 = {int (int)} 0x1016 <foo>

当启用覆盖调试时，无论覆盖是否被映射，GDB都可以找到覆盖中函数和变量的正确地址。这使得大多数GDB命令，如break和disassemble，即使在未映射的代码上也能正常工作。但是，GDB的断点支持有一些限制：

  • 只要GDB能够在其加载地址处写入覆盖，你就可以在未映射覆盖中的函数中设置断点。
  • GDB不能在未映射覆盖中设置硬件或基于模拟器的断点。但是，如果在覆盖管理器的末尾设置一个断点（如果你使用手动覆盖管理，还要告诉GDB哪些覆盖现在被映射），GDB会正确地重新设置其断点。
