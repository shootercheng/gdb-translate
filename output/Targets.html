19 指定调试目标 ¶

一个目标是你的程序所占用的执行环境。

通常，GDB与你的程序在同一主机环境中运行；在这种情况下，当你使用file或core命令时，调试目标会作为副作用被指定。当你需要更多灵活性时——例如在物理上分离的主机上运行GDB，或者通过串口控制独立系统或通过TCP/IP连接控制实时系统——你可以使用target命令来指定为GDB配置的目标类型之一（见 管理目标的命令）。

可以为几种不同的目标架构构建GDB。当以这种方式构建GDB时，你可以使用set architecture命令选择可用架构之一。

set architecture arch ¶

此命令将当前目标架构设置为arch。arch的值除了可以是支持的架构之一外，还可以是"auto"。

show architecture

显示当前目标架构。

set processor ¶
processor
