10.21字符集 ¶

如果你正在调试的程序使用的字符集与GDB自身使用的字符集不同来表示字符和字符串，GDB可以为你自动在所使用的字符集之间进行转换。我们将GDB使用的字符集称为主机字符集；将被调试程序使用的字符集称为目标字符集。

例如，如果你在使用ISO拉丁1字符集的GNU/Linux系统上运行GDB，但你使用GDB的远程协议（见调试远程程序）来调试在使用EBCDIC字符集的IBM大型机上运行的程序，那么主机字符集是拉丁1，目标字符集是EBCDIC。如果你给GDB命令set target-charset EBCDIC-US，那么当你打印字符或字符串值，或者在表达式中使用字符和字符串字面量时，GDB会在EBCDIC和拉丁1之间进行转换。

GDB无法自动识别被调试程序使用的字符集；你必须使用下面描述的set target-charset命令来告诉它。

以下是用于控制GDB字符集支持的命令：

set target-charset charset ¶

将当前目标字符集设置为charset。要显示支持的目标字符集列表，请键入 set target-charset TABTAB。

set host-charset charset ¶

将当前主机字符集设置为charset。

默认情况下，GDB使用适合其运行系统所使用的主机字符集；你可以使用set host-charset命令覆盖该默认设置。在某些系统上，GDB无法自动确定合适的主机字符集。在这种情况下，GDB使用“UTF-8”。

GDB只能使用某些字符集作为其主机字符集。如果你键入set host-charset TABTAB，GDB将列出它支持的主机字符集。

set charset charset ¶

将当前主机和目标字符集都设置为charset。如上所述，如果键入set charset TABTAB，GDB将列出可同时用于主机和目标的字符集名称。

show charset ¶

显示当前主机和目标字符集的名称。

show host-charset ¶

显示当前主机字符集的名称。

show target-charset ¶

显示当前目标字符集的名称。

set target-wide-charset charset ¶

将当前目标的宽字符集设置为charset。这是目标的wchar_t类型使用的字符集。要显示支持的宽字符集列表，请键入 set target-wide-charset TABTAB。

show target-wide-charset ¶

显示当前目标宽字符集的名称。

以下是GDB字符集支持的一个实际示例。 假设已将以下源代码放入文件charset-test.c中： 

#include <stdio.h>

char ascii_hello[]
  = {72, 101, 108, 108, 111, 44, 32, 119,
     111, 114, 108, 100, 33, 10, 0};
char ibm	1047_hello[]
  = {200, 133, 147, 147, 150, 107, 64, 166,
     150, 153, 147, 132, 90, 37, 0};

main ()
{
  printf ("Hello, world!\n");
}

在这个程序中，ascii_hello和ibm1047_hello是数组，包含字符串“Hello, world!”后跟一个换行符，分别以ASCII和IBM1047字符集编码。

我们编译该程序，并在其上调用调试器： 

$ gcc -g charset-test.c -o charset-test
$ gdb -nw charset-test
GNU gdb 2001-12-19-cvs
Copyright 2001 Free Software Foundation, Inc.
...
(gdb)

我们可以使用show charset命令查看GDB当前用于解释和显示字符及字符串的字符集： 

(gdb) show charset
当前主机和目标字符集是 `ISO-8859-1'。
(gdb)

为了打印本手册，让我们将ASCII用作初始字符集： 

(gdb) set charset ASCII
(gdb) show charset
当前主机和目标字符集是 `ASCII'。
(gdb)

假设ASCII确实是我们主机系统的正确字符集 —— 换句话说，假设如果GDB使用ASCII字符集打印字符，我们的终端将正确显示它们。由于我们当前的目标字符集也是ASCII，ascii_hello的内容可以清晰打印： 

(gdb) print ascii_hello
$1 = 0x401698 "Hello, world!\n"
(gdb) print ascii_hello[0]
$2 = 72 'H'
(gdb)

GDB在表达式中使用你使用的字符和字符串字面量的目标字符集： 

(gdb) print '+'
$3 = 43 '+'
(gdb)

ASCII字符集使用数字43来编码“+”字符。

GDB依赖用户告诉它目标程序使用的字符集是什么。如果我们在目标字符集仍为ASCII时打印ibm1047_hello，我们会得到乱码： 

(gdb) print ibm1047_hello
$4 = 0x4016a8 "\310\205\223\223\226k@\246\226\231\223\204Z%"
(gdb) print ibm1047_hello[0]
$5 = 200 '\310'
(gdb)

如果我们调用set target-charset并后跟TABTAB, GDB会告诉我们它支持的字符集： 

(gdb) set target-charset
ASCII       EBCDIC-US   IBM1047     ISO-8859-1
(gdb) set target-charset

我们可以选择IBM1047作为我们的目标字符集，然后再次检查程序的字符串。现在ASCII字符串显示错误，但GDB将ibm1047_hello的内容从目标字符集IBM1047转换为主机字符集ASCII，并且它们显示正确： 

(gdb) set target-charset IBM1047
(gdb) show charset
当前主机字符集是 `ASCII'。
当前目标字符集是 `IBM1047'。
(gdb) print ascii_hello
$6 = 0x401698 "\110\145%%?\054\040\167?\162%\144\041\012"
(gdb) print ascii_hello[0]
$7 = (gdb) print ibm1047_hello
$8 = 0x4016a8 "Hello, world!\n"
(gdb) print ibm1047_hello[0]
$9 = 200 'H'
(gdb)

如上所述，GDB在表达式中使用你使用的字符和字符串字面量的目标字符集以及目标字符集： 

(gdb) print '+'
$10 = 78 '+'
(gdb)

IBM1047字符集使用数字78来编码“+”字符。
