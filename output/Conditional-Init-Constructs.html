33.3.2条件初始化结构 ¶

Readline实现了一种与C预处理器的条件编译功能类似的机制，它允许根据测试结果来执行按键绑定和变量设置。这里使用了四个解析指令。

$if

$if结构允许根据编辑模式、正在使用的终端或使用Readline的应用程序来进行绑定。测试文本在任何比较运算符之后，一直延伸到行尾；除非另有说明，不需要任何字符来分隔它。

mode

$if指令的mode=形式用于测试Readline是处于emacs模式还是vi模式。例如，这可以与‘set keymap’命令结合使用，以便仅在Readline以emacs模式启动时，在emacs-standard和emacs-ctlx键映射中设置绑定。

term

term=形式可用于包含特定于终端的按键绑定，也许是为了绑定终端功能键输出的键序列。‘=’右侧的单词会与终端的全名以及终端名称中第一个‘-’之前的部分进行测试。例如，这允许sun匹配sun和sun-cmd。

version

version测试可用于与特定的Readline版本进行比较。version会扩展为当前的Readline版本。比较运算符包括‘=’（以及‘==’）、‘!=’、‘<=’、‘>=’、‘<’和‘>’。运算符右侧提供的版本号由主版本号、一个可选的小数点和一个可选的次版本号组成（例如，‘7.1’）。如果省略次版本号，则假定为‘0’。运算符可以与字符串version以及版本号参数用空格分隔。以下示例在使用的Readline版本为7.0或更高版本时设置一个变量： 

$if version >= 7.0
set show-mode-in-prompt on
$endif
application

application结构用于包含特定于应用程序的设置。每个使用Readline库的程序都会设置应用程序名称，你可以测试特定的值。 这可用于将键序列绑定到对特定程序有用的函数。例如，以下命令添加了一个在Bash中引用当前或前一个单词的键序列： 

$if Bash
# 引用当前或前一个单词
"\C-xq": "\eb\"\ef\""
$endif
variable

variable结构为Readline变量和值提供简单的相等性测试。 允许的比较运算符是‘=’、‘==’和‘!=’。变量名必须与比较运算符用空格分隔；运算符可以与右侧的值用空格分隔。 字符串和布尔变量都可以进行测试。布尔变量必须与值on和off进行测试。 以下示例等同于上述的mode=emacs测试： 

$if editing-mode == emacs
set show-mode-in-prompt on
$endif
$endif

如前一个示例所示，此命令终止一个$if命令。

$else

如果测试失败，则执行$if指令此分支中的命令。

$include

此指令接受一个文件名作为参数，并从该文件中读取命令和绑定。 例如，以下指令从/etc/inputrc读取： 

$include /etc/inputrc
