4.6你的程序的输入和输出 ¶

默认情况下，你在GDB下运行的程序对输入和输出使用与GDB相同的终端。GDB会将终端切换到它自己的终端模式来与你交互，但它会记录你的程序正在使用 的终端模式，并在你继续运行程序时切换回这些模式。

info terminal ¶

显示GDB记录的关于你的程序正在使用的终端模式的信息。

你可以使用带有 run 命令的 shell 重定向来重定向你的程序的输入和/或输出。例如， 

run > outfile

启动你的程序，将其输出重定向到文件 outfile。

指定你的程序应该在哪里进行输入和输出的另一种方法是使用 tty 命令。这个命令接受一个文件名作为参数，并使这个文件成为未来 run 命令的默认值。它还会为子进程重置控制终端，用于未来的 run 命令。例如， 

tty /dev/ttyb

指示后续使用 run 命令启动的进程默认在终端 /dev/ttyb 上进行输入和输出，并将其作为控制终端。

run 中的显式重定向会覆盖 tty 命令对输入/输出设备的影响，但不会影响其对控制终端的影响。

当你使用 tty 命令或在 run 命令中重定向输入时，只有你的程序的输入会受到影响。GDB的输入仍然来自你的终端。tty 是 set inferior-tty 的别名。

你可以使用 show inferior-tty 命令来告诉GDB显示将用于你未来运行程序的终端的名称。

set inferior-tty [ tty ] ¶

将正在调试 的程序的tty设置为 tty。省略 tty 会恢复默认行为，即使用与GDB相同的终端。

show inferior-tty ¶

显示正在调试的程序的当前tty。
