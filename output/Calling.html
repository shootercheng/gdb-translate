17.5调用程序函数 ¶

print 表达式 ¶

计算表达式表达式并显示结果值。 该表达式可能包括对正在调试的程序中的函数的调用。

call 表达式 ¶

计算表达式表达式，不显示返回值为void的值。

如果你想从程序中执行一个不返回任何值的函数（也称为无返回值函数），但又不想让GDB打印返回值为void的结果而使输出混乱，那么你可以使用这个变体的print命令。如果结果不是void，则会被打印并保存在值历史中。

通过print或call命令调用的函数有可能会产生一个信号（例如，如果函数中有错误，或者你传递给它的参数不正确）。在这种情况下发生的事情由set unwind-on-signal命令控制。

类似地，对于一个C++程序，通过print或call命令调用的函数有可能会产生一个由于虚拟帧的限制而未被处理的异常。在这种情况下，在帧中抛出但有帧外异常处理程序的任何异常将不会被找到。GDB为下级函数调用构建一个虚拟帧，并且展开器无法在这个虚拟帧之外寻找异常处理程序。在这种情况下发生的事情由set unwind-on-terminating-exception命令控制。

set unwind-on-signal ¶

如果在GDB在正在调试的程序中调用的函数中接收到信号，则设置堆栈展开。如果设置为on，GDB会展开它为调用创建的堆栈，并将上下文恢复到调用前的状态。如果设置为off（默认），GDB会在接收到信号的帧中停止。

命令set unwindonsignal是这个命令的别名，为了向后兼容而保留。

show unwind-on-signal ¶

显示GDB调用的函数中堆栈展开的当前设置。

命令show unwindonsignal是这个命令的别名，为了向后兼容而保留。

set unwind-on-terminating-exception ¶

如果在GDB在正在调试的程序中调用的函数中抛出了一个C++异常，但没有被处理，则设置堆栈展开。如果设置为on（默认），GDB会展开它为调用创建的堆栈，并将上下文恢复到调用前的状态。如果设置为off，该异常会被传递给默认的C++异常处理程序，并且下级进程会终止。

show unwind-on-terminating-exception ¶

显示GDB调用的函数中堆栈展开的当前设置。

set unwind-on-timeout ¶

如果从GDB调用的函数超时，则设置堆栈展开。如果设置为off（默认值），GDB会在超时发生的帧中停止。如果设置为on，GDB会展开它为调用创建的堆栈，并将上下文恢复到调用前的状态。

show unwind-on-timeout ¶

显示如果从GDB调用的函数超时，GDB是否会展开堆栈。

set may-call-functions ¶

设置在程序中调用函数的权限。 这控制着GDB是否会尝试在程序中调用函数，例如使用print命令中的表达式。它的默认值是on。

为了在程序中调用函数，GDB必须临时修改下级进程的状态。这可能会有潜在的不良副作用。此外，让GDB调用嵌套函数很可能是错误的，甚至可能导致正在调试的程序崩溃
