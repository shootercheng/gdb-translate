10.14 寄存器 ¶

在表达式中，你可以将机器寄存器的内容作为以“$”开头的变量来引用。不同机器上寄存器的名称不同；使用info registers查看你的机器上使用的名称。

info registers ¶

打印除浮点寄存器和向量寄存器之外的所有寄存器的名称和值（在选定的栈帧中）。

info all-registers ¶

打印所有寄存器的名称和值，包括浮点寄存器和向量寄存器（在选定的栈帧中）。

info registers 寄存器组 …

打印每个指定的寄存器组中的寄存器的名称和值。寄存器组可以是maint print reggroups返回的任何一个（见 维护命令）。

info registers 寄存器名 …

打印每个指定的寄存器寄存器名的相对化值。如下文详细讨论的，寄存器值通常相对于选定的栈帧。寄存器名可以是你正在使用的机器上有效的任何寄存器名，带或不带初始的“$”。

GDB有四个“标准”寄存器名，在大多数机器上（只要它们与架构的寄存器规范助记符不冲突），在表达式中都可用。寄存器名$pc和$sp分别用于程序计数器寄存器和栈指针。$fp用于包含指向当前栈帧指针的寄存器，$ps用于包含处理器状态的寄存器。例如，你可以用以下方式以十六进制打印程序计数器： 

p/x $pc

或者用以下方式打印下一条要执行的指令： 

x/i $pc

或者用以下方式将栈指针加412： 

set $sp += 4

只要没有冲突，如果可能的话，即使机器有不同的规范助记符，你的机器上也可以使用这四个标准寄存器名来引用。info registers命令显示规范名称。例如，在SPARC上，info registers将处理器状态寄存器显示为$psr，但你也可以将其称为$ps；在基于x86的机器上，$ps是EFLAGS寄存器的别名。

当以这种方式检查寄存器时，GDB始终将普通寄存器的内容视为整数。有些机器有只能保存浮点浮点的特殊寄存器，这些寄存器被视为具有浮点值。没有办法将普通寄存器的内容作为浮点值来引用（尽管你可以用“print/f $寄存器名”将其打印为浮点值）。

有些寄存器有不同的“原始”和“虚拟”数据格式。这意味着操作系统保存寄存器内容的数据格式与你的程序通常看到的格式不同样。例如，68881浮点协处理器的寄存器总是以“扩展”（原始）格式保存，但所有C程序都期望使用“双精度”（虚拟）格式工作。在这种情况下，GDB通常只使用虚拟格式（对你的程序有意义的格式），但info registers命令会以两种格式打印数据。

有些机器有特殊寄存器，其内容可以用几种不同的方式解释。例如，现代基于x86的机器有SSE和MMX寄存器，可以以几种不同的格式将几个值打包在一起保存。GDB以struct表示法引用此类寄存器： 

(gdb) print $xmm1
$1 = {
  v4_float = {0, 3.43859137e-038, 1.54142831e-044, 1.821688e-044},
  v2_double = {9.92129282474342e-303,......
  v16_int8 = "\000\000\000\000\3706;\001\v\000\000\000\r\000\000",
  v8_int16 = {0, 0, 14072, 315, 11, 0, 13, 0},
  v4_int32 = {0, 20657912, 11, 13},
  v2_int64 = {88725056443645952, 55834574859},
  uint128 = 0x0000000d0000000b013b36f800000000
}

要设置此类寄存器的值，你需要告诉GDB你希望更改寄存器的哪种视图，就好像你在为一个struct成员赋值一样： 

 (gdb) set $xmm1.uint128 = 0x000000000000000000000000FFFFFFFF

通常，寄存器值相对于选定的栈帧（见 选择一个帧）。这意味着你得到的是如果所有更内层的栈帧都退出并恢复其保存的寄存器时寄存器将包含的值。为了查看硬件寄存器的真实内容，你必须选择最内层的帧（使用“frame 0”）。

通常，ABI会保留一些寄存器，被调用者不需要保存这些寄存器（也称为：“调用者保存”、“调用时被破坏”或“易失性”寄存器）。因此，如果寄存器值此后被被调用者更改，GDB可能无法知道调用前寄存器的值（换句话说，在外层帧中）。GDB尝试从调试信息、展开信息或你的编译器生成的机器代码中推断内层帧保存（“被调用者保存”）寄存器的位置。如果某个寄存器没有被保存，并且GDB知道该寄存器是“调用者保存”的（通过其对ABI的了解，或者因为调试/展开信息明确表示该寄存器的值未定义），GDB会显示“<not saved>”作为该寄存器的值。对于GDB不了解寄存器保存约定的目标，如果某个寄存器没有被被调用者保存，那么假设其在外层帧中的值和位置与内层帧中的相同。这通常是无害的，因为如果该寄存器是调用时被破坏的，调用者要么不在乎调用后寄存器中的内容，要么有代码来恢复它关心的值。但是请注意，如果在外层帧中更改这样的寄存器，你也可能会影响内层帧。此外，你查看的帧越“外层”，调用时被破坏的寄存器的值就越有可能是错误的，因为它实际上并不代表调用前寄存器的值。
