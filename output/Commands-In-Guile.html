23.4.3.11 Guile中的命令 ¶

你可以在Guile中实现新的GDB CLI命令。CLI命令对象通过Guile函数 make-command 创建，并通过Guile函数 register-command! 添加到GDB中。采取这种两步走的方法是为了将把命令添加到GDB的副作用与 make-command 分离开来。

不支持多行命令，即由多行组成并以 end 结尾的命令。

Scheme过程： make-command name [#:invoke invoke] [#:command-class command-class] [#:completer-class completer] [#:prefix? prefix] [#:doc doc-string] ¶

参数 name 是命令的名称。如果 name 由多个单词组成，那么开头的单词会被视为前缀命令。在这种情况下，如果其中一个前缀命令不存在，就会引发异常。

结果是表示该命令的 <gdb:command> 对象。在使用 register-command! 将该命令注册到GDB之前，它是不可用的。

其余参数是可选的。

参数 invoke 是一个有三个参数的过程：self、args 和 from-tty。参数 self 是表示该命令的 <gdb:command> 对象。参数 args 是一个字符串，表示传递给命令的参数，去除了前后的空白字符。参数 from-tty 是一个布尔标志，指定命令是否应认为自己是由用户交互式调用产生的。如果这个函数抛出异常，它会被转换为GDB的 error 调用。否则，返回值将被忽略。

对于非前缀命令，invoke 是必需的。对于前缀命令，invoke 是可选的。只有需要处理未知子命令的前缀命令才应提供 invoke。
