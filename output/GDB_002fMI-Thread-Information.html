27.5.6 GDB/MI线程信息 ¶

每当GDB需要报告有关线程的信息时，它会使用一个包含以下字段的元组。除非另有说明，否则这些字段总是存在的。

id

GDB分配给线程的全局数字ID。

target-id

用于标识线程的特定于目标的字符串。

details

目标提供有关线程的其他信息。它应该是人类可读的，并且不被前端解释。此字段是可选的。

name

线程的名称。如果用户使用thread name命令指定了名称，则给出此名称。否则，如果GDB可以从目标中提取线程名称，则给出该名称；如果GDB找不到线程名称，则省略此字段。

state

线程的执行状态，根据线程当前是否正在运行，为‘stopped’或‘running’。

frame

线程当前正在执行的栈帧。仅当线程停止时此字段才存在。其格式在GDB/MI帧信息中有记录。

core

此字段的值是线程最后出现所在的处理器核心的整数编号。此字段是可选的。
