23.3.2.26 Python中的程序空间 ¶

一个程序空间，或称为progspace，表示一个地址空间的符号视图。 它由程序的所有目标文件组成。 请参阅 Python中的目标文件。 有关程序空间的更多详细信息，请参阅 程序空间。

在gdb模块中可以使用以下与程序空间相关函数：

函数：gdb.current_progspace () ¶

此函数返回当前选定的进程的程序空间。 请参阅 调试多个进程连接和程序。 这与 gdb.selected_inferior().progspace 相同（请参阅 Python中的进程），包含它是为了保持历史兼容性。

函数：gdb.progspaces () ¶

返回GDB当前已知的所有程序空间的序列。

每个程序空间由 gdb.Progspace 类的一个实例表示。

变量：Progspace.filename ¶

程序空间的主符号文件（已从中加载调试符号）的文件名，作为字符串，例如 symbol-file 或 file 命令的参数。

如果当前没有加载主符号表，那么这个属性将是 None。

变量：Progspace.symbol_file ¶

表示 gdb.Progspace 的主符号文件（已从中加载调试符号）的 gdb.Objfile。 这是由 symbol-file 或 file 命令设置的符号文件。

当 Progspace.filename 不为 None 时，这将是表示 Progspace.filename 的 gdb.Objfile。

如果当前没有加载主符号表，那么这个属性将是 None。

如果 Progspace 无效，即当 Progspace.is_valid() 返回 False 时，那么尝试访问这个属性将引发 RuntimeError 异常。
