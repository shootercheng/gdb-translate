10.23搜索内存 ¶

可以使用find命令在内存中搜索特定的字节序列。

find [/sn] start_addr, +len, val1 [, val2, …] ¶
find [/sn] start_addr, end_addr, val1 [, val2, …]

在内存中搜索由val1、val2等指定的字节序列。搜索从地址start_addr开始，持续len个字节或一直到包括end_addr的位置。

s和n是可选参数。 它们可以按任意顺序指定，可以分开指定，也可以一起指定。

s，搜索查询大小

每个搜索查询值的大小。

b

字节

h

半字（两个字节）

w

字（四个字节）

g

巨字（八个字节）

所有值都按照当前语言进行解释。 这意味着，例如，如果当前源语言是C/C++，那么搜索字符串“hello”时会包含尾随的'\0'。 可以通过使用强制类型转换从搜索中移除空终止符，例如：‘{char[5]}"hello"’。

如果未指定值的大小，则从当前语言中值的类型获取。 当想将搜索模式指定为多种类型的混合时，这很有用。 请注意，这意味着，例如，在类似C的语言中，搜索无类型的0x42时将搜索‘ (int) 0x42’，通常是四个字节。

n，最大查找次数

要打印的最大匹配次数。默认是打印所有找到的匹配项。

可以使用字符串作为搜索值。用双引号（"）引起来。 字符串值会逐个字节地复制到搜索模式中，而不管目标的字节序和大小规范。

找到的每个匹配项的地址以及找到匹配项的数量都会被打印出来。

最后找到的值的地址存储在便利变量‘$_’中。 匹配项的数量存储在‘$numfound’中。

例如，如果在这个函数中的printf处停止： 

void
hello ()
{
  static char hello[] = "hello-hello";
  static struct { char c; short s; int i; }
    __attribute__ ((packed)) mixed
    = { 'c', 0x1234, 0x87654321 };
  printf ("%s\n", hello);
}

在调试期间会得到： 

(gdb) find &hello[0], +sizeof(hello), "hello"
0x804956d <hello.1620+6>
1个模式找到
(gdb) find &hello[0], +sizeof(hello), 'h', 'e', 'l', 'l', 'o'
0x8049567 <hello.1620>
0x804956d <hello.1620+6>
2个模式找到。
(gdb) find &hello[0], +sizeof(hello), {char[5]}"hello"
0x8049567 <hello.1620>
0x804956d <hello.1620+6>
2个模式找到。
(gdb) find /b1 &hello[0], +sizeof(hello), 'h', 0x65, 'l'
0x8049567 <hello.1620>
1个模式找到
(gdb) find &mixed, +sizeof(mixed), (char) 'c', (short) 0x1234, (int) 0x87654321
0x8049560 <mixed.1625>
1个模式找到
(gdb) print $numfound
$1 = 1
(gdb) print $_
$2 = (void *) 0x8049560
