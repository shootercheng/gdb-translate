15.3.1类型检查概述 ¶

有些语言，如C和C++，是强类型的，这意味着运算符和函数的参数必须是正确的类型，否则会发生错误。这些检查可防止类型不匹配错误导致任何运行时问题。例如， 

int klass::my_method(char *b) { return  b? 1 : 2; }

(gdb) print obj.my_method (0)
$1 = 2

但是

(gdb) print obj.my_method (0x1234)
无法将方法klass::my_method解析为任何重载实例

第二个例子失败是因为在C++中，整数常量‘0x1234’与指针参数类型不兼容。

对于你在GDB命令中使用的表达式，你可以告诉GDB不要强制执行严格的类型检查，或者将任何不匹配视为错误并放弃该表达式；当禁用类型检查时，GDB可以成功地计算像上面第二个例子那样的表达式。

即使关闭了类型检查，可能还有其他与类型相关的原因阻止GDB计算表达式。例如，GDB不知道如何将一个int和一个struct foo相加。这些特定的类型错误与所使用的语言无关，通常是由无论如何都没有意义的表达式引起的。

GDB提供了一些用于控制类型检查的附加命令：

set check type on
set check type off

打开或关闭严格类型检查。如果在类型检查打开时计算表达式时发生任何类型不匹配，GDB会打印一条消息并中止表达式的计算。

show check type

显示类型检查的当前设置以及GDB是否正在强制执行严格的类型检查规则。
