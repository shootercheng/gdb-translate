21.4.9 AMD GPU ¶

GDB支持使用 AMD ROCm 平台调试卸载到 AMD GPU 设备上的程序。GDB 会将主机线程和GPU波前一起显示，允许同时调试程序中的主机部分和设备部分。

  • AMD GPU架构
  • AMD GPU设备驱动和AMD ROCm运行时
  • AMD GPU波前
  • AMD GPU代码对象
  • AMD GPU实体目标标识符和便利变量
  • AMD GPU信号
  • AMD GPU内存违规报告
  • AMD GPU日志记录
  • AMD GPU限制

21.4.9.1 AMD GPU架构 ¶

GDB 支持的 AMD GPU 架构列表取决于所使用的AMD调试器API库的版本。有关更多详细信息，请参阅其 文档。

21.4.9.2 AMD GPU设备驱动和AMD ROCm运行时 ¶

GDB 需要安装兼容的 AMD GPU 设备驱动。如果设备驱动版本或其实现的调试支持版本不受支持，将显示一条警告消息。除了无法进行 AMD GPU 调试外，GDB 将继续运行。

GDB 要求每个代理由设备驱动安装兼容的固件。如果检测到不支持的固件，将显示一条警告消息。除了无法在该代理上进行 AMD GPU 调试外，GDB 将继续运行。

GDB 需要加载兼容的 AMD ROCm 运行时才能检测 AMD GPU 代码对象和波前。如果检测到不受支持的 AMD ROCm 运行时，或者存在阻止调试的错误或限制，将显示一条警告消息。除了无法进行 AMD GPU 调试外， GDB 将继续运行。

21.4.9.3 AMD GPU波前 ¶

在 GDB 中，AMD GPU 波前表示为一个线程。

请注意，某些 AMD GPU 架构在提供有关在 GDB 未附加时创建的 AMD GPU 波前的信息方面可能有限制（请参阅 AMD GPU附加限制）。

当调度锁定生效时（请参阅 设置调度锁定），恢复的线程（无论是CPU线程还是GPU波前）创建的新波前将保持暂停状态。

21.4.9.4 AMD GPU代码对象 ¶

‘info sharedlibrary’ 命令将显示 AMD GPU 代码对象，以文件或内存URI的形式，以及主机的共享库。例如： 

(gdb) info sharedlibrary
From    To      Syms Read   Shared Object Library
0x1111  0x2222  Yes (*)     /lib64/ld-linux-x86-64.so.2
...
0x3333  0x4444  Yes (*)     /opt/rocm-4.5.0/.../libamd_comgr.so
0x5555  0x6666  Yes (*)     /lib/x86_64-linux-gnu/libtinfo.so.5 
0x7777  0x8888  Yes         file:///tmp/a.out#offset=6477&size=10832
0x9999  aaaa  Yes (*).     memory://95557/mem#offset=0x1234&size=100
(*): Shared library is missing debugging information.
(gdb)

对于 ‘file’ URI，路径部分是磁盘上包含代码对象的文件。offset 参数是此文件中代码对象开始处的基于0的偏移量。如果省略，它默认为0。size 参数是代码对象的字节大小。如果省略，它默认为文件的大小。

对于 ‘memory’ URI，路径部分是拥有包含代码对象的内存的进程的进程ID。offset 参数是找到代码对象的内存地址，size 参数是其字节大小。

AMD GPU 代码对象分别加载到每个 AMD GPU 设备中。因此，‘info sharedlibrary’ 命令可能会多次显示加载的相同代码对象。因此，如果有多个 AMD GPU 设备，在 AMD GPU代码中设置断点将导致多个断点位置。

21.4.9.5 AMD GPU实体目标标识符和便利变量 ¶

AMD GPU 实体具有以下目标标识符格式：

线程目标ID

AMD GPU 线程目标标识符（systag）字符串具有以下格式： 

AMDGPU Wave agent-id:queue-id:dispatch-id:wave-id (work-group-x,work-group-y,work-group-z)/work-group-thread-index

21.4.9.6 AMD GPU信号 ¶

对于 AMD GPU 波前，GDB 按以下方式将目标条件映射到停止信号：

SIGILL

执行非法指令。

SIGTRAP

执行 S_TRAP 指令，但不包括：

  • S_TRAP 1，它被 GDB 用于插入断点。
  • S_TRAP 2，它会引发 SIGABRT。
SIGABRT

执行 S_TRAP 2 指令。

SIGFPE

执行浮点或整数指令时检测到启用了引发信号的条件。这些条件包括：

  • 浮点运算无效。
  • 浮点运算有次正规输入并被舍入为零。
  • 浮点运算执行了除以零的操作。
  • 浮点运算产生了溢出结果。结果被舍入为无穷大。
  • 浮点运算产生了下溢结果。次正规结果被舍入为零。
  • 浮点运算产生了不精确结果。
  • 整数运算执行了除以零的操作。

默认情况下这些条件不会引发信号。‘set $mode’ 命令可用于更改 AMD GPU 波前的寄存器，该寄存器的某些位控制哪些条件会引发信号。‘print $trapsts’ 命令可用于检查哪些条件已被检测到，即使它们未被设置为引发信号。

SIGBUS

执行使用超出虚拟地址范围的地址访问全局内存的指令。

SIGSEGV

执行访问未映射或权限不兼容的全局内存页面的指令。

如果一条指令引发多个信号，每次波前恢复时它们将逐个报告。

21.4.9.7 AMD GPU内存违规报告 ¶

一个波前可以报告内存违规事件。但是，报告这些事件的程序位置可能在导致这些事件的机器指令之后。这可能导致报告的源语句不正确。可以使用以下命令控制此行为：

set amdgpu precise-memory mode ¶

控制 AMD GPU 设备如何检测内存违规，其中 mode 可以是：

off

程序位置可能不会紧跟在导致内存违规的指令之后。这是默认设置。

on

请求程序位置紧跟在导致内存违规的指令之后。启用此模式可能会使 AMD GPU 设备的执行显著变慢，因为它必须等待每个内存操作完成后才能执行下一条指令。

amdgpu precise-memory 参数是每个下级进程的。当一个下级进程分叉或执行，或者用户使用 clone-inferior 命令并因此创建一个下级进程时，新创建的下级进程继承原始下级进程的参数值。

show amdgpu precise-memory ¶

显示当前请求的AMD GPU精确内存设置。

21.4.9.8 AMD GPU日志记录 ¶

‘set debug amd-dbgapi’ 命令可用于在 ‘amd-dbgapi’ 目标中启用诊断消息。‘show debug amd-dbgapi’ 命令显示当前设置。请参阅 设置调试amd-dbgapi。

‘set debug amd-dbgapi-lib log-level level’ 命令可用于启用来自 ‘amd-dbgapi’ 库（GDB 在内部使用）的诊断消息。‘show debug amd-dbgapi-lib log-level’ 命令显示当前的 ‘amd-dbgapi’ 库日志级别。请参阅 设置调试amd-dbgapi-lib。

21.4.9.9 AMD GPU限制 ¶

  1. 在非停止模式下，波前可能不会命中在未停止时插入的断点，也不会看到在未停止时进行的内存更新，直到波前下次停止。在波前下次停止之前，未停止的波前更新的内存可能不可见。
  2. HIP运行时默认执行延迟代码对象加载。AMD GPU 代码对象直到第一个内核启动时才会加载。在此之前，所有断点都必须设置为待决断点。

    如果使用仅对应于未加载代码对象中的源行的源行位置，那么 GDB 可能无法设置待决断点，而是会在映射到主机代码的下一个源行上设置断点。这可能导致报告意外的断点命中。当包含源行的代码对象加载时，不正确的断点将被删除并替换为正确的断点。通过仅使用符号或函数名称在未加载的代码对象中设置断点，可以避免此问题。

    将 HIP_ENABLE_DEFERRED_LOADING 环境变量设置为 0 可用于禁用HIP运行时的延迟代码对象加载。这可确保当下级进程到达 main 函数开头时，所有代码对象都将被加载。

  3. 如果没有CPU线程正在运行，那么 ‘Ctrl-C’ 无法停止 AMD GPU 线程。例如，如果在整个程序停止后启用 scheduler-locking，然后恢复一个 AMD GPU 线程，就可能会发生这种情况。解除这种情况的唯一方法是杀死 GDB 进程。
  4. 默认情况下，对于某些架构，AMD GPU设备驱动会导致在 GDB 未附加时创建的所有 AMD GPU 波前无法报告与该波前相关的调度或波前的工作组位置。‘info threads’ 命令将用 ‘?’ 显示此缺失信息。

    这不会影响在 GDB 附加时创建的波前，这些波前始终能够报告此信息。

    如果在初始化 AMD ROCm 运行时将 HSA_ENABLE_DEBUG 环境变量设置为 ‘1’，那么即使对于在 GDB 未附加时创建的波前，所有架构的此信息也将可用。
