10.4人工数组 ¶

将内存中几个连续的相同类型对象打印出来通常很有用；可以是数组的一部分，或者是一个动态确定大小且程序中只有指针的数组部分。

你可以通过使用二元运算符‘@’将一段连续的内存视为一个人工数组来实现。‘@’的左操作数应该是所需数组的第一个元素，并且是一个单独的对象。右操作数应该是数组的所需长度。结果是一个数组值，其元素都是左操作数的类型。第一个元素实际上就是左操作数；第二个元素来自紧接着第一个元素所占用内存的字节，依此类推。下面是一个例子。如果一个程序中有： 

int *array = (int *) malloc (len * sizeof (int));

你可以使用以下命令打印‘array’的内容： 

p *array@len

‘@’的左操作数必须驻留在内存中。以这种方式用‘@’创建的值在进行下标操作时的行为与其他数组一样，并且在表达式中使用时会被强制转换为指针。人工数组最常通过值历史记录（见 值历史记录）在表达式中出现，即在打印出一个人工数组之后。

创建人工数组的另一种方法是使用强制类型转换。这会将一个值重新解释为好像它是一个数组。该值不一定在内存中： 

(gdb) p/x (short[2])0x12345678
$1 = {0x1234, 0x5678}

为了方便起见，如果省略数组长度（如‘(type[])value’），GDB会计算填充该值所需的大小（如‘sizeof(value)/sizeof(type)’）： 

(gdb) p/x (short[])0x12345678
$2 = {0x1234, 0x5678}

有时人工数组机制还不够；在中等复杂的数据结构中，感兴趣的元素可能实际上并不相邻 —— 例如，如果您对数组中指针的值感兴趣就会出现这种情况。在这种情况下，一个有用的解决方法是使用一个便利变量（见 便利变量）作为表达式中的计数器，该表达式打印出第一个感兴趣的值，然后通过RET重复该表达式。例如，假设你有一个指向结构体的指针数组‘dtab’，并且你对每个结构体中的一个字段‘fv’的值感兴趣。下面是你可能输入的内容示例： 

set $i = 0
p dtab[$i++]->fv
RET
RET
...
