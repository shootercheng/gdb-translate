3.4作为命令参数的文件名 ¶

当将文件名（或目录名）作为参数传递给命令时， 如果文件名参数不包含任何空格、双引号或单引号，那么对于所有命令，文件名可以写成一个简单字符串，例如： 

(gdb) file /path/to/some/file

如果文件名确实包含空格、双引号或单引号，那么GDB对于这些文件名的格式化有两种方法；使用哪种格式取决于正在使用的命令。

大多数GDB命令不需要或不支持引用和转义。这些命令将命令名后面的任何文本（不是命令选项，见命令选项）视为文件名，即使文件名包含空格或引号字符。在下面的示例中，用户正在将/path/that contains/two spaces/添加到自动加载安全路径（见add-auto-load-safe-path）： 

(gdb) add-auto-load-safe-path /path/that contains/two spaces/

少数命令要求包含空格或引号字符的文件名要么被引用，要么用反斜杠转义特殊字符。支持这种方式的命令在手册中有标记，任何未标记为接受其文件名参数的引用和转义的命令，都不接受这种文件名参数方式。

例如，要使用file命令（见指定文件的命令）加载文件/path/with spaces/to/a file，可以用反斜杠转义空格字符： 

(gdb) file /path/with\ spaces/to/a\ file

或者整个文件名可以用单引号或双引号括起来，在这种情况下不需要反斜杠，例如： 

(gdb) symbol-file "/path/with spaces/to/a file"
(gdb) exec-file '/path/with spaces/to/a file'

可以通过用反斜杠转义来在引用的文件名中包含引号字符，例如，在双引号包围的文件名中，双引号字符要用反斜杠转义，但单引号字符不用转义。在单引号字符串中，单引号字符需要转义，但双引号字符不用。

也可以通过用反斜杠转义来包含一个字面的反斜杠字符。
