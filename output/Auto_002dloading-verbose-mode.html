22.8.4 显示尝试自动加载的文件 ¶

为了更清楚地了解可以放置要与下级进程一起自动加载的脚本的所有文件位置 — 或者为了保护自己免受意外执行不可信脚本的影响 — GDB 提供了一个功能，用于打印所有尝试加载的文件。现有文件和不存在的文件都可能会被打印出来。

例如，从其中自动加载文件是安全的目录列表（见 自动加载的安全限制）也适用于规范化的文件名，在设置时可能不太明显。 

(gdb) set debug auto-load on
(gdb) file ~/src/t/true
自动加载：正在为目标文件“/tmp/true”加载命令脚本“/tmp/true-gdb.gdb”的预定义序列。
自动加载：正在更新“/usr:/opt”的目录。
自动加载：正在使用目录 “/usr”.
自动加载：正在使用目录 “/opt”.
警告：文件“/tmp/true-gdb.gdb”的自动加载已被您设置为“/usr:/opt”的“自动加载安全路径”拒绝。
set debug auto-load [on|off] ¶

设置是否打印尝试自动加载的文件名。

show debug auto-load ¶

显示是否开启了打印尝试自动加载的文件名的功能。
