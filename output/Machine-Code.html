9.6源文件和机器码 ¶

你可以使用命令info line将源行映射到程序地址（反之亦然），并使用命令disassemble将一系列地址显示为机器指令。你可以使用命令set disassemble-next-line来设置当执行停止时是否反汇编下一行源文件。在GNU Emacs模式下运行时，info line命令会使箭头指向指定的行。此外，info line会以符号形式以及十六进制形式打印地址。

info line ¶
info line locspec

打印通过解析locspec得到的代码位置的源行的编译代码的起始和结束地址。有关locspec的各种形式，请参阅 位置规范。 如果没有locspec，则打印有关当前源行的信息。

例如，我们可以使用info line来发现函数m4_changequote第一行的目标代码的位置： 

(gdb) info line m4_changequote
“builtin.c”文件的第895行从pc 处开始0x634c <m4_changequote> ，并在0x6350 <m4_changequote+4>处结束。

我们还可以使用*addr作为locspec的形式来询问哪个源行覆盖特定地址addr： 

(gdb) info line *0x63ff
“builtin.c”文件的第926行从pc 处开始0x63e4 <m4_changequote+152> ，并在0x6404 <m4_changequote+184>处结束。

在执行info line之后，x命令的默认地址会更改为该行的起始地址，因此只需输入“x/i”就可以开始检查机器码（请参阅 检查内存）。此外，这个地址会被保存为便利变量$_的值（请参阅 便利变量）。

在执行info line之后，再次使用info line而不指定位置将显示有关下一个源行的信息。

disassemble ¶
disassemble /m
disassemble /s
disassemble /r
disassemble /b

这个专门的命令将一段内存作为机器指令转储。它还可以通过指定/m或/s修饰符来打印混合的源文件+反汇编内容，并通过指定/r或/b修饰符以十六进制以及符号形式打印原始指令。

/m和/s只能使用其中一个，尝试同时使用两个会导致错误。

/r和/b只能使用其中一个，尝试同时使用两个会导致错误。

默认的内存范围是所选帧的程序计数器周围的函数。这个命令的单个参数是一个程序计数器值；GDB会转储这个值周围的函数。当给出两个参数时，它们应该用逗号分隔，可能会被空格包围。参数指定要转储的地址范围，有两种形式：

start,end

从start（包含）到end（不包含）的地址

start,+length

从start（包含）到start+length（不包含）的地址。

当指定两个参数时，还会打印函数的名称（因为给定范围内可能有多个函数）。

参数可以是任何产生数值的表达式，例如“0x32c4”、“&main+10”或“$pc - 8”。

如果正在反汇编的内存范围包含当前程序计数器，则该位置的指令会用“=>”标记显示。

以下示例展示了对HP PA-RISC 2.0代码的一系列地址的反汇编： 

(gdb) disas 0x32c4, 0x32e4
从0x32c4到0x32e4的汇编代码转储：
   0x32c4 <main+204>:      addil 0,dp
   0x
