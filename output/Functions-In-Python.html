23.3.2.25编写新的便利函数 ¶

你可以在Python中实现新的便利函数（见 便利变量） 便利函数是 gdb.Function 类的子类的一个实例。

函数：Function.__init__ (name) ¶

Function 的初始化器会向 GDB 注册新函数。参数 name 是函数的名称，为字符串类型。该函数对用户来说将作为一个类型为 内部函数 的便利变量可见，其名称与给定的 name 相同。

新函数的文档取自新类的文档字符串。

函数：Function.invoke (*args) ¶

当评估一个便利函数时，其参数会被转换为 gdb.Value 的实例，然后调用该函数的 invoke 方法。请注意，GDB 不会预先确定便利函数的参数个数。相反，所有可用的参数会按照标准的Python调用约定传递给 invoke。特别地，便利函数可以有默认参数值而不会产生不良影响。

此方法的返回值将用作其在包含表达式中的值。如果返回一个普通的Python值，它将按照通常的规则转换为 gdb.Value。

以下代码片段展示了如何在Python中实现一个简单的便利函数： 

class Greet (gdb.Function):
  """返回用于问候某人的字符串。
以名字作为参数。"""

def __init__ (self):
    super (Greet, self).__init__ ("greet")

def invoke (self, name):
    return "Hello, %s!" % name.string ()

Greet ()

最后一行实例化了这个类，并且这对于触发函数向 GDB 的注册是必要的。根据Python代码如何被读入 GDB，你可能需要显式地导入 gdb模块。

现在你可以在表达式中使用这个函数： 

(gdb) print $greet("Bob")
$1 = "Hello, Bob!"
