30.4.2编写即时调试信息读取器 ¶

如前所述，读取器本质上是一个符合特定ABI的共享对象。此ABI在jit-reader.h中描述。

jit-reader.h定义了编写读取器所需的结构、宏和函数。它（与GDB一起）安装在includedir/gdb中，其中includedir是系统包含目录。

读取器需要在GPL兼容许可证下发布。通过在源文件中放置宏GDB_DECLARE_GPL_COMPATIBLE_READER，可以声明读取器是在这样的许可证下发布的。

读取器的入口点是符号gdb_init_reader，它应该是一个具有以下原型的函数 

extern struct gdb_reader_funcs *gdb_init_reader (void);

struct gdb_reader_funcs包含一组指向回调函数的指针。这些函数用于读取JIT编译器生成的调试信息(read)、展开堆栈帧(unwind)以及创建规范的帧ID (get_frame_id)。它还有一个在读取器卸载时调用的回调函数(destroy)。该结构体如下所示 

struct gdb_reader_funcs
{
  /* 必须设置为GDB_READER_INTERFACE_VERSION。 */
  int reader_version;

  /* 供读取器使用。 */
  void *priv_data;

  gdb_read_debug_info *read;
  gdb_unwind_frame *unwind;
  gdb_get_frame_id *get_frame_id;
  gdb_destroy_reader *destroy;
};

GDB会为回调函数提供另一组回调函数来完成它们的工作。对于read，这些回调函数通过一个struct gdb_symbol_callbacks传递，对于unwind和get_frame_id，则通过一个struct gdb_unwind_callbacks传递。struct gdb_symbol_callbacks具有在对象文件中创建新对象文件和新符号表的回调函数。struct gdb_unwind_callbacks具有从当前帧读取寄存器并写出前一帧寄存器值的回调函数。两者都有一个用于从目标地址空间读取字节的回调函数(target_read)。
