E.21分支跟踪配置格式 ¶

对于每个下级线程，GDB可以使用‘qXfer:btrace-conf:read’数据包获取分支跟踪配置（见 qXfer btrace-conf read）。

该配置描述了分支跟踪格式以及该格式的配置设置。 描述了以下信息：

bts

此线程使用分支跟踪存储（BTS）格式。

size

BTS环形缓冲区的大小（以字节为单位）。

pt

此线程使用英特尔处理器跟踪（Intel PT）格式。

size

Intel PT环形缓冲区的大小（以字节为单位）。

GDB必须与Expat库链接以支持XML分支跟踪配置发现。 见 Expat。

分支跟踪配置格式的正式DTD如下： 

<!ELEMENT btrace-conf		(bts?, pt?)>
<!ATTLIST btrace-conf		version	CDATA	#FIXED "1.0">

<!ELEMENT bts	EMPTY>
<!ATTLIST bts	size		CDATA	#IMPLIED>

<!ELEMENT pt	EMPTY>
<!ATTLIST pt	size		CDATA	#IMPLIED>
<!ATTLIST pt	ptwrite		(yes | no)	#IMPLIED>
<!ATTLIST pt	event-tracing	(yes | no)	#IMPLIED>
