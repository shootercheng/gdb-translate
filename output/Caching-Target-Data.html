10.22 缓存目标数据 ¶

GDB会缓存调试器与目标之间交换的数据。 每个缓存都与子进程的地址空间相关联。 有关子进程和地址空间，请参阅 调试多个子进程、连接和程序。 这种缓存通常可以提高远程调试的性能（请参阅 调试远程程序），因为它通过将内存读写捆绑成大块来减少远程协议的开销。 不幸的是，简单地缓存所有内容会导致不正确的结果， 因为 GDB 不一定了解易失性值、内存映射I/O地址等。此外，在非停止模式下（请参阅 非停止模式），在执行gdb命令 期间 内存可能会被更改。 因此，默认情况下，GDB 只缓存已知在堆栈上的数据13 或代码段中的数据。 内存的其他区域可以显式标记为可缓存；请参阅 内存区域属性。

set remotecache on ¶
set remotecache off

此选项不再起任何作用；它存在是为了与旧脚本兼容。

show remotecache ¶

显示过时的remotecache标志的当前状态。

set stack-cache on ¶
set stack-cache off

启用或禁用堆栈访问的缓存。当设置为 on 时，使用缓存。默认情况下，此选项为 on。

show stack-cache ¶

显示内存访问数据缓存的当前状态。

set code-cache on ¶
set code-cache off

启用或禁用代码段访问的缓存。当设置为 on 时，使用缓存。默认情况下，此选项为 on。这可以提高远程调试中反汇编的性能。

show code-cache ¶

显示代码段访问的目标内存缓存的当前状态。

info dcache [line] ¶

打印有关当前子进程地址空间数据缓存性能的信息。显示的信息包括数据缓存的宽度和深度，以及每个缓存行的编号、地址和被引用的次数。此命令对于调试数据缓存操作很有用。

如果指定了行号，则该行的内容将以十六进制打印。

set dcache size size ¶

设置数据缓存中的最大条目数（即上述数据缓存深度）。

set dcache line-size line-size ¶

设置每个数据缓存条目缓存的字节数（即上述数据缓存宽度）。必须是2的幂。

show dcache size ¶

显示数据缓存条目的最大数量。请参阅 info dcache。

show dcache line-size ¶

显示数据缓存行的默认大小。

maint flush dcache ¶

清除数据缓存的内容（如果有）。此维护命令在调试数据缓存实现时很有用。

脚注

(13)

在非停止模式下，运行中的线程以干扰回溯跟踪的方式修改已停止线程的堆栈的情况相当罕见，并且堆栈读取的缓存显著加快了远程回溯跟踪的速度。
