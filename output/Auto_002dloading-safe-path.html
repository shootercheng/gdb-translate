22.8.3自动加载的安全限制 ¶

由于下级进程的文件可能来自不可信的源（例如由应用程序用户提交），GDB并不总是自动加载任何文件。GDB提供了“set auto-load safe-path”设置，用于列出可信任的目录，这些目录用于加载用户未明确请求的文件。每个目录也可以是一个shell通配符模式。

如果路径设置不正确，你将看到一个警告，并且文件不会被加载： 

$ ./gdb -q ./gdb
Reading symbols from /home/user/gdb/gdb...
warning: File "/home/user/gdb/gdb-gdb.gdb" auto-loading has been
         declined by your `auto-load safe-path' set
         to "$debugdir:$datadir/auto-load".
warning: File "/home/user/gdb/gdb-gdb.py" auto-loading has been
         declined by your `auto-load safe-path' set
         to "$debugdir:$datadir/auto-load".

要指示GDB无论如何都继续使用初始化文件，请像这样调用GDB： 

$ gdb -q -iex "set auto-load safe-path /home/user/gdb" ./gdb

可信任目录列表由以下命令控制：

set auto-load safe-path [directories] ¶

设置可信任的目录列表（及其子目录），用于自动加载和执行脚本。你也可以输入一个特定的可信任文件。每个目录也可以是一个shell通配符模式；通配符不匹配目录分隔符 - 请参阅系统函数fnmatch的FNM_PATHNAME（请参阅《GNU C库参考手册》中的 fnmatch）。 如果你省略directories，“auto-load safe-path”将被重置为在编译 GDB 期间指定的默认值。

目录列表使用路径分隔符（在GNU和Unix系统上是“:”，在MS-Windows和MS-DOS上是“;”）来分隔目录，类似于PATH环境变量。

show auto-load safe-path ¶

显示可信任的目录列表，用于自动加载和执行脚本。

add-auto-load-safe-path ¶

将一个条目（或条目列表）添加到可信任的目录列表中，用于自动加载和执行脚本。多个条目可以用主机平台使用的路径分隔符分隔。

这个变量的默认值是由“--with-auto-load-dir”配置的内容（请参阅 with-auto-load-dir）。$debugdir 和 $datadir 的替换与 set auto-load scripts-directory 相同。 “set auto-load safe-path”的默认值也可以被 GDB 的配置选项“--with-auto-load-safe-path”覆盖。

将这个变量设置为“/”会禁用此安全保护，相应的 GDB 配置选项是“--without-auto-load-safe-path”。 这个变量应该只设置为系统超级用户可写的系统目录。用户可以在他们主目录中的初始化文件中添加他们的源目录（请参阅 主目录初始化文件）。另请参阅当前目录中的弃用初始化文件（请参阅 启动期间当前目录中的初始化文件）。

要强制GDB加载它在前面的示例中拒绝加载的文件，你可以使用以下方法之一：

~/.gdbinit：“add-auto-load-safe-path ~/src/gdb”

将这个可信任的目录（或文件）指定为列表的附加组件。你还必须指定由“show auto-load safe-path”显示的任何现有目录（例如在这个示例中的“/usr:/bin” )。

gdb -iex "set auto-load safe-path /usr:/bin:~/src/gdb" …

像前面的情况一样指定这个目录，但仅用于单个GDB会话。

gdb -iex "set auto-load safe-path /" …

为单个 GDB 会话禁用自动加载安全性。 这假设你在这个 GDB 会话期间调试的所有文件都来自可信任的源。

./configure --without-auto-load-safe-path

在编译 GDB 期间，你可以禁用任何自动加载安全性。 这假设你将使用这个 GDB 调试的所有文件都来自可信任的源。

另一方面你也可以明确禁止自动加载文件，这也会抑制任何此类警告消息：

gdb -iex "set auto-load no" …

你可以在单个GDB会话中使用GDB命令行选项。

~/.gdbinit：“set auto-load no”

为用户全局禁用自动加载（请参阅 主目录初始化文件）。虽然不太可能，但你也可以使用系统初始化文件代替（请参阅 系统范围的配置和设置）。

此设置适用于用户输入的文件名。如果没有匹配项，GDB会在最后尝试将所有文件名解析为其规范形式（通常是解析符号链接），然后再次比较这些条目。GDB在开始比较之前已经自行将大多数文件名规范化，因此建议输入目录的规范形式。
