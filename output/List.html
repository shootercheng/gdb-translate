9.1 打印源行 ¶

要从源文件中打印行，请使用list命令（缩写为l）。默认情况下，打印十行。有几种方法可以指定要打印文件的哪一部分；有关完整列表，请参阅 位置规范。

以下是最常用的list命令形式：

list linenum

打印当前源文件中围绕行号linenum居中的行。

list function

打印围绕函数function开头的行。

list

打印更多行。如果最后打印的行是用list命令打印的，那么这次打印最后打印的行之后的行；但是，如果最后打印的行是作为显示堆栈帧的一部分而单独打印的一行（见 检查堆栈），那么这次打印围绕该行居中的行。如果没有使用list命令且没有单独打印一行，它将打印围绕函数main的行。

list +

与不带参数使用时相同。

list -

打印紧接在最后打印的行之前的行。

list.

打印当前选定帧内执行点周围的行。如果下级程序没有运行，则改为打印围绕主函数开头的行。

默认情况下，GDB使用这些形式的list命令中的任何一个打印十行源行。你可以使用set listsize来更改此设置：

set listsize count ¶
set listsize unlimited

使list命令显示count行源行（除非list参数明确指定了其他数字）。将count设置为unlimited或0表示没有限制。

show listsize ¶

显示list打印的行数。

使用RET重复list命令会丢弃参数，因此它等同于只输入list。这比再次列出相同的行更有用。对于参数为“-”的情况是个例外；该参数在重复时会保留，以便每次重复都在源文件中向上移动。

一般来说，list命令期望你提供零个、一个或两个位置规范。这些位置规范被解释为解析为源代码行；有几种编写它们的方法（见 位置规范），但效果总是解析为要显示的一些源代码行。

以下是list可能的参数的完整描述：

list locspec

打印围绕通过解析locspec得到的所有代码位置处的行或多行居中的行。

list first,last

打印从first到last的行。两个参数都是位置规范。当一个list命令有两个位置规范，并且第二个位置规范的源文件被省略时，这指的是与第一个位置规范相同的源文件。如果first或last在程序中解析为多个源代码行，那么列表命令将显示解析后的源代码行列表，而不会继续列出源代码。

list,last

打印以last结尾的行。

同样，如果last在程序中解析为多个源代码行，那么列表命令将打印解析后的源代码行列表，而不会继续列出源代码。

list first,

打印以first开头的行。

list +

打印紧接在最后打印的行之后的行。

list -

打印紧接在最后打印的行之前的行。

list

如前表所述。
