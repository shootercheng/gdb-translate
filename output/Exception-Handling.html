23.3.2.3异常处理 ¶

在执行python命令时，Python代码中未捕获的Python异常会被转换为对GDB错误报告机制的调用。如果调用python的命令没有处理该错误，GDB将终止它并打印错误消息。具体打印的内容取决于set python print-stack（见 Python命令）。示例： 

(gdb) python print foo
回溯（最近一次调用）：
  文件“<string>”，第1行，在 <module> 中
NameError: 未定义名称 'foo'

在由Python代码调用的GDB命令中发生的GDB错误会被转换为Python异常。Python异常的类型取决于错误。

gdb.error ¶

这是GDB生成的大多数异常的基类。 它派生自RuntimeError，以与早期版本的GDB兼容。

如果在GDB中发生的错误不属于某些更具体的类别，那么生成的异常将具有此类型。

gdb.MemoryError ¶

这是gdb.error的子类，当操作试图访问下级进程中的无效内存时抛出。

KeyboardInterrupt ¶

用户中断（通过C-c或在分页提示符处键入q）会被转换为Python KeyboardInterrupt异常。

在所有情况下，你的异常处理程序将把GDB错误消息作为其值，并将最接近GDB错误发生位置的Python语句处的Python调用栈回溯作为回溯信息。

当通过gdb.Command在Python中实现GDB命令，或通过gdb.Function实现函数时，能够抛出不会导致打印回溯的异常是很有用的。例如，用户可能错误地调用了命令。GDB提供了一个特殊的异常类可用于此目的。

gdb.GdbError ¶

当从命令或函数中抛出时，此异常将导致命令或函数失败，但不会显示Python栈。 GDB本身不会抛出此异常，而是在从用户Python代码中抛出时识别它。示例： 

(gdb) python
>class HelloWorld (gdb.Command):
>  """向整个世界问候。"""
>  def __init__ (self):
>    super (HelloWorld, self).__init__ ("hello-world", gdb.COMMAND_USER)
>  def invoke (self, args, from_tty):
>    argv = gdb.string_to_argv (args)
>    if len (argv) != 0:
>      raise gdb.GdbError ("hello-world不接受参数")
>    print ("Hello, World!")
>HelloWorld ()
>end
(gdb) hello-world 42
hello-world不接受参数
