21.3.8 PowerPC嵌入式 ¶

GDB支持使用DVC（数据值比较）寄存器在硬件中实现以下形式的简单硬件观察点条件： 

(gdb) watch 地址|变量 \
  if  地址|变量 == 常量表达式

当GDB在条件表达式中检测到这种模式时，DVC寄存器将被自动使用，并且创建的观察点使用一个调试寄存器（要么exact - watchpoints选项开启且变量是标量，要么变量长度为一个字节）。此功能在运行于Linux内核版本2.6.34或更高版本的原生GDB中可用。

在PowerPC嵌入式处理器上运行时，GDB会自动使用范围硬件观察点，除非exact - watchpoints选项开启，在这种情况下，观察标量类型变量时会创建仅使用一个调试寄存器的观察点。

你可以使用以下命令之一创建一个伪数组来观察任意内存区域（见 表达式）： 

(gdb) watch *((char *) 地址)@长度
(gdb) watch {char[长度]} 地址

PowerPC嵌入式处理器支持掩码观察点。有关在 设置观察点 中关于 mask 参数的讨论。

PowerPC嵌入式处理器支持硬件加速的范围断点。范围断点会在被调试程序在其设置的范围内的任何地址执行指令时停止其执行。要在GDB中设置范围断点，请使用break - range命令。

GDB提供了以下特定于PowerPC的命令：

break - range 起始位置规范, 结束位置规范 ¶

为给定的由起始位置规范和结束位置规范指定的地址范围设置断点，它们是位置规范。有关所有可能的位置规范形式列表，请参阅 位置规范。GDB会解析起始位置规范和结束位置规范，并使用解析后的代码位置的地址作为要中断的范围的起始和结束地址。当被调试程序在起始和结束地址之间（包括两端）的任何地址执行指令时，该断点将停止其执行。如果起始位置规范或结束位置规范解析为程序中的多个代码位置，则该命令将出错并中止，不会创建断点。

set powerpc soft - float ¶
show powerpc soft - float

强制GDB使用（或不使用）软件浮点调用约定。默认情况下，GDB根据所选架构和提供的可执行文件选择调用约定。

set powerpc vector - abi
show powerpc vector - abi

强制GDB对向量参数和返回值使用指定的调用约定。有效的选项是‘auto’；‘generic’，即使存在向量寄存器也避免使用；‘altivec’，使用AltiVec寄存器；以及‘spe’使用SPE寄存器。默认情况下，GDB根据所选架构和提供的可执行文件选择调用约定。

set powerpc exact - watchpoints
show powerpc exact - watchpoints

允许GDB在观察标量类型变量时仅使用一个调试寄存器，从而假定该变量是通过其第一个字节的地址进行访问的。
