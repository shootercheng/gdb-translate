17.6 补丁程序 ¶

默认情况下，GDB以只读方式打开包含程序可执行代码的文件（或核心文件）。这可防止意外更改机器代码；但它也阻止你有意修补程序的二进制文件。

如果你希望能够修补二进制文件，可以使用set write命令明确指定。例如，你可能想要启用内部调试标志，甚至进行紧急修复。

set write on ¶
set write off

如果你指定‘set write on’，GDB会以读写方式打开可执行文件和核心文件；如果你指定set write off（默认设置），GDB会以只读方式打开它们。

如果你已经加载了一个文件，在更改set write后，必须再次加载它（使用exec-file或core-file命令），以便新设置生效。

show write ¶

显示可执行文件和核心文件是否以读写方式打开。
