5.5.6观察模式 ¶

如果你想在不间断模式的基础上观察程序行为，而不让GDB有任何干扰的机会，你可以设置变量来禁用调试器所有修改状态的尝试，无论是通过写入内存、插入断点等。这些操作在底层进行，拦截所有命令的操作。

当所有这些都设置为“off”时，GDB就处于所谓的观察模式。为了方便起见，可以设置变量“observer”来禁用这些设置，并启用不间断模式。

请注意，GDB不会阻止你进行这些设置的无意义组合。例如，如果启用了“may-insert-breakpoints”但禁用了“may-write-memory”，那么通过将陷阱指令写入代码流来工作的断点仍然无法设置。

set observer on ¶
set observer off

设置为“on”时，这将禁用下面所有的权限变量（除了“insert-fast-tracepoints ”），并启用不间断调试。将其设置为“off”会切换回正常调试，不过仍保持不间断模式。

show observer

显示观察模式是开启还是关闭。

set may-write-registers on ¶
set may-write-registers off

这控制着GDB是否会尝试改变寄存器的值，比如使用“print”中的赋值表达式或“jump”命令。它默认设置为“on”。

show may-write-registers

显示当前写入寄存器的权限。

set may-write-memory on ¶
set may-write-memory off

这控制着GDB是否会尝试改变内存的内容, 比如使用“print”中的赋值表达式。它默认设置为“on”。

show may-write-memory

显示当前写入内存的权限。

set may-insert-breakpoints on ¶
set may-insert-breakpoints off

这控制着GDB是否会尝试插入断点。这会影响所有断点，包括GDB定义的内部断点。它默认设置为“on”。

show may-insert-breakpoints

显示当前插入断点的权限。

set may-insert-tracepoints on ¶
set may-insert-tracepoints off

这控制着GDB在跟踪实验开始时是否会尝试插入（常规）跟踪点。它只影响非快速跟踪点，快速跟踪点由“may-insert-fast-tracepoints”控制。它默认设置为“on ”。

show may-insert-tracepoints

显示当前插入跟踪点的权限。

set may-insert-fast-tracepoints on ¶
set may-insert-fast-tracepoints off

这控制着GDB在跟踪实验开始时是否会尝试插入快速跟踪点。它只影响快速跟踪点，常规（非快速）跟踪点由“may-insert-tracepoints”控制。它默认设置为“on”。

show may-insert-fast-tracepoints

显示当前插入快速跟踪点的权限。

set may-interrupt on ¶
set may-interrupt off

这控制着GDB是否会尝试中断或停止程序执行。当这个变量为“off”时，“interrupt”命令将没有效果，Ctrl-c也一样。它默认设置为“on”。

show may-interrupt

显示当前中断或停止程序的权限。
