15.4.9.4 Modula-2类型 ¶

目前，GDB可以用Modula-2语法打印以下数据类型：数组类型、记录类型、集合类型、指针类型、过程类型、枚举类型、子界类型和基类型。你也可以打印使用这些类型声明的变量的内容。 本节给出了一些简单的源代码示例以及GDB会话示例。

第一个示例包含以下代码段： 

VAR
   s: SET OF CHAR ;
   r: [20..40] ;

你可以请求GDB询问r和s的类型和值。 

(gdb) print s
{'A'..'C', 'Z'}
(gdb) ptype s
SET OF CHAR
(gdb) print r
21 
(gdb) ptype r
[20..40]

同样，如果你的源代码将s声明为： 

VAR
   s: SET ['A'..'Z'] ;

那么你可以通过以下方式查询s的类型： 

(gdb) ptype s
type = SET ['A'..'Z']

请注意，目前你不能使用调试器交互式地操作集合表达式。

以下示例展示了如何在Modula-2中声明一个数组，以及如何与GDB交互以打印其类型和内容： 

VAR
   s: ARRAY [-10..10] OF CHAR ;

(gdb) ptype s
ARRAY [-10..10] OF CHAR

请注意，数组处理尚未完成，虽然类型打印正确，但表达式处理仍然假设所有数组的下界为零，而不是像上面示例中的-10。

这里还有一些与Modula-2类型相关的示例： 

TYPE
   colour = (blue, red, yellow, green) ;
   t = [blue..yellow] ;
VAR
   s: t ;
BEGIN
   s := blue ;

GDB交互展示了如何查询变量的数据类型和值。 

(gdb) print s
$1 = blue
(gdb) ptype t
type = [blue..yellow]

在这个示例中，声明了一个Modula-2数组并显示其内容。请注意，内容的写法与它们在C中的对应写法相同。 

VAR
   s: ARRAY [1..5] OF CARDINAL ;
BEGIN
   s[1] := 1 ;

(gdb) print s
$1 = {1, 0, 0, 0, 0}
(gdb) ptype s
type = ARRAY [1..5] OF CARDINAL

GDB的Modula-2语言接口也理解指针类型，如这个示例所示： 

VAR
   s: POINTER TO ARRAY [1..5] OF CARDINAL ;
BEGIN
   NEW(s) ;
   s^[1] := 1 ;

你可以请求GDB描述s的类型。 

(gdb) ptype s
type = POINTER TO ARRAY [1..5] OF CARDINAL

GDB处理复合类型，正如我们在这个示例中看到的。这里我们组合了数组类型、记录类型、指针类型和子界类型： 

TYPE
   foo = RECORD
            f1: CARDINAL ;
            f2: CHAR ;
            f3: myarray ;
         END ;

   myarray = ARRAY myrange OF CARDINAL ;
   myrange = [-2..2] ;
VAR
   s: POINTER TO ARRAY myrange OF foo ;

你可以像下面这样请求GDB描述s的类型。 

(gdb) ptype s
type = POINTER TO ARRAY [-2..2] OF foo = RECORD
    f1 : CARDINAL;
    f2 : CHAR;
    f3 : ARRAY [-2..2] OF CARDINAL;
END
