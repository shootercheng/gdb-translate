附录I跟踪文件格式 ¶

跟踪文件分为三个部分：一个头部、一个文本描述部分和一个带有二进制数据的跟踪帧部分。

头部的形式为 \x7fTRACE0\n。第一个字节是 0x7f，用于表示文件包含二进制数据，而 0 是一个版本号，未来可能会有不同的值。

描述部分由多行ASCII文本组成，用换行符（0xa）分隔。这些行可能包括各种可选的描述性或上下文设置信息，例如跟踪点定义或寄存器集大小。GDB 将忽略它不识别的任何行。空行标记此部分的结束。

R 大小

指定以字节为单位的寄存器块的大小。这等于远程协议中 g 数据包有效负载的大小。大小 是一个ASCII十进制数。在单个跟踪文件中应该只有一行这样的内容。

状态 状态

跟踪状态。状态 与 qTStatus 远程数据包回复的格式相同。在单个跟踪文件中应该只有一行这样的内容。

tp 有效负载

跟踪点定义。有效负载 与 qTfP/qTsP 远程数据包回复有效负载格式相同。单个跟踪点可能需要多行定义，对应多个回复数据包。

tsv 有效负载

跟踪状态变量定义。有效负载 与 qTfV/qTsV 远程数据包回复有效负载格式相同。单个变量可能需要多行定义，对应多个回复数据包。

tdesc 有效负载

XML格式的目标描述。有效负载 是XML文件的一行。所有这些行应连接在一起以获取原始XML文件。此文件与 qXfer features 有效负载格式相同，并且对应于主 target.xml 文件。不允许包含。

跟踪帧部分由多个连续的帧组成。每个帧以一个两字节的跟踪点编号开头，后跟一个四字节大小，表示帧中的数据量。帧中的数据由多个块组成，每个块由一个字符引入，指示其类型（至少有寄存器、内存和跟踪状态变量）。此部分中的数据是原始二进制数据，不是十六进制或其他编码；其字节序与目标的字节序匹配。

R 字节数

寄存器块。字节的数量和顺序与远程协议中 g 数据包的字节数和顺序匹配。请注意，这些是实际字节，按目标顺序排列，而不是十六进制编码。

M 地址 长度 字节数...

内存块。这是一个连续的内存块，位于8字节地址 地址 处，长度为2字节 长度，后跟 长度 个字节。

V 编号 值

跟踪状态变量块。这记录了编号为 编号 的跟踪状态变量的8字节有符号值 值。

跟踪文件格式的未来增强功能可能包括其他类型的块。
