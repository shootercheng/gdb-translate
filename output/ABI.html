22.7配置当前ABI ¶

GDB可以自动确定应用程序的ABI（应用程序二进制接口）。但是，有时你需要覆盖它的结论。使用这些命令来管理GDB对当前ABI的视图。

一个GDB配置可以通过远程调试或本机仿真来调试针对多个操作系统目标的二进制文件。GDB将自动检测正在使用的OS ABI（操作系统ABI），但你可以使用set osabi命令覆盖其结论。一个有用的例子是调试使用替代C库（例如用于GNU/Linux的UCLIBC）的二进制文件，该库没有你的平台的标准C库提供的相同标识标记。

当GDB调试AArch64架构时，它提供了一个“Newlib”OS ABI。这对于调试使用NEWLIB C库的二进制文件时处理setjmp和longjmp很有用。可以通过set osabi Newlib选择“Newlib”OS ABI。

show osabi

显示当前正在使用的OS ABI。

set osabi

不带参数时，显示已注册的可用OS ABI列表。

set osabi abi

将当前OS ABI设置为abi。

一般来说，类型为float的参数传递给函数的方式取决于函数是否有原型。对于有原型（即ANSI/ISO风格）的函数，float参数将按架构对float的约定原样传递。对于无原型（即K&R风格）的函数，则先将float参数提升为double类型，然后再传递。

不幸的是，某些形式的调试信息不能可靠地指示函数是否有原型。如果GDB调用未标记为有原型的函数，它会参考set coerce-float-to-double。
