23.3.4.2 gdb.types ¶

本模块提供了一组用于处理 gdb.Type 对象的实用工具。

get_basic_type (type)

返回去除了const和volatile限定符的 type，并将typedef和C++引用转换为底层类型。

C++示例： 

typedef const int const_int;
const_int foo (3);
const_int& foo_ref (foo);
int main () { return 0; }

然后在gdb中： 

(gdb) start
(gdb) python import gdb.types
(gdb) python foo_ref = gdb.parse_and_eval("foo_ref")
(gdb) python print gdb.types.get_basic_type(foo_ref.type)
int
has_field (type, field)

如果 type（假定为具有字段的类型，例如结构体或联合体）具有字段 field 返回 True。

make_enum_dict (enum_type)

返回从 enum_type 生成的Python 字典 类型。

deep_items (type)

返回一个类似于标准 gdb.Type.iteritems 方法的Python迭代器，不同之处在于 deep_items 返回的迭代器将递归遍历匿名结构体或联合体字段。例如： 

struct A
{
    int a;
    union {
        int b0;
        int b1;
    };
};

然后在GDB中： 

(gdb) python import gdb.types
(gdb) python struct_a = gdb.lookup_type("struct A")
(gdb_python print struct_a.keys ()
{['a', '']}](gdb) python print [k for k,v in gdb.types.deep_items(struct_a)]
{['a', 'b0', 'b1']}
get_type_recognizers ()

返回当前上下文中启用的类型识别器列表。这在GDB的类型打印过程中被调用（见 类型打印API）。

apply_type_recognizers (recognizers, type_obj)

将类型识别器 recognizers 应用于类型对象 type_obj。如果任何识别器返回一个字符串，则返回该字符串。否则，返回 None。这在GDB的类型打印过程中被调用（见 类型打印API）。

register_type_printer (locus, printer)

这是一个用于注册类型打印机 printer 的便利函数。打印机必须实现类型打印机协议。locus 参数可以是一个 gdb.Objfile，在这种情况下，打印机将注册到该目标文件；一个 gdb.Progspace，在这种情况下，打印机将注册到该程序空间；或者 None，在这种情况下，打印机将全局注册。
